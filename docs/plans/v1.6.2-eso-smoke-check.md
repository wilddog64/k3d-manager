# v1.6.2 — ESO health checks in smoke test

**Branch:** `k3d-manager-v1.6.2`
**Files:** `bin/k3dm-webhook`

---

## Problem

`_smoke_test_services()` checks HTTP endpoints but does not verify ESO (External Secrets
Operator) health. ESO sync failure is the most common root cause of "pod Running but app
broken" scenarios: the pod starts, the ESO-backed secret is missing, the app crashes.
Adding ESO checks to the smoke report surfaces this before Gemini has to guess.

Two checks are needed:
1. `ClusterSecretStore/vault-backend` — is Vault reachable and the store Ready?
2. ExternalSecrets across all namespaces — are any not in `SecretSynced` state?

The on-demand health endpoint (spec `v1.6.2-acg-status-health-check.md`) needs a 1-retry

### Change 2 — `bin/k3dm-webhook`: add ESO checks at end of `_smoke_test_services()`

Insert before the final `return results` line (currently at line 806).

**Exact old block (line 803–806):**

```python
    except Exception as exc:
        results.append(("Product images", False, str(exc)))

    return results
```

**Exact new block:**

```python
    except Exception as exc:
        results.append(("Product images", False, str(exc)))

    # ESO: ClusterSecretStore vault-backend
    try:
        eso_store = subprocess.run(
            ["kubectl", "get", "clustersecretstore", "vault-backend",
             "--context", "ubuntu-k3s", "--no-headers",
             "-o", "jsonpath={.status.conditions[0].type}={.status.conditions[0].status}"],
            capture_output=True, text=True, timeout=8,
        )
        if eso_store.returncode == 0 and eso_store.stdout.strip():
            val = eso_store.stdout.strip()
            ok = val == "Ready=True"
            results.append(("ESO ClusterSecretStore", ok, val))
        else:
            results.append(("ESO ClusterSecretStore", False, "kubectl returned no output"))
    except Exception as exc:
        results.append(("ESO ClusterSecretStore", False, str(exc)))

    # ESO: ExternalSecrets not in Ready=True state
    try:
        eso_secrets = subprocess.run(
            ["kubectl", "get", "externalsecrets", "-A",
             "--context", "ubuntu-k3s", "--no-headers",
             "-o", "custom-columns=NS:.metadata.namespace,NAME:.metadata.name,"
                   "READY:.status.conditions[0].status"],
            capture_output=True, text=True, timeout=10,
        )
        if eso_secrets.returncode == 0 and eso_secrets.stdout.strip():
            es_lines = [l for l in eso_secrets.stdout.strip().splitlines() if l]
            not_ready = [l for l in es_lines if not l.strip().endswith("True")]
            total = len(es_lines)
            if not_ready:
                sample = ", ".join(l.split()[1] for l in not_ready[:3] if len(l.split()) >= 2)
                results.append(("ESO ExternalSecrets", False,
                                 f"{len(not_ready)}/{total} not synced: {sample}"))
            else:
                results.append(("ESO ExternalSecrets", True, f"{total}/{total} synced"))
        else:
            results.append(("ESO ExternalSecrets", False, "kubectl returned no output"))
    except Exception as exc:
        results.append(("ESO ExternalSecrets", False, str(exc)))

    return results
```
