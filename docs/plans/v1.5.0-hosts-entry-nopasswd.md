# Spec: Passwordless /etc/hosts update for pipeline compatibility

**Branch:** `k3d-manager-v1.5.0`
**Milestone:** v1.5.0

## Problem

`bin/acg-up` has two constructs that block non-interactive (pipeline) execution:

1. `sudo -v` prewarm (line 50) — requires a TTY to prompt for a password.
2. `/etc/hosts` update via `_run_command --interactive-sudo -- sh -c "..."` — `sudo sh -c`
   is intentionally absent from the NOPASSWD rules (shell wildcard is a root-shell giveaway).

Both will hang or fail in CI where no terminal is available.

## Fix

Extract the `/etc/hosts` update into a dedicated helper script installed at a fixed path
(`/usr/local/bin/k3d-manager-update-hosts`). Add a scoped NOPASSWD rule for that exact
path. Remove the `sudo -v` prewarm — it is no longer needed once all macOS host operations
are covered by NOPASSWD.

## Change 1 — new file `bin/update-hosts-entry.sh`

Create this file (executable, `chmod 755`):

```bash
#!/usr/bin/env bash
# bin/update-hosts-entry.sh
# Update or add a single /etc/hosts entry. Must be run as root.
# Usage: update-hosts-entry.sh <hostname> <ip>
# Installed to: /usr/local/bin/k3d-manager-update-hosts by make install-sudoers

set -euo pipefail

_h="${1:?hostname required}"
_ip="${2:?ip required}"

_desired="${_ip} ${_h}"
_tmp="$(mktemp)"
grep -vE "^[0-9.]+[[:space:]]+${_h//./\\.}$" /etc/hosts > "$_tmp"
printf '%s\n' "$_desired" >> "$_tmp"
cat "$_tmp" > /etc/hosts
rm -f "$_tmp"
```

## Change 2 — `bin/install-sudoers.sh`

### 2a — Add NOPASSWD rule to `_SUDOERS_CONTENT`

Add one line after the existing loopback/launchctl rules (after the `rm -f` line, before
the blank line that precedes the binary-install block):

```bash
# OLD (excerpt from _SUDOERS_CONTENT, line ~36)
%admin  ALL=(root) NOPASSWD: /bin/rm -f /Library/LaunchDaemons/com.k3d-manager.*.plist *

# Binary install to /usr/local/bin (k3s, kubectl, istioctl, vcluster, etc.)
```

```bash
# NEW
%admin  ALL=(root) NOPASSWD: /bin/rm -f /Library/LaunchDaemons/com.k3d-manager.*.plist *

# /etc/hosts management — scoped helper only (no bare sh -c wildcard)
%admin  ALL=(root) NOPASSWD: /usr/local/bin/k3d-manager-update-hosts *

# Binary install to /usr/local/bin (k3s, kubectl, istioctl, vcluster, etc.)
```

### 2b — Install helper in `install` mode

After the `visudo -c` check succeeds and before the sudoers file is written, install the
helper script. Add these lines immediately before the `_run_command --interactive-sudo --
install -m 0440 ...` call:

```bash
# OLD (lines ~98–105)
    if ! visudo -c -f "$_tmpfile"; then
      rm -f "$_tmpfile"
      echo "[install-sudoers] ERROR: sudoers syntax check failed — aborting" >&2
      exit 1
    fi
    _run_command --interactive-sudo -- install -m 0440 "$_tmpfile" "$_SUDOERS_FILE"
    rm -f "$_tmpfile"
    echo "[install-sudoers] Installed $_SUDOERS_FILE"
```

```bash
# NEW
    if ! visudo -c -f "$_tmpfile"; then
      rm -f "$_tmpfile"
      echo "[install-sudoers] ERROR: sudoers syntax check failed — aborting" >&2
      exit 1
    fi
    _run_command --interactive-sudo -- install -m 0755 \
      "${REPO_ROOT}/bin/update-hosts-entry.sh" /usr/local/bin/k3d-manager-update-hosts
    _run_command --interactive-sudo -- install -m 0440 "$_tmpfile" "$_SUDOERS_FILE"
    rm -f "$_tmpfile"
    echo "[install-sudoers] Installed $_SUDOERS_FILE"
    echo "[install-sudoers] Installed /usr/local/bin/k3d-manager-update-hosts"
```
