# Refactor: v1.4.10 - Decompose `bin/acg-up` into reusable bootstrap helpers

**Branch:** `k3d-manager-v1.4.10`

## Context

`bin/acg-up` has grown into a catch-all provision script. It now handles ACG login, cluster bring-up, host-side launchd plumbing, shopping-cart bootstrap, Vault seeding, ESO wiring, ArgoCD registration, Keycloak realm reconciliation, and final product seeding.

Several of those responsibilities already belong to reusable `k3d-manager` helpers or should become them. The current shape makes the script hard to reason about, hard to test, and easy to accidentally break when a change in one area touches unrelated startup behavior.

## Goal

Reduce `bin/acg-up` to a thin orchestration entrypoint while preserving all current behavior, log messages, and ordering.

The refactor should:

- keep `bin/acg-up` as the top-level conductor
- move reusable bootstrap logic into existing or new `k3d-manager` plugin helpers
- keep macOS-specific UX plumbing local where it is genuinely host-specific
- avoid changing any cluster behavior, URLs, or user-visible flow unless the spec explicitly says so

## Problem Statement

Today `bin/acg-up` mixes four different concerns:

1. ACG sandbox/login and TTL handling
2. Cluster bootstrap and application registration
3. Shopping-cart runtime bootstrap and secret seeding
4. macOS launchd, hosts file, and tunnel management

That coupling creates a maintenance problem:

- the file is long enough that unrelated changes become risky
- reusable logic is duplicated or buried inside the orchestration flow
- helper boundaries are unclear, so it is hard to test pieces independently

## Proposed Boundary

### Keep in `bin/acg-up`

- ACG sandbox prompt and credential acquisition
- High-level step ordering and status logging
- macOS host UX wiring that is inherently entrypoint-specific:
  - `/etc/hosts` edits
  - launchd agents
  - local browser listeners
  - Cloudflare tunnel setup

### Move into `k3d-manager` helpers

- GHCR secret selection and creation
- `shopping-cart-apps` ServiceAccount imagePullSecrets patching
- Vault bridge setup
- Vault KV seeding for shopping-cart sandbox secrets
- ESO `ClusterSecretStore` setup
- ArgoCD application-cluster registration
- shopping-cart data-layer deployment
- Keycloak / ArgoCD bootstrap reconciliation
- product-catalog post-sync reconciliation and seed logic

### Keep in existing plugin homes where possible

- `scripts/plugins/argocd.sh`
  - ArgoCD port-forward helpers
  - browser HTTPS readiness helpers
  - `register_app_cluster`
- `scripts/plugins/keycloak.sh`
  - realm/client reconciliation helpers
- `scripts/plugins/shopping_cart.sh`
  - shopping-cart data-layer deployment and other shopping-cart-specific helpers

## Refactor Strategy

### Phase 1 - Carve out reusable bootstrap helpers

Extract the inline bootstrap blocks from `bin/acg-up` into named helper functions. The first extraction should focus on the shopping-cart runtime bootstrap path:

- GHCR secret creation and ServiceAccount patching
- Vault bridge creation
- Vault KV seeding
- ClusterSecretStore creation
- shopping-cart data-layer deployment
- Keycloak realm and client reconciliation
- product-catalog secret reconciliation and seeding

### Phase 2 - Preserve orchestration in `bin/acg-up`

Replace each inline block with a single helper call, but keep the existing step labels and step order intact.

The user-visible flow should still read like the current script:

- credentials
- cluster
- tunnel / host listeners
- bootstrap
- verification
- final URLs

### Phase 3 - Tighten the helper API

Where a helper is reusable across other commands, promote it to a stable plugin function instead of leaving it as an anonymous inline block in `bin/acg-up`.

Examples:

- cluster registration should stay under `scripts/plugins/argocd.sh`
- shopping-cart data deployment should stay under `scripts/plugins/shopping_cart.sh`
- Keycloak reconciliation should stay under `scripts/plugins/keycloak.sh`

## Files In Scope

| File | Planned Change |
|------|----------------|
| `bin/acg-up` | Replace inline bootstrap blocks with helper calls; keep orchestration and host UX |
| `scripts/plugins/argocd.sh` | Add or expose reusable ArgoCD bootstrap helpers if needed |
| `scripts/plugins/shopping_cart.sh` | Add shopping-cart bootstrap helpers and keep data-layer deployment logic here |
| `scripts/plugins/keycloak.sh` | Reuse or expose Keycloak reconciliation helpers as needed |
| `scripts/tests/bin/acg_up.bats` | Update orchestration assertions to match the refactor |
| `scripts/tests/plugins/shopping_cart.bats` | Add coverage for any new shopping-cart helper entrypoints |

## Expected Result

After the refactor:

- `bin/acg-up` should read like an ordered checklist, not a monolithic implementation
- shared bootstrap logic should live in the plugin layer instead of being duplicated inline
- the shopping-cart bootstrap flow should remain functionally identical
- tests should prove the same startup behavior still exists through smaller helper seams

## Verification

Use the existing shell and BATS checks that cover the touched areas:

```bash
shellcheck -S warning bin/acg-up
bats scripts/tests/bin/acg_up.bats
bats scripts/tests/plugins/shopping_cart.bats
bats scripts/tests/plugins/argocd.bats
bats scripts/tests/plugins/keycloak.bats
```

For any helper extraction that changes public function surfaces, add or update focused BATS coverage in the same commit.

## Out of Scope

- No new runtime features
- No changes to shopping-cart repo manifests
- No changes to public URLs or hostnames
- No removal of host-side launchd or tunnel behavior unless the helper extraction preserves identical behavior
- No rewrite of the ACG credential extraction flow

## Definition of Done

- `bin/acg-up` is significantly slimmer and acts primarily as orchestration
- shopping-cart bootstrap logic has named helper boundaries in the plugin layer
- existing startup behavior remains unchanged
- touched shell files pass `shellcheck -S warning`
- the relevant BATS suites pass
- the work is committed and pushed on `k3d-manager-v1.4.10`
