## Problem

Calling `_ai_agent_review --prompt "hello"` without `K3DM_ENABLE_AI=1` fails immediately:

```
ERROR: Copilot CLI is disabled. Set K3DM_ENABLE_AI=1 to enable AI tooling.
```

`_ai_agent_review` is a generic lib-foundation dispatch wrapper designed to be used by any project. Requiring `K3DM_ENABLE_AI=1` — a k3d-manager-specific env var — defeats its purpose as a reusable library function.

**Root cause:** `_copilot_review` in lib-foundation has a hardcoded `K3DM_ENABLE_AI` gate at lines 1656–1658. The gate belongs in the public callers (`copilot_triage_pod`, `copilot_draft_spec` in k3d-manager), which already have it. The backend implementation should not re-gate on a consumer-specific env var.

---

## Reproduction

```bash
# In k3d-manager root, with system.sh sourced:
source scripts/lib/foundation/scripts/lib/system.sh
_ai_agent_review --prompt "say hello"
# Expected: invokes copilot CLI (or fails with auth/CLI-not-found error)
# Actual:   ERROR: Copilot CLI is disabled. Set K3DM_ENABLE_AI=1 to enable AI tooling.
```

## Fix

### Change 1 — `scripts/lib/system.sh`: remove `K3DM_ENABLE_AI` gate from `_copilot_review`

**File:** `/Users/cliang/src/gitrepo/personal/lib-foundation/scripts/lib/system.sh`

**Exact old block (lines 1655–1661):**

```bash
function _copilot_review() {
   if [[ "${K3DM_ENABLE_AI:-0}" != "1" ]]; then
      _err "Copilot CLI is disabled. Set K3DM_ENABLE_AI=1 to enable AI tooling."
   fi

   _safe_path
   _ensure_copilot_cli
```

**Exact new block:**

```bash
function _copilot_review() {
   _safe_path
   _ensure_copilot_cli
```

## Why `_copilot_auth_check` is NOT changed

`_copilot_auth_check` at line 1592 also references `K3DM_ENABLE_AI`, but as a skip-guard (returns 0 early if AI is disabled, meaning auth validation is skipped rather than blocked). This is harmless for generic callers — if `K3DM_ENABLE_AI` is unset, auth check becomes a no-op and the actual CLI invocation will surface a real auth error if the user is not logged in. Changing `_copilot_auth_check` is out of scope for this fix.
