shipped across v0.9.12 and v0.9.14 (see CHANGE.md). This spec closes out the one remaining item

## Implementation

### `scripts/etc/ldap/ldap-password-rotator.sh`

Replace the `vault kv put` block inside the heredoc that runs in the Vault pod.

**Old (`vault kv put` with command-line args):**
```bash
export VAULT_TOKEN
vault kv put "$TARGET_VAULT_PATH" \
  username="$username" \
  password="$password" \
  dn="$dn" \
  rotated_at="$rotated_at" >/dev/null 2>&1
```

**New (`vault kv put @-` reads JSON from stdin — no secrets in ps aux):**
```bash
export VAULT_TOKEN
printf '{"username":"%s","password":"%s","dn":"%s","rotated_at":"%s"}' \
  "$username" "$password" "$dn" "$rotated_at" \
  | vault kv put "$TARGET_VAULT_PATH" @- >/dev/null 2>&1
```

**Why:** `vault kv put path key=value` expands to a process with visible arguments.
`vault kv put path @-` reads a JSON object from stdin — credentials never appear in
`ps aux` or audit logs inside the Vault pod.

**Constraint:** The existing escaping context is inside a bash heredoc passed to
`kubectl exec -i … bash -c "…"`. The `\`-escaped variables (`\$username`, `\$password`,
`\$dn`, `\$rotated_at`) must remain escaped so they are evaluated inside the pod, not
on the host. Do not change the escaping of the surrounding heredoc.

**Exact edit — old inside heredoc (with escaping):**
```
export VAULT_TOKEN
vault kv put \"\$TARGET_VAULT_PATH\" \\
  username=\"\$username\" \\
  password=\"\$password\" \\
  dn=\"\$dn\" \\
  rotated_at=\"\$rotated_at\" >/dev/null 2>&1
```

**Exact edit — new inside heredoc (with escaping):**
```
export VAULT_TOKEN
printf '{\"username\":\"%s\",\"password\":\"%s\",\"dn\":\"%s\",\"rotated_at\":\"%s\"}' \\
  \"\$username\" \"\$password\" \"\$dn\" \"\$rotated_at\" \\
  | vault kv put \"\$TARGET_VAULT_PATH\" @- >/dev/null 2>&1
```
