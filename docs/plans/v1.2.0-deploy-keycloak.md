# Deploy Keycloak for shopping-cart frontend login

**Date:** 2026-04-28
**Repo:** `k3d-manager`
**Branch:** `k3d-manager-v1.2.0`

---

## Background

The shopping-cart frontend uses Keycloak OIDC for authentication. The JS bundle has
`http://localhost:8080` baked in as the Keycloak URL (via `VITE_KEYCLOAK_URL` default
in `src/config/auth.ts`). Keycloak is not yet deployed to the sandbox, so Login fails.

The Keycloak manifests already exist in `shopping-cart-infra/identity/keycloak/`:
- PostgreSQL StatefulSet for Keycloak's database
- Keycloak Deployment (image: `quay.io/keycloak/keycloak:24.0`)
- ConfigMap with `KC_DB`, `KC_REALM=shopping-cart`, LDAP config
- Secrets with `CHANGE_ME` placeholders

ArgoCD currently forwards to `localhost:8080`. Keycloak needs `localhost:8080` (frontend
hardcoded). Fix: move ArgoCD to `localhost:9090`, Keycloak to `localhost:8080`.

## Fix 1 — Add `services/shopping-cart-identity/` kustomization

Create an ArgoCD-managed Keycloak deployment. The ApplicationSet git generator already
discovers `services/*` automatically — no ApplicationSet change needed.

### New file: `services/shopping-cart-identity/realm.json`

This file is mounted into Keycloak at startup to auto-create the `shopping-cart` realm,
the `frontend` OIDC client, and a test user. Without this, login cannot work.

```json
{
  "id": "shopping-cart",
  "realm": "shopping-cart",
  "displayName": "Shopping Cart",
  "enabled": true,
  "sslRequired": "none",
  "registrationAllowed": false,
  "clients": [
    {
      "clientId": "frontend",
      "name": "Shopping Cart Frontend",
      "enabled": true,
      "publicClient": true,
      "standardFlowEnabled": true,
      "directAccessGrantsEnabled": true,
      "redirectUris": [
        "http://localhost:8081/*"
      ],
      "webOrigins": [
        "http://localhost:8081"
      ],
      "protocol": "openid-connect"
    }
  ],
  "users": [
    {
      "username": "testuser",
      "email": "testuser@shopping-cart.local",
      "firstName": "Test",
      "lastName": "User",
      "enabled": true,
      "credentials": [
        {
          "type": "password",
          "value": "testpassword",
          "temporary": false
        }
      ]
    }
  ]
}
```

### New file: `services/shopping-cart-identity/kustomization.yaml`

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - https://github.com/wilddog64/shopping-cart-infra//identity/keycloak?ref=main
namespace: identity
configMapGenerator:
  - name: keycloak-realm-import
    files:
      - realm.json
generatorOptions:
  disableNameSuffixHash: true
patches:
  - patch: |-
      apiVersion: apps/v1
      kind: Deployment
      metadata:
        name: keycloak
        namespace: identity
      spec:
        template:
          spec:
            containers:
            - name: keycloak
              args:
                - start-dev
                - --import-realm
              volumeMounts:
              - name: realm-import
                mountPath: /opt/keycloak/data/import
            volumes:
            - name: realm-import
              configMap:
                name: keycloak-realm-import
    target:
      kind: Deployment
      name: keycloak
```

**Why `start-dev` instead of `start --optimized`:** The upstream manifest uses
`start --optimized` which requires the image to be pre-built with `kc.sh build`.
The standard `quay.io/keycloak/keycloak:24.0` image is not pre-built — it will
crash. `start-dev` works with the standard image for dev sandbox use.

**Why `--import-realm`:** Tells Keycloak to import all JSON files found in
`/opt/keycloak/data/import/` on first startup. Creates the `shopping-cart` realm,
`frontend` client, and `testuser` automatically.

## Fix 3 — Move ArgoCD port-forward from 9090, add Keycloak on 8080

The frontend has `localhost:8080` baked in for Keycloak. ArgoCD currently owns 8080.
Fix: ArgoCD → 9090, Keycloak → 8080.

Then add Keycloak port-forward after the existing frontend port-forward block
(after the `_info "[sync-apps] Frontend available..."` line):

```bash
_info "[sync-apps] Starting Keycloak port-forward (localhost:8080 → identity/keycloak:80)..."
_keycloak_pf_pid_file="${HOME}/.local/share/k3d-manager/keycloak-pf.pid"
_keycloak_pf_log="${HOME}/.local/share/k3d-manager/keycloak-pf.log"
mkdir -p "$(dirname "${_keycloak_pf_pid_file}")"

if [[ -f "${_keycloak_pf_pid_file}" ]]; then
  kill "$(cat "${_keycloak_pf_pid_file}")" 2>/dev/null || true
  rm -f "${_keycloak_pf_pid_file}"
fi

if kubectl get svc keycloak -n identity --context "${APP_CONTEXT}" >/dev/null 2>&1; then
  kubectl port-forward svc/keycloak -n identity \
    --context "${APP_CONTEXT}" 8080:80 \
    >"${_keycloak_pf_log}" 2>&1 &
  _keycloak_pf_pid=$!
  disown "${_keycloak_pf_pid}"
  echo "${_keycloak_pf_pid}" > "${_keycloak_pf_pid_file}"
  _info "[sync-apps] Keycloak available at http://localhost:8080 (admin: admin / CHANGE_ME)"
  _info "[sync-apps] ArgoCD available at http://localhost:9090"
else
  _info "[sync-apps] Keycloak not yet deployed — skipping port-forward"
fi
```

### Fix 3c — `bin/acg-down`: add Keycloak PID cleanup
