# Bugfix: v1.6.0 — Wait for data-layer ArgoCD sync before StatefulSet loop

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/acg-up`, `scripts/plugins/shopping_cart.sh`

---

## Problem

`deploy_shopping_cart_data` waits for PostgreSQL StatefulSets to appear in
`shopping-cart-data`, but the wait is passive — it polls every 10 s for up to 300 s with
no confirmation that ArgoCD has even started syncing the `data-layer` Application.
If ArgoCD is slow to pick up the new cluster registration (common right after
`argocd-application-controller` restarts), the 300 s deadline expires before a single
StatefulSet is created, producing the misleading error:

```
ERROR: [shopping_cart] Timed out waiting for StatefulSet postgresql-orders to be created by ArgoCD
```

**Root cause:** the `data-layer` Application is applied immediately before
`deploy_shopping_cart_data` (fix from `deb4c3b2`), but there is no check that ArgoCD has
actually acknowledged and synced the Application before the StatefulSet wait begins.

**Goal:** gate the StatefulSet wait on `data-layer` reaching `Synced` status in ArgoCD,
so the function fails fast with a clear error if ArgoCD never syncs, and the StatefulSet
wait only runs after resources are already created.

---

## Fix

### Change 1 — `bin/acg-up`: add sync-wait between apply and `deploy_shopping_cart_data`

**Exact old block (lines 562–566):**

```bash
_info "[acg-up] Step 10b/14 — Deploying shopping-cart data layer (PostgreSQL, Redis, RabbitMQ)..."
_info "[acg-up] Applying data-layer ArgoCD Application so sync starts before wait loop..."
_run_command -- kubectl apply --context k3d-k3d-cluster \
  -f "${REPO_ROOT}/../shopping-carts/shopping-cart-infra/argocd/applications/data-layer.yaml"
deploy_shopping_cart_data
```

**Exact new block:**

```bash
_info "[acg-up] Step 10b/14 — Deploying shopping-cart data layer (PostgreSQL, Redis, RabbitMQ)..."
_info "[acg-up] Applying data-layer ArgoCD Application so sync starts before wait loop..."
_run_command -- kubectl apply --context k3d-k3d-cluster \
  -f "${REPO_ROOT}/../shopping-carts/shopping-cart-infra/argocd/applications/data-layer.yaml"
_info "[acg-up] Waiting for ArgoCD to sync data-layer Application (max 300s)..."
_dl_sync_deadline=$(( $(date +%s) + 300 ))
until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
    -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
  if [[ $(date +%s) -ge ${_dl_sync_deadline} ]]; then
    _err "[acg-up] data-layer ArgoCD Application did not reach Synced within 300s — check: kubectl get application data-layer -n cicd --context k3d-k3d-cluster"
    exit 1
  fi
  _info "[acg-up] data-layer not yet Synced — waiting..."
  sleep 10
done
_info "[acg-up] data-layer Application Synced — proceeding to StatefulSet readiness check"
deploy_shopping_cart_data
```

---

### Change 2 — `scripts/plugins/shopping_cart.sh`: remove redundant StatefulSet existence wait

Now that the sync-wait in `bin/acg-up` guarantees `data-layer` is `Synced` before
`deploy_shopping_cart_data` runs, the StatefulSets already exist by the time the function
is called. Replace the existence-polling loop with a single presence check that errors
immediately if a StatefulSet is missing (should never happen after a Synced Application).

**Exact old block (lines 96–111):**

```bash
  _info "[shopping_cart] Data layer managed by ArgoCD — waiting for readiness..."

  _info "[shopping_cart] Waiting for ArgoCD to create data-layer StatefulSets..."
  local _sts_deadline
  for pg in postgresql-orders postgresql-payment postgresql-products; do
    _sts_deadline=$(( $(date +%s) + 300 ))
    until kubectl get statefulset/"${pg}" \
        -n shopping-cart-data --context ubuntu-k3s >/dev/null 2>&1; do
      if [[ $(date +%s) -ge ${_sts_deadline} ]]; then
        _err "[shopping_cart] Timed out waiting for StatefulSet ${pg} to be created by ArgoCD"
        return 1
      fi
      _info "[shopping_cart] StatefulSet ${pg} not yet created — waiting..."
      sleep 10
    done
  done
```

**Exact new block:**

```bash
  _info "[shopping_cart] Data layer managed by ArgoCD — verifying StatefulSet presence..."

  for pg in postgresql-orders postgresql-payment postgresql-products; do
    if ! kubectl get statefulset/"${pg}" \
        -n shopping-cart-data --context ubuntu-k3s >/dev/null 2>&1; then
      _err "[shopping_cart] StatefulSet ${pg} not found in shopping-cart-data after ArgoCD sync — check: kubectl get application data-layer -n cicd --context k3d-k3d-cluster"
      return 1
    fi
  done
```
