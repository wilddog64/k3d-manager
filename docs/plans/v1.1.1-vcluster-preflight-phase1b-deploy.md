# v1.1.1 — vCluster Preflight Phase 1b: wire the deploy path

**Branch:** `k3d-manager-v1.7.1`
**File:** `bin/cluster-preflight`
**Status:** PLAN — ready for Codex
**Depends on:** Phase 1a spike (already committed; proven live 2026-06-16 — vcluster on Hostinger + NodePort + ArgoCD registration all work)

---

## Problem

`bin/cluster-preflight` is a **Phase 1a spike**: `main` creates the vcluster, registers it
with hub ArgoCD, then **stops at `exit 0` (line ~569)** and never deploys anything. The
functions that render + apply the **prefixed-clone** ArgoCD resources
(`_cluster_preflight_apply_argocd_resources`, line ~503) and wait for them
(`_cluster_preflight_wait_for_apps`, line ~98) are **fully written but never called** — dead code.

Two gaps to close:

1. **Deploy is unwired** — wire the existing apply + wait functions into `main` so the stack
   actually deploys into the registered vcluster.
2. **Exit code is wrong on success** — the spike exits such that `make preflight` reports
   `make: Error 1` even though registration succeeded. A non-zero exit on success will break
   the future Phase 3 commit-status merge gate (CI keys off the exit code). Success must `exit 0`.

Also add a **`--reuse` mode** so we can deploy into an **already-running, already-registered**
vcluster (e.g. the live `pr33`) without tearing it down and rebuilding — this is how the deploy
path gets tested against the live Hostinger vcluster.

**Root cause:** the spike intentionally returned at the registration checkpoint; the deploy
increment was deferred.

---

## Reproduction

```bash
make preflight NAME=pr33 MODE=--keep
# → registers cluster-pr33, logs "Phase 1a spike reached registration stop point",
#   deploys NOTHING, and make exits with "Error 1" despite success.
```

## Fix

> All line numbers are approximate — match on the literal blocks.

### Change 1 — accept `--reuse` mode

**Exact old block (the mode `case`, lines ~32–39):**

```bash
case "${PREFLIGHT_MODE}" in
  --keep|--destroy)
    ;;
  *)
    _cluster_preflight_usage
    exit 1
    ;;
esac
```

**Exact new block:**

```bash
PREFLIGHT_REUSE=0
case "${PREFLIGHT_MODE}" in
  --keep|--destroy)
    ;;
  --reuse)
    PREFLIGHT_REUSE=1
    PREFLIGHT_MODE="--keep"
    ;;
  *)
    _cluster_preflight_usage
    exit 1
    ;;
esac
```

Also update the usage string.

**Exact old block (lines ~13–17):**

```bash
function _cluster_preflight_usage() {
  cat <<'EOF'
Usage: cluster-preflight <name> [--keep|--destroy]
EOF
}
```

**Exact new block:**

```bash
function _cluster_preflight_usage() {
  cat <<'EOF'
Usage: cluster-preflight <name> [--keep|--destroy|--reuse]
  --keep     create the vCluster, deploy the stack, leave it running (default)
  --destroy  create, deploy, then tear down + prune ArgoCD resources on exit
  --reuse    skip create/register; deploy into an already-registered vCluster, keep it
EOF
}
```

### Change 2 — guard create/register on `--reuse`, then deploy + wait, then exit 0

**Exact old block (main flow, lines ~530–569 — the create→register→spike-stop region):**

```bash
_info "[preflight] Creating vCluster ${PREFLIGHT_NAME}..."
VCLUSTER_VALUES_FILE="${PREFLIGHT_VCLUSTER_VALUES_FILE}" \
VCLUSTER_HOST_CONTEXT="${VCLUSTER_HOST_CONTEXT}" \
  _run_command -- "${REPO_ROOT}/scripts/k3d-manager" vcluster_create "${PREFLIGHT_NAME}"

if [[ ! -f "${VC_KCFG}" ]]; then
  _err "[preflight] Expected vCluster kubeconfig not found: ${VC_KCFG}"
fi

_info "[preflight] Applying argocd-manager RBAC inside ${PREFLIGHT_NAME}..."
KUBECONFIG="${VC_KCFG}" _kubectl apply -f "${REPO_ROOT}/scripts/etc/vcluster/argocd-manager-rbac.yaml" >/dev/null

_info "[preflight] Minting argocd-manager token inside ${PREFLIGHT_NAME}..."
VC_TOKEN="$(KUBECONFIG="${VC_KCFG}" _kubectl create token argocd-manager -n kube-system --duration=8760h)"
if [[ -z "${VC_TOKEN}" ]]; then
  _err "[preflight] Failed to mint argocd-manager token inside ${PREFLIGHT_NAME}"
fi

_info "[preflight] Deriving ${PREFLIGHT_NAME} API URL from NodePort..."
_cluster_preflight_resolve_app_cluster_server
_info "[preflight] Verifying ${APP_CLUSTER_SERVER}/healthz from this host..."
_cluster_preflight_verify_app_cluster_server

_info "[preflight] Registering ${PREFLIGHT_NAME} with hub ArgoCD..."
if [[ -n "${REGISTER_KUBE_CONTEXT}" && "${REGISTER_KUBE_CONTEXT}" != "$(_kubectl config current-context 2>/dev/null || true)" ]]; then
  _info "[preflight] Switching kubectl context to ${REGISTER_KUBE_CONTEXT} for ArgoCD registration..."
  _kubectl config use-context "${REGISTER_KUBE_CONTEXT}" >/dev/null
fi
ARGOCD_APP_CLUSTER_NAME="${APP_CLUSTER_NAME}" \
ARGOCD_APP_CLUSTER_SECRET_NAME="${APP_CLUSTER_SECRET_NAME}" \
ARGOCD_APP_CLUSTER_SERVER="${APP_CLUSTER_SERVER}" \
ARGOCD_APP_CLUSTER_TOKEN="${VC_TOKEN}" \
  _run_command -- "${REPO_ROOT}/scripts/k3d-manager" register_app_cluster

if [[ "${PREFLIGHT_MODE}" == "--keep" ]]; then
  _info "[preflight] Phase 1a spike reached registration stop point; vCluster ${PREFLIGHT_NAME} is left running"
else
  _info "[preflight] Phase 1a spike reached registration stop point; vCluster ${PREFLIGHT_NAME} will be destroyed on exit"
fi
exit 0
```

**Exact new block:**

```bash
if [[ "${PREFLIGHT_REUSE}" -eq 1 ]]; then
  _info "[preflight] --reuse: skipping create/register for existing vCluster ${PREFLIGHT_NAME}..."
  if [[ ! -f "${VC_KCFG}" ]]; then
    _err "[preflight] --reuse requires an existing vCluster kubeconfig: ${VC_KCFG} (run --keep first)"
  fi
  _info "[preflight] Deriving ${PREFLIGHT_NAME} API URL from NodePort..."
  _cluster_preflight_resolve_app_cluster_server
  _info "[preflight] Verifying ${APP_CLUSTER_SERVER}/healthz from this host..."
  _cluster_preflight_verify_app_cluster_server
else
  _info "[preflight] Creating vCluster ${PREFLIGHT_NAME}..."
  VCLUSTER_VALUES_FILE="${PREFLIGHT_VCLUSTER_VALUES_FILE}" \
  VCLUSTER_HOST_CONTEXT="${VCLUSTER_HOST_CONTEXT}" \
    _run_command -- "${REPO_ROOT}/scripts/k3d-manager" vcluster_create "${PREFLIGHT_NAME}"

  if [[ ! -f "${VC_KCFG}" ]]; then
    _err "[preflight] Expected vCluster kubeconfig not found: ${VC_KCFG}"
  fi

  _info "[preflight] Applying argocd-manager RBAC inside ${PREFLIGHT_NAME}..."
  KUBECONFIG="${VC_KCFG}" _kubectl apply -f "${REPO_ROOT}/scripts/etc/vcluster/argocd-manager-rbac.yaml" >/dev/null

  _info "[preflight] Minting argocd-manager token inside ${PREFLIGHT_NAME}..."
  VC_TOKEN="$(KUBECONFIG="${VC_KCFG}" _kubectl create token argocd-manager -n kube-system --duration=8760h)"
  if [[ -z "${VC_TOKEN}" ]]; then
    _err "[preflight] Failed to mint argocd-manager token inside ${PREFLIGHT_NAME}"
  fi

  _info "[preflight] Deriving ${PREFLIGHT_NAME} API URL from NodePort..."
  _cluster_preflight_resolve_app_cluster_server
  _info "[preflight] Verifying ${APP_CLUSTER_SERVER}/healthz from this host..."
  _cluster_preflight_verify_app_cluster_server

  _info "[preflight] Registering ${PREFLIGHT_NAME} with hub ArgoCD..."
  if [[ -n "${REGISTER_KUBE_CONTEXT}" && "${REGISTER_KUBE_CONTEXT}" != "$(_kubectl config current-context 2>/dev/null || true)" ]]; then
    _info "[preflight] Switching kubectl context to ${REGISTER_KUBE_CONTEXT} for ArgoCD registration..."
    _kubectl config use-context "${REGISTER_KUBE_CONTEXT}" >/dev/null
  fi
  ARGOCD_APP_CLUSTER_NAME="${APP_CLUSTER_NAME}" \
  ARGOCD_APP_CLUSTER_SECRET_NAME="${APP_CLUSTER_SECRET_NAME}" \
  ARGOCD_APP_CLUSTER_SERVER="${APP_CLUSTER_SERVER}" \
  ARGOCD_APP_CLUSTER_TOKEN="${VC_TOKEN}" \
    _run_command -- "${REPO_ROOT}/scripts/k3d-manager" register_app_cluster
fi

if [[ -n "${REGISTER_KUBE_CONTEXT}" && "${REGISTER_KUBE_CONTEXT}" != "$(_kubectl config current-context 2>/dev/null || true)" ]]; then
  _info "[preflight] Switching kubectl context to ${REGISTER_KUBE_CONTEXT} for ArgoCD deploy..."
  _kubectl config use-context "${REGISTER_KUBE_CONTEXT}" >/dev/null
fi

_info "[preflight] Deploying preflight stack (prefixed-clone ArgoCD resources) for ${PREFLIGHT_NAME}..."
_cluster_preflight_apply_argocd_resources "${APP_CLUSTER_NAME}"

_info "[preflight] Waiting up to ${PREFLIGHT_WAIT_TIMEOUT}s for preflight apps to become Synced/Healthy..."
_cluster_preflight_wait_for_apps "${APP_CLUSTER_NAME}" "${PREFLIGHT_WAIT_TIMEOUT}"

if [[ "${PREFLIGHT_MODE}" == "--keep" ]]; then
  _info "[preflight] Preflight stack deployed; vCluster ${PREFLIGHT_NAME} is left running"
else
  _info "[preflight] Preflight stack deployed and verified; vCluster ${PREFLIGHT_NAME} will be destroyed on exit"
fi
exit 0
```

> `register_app_cluster` (when not `--reuse`) leaves the context on `REGISTER_KUBE_CONTEXT`
> already, so the second context-switch block is a no-op there; it exists so `--reuse` (which
> skipped the register block) lands on the hub context before applying. Do not remove it.
>
> `_cluster_preflight_wait_for_apps` returns non-zero on timeout — with `set -euo pipefail`
> that propagates as a real failure exit (correct: a stuck deploy must fail the gate). Only the
> success path reaches `exit 0`. Do NOT swallow the wait's exit code.
