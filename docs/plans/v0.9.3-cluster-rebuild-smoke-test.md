# Task: Cluster Rebuild Smoke Test

**Repo:** `k3d-manager`
**Branch:** `k3d-manager-v0.9.3`
**Agent:** Gemini (M2 Air — has live cluster access)
**Purpose:** Verify lib-foundation v0.3.2 changes and TTY fix work correctly end-to-end on a real cluster rebuild.

---

## Step 0 — Machine check

```bash
hostname && uname -n
```

Must be on M2 Air. Stop if not.

---

## Step 1 — Sync branch

```bash
git -C ~/src/gitrepo/personal/k3d-manager fetch origin
git -C ~/src/gitrepo/personal/k3d-manager checkout k3d-manager-v0.9.3
git -C ~/src/gitrepo/personal/k3d-manager pull origin k3d-manager-v0.9.3
```

Confirm latest commit is `04522b5` (TTY fix). Stop if not.

---

## Step 2 — Destroy existing clusters

**Infra cluster (k3d on OrbStack):**

```bash
cd ~/src/gitrepo/personal/k3d-manager
CLUSTER_ROLE=infra ./scripts/k3d-manager destroy_cluster
```

**App cluster (Ubuntu k3s — SSH from M2 Air):**

```bash
ssh ubuntu "sudo k3s-uninstall.sh"
```

Confirm both clusters are gone before proceeding:

```bash
# Infra: no k3d clusters
k3d cluster list

# App: k3s gone
ssh ubuntu "kubectl get nodes 2>&1 || echo 'k3s removed'"
```

---

## Step 3 — Rebuild infra cluster

```bash
cd ~/src/gitrepo/personal/k3d-manager
CLUSTER_ROLE=infra CLUSTER_PROVIDER=k3d ./scripts/k3d-manager deploy_cluster
```

**What to verify during deploy:**
- No `Non-interactive session detected; defaulting to k3d provider` message when `CLUSTER_PROVIDER` is set explicitly (that message should only appear when no provider is set and stdin is not a TTY)
- No errors during cluster creation

Then deploy all infra services in order:

```bash
./scripts/k3d-manager deploy_vault
./scripts/k3d-manager unseal_vault
./scripts/k3d-manager deploy_eso
./scripts/k3d-manager deploy_ldap
./scripts/k3d-manager deploy_istio
./scripts/k3d-manager deploy_jenkins
./scripts/k3d-manager deploy_argocd
./scripts/k3d-manager deploy_keycloak
./scripts/k3d-manager deploy_cert_manager
```

## Step 4 — Verify infra cluster health

```bash
kubectl get pods -A | grep -v Running | grep -v Completed
```

Expected: no pods in error state. All components from the table below must be Running:

| Component | Namespace |
|---|---|
| Vault | `secrets` |
| ESO | `secrets` |
| OpenLDAP | `identity` / `directory` |
| Istio | `istio-system` |
| Jenkins | `cicd` |
| ArgoCD | `cicd` |
| Keycloak | `identity` |
| cert-manager | `cert-manager` |

Also verify Vault is unsealed:

```bash
kubectl exec -n secrets vault-0 -- vault status | grep Sealed
```

Expected: `Sealed  false`

## Step 5 — Rebuild app cluster (Ubuntu k3s)

SSH to Ubuntu and install k3s:

```bash
ssh ubuntu
curl -sfL https://get.k3s.io | sh -
exit
```

Wait for node to be Ready, then register it with the infra cluster and deploy services:

```bash
cd ~/src/gitrepo/personal/k3d-manager
./scripts/k3d-manager add_ubuntu_k3s_cluster
./scripts/k3d-manager deploy_vault        # cross-cluster Vault auth
./scripts/k3d-manager deploy_eso          # ESO on app cluster
./scripts/k3d-manager deploy_ldap         # OpenLDAP on app cluster
./scripts/k3d-manager deploy_istio        # Istio on app cluster
./scripts/k3d-manager register_shopping_cart_apps
```

Verify app cluster health:

```bash
ssh ubuntu "kubectl get nodes"
ssh ubuntu "kubectl get pods -A" | grep -v Running | grep -v Completed
```

Expected services Running on app cluster:

| Component | Namespace |
|---|---|
| k3s node | Ready |
| Istio | `istio-system` |
| ESO | `secrets` |
| Vault (cross-cluster auth) | configured |
| OpenLDAP | `identity` |
| shopping-cart apps | `default` / ArgoCD Synced |
