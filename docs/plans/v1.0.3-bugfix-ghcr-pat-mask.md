## Problem

`make up` prints the full command line before executing it:

```
GHCR_PAT="ghp_xxx..." bin/acg-up "https://..."
```

This is Make's default command echo behaviour. The PAT value is visible in the terminal
and any CI/CD log that captures stdout.

**Root cause:** The `up` recipe line in `Makefile` has no `@` prefix, so Make echoes
`GHCR_PAT="$(GHCR_PAT)" bin/acg-up "$(URL)"` — with the PAT fully expanded — before
running it.

**Note:** `--docker-password` in `bin/acg-up` line 102 is a separate issue that requires
a lib-foundation change first (`--docker-password` is not in `_args_have_sensitive_flag`).
That is out of scope here.

---

## Fix

### Change 1 — `Makefile` lines 12–13: suppress Make command echo on `up` target

**Exact old block:**

```makefile
## Provision full stack: credentials → cluster → ESO → ArgoCD
up:
	GHCR_PAT="$(GHCR_PAT)" bin/acg-up "$(URL)"
```

**Exact new block:**

```makefile
## Provision full stack: credentials → cluster → ESO → ArgoCD
up:
	@echo "[make] Running bin/acg-up..."
	@GHCR_PAT="$(GHCR_PAT)" bin/acg-up "$(URL)"
```
