# Spec: v1.6.3 — `make fix-*` agent fix targets

**Branch:** `k3d-manager-v1.6.3`
**Files:** `Makefile`, `bin/k3dm-ask-bash`, `bin/k3dm-webhook`, `docs/howto/slack-slash-commands.md`

---

## Problem

The `/ask` agent fix mode currently allows raw `kubectl rollout restart`, `kubectl delete pod`,
and `argocd app sync` in the bash sandbox. These are unbounded in how Claude can phrase them —
the sandbox pattern-matches against flag combinations, which is brittle and hard to audit.

There is no way for Claude to discover what fix operations are available. It must invent
commands from scratch, which increases the chance of typos, wrong namespaces, or unsafe
argument combinations.

**Goal:** Replace ad-hoc raw mutations with a named, discoverable `make fix-*` target set.
Each target encodes the safe sequence for a common recovery operation (restart + wait, sync
+ force, delete pod by label). Claude calls `make fix-list` first to discover available
targets, then invokes the right one. Raw kubectl reads remain available for diagnosis.

---

## Design

**Hybrid model (not a full replacement):**

The bash sandbox still allows the narrow kubectl allowlist in fix mode because `make`
spawns bash sub-processes that call kubectl internally — and those sub-processes hit the
same sandbox. The new work adds:

1. `make fix-*` targets to the sandbox allowlist in fix mode (new pattern)
2. `make fix-*` targets to the Makefile (new section)
3. System prompt update: Claude is told to prefer `make fix-*` and to run `make fix-list`
   first to discover what's available
4. Docs update: fix mode section updated to mention make targets

**Sandbox logic stays coherent:** `make fix-restart APP=data-layer NS=cicd` is allowed;
internally make calls `kubectl rollout restart deployment/data-layer -n cicd --context
ubuntu-k3s`, which passes the sandbox check because `kubectl rollout restart` is in the
fix-mode allowlist. No bypass needed.

## Change 1 — `Makefile`: add `fix-*` targets section

Insert the following block **before** the `## Run all BATS test suites` comment
(after the `vuln-scan` target, before line 341). Read the file first to confirm the
insertion point.

**Exact new block:**

```makefile
## ── Agent Fix Targets ────────────────────────────────────────────────────────
## Callable by /ask agents in fix mode. Use 'make fix-list' to discover targets.
## All targets accept CONTEXT (default: ubuntu-k3s) and NS (namespace).

FIX_CONTEXT ?= ubuntu-k3s

## List all available fix targets with descriptions
fix-list:
	@grep -E '^fix-[a-z].*:.*##' Makefile | sort | awk -F':.*##' '{printf "  make %-30s %s\n", $$1, $$2}'

## Rollout restart a deployment and wait for rollout (APP=<name> NS=<namespace>)
fix-restart: ## APP and NS are required
	@test -n "$(APP)" || { echo "Usage: make fix-restart APP=<deployment> NS=<namespace>"; exit 1; }
	@test -n "$(NS)"  || { echo "Usage: make fix-restart APP=<deployment> NS=<namespace>"; exit 1; }
	kubectl rollout restart deployment/$(APP) -n $(NS) --context $(FIX_CONTEXT)
	kubectl rollout status  deployment/$(APP) -n $(NS) --context $(FIX_CONTEXT) --timeout=120s

## Delete all pods matching label app=<APP> (forces pod restart via ReplicaSet)
fix-delete-pod: ## APP and NS are required
	@test -n "$(APP)" || { echo "Usage: make fix-delete-pod APP=<label> NS=<namespace>"; exit 1; }
	@test -n "$(NS)"  || { echo "Usage: make fix-delete-pod APP=<label> NS=<namespace>"; exit 1; }
	kubectl delete pod -l app=$(APP) -n $(NS) --context $(FIX_CONTEXT) --grace-period=0

## ArgoCD app sync with 120s timeout (APP=<argocd-app-name>)
fix-sync: ## APP is required
	@test -n "$(APP)" || { echo "Usage: make fix-sync APP=<argocd-app-name>"; exit 1; }
	argocd app sync $(APP) --timeout 120 --server localhost:8080 --insecure

## ArgoCD force sync — discards local state (APP=<argocd-app-name>)
fix-force-sync: ## APP is required
	@test -n "$(APP)" || { echo "Usage: make fix-force-sync APP=<argocd-app-name>"; exit 1; }
	argocd app sync $(APP) --force --timeout 180 --server localhost:8080 --insecure

## Force ESO ClusterSecretStore reconcile (annotates vault-backend to trigger re-sync)
fix-eso-refresh: ## No arguments needed
	kubectl annotate clustersecretstore vault-backend \
	  k3d-manager/reconcile-at="$$(date -u +%Y-%m-%dT%H:%M:%SZ)" --overwrite \
	  --context $(FIX_CONTEXT)

## Print node + pod status for a namespace (NS=<namespace>)
fix-status: ## NS is required
	@test -n "$(NS)" || { echo "Usage: make fix-status NS=<namespace>"; exit 1; }
	kubectl get nodes --context $(FIX_CONTEXT) --no-headers
	kubectl get pods -n $(NS) --context $(FIX_CONTEXT)

```

## Change 2 — `bin/k3dm-ask-bash`: add `make fix-*` to fix mode allowlist

**Exact old block (lines 24–35):**

```bash
if [[ "$FIX_MODE" == "1" ]]; then
  # ── Fix mode: allow specific recovery operations, block everything else ──────

  # kubectl: allow rollout restart and delete pod only
  if echo "$CMD" | grep -qE '\bkubectl\b'; then
    if echo "$CMD" | grep -qE '\bkubectl\b.*\bdelete\b' && ! echo "$CMD" | grep -qE '\bkubectl\b.*delete\s+pod'; then
      _deny "$CMD"
    fi
    if echo "$CMD" | grep -qE '\bkubectl\b.*(apply|create|edit|patch|replace|scale|cordon|drain|taint|label|annotate|expose|run\b|cp\b|exec\b|port-forward)'; then
      _deny "$CMD"
    fi
  fi
```

**Exact new block:**

```bash
if [[ "$FIX_MODE" == "1" ]]; then
  # ── Fix mode: allow make fix-* targets + narrow kubectl/argocd ops ───────────
  # make fix-* targets are the preferred path; raw kubectl ops remain in the
  # allowlist because make targets call kubectl in sub-processes via this sandbox.

  # make: only fix-* targets allowed; all other make invocations are blocked
  if echo "$CMD" | grep -qE '\bmake\b'; then
    if ! echo "$CMD" | grep -qE '\bmake\s+fix-[a-z]'; then
      _deny "$CMD"
    fi
  fi

  # kubectl: allow rollout restart and delete pod only
  if echo "$CMD" | grep -qE '\bkubectl\b'; then
    if echo "$CMD" | grep -qE '\bkubectl\b.*\bdelete\b' && ! echo "$CMD" | grep -qE '\bkubectl\b.*delete\s+pod'; then
      _deny "$CMD"
    fi
    if echo "$CMD" | grep -qE '\bkubectl\b.*(apply|create|edit|patch|replace|scale|cordon|drain|taint|label|annotate|expose|run\b|cp\b|exec\b|port-forward)'; then
      _deny "$CMD"
    fi
  fi
```

```markdown
If the question contains phrasing like *"fix this"*, *"restart the pod"*, or *"sync the app"*,
```

**Exact new block:**

```markdown
### Fix mode

If the question contains phrasing like *"fix this"*, *"restart the pod"*, or *"sync the app"*,
the agent switches to **fix mode** (`K3DM_FIX_MODE=1`):

- Claude first runs `make fix-list` to discover available fix targets, then invokes the
  appropriate `make fix-*` target. Named targets encode the safe operation sequence
  (e.g., `fix-restart` does `rollout restart` + `rollout status --timeout=120s`).
- Raw kubectl (`rollout restart`, `delete pod`) and `argocd app sync` remain available
  as a fallback when no make target fits — and are required internally by make sub-processes.

  | Allowed in fix mode | Purpose |
  |---------------------|---------|
  | `make fix-*` | Named recovery targets (preferred) |
  | `kubectl rollout restart` | Fallback / used by `make fix-restart` internally |
  | `kubectl delete pod` | Fallback / used by `make fix-delete-pod` internally |
  | `argocd app sync` | Fallback / used by `make fix-sync` internally |

- All other kubectl writes, Helm writes, and ArgoCD mutations remain blocked.
- If fix + file modes are both detected, Claude runs the fix **and** writes a bug doc
  in a single pass. The Slack reply posts a `Status: Fixed` block followed by the
  `## Fix Applied` section.

Fix mode uses the same timeout and turn limit as standard ask mode (300 s / 5 turns).

**Available `make fix-*` targets** (run `make fix-list` in the repo to see current list):

| Target | Arguments | What it does |
|--------|-----------|--------------|
| `fix-list` | — | Print all fix targets with descriptions |
| `fix-restart` | `APP NS` | `kubectl rollout restart` + `rollout status --timeout=120s` |
| `fix-delete-pod` | `APP NS` | `kubectl delete pod -l app=<APP>` with grace-period=0 |
| `fix-sync` | `APP` | `argocd app sync --timeout 120` |
| `fix-force-sync` | `APP` | `argocd app sync --force --timeout 180` |
| `fix-eso-refresh` | — | Annotate `vault-backend` ClusterSecretStore to force ESO reconcile |
| `fix-status` | `NS` | Print nodes + pods for a namespace |

`CONTEXT` defaults to `ubuntu-k3s`. Override with `make fix-restart APP=x NS=y CONTEXT=k3d-k3d-cluster`.
```
