# Bugfix: v1.4.12 — named ServiceAccounts missing imagePullSecrets for ghcr.io

**Branch:** `k3d-manager-v1.4.12`
**Files:** `services/shopping-cart-basket/kustomization.yaml`,
`services/shopping-cart-order/kustomization.yaml`,
`services/shopping-cart-payment/kustomization.yaml`,
`services/shopping-cart-product-catalog/kustomization.yaml`

---

## Problem

Four service pods are stuck in `ImagePullBackOff` with `401 Unauthorized` from `ghcr.io`:

- `basket-service` (shopping-cart-apps ns)
- `order-service` (shopping-cart-apps ns)
- `product-catalog` (shopping-cart-apps ns)
- `payment-service` (shopping-cart-payment ns)

`ghcr-pull-secret` is set as `imagePullSecrets` on the `default` ServiceAccount in
`shopping-cart-apps`. Kubernetes does NOT propagate `imagePullSecrets` from `default` to
other ServiceAccounts. Each service uses a named SA (`basket-service`, `order-service`,
etc.) which has no `imagePullSecrets`.

**Root cause:** v1.4.11 moved `imagePullSecrets` from per-app patches to the `default` SA,
but this only works for pods that use `default`. Named SAs are unaffected.

---

## Reproduction

```bash
kubectl --context=ubuntu-k3s describe pod -n shopping-cart-apps \
  $(kubectl --context=ubuntu-k3s get pod -n shopping-cart-apps -l app=basket-service -o name) \
  | grep "401 Unauthorized"
```

---

## Fix

Add a Kustomize strategic-merge patch to each affected service's `kustomization.yaml` that
adds `imagePullSecrets: [{name: ghcr-pull-secret}]` to the named ServiceAccount.

---

### Change 1 — `services/shopping-cart-basket/kustomization.yaml`

**Exact old content:**

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - https://github.com/wilddog64/shopping-cart-basket//k8s/base?ref=main
```

**Exact new content:**

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - https://github.com/wilddog64/shopping-cart-basket//k8s/base?ref=main
patches:
  - patch: |-
      apiVersion: v1
      kind: ServiceAccount
      metadata:
        name: basket-service
      imagePullSecrets:
        - name: ghcr-pull-secret
    target:
      kind: ServiceAccount
      name: basket-service
```
