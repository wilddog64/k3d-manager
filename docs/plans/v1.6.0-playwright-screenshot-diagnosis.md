# Feature: v1.6.0 — Playwright screenshot diagnosis on failure

**Branch (k3d-manager):** `k3d-manager-v1.6.0`
**Branch (lib-acg):** `fix/playwright-screenshot-diagnosis`

---

## Problem

When `acg_credentials.js` or `acg_restart.js` fails, the webhook's failure analysis
(`_analyze_failure`) only has text log output. The log says *what* failed but not *what
the browser showed* — a survey modal, a redirect to login, an empty panel, an error banner.
Adding a screenshot at the failure point gives the AI visual context to name the exact
UI state instead of guessing.

---

## Solution

**Phase A (lib-acg):** Both Playwright scripts save a screenshot on any unhandled error
and print the path to stderr as `INFO: Screenshot saved to /tmp/k3dm-acg-screenshot-<ts>.png`.

**Phase B (k3d-manager):** The webhook parses the job log for that line, base64-encodes
the image, and calls the Claude API (Anthropic) with the screenshot appended to the
existing failure analysis prompt. Falls back silently to text-only analysis when
no screenshot is found or `ANTHROPIC_API_KEY` is not in the webhook's environment.

A new `update-webhook-env` Makefile target injects `ANTHROPIC_API_KEY` from the shell
environment into the installed LaunchAgent plist and restarts the webhook.

```python
def _call_claude_vision(prompt, screenshot_path):
    """Call Claude API with a PNG screenshot + text prompt. Returns description or error string."""
    if not ANTHROPIC_API_KEY:
        return "vision analysis skipped — ANTHROPIC_API_KEY not set in webhook environment"
    try:
        with open(screenshot_path, "rb") as f:
            image_data = base64.b64encode(f.read()).decode("utf-8")
    except OSError as exc:
        return f"vision analysis skipped — could not read screenshot: {exc}"
    payload = json.dumps({
        "model": "claude-haiku-4-5-20251001",
        "max_tokens": 256,
        "messages": [
            {
                "role": "user",
                "content": [
                    {
                        "type": "image",
                        "source": {
                            "type": "base64",
                            "media_type": "image/png",
                            "data": image_data,
                        },
                    },
                    {"type": "text", "text": prompt},
                ],
            }
        ],
    }).encode()
    req = urllib.request.Request(
        "https://api.anthropic.com/v1/messages",
        data=payload,
        headers={
            "x-api-key": ANTHROPIC_API_KEY,
            "anthropic-version": "2023-06-01",
            "content-type": "application/json",
        },
        method="POST",
    )
    try:
        with urllib.request.urlopen(req, timeout=30) as resp:
            result = json.loads(resp.read())
            return result["content"][0]["text"].strip()
    except Exception as exc:
        return f"vision analysis error: {exc}"


def _analyze_stall(lines):
```

### Change 7 — `bin/k3dm-webhook`: modify `_analyze_failure()` to attach screenshot

Add screenshot detection and vision call at the end of `_analyze_failure`, just before
the `return _call_gemini(prompt)` line.

**Exact old block:**
```python
        "Reply in exactly this format:\n"
        "CLUSTER_FAILURE|APP_FAILURE\n"
        "TRANSIENT|PERMANENT\n"
        "<root cause — one sentence>\n"
        "<next step — one sentence, include the exact command if possible>"
    )
    return _call_gemini(prompt)
```

**Exact new block:**
```python
        "Reply in exactly this format:\n"
        "CLUSTER_FAILURE|APP_FAILURE\n"
        "TRANSIENT|PERMANENT\n"
        "<root cause — one sentence>\n"
        "<next step — one sentence, include the exact command if possible>"
    )
    screenshot_path = None
    for line in reversed(lines):
        m = _SCREENSHOT_RE.search(line)
        if m:
            screenshot_path = m.group(1)
            break
    if screenshot_path and os.path.isfile(screenshot_path):
        vision_prompt = (
            "This is a screenshot of the Pluralsight ACG sandbox browser at the moment "
            "a Playwright automation script failed. Describe what you see in 2-3 sentences: "
            "which page or dialog is visible, what UI elements are present, and what "
            "likely caused the automation script to fail."
        )
        visual_note = _call_claude_vision(vision_prompt, screenshot_path)
        prompt = prompt.rstrip() + f"\n\nBrowser screenshot at time of failure: {visual_note}"
    return _call_gemini(prompt)
```
