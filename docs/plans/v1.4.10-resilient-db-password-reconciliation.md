# Plan: Resilient DB Password Reconciliation

## Objective
Fix the persistent `FATAL: password authentication failed` error in the `order-service` (and potentially other services) by eliminating legacy placeholder password logic and ensuring Vault-backed passwords are correctly applied to stateful databases during the `acg-up` bootstrap.

## Root Cause Analysis
1. **Legacy Override:** The `deploy_shopping_cart_data` function in `scripts/plugins/shopping_cart.sh` contained a legacy loop that blindly executed `ALTER USER postgres WITH PASSWORD 'CHANGE_ME';` on all PostgreSQL pods.
2. **Execution Timing:** This legacy loop ran *after* the `shopping_cart_reconcile_order_service` attempt, immediately overwriting the correct Vault password with `CHANGE_ME`.
3. **Deployment Race:** `shopping_cart_reconcile_order_service` was called too early in the bootstrap sequence (inside `deploy_shopping_cart_data`). At that point, ArgoCD hadn't fully deployed the application pods, so the logic that restarts the deployment to pick up the new secret was skipped.

## Implementation Steps

### Step 1: Cleanup `scripts/plugins/shopping_cart.sh`
Remove the legacy `CHANGE_ME` alignment loop and the prematurely placed `shopping_cart_reconcile_order_service` call from the `deploy_shopping_cart_data` function.

**Target File:** `scripts/plugins/shopping_cart.sh`
*   **Remove** the `shopping_cart_reconcile_order_service` line inside `deploy_shopping_cart_data`.
*   **Remove** the block that prints `[shopping_cart] Aligning PostgreSQL passwords to match app secrets (CHANGE_ME)...` and loops over `postgresql-orders-0` and `postgresql-products-0` to set the password to `CHANGE_ME`.

### Step 2: Update `bin/acg-up` Execution Order
Call both reconcilers at the end of the `acg-up` sequence, after the `ClusterSecretStore` is verified as `Ready` and ArgoCD has had time to deploy the application resources.

**Target File:** `bin/acg-up`
*   Locate the `shopping_cart_reconcile_product_catalog` call (around line 1154).
*   Add `shopping_cart_reconcile_order_service` immediately before it.

## Verification & Testing
1.  **Cold Run:** Execute `make down` followed by a full `make up`.
2.  **Health Check:** Verify that both `order-service` and `product-catalog` ArgoCD applications eventually reach `Synced` and `Healthy` without any `CrashLoopBackOff` from `password authentication failed`.
3.  **Logs:** Verify `acg-up` output shows both "PostgreSQL orders password reconciled with Vault" and the corresponding log for `product-catalog`.
