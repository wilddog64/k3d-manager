## Problem

`_provider_k3s_gcp_deploy_cluster` hard-fails with:

```
ERROR: [k3s-gcp] gcloud CLI not found in PATH
```

The raw `command -v gcloud` check does not attempt installation. Every other
prerequisite in the codebase auto-installs (`_ensure_k3sup`, `_ensure_node`,
`_ensure_copilot_cli`). `gcloud` must follow the same pattern.

## Changes

### 1. `scripts/plugins/gcp.sh` — add `_ensure_gcloud`

Add the function immediately before `gcp_get_credentials`.

**Before** (no such function exists):

```bash
function gcp_get_credentials() {
```

**After** (insert new function above):

```bash
function _ensure_gcloud() {
  if command -v gcloud >/dev/null 2>&1; then
    return 0
  fi
  _info "[gcp] gcloud CLI not found — installing..."
  if _command_exist brew; then
    _run_command --soft -- brew install --cask google-cloud-sdk
    if command -v gcloud >/dev/null 2>&1; then
      return 0
    fi
  fi
  if _is_debian_family && _command_exist curl; then
    local _gcloud_installer
    _gcloud_installer="$(mktemp)"
    if ! curl -fsSL -o "${_gcloud_installer}" \
        https://sdk.cloud.google.com; then
      rm -f "${_gcloud_installer}"
      _err "[gcp] Failed to download Google Cloud SDK installer"
      return 1
    fi
    CLOUDSDK_CORE_DISABLE_PROMPTS=1 \
      _run_command --soft -- bash "${_gcloud_installer}" --disable-prompts \
        --install-dir="${HOME}/.local/share/google-cloud-sdk"
    rm -f "${_gcloud_installer}"
    # SDK installs gcloud to ${HOME}/.local/share/google-cloud-sdk/google-cloud-sdk/bin
    if command -v gcloud >/dev/null 2>&1; then
      return 0
    fi
    # Try adding bin dir to PATH for remainder of this shell session
    local _gcloud_bin="${HOME}/.local/share/google-cloud-sdk/google-cloud-sdk/bin"
    if [[ -f "${_gcloud_bin}/gcloud" ]]; then
      export PATH="${_gcloud_bin}:${PATH}"
      return 0
    fi
  fi
  _err "[gcp] gcloud CLI not found and automatic installation failed — install manually: brew install --cask google-cloud-sdk"
  return 1
}

function gcp_get_credentials() {
```

> **Note:** The `_run_command --soft` flag means the command is attempted but
> a non-zero exit does not abort the script — the subsequent `command -v gcloud`
> check decides whether to continue or fall through.

### 3. `scripts/tests/providers/k3s_gcp.bats` — add `_ensure_gcloud` stub + test

**In `setup()`**, add stub and export (alongside existing stubs):

```bash
  _ensure_gcloud() { :; }
  _command_exist() { command -v "$1" >/dev/null 2>&1; }
  export -f _ensure_gcloud _command_exist
```

**Add two new tests** at the end of the file:

```bash
@test "_ensure_gcloud returns 0 when gcloud is already installed" {
  gcloud() { :; }
  export -f gcloud
  run _ensure_gcloud
  [ "$status" -eq 0 ]
}

@test "_ensure_gcloud errors when gcloud missing and brew missing" {
  unset -f gcloud 2>/dev/null || true
  PATH_ORIG="$PATH"
  export PATH="/nonexistent"
  _command_exist() { return 1; }
  export -f _command_exist
  run _ensure_gcloud
  export PATH="$PATH_ORIG"
  [ "$status" -ne 0 ]
  [[ "$output" == *"install manually"* ]] || [[ "$stderr" == *"install manually"* ]]
}
```

> The second test overrides `PATH` and `_command_exist` so no real brew/curl is
> invoked. Restore `PATH` after the test to avoid polluting subsequent tests.
