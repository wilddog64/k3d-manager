## Background

`kubectl port-forward vault-0 18200:8200 -n secrets` must be running on the mac at all times
when the ACG cluster is live. It is the final hop in the ESO→Vault chain:

```
ESO pod → vault-bridge ClusterIP:8201 → socat on ubuntu node → autossh reverse tunnel
  → mac port 18200 → [this port-forward] → vault-0 pod:8200
```

When the port-forward dies (reconnect, mac sleep, crash), the ClusterSecretStore
`vault-backend` enters `InvalidProviderConfig` and all ExternalSecrets fail to sync.
Currently there is no persistence mechanism — it must be restarted manually.

The fix is a `KeepAlive` LaunchAgent, following the same pattern as
`com.k3d-manager.webhook-token-rotate.plist.tmpl` for template rendering and
`com.k3d-manager.argocd-port-forward.plist` for the port-forward pattern.

## Implementation

### File 1 — `scripts/etc/launchd/com.k3d-manager.vault-port-forward.plist.tmpl` (new)

Create this file with the following exact content:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.k3d-manager.vault-port-forward</string>
  <key>ProgramArguments</key>
  <array>
    <string>{{KUBECTL_PATH}}</string>
    <string>port-forward</string>
    <string>vault-0</string>
    <string>18200:8200</string>
    <string>-n</string>
    <string>secrets</string>
    <string>--context</string>
    <string>k3d-k3d-cluster</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>{{HOME}}/Library/Logs/k3dm-vault-port-forward.log</string>
  <key>StandardErrorPath</key>
  <string>{{HOME}}/Library/Logs/k3dm-vault-port-forward.log</string>
</dict>
</plist>
```

`{{KUBECTL_PATH}}` is needed because LaunchAgents do not inherit `$PATH`, so
`kubectl` must be an absolute path. Render it from `$(which kubectl)` in the Makefile.

### File 2 — `Makefile`

Add two new targets after the `install-token-rotator` block (after line 166):

#### Target: `install-vault-port-forward`

```makefile
## Install the Vault port-forward LaunchAgent — keeps kubectl port-forward vault-0 18200:8200 alive
install-vault-port-forward:
	sed \
	  -e "s|{{KUBECTL_PATH}}|$$(which kubectl)|g" \
	  -e "s|{{HOME}}|$(HOME)|g" \
	  scripts/etc/launchd/com.k3d-manager.vault-port-forward.plist.tmpl \
	  > "$(HOME)/Library/LaunchAgents/com.k3d-manager.vault-port-forward.plist"
	launchctl bootout "gui/$$(id -u)/com.k3d-manager.vault-port-forward" 2>/dev/null || true
	launchctl bootstrap "gui/$$(id -u)" \
	  "$(HOME)/Library/LaunchAgents/com.k3d-manager.vault-port-forward.plist"
	@echo "Vault port-forward agent installed — port 18200 will stay open while k3d-cluster is running"
```

#### Target: `uninstall-vault-port-forward`

```makefile
## Stop and remove the Vault port-forward LaunchAgent
uninstall-vault-port-forward:
	launchctl bootout "gui/$$(id -u)/com.k3d-manager.vault-port-forward" 2>/dev/null || true
	rm -f "$(HOME)/Library/LaunchAgents/com.k3d-manager.vault-port-forward.plist"
	@echo "Vault port-forward agent removed"
```
