# v1.0.2 — Fix `bin/acg-up` Full Stack Automation + Makefile

**Date**: 2026-04-03
**Branch**: k3d-manager-v1.0.2
**Status**: ASSIGNED to Codex

## Problem

`bin/acg-up` provisions the cluster but leaves 8 steps that must be run manually:
1. Vault port-forward (Mac → k3d Vault) is never started — reverse SSH tunnel has nothing to forward to
2. vault-bridge Service never applied — Endpoints exist but no ClusterIP DNS name
3. `argocd-manager` SA + static token Secret never bootstrapped on remote cluster
4. `helm` not installed on fresh EC2 nodes
5. ESO operator never installed on remote cluster
6. `vault-token` secret + `ClusterSecretStore` never applied on remote cluster
7. ArgoCD cluster registration never called (or uses wrong `kubectl create token` token type)
8. ArgoCD application controller never restarted after cluster secret update

There is also no `Makefile` — every command requires remembering the full `bin/` path.

## Files Changed

1. `bin/acg-up` — add 8 new steps (steps 4–11); renumber all step banners to `/12`
2. `bin/acg-down` — kill Vault port-forward PID on teardown
3. `Makefile` (NEW) — repo-root Makefile with `up`, `down`, `refresh`, `status`, `creds`, `help`

### 1c. Add Step 4 — Vault port-forward (insert after `tunnel_start` line, before Step 5/ghcr-pull-secret)

Insert this block after the `tunnel_start` line and before the ghcr-pull-secret block:

```bash
_info "[acg-up] Step 4/12 — Starting Vault port-forward (k3d → localhost:8200)..."
_vault_pf_pid_file="${HOME}/.local/share/k3d-manager/vault-pf.pid"
mkdir -p "$(dirname "${_vault_pf_pid_file}")"
# Kill any stale port-forward from a previous run
if [[ -f "${_vault_pf_pid_file}" ]]; then
  kill "$(cat "${_vault_pf_pid_file}")" 2>/dev/null || true
  rm -f "${_vault_pf_pid_file}"
fi
kubectl port-forward svc/vault -n secrets --context k3d-k3d-cluster 8200:8200 \
  >>"${HOME}/.local/share/k3d-manager/vault-pf.log" 2>&1 &
_vault_pf_pid=$!
disown "${_vault_pf_pid}"
echo "${_vault_pf_pid}" > "${_vault_pf_pid_file}"
sleep 3
_info "[acg-up] Vault port-forward PID: ${_vault_pf_pid} (log: ${HOME}/.local/share/k3d-manager/vault-pf.log)"
```

### 1f. Add Step 8 — Install helm + ESO on remote cluster

```bash
_info "[acg-up] Step 8/12 — Installing helm + ESO on remote cluster..."
_eso_version="${ESO_VERSION:-0.9.20}"
if ! ssh ubuntu command -v helm >/dev/null 2>&1; then
  _info "[acg-up] helm not found — installing..."
  ssh ubuntu "curl -fsSL https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash" >/dev/null
fi
if ! ssh ubuntu "sudo KUBECONFIG=/etc/rancher/k3s/k3s.yaml helm status external-secrets -n secrets >/dev/null 2>&1"; then
  _info "[acg-up] ESO not installed — installing v${_eso_version}..."
  ssh ubuntu "sudo KUBECONFIG=/etc/rancher/k3s/k3s.yaml helm repo add external-secrets https://charts.external-secrets.io >/dev/null 2>&1 && \
    sudo KUBECONFIG=/etc/rancher/k3s/k3s.yaml helm repo update >/dev/null 2>&1 && \
    sudo KUBECONFIG=/etc/rancher/k3s/k3s.yaml helm install external-secrets external-secrets/external-secrets \
      -n secrets --create-namespace --set installCRDs=true --version ${_eso_version}" >/dev/null
  _info "[acg-up] ESO v${_eso_version} installed"
else
  _info "[acg-up] ESO already installed — skipping"
fi
```

### 1g. Add Step 9 — Apply vault-token secret + ClusterSecretStore

```bash
_info "[acg-up] Step 9/12 — Applying vault-token secret + ClusterSecretStore on remote cluster..."
_vault_root_token=$(kubectl get secret vault-root -n secrets --context k3d-k3d-cluster \
  -o jsonpath='{.data.token}' | base64 -d)
if [[ -z "${_vault_root_token}" ]]; then
  _err "[acg-up] Could not read vault-root token from k3d-k3d-cluster — is Vault running?"
fi
kubectl create namespace secrets --context ubuntu-k3s \
  --dry-run=client -o yaml | kubectl apply --context ubuntu-k3s -f - >/dev/null
kubectl create secret generic vault-token \
  -n secrets --context ubuntu-k3s \
  --from-literal=token="${_vault_root_token}" \
  --dry-run=client -o yaml | kubectl apply --context ubuntu-k3s -f -
sleep 10
kubectl apply --context ubuntu-k3s -f - <<'CSSEOF'
apiVersion: external-secrets.io/v1beta1
kind: ClusterSecretStore
metadata:
  name: vault-backend
spec:
  provider:
    vault:
      server: "http://vault-bridge.secrets.svc.cluster.local:8201"
      path: "secret"
      version: "v2"
      auth:
        tokenSecretRef:
          name: vault-token
          namespace: secrets
          key: token
CSSEOF
_info "[acg-up] vault-token secret + ClusterSecretStore applied"
```

### 1h. Add Step 10 — Register ArgoCD app cluster

```bash
_info "[acg-up] Step 10/12 — Registering app cluster with ArgoCD..."
_argocd_sa_token=$(kubectl get secret argocd-manager-token -n kube-system --context ubuntu-k3s \
  -o jsonpath='{.data.token}' | base64 -d | tr -d '\n')
if [[ -z "${_argocd_sa_token}" ]]; then
  _err "[acg-up] argocd-manager-token secret not yet populated — wait and retry"
fi
kubectl create namespace cicd --context k3d-k3d-cluster \
  --dry-run=client -o yaml | kubectl apply --context k3d-k3d-cluster -f - >/dev/null
ARGOCD_APP_CLUSTER_TOKEN="${_argocd_sa_token}" \
  "${REPO_ROOT}/scripts/k3d-manager" register_app_cluster
_info "[acg-up] Restarting ArgoCD application controller to pick up new cluster secret..."
kubectl rollout restart statefulset/argocd-application-controller \
  -n cicd --context k3d-k3d-cluster
kubectl rollout status statefulset/argocd-application-controller \
  -n cicd --context k3d-k3d-cluster --timeout=90s
_info "[acg-up] ArgoCD cluster registered: ubuntu-k3s → https://host.k3d.internal:6443"
```

## Change 3 — `Makefile` (new file at repo root)

```makefile
# Makefile — k3d-manager ACG cluster lifecycle shortcuts
# Usage: make [target] [URL=https://...]

.DEFAULT_GOAL := help

URL ?= https://app.pluralsight.com/cloud-playground/cloud-sandboxes

.PHONY: up down refresh status creds help

## Provision full stack: credentials → cluster → ESO → ArgoCD
up:
	bin/acg-up "$(URL)"

## Tear down cluster and stop all background processes
down:
	bin/acg-down --confirm

## Refresh AWS credentials and restart tunnel (use when creds expire)
refresh:
	bin/acg-refresh "$(URL)"

## Show cluster nodes, pod status, tunnel health
status:
	bin/acg-status

## Extract AWS credentials only (no cluster changes)
creds:
	scripts/k3d-manager acg_get_credentials "$(URL)"

## Show this help
help:
	@echo ""
	@echo "  k3d-manager — ACG cluster lifecycle"
	@echo ""
	@echo "  Targets:"
	@echo "    make up        Provision full stack (credentials → cluster → ESO → ArgoCD)"
	@echo "    make down      Tear down cluster and stop all background processes"
	@echo "    make refresh   Refresh AWS credentials and restart tunnel"
	@echo "    make status    Show cluster nodes, pod status, tunnel health"
	@echo "    make creds     Extract AWS credentials only"
	@echo ""
	@echo "  Override sandbox URL (falls back to default if omitted):"
	@echo "    make up URL=https://app.pluralsight.com/hands-on/playground/cloud-sandboxes/..."
	@echo ""
	@echo "  Default URL: $(URL)"
	@echo ""
```
