## Background

`antigravity_acg_extend` currently navigates to the ACG sandbox page via CDP (port 9222)
but has no way to authenticate — if the ACG session has expired, it fails silently.

The fix is Playwright `storageState`: capture cookies + localStorage from a live logged-in
browser context and persist to `scratch/acg_session.json`. Subsequent automation runs inject
this state before navigation, bypassing the login wall.

**Antigravity is the headed browser.** Do NOT launch a new Playwright browser for state
capture. Connect via CDP (`chromium.connectOverCDP('http://localhost:9222')`) and call
`context.storageState({ path: ... })` on the existing context.

```bash
_ACG_SESSION_FILE="${HOME}/src/gitrepo/personal/k3d-manager/scratch/acg_session.json"
```

Use `SCRIPT_DIR` if available, or derive from the plugin location. The path must resolve
to `scratch/acg_session.json` inside the k3d-manager repo root.

### 2. `_acg_save_session` (private helper)

Generates a Playwright script via gemini that:
1. Connects to Antigravity via CDP
2. Uses the first browser context
3. Navigates to `https://learn.acloud.guru` and waits for load
4. Checks if logged in (look for user avatar or account menu elements)
5. If NOT logged in: prints `ACTION REQUIRED: Log into ACG in the Antigravity browser, then press Enter` and waits for the URL to no longer be a login page (poll every 5s, timeout 300s)
6. Once confirmed logged in: calls `context.storageState({ path: '<_ACG_SESSION_FILE>' })` to write the JSON
7. Prints `ACG_SESSION_SAVED` and exits 0
8. On any failure: exits 1

Writes the script to `/tmp/ag_acg_save_session.js`, executes with `node`.

Gemini prompt drives script generation — follow the same pattern as `_antigravity_ensure_github_session`.

### 3. `_acg_load_session` (private helper)

Returns the session file path if it exists and is non-empty. Prints a warning and returns 1
if the file is missing or empty (caller should run `acg_login` first).

```bash
function _acg_load_session() {
  if [[ ! -s "${_ACG_SESSION_FILE}" ]]; then
    _warn "ACG session file not found — run: ./scripts/k3d-manager acg_login"
    return 1
  fi
  printf '%s' "${_ACG_SESSION_FILE}"
}
```

### 4. `acg_login` (public function)

```bash
function acg_login() {
  _ensure_antigravity
  _antigravity_launch
  _acg_save_session
}
```

### 5. Update `antigravity_acg_extend` to load session

In the Gemini prompt inside `antigravity_acg_extend`, add session injection before navigation:

After step 1 (connect via CDP), add:
```
2. Load the saved session state: await context.addCookies(JSON.parse(require('fs').readFileSync('<session_file>', 'utf8')).cookies);
```

Where `<session_file>` is the value of `_ACG_SESSION_FILE`.

Call `_acg_load_session` before building the gemini prompt. If it returns 1, print an error
and return 1 (do not attempt automation without a session).
