# v0.6.2 Plan — Copilot CLI Tool Management

## Context

v0.6.1 merged. The v0.7.0 Keycloak provider interface plan invokes Copilot CLI (Step 6)
via a `copilot -p ...` shell command. Currently `copilot` must be pre-installed manually.

v0.6.2 lifts Copilot CLI onto the same auto-install pattern as `bats` and `cargo`
(`_ensure_bats`, `_ensure_cargo` in `scripts/lib/system.sh`), ensuring the agent
workflow is fully self-contained on any supported platform.

Branch: `feature/ensure-copilot-cli`
Target release: **v0.6.2**
Prerequisite for: **v0.7.0** (Step 6 uses `_ensure_copilot_cli` + `_run_command`)

## Background: What is `copilot`?

- Package: `@github/copilot` on the npm registry (current stable: `0.0.420`)
- Binary: a Node.js wrapper script — **requires Node.js to run**
- Install: `npm install -g @github/copilot`
- There is no standalone native binary; `brew install copilot` installs the unrelated AWS
  Copilot CLI

## Dependency Chain

```
_ensure_copilot_cli()
    └── _ensure_node()                        # prerequisite — node + npm
            ├── _command_exist node → return 0
            ├── macOS + brew  → brew install node
            ├── Debian        → apt-get install -y nodejs npm
            ├── RedHat        → dnf install -y nodejs npm
            └── _install_node_from_release()  # direct download fallback
                    └── https://nodejs.org/dist/vX.Y.Z/node-vX.Y.Z-<os>-<arch>.tar.gz
                        → extract to ~/.local, add ~/.local/bin to PATH
    └── npm install -g @github/copilot
```

`_ensure_node` is a first-class function — composable and reusable for future Node
tooling.

## Implementation Details

### `_install_node_from_release()` (direct download fallback)

Mirrors `_install_bats_from_source()`:
- Detects platform (`uname -s`) and arch (`uname -m`) to select correct tarball
- Downloads `node-v${NODE_LTS_VERSION}-<os>-<arch>.tar.gz` from `nodejs.org/dist`
- Extracts to `~/.local`
- `~/.local/bin/node` and `~/.local/bin/npm` become available

Version pin: `NODE_LTS_VERSION="${NODE_LTS_VERSION:-22.14.0}"` (LTS at time of writing)

### `_ensure_node()`

```
1. _command_exist node → return 0
2. macOS + brew  → _run_command -- brew install node
3. Debian family → _run_command --prefer-sudo -- apt-get install -y nodejs npm
4. RedHat family → _run_command --prefer-sudo -- dnf install -y nodejs npm
5. _install_node_from_release → return 0 if succeeds
6. exit 127 "Cannot install node: ..."
```

### `_ensure_copilot_cli()`

```
1. _command_exist copilot → return 0
2. _ensure_node                                    # guarantee node + npm
3. _run_command -- npm install -g @github/copilot
4. _command_exist copilot → return 0
5. exit 127 "Cannot install copilot CLI: ..."
```

Version: always installs `latest` tag; override via `COPILOT_CLI_VERSION` env var
(`npm install -g @github/copilot@${COPILOT_CLI_VERSION:-latest}`).

## Test Cases

### `ensure_node.bats` (mirrors `ensure_bats.bats` structure)

1. No-op when `node` already installed — no commands run
2. Installs via `brew install node` on macOS (brew available)
3. Installs via `apt-get install nodejs npm` on Debian
4. Installs via `dnf install nodejs npm` on RedHat
5. Falls back to `_install_node_from_release` when no package manager available

### `ensure_copilot_cli.bats` (stubs `_ensure_node` as a no-op)

1. No-op when `copilot` already installed — `_ensure_node` never called
2. Calls `_ensure_node` then `npm install -g @github/copilot` when `copilot` missing

## Security Considerations

**Do not pass live credentials to Copilot CLI via `--secret-env-vars`.**

A known prompt injection vulnerability allows malicious content in repository files (READMEs, docs)
to bypass Copilot's deny rules using shell indirection (`env curl -s URL | env sh`). Since `env`
is allowlisted, the regex validator does not flag the wrapped commands.

In our workflow, Copilot's role is **file generation only** — writing stub shell files and bats
tests. It requires no cluster access, no Vault token, and no kubeconfig. Credentials must never
be injected.

Reference: https://dev.to/matthewhou/github-copilot-cli-executes-malware-with-zero-approval-your-cicd-pipeline-would-have-caught-it-4g19
