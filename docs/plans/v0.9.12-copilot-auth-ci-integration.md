## Change 1 — `.github/workflows/ci.yml`

Add two new steps to the `lint` job **between** "Setup dependencies" and "Run shellcheck".

### Step A — Install Copilot CLI

Insert after the "Setup dependencies" step:

```yaml
      - name: Install Copilot CLI
        if: env.COPILOT_GITHUB_TOKEN != ''
        env:
          COPILOT_GITHUB_TOKEN: ${{ secrets.COPILOT_TOKEN }}
        shell: bash
        run: |
          set -euo pipefail
          tmp_dir="$(mktemp -d)"
          curl -fsSL https://gh.io/copilot-install -o "${tmp_dir}/copilot-install.sh"
          chmod +x "${tmp_dir}/copilot-install.sh"
          env VERSION=latest bash "${tmp_dir}/copilot-install.sh"
          rm -rf "$tmp_dir"
          echo "$HOME/.local/bin" >> "$GITHUB_PATH"
```

### Step B — Pass `COPILOT_GITHUB_TOKEN` + `K3DM_ENABLE_AI` to BATS

Modify the existing "Run lib unit BATS" step to add an `env:` block:

**Before:**
```yaml
      - name: Run lib unit BATS
        shell: bash
        run: |
          set -euo pipefail
          bats scripts/tests/lib
```

**After:**
```yaml
      - name: Run lib unit BATS
        shell: bash
        env:
          COPILOT_GITHUB_TOKEN: ${{ secrets.COPILOT_TOKEN }}
          K3DM_ENABLE_AI: "1"
        run: |
          set -euo pipefail
          bats scripts/tests/lib
```

## Change 2 — `scripts/tests/lib/ensure_copilot_cli.bats`

Append one new test at the end of the file (after the last `}`) — a real integration test
that calls `_copilot_auth_check` without stubbing `_run_command`:

```bash
@test "_copilot_auth_check passes with real COPILOT_GITHUB_TOKEN" {
  if [[ -z "${COPILOT_GITHUB_TOKEN:-}" ]]; then
    skip "COPILOT_GITHUB_TOKEN not set — skipping live auth check"
  fi
  if ! command -v copilot >/dev/null 2>&1; then
    skip "copilot binary not found — skipping live auth check"
  fi

  export K3DM_ENABLE_AI=1
  # Do NOT stub _run_command — use the real copilot binary
  run _copilot_auth_check
  [ "$status" -eq 0 ]
}
```

**Why two skip guards:**
- `COPILOT_GITHUB_TOKEN` not set → skip (local dev without the secret)
- `copilot` binary not found → skip (defensive; install step should put it in PATH, but belt-and-suspenders)

**Why no stubs:**
The existing 4 tests cover all stub-based paths. This test's sole purpose is to verify that
`copilot auth status` returns 0 when called with a real fine-grained PAT via `COPILOT_GITHUB_TOKEN`.
Stubbing `_run_command` would make it identical to test 4 (already exists).
