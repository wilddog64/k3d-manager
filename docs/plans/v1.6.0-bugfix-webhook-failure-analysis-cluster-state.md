# Bugfix: v1.6.0 — Enrich failure analysis with live cluster state

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/k3dm-webhook`

---

## Problem

`_analyze_failure` feeds Gemini only the last 50 lines of `make up` stdout. When the
failure is an application-layer problem (CrashLoopBackOff pods, Degraded ArgoCD apps),
those details never appear in the `make up` tail — they exist only in `kubectl` output.
Gemini receives no pod state and no ArgoCD health status, so it produces generic advice
("increase the synchronization timeout") instead of naming the failing pods and apps.

**Root cause:** `_analyze_failure` collects no live cluster state before building the
Gemini prompt.

---

## Fix

Add a `_collect_cluster_state()` helper that runs two `kubectl` queries immediately
before `_call_gemini`:

1. `kubectl get pods -A --context ubuntu-k3s` — filter for unhealthy pods
   (CrashLoopBackOff, Error, OOMKilled, ImagePullBackOff, ErrImagePull, Pending)
2. `kubectl get applications -A --context k3d-k3d-cluster` — get health + sync status
   for every ArgoCD Application

Both calls use a 15-second timeout and are silently skipped if the cluster is
unreachable (failure may be early in provisioning before nodes exist).

Inject the collected state as a new section in the Gemini prompt between the make output
and the diagnosis instructions.

### Change 1 — `bin/k3dm-webhook`: add `_collect_cluster_state` function

Insert this function immediately **before** `def _analyze_failure(`:

**Insert this block:**

```python
def _collect_cluster_state():
    """Collect live cluster state to enrich failure analysis. Returns a string or empty string."""
    sections = []

    try:
        result = subprocess.run(
            ["kubectl", "get", "pods", "-A", "--context", "ubuntu-k3s", "--no-headers"],
            capture_output=True, text=True, timeout=15,
        )
        if result.returncode == 0 and result.stdout.strip():
            bad = [
                ln for ln in result.stdout.splitlines()
                if any(s in ln for s in (
                    "CrashLoopBackOff", "Error", "OOMKilled",
                    "ImagePullBackOff", "ErrImagePull", "Pending",
                ))
                and "Completed" not in ln
            ]
            if bad:
                sections.append("Unhealthy pods (ubuntu-k3s):\n" + "\n".join(bad))
    except Exception:
        pass

    try:
        result = subprocess.run(
            [
                "kubectl", "get", "applications", "-A",
                "--context", "k3d-k3d-cluster",
                "-o", (
                    "custom-columns="
                    "NAME:.metadata.name,"
                    "HEALTH:.status.health.status,"
                    "SYNC:.status.sync.status"
                ),
                "--no-headers",
            ],
            capture_output=True, text=True, timeout=15,
        )
        if result.returncode == 0 and result.stdout.strip():
            sections.append(
                "ArgoCD application state (k3d-k3d-cluster):\n" + result.stdout.strip()
            )
    except Exception:
        pass

    return "\n\n".join(sections)
```

    last_output = "\n".join(last_50)
    cluster_state = _collect_cluster_state()
    cluster_section = (
        f"Live cluster state at time of failure:\n\n{cluster_state}\n\n"
        if cluster_state else ""
    )
    prompt = (
        "acg-up is a multi-step script that provisions a 3-node k3s cluster on AWS "
        "(via CloudFormation), installs ArgoCD on a local k3d hub cluster, registers "
        "the remote k3s cluster with ArgoCD, then deploys a shopping-cart application "

        f"Last 50 lines of output:\n\n{last_output}\n\n"
        f"{cluster_section}"
        "Diagnose the failure:\n"
        "1. TRANSIENT or PERMANENT?\n"
        "   TRANSIENT = a retry of acg-up is likely to fix it (e.g. ArgoCD sync lag, "
        "pod not ready yet, API server briefly unreachable).\n"
        "   PERMANENT = human action is required before a retry will work (e.g. "
