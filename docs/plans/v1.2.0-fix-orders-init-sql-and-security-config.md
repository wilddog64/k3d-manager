## Fix 1 â€” shopping-cart-infra: orders init SQL

**File:** `data-layer/postgresql/orders/configmap.yaml`

Replace the entire `01-init-schema.sql` value. The current SQL creates `SERIAL` primary keys,
but the `Order` and `OrderItem` entities use `@GeneratedValue(strategy = GenerationType.UUID)`.
Hibernate cannot automatically cast existing SERIAL columns to UUID, causing DDL failures on
every fresh cluster.

The new SQL must:
- Use `UUID DEFAULT gen_random_uuid()` for all `id` columns
- Use `UUID` (not `INTEGER`) for `order_id` foreign key in `order_items`
- Match the entity field names exactly: `customer_id`, `total_amount`, `currency`, `created_at`, `updated_at`
- Remove all sample `INSERT` statements (they reference integer IDs)
- Remove all functions that reference `orders_id_seq`
- Remove views that reference the old schema

### Exact old content (key lines â€” do NOT use SERIAL anywhere):
```yaml
  01-init-schema.sql: |
    -- Orders Database Schema
    ...
    CREATE TABLE IF NOT EXISTS orders (
        id SERIAL PRIMARY KEY,
        order_number VARCHAR(50) UNIQUE NOT NULL,
        user_id INTEGER NOT NULL,
        ...
    );

    CREATE TABLE IF NOT EXISTS order_items (
        id SERIAL PRIMARY KEY,
        order_id INTEGER NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
        ...
    );
```

### Exact new content (replace the entire `01-init-schema.sql` value with this):
```yaml
  01-init-schema.sql: |
    -- Orders Database Schema
    -- Aligned with Order and OrderItem JPA entities (UUID primary keys)

    CREATE EXTENSION IF NOT EXISTS "pgcrypto";

    CREATE TABLE IF NOT EXISTS orders (
        id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
        customer_id VARCHAR(255) NOT NULL,
        status VARCHAR(50) NOT NULL DEFAULT 'PENDING',
        total_amount NUMERIC(10, 2) NOT NULL,
        currency VARCHAR(3) NOT NULL DEFAULT 'USD',
        created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
    );

    CREATE TABLE IF NOT EXISTS order_items (
        id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
        order_id UUID NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
        product_id VARCHAR(255) NOT NULL,
        product_name VARCHAR(255) NOT NULL,
        quantity INTEGER NOT NULL CHECK (quantity > 0),
        unit_price NUMERIC(10, 2) NOT NULL CHECK (unit_price >= 0),
        total_price NUMERIC(10, 2) NOT NULL CHECK (total_price >= 0)
    );

    CREATE INDEX IF NOT EXISTS idx_orders_customer_id ON orders(customer_id);
    CREATE INDEX IF NOT EXISTS idx_orders_status ON orders(status);
    CREATE INDEX IF NOT EXISTS idx_orders_created_at ON orders(created_at DESC);
    CREATE INDEX IF NOT EXISTS idx_order_items_order_id ON order_items(order_id);

    -- Create function to update updated_at timestamp
    CREATE OR REPLACE FUNCTION update_updated_at_column()
    RETURNS TRIGGER AS $$
    BEGIN
        NEW.updated_at = CURRENT_TIMESTAMP;
        RETURN NEW;
    END;
    $$ LANGUAGE plpgsql;

    DROP TRIGGER IF EXISTS update_orders_updated_at ON orders;
    CREATE TRIGGER update_orders_updated_at
        BEFORE UPDATE ON orders
        FOR EACH ROW
        EXECUTE FUNCTION update_updated_at_column();
```

Everything else in the YAML file (metadata, labels) stays unchanged. Only `data.01-init-schema.sql` changes.

### Commit (shopping-cart-infra):
```
fix(postgresql): align orders init SQL to entity model â€” UUID primary keys

orders and order_items tables used SERIAL PRIMARY KEY but Order/OrderItem
entities use @GeneratedValue(strategy = GenerationType.UUID). Hibernate
ddl-auto=validate fails on fresh clusters; ddl-auto=update logs WARNINGs
and cannot cast existing SERIAL rows to UUID. Replace SERIAL with UUID
DEFAULT gen_random_uuid(); remove sample INSERTs and stale functions.
```

## Fix 2 â€” shopping-cart-order: SecurityConfig health probe gap

**File:** `src/main/java/com/shoppingcart/order/config/SecurityConfig.java`

The Kubernetes readiness and liveness probes hit `/actuator/health/readiness` and
`/actuator/health/liveness`. The current `permitAll()` only covers `/actuator/health`
(exact match), causing probes to get `401 Unauthorized` and the pod to stay `0/1`.

### Exact old line (line 67):
```java
                .requestMatchers("/actuator/health", "/actuator/info", "/actuator/prometheus").permitAll()
```

### Exact new line:
```java
                .requestMatchers("/actuator/health", "/actuator/health/**", "/actuator/info", "/actuator/prometheus").permitAll()
```

One line change. No other files in `shopping-cart-order` are touched.

### Commit (shopping-cart-order):
```
fix(security): permit /actuator/health/** for Kubernetes probe access

SecurityConfig only permitted /actuator/health (exact match), causing
/actuator/health/readiness and /actuator/health/liveness to return 401.
Kubernetes probes cannot pass authentication, so pods stayed 0/1 Ready.
```
