# feat: v1.6.5 — wire k3s-azure into bin/acg-up and bin/acg-down

**Branch:** `k3d-manager-v1.6.5`
**Files:** `bin/acg-up`, `bin/acg-down`

---

## Problem

`bin/acg-up` and `bin/acg-down` do not handle `CLUSTER_PROVIDER=k3s-azure`.
Running `bin/acg-up` with that env var hits the `*` guard and exits with an error.
The Slack webhook already routes `/k3dm up azure` → `CLUSTER_PROVIDER=k3s-azure`,
so the provider is unreachable from both terminal and Slack.

---

## Fix

Four targeted edits to `bin/acg-up` and one to `bin/acg-down`.
No other files touched.

---

### Change 1 — `bin/acg-up`: add k3s-azure to provider source block (~line 60)

**Exact old block:**

```bash
_cluster_provider="${CLUSTER_PROVIDER:-k3s-aws}"
case "${_cluster_provider}" in

esac
```

**Exact new block:**


### Change 5 — `bin/acg-down`: add k3s-azure teardown case (~line 65)

**Exact old block:**

```bash
  k3s-gcp)
    source "${REPO_ROOT}/scripts/lib/providers/k3s-gcp.sh"
    _info "[acg-down] Tearing down GCP cluster..."
    if [[ -z "${GCP_PROJECT:-}" ]]; then
      _gcp_sa_json="${HOME}/.local/share/k3d-manager/gcp-service-account.json"
      if [[ -f "${_gcp_sa_json}" ]]; then
        GCP_PROJECT="$(python3 -c "import json; print(json.load(open('${_gcp_sa_json}')).get('project_id',''))" 2>/dev/null || true)"
        export GCP_PROJECT
      fi
    fi
    _provider_k3s_gcp_destroy_cluster --confirm
    ;;
  *)
    _info "[acg-down] Unknown CLUSTER_PROVIDER '${_cluster_provider}' — skipping remote teardown"
    ;;
esac
```

**Exact new block:**

```bash
  k3s-gcp)
    source "${REPO_ROOT}/scripts/lib/providers/k3s-gcp.sh"
    _info "[acg-down] Tearing down GCP cluster..."
    if [[ -z "${GCP_PROJECT:-}" ]]; then
      _gcp_sa_json="${HOME}/.local/share/k3d-manager/gcp-service-account.json"
      if [[ -f "${_gcp_sa_json}" ]]; then
        GCP_PROJECT="$(python3 -c "import json; print(json.load(open('${_gcp_sa_json}')).get('project_id',''))" 2>/dev/null || true)"
        export GCP_PROJECT
      fi
    fi
    _provider_k3s_gcp_destroy_cluster --confirm
    ;;
  k3s-azure)
    source "${REPO_ROOT}/scripts/plugins/azure.sh"
    source "${REPO_ROOT}/scripts/lib/providers/k3s-azure.sh"
    _info "[acg-down] Tearing down Azure cluster..."
    _provider_k3s_azure_destroy_cluster --confirm
    ;;
  *)
    _info "[acg-down] Unknown CLUSTER_PROVIDER '${_cluster_provider}' — skipping remote teardown"
    ;;
esac
```
