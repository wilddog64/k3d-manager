# Plan: Vault-managed TLS listener for Argo CD browser SSO

Status: Implemented

Implemented in commit `7e85830e`.

## Goal

Replace the current raw TCP browser bridge with a real TLS listener on
`https://argocd.shopping-cart.local` so Safari can reach the canonical Argo CD
SSO entrypoint without redirect or scheme mismatches.

## Problem Statement

The browser URL is currently backed by a plain TCP forward to the Argo CD
`localhost:8080` port-forward. That makes the hostname reachable only if the
client is already speaking TLS correctly, which Safari is not when the listener
is just relaying bytes.

## Desired Behavior

- `bin/acg-up` provisions a local HTTPS listener for `argocd.shopping-cart.local:443`.
- The listener presents a Vault PKI-issued certificate for
  `argocd.shopping-cart.local`.
- The listener terminates TLS locally and proxies cleartext HTTP to the existing
  Argo CD port-forward on `localhost:8080`.
- `bin/acg-down` tears the listener down and removes the generated cert material.
- The browser SSO flow continues to use the canonical hostname, while terminal
  smoke tests can still use the `localhost:8080` port-forward directly.

## Scope

- Add a Vault-backed cert issuance helper for the browser listener.
- Replace the current raw TCP browser bridge with TLS termination via `socat`
  `OPENSSL-LISTEN`.
- Update bootstrap/teardown scripts and docs.
- Add regression tests for the new TLS listener generation and teardown.

## Non-Goals

- Do not change the Argo CD login flow itself.
- Do not move the browser listener away from `launchd`.
- Do not change the existing `localhost:8080` terminal login path.

## Acceptance Criteria

- Safari can open `https://argocd.shopping-cart.local` without the address
  invalid error.
- The listener uses Vault-issued material, not an ad hoc self-signed cert.
- `bin/acg-down` removes the listener and generated files cleanly.
- BATS and shellcheck stay green.
