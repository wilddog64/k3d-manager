# Spec: v1.6.3 ‚Äî Gemini side-observation bug filing

**Branch:** `k3d-manager-v1.6.3`
**Files:** `Makefile`, `bin/k3dm-ask-bash`, `bin/k3dm-webhook`

---

## Problem

When Gemini answers an `/ask` question it sometimes notices real issues adjacent to the
question asked (e.g. a wrong DNS fix recommendation, a misconfigured ArgoCD cluster
secret). Currently the system prompt has no way for Gemini to surface those observations
without dumping them into the Slack reply as unsolicited diagnosis ‚Äî which looks like
scope creep and creates noise.

**Goal:** Give Gemini a sanctioned channel for side observations:
- Answer the question asked (focused, concise Slack reply)
- If Gemini notices something else, emit it in a structured `OBSERVATIONS:` section
- Webhook parses that section, writes one `docs/bugs/` file per observation, strips the
  raw observations from the Slack reply, and appends a one-line "Also filed: ‚Ä¶" note

No cluster mutations. No bash execution required on Gemini's side ‚Äî the webhook does
the filing.

A `make file-bug` Makefile target is also added so Claude (which has bash access via
`k3dm-ask-bash`) can self-file observations using the same format.

---

## Design

**Gemini response format (new):**

```
ANSWER:
<answer to the question>

OBSERVATIONS:
- TITLE: <short title> | BODY: <one paragraph>
- TITLE: <short title> | BODY: <one paragraph>
```

`OBSERVATIONS:` block is optional ‚Äî Gemini omits it when there is nothing to report.

**Webhook parsing:**
1. Split Gemini's raw response on `\nOBSERVATIONS:\n`
2. Left side ‚Üí Slack reply (strip the `ANSWER:` prefix if present)
3. Right side ‚Üí parse each `- TITLE: ‚Ä¶ | BODY: ‚Ä¶` line ‚Üí write bug doc ‚Üí append
   "Also filed: `docs/bugs/<filename>`" to the Slack reply

**Bug doc format** (written by both `make file-bug` and the webhook parser):

```markdown
# Bug: <title>

**Filed:** YYYY-MM-DD
**Source:** /ask agent observation

## Description

<body>
```

def _parse_gemini_observations(raw):
    """Split Gemini's structured response into (answer_text, [filed_paths]).

    Expected format:
        ANSWER:
        <answer text>

        OBSERVATIONS:
        - TITLE: <title> | BODY: <body>

    If the format is not present, returns (raw, []) unchanged.
    """
    obs_marker = "\nOBSERVATIONS:\n"
    if obs_marker not in raw:
        answer = raw.removeprefix("ANSWER:\n").strip()
        return answer, []

    answer_part, obs_part = raw.split(obs_marker, 1)
    answer = answer_part.removeprefix("ANSWER:\n").strip()

    filed = []
    for line in obs_part.splitlines():
        line = line.strip().lstrip("- ")
        if not line.startswith("TITLE:"):
            continue
        try:
            title_part, body_part = line.split("| BODY:", 1)
        except ValueError:
            continue
        title = title_part.removeprefix("TITLE:").strip()
        body = body_part.strip()
        if not title or not body:
            continue
        slug = re.sub(r"[^a-z0-9-]", "", title.lower().replace(" ", "-"))
        today = __import__("datetime").date.today().isoformat()
        fname = Path(REPO_ROOT) / "docs" / "bugs" / f"{today}-{slug}.md"
        try:
            fname.parent.mkdir(parents=True, exist_ok=True)
            fname.write_text(
                f"# Bug: {title}\n\n"
                f"**Filed:** {today}\n"
                f"**Source:** /ask agent observation\n\n"
                f"## Description\n\n{body}\n"
            )
            filed.append(str(fname.relative_to(REPO_ROOT)))
        except OSError:
            pass
    return answer, filed
