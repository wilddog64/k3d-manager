# Spec: v1.4.3 — Keycloak Frontend Login (LDAP + Vault ESO)

**Branch (k3d-manager):** `k3d-manager-v1.4.3`
**Branch (shopping-cart-infra):** `feat/keycloak-vault-eso` — create from `docs/next-improvements`

Enable the shopping-cart-frontend to log in via Keycloak. Three gaps to close:
1. `frontend` public OIDC client is missing from the Keycloak realm
2. Keycloak credentials are static `CHANGE_ME` placeholders — migrate to Vault + ESO
3. LDAP sample-user password hashes are invalid placeholders — replace with real SSHA hashes

## Change 3 — NEW `identity/keycloak/keycloak-secrets-externalsecret.yaml`

Create this file with the exact content below:

```yaml
---
apiVersion: external-secrets.io/v1
kind: ExternalSecret
metadata:
  name: keycloak-secrets
  namespace: identity
  annotations:
    argocd.argoproj.io/sync-wave: "0"
  labels:
    app.kubernetes.io/name: external-secret
    app.kubernetes.io/instance: keycloak-secrets
    app.kubernetes.io/component: identity-credentials
    app.kubernetes.io/part-of: shopping-cart
spec:
  refreshInterval: 24h
  secretStoreRef:
    name: vault-backend
    kind: ClusterSecretStore
  target:
    name: keycloak-secrets
    creationPolicy: Owner
    template:
      data:
        KEYCLOAK_ADMIN: "admin"
        KC_DB_USERNAME: "keycloak"
  data:
    - secretKey: KEYCLOAK_ADMIN_PASSWORD
      remoteRef:
        key: secret/data/keycloak/admin
        property: admin_password
    - secretKey: KC_DB_PASSWORD
      remoteRef:
        key: secret/data/keycloak/admin
        property: db_password
    - secretKey: LDAP_BIND_CREDENTIAL
      remoteRef:
        key: secret/data/keycloak/admin
        property: ldap_bind_credential
```

## Change 8 — `bin/acg-up` (k3d-manager): Vault KV seeding for Keycloak

**Exact old block (line 458–459):**

```bash
_vault_kv_put "{\"admin_password\":\"${_ldap_admin_pass}\",\"readonly_password\":\"${_ldap_readonly_pass}\"}" ldap/admin
_info "[acg-up] Vault KV seeded (redis, postgres, payment, rabbitmq, ldap sandbox secrets)"
```

**Exact new block:**

```bash
_vault_kv_put "{\"admin_password\":\"${_ldap_admin_pass}\",\"readonly_password\":\"${_ldap_readonly_pass}\"}" ldap/admin
_kc_admin_pass=$(openssl rand -base64 24 | tr -d '=+/')
_kc_db_pass=$(openssl rand -base64 24 | tr -d '=+/')
_argocd_client_secret=$(openssl rand -base64 24 | tr -d '=+/')
_order_client_secret=$(openssl rand -base64 24 | tr -d '=+/')
_product_client_secret=$(openssl rand -base64 24 | tr -d '=+/')
_grafana_client_secret=$(openssl rand -base64 24 | tr -d '=+/')
_vault_kv_put "{\"admin_password\":\"${_kc_admin_pass}\",\"db_password\":\"${_kc_db_pass}\",\"ldap_bind_credential\":\"${_ldap_admin_pass}\"}" keycloak/admin
_vault_kv_put "{\"argocd_client_secret\":\"${_argocd_client_secret}\",\"order_service_client_secret\":\"${_order_client_secret}\",\"product_catalog_client_secret\":\"${_product_client_secret}\",\"grafana_client_secret\":\"${_grafana_client_secret}\"}" keycloak/clients
_info "[acg-up] Vault KV seeded (redis, postgres, payment, rabbitmq, ldap, keycloak sandbox secrets)"
```

Note: `ldap_bind_credential` intentionally reuses `_ldap_admin_pass` — Keycloak's LDAP bind password
must match the LDAP admin password already seeded at `secret/data/ldap/admin`.
