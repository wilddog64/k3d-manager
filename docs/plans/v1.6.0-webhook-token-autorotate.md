# Feature: v1.6.0 — Auto-rotate webhook bearer token every 6 hours

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/rotate-webhook-token`, `scripts/etc/com.k3d-manager.webhook-token-rotate.plist`, `Makefile`

---

## Problem

The `k3dm-webhook-token` Keychain entry is a static bearer token that never rotates.
If the token leaks (e.g. via shell history, log scraping, or Cloudflare Worker secret
exposure), an attacker can call the webhook indefinitely.

**Goal:** Rotate the token automatically every 6 hours — update Keychain, push the new
secret to the Cloudflare Worker, restart the webhook daemon — with no manual intervention.

---

## Design

The webhook reads its token from Keychain at startup (`security find-generic-password
-s k3dm-webhook-token -a k3dm -w`). Rotation therefore requires a process restart to
pick up the new value.

The Cloudflare Worker authenticates incoming Slack relay requests using `WEBHOOK_TOKEN`
(set via `wrangler secret put`). Rotation must update the Worker secret **before**
restarting the webhook so there is no window where the two sides disagree.

Rotation sequence:
1. Generate a new 32-byte hex token (`openssl rand -hex 32`)
2. Update Keychain entry
3. Push new token to Cloudflare Worker secret (secret-only — no full `wrangler deploy`)
4. Restart webhook daemon (`make restart-webhook`)

A LaunchAgent timer fires the rotate script every 6 hours.

## Change 1 — `bin/rotate-webhook-token`: new rotation script

Create this file (mode 755):

```bash
#!/usr/bin/env bash
set -euo pipefail

REPO_ROOT="$(cd "$(dirname "$0")/.." && pwd)"

_log() { echo "[rotate-webhook-token] $*"; }
_err() { echo "[rotate-webhook-token] ERROR: $*" >&2; exit 1; }

NEW_TOKEN="$(openssl rand -hex 32)"

_log "Updating Keychain entry k3dm-webhook-token..."
security add-generic-password -U -s k3dm-webhook-token -a k3dm -w "${NEW_TOKEN}" \
  || _err "Failed to update Keychain"

_log "Reading Cloudflare API token from Keychain..."
CF_TOKEN="$(security find-generic-password -s cloudflare-api-token -a k3dm -w 2>/dev/null)" \
  || _err "cloudflare-api-token not found in Keychain — run make setup-worker first"

_log "Pushing new WEBHOOK_TOKEN to Cloudflare Worker..."
printf '%s' "${NEW_TOKEN}" | \
  CLOUDFLARE_API_TOKEN="${CF_TOKEN}" \
  npx --yes wrangler secret put WEBHOOK_TOKEN \
    --cwd "${REPO_ROOT}/workers/slack-relay" \
  || _err "Failed to push WEBHOOK_TOKEN to Cloudflare"

_log "Restarting webhook daemon..."
make -C "${REPO_ROOT}" restart-webhook \
  || _err "Failed to restart webhook"

_log "Token rotation complete."
```

## Change 3 — `Makefile`: add `rotate-webhook-token` and `install-token-rotator` targets

Find the `restart-webhook` target block and add these two targets immediately after it.

**Exact old block (restart-webhook target — find by searching for `restart-webhook:`):**

```makefile
restart-webhook:
	launchctl bootout "gui/$(id -u)/com.k3d-manager.webhook" 2>/dev/null || true
	launchctl bootstrap "gui/$(id -u)" "/Users/cliang/Library/LaunchAgents/com.k3d-manager.webhook.plist"
```

**Exact new block:**

```makefile
restart-webhook:
	launchctl bootout "gui/$(id -u)/com.k3d-manager.webhook" 2>/dev/null || true
	launchctl bootstrap "gui/$(id -u)" "/Users/cliang/Library/LaunchAgents/com.k3d-manager.webhook.plist"

## Rotate webhook bearer token now (updates Keychain + Cloudflare Worker secret + restarts webhook)
rotate-webhook-token:
	bin/rotate-webhook-token

## Install the 6-hour token rotation LaunchAgent (run once; safe to re-run)
install-token-rotator:
	cp scripts/etc/com.k3d-manager.webhook-token-rotate.plist \
	  "$(HOME)/Library/LaunchAgents/com.k3d-manager.webhook-token-rotate.plist"
	launchctl bootout "gui/$$(id -u)/com.k3d-manager.webhook-token-rotate" 2>/dev/null || true
	launchctl bootstrap "gui/$$(id -u)" \
	  "$(HOME)/Library/LaunchAgents/com.k3d-manager.webhook-token-rotate.plist"
	@echo "Token rotator installed — fires every 6 hours"
```
