## Problem

When the ArgoCD cluster secret goes stale (e.g. after sandbox recreation), re-registration
requires knowing to call `register_app_cluster` with `ARGOCD_APP_CLUSTER_TOKEN` set correctly.
There is no Makefile shortcut — users must know the internal function name and how to extract
the token. `bin/acg-up` already handles this inline in Step 10, but there is no standalone
target for re-running registration without reprovisioning the full stack.

## Files Changed

1. `Makefile` — add `register-apps` and `register-apps-stop` targets + `.PHONY` entries + help text

### Step 2 — Add target after `chrome-cdp-stop`

```makefile
## Re-register ubuntu-k3s app cluster with ArgoCD (use after sandbox recreation)
register-apps:
	@_token=$$(kubectl get secret argocd-manager-token -n kube-system --context ubuntu-k3s \
	  -o jsonpath='{.data.token}' 2>/dev/null | base64 -d | tr -d '\n'); \
	if [[ -z "$$_token" ]]; then \
	  echo "ERROR: argocd-manager-token not found on ubuntu-k3s — is the cluster up?"; \
	  exit 1; \
	fi; \
	ARGOCD_APP_CLUSTER_TOKEN="$$_token" scripts/k3d-manager register_app_cluster; \
	kubectl rollout restart statefulset/argocd-application-controller -n cicd --context k3d-k3d-cluster; \
	kubectl rollout status statefulset/argocd-application-controller -n cicd --context k3d-k3d-cluster --timeout=90s
```
