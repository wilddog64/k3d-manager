## Problem

After `make up` + `make sync-apps`, ArgoCD data-layer sync fails with:

```
The Kubernetes API could not find version "v1" of external-secrets.io/ExternalSecret.
Version "v1beta1" of external-secrets.io/ExternalSecret is installed on the destination cluster.
```

**Root cause:** ESO v0.14.0 only serves `v1alpha1` and `v1beta1`. The `v1` GA API was
introduced in ESO **1.0.0**. The shopping-cart-infra manifests use `external-secrets.io/v1`,
so they require ESO 1.0.0+. The previous bump from `0.9.20` to `0.14.0` was insufficient.
