# Plan: Robust ACG Extend Script (v1.0.4)

**Objective:** Improve `scripts/playwright/acg_extend.js` to handle early runs gracefully by parsing the sandbox TTL (Time To Live). This prevents the `acg_watch` loop from reporting false failures and shutting down the watcher when the "Extend" button is naturally absent because more than 1 hour remains.

**Target File:** `scripts/playwright/acg_extend.js`

## Problem Context
The current script fails with "Extend button not found" if it runs before the final hour of the sandbox TTL. Because Pluralsight hides the "Extend Session" button until < 1h remains, the `acg_watch` loop (running every 3.5h) treats this missing button as a fatal error, triggering an "Instance gone" chain reaction that stops the watcher prematurely.

## Implementation Steps

1.  **Standardize Target URL:**
    - Standardized navigation to `https://app.pluralsight.com/hands-on/playground/cloud-sandboxes` to match the exact URL where the cards are rendered, minimizing SPA redirects.

2.  **Add CDP Session Reuse:**
    - Implemented `chromium.connectOverCDP('http://localhost:9222')` as the primary connection method. This allows the script to attach to the user's active Chrome instance, reuse existing login sessions, and "see" already-open sandbox panels or modals.

3.  **Refine Sandbox Interaction:**
    - Added a check to skip the "Open Sandbox" click if the details panel (containing TTL info) is already visible.
    - Used `{ force: true }` for the "Open Sandbox" click to bypass any invisible SPA overlays that might intercept pointer events.

4.  **Add TTL Parsing & Graceful Exit Logic:**
    - Used a broad text-based locator (`text=/Auto Shutdown/i`) to find the TTL information.
    - Extracted text from the parent element to ensure the full time string (e.g., "Auto Shutdown at 10:19PM") is captured even if split across child tags.
    - Parsed the extracted time and compared it to the current local time to calculate the remaining TTL.
    - **Logic Check before Button Search:**
        - If remaining time is > 65 minutes: Do *not* search for the button. Exit `0` with a message: `INFO: Extension window not open yet (XXm remaining)`.
        - If remaining time is <= 65 minutes: Proceed to search for the "Extend Session" button using robust selectors.

5.  **Refine Button Selectors:**
    - Prioritized selectors identified from the Pluralsight React bundle and Pando design system:
      - `[data-heap-id="Hands-on Playground - Click - AWS Sandbox - Extend Sandbox"]`
      - `button:has-text("Extend Session")`
      - `[id="extend-sandbox"] button`

## Definition of Done
- [x] `acg_extend.js` successfully parses the Auto Shutdown time.
- [x] Script exits `0` (success) when run against a fresh sandbox (>1h remaining) without throwing an error.
- [x] Script successfully finds and clicks the "Extend Session" button when run against an expiring sandbox (<1h remaining).
- [x] Changes are committed to the `k3d-manager-v1.0.4` branch.
