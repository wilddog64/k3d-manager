# v0.9.6 ŌĆö Lab Accessibility: LoadBalancer Services

**Branch:** `k3d-manager-v0.9.6`
**Repos:** `k3d-manager`, `shopping-cart-frontend`
**Milestone gate:** All UI services reachable via OrbStack-assigned LoadBalancer IP ŌĆö no port-forward required

---

## Context

All infra cluster UI services (`argocd-server`, `keycloak`, `jenkins`) are deployed with
`type: ClusterIP`. Accessing them requires running `kubectl port-forward` each session.
OrbStack natively supports `LoadBalancer` services and assigns stable LAN IPs, making
this a zero-infrastructure change ŌĆö just update the Helm values templates and k8s manifests.

LDAP is excluded ŌĆö it is a protocol service (no browser UI); ClusterIP is appropriate.
Data-layer services (PostgreSQL, Redis, RabbitMQ) are excluded ŌĆö internal only, no direct
access needed.

## Notes

- Changes to `values.yaml.tmpl` take effect on next `helm upgrade` or cluster rebuild ŌĆö
  existing `patch` changes are ephemeral and not needed after this
- Jenkins chart uses `controller.serviceType` (not `service.type`) ŌĆö verify with
  `helm show values jenkins/jenkins | grep serviceType` if unsure
- OrbStack LoadBalancer IPs are LAN-stable but not guaranteed across OrbStack version
  upgrades ŌĆö document the current IPs in `memory-bank/activeContext.md` after first assignment
- `shopping-cart-frontend` change requires a PR on that repo ŌĆö branch protection applies
