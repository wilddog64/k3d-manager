## Problem

`acg_import_credentials` parses AWS credentials from stdin. The three perl regex lines
use `\S+` to capture values. When the AWS Console "Command line or programmatic access"
panel outputs **quoted** values, the quotes are included in the capture:

```
export AWS_ACCESS_KEY_ID="AKIAIOSFODNN7EXAMPLE"
```

`\S+` captures `"AKIAIOSFODNN7EXAMPLE"` (with quotes). Writing that value to
`~/.aws/credentials` breaks `aws sts get-caller-identity` with an authentication error.

---

## Supported Formats After Fix

All four formats must work:

| Format | Source |
|--------|--------|
| `AWS Access Key ID: ASIA...` | Pluralsight Cloud Access panel |
| `export AWS_ACCESS_KEY_ID=ASIA...` | AWS Console unquoted export |
| `export AWS_ACCESS_KEY_ID="ASIA..."` | AWS Console quoted export (IAM Identity Center) |
| `aws_access_key_id=ASIA...` | AWS credentials file section |

## Changes

### Change 1 — `scripts/plugins/acg.sh`: strip quotes from captured values (lines 230–232)

**Exact old block:**

```bash
  access_key=$(printf '%s' "$input" | perl -ne 'if (/AWS(?:_ACCESS_KEY_ID| Access Key ID)[\s:=]+(\S+)/i) {print $1; exit}')
  secret_key=$(printf '%s' "$input" | perl -ne 'if (/AWS(?:_SECRET_ACCESS_KEY| Secret Access Key)[\s:=]+(\S+)/i) {print $1; exit}')
  session_token=$(printf '%s' "$input" | perl -ne 'if (/AWS(?:_SESSION_TOKEN| Session Token)[\s:=]+(\S+)/i) {print $1; exit}')
```

**Exact new block:**

```bash
  access_key=$(printf '%s' "$input" | perl -ne 's/["\x27]//g; if (/AWS(?:_ACCESS_KEY_ID| Access Key ID)[\s:=]+(\S+)/i) {print $1; exit}')
  secret_key=$(printf '%s' "$input" | perl -ne 's/["\x27]//g; if (/AWS(?:_SECRET_ACCESS_KEY| Secret Access Key)[\s:=]+(\S+)/i) {print $1; exit}')
  session_token=$(printf '%s' "$input" | perl -ne 's/["\x27]//g; if (/AWS(?:_SESSION_TOKEN| Session Token)[\s:=]+(\S+)/i) {print $1; exit}')
```

The `s/["\x27]//g` strips double-quotes and single-quotes from each line before matching.
`\x27` is the hex escape for single-quote, avoiding shell quoting conflicts inside the
single-quoted perl string.

### Change 2 — `scripts/plugins/acg.sh`: update help text (lines 206–222)

**Exact old block:**

```bash
    cat <<'HELP'
Usage: pbpaste | acg_import_credentials
       acg_import_credentials < credentials.txt

Read AWS credentials block from stdin and write to ~/.aws/credentials.
Expected input format (copy from Pluralsight Cloud Access panel):

  AWS Access Key ID: ASIA...
  AWS Secret Access Key: abc123...
  AWS Session Token: IQo...

Or the export block format:

  export AWS_ACCESS_KEY_ID=ASIA...
  export AWS_SECRET_ACCESS_KEY=abc123...
  export AWS_SESSION_TOKEN=IQo...
HELP
```

**Exact new block:**

```bash
    cat <<'HELP'
Usage: pbpaste | acg_import_credentials
       acg_import_credentials < credentials.txt

Read AWS credentials block from stdin and write to ~/.aws/credentials.
Supports all standard AWS credential formats:

  # Pluralsight Cloud Access panel
  AWS Access Key ID: ASIA...
  AWS Secret Access Key: abc123...
  AWS Session Token: IQo...

  # AWS Console export (unquoted or quoted)
  export AWS_ACCESS_KEY_ID=ASIA...
  export AWS_SECRET_ACCESS_KEY=abc123...
  export AWS_SESSION_TOKEN=IQo...

  # AWS credentials file section
  aws_access_key_id=ASIA...
  aws_secret_access_key=abc123...
  aws_session_token=IQo...
HELP
```
