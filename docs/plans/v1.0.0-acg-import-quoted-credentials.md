## Problem

`acg_import_credentials` cannot parse two common credential formats:

**Gap 1 — AWS IAM CSV download:** When an admin creates a new IAM user and generates
an access key, the console offers a "Download .csv" button. The downloaded file looks like:

```
Access key ID,Secret access key
AKIAIOSFODNN7EXAMPLE,wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

The current perl regex has no anchor to parse CSV — it looks for `AWS` or `Access Key ID:`
prefixes that don't exist in CSV rows.

**Gap 2 — AWS Console quoted export:** The AWS IAM Identity Center "Command line or
programmatic access" panel (Option 1) outputs quoted values:

```
export AWS_ACCESS_KEY_ID="AKIAIOSFODNN7EXAMPLE"
export AWS_SECRET_ACCESS_KEY="wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
export AWS_SESSION_TOKEN="AQo..."
```

The current `\S+` captures `"AKIAIOSFODNN7EXAMPLE"` with quotes, breaking
`aws sts get-caller-identity`.

---

## Supported Formats After Fix

All five formats must work:

| Format | Source | Session Token |
|--------|--------|---------------|
| `AWS Access Key ID: ASIA...` | Pluralsight Cloud Access panel | yes |
| `export AWS_ACCESS_KEY_ID=ASIA...` | AWS Console unquoted export | yes |
| `export AWS_ACCESS_KEY_ID="ASIA..."` | AWS Console quoted export (IAM Identity Center) | yes |
| `aws_access_key_id=ASIA...` | AWS credentials file section | yes |
| CSV (two-row, comma-separated) | AWS IAM "Download .csv" | no (long-term key) |

## Changes

### Change 1 — `scripts/plugins/acg.sh`: add CSV detection + strip quotes (lines 226–232)

**Exact old block:**

```bash
  _info "[acg] Reading credentials from stdin..."
  local input access_key secret_key session_token
  input=$(cat)

  access_key=$(printf '%s' "$input" | perl -ne 'if (/AWS(?:_ACCESS_KEY_ID| Access Key ID)[\s:=]+(\S+)/i) {print $1; exit}')
  secret_key=$(printf '%s' "$input" | perl -ne 'if (/AWS(?:_SECRET_ACCESS_KEY| Secret Access Key)[\s:=]+(\S+)/i) {print $1; exit}')
  session_token=$(printf '%s' "$input" | perl -ne 'if (/AWS(?:_SESSION_TOKEN| Session Token)[\s:=]+(\S+)/i) {print $1; exit}')
```

**Exact new block:**

```bash
  _info "[acg] Reading credentials from stdin..."
  local input access_key secret_key session_token
  input=$(cat)

  if printf '%s' "$input" | grep -q '^Access key ID,'; then
    access_key=$(printf '%s' "$input" | awk -F',' 'NR==2 {print $1}')
    secret_key=$(printf '%s' "$input" | awk -F',' 'NR==2 {print $2}')
    session_token=""
  else
    access_key=$(printf '%s' "$input" | perl -ne 's/["\x27]//g; if (/AWS(?:_ACCESS_KEY_ID| Access Key ID)[\s:=]+(\S+)/i) {print $1; exit}')
    secret_key=$(printf '%s' "$input" | perl -ne 's/["\x27]//g; if (/AWS(?:_SECRET_ACCESS_KEY| Secret Access Key)[\s:=]+(\S+)/i) {print $1; exit}')
    session_token=$(printf '%s' "$input" | perl -ne 's/["\x27]//g; if (/AWS(?:_SESSION_TOKEN| Session Token)[\s:=]+(\S+)/i) {print $1; exit}')
  fi
```

**How it works:**
- If input starts with the CSV header `Access key ID,`, parse as CSV: row 2, columns 1 and 2.
  Session token is empty (IAM long-term keys have no session token).
- Otherwise, fall through to the existing perl regex path with the `s/["\x27]//g` quote-strip
  prefix added to each line before matching.

### Change 2 — `scripts/plugins/acg.sh`: update help text (lines 206–222)

**Exact old block:**

```bash
    cat <<'HELP'
Usage: pbpaste | acg_import_credentials
       acg_import_credentials < credentials.txt

Read AWS credentials block from stdin and write to ~/.aws/credentials.
Expected input format (copy from Pluralsight Cloud Access panel):

  AWS Access Key ID: ASIA...
  AWS Secret Access Key: abc123...
  AWS Session Token: IQo...

Or the export block format:

  export AWS_ACCESS_KEY_ID=ASIA...
  export AWS_SECRET_ACCESS_KEY=abc123...
  export AWS_SESSION_TOKEN=IQo...
HELP
```

**Exact new block:**

```bash
    cat <<'HELP'
Usage: pbpaste | acg_import_credentials
       acg_import_credentials < credentials.csv
       acg_import_credentials < credentials.txt

Read AWS credentials from stdin and write to ~/.aws/credentials.
Supports all standard AWS credential formats:

  # AWS IAM "Download .csv" (new developer onboarding)
  Access key ID,Secret access key
  AKIAIOSFODNN7EXAMPLE,wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

  # Pluralsight Cloud Access panel
  AWS Access Key ID: ASIA...
  AWS Secret Access Key: abc123...
  AWS Session Token: IQo...

  # AWS Console export (unquoted or quoted)
  export AWS_ACCESS_KEY_ID=ASIA...
  export AWS_SECRET_ACCESS_KEY=abc123...
  export AWS_SESSION_TOKEN=IQo...

  # AWS credentials file section
  aws_access_key_id=ASIA...
  aws_secret_access_key=abc123...
  aws_session_token=IQo...
HELP
```
