# Public/Private Function Audit — k3d-manager

**Branch:** `k3d-manager-v1.2.0`
**Rule:** `deploy_*` and named lifecycle entry points are public. Supporting functions that
are not useful on their own are private (underscore prefix). See CLAUDE.md plugin conventions.

---

## Before You Start

- Read callers of each function before renaming — grep for the current name across all scripts
- `create_cluster` and `destroy_cluster` aliases (`k3d`/`k3s` variants) also appear in `scripts/lib/foundation/` subtree — do NOT edit those; edits to subtree must go upstream to lib-foundation first
- `acg_extend_playwright` and `acg_import_credentials` are in lib-acg — rename must be done there and subtree-pulled into k3d-manager
- The `help` category patterns in `scripts/lib/help/utils.sh` must be updated to match the new private names (remove renamed functions from public patterns)
