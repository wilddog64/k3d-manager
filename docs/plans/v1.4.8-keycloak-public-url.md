## Problem

The Keycloak URL is set to internal cluster hostnames throughout the stack:
- Frontend image builds with `VITE_KEYCLOAK_URL=http://keycloak.identity.svc.cluster.local:8080` ‚Äî Login button redirects to an unreachable host when accessed via `https://frontend.3ai-talk.org`
- Backend services use `OAUTH2_ISSUER_URI=http://keycloak.identity.svc.cluster.local/realms/shopping-cart` ‚Äî token issuer claim will mismatch after Keycloak `frontendUrl` is set
- Keycloak realm has no `frontendUrl` ‚Äî tokens carry the internal URL as `iss` claim

**Goal:** Login via `https://frontend.3ai-talk.org` ‚Üí Keycloak at `https://keycloak.3ai-talk.org` ‚Üí redirect back ‚Üí tokens validated by backend services.

## Fix

### Change 1 ‚Äî `k3d-manager/bin/acg-up`: set Keycloak `frontendUrl` after realm import

In `bin/acg-up`, find the block that ends Step 10d (realm import). It contains a `_keycloak_reconcile_realm_client` call followed by realm cleanup. Add the `frontendUrl` patch immediately after the realm import completes and before the Keycloak port-forward is stopped.

Look for this pattern around line 793:
```bash
if _keycloak_reconcile_realm_client "http://localhost:${_kc_pf_port}" "${_kc_token}" "shopping-cart" "argocd" "${_realm_json_file}";
```

After the realm reconciliation block (after the clients loop ends, before port-forward kill), insert:

**Exact new block to insert:**
```bash
# Set Keycloak frontendUrl so tokens carry the public Cloudflare domain as issuer
_kc_frontend_url="${KEYCLOAK_PUBLIC_URL:-https://keycloak.3ai-talk.org}"
curl -sf -X PUT \
  -H "Authorization: Bearer ${_kc_token}" \
  -H "Content-Type: application/json" \
  "http://localhost:${_kc_pf_port}/admin/realms/shopping-cart" \
  -d "{\"frontendUrl\":\"${_kc_frontend_url}\"}" \
  >/dev/null 2>&1 \
  && _info "[acg-up] Keycloak frontendUrl set to ${_kc_frontend_url}" \
  || _warn "[acg-up] Keycloak frontendUrl update failed ‚Äî external SSO may not work"
```

**Exact insertion point** ‚Äî insert BEFORE line 808 (`kill "${_kc_pf_pid}" 2>/dev/null || true` / `rm -f "${_realm_json_file}"`):

```bash
    kill "${_kc_pf_pid}" 2>/dev/null || true
    rm -f "${_realm_json_file}"
  fi
fi
```

becomes:

```bash
    # Set Keycloak frontendUrl so tokens carry the public Cloudflare domain as issuer
    _kc_frontend_url="${KEYCLOAK_PUBLIC_URL:-https://keycloak.3ai-talk.org}"
    curl -sf -X PUT \
      -H "Authorization: Bearer ${_kc_token}" \
      -H "Content-Type: application/json" \
      "http://localhost:${_kc_pf_port}/admin/realms/shopping-cart" \
      -d "{\"frontendUrl\":\"${_kc_frontend_url}\"}" \
      >/dev/null 2>&1 \
      && _info "[acg-up] Keycloak frontendUrl set to ${_kc_frontend_url}" \
      || _warn "[acg-up] Keycloak frontendUrl update failed ‚Äî external SSO may not work"

    kill "${_kc_pf_pid}" 2>/dev/null || true
    rm -f "${_realm_json_file}"
  fi
fi
```
