1. **Do not modify `scripts/lib/system.sh` or any non-test production code.**
   Codex owns implementation files. If you find a bug, report it — do not fix it.
2. **Do not modify `memory-bank/`.** Claude owns memory bank updates.

5. **Do not commit on failure.** If tests fail due to implementation bugs, stop and
   report to Claude with: file, line number, expected vs actual behavior.

## Phase 3 — Red Team Audit

Perform the following security checks on the v0.6.2 changes. For each check,
report: PASS, FAIL, or N/A with a brief justification.

### RT-1: PATH Poisoning

Verify `_safe_path` correctly identifies world-writable directories.

```
Test scenario:
1. Create a temp world-writable dir: tmp_dir=$(mktemp -d) && chmod 777 "$tmp_dir"
2. Prepend to PATH: PATH="$tmp_dir:$PATH"
3. Call _safe_path
4. Assert: returns non-zero
5. Cleanup: rmdir "$tmp_dir"
```

### RT-2: Secret Exposure in Process Listing

Verify the `ldap-password-rotator.sh` stdin fix works:

```
Check:
1. Read the refactored vault kv put command in ldap-password-rotator.sh
2. Verify: $new_password does NOT appear in any command argument string
3. Verify: $new_password is passed via stdin (heredoc or <<<)
4. Verify: kubectl exec uses -i flag for stdin passthrough
```

### RT-3: Trace Isolation for Copilot CLI

Verify that `_k3d_manager_copilot` does not leak secrets when tracing is enabled:

```
Check:
1. Verify copilot invocation goes through _run_command (which honors _args_have_sensitive_flag)
2. Verify no Vault tokens, passwords, or kubeconfig contents are passed as
   arguments to the copilot command
3. Verify CDPATH and OLDPWD are sanitized before invocation
```

### RT-4: Deny-Tool Guardrails

Verify `_k3d_manager_copilot` deny-tool configuration:

```
Check:
1. Verify --deny-tool 'shell(cd ..)' is present
2. Verify --deny-tool 'shell(git push)' is present
3. Verify --deny-tool 'shell(git push --force)' is present
4. Verify --deny-tool 'shell(rm -rf)' is present
5. Check: are there other dangerous commands that should be denied?
   Candidates: shell(curl|wget piped to sh), shell(eval), shell(sudo)
   Report any recommended additions.
```

### RT-5: AI Gating Bypass

Verify the `K3DM_ENABLE_AI` gate cannot be bypassed:

```
Check:
1. Verify _k3d_manager_copilot checks K3DM_ENABLE_AI before calling _ensure_copilot_cli
2. Verify the check is [[ "${K3DM_ENABLE_AI:-0}" != "1" ]] (not just -z check)
3. Verify no other code path calls copilot directly (bypassing _k3d_manager_copilot)
   Search: grep -r 'copilot' scripts/ --include='*.sh' — should only appear in system.sh
```

### RT-6: Copilot CLI Prompt Injection Surface

Review the known vulnerability (referenced in plan):

```
Check:
1. Verify --secret-env-vars is NOT used anywhere
2. Verify no Vault tokens, kubeconfig, or cluster credentials are passed to copilot
3. Verify copilot's scope is limited to file generation (no cluster access)
```
