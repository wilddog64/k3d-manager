# Feature: v1.6.0 — Slack threading for acg-up job notifications

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/k3dm-webhook`, `Makefile`

---

## Problem

All Slack notifications from the webhook post as independent messages, making it hard
to follow the lifecycle of a single acg-up job (started → progress → stall alert →
failure analysis → cleanup → post-provision health check). They appear scattered across
the channel with no visual grouping.

Additionally `_run_cleanup` calls `_notify` which does not exist at module scope — it
would raise `NameError` if the function were ever reached.

---

## Solution

Each acg-up/down job gets a single Slack thread. The first notification for a job
posts a parent message via `chat.postMessage` and saves the returned `ts` to
`JOB_DIR/<job_id>/thread_ts`. Every subsequent notification for the same job passes
`thread_ts` to `chat.postMessage`, producing a reply in the same thread.

Falls back to the old `_slack_post` incoming-webhook path when `SLACK_BOT_TOKEN` /
`SLACK_CHANNEL_ID` are not configured — backward compat is preserved.

A new `update-webhook-slack` Makefile target injects `SLACK_BOT_TOKEN` and
`SLACK_CHANNEL_ID` into the installed LaunchAgent plist and restarts the webhook.

```python
def _post_slack_bot(text, thread_ts=None):
    """POST to Slack chat.postMessage API. Returns ts on success, empty string on failure."""
    if not SLACK_BOT_TOKEN or not SLACK_CHANNEL_ID:
        return ""
    payload = {"channel": SLACK_CHANNEL_ID, "text": text}
    if thread_ts:
        payload["thread_ts"] = thread_ts
    data = json.dumps(payload).encode()
    req = urllib.request.Request(
        "https://slack.com/api/chat.postMessage",
        data=data,
        headers={
            "Authorization": f"Bearer {SLACK_BOT_TOKEN}",
            "Content-Type": "application/json; charset=utf-8",
        },
        method="POST",
    )
    try:
        with urllib.request.urlopen(req, timeout=10) as resp:
            result = json.loads(resp.read())
            return result.get("ts", "") if result.get("ok") else ""
    except Exception:
        return ""


def _notify_job(job_id, text):
    """Post a Slack message in the thread for job_id.

    First call per job creates the parent message and saves ts.
    Subsequent calls reply in-thread. Falls back to webhook URL if bot token not set.
    """
    thread_ts_file = JOB_DIR / job_id / "thread_ts"
    if SLACK_BOT_TOKEN and SLACK_CHANNEL_ID:
        thread_ts = thread_ts_file.read_text().strip() if thread_ts_file.exists() else None
        ts = _post_slack_bot(text, thread_ts=thread_ts)
        if ts and not thread_ts:
            thread_ts_file.write_text(ts)
    elif SLACK_WEBHOOK_URL:
        _slack_post(SLACK_WEBHOOK_URL, text)


def _call_gemini(prompt):
```
