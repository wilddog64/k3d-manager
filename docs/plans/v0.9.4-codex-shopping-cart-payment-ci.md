## Problem

`shopping-cart-payment` CI fails in the Docker build step. Root cause: `./mvnw
dependency:go-offline -B` exits 1 with `maven.multiModuleProjectDirectory system property
is not set` on JDK 21 Alpine (Maven Wrapper incompatibility).

A fix was pushed at commit `8a77199` on 2026-03-17:
- Swapped Maven Wrapper (`./mvnw`) to `apk add maven` + `mvn` in the Dockerfile
- Same pattern used by shopping-cart-order (which now passes)

The CI run for `8a77199` may still be in progress when you start. Check it first.

The payment CI has **two** Docker build jobs that may both be affected:
1. `Build, Scan & Push` — calls the infra reusable workflow
   (`wilddog64/shopping-cart-infra/.github/workflows/build-push-deploy.yml@8363caf`)
2. `Build Docker Image` — standalone job in `ci.yaml` that also builds+pushes the image

Both use the same `Dockerfile`. The fix at `8a77199` should resolve both.
