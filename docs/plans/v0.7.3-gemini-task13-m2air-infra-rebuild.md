## Background

Root cause of all ArgoCD→Ubuntu connectivity failures: the infra cluster (with ArgoCD)
is on M4 Air, which has no route to Ubuntu at `10.211.55.14` (Parallels VM on M2 Air).

M2 Air has a partial k3d cluster (`istio-system` + `secrets` — no `cicd`, no `identity`).
This task tears it down and rebuilds the full infra stack on M2 Air. Once ArgoCD runs on
M2 Air, it can reach Ubuntu at `10.211.55.14` directly.

## Task 5 — Register Ubuntu k3s Cluster in ArgoCD

Re-export Ubuntu kubeconfig:
```bash
ssh ubuntu "sudo cat /etc/rancher/k3s/k3s.yaml \
  | sed 's|127.0.0.1|10.211.55.14|g'" > ~/.kube/k3s-ubuntu.yaml
chmod 600 ~/.kube/k3s-ubuntu.yaml

# Verify kubectl can reach Ubuntu
KUBECONFIG=~/.kube/k3s-ubuntu.yaml kubectl get nodes
# Expected: node Ready
```

Port-forward ArgoCD and register:
```bash
nohup kubectl port-forward svc/argocd-server -n cicd 8080:443 \
  > /tmp/argocd-pf.log 2>&1 &
sleep 5
curl -k https://localhost:8080/healthz

argocd login localhost:8080 --insecure
argocd cluster add k3s-automation \
  --name ubuntu-k3s \
  --kubeconfig ~/.kube/k3s-ubuntu.yaml
argocd cluster list
# Expected: https://10.211.55.14:6443  ubuntu-k3s  Ready
```

**STOP — report `argocd cluster list` output.**
