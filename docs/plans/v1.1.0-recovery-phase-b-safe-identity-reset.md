# Plan: v1.1.0 Recovery Phase B — Safe Identity Reset

**Branch:** `k3d-manager-v1.1.0`
**Files:** `scripts/playwright/gcp_login.js`

---

## Objective

Harden the GCP login robot to handle "Signed Out" account traps and "Session Expired" redirects without manual intervention, while strictly preserving the active Pluralsight session.

---

## Proposed Logic (Surgical Cleanup)

1.  **Domain-Isolated Cookie Wipe:** 
    - At the start of the flow, use `context.clearCookies({ domain: '.google.com' })`.
    - **Goal:** Clear the "Choose an account" list of 4+ old sandbox users.
    - **Safety:** Explicitly avoids clearing `.pluralsight.com` cookies to preserve the dashboard session.

2.  **The "Trap Escape" Loop:**
    - Detect `sessionexpired` or `/info/` in the URL.
    - **Action:** Close the current tab and open a fresh tab at `https://accounts.google.com/AddSession`.
    - **Goal:** Bypasses Google's internal flow-state errors (CSRF) that happen during redirects.

3.  **Clean Entry Automation:**
    - Fill in the current `GCP_ACCOUNT` and `GCP_PASSWORD`.
    - Automated clicks for "Agree and continue", "Confirm", and "Allow".

4.  **Completion Gate:**
    - Wait for the browser to navigate to `localhost` before detaching.
    - **Goal:** Ensure the OAuth token is actually received by the gcloud CLI.

---

## Verification Criteria (Functional E2E)

- [ ] Pluralsight dashboard remains logged in after the robot finishes.
- [ ] Google "Choose an account" list is empty or shows only the current user.
- [ ] CLI identity (`gcloud auth list`) is active for the new sandbox user.
- [ ] SSH and instance management (`gcloud compute instances list`) are unblocked.
