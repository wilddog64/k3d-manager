# Bugfix: v1.6.0 — Gemini timeout leaks prompt into Slack notification

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/k3dm-webhook`

---

## Problem

When the Gemini CLI subprocess times out, `except Exception as exc` catches
`subprocess.TimeoutExpired`. Python's string representation of `TimeoutExpired`
includes the full command argument list — which contains the entire `--prompt`
value (hundreds of characters of cluster state). This leaks into the Slack
notification as a wall of text with literal `\n` escape sequences.

Additionally, the 60-second timeout is too short when `_collect_cluster_state()`
adds node/pod/ArgoCD output to the prompt — the combined payload is large enough
that the Gemini CLI regularly exceeds the limit.

**Root cause:** `except Exception as exc: return f"gemini error: {exc}"` converts
`TimeoutExpired` to a string that embeds the full `--prompt` argument; and the
60-second timeout cap is too low for prompts enriched with live cluster state.

---

## Reproduction

Trigger `_analyze_failure()` while the Gemini CLI is slow (large cluster state
in prompt). The Slack notification shows the raw prompt text with `\n` sequences
and ends with `timed out after 60 seconds`.

---

## Fix

### Change 1 — `bin/k3dm-webhook`: catch `TimeoutExpired` before `Exception`

**Exact old block:**
```python
    try:
        r = subprocess.run(
            [gemini_bin, "--model", GEMINI_MODEL, "--prompt", prompt],
            capture_output=True,
            text=True,
            timeout=60,
            env=env,
        )
        return r.stdout.strip() or r.stderr.strip() or "gemini returned no output"
    except FileNotFoundError:
        return "gemini CLI not found — skipping AI analysis"
    except Exception as exc:
        return f"gemini error: {exc}"
```

**Exact new block:**
```python
    try:
        r = subprocess.run(
            [gemini_bin, "--model", GEMINI_MODEL, "--prompt", prompt],
            capture_output=True,
            text=True,
            timeout=120,
            env=env,
        )
        return r.stdout.strip() or r.stderr.strip() or "gemini returned no output"
    except FileNotFoundError:
        return "gemini CLI not found — skipping AI analysis"
    except subprocess.TimeoutExpired:
        return "gemini analysis timed out after 120s — cluster state may be too large"
    except Exception as exc:
        return f"gemini error: {exc}"
```
