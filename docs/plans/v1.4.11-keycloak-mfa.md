# Plan: v1.4.11 — Keycloak Role-Based MFA

**Branch (k3d-manager — spec source):** `k3d-manager-v1.4.11`
**Branch (work repo — shopping-cart-infra):** `feat/v1.4.11-keycloak-mfa`

---

## Objective

Enable TOTP-based MFA for users with `platform-admin` or `platform-developer` roles. All
other users (regular shoppers, service accounts) are unaffected. `make up` and all CI/CD
automation are explicitly safe — they use kubectl service-account tokens and `kcadm.sh` admin
API, which never touch the realm browser authentication flow.

**Both ArgoCD and the shopping-cart frontend already route through the Keycloak `shopping-cart`
realm browser flow.** Adding MFA to that flow automatically covers both surfaces with no changes
to either the ArgoCD config or the frontend code.

---

## Design

### Why role-based instead of realm-wide

| Users | Role | Gets MFA? |
|---|---|---|
| Platform admins (ArgoCD full access) | `platform-admin` | Yes |
| Platform developers (ArgoCD read/deploy) | `platform-developer` | Yes |
| Regular shoppers | `order-user`, `catalog-user` | No |
| Service accounts | N/A | No |
| CI/CD automation (`make up`) | kubectl SA tokens | No — never touches browser flow |

### Mechanism: composite role + conditional OTP

1. A new `platform-mfa` realm role is added.
2. `platform-admin` and `platform-developer` become composite roles that include `platform-mfa`.
   Users in `platform-admins` / `platform-developers` groups inherit `platform-mfa` automatically.
3. The reconcile job creates a `browser-with-conditional-otp` flow (copied from built-in browser
   flow) with a conditional OTP sub-flow that fires when the authenticated user has `platform-mfa`.
4. The realm's `browserFlow` is set to `browser-with-conditional-otp`.

### CI/CD safety

- `make up` uses kubectl with Kubernetes service-account tokens — never triggers a browser OIDC flow.
- The reconcile job authenticates with `kcadm.sh` admin credentials against the `master` realm —
  completely separate from the `shopping-cart` realm browser flow.
- ArgoCD sync operations use service-account tokens — not browser OIDC.
- `partialImport` does not import authentication flows — flows are created imperatively in the
  reconcile job with idempotency guards (skip if flow already exists).

### Change 2 — `keycloak-reconcile-hook-job.yaml`: add conditional OTP browser flow

Add the following block to the reconcile job's shell script, **immediately after** the `partialImport` block (after line 98 — the `if [[ ${_pi_rc} -ne 0 ]]` block), **before** the `ldap_id=` line:

**Exact insertion point — insert after this existing block:**
```bash
          if [[ ${_pi_rc} -ne 0 ]]; then
            echo "Warning: partialImport exited ${_pi_rc} (re-run duplicate-key is expected); continuing with LDAP setup"
          fi
```

**Block to insert:**
```bash

          if /opt/keycloak/bin/kcadm.sh get "authentication/flows" \
               -r "${KC_REALM}" 2>/dev/null \
             | grep -q '"browser-with-conditional-otp"'; then
            echo "browser-with-conditional-otp flow already exists; skipping"
          else
            echo "Creating browser-with-conditional-otp flow..."

            /opt/keycloak/bin/kcadm.sh create "authentication/flows/browser/copy" \
              -r "${KC_REALM}" -s newName=browser-with-conditional-otp

            /opt/keycloak/bin/kcadm.sh create \
              "authentication/flows/browser-with-conditional-otp/executions/flow" \
              -r "${KC_REALM}" \
              -s alias=otp-conditional-subflow \
              -s type=basic-flow

            _otp_subflow_exec_id=$(
              /opt/keycloak/bin/kcadm.sh get \
                "authentication/flows/browser-with-conditional-otp/executions" \
                -r "${KC_REALM}" --fields id,displayName --format csv 2>/dev/null \
              | grep "otp-conditional-subflow" | cut -d',' -f1 | tr -d '"'
            )
            /opt/keycloak/bin/kcadm.sh update \
              "authentication/executions/${_otp_subflow_exec_id}" \
              -r "${KC_REALM}" -s requirement=CONDITIONAL

            /opt/keycloak/bin/kcadm.sh create \
              "authentication/flows/otp-conditional-subflow/executions/execution" \
              -r "${KC_REALM}" -s provider=conditional-user-role

            _role_cond_id=$(
              /opt/keycloak/bin/kcadm.sh get \
                "authentication/flows/otp-conditional-subflow/executions" \
                -r "${KC_REALM}" --fields id,providerId --format csv 2>/dev/null \
              | grep "conditional-user-role" | cut -d',' -f1 | tr -d '"'
            )
            /opt/keycloak/bin/kcadm.sh create \
              "authentication/executions/${_role_cond_id}/config" \
              -r "${KC_REALM}" \
              -s alias=platform-mfa-role-condition \
              -s 'config={"conditionRoleAlias":"platform-mfa","negate":"false"}'
            /opt/keycloak/bin/kcadm.sh update \
              "authentication/executions/${_role_cond_id}" \
              -r "${KC_REALM}" -s requirement=REQUIRED

            /opt/keycloak/bin/kcadm.sh create \
              "authentication/flows/otp-conditional-subflow/executions/execution" \
              -r "${KC_REALM}" -s provider=auth-otp-form

            _otp_form_id=$(
              /opt/keycloak/bin/kcadm.sh get \
                "authentication/flows/otp-conditional-subflow/executions" \
                -r "${KC_REALM}" --fields id,providerId --format csv 2>/dev/null \
              | grep "auth-otp-form" | cut -d',' -f1 | tr -d '"'
            )
            /opt/keycloak/bin/kcadm.sh update \
              "authentication/executions/${_otp_form_id}" \
              -r "${KC_REALM}" -s requirement=REQUIRED

            /opt/keycloak/bin/kcadm.sh update "realms/${KC_REALM}" \
              -s browserFlow=browser-with-conditional-otp

            echo "browser-with-conditional-otp flow created and activated"
          fi

```
