## Problem

All shopping-cart services run `replicas: 2`. Three services (order, payment,
product-catalog) also have HPA with `minReplicas: 2`. On a constrained dev cluster
(t3.medium, 4GB RAM), 10 pods simultaneously thrash the node and cause
ServiceUnavailable on the k3s API server.

HPAs are not useful on a single-node dev cluster — there is no spare capacity to scale
into. They add complexity with no benefit and will scale pods back up even after
`replicas` is set to 1.

Fix: `replicas: 1` on all deployments + delete HPA manifests entirely. HPA will be
reintroduced in v1.1.0 on EKS with a proper multi-node autoscaling group.
