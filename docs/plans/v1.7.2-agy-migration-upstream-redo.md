## Why this spec exists (what went wrong)

The previous pass implemented the migration **in the wrong repository.** It committed the code
directly into **k3d-manager's subtree directories** instead of the upstream source repos:

- `c84f6a5d` (k3d-manager) — `feat(cdp): replace dead gemini-cli session check…` — committed
  the cdp.sh rewrite PLUS `scripts/lib/acg_session_check.js` and
  `scripts/tests/lib/acg_session_check.bats` **into k3d-manager**.
- `8c4e44dc` (k3d-manager) — `feat(system): add agy CLI installer helper` — committed
  `_ensure_agy_cli` into k3d-manager's `scripts/lib/foundation/` subtree copy.

Neither commit exists in lib-acg or lib-foundation — their upstream branches still hold only
the spec. This violates the **subtree-upstream-first** rule: `scripts/lib/acg/` is a subtree of
lib-acg, `scripts/lib/foundation/` is a subtree of lib-foundation; edits must land upstream and
arrive in k3d-manager only via `git subtree pull`.

It also carried a **portability bug**: the JS landed at k3d-manager's top-level `scripts/lib/`
(outside the acg subtree) and cdp.sh resolved it as `${_LIB_ACG_ROOT}/../acg_session_check.js`
— reaching *out of* the lib-acg repo. The script must live **inside lib-acg** so it round-trips.
