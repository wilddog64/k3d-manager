## Problem

`make up` fails at step 9 with:

```
error: resource mapping not found for name: "vault-backend" namespace: "" from "STDIN":
no matches for kind "ClusterSecretStore" in version "external-secrets.io/v1beta1"
ensure CRDs are installed first
```

**Root cause:** ESO 1.0.0 dropped `v1beta1` serving. The ClusterSecretStore manifest
in `bin/acg-up` still uses `apiVersion: external-secrets.io/v1beta1`. It must be
updated to `apiVersion: external-secrets.io/v1` to match ESO 1.0.0.

---

## Fix

### Change 1 — `bin/acg-up` line 235: bump ClusterSecretStore apiVersion

**Exact old block (lines 234–250):**

```bash
kubectl apply --context ubuntu-k3s -f - <<'CSSEOF'
apiVersion: external-secrets.io/v1beta1
kind: ClusterSecretStore
metadata:
  name: vault-backend
spec:
  provider:
    vault:
      server: "http://vault-bridge.secrets.svc.cluster.local:8201"
      path: "secret"
      version: "v2"
      auth:
        tokenSecretRef:
          name: vault-token
          namespace: secrets
          key: token
CSSEOF
```

**Exact new block:**

```bash
kubectl apply --context ubuntu-k3s -f - <<'CSSEOF'
apiVersion: external-secrets.io/v1
kind: ClusterSecretStore
metadata:
  name: vault-backend
spec:
  provider:
    vault:
      server: "http://vault-bridge.secrets.svc.cluster.local:8201"
      path: "secret"
      version: "v2"
      auth:
        tokenSecretRef:
          name: vault-token
          namespace: secrets
          key: token
CSSEOF
```
