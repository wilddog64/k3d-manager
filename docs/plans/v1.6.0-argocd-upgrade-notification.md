# v1.6.0 — ArgoCD upgrade pipeline notifications (email + PagerDuty)

## Context

The CVE scan CronJob needs to notify the user when:
- **A CVE is detected** and an upgrade is starting
- **A stage is unreachable** (laptop off, ACG not running) — skipped, not failed
- **A stage fails** smoke test — rolled back, pipeline stopped
- **Upgrade completes** — summary of what was upgraded

Channels:
- **Email (SendGrid)** — all events; low-friction summary
- **PagerDuty** — failures and unreachable stages only; wakes the user up if needed

Both are optional: if credentials are missing the notification is skipped with a warning
log line. The pipeline always continues regardless of notification success.

**Prerequisites:** `platform-ops` namespace and ServiceAccount from the CVE scan spec
(`v1.6.0-argocd-cve-scan-promotion.md`) must exist. Deploy that spec first.

## File 1: `scripts/etc/argocd/platform-ops/notify.sh`

```bash
#!/bin/sh
# notify.sh — send email and/or PagerDuty alert from inside the CVE scan CronJob.
#
# Usage:
#   notify.sh info    "Title" "Message body"
#   notify.sh warning "Title" "Message body"
#   notify.sh critical "Title" "Message body"
#
# Severity mapping:
#   info     → email only
#   warning  → email + PagerDuty trigger
#   critical → email + PagerDuty trigger
#
# Credentials (all optional — missing = skip that channel):
#   SENDGRID_API_KEY    — SendGrid v3 API key
#   NOTIFICATION_EMAIL  — recipient email address
#   NOTIFICATION_FROM   — sender email address (default: argocd-cve@k3d-manager)
#   PAGERDUTY_ROUTING_KEY — PagerDuty Events API v2 routing/integration key

set -eu

SEVERITY="${1:-info}"
TITLE="${2:-[k3d-manager] notification}"
MESSAGE="${3:-no message}"

NOTIFICATION_FROM="${NOTIFICATION_FROM:-argocd-cve@k3d-manager}"

_log() { echo "[notify] $*"; }

# --- Email via SendGrid ---
if [ -n "${SENDGRID_API_KEY:-}" ] && [ -n "${NOTIFICATION_EMAIL:-}" ]; then
  _PAYLOAD=$(printf '{"personalizations":[{"to":[{"email":"%s"}]}],"from":{"email":"%s"},"subject":"[k3d-manager] %s","content":[{"type":"text/plain","value":"%s"}]}' \
    "${NOTIFICATION_EMAIL}" "${NOTIFICATION_FROM}" "${TITLE}" "${MESSAGE}")

  _STATUS=$(curl -sf -o /dev/null -w "%{http_code}" \
    -X POST "https://api.sendgrid.com/v3/mail/send" \
    -H "Authorization: Bearer ${SENDGRID_API_KEY}" \
    -H "Content-Type: application/json" \
    -d "${_PAYLOAD}" 2>/dev/null || echo "000")

  if [ "${_STATUS}" = "202" ]; then
    _log "Email sent to ${NOTIFICATION_EMAIL} (${SEVERITY})"
  else
    _log "Email failed: HTTP ${_STATUS}"
  fi
else
  _log "Email skipped: SENDGRID_API_KEY or NOTIFICATION_EMAIL not set"
fi

# --- PagerDuty (warning and critical only) ---
if [ "${SEVERITY}" != "info" ] && [ -n "${PAGERDUTY_ROUTING_KEY:-}" ]; then
  _PD_SEVERITY="warning"
  [ "${SEVERITY}" = "critical" ] && _PD_SEVERITY="critical"

  _PD_PAYLOAD=$(printf '{"routing_key":"%s","event_action":"trigger","payload":{"summary":"%s","source":"k3d-manager-oci","severity":"%s","custom_details":{"message":"%s"}}}' \
    "${PAGERDUTY_ROUTING_KEY}" "${TITLE}" "${_PD_SEVERITY}" "${MESSAGE}")

  _PD_STATUS=$(curl -sf -o /dev/null -w "%{http_code}" \
    -X POST "https://events.pagerduty.com/v2/enqueue" \
    -H "Content-Type: application/json" \
    -d "${_PD_PAYLOAD}" 2>/dev/null || echo "000")

  if [ "${_PD_STATUS}" = "202" ]; then
    _log "PagerDuty triggered (${_PD_SEVERITY})"
  else
    _log "PagerDuty failed: HTTP ${_PD_STATUS}"
  fi
elif [ "${SEVERITY}" != "info" ]; then
  _log "PagerDuty skipped: PAGERDUTY_ROUTING_KEY not set"
fi
```

## File 2: `scripts/etc/argocd/platform-ops/notification-secret.yaml.tmpl`

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: platform-ops-notifications
  namespace: platform-ops
  labels:
    managed-by: k3d-manager
type: Opaque
stringData:
  sendgrid-api-key: "${SENDGRID_API_KEY}"
  pagerduty-routing-key: "${PAGERDUTY_ROUTING_KEY}"
  notification-email: "${NOTIFICATION_EMAIL}"
  notification-from: "${NOTIFICATION_FROM:-argocd-cve@k3d-manager}"
```

**Note:** `envsubst` substitutes these at deploy time. If any variable is unset,
the field will be an empty string — the notify script handles missing values gracefully
by skipping that channel.

## File 4: `scripts/plugins/argocd.sh` — update `deploy_argocd_platform_ops`

Extend the function to deploy the notification Secret scaffold:

```bash
# Add after the ConfigMap apply line:
_info "[argocd] Deploying notification Secret scaffold..."
envsubst '$SENDGRID_API_KEY $PAGERDUTY_ROUTING_KEY $NOTIFICATION_EMAIL $NOTIFICATION_FROM' \
  < "${_dir}/notification-secret.yaml.tmpl" | _kubectl apply -f -

# Also deploy notify.sh as part of the scan-script ConfigMap:
_kubectl create configmap argocd-cve-scan-script \
  --from-file=cve-scan.sh="${_dir}/cve-scan.sh" \
  --from-file=notify.sh="${_dir}/notify.sh" \
  --namespace platform-ops \
  --dry-run=client -o yaml | _kubectl apply -f -
```

**Note:** `notify.sh` is bundled into the same ConfigMap as `cve-scan.sh`. The CronJob
mounts the ConfigMap at `/scripts/`; `cve-scan.sh` calls `/scripts/notify.sh`.

## Usage in cve-scan.sh

After this spec is implemented, `cve-scan.sh` (from `v1.6.0-argocd-cve-scan-promotion.md`)
calls notify like this:

```sh
/scripts/notify.sh info \
  "ArgoCD CVE upgrade started" \
  "Upgrading ${CURRENT_CHART} → ${LATEST_CHART}; CVEs: ${CVE_COUNT}"

/scripts/notify.sh warning \
  "ArgoCD upgrade stage unreachable: ACG" \
  "Laptop webhook did not respond — skipping ACG stage"

/scripts/notify.sh critical \
  "ArgoCD upgrade FAILED: infra smoke test" \
  "chart ${LATEST_CHART} failed on infra hub — rolled back; OCI not upgraded"

/scripts/notify.sh info \
  "ArgoCD upgrade complete" \
  "chart ${CURRENT_CHART}→${LATEST_CHART}; ACG=${ACG_UPGRADED} infra=${INFRA_UPGRADED} OCI=patched"
```

## Setting credentials on OCI

Run once after OCI cluster is up:

```bash
SENDGRID_API_KEY=<key> \
PAGERDUTY_ROUTING_KEY=<key> \
NOTIFICATION_EMAIL=ckm.liang@gmail.com \
  scripts/k3d-manager deploy_argocd_platform_ops
```

```bash
kubectl patch secret platform-ops-notifications -n platform-ops \
  --type=merge -p \
  '{"stringData":{"sendgrid-api-key":"<key>","pagerduty-routing-key":"<key>","notification-email":"ckm.liang@gmail.com"}}'
```
