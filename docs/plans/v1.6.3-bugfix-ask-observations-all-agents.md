# Bugfix: v1.6.3 — /ask codex and /ask claude never file OBSERVATIONS bugs

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/k3dm-webhook`

---

## Problem

`_parse_gemini_observations` is called only for the `gemini` agent path. When `/ask codex`
or `/ask claude` (non-filing mode) runs, the response is posted to Slack as raw text and
nothing is written to `docs/bugs/` — even when the agent identifies a real bug.

Codex's DNS analysis (`host.k3d.internal` still not resolving after `acg-refresh`) is the
concrete incident: the analysis reached Slack but no bug doc was filed.

**Root cause:** `_run_cluster_ask` calls `_parse_gemini_observations` inside the
`if agent == "gemini":` branch and returns early. The shared `subprocess.run` block
that handles claude and codex posts `_finish(f"🤖 *{agent}:* {answer}")` unconditionally
without any observation extraction.

---

## Reproduction

1. `/ask codex` with a question that reveals a real bug (e.g. DNS resolution failure)
2. Codex returns an analysis to Slack
3. `ls docs/bugs/` — nothing new was written

---

Four changes, all in `bin/k3dm-webhook`.

### Change 4 — parse observations in shared subprocess finish block (line ~1832)

**Exact old block:**

```python
            answer = _ANSI.sub("", result.stdout).strip()
            if not answer:
                answer = _ANSI.sub("", result.stderr).strip() or f"No output from {agent}."
            _finish(f"🤖 *{agent}:* {answer}")
```

**Exact new block:**

```python
            answer = _ANSI.sub("", result.stdout).strip()
            if not answer:
                answer = _ANSI.sub("", result.stderr).strip() or f"No output from {agent}."
            if _observe:
                answer, filed = _parse_gemini_observations(answer)
                suffix = "\n" + "\n".join(f"Also filed: `{f}`" for f in filed) if filed else ""
            else:
                suffix = ""
            _finish(f"🤖 *{agent}:* {answer}{suffix}")
```
