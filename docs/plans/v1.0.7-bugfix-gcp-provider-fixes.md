# Bugfix: v1.0.7 — GCP Provider Post-Review Fixes

**Branch:** `k3d-manager-v1.0.7`
**Files:** `scripts/lib/providers/k3s-gcp.sh`, `scripts/plugins/gcp.sh`

---

## Problem

Five issues found during post-commit review of `1c620795`:

1. **`--kubeconfig` is not a valid k3sup flag** — the correct flag is `--local-path`. k3sup will error or write the kubeconfig to an unexpected location.
2. **`--image-family` and `--image-project` missing from `gcloud compute instances create`** — GCE will use its default image (Debian) instead of Ubuntu 22.04 LTS. k3sup and k3d-manager assume Ubuntu.
3. **`--k3s-extra-args="--disable=traefik"` missing from `k3sup install`** — Traefik is installed alongside k3s, competing with Istio IngressGateway for ports 80/443.
4. **`destroy_cluster` does not clean up the kubeconfig context** — `kubectl config delete-context k3s-gcp`, `delete-cluster k3s-gcp`, and `rm kubeconfig` are missing. `make down` leaves a stale context.
5. **`gcp.sh` assigns directly to globals before validating** — if Playwright output is partial, empty values are silently exported. Locals should be used for parsing; validation should run before `export`.

---

## Reproduction

| Bug | Trigger | Symptom |
|-----|---------|---------|
| 1 | `make up CLUSTER_PROVIDER=k3s-gcp` | k3sup errors: `unknown flag: --kubeconfig` |
| 2 | `make up CLUSTER_PROVIDER=k3s-gcp` | Instance boots Debian; k3sup SSH fails or k3s install fails |
| 3 | `make up CLUSTER_PROVIDER=k3s-gcp` | Traefik occupies port 80/443; Istio IngressGateway fails to start |
| 4 | `make down CLUSTER_PROVIDER=k3s-gcp` | `kubectl config get-contexts` still shows `k3s-gcp` after teardown |
| 5 | Playwright fails to extract GCP_PROJECT | `gcp_get_credentials` returns 0 with empty `GCP_PROJECT` exported |

### Change 4 — `scripts/lib/providers/k3s-gcp.sh`: clean up kubeconfig on destroy

**Exact old block (lines 155–159):**

```bash
  _info "[k3s-gcp] Deleting compute instance ${_GCP_INSTANCE_NAME}..."
  gcloud compute instances delete "${_GCP_INSTANCE_NAME}" \
    --zone="${_GCP_ZONE}" --quiet || return 1

  _info "[k3s-gcp] Cluster destroyed."
```

**Exact new block:**

```bash
  _info "[k3s-gcp] Deleting compute instance ${_GCP_INSTANCE_NAME}..."
  gcloud compute instances delete "${_GCP_INSTANCE_NAME}" \
    --zone="${_GCP_ZONE}" --quiet || return 1

  _info "[k3s-gcp] Removing kubeconfig context k3s-gcp..."
  kubectl config delete-context k3s-gcp 2>/dev/null || true
  kubectl config delete-cluster k3s-gcp 2>/dev/null || true
  rm -f "${_GCP_KUBECONFIG}"

  _info "[k3s-gcp] Cluster destroyed."
```

### Change 5 — `scripts/plugins/gcp.sh`: parse to locals, validate, then export

**Exact old block (lines 23–31):**

```bash
  GCP_PROJECT=$(printf '%s\n' "${output}" | grep '^GCP_PROJECT=' | cut -d= -f2-)
  GOOGLE_APPLICATION_CREDENTIALS=$(printf '%s\n' "${output}" | grep '^GOOGLE_APPLICATION_CREDENTIALS=' | cut -d= -f2-)
  GCP_USERNAME=$(printf '%s\n' "${output}" | grep '^GCP_USERNAME=' | cut -d= -f2-)
  GCP_PASSWORD=$(printf '%s\n' "${output}" | grep '^GCP_PASSWORD=' | cut -d= -f2-)

  export GCP_PROJECT GOOGLE_APPLICATION_CREDENTIALS GCP_USERNAME GCP_PASSWORD

  _info "[gcp] GCP_PROJECT=${GCP_PROJECT}"
  _info "[gcp] GOOGLE_APPLICATION_CREDENTIALS=${GOOGLE_APPLICATION_CREDENTIALS}"
```

**Exact new block:**

```bash
  local project key_path username password
  project=$(printf '%s\n' "${output}" | grep '^GCP_PROJECT=' | cut -d= -f2-)
  key_path=$(printf '%s\n' "${output}" | grep '^GOOGLE_APPLICATION_CREDENTIALS=' | cut -d= -f2-)
  username=$(printf '%s\n' "${output}" | grep '^GCP_USERNAME=' | cut -d= -f2-)
  password=$(printf '%s\n' "${output}" | grep '^GCP_PASSWORD=' | cut -d= -f2-)

  if [[ -z "${project}" || "${project}" == "None" || "${project}" == "null" ]]; then
    _err "[gcp] Could not extract GCP_PROJECT from Playwright output"
    return 1
  fi
  if [[ -z "${key_path}" || ! -f "${key_path}" ]]; then
    _err "[gcp] GOOGLE_APPLICATION_CREDENTIALS not set or key file not found: ${key_path}"
    return 1
  fi

  export GCP_PROJECT="${project}"
  export GOOGLE_APPLICATION_CREDENTIALS="${key_path}"
  export GCP_USERNAME="${username}"
  export GCP_PASSWORD="${password}"

  _info "[gcp] GCP_PROJECT=${project}"
  _info "[gcp] GOOGLE_APPLICATION_CREDENTIALS=${key_path}"
```
