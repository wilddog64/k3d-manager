# v0.6.4 — Gemini Task: Linux k3s Validation Gate

## Context

v0.6.3 refactored `_ensure_path_exists`, `_start_k3s_service`, `_k3s_stage_file`,
`_install_k3s`, `_install_docker`, `_create_nfs_share`, and `deploy_cluster` platform
detection. All of these functions execute exclusively on the Linux/k3s code path.

The v0.6.3 cluster rebuild was on macOS/OrbStack — it never exercised the Linux paths.
This validation gate confirms the refactored code works correctly on a real Ubuntu machine
before `core.sh` and `system.sh` are extracted into `lib-foundation`.

---

## Critical Rules

1. **Do not modify any production code or test files.**
2. **Do not modify `memory-bank/`.** Claude owns memory-bank writes.
3. **Commit your own work** if you create any helper scripts — sign-off on what you did.
4. **Update memory-bank to report completion** — this is how you communicate back to Claude.
5. **If any step requires a sudo password prompt, stop and report immediately.**
6. **Report each phase individually** with full output on any failure.

---

## Environment

- **Target machine**: Ubuntu VM (`ssh ubuntu`)
- **Provider**: `CLUSTER_PROVIDER=k3s`
- **Branch**: `k3d-manager-v0.6.4` — ensure the repo on the Ubuntu VM is on this branch before starting

```bash
ssh ubuntu
cd <k3d-manager repo>
git fetch origin
git checkout k3d-manager-v0.6.4
```

---

## Task Sequence

### Phase 1 — Destroy existing k3s cluster

```bash
CLUSTER_PROVIDER=k3s ./scripts/k3d-manager destroy_cluster
```

Expected: k3s cluster torn down cleanly. No errors.

---

### Phase 2 — Install and create k3s cluster

```bash
CLUSTER_PROVIDER=k3s ./scripts/k3d-manager create_cluster
```

Expected: k3s installed, cluster created, kubeconfig configured.

**Watch for:**
- `_ensure_path_exists` creating `/etc/rancher/k3s/` via `_run_command --prefer-sudo`
- `_install_k3s` downloading and running the installer via `_run_command --prefer-sudo`
- `_start_k3s_service` starting via systemd (`_run_command --prefer-sudo -- systemctl start`)
- kubeconfig copied and chowned to calling user (not root-owned)

---

### Phase 3 — Deploy full stack

```bash
CLUSTER_PROVIDER=k3s ./scripts/k3d-manager deploy_cluster
```

Expected: Vault, ESO, Istio, OpenLDAP, Jenkins, ArgoCD, Keycloak all deployed and healthy.

**Watch for:**
- `_detect_platform` returning `debian` (Ubuntu is Debian-family)
- `deploy_cluster` dispatching correctly via `_detect_platform`
- No permission errors from collapsed `_run_command --prefer-sudo` calls

---

### Phase 4 — Smoke tests

Run each individually and report pass/fail:

```bash
CLUSTER_PROVIDER=k3s ./scripts/k3d-manager test_vault
CLUSTER_PROVIDER=k3s ./scripts/k3d-manager test_eso
CLUSTER_PROVIDER=k3s ./scripts/k3d-manager test_istio
```

---

```bash
CLUSTER_PROVIDER=k3s ./scripts/k3d-manager test all 2>&1
```

Report total pass count. Include full output for any failing test.
