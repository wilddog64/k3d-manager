# Bugfix: v1.6.0 вЂ” Auto-cleanup after acg-up failure

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/k3dm-webhook`

---

## Problem

When `make up` fails, `_finish("failed")` marks the job done and Gemini sends a failure
analysis to Slack вЂ” but the partially-provisioned AWS cluster (CloudFormation stack, EC2
nodes) keeps running and accruing ACG credit, requiring the user to manually trigger
`/acg-down` or `make down`.

**Root cause:** The `except` block in `_run_cluster` calls `_analyze_failure` then
`_finish("failed")` with no subsequent teardown step for `action == "up"` failures.

---

## Fix

Add a `_run_cleanup` function that runs `make down KEEP_LOCAL=1` in a daemon thread.
Fire it from the `except` block only when `action == "up"`.

`_finish("failed")` still runs immediately (marks job done, sends Slack summary).
Cleanup runs concurrently in the background and posts its own Slack notifications at
start and completion so the user knows teardown is happening.

---

### Change 1 вЂ” `bin/k3dm-webhook`: add `_run_cleanup` function

Insert this function immediately **before** `_run_cluster` (before line ~170 where
`def _run_cluster(` appears). Do not place it inside `_run_cluster`.

**Insert this block:**

```python
def _run_cleanup(provider):
    """Run make down KEEP_LOCAL=1 in the background after a failed acg-up."""
    _notify(f"рџ§№ *Cleanup triggered* ({provider}) вЂ” running `make down KEEP_LOCAL=1`")
    try:
        cmd = ["make", "down", "KEEP_LOCAL=1"]
        proc = subprocess.Popen(
            cmd,
            cwd=REPO_ROOT,
            stdout=subprocess.PIPE,
            stderr=subprocess.STDOUT,
            text=True,
            preexec_fn=os.setsid,
        )
        pgid = os.getpgid(proc.pid)
        out_lines = []

        def _drain():
            for line in proc.stdout:
                out_lines.append(line)

        reader = threading.Thread(target=_drain, daemon=True)
        reader.start()
        proc.wait()
        try:
            os.killpg(pgid, signal.SIGTERM)
        except (ProcessLookupError, OSError):
            pass
        reader.join(timeout=5)
        if proc.returncode == 0:
            _notify(f"вњ… *Cleanup complete* ({provider}) вЂ” AWS resources removed")
        else:
            _notify(f"вљ пёЏ *Cleanup failed* ({provider}) вЂ” `make down` exited {proc.returncode}; manual teardown may be needed")
    except Exception as exc:
        _notify(f"вљ пёЏ *Cleanup error* ({provider}): {exc}")
```

        _log(f"ERROR: {exc}")

            analysis = _analyze_failure(lines)
            _notify(f"рџ”Ќ *Failure analysis* (job `{job_id}`)\n{analysis}")

                target=_run_cleanup,
                args=(provider,),
                daemon=True,
            ).start()
