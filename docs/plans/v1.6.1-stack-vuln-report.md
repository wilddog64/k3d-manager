# v1.6.1 — Full-stack vulnerability scan + promotion pipeline

## Context

Extends the ArgoCD CVE pipeline (`v1.6.0`) to cover the entire k3d-manager stack.
A bi-weekly in-cluster CronJob on OCI scans all running workloads with `trivy k8s`,
generates a Markdown + JSON report, uploads it to OCI object storage, and notifies.

When CVEs require patching, a human reviews the report and triggers the same
three-stage promotion pipeline used by v1.6.0:

```
ACG (dev, via laptop webhook) → laptop infra hub (via laptop webhook) → OCI (in-cluster)
```

Auto-upgrade is intentionally **not done** for the full stack — Vault, Istio, Keycloak,
and Jenkins are stateful and must be tested before promotion.

**Prerequisites (must be shipped before this spec is implemented):**
1. `v1.5.1-oci-object-storage.md` — OCI object storage bucket + upload helpers
2. `v1.6.0-k3dm-webhook-server.md` — laptop webhook server
3. `v1.6.0-argocd-upgrade-notification.md` — notify.sh + notification Secret

## Promotion pipeline

The same webhook infrastructure from v1.6.0 is reused. The webhook server gains
a new endpoint `/api/v1/stack-upgrade` that accepts:

```json
{ "component": "vault", "chart_version": "0.31.0", "stage": "acg" }
```

On ACG/laptop stages, the webhook server runs:
```bash
make stack-upgrade COMPONENT=<component> VERSION=<version> CLUSTER_PROVIDER=<provider>
```

This maps to the new `stack_upgrade_component` function in the stack plugin.

## File 2: `scripts/etc/platform-ops/rbac.yaml`

Needs ClusterRole (read-only) to enumerate pods and Helm secrets across all namespaces.

```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: stack-scanner
  namespace: platform-ops
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: stack-scanner-readonly
  labels:
    managed-by: k3d-manager
rules:
  - apiGroups: [""]
    resources: ["pods", "namespaces", "nodes", "secrets", "configmaps"]
    verbs: ["get", "list"]
  - apiGroups: ["apps"]
    resources: ["deployments", "statefulsets", "daemonsets", "replicasets"]
    verbs: ["get", "list"]
  - apiGroups: ["batch"]
    resources: ["jobs", "cronjobs"]
    verbs: ["get", "list"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: stack-scanner-readonly
  labels:
    managed-by: k3d-manager
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: stack-scanner-readonly
subjects:
  - kind: ServiceAccount
    name: stack-scanner
    namespace: platform-ops
```

**Justification for ClusterRole:** `trivy k8s` must enumerate pods and Helm release
Secrets (prefixed `sh.helm.release`) across all namespaces to build the component list.
Read-only verbs only — no write access granted.

                  valueFrom:
                    secretKeyRef:
                      name: laptop-webhook-token
                      key: token
                      optional: true
                - name: SENDGRID_API_KEY
                  valueFrom:
                    secretKeyRef:
                      name: platform-ops-notifications
                      key: sendgrid-api-key
                      optional: true
                - name: NOTIFICATION_EMAIL
                  valueFrom:
                    secretKeyRef:
                      name: platform-ops-notifications
                      key: notification-email
                      optional: true

server runs on the laptop which already has `~/.oci/config` configured.

## File 4: `scripts/etc/platform-ops/stack-scan.sh`

```bash
#!/bin/sh
# Full-stack CVE scan for k3d-manager components.
# Runs inside the CronJob pod on OCI.
# Outputs: JSON + Markdown report, sent to laptop webhook for OCI storage upload.
# Triggers: notification with summary. Does NOT auto-upgrade (human decides).

set -eu

WEBHOOK_URL="${WEBHOOK_URL:-https://webhook.3ai-talk.org}"
LAPTOP_WEBHOOK_TOKEN="${LAPTOP_WEBHOOK_TOKEN:-}"
REPORT_DATE=$(date +%Y-%m-%d)
REPORT_JSON="/tmp/stack-scan-${REPORT_DATE}.json"
REPORT_MD="/tmp/stack-scan-${REPORT_DATE}.md"

_log()    { echo "[stack-scan] $*"; }
_notify() { [ -f /scripts/notify.sh ] && /scripts/notify.sh "$@" || true; }

# --- Scan ---

_log "Starting trivy k8s cluster scan (HIGH/CRITICAL)..."
trivy k8s \
  --severity HIGH,CRITICAL \
  --report summary \
  --all-namespaces \
  --format json \
  --output "${REPORT_JSON}" \
  cluster 2>/dev/null || true

if [ ! -s "${REPORT_JSON}" ]; then
  _log "Scan produced no output — cluster unreachable or trivy error"
  exit 1
fi

# --- Build Markdown report ---

TOTAL_CRIT=$(grep -o '"Severity":"CRITICAL"' "${REPORT_JSON}" | wc -l | tr -d ' ')
TOTAL_HIGH=$(grep -o '"Severity":"HIGH"' "${REPORT_JSON}"     | wc -l | tr -d ' ')
TOTAL=$(( TOTAL_CRIT + TOTAL_HIGH ))

cat > "${REPORT_MD}" <<EOF
# Stack Vulnerability Report — ${REPORT_DATE}

Generated by: k3d-manager stack-vuln-scan CronJob (OCI)
Scan scope: all namespaces, HIGH + CRITICAL only

## Summary

| Severity | Count |
|----------|-------|
| CRITICAL | ${TOTAL_CRIT} |
| HIGH     | ${TOTAL_HIGH} |
| **Total**| **${TOTAL}** |

## Component Detail

See attached JSON for full CVE list per component.

## Patch Workflow

1. Review JSON report in OCI object storage: k3d-manager-oci/stack-scan/
2. Identify components needing patch and target version
3. Run: make stack-upgrade COMPONENT=<name> VERSION=<version> CLUSTER_PROVIDER=acg
4. Verify on ACG, then promote:
   make stack-upgrade COMPONENT=<name> VERSION=<version> CLUSTER_PROVIDER=k3s-remote
   make stack-upgrade COMPONENT=<name> VERSION=<version> CLUSTER_PROVIDER=k3s-oci
EOF

_log "Report: CRITICAL=${TOTAL_CRIT} HIGH=${TOTAL_HIGH} TOTAL=${TOTAL}"

# --- Upload report via laptop webhook ---

if [ -n "${LAPTOP_WEBHOOK_TOKEN}" ]; then
  REPORT_B64=$(base64 < "${REPORT_JSON}" | tr -d '\n')
  MD_B64=$(base64 < "${REPORT_MD}" | tr -d '\n')

  curl -sf --max-time 30 \
    -X POST "${WEBHOOK_URL}/api/v1/stack-scan-upload" \
    -H "Authorization: Bearer ${LAPTOP_WEBHOOK_TOKEN}" \
    -H "Content-Type: application/json" \
    -d "{\"date\":\"${REPORT_DATE}\",\"json_b64\":\"${REPORT_B64}\",\"md_b64\":\"${MD_B64}\"}" \
    >/dev/null 2>&1 || _log "Webhook upload failed — report stays local only"
else
  _log "No webhook token — skipping OCI storage upload"
fi

# --- Notify ---

if [ "${TOTAL}" -eq 0 ]; then
  _log "No HIGH/CRITICAL CVEs found — stack clean"
  _notify info \
    "Stack scan clean — ${REPORT_DATE}" \
    "No HIGH or CRITICAL CVEs found across all namespaces"
  exit 0
fi

_notify warning \
  "Stack CVEs found: CRITICAL=${TOTAL_CRIT} HIGH=${TOTAL_HIGH} — ${REPORT_DATE}" \
  "Review report in OCI storage: k3d-manager-oci/stack-scan/stack-scan-${REPORT_DATE}.json
Patch workflow: make stack-upgrade COMPONENT=<name> VERSION=<version>"

_log "Done — ${TOTAL} findings require review"
exit 0
```

## File 5: `scripts/plugins/platform_ops.sh`

```bash
#!/usr/bin/env bash
# scripts/plugins/platform_ops.sh
# Platform operations: stack vulnerability scan + component upgrade promotion.

function scan_stack() {
  _info "[platform-ops] Deploying stack-vuln-scan CronJob to OCI cluster..."
  local _dir="${K3D_MANAGER_ROOT}/scripts/etc/platform-ops"

  for _file in namespace.yaml rbac.yaml; do
    _kubectl apply -f "${_dir}/${_file}"
  done

  envsubst '$OCI_BUCKET_NAME $OCI_REGION' \
    < "${_dir}/stack-scan-cronjob.yaml" | _kubectl apply -f -

  if [[ -f "${_dir}/notify.sh" ]]; then
    _kubectl create configmap stack-scan-script \
      --from-file=stack-scan.sh="${_dir}/stack-scan.sh" \
      --from-file=notify.sh="${_dir}/notify.sh" \
      --namespace platform-ops \
      --dry-run=client -o yaml | _kubectl apply -f -
  else
    _kubectl create configmap stack-scan-script \
      --from-file=stack-scan.sh="${_dir}/stack-scan.sh" \
      --namespace platform-ops \
      --dry-run=client -o yaml | _kubectl apply -f -
    _warn "[platform-ops] notify.sh missing — implement v1.6.0-argocd-upgrade-notification.md first"
  fi

  _info "[platform-ops] CronJob deployed — runs 1st and 15th of each month at 02:00 UTC"
  _info "[platform-ops] To run immediately: kubectl create job stack-vuln-scan-manual --from=cronjob/stack-vuln-scan -n platform-ops"
}

function stack_upgrade_component() {
  local _component="" _version="" _stage=""

  while [[ $# -gt 0 ]]; do
    case "$1" in
      --component) _component="$2"; shift 2 ;;
      --version)   _version="$2";   shift 2 ;;
      --stage)     _stage="$2";     shift 2 ;;
      *) _err "[platform-ops] Unknown argument: $1"; return 1 ;;
    esac
  done

  if [[ -z "${_component}" || -z "${_version}" ]]; then
    _err "[platform-ops] Usage: stack_upgrade_component --component <name> --version <ver> [--stage <acg|infra|oci>]"
    return 1
  fi

  _stage="${_stage:-oci}"
  _info "[platform-ops] Upgrading ${_component} to ${_version} on stage=${_stage}..."

  case "${_component}" in
    argocd)
      ARGOCD_HELM_CHART_VERSION="${_version}" \
        ./scripts/k3d-manager deploy_argocd ;;
    vault)
      VAULT_CHART_VERSION="${_version}" \
        ./scripts/k3d-manager deploy_vault ;;
    cert-manager)
      CERT_MANAGER_HELM_CHART_VERSION="${_version}" \
        ./scripts/k3d-manager deploy_cert_manager ;;
    eso)
      ESO_HELM_CHART_VERSION="${_version}" \
        ./scripts/k3d-manager deploy_eso ;;
    keycloak)
      KEYCLOAK_HELM_CHART_VERSION="${_version}" \
        ./scripts/k3d-manager deploy_keycloak ;;
    ldap)
      LDAP_HELM_CHART_VERSION="${_version}" \
        ./scripts/k3d-manager deploy_ldap ;;
    *)
      _err "[platform-ops] Unknown component: ${_component}"
      _err "[platform-ops] Supported: argocd vault cert-manager eso keycloak ldap"
      return 1 ;;
  esac

  _info "[platform-ops] ${_component} upgraded to ${_version} on ${CLUSTER_PROVIDER}"
}
```

## Makefile targets

Add to `Makefile`:

```makefile
## Run full-stack CVE scan immediately (triggers the CronJob as a one-shot Job)
vuln-scan:
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-oci) CLUSTER_PROVIDER=k3s-oci ./scripts/k3d-manager scan_stack ;; \
	  *)       echo "[make] vuln-scan deploys the CronJob — run with CLUSTER_PROVIDER=k3s-oci" ; exit 1 ;; \
	esac

## Upgrade a stack component through ACG → laptop → OCI promotion
## Usage: make stack-upgrade COMPONENT=vault VERSION=0.31.0 CLUSTER_PROVIDER=k3s-oci
stack-upgrade:
	@if [ -z "$(COMPONENT)" ] || [ -z "$(VERSION)" ]; then \
	  echo "[make] Usage: make stack-upgrade COMPONENT=<name> VERSION=<version> CLUSTER_PROVIDER=<provider>"; \
	  exit 1; \
	fi
	@CLUSTER_PROVIDER=$(CLUSTER_PROVIDER) ./scripts/k3d-manager \
	  stack_upgrade_component --component "$(COMPONENT)" --version "$(VERSION)"
```

Add `vuln-scan stack-upgrade` to `.PHONY`.

## Promotion workflow (human-driven)

After reviewing the scan report:

```bash
# Step 1 — Test on ACG
make stack-upgrade COMPONENT=vault VERSION=0.31.0 CLUSTER_PROVIDER=acg

# Step 2 — Promote to laptop infra hub
make stack-upgrade COMPONENT=vault VERSION=0.31.0 CLUSTER_PROVIDER=k3s-remote

# Step 3 — Promote to OCI
make stack-upgrade COMPONENT=vault VERSION=0.31.0 CLUSTER_PROVIDER=k3s-oci
```

The laptop webhook server (`v1.6.0-k3dm-webhook-server.md`) exposes
`POST /api/v1/stack-upgrade` to allow the OCI CronJob to trigger ACG/laptop
upgrades remotely in future automation.

## What NOT to Do

- Do NOT auto-upgrade any component — scan and notify only; human decides what to patch
- Do NOT grant write verbs to the ClusterRole — read-only only
- Do NOT use `latest` Trivy image tag
- Do NOT fail hard when webhook is unreachable — log + continue
- Do NOT create a PR
- Do NOT skip pre-commit hooks (`--no-verify`)
- Do NOT modify files outside the listed targets
- Do NOT commit to `main`
