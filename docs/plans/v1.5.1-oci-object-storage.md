# v1.5.1 — OCI Object Storage integration

## Goal

Use OCI Always Free object storage (10 GB free forever) to back up k3s state and
kubeconfig from the OCI cluster. Adds `make backup` and `make restore` targets for
`CLUSTER_PROVIDER=k3s-oci`.

## Always Free quota

- 10 GB standard storage
- 50,000 API requests/month
- No egress charges within the same region

## Scope

**New file:** `scripts/lib/providers/k3s-oci-storage.sh`
**Modified:** `scripts/lib/providers/k3s-oci.sh` — source storage plugin + call `_oci_storage_ensure_bucket` at end of deploy
**Modified:** `Makefile` — add `backup` and `restore` targets

## Bucket design

| Item | OCI path |
|------|----------|
| Bucket name | `k3d-manager-oci` (created once, idempotent) |
| k3s etcd snapshot | `k3s-oci/etcd/k3s-etcd-<timestamp>.db` |
| kubeconfig | `k3s-oci/kubeconfig/k3s-oci.yaml` |

---

## Public functions

### `oci_backup`
```
Usage: CLUSTER_PROVIDER=k3s-oci ./scripts/k3d-manager oci_backup
       make backup CLUSTER_PROVIDER=k3s-oci
```
Steps:
1. Ensure bucket exists (`_oci_storage_ensure_bucket`)
2. SSH to instance → trigger k3s etcd snapshot → download to `/tmp/k3s-etcd-<timestamp>.db`
3. Upload snapshot to `k3s-oci/etcd/k3s-etcd-<timestamp>.db`
4. Upload current `~/.kube/k3s-oci.yaml` to `k3s-oci/kubeconfig/k3s-oci.yaml`
5. Print object URLs

### `oci_restore`
```
Usage: CLUSTER_PROVIDER=k3s-oci ./scripts/k3d-manager oci_restore [--snapshot <name>]
       make restore CLUSTER_PROVIDER=k3s-oci
```
Steps:
1. List available snapshots in bucket
2. Download latest (or named) snapshot
3. SSH to instance → restore etcd snapshot
4. Download kubeconfig → merge into `~/.kube/k3s-oci.yaml`

## Private helpers

```bash
_oci_storage_ensure_bucket        # create bucket if not exists (idempotent)
_oci_storage_upload <local> <obj> # oci os object put wrapper
_oci_storage_download <obj> <local> # oci os object get wrapper
_oci_storage_list                  # list objects in bucket
```

## Makefile targets

```makefile
## Backup k3s etcd snapshot + kubeconfig to OCI object storage (k3s-oci only)
backup:
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-oci) CLUSTER_PROVIDER=k3s-oci ./scripts/k3d-manager oci_backup ;; \
	  *)       echo "[make] backup only supported for CLUSTER_PROVIDER=k3s-oci" ; exit 1 ;; \
	esac

## Restore k3s etcd snapshot + kubeconfig from OCI object storage (k3s-oci only)
restore:
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-oci) CLUSTER_PROVIDER=k3s-oci ./scripts/k3d-manager oci_restore ;; \
	  *)       echo "[make] restore only supported for CLUSTER_PROVIDER=k3s-oci" ; exit 1 ;; \
	esac
```

## Auto-backup on deploy

At the end of `_provider_k3s_oci_deploy_cluster` (Step 8/8), call `oci_backup` automatically
so the first backup is always taken right after a successful provision.
