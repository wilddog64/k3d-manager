## Context

lib-foundation **v0.4.0** absorbed lib-acg as the optional module `scripts/lib/acg/` (merged to
lib-foundation main `aed8c56`). k3d-manager currently carries **two** subtrees:

| Subtree | Prefix | Remote | Role |
|---------|--------|--------|------|
| lib-foundation | `scripts/lib/foundation/` | `lib-foundation` main | core libs (`system.sh`, `core.sh`, ‚Ä¶) ‚Äî and **now** the absorbed `scripts/lib/acg/` module |
| lib-acg (standalone) | `scripts/lib/acg/` | `lib-acg` main | the full nested legacy repo ‚Äî **to be removed** |

After this change k3d-manager has **one** subtree (lib-foundation), which carries the acg module at
`scripts/lib/foundation/scripts/lib/acg/`. lib-acg is then dead to k3d-manager (Phase 3 archives the repo).

**Antigravity status (already partly done upstream):** the absorbed module‚Äôs `_cdp_ensure_acg_session`
already targets the *Antigravity browser* via `node acg_session_check.js` + `_antigravity_browser_ready`
‚Äî **no `gemini`/`agy` prompt wrapper remains in the acg module**. The only legacy `gemini-cli` surface
left in k3d-manager is the **k3d-manager-owned** `scripts/plugins/gemini.sh` (GitHub Copilot review
automation), which still calls `gemini ‚Ä¶`. Part B retargets that to `agy`.

## Part B ‚Äî Antigravity (`agy`) wrapper retarget in `scripts/plugins/gemini.sh`

Google retired `@google/gemini-cli`; replace it with the Go-based **Antigravity CLI (`agy`)**.
`_ensure_agy_cli` and `_antigravity_browser_ready` already exist in the foundation subtree
(`scripts/lib/foundation/scripts/lib/system.sh`).

### B1 ‚Äî Add `_agy_prompt`, retire `_gemini_prompt`

Define `_agy_prompt` modeled on the current `_gemini_prompt` (the model-fallback loop), driving `agy`
instead of `gemini`. Replace the `gemini --model ‚Ä¶ --approval-mode yolo --prompt` /
`gemini --model ‚Ä¶ --prompt` calls with the `agy` equivalent using `--dangerously-skip-permissions`
(the agy replacement for `--approval-mode yolo` / `--yolo`).

> **VERIFY ‚Äî do not fabricate flags.** The exact `agy` invocation (how it takes a prompt and a model)
> is **not** documented in the upstream lib-acg v0.1.8 spec (`371d01e`), which left `_agy_prompt`‚Äôs body
> undefined. Run `agy --help` (install via `_ensure_agy_cli`) and confirm the prompt/model flag syntax
> before writing the call. The only flag confirmed by upstream is `--dangerously-skip-permissions`.
> Capture the verified `agy` command line in the commit message.

### B2 ‚Äî Retarget callers + naming

- `_ensure_gemini` ‚Üí use `_ensure_agy_cli` (already in foundation); drop the `npm install -g @google/gemini-cli` path.
- `_gemini_ensure_github_session`, `gemini_trigger_copilot_review`, `gemini_poll_task`: rename the
  `gemini_prompt` locals to `agy_prompt`, change `_gemini_prompt "$‚Ä¶ " --yolo` ‚Üí
  `_agy_prompt "$‚Ä¶" --dangerously-skip-permissions`, and rename user-facing strings
  "Gemini browser" ‚Üí "Antigravity browser".
- Decide public-function naming: keep `gemini_*` public names for back-compat **or** rename to `agy_*`
  with `gemini_*` shims. Recommend keeping `gemini_*` public names this release (no dispatcher churn)
  and only swapping the backend. State the choice in the commit message.

### B3 ‚Äî References scan

`grep -rn "gemini" scripts/ | grep -vE "foundation/"` should return only intentional residue
(e.g. the plugin filename or back-compat shims). No live `gemini` CLI invocation should remain.
