# v1.5.3 — OCI Multi-AD Capacity Retry

## Problem

`_oci_provision_instance` retries up to 288 times (24 h) against a **single**
availability domain (AD). When OCI has no A1 capacity in that AD the process
spins for up to a day before failing, with no attempt to try the other ADs in
the same region (Ashburn has AD-1, AD-2, AD-3; Phoenix has AD-1).

## Goal

On `InsufficientCapacity`, rotate to the next AD in the region before sleeping.
Full cycle through all ADs before taking the 5-minute back-off sleep. No change
to the overall 24-hour ceiling.

---

## Target File

```
scripts/lib/providers/k3s-oci.sh
```

---

## Design

### 1 — Regional subnet (remove AD pin from subnet)

OCI supports **regional** subnets (no `--availability-domain` flag). Instances
in any AD can be placed in a regional subnet. The current subnet is AD-scoped,
which locks instances to one AD anyway.

Change `_oci_provision_infrastructure` subnet creation: drop
`--availability-domain "${OCI_AVAILABILITY_DOMAIN}"`.

### 2 — Auto-discover ADs

Add a helper `_oci_list_ads` that calls:

```bash
oci iam availability-domain list \
  --compartment-id "${OCI_COMPARTMENT_ID}" \
  --query 'data[].name' --raw-output 2>/dev/null
```

Returns newline-separated AD names, e.g.:
```
qIZq:US-ASHBURN-AD-1
qIZq:US-ASHBURN-AD-2
qIZq:US-ASHBURN-AD-3
```

Cache result to `${_OCI_STATE_DIR}/availability-domains` after first call.

### 3 — Multi-AD rotation in `_oci_provision_instance`

Replace the current single-AD while loop with:

```bash
# Build AD array (auto-discovered or override)
local -a _ads=()
if [[ -n "${OCI_AVAILABILITY_DOMAIN:-}" ]]; then
  _ads=("${OCI_AVAILABILITY_DOMAIN}")
else
  while IFS= read -r _ad; do
    [[ -n "${_ad}" ]] && _ads+=("${_ad}")
  done < <(_oci_list_ads)
fi
if [[ "${#_ads[@]}" -eq 0 ]]; then
  _err "[k3s-oci] No availability domains found"
  return 1
fi

local _retry_interval=300 _max_cycles=96 _cycle=0
local _instance_id=""
while (( _cycle < _max_cycles )); do
  for _ad in "${_ads[@]}"; do
    _instance_id=$(oci compute instance launch \
      --compartment-id "${OCI_COMPARTMENT_ID}" \
      --availability-domain "${_ad}" \
      --shape "${_OCI_INSTANCE_SHAPE}" \
      --shape-config "{\"ocpus\":${_OCI_OCPUS},\"memoryInGBs\":${_OCI_MEMORY_GB}}" \
      --image-id "${OCI_IMAGE_ID}" \
      --subnet-id "${_subnet_id}" \
      --display-name "${_name}" \
      --assign-public-ip true \
      --ssh-authorized-keys-file "${_OCI_SSH_KEY}.pub" \
      --query 'data.id' --raw-output 2>/dev/null) || true
    if [[ -n "${_instance_id}" && "${_instance_id}" != "null" ]]; then
      _info "[k3s-oci] Instance '${_name}' launched in AD: ${_ad}"
      break 2
    fi
    _info "[k3s-oci] No capacity in ${_ad} for '${_name}' — trying next AD..."
  done
  (( _cycle++ )) || true
  _info "[k3s-oci] All ADs exhausted (cycle ${_cycle}/${_max_cycles}) — sleeping $(( _retry_interval / 60 )) min..."
  sleep "${_retry_interval}"
done
```

`96 cycles × 5 min = 8 hours` ceiling (down from 24h; more realistic expectation).

### 4 — `OCI_AVAILABILITY_DOMAIN` becomes optional

- If set: use only that AD (backward compatible; single-AD behaviour preserved).
- If unset: auto-discover and rotate through all ADs.

Remove `OCI_AVAILABILITY_DOMAIN` from the required-vars check in
`_oci_validate_prereqs`. Remove the prompt for it in first-time setup.
Remove it from the `${_OCI_STATE_DIR}/env` file write.

Update the `--help` block to reflect optional vs required status.

**B — New helper `_oci_list_ads` (insert after `_oci_reconfigure`, before `_oci_provision_infrastructure`)**

```bash
function _oci_list_ads() {
  local _cache="${_OCI_STATE_DIR}/availability-domains"
  if [[ -f "${_cache}" ]]; then
    cat "${_cache}"
    return 0
  fi
  local _ads
  _ads=$(oci iam availability-domain list \
    --compartment-id "${OCI_COMPARTMENT_ID}" \
    --query 'data[].name' --raw-output 2>/dev/null || true)
  if [[ -z "${_ads}" ]]; then
    _err "[k3s-oci] Could not list availability domains for ${OCI_REGION}"
    return 1
  fi
  mkdir -p "${_OCI_STATE_DIR}"
  printf '%s\n' "${_ads}" > "${_cache}"
  printf '%s\n' "${_ads}"
}
```
