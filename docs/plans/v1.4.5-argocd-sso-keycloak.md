## Problem

ArgoCD UI SSO via Keycloak fails for three reasons:

1. **`argocd-cm.yaml` issuer is an internal cluster URL** — `http://keycloak.identity.svc.cluster.local/realms/shopping-cart` — the browser is redirected to this hostname during OIDC auth but cannot resolve it.
2. **All three `argocd/config/` files use `namespace: argocd`** — ArgoCD is deployed in `cicd` (see `ARGOCD_NAMESPACE` in `scripts/etc/argocd/vars.sh`). The configs are applied to the wrong namespace and are silently ignored.
3. **`argocd-secret.yaml` has placeholder `argocd-oidc-secret-changeme`** — the real OIDC client secret is generated by `acg-up` and stored in Vault; it is never injected into the ArgoCD secret.

## Changes

### shopping-cart-infra — Branch: `docs/next-improvements`

#### Change 1 — `identity/keycloak/configmap.yaml`: add KC_HOSTNAME

**Exact old block:**
```yaml
  KC_HOSTNAME_STRICT: "false"
  KC_PROXY: edge
```

**Exact new block:**
```yaml
  KC_HOSTNAME: keycloak.shopping-cart.local
  KC_HOSTNAME_STRICT: "false"
  KC_PROXY: edge
```

This makes Keycloak's OIDC discovery doc and token `iss` claim use `keycloak.shopping-cart.local` regardless of the incoming request's Host header. ArgoCD's `issuer` config and the JWT `iss` claim must match; this sets the canonical value.

#### Change 4 — `argocd/config/argocd-secret.yaml`: replace with ExternalSecret

Delete the entire file and replace it with the following ExternalSecret. The `creationPolicy: Merge` policy adds `oidc.keycloak.clientSecret` to the pre-existing `argocd-secret` without overwriting ArgoCD's own keys (admin password, TLS cert, etc.).

**New file content:**
```yaml
---
apiVersion: external-secrets.io/v1
kind: ExternalSecret
metadata:
  name: argocd-oidc-secret
  namespace: cicd
  annotations:
    argocd.argoproj.io/sync-wave: "0"
  labels:
    app.kubernetes.io/name: argocd-oidc-secret
    app.kubernetes.io/part-of: argocd
spec:
  refreshInterval: 24h
  secretStoreRef:
    name: vault-backend
    kind: ClusterSecretStore
  target:
    name: argocd-secret
    creationPolicy: Merge
  data:
    - secretKey: oidc.keycloak.clientSecret
      remoteRef:
        key: secret/data/keycloak/clients
        property: argocd_client_secret
```

### k3d-manager — Branch: `k3d-manager-v1.4.5`

#### Change 5 — `bin/acg-up`: add Step 10e — SSO wiring

**Exact insertion point — after this block (end of the realm import section, before Step 11):**
```bash
    kill "${_kc_pf_pid}" 2>/dev/null || true
  fi
fi


_info "[acg-up] Step 11/14 — Verifying ClusterSecretStore..."
```

**Insert this block between the `fi` and Step 11:**
```bash

_info "[acg-up] Step 10e/14 — Wiring ArgoCD SSO (Keycloak VirtualService + argocd-cm + secret patch)..."

# Apply Keycloak VirtualService so in-cluster pods reach keycloak.shopping-cart.local via Istio
KEYCLOAK_VIRTUALSERVICE_HOST=keycloak.shopping-cart.local \
KEYCLOAK_VIRTUALSERVICE_GATEWAY=istio-system/default-gateway \
KEYCLOAK_NAMESPACE=identity \
KEYCLOAK_SERVICE_PORT=80 \
  envsubst < "${REPO_ROOT}/scripts/etc/keycloak/virtualservice.yaml.tmpl" | \
  kubectl apply --context k3d-k3d-cluster -f -

# Patch CoreDNS so in-cluster pods resolve keycloak.shopping-cart.local → IngressGateway IP
_gw_ip=$(kubectl get svc istio-ingressgateway -n istio-system --context k3d-k3d-cluster \
  -o jsonpath='{.status.loadBalancer.ingress[0].ip}' 2>/dev/null || echo "")
if [[ -n "${_gw_ip}" ]]; then
  _cf=$(kubectl get cm coredns -n kube-system --context k3d-k3d-cluster \
    -o jsonpath='{.data.Corefile}')
  if ! printf '%s' "${_cf}" | grep -qF 'keycloak.shopping-cart.local'; then
    _new_cf=$(printf '%s' "${_cf}" | awk \
      -v ip="${_gw_ip}" \
      '/^    health \{/{print "    hosts {"; printf "      %s keycloak.shopping-cart.local\n", ip; print "      fallthrough"; print "    }"} {print}')
    kubectl patch cm coredns -n kube-system --context k3d-k3d-cluster \
      --type merge \
      --patch "$(printf '{"data":{"Corefile":%s}}' "$(printf '%s' "${_new_cf}" | jq -Rs .)")"
    kubectl rollout restart deployment/coredns -n kube-system --context k3d-k3d-cluster
    kubectl rollout status deployment/coredns -n kube-system --context k3d-k3d-cluster --timeout=60s
    _info "[acg-up] CoreDNS patched: keycloak.shopping-cart.local → ${_gw_ip}"
  fi
else
  _info "[acg-up] WARNING: IngressGateway LB IP not found — CoreDNS not patched; ArgoCD SSO in-cluster resolution may fail"
fi

# Apply argocd-cm and argocd-rbac-cm from shopping-cart-infra (now using namespace: cicd)
_infra_root="${REPO_ROOT}/../shopping-carts/shopping-cart-infra"
kubectl apply --context k3d-k3d-cluster \
  -f "${_infra_root}/argocd/config/argocd-cm.yaml" \
  -f "${_infra_root}/argocd/config/argocd-rbac-cm.yaml"

# Patch argocd-secret directly with the OIDC client secret generated earlier in this run.
# The ExternalSecret (argocd-oidc-secret) will take over once ClusterSecretStore is Ready;
# this immediate patch ensures ArgoCD SSO works on first boot before ESO syncs.
kubectl patch secret argocd-secret -n cicd --context k3d-k3d-cluster \
  --type merge \
  --patch "{\"stringData\":{\"oidc.keycloak.clientSecret\":\"${_argocd_client_secret}\"}}"

# Add /etc/hosts entry on the Mac host so the browser reaches keycloak.shopping-cart.local
if ! grep -qF 'keycloak.shopping-cart.local' /etc/hosts; then
  _run_command --require-sudo -- \
    sh -c 'printf "127.0.0.1 keycloak.shopping-cart.local\n" >> /etc/hosts'
  _info "[acg-up] /etc/hosts: added keycloak.shopping-cart.local → 127.0.0.1"
fi

# Restart argocd-server to pick up the updated argocd-cm and argocd-secret
kubectl rollout restart deployment/argocd-server -n cicd --context k3d-k3d-cluster
kubectl rollout status deployment/argocd-server -n cicd --context k3d-k3d-cluster --timeout=120s
_info "[acg-up] ArgoCD SSO wired: login at https://argocd.shopping-cart.local → Keycloak realm shopping-cart"
```

## How SSO Works After This Change

```
Browser → https://argocd.shopping-cart.local
  → ArgoCD server redirects to http://keycloak.shopping-cart.local/realms/shopping-cart/protocol/openid-connect/auth
  → Browser reaches Keycloak via /etc/hosts (127.0.0.1) + Istio IngressGateway VirtualService
  → User logs in with LDAP credentials (alice / Admin@Cart2024)
  → Keycloak redirects back to https://argocd.shopping-cart.local/auth/callback with auth code
  → ArgoCD server exchanges code for token via http://keycloak.identity.svc.cluster.local (in-cluster)
     (CoreDNS resolves keycloak.shopping-cart.local → IngressGateway IP for server-side calls)
  → ArgoCD maps Keycloak groups → RBAC roles via argocd-rbac-cm
```

Test users from LDAP:
- `admin@shopping-cart.local` / `Admin@Cart2024` → `platform-admins` group → ArgoCD `role:admin`
- `developer@shopping-cart.local` / `Dev@Cart2024` → `argocd-developers` group → ArgoCD `role:argocd-developer`
