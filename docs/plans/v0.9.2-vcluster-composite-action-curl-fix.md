## Context

Copilot posted two new findings on PR #34 after the `action_path` fix:

**Finding 1 (id: 2937289569)** — `curl -s` brittle and high-risk:
> `curl -s "$installer" | sudo bash` won't fail clearly on 404/redirects and
> executes remote content as root without error checking. Use `curl -fsSL` at
> minimum; prefer downloading first then executing separately.

**Finding 2 (id: 2937289578)** — `k3d-version` input not validated:
> The input is interpolated directly into the raw.githubusercontent.com URL.
> A caller setting `k3d-version: main` reintroduces the supply-chain risk.
> Validate the input against a `vX.Y.Z` pattern before use.

## Step 2 — Fix `curl` in Install k3d step

File: `.github/actions/vcluster-e2e-setup/action.yml`

Replace the `Install k3d` step `run` block:

**Before:**
```yaml
      run: |
        set -euo pipefail
        base_url="https://raw.githubusercontent.com/k3d-io/k3d"
        installer="${base_url}/${{ inputs.k3d-version }}/install.sh"
        curl -s "$installer" | sudo bash
```

**After:**
```yaml
      run: |
        set -euo pipefail
        k3d_version="${{ inputs.k3d-version }}"
        if [[ ! "$k3d_version" =~ ^v[0-9]+\.[0-9]+\.[0-9]+$ ]]; then
          echo "Error: k3d-version must match vX.Y.Z (got: $k3d_version)" >&2
          exit 1
        fi
        base_url="https://raw.githubusercontent.com/k3d-io/k3d"
        installer="${base_url}/${k3d_version}/install.sh"
        tmp_script="$(mktemp /tmp/k3d-install.XXXXXX)"
        trap 'rm -f "$tmp_script"' EXIT
        curl -fsSL "$installer" -o "$tmp_script"
        chmod 600 "$tmp_script"
        sudo bash "$tmp_script"
```

This addresses both findings:
- `-fsSL`: fails on HTTP errors (`-f`), shows errors in silent mode (`-S`), follows redirects (`-L`)
- Downloads to a unique temp file (single `mktemp` call stored in `$tmp_script`), then executes the same file — avoids piping remote content directly to shell and prevents TOCTOU
- Validates `k3d-version` matches `vX.Y.Z` before use — blocks `main` or path-like values
