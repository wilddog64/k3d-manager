# Plan: v1.2.0 — Extract ACG Browser Automation into lib-acg

**Status:** DESIGN — implementation tasks to be specced per-phase before Codex handoff
**Supersedes:** `docs/plans/v1.1.0-acg-extraction-repo-split.md` (high-level; this is the executable plan)

---

## Problem

See `docs/plans/v1.1.0-acg-extraction-repo-split.md` for full motivation. Short form:
browser automation and CDP-specific code destabilizes `k3d-manager` orchestration whenever
GCP or provider-specific UI flows evolve. The extraction gives browser automation its own
release cadence, test loop, and provider-isolated file structure.

## lib-acg Repo Structure

```
lib-acg/
├── scripts/
│   ├── lib/
│   │   └── cdp.sh            # _browser_launch, _cdp_ensure_acg_session
│   ├── plugins/
│   │   ├── acg.sh            # ACG sandbox lifecycle
│   │   └── gcp.sh            # GCP identity bridge
│   └── vars.sh               # PLAYWRIGHT_AUTH_DIR, PLAYWRIGHT_CDP_PORT, URLs
├── playwright/
│   ├── acg_credentials.js
│   ├── acg_extend.js
│   └── gcp_login.js
├── package.json
├── CLAUDE.md
└── README.md
```

lib-acg will pull lib-foundation via git subtree (`scripts/lib/foundation/`) — same
pattern as k3d-manager — so `_antigravity_browser_ready` and friends are available
inside lib-acg without copying.

### Integration into k3d-manager

lib-acg is pulled into k3d-manager via git subtree at `scripts/lib/acg/`:

```bash
git subtree add --prefix=scripts/lib/acg \
  https://github.com/wilddog64/lib-acg.git main --squash
```

Stubs in `scripts/plugins/` forward to lib-acg so the existing dispatcher requires
no changes:

```bash
# scripts/plugins/acg.sh (stub — replaces the moved file)
source "${REPO_ROOT}/scripts/lib/acg/scripts/plugins/acg.sh"

# scripts/plugins/gcp.sh (stub)
source "${REPO_ROOT}/scripts/lib/acg/scripts/plugins/gcp.sh"
```

`gemini.sh` sources lib-acg's CDP primitive directly:

```bash
# near top of scripts/plugins/gemini.sh
source "${REPO_ROOT}/scripts/lib/acg/scripts/lib/cdp.sh"
```

---

## Interface Contract

k3d-manager consumes lib-acg through two surfaces only:

1. **Shell plugin stubs** — `scripts/plugins/acg.sh` and `scripts/plugins/gcp.sh` forward
   to `scripts/lib/acg/scripts/plugins/`. All public function names stay identical.

2. **CDP primitive** — `scripts/lib/acg/scripts/lib/cdp.sh` exports `_browser_launch`
   and `_cdp_ensure_acg_session`. `gemini.sh` sources this file directly.

Playwright scripts remain at the same relative paths inside the subtree. References in
`acg.sh` and `gcp.sh` use `${REPO_ROOT}/scripts/lib/acg/playwright/`.

- Do NOT create a PR for Phase 4 until Phase 3 lib-acg is independently green.
