# v1.4.5 — `make up TRUST_CA=1` host CA trust bootstrap

**Branch:** `k3d-manager-v1.4.5`

---

## Context

The Argo CD browser SSO flow now depends on a local Vault-issued TLS listener
for `https://argocd.shopping-cart.local`. That certificate must be trusted by
the host browser for Safari to complete SSO cleanly.

The trust-store step should remain opt-in, but it needs to be smooth enough
that a user can enable it with a single `make up TRUST_CA=1` invocation. The
helper should detect the host OS internally instead of making `make` itself
platform-aware.

---

## Goal

Add a single opt-in trust bootstrap so a user can run:

```bash
make up TRUST_CA=1
```

and have the local browser trust the Vault-issued Argo CD certificate on the
current host OS.

---

## Problem

The browser SSO path depends on a locally trusted CA, but the trust installation
is not yet wired into the main bootstrap in a host-agnostic way. If the trust
step is embedded directly in the bootstrap as macOS-only logic, the flow becomes
fragile. If it is omitted entirely, Safari cannot trust the canonical HTTPS
entrypoint.

---

## Desired Behavior

- `make up` remains the default bootstrap path and does not force a trust-store
  mutation unless explicitly requested.
- `make up TRUST_CA=1` runs the existing CA trust helper as part of the ACG
  bootstrap.
- The helper detects the host OS internally and performs the appropriate trust
  install for that platform.
- The main bootstrap stays ACG-aware, but not macOS-only.
- The browser SSO flow continues to use the canonical
  `https://argocd.shopping-cart.local` entrypoint.

---

## Scope

- Extend the existing host CA trust helper so it can auto-detect the platform
  and install trust for the current OS.
- Teach `make up` / `bin/acg-up` to invoke that helper only when
  `TRUST_CA=1` is set.
- Update docs so the new opt-in flag is discoverable.
- Add tests for the `TRUST_CA=1` path and the non-trusting default path.

---

## Non-Goals

- Do not move browser TLS issuance back to a raw TCP-only listener.
- Do not make `make` itself OS-aware.
- Do not touch `scripts/lib/core.sh` or `scripts/lib/system.sh`.
- Do not change the in-cluster Argo CD deployment.

---

## Acceptance Criteria

- `make up` still works without mutating the host trust store.
- `make up TRUST_CA=1` installs trust for the current host OS through a single
  documented flow.
- macOS and Linux both have a defined trust path.
- The browser SSO flow works from the canonical HTTPS hostname after the trust
  helper runs.
- The change is covered by tests and documented in the repo.
