## Context

`scripts/lib/foundation/scripts/lib/system.sh` is the canonical upstream version (lib-foundation
`feat/v0.3.4`, commit `b60ddc6`). `scripts/lib/system.sh` is the local copy k3d-manager uses
at runtime. The two are out of sync ŌĆö the local copy is missing:

- `# shellcheck shell=bash` header
- `_run_command_resolve_sudo()` helper function (replaces `_run_command_has_tty`)
- `resolver_rc` unified error handling in `_run_command`
- `--interactive-sudo` flag on all privileged `apt-get`, `dnf`, `systemctl`, `tee`, `gpg`, `usermod` calls (replaces bare `-- sudo <cmd>` pattern)
- `helm_global_arr` array split for `HELM_GLOBAL_ARGS` (fixes word-splitting bug)
- `return 1` instead of `exit -1` in one location
- `printf -v cmd_str` for safe error message formatting
- `# shellcheck disable=SC2016` annotations on trap strings
- Trap quoting fixes (`"${cur}; ${handler}"` instead of `'"$cur"; "$handler"'`)

**The fix is a direct file copy ŌĆö do not do it surgically line by line.**
