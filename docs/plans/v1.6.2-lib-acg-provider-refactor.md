# v1.6.2 — lib-acg: provider pattern refactor

**Target repo:** `lib-acg`
**Branch (lib-acg):** `feat/v0.2.1-provider-refactor` (freshly cut from current `main` — 672-line source)
**Branch (k3d-manager):** `k3d-manager-v1.6.2` (spec lives here; subtree pull after merge)

---

## Objective

`playwright/acg_credentials.js` is 672 lines — a single monolithic file containing
CDP browser setup, Pluralsight navigation, sign-in flow, sandbox start/wait logic,
extend-dialog handling, credential extraction for AWS, GCP, and a stub for Azure.
Adding Azure extraction or sandbox switching directly to this file risks breaking the
stable AWS flow.

Refactor into a module tree with isolated provider files and shared library modules.
**No behavior change.** AWS and GCP must work identically after the refactor.

---

## Target module structure

```
playwright/
  acg_credentials.js          ← thin orchestrator (~80 lines)
  lib/
    browser.js                ← CDP connect + launchPersistentContext fallback
    sandbox.js                ← page find, navigate, sign-in, start/wait for sandbox
    output.js                 ← _outputCredentials, _isFirstRun, AUTH_DIR
  providers/
    aws.js                    ← AWS credential extraction
    gcp.js                    ← GCP credential extraction (moved as-is; marked fragile)
    azure.js                  ← stub (throws "not yet implemented")
tests/
  providers/
    aws.test.js               ← unit tests for AWS extraction with mock page
    output.test.js            ← unit tests for _outputCredentials
```

### `playwright/acg_credentials.js` (after refactor)

Thin orchestrator — parses args, calls modules in sequence, cleans up:

```js
const { connectBrowser }     = require('./lib/browser');
const { findOrCreatePage, navigateToSandbox, waitForSkeleton, handleSignIn, startSandbox } = require('./lib/sandbox');
const { _outputCredentials, _isFirstRun, AUTH_DIR } = require('./lib/output');
const providers = {
  aws:   require('./providers/aws'),
  gcp:   require('./providers/gcp'),
  azure: require('./providers/azure'),
};

async function extractCredentials() {
  // 1. parse args
  // 2. validate provider
  // 3. log first-run bootstrap message if needed
  // 4. connectBrowser()
  // 5. findOrCreatePage()
  // 6. navigateToSandbox()
  // 7. waitForSkeleton()
  // 8. handleSignIn()
  // 9. startSandbox()
  // 10. providers[PROVIDER].extractCredentials(page, _outputCredentials)
  // finally: close page if created; disconnect CDP or close context
}

// Overall timeout: 780000ms (same as current)
Promise.race([extractCredentials(), _timeoutPromise])
  .then(...)
  .catch(...)
```

All logic moved to modules — orchestrator contains no inline browser, navigation, or
extraction logic.

```js
function makeMockPage(inputs) {

    locator: jest.fn().mockReturnValue({

          evaluateHandle: jest.fn().mockResolvedValue({
            evaluate: jest.fn().mockResolvedValue(inp.label),
          }),
        }))
      ),
    }),
  };
}
```

**Required test cases:**

| Test | Mock inputs | Expected output |
|------|-------------|-----------------|
| Label-matched extraction | labels containing "Access Key ID", "Secret Access Key", "Session Token" | `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_SESSION_TOKEN` all set |
| Positional fallback (no labels) | 5 inputs, empty labels | positions [2],[3],[4] used |

| No session token | only access key + secret | `AWS_SESSION_TOKEN` absent from output |
| outputFn called with correct keys | happy-path | spy called once with correct object |
