## Background

ArgoCD→Ubuntu k3s gRPC connectivity fails with i/o timeout despite TCP being reachable
(`curl -k https://10.211.55.14:6443/healthz` returns 401 from M2 Air).

Root cause hypothesis: k3s API server TLS cert does not include `10.211.55.14` as a SAN.
By default k3s only adds `127.0.0.1` and the node hostname. ArgoCD validates the server
certificate during the gRPC TLS handshake — if the IP is not in the SAN, the handshake fails.

This task fixes the SAN, re-exports kubeconfig, re-registers the cluster in ArgoCD, and
verifies end-to-end GitOps sync.

## Task 2 — Fix: Add TLS SAN to k3s Config

On Ubuntu, add `10.211.55.14` to k3s TLS SANs and restart:

```bash
ssh ubuntu "sudo bash -c 'mkdir -p /etc/rancher/k3s && \
  if grep -q tls-san /etc/rancher/k3s/config.yaml 2>/dev/null; then \
    echo \"tls-san entry already exists — edit manually\"; \
  else \
    echo -e \"tls-san:\n  - 10.211.55.14\n  - 127.0.0.1\n  - localhost\" >> /etc/rancher/k3s/config.yaml; \
  fi && cat /etc/rancher/k3s/config.yaml'"
```

Restart k3s to regenerate the API server cert:

```bash
ssh ubuntu "sudo systemctl restart k3s"
sleep 15
ssh ubuntu "sudo systemctl is-active k3s"
# Expected: active
```

Verify new cert has the SAN:

```bash
ssh ubuntu "sudo openssl s_client -connect 10.211.55.14:6443 </dev/null 2>/dev/null \
  | openssl x509 -noout -text | grep -A5 'Subject Alternative Name'"
# Expected: IP Address:10.211.55.14 present
```

**STOP — report SAN output and k3s status before continuing.**

## Task 3 — Re-export Kubeconfig and Re-register Cluster

Re-export kubeconfig with external IP:

```bash
ssh ubuntu "sudo cat /etc/rancher/k3s/k3s.yaml \
  | sed 's|127.0.0.1|10.211.55.14|g'" > ~/.kube/k3s-ubuntu.yaml
chmod 600 ~/.kube/k3s-ubuntu.yaml
```

Verify kubectl can reach the cluster:

```bash
KUBECONFIG=~/.kube/k3s-ubuntu.yaml kubectl get nodes
# Expected: node Ready
```

Re-register in ArgoCD (port-forward must be active):

```bash
nohup kubectl port-forward svc/argocd-server -n cicd 8080:443 > /tmp/argocd-pf.log 2>&1 &
sleep 5
curl -k https://localhost:8080/healthz
# Expected: Healthy

argocd login localhost:8080 --insecure
argocd cluster add k3s-automation \
  --name ubuntu-k3s \
  --kubeconfig ~/.kube/k3s-ubuntu.yaml \
  --insecure=false
argocd cluster list
# Expected: https://10.211.55.14:6443  ubuntu-k3s  Ready
```
