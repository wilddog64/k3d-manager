## Problem / Decision

Hostinger is now the permanent app cluster; the ephemeral ACG cloud sandbox (aws/gcp/az) is the
special case. But `/cluster-up`, `/cluster-status`, `/cluster-refresh` still default to the `aws`
sandbox. Flip the defaults so the everyday commands operate on Hostinger.

**Decision (user, 2026-06-14):**
- `/cluster-up`, `/cluster-status`, `/cluster-refresh` → **default to Hostinger**; the cloud
  sandbox is opt-in via an explicit `aws`/`gcp`/`az` arg.
- `/cluster-down` → **stays sandbox-default** (no destructive Hostinger default). Hostinger
  teardown only via explicit `/cluster-down hostinger`.
- `/cluster-resume` → **unchanged** (ACG checkpoint concept; out of scope).
- `/cluster-refresh` for Hostinger is **new functionality**: restore the `ubuntu-hostinger`
  kubeconfig context + re-register the hub ArgoCD cluster secret + verify `/healthz`.
  **Non-destructive** — no k3s reinstall.

Provider arg → `CLUSTER_PROVIDER` mapping gains `hostinger → k3s-hostinger`.

### B6 — new `_run_hostinger_refresh` (add immediately AFTER the `_run_hostinger_status` function)

Mirror `_run_hostinger_status` exactly: batch output in a `lines` list via `_log`, post once in
`_finish` (via `_slack_post` when `response_url`, else `_notify_job`), append the
`__WEBHOOK_SUCCESS__` sentinel **inside** the `bash -c` command, wrap in try/except.
Verified helper signatures: `_posix_spawn_capture(cmd, timeout, cwd=None, env=None)`,
`_slack_post(url, text)`, `_notify_job(job_id, text)`.

```python
def _run_hostinger_refresh(job_id, response_url, thread_ts=None):
    """Run make refresh for k3s-hostinger (restore kubeconfig + ArgoCD); post result."""
    job_dir = JOB_DIR / job_id
    if thread_ts:
        (job_dir / "thread_ts").write_text(thread_ts)
    lines = []

    def _log(msg):
        lines.append(msg + "\n")

    def _finish(status):
        (job_dir / "status").write_text(status)
        output = "".join(lines)
        (job_dir / "output").write_text(output)
        if response_url:
            _slack_post(response_url, output)
        else:
            _notify_job(job_id, output)

    try:
        (job_dir / "status").write_text("running")
        import shlex as _shlex_hr
        repo_root = Path(__file__).resolve().parent.parent
        _log("🔄 *Hostinger refresh* — restoring kubeconfig + ArgoCD registration…")
        refresh_out, refresh_timeout = _posix_spawn_capture(
            ["/bin/bash", "-c",
             f"cd {_shlex_hr.quote(str(repo_root))} && make refresh CLUSTER_PROVIDER=k3s-hostinger && echo __WEBHOOK_SUCCESS__"],
            timeout=180,
        )
        if refresh_timeout:
            _log("\n❌ *Hostinger refresh timed out* after 180s — run `make refresh CLUSTER_PROVIDER=k3s-hostinger` manually")
            _finish("failed")
            return
        if "__WEBHOOK_SUCCESS__" in refresh_out:
            _log("\n✅ *Hostinger refresh complete* — `ubuntu-hostinger` context restored")
        else:
            tail_lines = [l for l in refresh_out.strip().splitlines()[-8:] if l]
            _log("\n❌ *Hostinger refresh failed*")
            if tail_lines:
                _log("```\n{}\n```".format("\n".join(tail_lines)))
        _finish("success")
    except Exception as exc:
        _log(f"ERROR: {exc}")
        _finish("failed")
```

> This mirrors the current `_run_hostinger_status` structure exactly (lines→`_finish`→`_slack_post`,
> inline sentinel). Do NOT introduce `subprocess.run`.

> Thread-command parity (lines 607 + 637 dispatch): the thread forms `cluster-status` /
> `cluster-refresh` (typed in a Slack thread without slash) currently call `_run_cluster_status`
> / `_run_cluster_refresh`. Update each to parse an optional provider arg (default `hostinger`)
> and route to the hostinger function when provider is `hostinger`, mirroring B4/B5. Keep the
> short alias `refresh` routing to the sandbox path.

## Fix — Part E: `scripts/lib/providers/k3s-hostinger.sh` — `_provider_k3s_hostinger_refresh_cluster`

```bash
function _provider_k3s_hostinger_refresh_cluster() {
  _hostinger_require_host >/dev/null || return 1
  if [[ ! -f "${_HOSTINGER_KUBECONFIG}" ]]; then
    printf 'ERROR: %s\n' "[k3s-hostinger] saved kubeconfig ${_HOSTINGER_KUBECONFIG} missing — run deploy_cluster (/cluster-up) first" >&2
    return 1
  fi
  _info "[k3s-hostinger] Refreshing ${_HOSTINGER_KUBE_CONTEXT} kubeconfig + ArgoCD registration…"
  _hostinger_merge_kubeconfig || return 1
  _hostinger_register_cluster || return 1
  if kubectl --context "${_HOSTINGER_KUBE_CONTEXT}" get --raw='/healthz' >/dev/null 2>&1; then
    _info "[k3s-hostinger] Refresh complete — ${_HOSTINGER_KUBE_CONTEXT} reachable"
    printf '%s\n' "__WEBHOOK_SUCCESS__"
  else
    printf 'ERROR: %s\n' "[k3s-hostinger] ${_HOSTINGER_KUBE_CONTEXT} still unreachable after refresh" >&2
    return 1
  fi
}
```

> The `__WEBHOOK_SUCCESS__` sentinel print on success is what `_run_hostinger_refresh` greps for.
> Confirm `_hostinger_require_host` prints the host to stdout (redirect to /dev/null here if so),
> and that `_HOSTINGER_KUBECONFIG` / `_HOSTINGER_KUBE_CONTEXT` are the actual var names in this file.
