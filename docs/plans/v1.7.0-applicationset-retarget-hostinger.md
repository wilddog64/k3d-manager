## Problem

The ACG EC2 app cluster (`ubuntu-k3s`) is being retired in favour of the permanent Hostinger box
(`ubuntu-hostinger`). Two ApplicationSets still hardcode the old destination, so shopping-cart and
observability workloads still target the gone ACG cluster:

- `services-git.yaml:40` → `destination.name: ubuntu-k3s`
- `observability-acg.yaml:32` → `destination.name: ubuntu-k3s`

This is exactly why `make status CLUSTER_PROVIDER=k3s-hostinger` shows those ArgoCD apps as
`SYNC STATUS: Unknown` — ArgoCD cannot reach the destination cluster they name.

**Root cause:** the destination cluster name is a hardcoded literal; the ApplicationSet render
(`_argocd_deploy_applicationsets`) only substitutes `$ARGOCD_NAMESPACE` and `$K3D_MANAGER_BRANCH`,
so there is no way to point the destination at a different cluster without editing the YAML.

---

## Reproduction

```bash
make status CLUSTER_PROVIDER=k3s-hostinger
# ArgoCD Apps section: shopping-cart-*, acg-* show SYNC STATUS: Unknown
# because their ApplicationSet destination is name: ubuntu-k3s (no such registered cluster)
```

---

## Fix

Parameterize the destination cluster name with a new `${APP_CLUSTER_NAME}` envsubst placeholder,
default it to `ubuntu-hostinger` at render time, and add it to the `envsubst` allowlist. This makes
the destination a **config switch** — `ubuntu-hostinger` by default, still
`APP_CLUSTER_NAME=ubuntu-k3s` for ACG — instead of a hard-coded edit. The registered cluster secret
`cluster-ubuntu-hostinger` has `stringData.name: ubuntu-hostinger`, so this destination resolves.

### Change 1 — `scripts/etc/argocd/applicationsets/services-git.yaml`: parameterize destination

**Exact old block (lines 39–41):**

```yaml
      destination:
        name: ubuntu-k3s
        namespace: shopping-cart-apps
```

**Exact new block:**

```yaml
      destination:
        name: ${APP_CLUSTER_NAME}
        namespace: shopping-cart-apps
```

### Change 2 — `scripts/etc/argocd/applicationsets/observability-acg.yaml`: parameterize destination

**Exact old block (lines 31–33):**

```yaml
      destination:
        name: ubuntu-k3s
        namespace: '{{.namespace}}'
```

**Exact new block:**

```yaml
      destination:
        name: ${APP_CLUSTER_NAME}
        namespace: '{{.namespace}}'
```

### Change 3 — `scripts/plugins/argocd.sh`: default + export `APP_CLUSTER_NAME`, add to envsubst allowlist

**Exact old block (inside `_argocd_deploy_applicationsets`):**

```bash
   K3D_MANAGER_BRANCH="${K3D_MANAGER_BRANCH:-$(git -C "${REPO_ROOT}" rev-parse --abbrev-ref HEAD 2>/dev/null || echo main)}"
   export K3D_MANAGER_BRANCH
```

**Exact new block:**

```bash
   K3D_MANAGER_BRANCH="${K3D_MANAGER_BRANCH:-$(git -C "${REPO_ROOT}" rev-parse --abbrev-ref HEAD 2>/dev/null || echo main)}"
   export K3D_MANAGER_BRANCH

   APP_CLUSTER_NAME="${APP_CLUSTER_NAME:-ubuntu-hostinger}"
   export APP_CLUSTER_NAME
```

**Exact old block (the render line in the deploy loop):**

```bash
      if envsubst '$ARGOCD_NAMESPACE $K3D_MANAGER_BRANCH' < "$file" | _kubectl apply -f - >/dev/null 2>&1; then
```

**Exact new block:**

```bash
      if envsubst '$ARGOCD_NAMESPACE $K3D_MANAGER_BRANCH $APP_CLUSTER_NAME' < "$file" | _kubectl apply -f - >/dev/null 2>&1; then
```
