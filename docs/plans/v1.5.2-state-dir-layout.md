# v1.5.2 — State Directory Layout Reorganization

**Branch:** `k3d-manager-v1.5.0`
**Type:** refactor (no functional change)

## Context

`~/.local/share/k3d-manager/` is a runtime state directory (equivalent to `/var/lib/`).
Currently all generated wrapper scripts, PID files, log files, and data files land flat in
the root. PNG screenshots and log files from browser automation also land here and must be
cleaned up manually. This spec reorganizes into typed subdirectories.

## New Layout

```
~/.local/share/k3d-manager/
  bin/          # runtime-generated wrapper scripts (*.sh)
  run/          # PID files (*.pid), plist temp files (*.plist)
  logs/         # log files (*.log) — replaces/extends existing logs/ subdir
  oci/          # OCI provider state (unchanged — already a subdir)
  screenshots/  # *.png from browser automation
  argocd-browser-https-tls/  # TLS certs (unchanged — already a subdir)
  gcp-console-auth/          # GCP OAuth tokens (unchanged — already a subdir)
  playwright-auth/ → profile/  # unchanged — already a subdir (keep as profile/)
  gcp-service-account.json  # stays in root — it's a credentials file, not a log/script
  tunnel-urls.txt            # stays in root — it's active state config, not a log
```

**Not moved:** `gcp-service-account.json`, `tunnel-urls.txt`, `profile/`,
`oci/`, `argocd-browser-https-tls/`, `gcp-console-auth/`.

## Migration

The scripts create these paths at runtime (`mkdir -p` each parent before writing).
No migration script is needed — on next `make up` / `acg-up` the new paths will be
created fresh. User can manually remove flat root files after first successful run:

```bash
rm -f ~/.local/share/k3d-manager/*.sh \
       ~/.local/share/k3d-manager/*.pid \
       ~/.local/share/k3d-manager/*.log \
       ~/.local/share/k3d-manager/*.plist
```
