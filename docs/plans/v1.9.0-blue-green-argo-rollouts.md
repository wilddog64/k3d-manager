# v1.9.0 — Blue/green deployment on OCI via Argo Rollouts + vCluster stress testing

## Context

OCI is the permanent production cluster. When shopping-cart apps run there, deployments
need a safe promotion gate: spin up an isolated full-stack green environment (vCluster),
run k6 against it, promote the real green ReplicaSet to production only if all thresholds
pass — otherwise auto-abort and stay on blue.

**Decision recorded:** 2026-05-29 (vCluster amendment 2026-05-29)

### Two-layer design: vCluster for isolation, Argo Rollouts for promotion

These solve different problems and both are needed:

| Layer | Tool | What it does |
|-------|------|--------------|
| Test isolation | vCluster | Ephemeral full-stack cluster inside OCI; k6 hits it without any production blast radius |
| Traffic migration | Argo Rollouts | Manages the blue→green selector swap on the real production cluster |

```
Push new image tag → ArgoCD syncs Rollout CRD
  → Rollouts controller creates green ReplicaSet (inactive, 0% traffic)
  → prePromotionAnalysis fires stress-runner Job:
      1. kubectl create namespace vcluster-stress-<run-id>
      2. vcluster create — full virtual cluster in that namespace
      3. kubectl apply -k shopping-cart-infra — full stack deployed into vCluster
         (image tag patched to the new version under test)
      4. k6 stress test hits vCluster services (in-cluster endpoint)
         checks: throughput > 50 RPS, error rate < 1%, p95 < 500ms
      5. trap: vcluster delete — namespace gone regardless of pass/fail
  → PASS → autoPromotionEnabled: false → you run: rollout_promote <app>
           active-service selector flips to green; blue RS scaled to 0
  → FAIL → abort; green RS deleted; blue stays active; Alertmanager notified
```

**Why vCluster beats a preview-service for stress testing:**
- Tests the full stack in realistic conditions (all 5 services + DBs + auth), not one service in isolation
- Zero blast radius — vCluster is a separate virtual control plane; its traffic never touches production services
- Ephemeral by design — `trap EXIT` deletes it; no cleanup debt
- ARM64 native — runs on the same OCI hardware as production; no architecture mismatch

**Prerequisites:**
- OCI cluster live (`CLUSTER_PROVIDER=k3s-oci`; v1.5.0)
- kube-prometheus-stack on OCI (v1.7.0) — Prometheus metrics for post-promotion analysis
- ArgoCD on OCI (v1.5.0 platform-helm)
- `shopping-cart-infra` repo accessible from inside OCI pods (public GitHub or deploy key)

## File 2: `scripts/etc/rollouts/stress-runner/Dockerfile`

Single image containing everything the analysis Job needs. Built for `linux/arm64`
and pushed to `ghcr.io/wilddog64/k3d-manager/stress-runner:<tag>`.

```dockerfile
FROM alpine:3.21 AS base

ARG KUBECTL_VERSION=v1.32.0
ARG VCLUSTER_VERSION=v0.22.0
ARG HELM_VERSION=3.17.0
ARG K6_VERSION=0.56.0
ARG KUSTOMIZE_VERSION=v5.4.3

RUN apk add --no-cache bash curl git ca-certificates

# kubectl (arm64)
RUN curl -fsSL "https://dl.k8s.io/release/${KUBECTL_VERSION}/bin/linux/arm64/kubectl" \
    -o /usr/local/bin/kubectl && chmod +x /usr/local/bin/kubectl

# vcluster CLI (arm64)
RUN curl -fsSL "https://github.com/loft-sh/vcluster/releases/download/${VCLUSTER_VERSION}/vcluster-linux-arm64" \
    -o /usr/local/bin/vcluster && chmod +x /usr/local/bin/vcluster

# helm (arm64)
RUN curl -fsSL "https://get.helm.sh/helm-v${HELM_VERSION}-linux-arm64.tar.gz" \
    | tar xz -C /tmp && mv /tmp/linux-arm64/helm /usr/local/bin/helm

# kustomize (arm64)
RUN curl -fsSL "https://github.com/kubernetes-sigs/kustomize/releases/download/kustomize%2F${KUSTOMIZE_VERSION}/kustomize_${KUSTOMIZE_VERSION}_linux_arm64.tar.gz" \
    | tar xz -C /usr/local/bin && chmod +x /usr/local/bin/kustomize

# k6 (arm64) — copy binary from official image
FROM grafana/k6:0.56.0 AS k6
FROM base
COPY --from=k6 /usr/bin/k6 /usr/local/bin/k6

COPY entrypoint.sh /entrypoint.sh
COPY k6-stress-script.js /scripts/stress-test.js
RUN chmod +x /entrypoint.sh

ENTRYPOINT ["/entrypoint.sh"]
```

## File 3: `scripts/etc/rollouts/stress-runner/entrypoint.sh`

```bash
#!/usr/bin/env bash
# Orchestrates: vCluster create → full-stack deploy → k6 → vCluster delete
#
# Required env vars (injected by AnalysisTemplate):
#   IMAGE_TAG          — new image tag under test (e.g. "sha-abc1234")
#   APP_NAME           — primary app being rolled out (e.g. "shopping-cart-frontend")
#   INFRA_REPO         — shopping-cart-infra git URL
#   INFRA_REF          — git ref to deploy (default: main)
#   K6_DURATION        — k6 duration (default: 120s)
#   K6_VUS             — k6 virtual users (default: 20)

set -euo pipefail

RUN_ID="$(date +%s)"
VC_NS="vcluster-stress-${RUN_ID}"
VC_NAME="stress"
VC_KUBECONFIG="/tmp/vcluster-${RUN_ID}.kubeconfig"
INFRA_REF="${INFRA_REF:-main}"
K6_DURATION="${K6_DURATION:-120s}"
K6_VUS="${K6_VUS:-20}"

_log() { echo "[stress-runner] $*"; }

cleanup() {
  _log "Cleanup: deleting vCluster ${VC_NS}..."
  vcluster delete "${VC_NAME}" --namespace "${VC_NS}" 2>/dev/null || true
  kubectl delete namespace "${VC_NS}" --ignore-not-found=true 2>/dev/null || true
}
trap cleanup EXIT

# --- 1. Create vCluster ---
_log "Creating namespace ${VC_NS}..."
kubectl create namespace "${VC_NS}"

_log "Creating vCluster (this takes ~60s)..."
vcluster create "${VC_NAME}" \
  --namespace "${VC_NS}" \
  --connect=false \
  --wait \
  --set "sync.fromHost.nodes.enabled=true"

# --- 2. Get vCluster kubeconfig ---
vcluster connect "${VC_NAME}" \
  --namespace "${VC_NS}" \
  --update-current=false \
  --kube-config "${VC_KUBECONFIG}"

_log "vCluster ready: ${VC_KUBECONFIG}"

# --- 3. Clone shopping-cart-infra and patch image tag ---
_log "Cloning ${INFRA_REPO} @ ${INFRA_REF}..."
git clone --depth=1 --branch "${INFRA_REF}" "${INFRA_REPO}" /tmp/infra

# Patch image tag for the app under test using kustomize
cd /tmp/infra
kustomize edit set image \
  "ghcr.io/wilddog64/${APP_NAME}=ghcr.io/wilddog64/${APP_NAME}:${IMAGE_TAG}"

# --- 4. Deploy full stack into vCluster ---
_log "Deploying shopping-cart stack into vCluster..."
KUBECONFIG="${VC_KUBECONFIG}" kubectl create namespace shopping-cart 2>/dev/null || true
KUBECONFIG="${VC_KUBECONFIG}" kustomize build /tmp/infra/k8s/overlays/oci \
  | KUBECONFIG="${VC_KUBECONFIG}" kubectl apply -f -

# Wait for all deployments ready (5 min timeout)
_log "Waiting for stack ready..."
for deploy in shopping-cart-frontend shopping-cart-basket shopping-cart-order \
              shopping-cart-payment shopping-cart-product-catalog; do
  KUBECONFIG="${VC_KUBECONFIG}" kubectl rollout status deployment/"${deploy}" \
    -n shopping-cart --timeout=300s || {
    _log "ERROR: ${deploy} did not become ready"
    exit 1
  }
done

# --- 5. Get frontend ClusterIP inside vCluster ---
TARGET_URL="http://$(KUBECONFIG="${VC_KUBECONFIG}" \
  kubectl get svc shopping-cart-frontend -n shopping-cart \
  -o jsonpath='{.spec.clusterIP}'):80"
_log "k6 target: ${TARGET_URL}"

# --- 6. Run k6 stress test ---
_log "Running k6: ${K6_VUS} VUs × ${K6_DURATION}..."
k6 run \
  --vus "${K6_VUS}" \
  --duration "${K6_DURATION}" \
  --env TARGET_URL="${TARGET_URL}" \
  /scripts/stress-test.js

_log "k6 passed — vCluster cleanup will run via trap"
```

## File 4: `scripts/etc/rollouts/stress-runner/k6-stress-script.js`

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  thresholds: {
    http_req_failed:   ['rate<0.01'],       // < 1% error rate
    http_req_duration: ['p(95)<500'],       // p95 latency < 500ms
    http_reqs:         ['rate>50'],         // throughput > 50 RPS
  },
};

const TARGET = __ENV.TARGET_URL || 'http://localhost:8080';

export default function () {
  const health = http.get(`${TARGET}/actuator/health`);
  check(health, { 'health 200': (r) => r.status === 200 });

  const res = http.get(`${TARGET}/`);
  check(res, {
    'status 2xx': (r) => r.status >= 200 && r.status < 300,
    'p95 ok':     (r) => r.timings.duration < 500,
  });

  sleep(0.1);
}
```

### RBAC: `rollouts-stress-runner` ServiceAccount

The stress-runner Job needs cluster-level permission to create/delete namespaces and
vCluster resources. Add to `scripts/etc/rollouts/`:

```yaml
# stress-runner-rbac.yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: rollouts-stress-runner
  namespace: "{{ROLLOUT_NAMESPACE}}"
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: rollouts-stress-runner
rules:
  - apiGroups: [""]
    resources: [namespaces]
    verbs: [create, delete, get, list]
  - apiGroups: [""]
    resources: [pods, services, configmaps, secrets]
    verbs: ["*"]
  - apiGroups: [apps]
    resources: [deployments, statefulsets]
    verbs: ["*"]
  # vCluster objects live in the host cluster namespace
  - apiGroups: [storage.loft.sh, management.loft.sh]
    resources: ["*"]
    verbs: ["*"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: rollouts-stress-runner
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: rollouts-stress-runner
subjects:
  - kind: ServiceAccount
    name: rollouts-stress-runner
    namespace: "{{ROLLOUT_NAMESPACE}}"
```

**Note:** Scope ClusterRole down further at implementation time — grant only the
minimum paths actually used by vcluster CLI during `vcluster create/delete`.

  name: ${APP_NAME}

  replicas: 2

    matchLabels:
      app: ${APP_NAME}

      labels:
        app: ${APP_NAME}

            value: ${APP_NAME}

            value: "120s"

            value: "20"

## File 7: `scripts/plugins/rollouts.sh`

```bash
#!/usr/bin/env bash
# rollouts.sh — Argo Rollouts + vCluster stress-test management on OCI

function deploy_rollouts() {
  local _dir="${SCRIPT_DIR}/etc/rollouts"
  local _ns="${ROLLOUTS_NAMESPACE:-shopping-cart}"

  _info "[rollouts] Deploying stress-runner RBAC..."
  envsubst '$ROLLOUT_NAMESPACE' \
    < "${_dir}/stress-runner-rbac.yaml" | _kubectl apply -f -

  _info "[rollouts] Deploying AnalysisTemplate..."
  envsubst '$ROLLOUT_NAMESPACE' \
    < "${_dir}/analysis-template-k6.yaml" | _kubectl apply -f -

  _info "[rollouts] Rollouts controller deployed via ApplicationSet (ArgoCD manages chart)"
  _info "[rollouts] Dashboard: https://rollouts.3ai-talk.org"
}

function rollout_status() {
  local _app="${1:?app name required}"
  local _ns="${2:-shopping-cart}"
  _kubectl argo rollouts status "${_app}" --namespace "${_ns}" --watch=false \
    --context "${OCI_CONTEXT:-}"
}

function rollout_promote() {
  local _app="${1:?app name required}"
  local _ns="${2:-shopping-cart}"
  _info "[rollouts] Promoting ${_app} — vCluster stress test already passed"
  _kubectl argo rollouts promote "${_app}" --namespace "${_ns}" \
    --context "${OCI_CONTEXT:-}"
}

function rollout_abort() {
  local _app="${1:?app name required}"
  local _ns="${2:-shopping-cart}"
  _info "[rollouts] Aborting ${_app} — rolling back to blue..."
  _kubectl argo rollouts abort "${_app}" --namespace "${_ns}" \
    --context "${OCI_CONTEXT:-}"
}
```

## Migration: Deployment → Rollout

For each shopping-cart app moving to OCI:

1. In shopping-cart-infra, add `k8s/overlays/oci/` kustomize overlay (needed by
   the entrypoint for `kustomize build /tmp/infra/k8s/overlays/oci`).

2. Replace `Deployment` with `Rollout` in the app's kustomize base:
   ```yaml
   kind: Deployment  →  kind: Rollout
   apiVersion: apps/v1  →  apiVersion: argoproj.io/v1alpha1
   ```
   Add the `strategy.blueGreen` block from the template above.

3. Add the preview Service to the app's kustomize base.

4. ArgoCD syncs → Rollouts controller manages subsequent deployments.

This migration is a shopping-cart-infra change — write a spec in that repo when ready.
