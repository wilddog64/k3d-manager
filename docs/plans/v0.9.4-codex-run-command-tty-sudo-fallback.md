## Problem

`_run_command --prefer-sudo` silently falls back to running as the current user when
`sudo -n` is unavailable (e.g., cold timestamp after a VM reboot). This causes misleading
permission errors instead of prompting for a password.

**Failing call chain:**
```
deploy_cluster (k3s)
  ‚Üí _ensure_path_exists "/etc/rancher/k3s"       # core.sh:36
    ‚Üí _run_command --prefer-sudo -- mkdir -p "/etc/rancher/k3s"
      ‚Üí sudo -n true              ‚Üí FAILS (cold sudo timestamp)
      ‚Üí falls through to runner=("$prog")         # system.sh:118
      ‚Üí mkdir -p /etc/rancher/k3s                 ‚Üí Permission denied
```

The same silent fallback exists in the `--require-sudo` path (exits 127 immediately instead
of trying interactive sudo) and in the `--probe` path.

**Issue doc:** `docs/issues/2026-03-19-run-command-non-interactive-sudo-failure.md`

## What to Change

**File:** `scripts/lib/system.sh`

### Change 1 ‚Äî Add `_run_command_has_tty` helper before `_run_command`

Add this function immediately before the `_run_command` function definition (before line 49):

```bash
# Testable TTY check. Override with _RC_FORCE_TTY=1 in tests.
_run_command_has_tty() {
  [[ "${_RC_FORCE_TTY:-0}" == "1" ]] || [[ -t 0 && -t 1 ]]
}
```

### Change 2 ‚Äî `--require-sudo` path: try interactive sudo before exit 127

**Current code** (inside `_run_command`, `require_sudo` block):
```bash
  if (( require_sudo )); then
    if (( interactive_sudo )) || sudo -n true >/dev/null 2>&1; then
      runner=(sudo "${sudo_flags[@]}" "$prog")
    else
      (( quiet )) || echo "sudo non-interactive not available" >&2
      exit 127
    fi
```

**Replace with:**
```bash
  if (( require_sudo )); then
    if (( interactive_sudo )) || sudo -n true >/dev/null 2>&1; then
      runner=(sudo "${sudo_flags[@]}" "$prog")
    elif _run_command_has_tty; then
      runner=(sudo "$prog")
    else
      (( quiet )) || echo "sudo non-interactive not available" >&2
      exit 127
    fi
```

### Change 3 ‚Äî `--probe` path: TTY fallback when `sudo -n` probe fails

**Current code** (inside `_run_command`, `probe_args` block, last two branches):
```bash
    elif (( prefer_sudo )) && sudo -n true >/dev/null 2>&1; then
        runner=(sudo -n "$prog")
      else
        runner=("$prog")
      fi
```

**Replace with:**
```bash
    elif (( prefer_sudo )) && sudo -n true >/dev/null 2>&1; then
        runner=(sudo -n "$prog")
      elif (( prefer_sudo )) && _run_command_has_tty; then
        runner=(sudo "$prog")
      else
        runner=("$prog")
      fi
```

### Change 4 ‚Äî Non-probe `--prefer-sudo` path: TTY fallback

**Current code** (inside `_run_command`, non-probe `else` block):
```bash
    if (( prefer_sudo )) && (( interactive_sudo )); then
        runner=(sudo "${sudo_flags[@]}" "$prog")
      elif (( prefer_sudo )) && sudo -n true >/dev/null 2>&1; then
        runner=(sudo -n "$prog")
      else
        runner=("$prog")
      fi
```

**Replace with:**
```bash
    if (( prefer_sudo )) && (( interactive_sudo )); then
        runner=(sudo "${sudo_flags[@]}" "$prog")
      elif (( prefer_sudo )) && sudo -n true >/dev/null 2>&1; then
        runner=(sudo -n "$prog")
      elif (( prefer_sudo )) && _run_command_has_tty; then
        runner=(sudo "$prog")
      else
        runner=("$prog")
      fi
```

## What to Change

**File:** `scripts/tests/lib/run_command.bats`

Add these three new test cases after the existing `--prefer-sudo falls back when sudo unavailable` test:

```bash
@test "--prefer-sudo uses interactive sudo when sudo -n unavailable but TTY present" {
  sudo() {
    echo "sudo $*" >> "$RUN_LOG"
    [[ "$1" == "-n" ]] && return 1
    shift; "$@"
  }
  export -f sudo
  run env _RC_FORCE_TTY=1 _run_command --prefer-sudo -- echo hi
  [ "$status" -eq 0 ]
  [[ "$output" = "hi" ]]
  read_lines "$RUN_LOG" log
  [ "${log[1]}" = "sudo echo hi" ]
}

@test "--require-sudo uses interactive sudo when sudo -n unavailable but TTY present" {
  sudo() {
    echo "sudo $*" >> "$RUN_LOG"
    [[ "$1" == "-n" ]] && return 1
    shift; "$@"
  }
  export -f sudo
  run env _RC_FORCE_TTY=1 _run_command --require-sudo -- echo hi
  [ "$status" -eq 0 ]
  [[ "$output" = "hi" ]]
  read_lines "$RUN_LOG" log
  [ "${log[1]}" = "sudo echo hi" ]
}

@test "--require-sudo fails when sudo -n unavailable and no TTY" {
  sudo() { return 1; }
  export -f sudo
  run -127 _run_command --require-sudo -- echo hi
  [[ "$output" == *"sudo non-interactive not available"* ]]
}
```
