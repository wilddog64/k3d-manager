# Bugfix: v1.4.12 — remove redundant directory.recurse: false from networking.yaml

**Branch (spec repo):** `k3d-manager-v1.4.12`
**Work repo:** `shopping-cart-infra`
**Work branch:** `docs/next-improvements`

---

## Problem

`argocd/applications/networking.yaml` explicitly sets `directory.recurse: false`, which is
the ArgoCD default. ArgoCD strips default-false values from live Application resources,
so the live resource has no `directory` block. This causes a perpetual diff:

```
===== argoproj.io/Application cicd/shopping-cart-networking ======
158a159,160
>     directory:
>       recurse: false
```

`shopping-cart-apps` (the root app) stays permanently OutOfSync because of this one-line diff.

**Root cause:** `directory.recurse: false` was specified explicitly in the manifest. ArgoCD
does not store zero/false default values in etcd, so the live state never matches.

---

## Fix

### Change 1 — `argocd/applications/networking.yaml`: remove the `directory` block

**Exact old block (lines 20-22):**

```yaml
    path: networking/istio
    directory:
      recurse: false
```

**Exact new block:**

```yaml
    path: networking/istio
```
