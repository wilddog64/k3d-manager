# Bugfix: v1.6.0 — acg-down leaves four daemons running after teardown

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/acg-down`

---

## Problem

Running `make down` followed by `make up` leaves the stack degraded. After
teardown, four launchd services keep running with stale state (looping against
clusters that no longer exist). When `bin/acg-up` reinstalls them, it compares
new vs existing plist content — if unchanged, it skips reinstall and the old
process continues targeting the dead cluster.

**Root cause:** `bin/acg-down` stops the ArgoCD and Keycloak browser daemons
but omits four others installed by `bin/acg-up`: the Cloudflare named tunnel,
the frontend HTTP listener, the loopback alias, and the Keycloak port-forward
agent.

---

## Reproduction

```
make down          # tears down remote + Hub
make up            # re-provisions
# Result: frontend, keycloak, and cloudflare tunnel still pointing at dead cluster
# /acg-status shows degraded apps even after full make up
```

---

## Fix

### Change 1 — `bin/acg-down`: stop four missing daemons before Hub cluster delete

Insert the following block immediately after line 149 (after the existing
ArgoCD browser HTTPS TLS cert removal, inside the `if _is_mac; then` block),
before the closing `fi` on line 150.

**Exact old block (lines 148–150):**

```bash
  _run_command --prefer-sudo --soft -- rm -f "${_argocd_browser_plist}" "${_argocd_browser_wrapper}" >/dev/null 2>&1 || true
  _run_command --prefer-sudo --soft -- rm -f "${_argocd_browser_tls_cert}" "${_argocd_browser_tls_key}" "${_argocd_browser_tls_ca}" "${_argocd_browser_tls_dir}/tls.crt" >/dev/null 2>&1 || true
fi
```

**Exact new block:**

```bash
  _run_command --prefer-sudo --soft -- rm -f "${_argocd_browser_plist}" "${_argocd_browser_wrapper}" >/dev/null 2>&1 || true
  _run_command --prefer-sudo --soft -- rm -f "${_argocd_browser_tls_cert}" "${_argocd_browser_tls_key}" "${_argocd_browser_tls_ca}" "${_argocd_browser_tls_dir}/tls.crt" >/dev/null 2>&1 || true

  _info "[acg-down] Stopping Cloudflare named tunnel launchd daemon..."
  _cf_tunnel_plist="/Library/LaunchDaemons/com.k3d-manager.cloudflare-tunnel.plist"
  _cf_tunnel_launchctl_log="${_ACG_STATE_DIR}/logs/cloudflare-tunnel-launchctl.log"
  : > "${_cf_tunnel_launchctl_log}"
  if _run_command --prefer-sudo --quiet --soft -- launchctl bootout system "${_cf_tunnel_plist}" \
      >"${_cf_tunnel_launchctl_log}" 2>&1; then
    _info "[acg-down] Cloudflare named tunnel launchd daemon unloaded"
  fi
  _run_command --prefer-sudo --soft -- rm -f "${_cf_tunnel_plist}" >/dev/null 2>&1 || true

  _info "[acg-down] Stopping frontend browser HTTP listener launchd daemon..."
  _frontend_browser_plist="${FRONTEND_BROWSER_LISTENER_PLIST:-/Library/LaunchDaemons/com.k3d-manager.frontend-browser-http.plist}"
  _frontend_browser_wrapper="${_ACG_STATE_DIR}/bin/frontend-browser-http.sh"
  _frontend_browser_launchctl_log="${_ACG_STATE_DIR}/logs/frontend-browser-http-launchctl.log"
  : > "${_frontend_browser_launchctl_log}"
  if _run_command --prefer-sudo --quiet --soft -- launchctl bootout system "${_frontend_browser_plist}" \
      >"${_frontend_browser_launchctl_log}" 2>&1; then
    _info "[acg-down] Frontend browser HTTP listener launchd daemon unloaded"
  fi
  _run_command --prefer-sudo --soft -- rm -f "${_frontend_browser_plist}" "${_frontend_browser_wrapper}" >/dev/null 2>&1 || true

  _info "[acg-down] Stopping Keycloak port-forward launchd agent..."
  _kc_pf_label="com.k3d-manager.keycloak-port-forward"
  _kc_pf_plist="${HOME}/Library/LaunchAgents/${_kc_pf_label}.plist"
  _kc_pf_wrapper="${_ACG_STATE_DIR}/bin/keycloak-port-forward.sh"
  if launchctl list "${_kc_pf_label}" >/dev/null 2>&1; then
    launchctl unload "${_kc_pf_plist}" 2>/dev/null || true
    _info "[acg-down] Keycloak port-forward launchd agent unloaded"
  fi
  rm -f "${_kc_pf_plist}" "${_kc_pf_wrapper}"
fi
```
