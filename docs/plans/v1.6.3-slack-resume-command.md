# v1.6.3 — `/acg-resume` Slack command

**Branch:** `k3d-manager-v1.6.3`
**Files:**
- `bin/acg-up`
- `Makefile`
- `bin/k3dm-webhook`
- `workers/slack-relay/index.js`

---

## Problem

When `bin/acg-up` fails partway through (e.g., Keycloak realm import at Step 10d after a
15-minute CloudFormation provision), there is no way to resume. The only option is a full
re-run: Steps 1–14 from scratch, including re-provisioning CloudFormation (10–15 min) and
waiting on all StatefulSet readiness loops again.

A `/resume` Slack thread command lets the user re-enter `make up` from the last completed
step — skipping everything already done.

---

## Design

### Checkpoint files

`bin/acg-up` writes a marker file after each expensive step completes:

```
~/.local/share/k3d-manager/checkpoints/<slug>.done
```

Helper functions:
- `_cp_skip <slug>` — returns 0 (skip) if marker exists; 1 (run) if not
- `_cp_write <slug>` — creates the marker

### Fresh start vs resume

At the start of `bin/acg-up`, before Step 1:
- **Fresh start (default):** clear all checkpoints: `rm -rf "${_ACG_CHECKPOINT_DIR}"`
- **Resume mode:** set `K3DM_RESUME=1` in the environment → checkpoints are preserved

### Makefile

The `up` target passes `K3DM_RESUME` to `bin/acg-up` so the webhook can set it as a
Makefile variable: `make up CLUSTER_PROVIDER=k3s-aws K3DM_RESUME=1`

### Webhook

New function `_run_cluster_resume(job_id, provider)` runs `make up CLUSTER_PROVIDER=...
K3DM_RESUME=1`. Same progress/failure reporting as `_run_cluster(... "up" ...)` but without
cleanup on failure (checkpoints are preserved so the user can retry immediately).

New thread command `resume`/`acg-resume` and endpoint `/api/v1/cluster-resume`.

### Worker

`/acg-resume` slash command relays to `/api/v1/cluster-resume`.

def _run_cluster_resume(job_id, provider="aws"):
    """Re-run make up with K3DM_RESUME=1 to skip completed checkpoints."""
    job_dir = JOB_DIR / job_id
    lines = []
    _provider_map = {"aws": "k3s-aws", "gcp": "k3s-gcp", "azure": "k3s-azure"}
    _cluster_provider = _provider_map.get(provider, "k3s-aws")
    _start = datetime.datetime.utcnow().timestamp()
    _progress_timer = None

    def _log(msg):
        lines.append(msg + "\n")

    _notify_job(job_id, f"🔄 *acg-resume* ({provider}) started — job `{job_id}`")

    def _post_progress():
        nonlocal _progress_timer
        elapsed = int((datetime.datetime.utcnow().timestamp() - _start) / 60)
        tail = [l.rstrip() for l in lines[-3:] if l.strip()]
        tail_text = "\n".join(tail) if tail else "—"
        _notify_job(job_id, f"⏳ *acg-resume* ({provider}) still running — {elapsed}m elapsed\n```{tail_text}```")
        _progress_timer = threading.Timer(120, _post_progress)
        _progress_timer.daemon = True
        _progress_timer.start()

    def _finish(status):
        if _progress_timer:
            _progress_timer.cancel()
        with _running_procs_lock:
            _running_procs.pop(job_id, None)
        (job_dir / "status").write_text(status)
        (job_dir / "output").write_text("".join(lines))
        emoji = "✅" if status == "success" else "❌"
        elapsed = int((datetime.datetime.utcnow().timestamp() - _start) / 60)
        _notify_job(job_id, f"{emoji} *acg-resume* ({provider}) — {status} in {elapsed}m")

    try:
        cmd = ["make", "up", f"CLUSTER_PROVIDER={_cluster_provider}", "K3DM_RESUME=1"]
        _log(f"Running {' '.join(cmd)}...")
        _progress_timer = threading.Timer(120, _post_progress)
        _progress_timer.daemon = True
        _progress_timer.start()
        proc = subprocess.Popen(
            cmd,
            cwd=REPO_ROOT,
            stdout=subprocess.PIPE,
            stderr=subprocess.STDOUT,
            text=True,
            preexec_fn=os.setsid,
        )
        with _running_procs_lock:
            _running_procs[job_id] = proc

        def _drain():
            for line in proc.stdout:
                lines.append(line)

        _reader = threading.Thread(target=_drain, daemon=True)
        _reader.start()
        pgid = os.getpgid(proc.pid)
        proc.wait()

            target=_run_post_provision_check,

            daemon=True
        ).start()
