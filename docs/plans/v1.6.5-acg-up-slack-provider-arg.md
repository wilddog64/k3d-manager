# feat: v1.6.5 — provider argument for /acg-up, /acg-down, /acg-resume Slack commands

**Branch:** `k3d-manager-v1.6.5`
**Files:** `bin/k3dm-webhook`

---

## Problem

`/acg-up`, `/acg-down`, and `/acg-resume` in Slack hardcode `"aws"` as the provider.
There is no way to provision an Azure or GCP cluster from Slack without editing the webhook.

The `_provider_map` already maps `"aws"` / `"gcp"` / `"azure"` → `CLUSTER_PROVIDER` values,
so routing is already in place — the command parsers just don't extract the provider token.

---

## Desired Slack syntax

```
/acg-up              → provider = aws  (default, unchanged)
/acg-up aws          → provider = aws
/acg-up azure        → provider = azure
/acg-up gcp          → provider = gcp

/acg-down            → provider = aws  (default)
/acg-down azure      → provider = azure
/acg-down gcp        → provider = gcp

/acg-resume          → provider = aws  (default)
/acg-resume azure    → provider = azure
/acg-resume gcp      → provider = gcp
```

---

## Fix

Three targeted edits to `bin/k3dm-webhook` — one per command handler.
No other files touched.
