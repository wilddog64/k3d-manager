## Background

v0.7.2 shipped with:
- Tracked pre-commit hook (`scripts/hooks/pre-commit`) — `_agent_audit` always runs,
  `_agent_lint` runs when `K3DM_ENABLE_AI=1`
- `_agent_audit` checks: bare sudo, if-count, BATS assertion removal, kubectl exec
  credential pattern
- Subtree guard: blocks direct edits to `scripts/lib/foundation/` (bypass:
  `K3DM_SUBTREE_SYNC=1`)
- CI: bats pinned to v1.11.0

This task validates the full stack end-to-end on both clusters with the new hook in place.

## Task 3 — Pre-commit Hook Smoke Test (macOS)

Test that the hook enforces the rules. Run each test from the repo root.

### Test A — Clean commit passes

```bash
echo "# comment" >> scripts/plugins/eso.sh
git add scripts/plugins/eso.sh
git commit -m "test: clean commit"
# Expected: commit succeeds, _agent_audit reports no violations
git revert HEAD --no-edit
```

### Test B — Bare sudo is blocked

```bash
echo "sudo apt-get install -y curl" >> scripts/plugins/eso.sh
git add scripts/plugins/eso.sh
git commit -m "test: bare sudo"
# Expected: pre-commit hook blocks with "bare sudo call" message
git checkout -- scripts/plugins/eso.sh
```

### Test C — Subtree guard fires

```bash
echo "# test" >> scripts/lib/foundation/scripts/lib/system.sh
git add scripts/lib/foundation/scripts/lib/system.sh
git commit -m "test: subtree edit"
# Expected: blocked with "direct edits to scripts/lib/foundation/ are not allowed"
git checkout -- scripts/lib/foundation/scripts/lib/system.sh
```

### Test D — Subtree bypass works

```bash
echo "# test" >> scripts/lib/foundation/scripts/lib/system.sh
git add scripts/lib/foundation/scripts/lib/system.sh
K3DM_SUBTREE_SYNC=1 git commit -m "test: subtree bypass"
# Expected: commit succeeds
git revert HEAD --no-edit
```
