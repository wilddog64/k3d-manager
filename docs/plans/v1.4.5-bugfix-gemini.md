# Bugfix: v1.4.5 — acg-up Step 10e CoreDNS + /etc/hosts failures

**Branch:** `k3d-manager-v1.4.5`
**Files:** `bin/acg-up`

---

## Problem

`make up` fails with `sudo non-interactive not available` + `Error 127`, aborting before `argocd-server` is restarted.
Additionally, the CoreDNS patch silently no-ops: `keycloak.shopping-cart.local` is never injected, so ArgoCD cannot resolve the OIDC issuer in-cluster.

**Root cause (bug 1 — CoreDNS):** The awk pattern `/^    health \{/` assumes `health` has a brace (`health {`), but the k3d CoreDNS Corefile uses `health` on its own line with no braces. The pattern never matches, so `_new_cf` equals `_cf`, the patch is a no-op, and the hostname is not added.

**Root cause (bug 2 — /etc/hosts):** `_run_command --require-sudo` hard-exits (error 127) when sudo is unavailable non-interactively (e.g. Touch ID / passphrase required), propagating through `set -euo pipefail` and aborting `make up` before `argocd-server` restarts.

---

## Reproduction

```bash
make up
# Bug 1: argocd-server cannot reach keycloak.shopping-cart.local — OIDC login fails
# Bug 2: output shows "sudo non-interactive not available" then "make: *** [up] Error 127"
```

---

## Fix

### Change 1 — `bin/acg-up` lines 607–623: CoreDNS — append to NodeHosts instead of injecting a second hosts block

The `coredns` ConfigMap has a `NodeHosts` key mounted at `/etc/coredns/NodeHosts`. The existing
`hosts /etc/coredns/NodeHosts` stanza in the Corefile reads from that file. Appending to `NodeHosts`
avoids the awk pattern fragility and sidesteps the "plugin/hosts: this plugin can only be used once
per Server Block" crash that a second `hosts {}` block causes.

**Exact old block (lines 607–623):**

```bash
if [[ -n "${_gw_ip}" ]]; then
  _cf=$(kubectl get cm coredns -n kube-system --context k3d-k3d-cluster \
    -o jsonpath='{.data.Corefile}')
  if ! printf '%s' "${_cf}" | grep -qF 'keycloak.shopping-cart.local'; then
    _new_cf=$(printf '%s' "${_cf}" | awk \
      -v ip="${_gw_ip}" \
      '/^    health \{/{print "    hosts {"; printf "      %s keycloak.shopping-cart.local\n", ip; print "      fallthrough"; print "    }"} {print}')
    kubectl patch cm coredns -n kube-system --context k3d-k3d-cluster \
      --type merge \
      --patch "$(printf '{"data":{"Corefile":%s}}' "$(printf '%s' "${_new_cf}" | jq -Rs .)")"
    kubectl rollout restart deployment/coredns -n kube-system --context k3d-k3d-cluster
    kubectl rollout status deployment/coredns -n kube-system --context k3d-k3d-cluster --timeout=60s
    _info "[acg-up] CoreDNS patched: keycloak.shopping-cart.local → ${_gw_ip}"
  fi
else
  _info "[acg-up] WARNING: IngressGateway LB IP not found — CoreDNS not patched; ArgoCD SSO in-cluster resolution may fail"
fi
```

**Exact new block:**

```bash
if [[ -n "${_gw_ip}" ]]; then
  _nh=$(kubectl get cm coredns -n kube-system --context k3d-k3d-cluster \
    -o jsonpath='{.data.NodeHosts}')
  if ! printf '%s' "${_nh}" | grep -qF 'keycloak.shopping-cart.local'; then
    _new_nh="${_nh}
${_gw_ip} keycloak.shopping-cart.local"
    kubectl patch cm coredns -n kube-system --context k3d-k3d-cluster \
      --type merge \
      --patch "$(python3 -c "import json,sys; print(json.dumps({'data':{'NodeHosts':sys.argv[1]}}))" "${_new_nh}")"
    kubectl rollout restart deployment/coredns -n kube-system --context k3d-k3d-cluster
    kubectl rollout status deployment/coredns -n kube-system --context k3d-k3d-cluster --timeout=60s
    _info "[acg-up] CoreDNS NodeHosts patched: keycloak.shopping-cart.local → ${_gw_ip}"
  fi
else
  _info "[acg-up] WARNING: IngressGateway LB IP not found — CoreDNS not patched; ArgoCD SSO in-cluster resolution may fail"
fi
```

### Change 2 — `bin/acg-up` lines 638–643: /etc/hosts — degrade gracefully when sudo unavailable

Replace `--require-sudo` (hard-fail) with `--prefer-sudo` (falls back to user-level execution).
If the user context also lacks write permission to `/etc/hosts`, print a warning instead of aborting.

**Exact old block (lines 638–643):**

```bash
# Add /etc/hosts entry on the Mac host so the browser reaches keycloak.shopping-cart.local
if ! grep -qF 'keycloak.shopping-cart.local' /etc/hosts; then
  _run_command --require-sudo -- \
    sh -c 'printf "127.0.0.1 keycloak.shopping-cart.local\n" >> /etc/hosts'
  _info "[acg-up] /etc/hosts: added keycloak.shopping-cart.local → 127.0.0.1"
fi
```

**Exact new block:**

```bash
# Add /etc/hosts entry on the Mac host so the browser reaches keycloak.shopping-cart.local
if ! grep -qF 'keycloak.shopping-cart.local' /etc/hosts; then
  if _run_command --prefer-sudo -- \
      sh -c 'printf "127.0.0.1 keycloak.shopping-cart.local\n" >> /etc/hosts' 2>/dev/null; then
    _info "[acg-up] /etc/hosts: added keycloak.shopping-cart.local → 127.0.0.1"
  else
    _info "[acg-up] WARNING: could not update /etc/hosts — add manually: 127.0.0.1 keycloak.shopping-cart.local"
  fi
fi
```
