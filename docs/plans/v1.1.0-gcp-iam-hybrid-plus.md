# Plan: v1.1.0 — Secure GCP IAM Hybrid+ Automation

> **STATUS: SUPERSEDED — 2026-04-16**
> The ACG `cloud_user` account has sufficient compute permissions without any IAM grant.
> `gcp_grant_compute_admin` and all Playwright IAM automation are unnecessary.
> `gcloud auth login` with `cloud_user` credentials is sufficient for `deploy_cluster`.
> This plan is retained for reference only — do not implement.

## Problem Statement
Full automation of the GCP IAM grant is blocked by Google's bot detection on login screens and ACG's API-level restrictions on `setIamPolicy`. Manual granting is tedious (10+ clicks). 

## Solution: The "Hybrid+" Workflow
A guided automation flow that combines human authentication with surgical robot interaction.

### 1. Automated Extraction (Gemini)
- Extract sandbox credentials (Username, Password, Project ID, and the specific "AddSession" Sandbox URL) from the ACG dashboard.

### 2. The Manual Bridge (User + Gemini)
- **Chrome launch/attach:** `scripts/plugins/gcp.sh` will start Chrome (if not already running) with `--remote-debugging-port=9222` and record the port in `~/.local/share/k3d-manager/playwright-auth/chrome.port`. The AddSession URL is opened with `open -a "Google Chrome" --args --profile-directory=Default <url>` so the same profile hosts both the manual login and the automation.
- **Human Role:** User selects the sandbox account in the Chrome UI. This bypasses Google's bot detection and any reCAPTCHA.
- **Robot Role (Hybrid+):** Once the user confirms login, Playwright attaches to the Chrome session via CDP (port from the file above) and clicks the consent dialogs in order:
    - "I understand" (New account welcome)
    - "Continue" (Sign in confirmation)
    - "Allow" (Google Cloud SDK access)
  If any dialog is missing, the script logs `DIAGNOSTIC: consent dialog missing` and prompts the user to continue manually.

### 3. Surgical IAM Grant (Gemini)
- Once logged in, the script navigates the active tab to `https://console.cloud.google.com/iam-admin/iam?project=<GCP_PROJECT>#permissions`.
- **Identity Guard:** The script queries `div[aria-label="Account"] span` and ensures the email matches `cloud_user_p_.*@qwiklabs.net`. If the pattern fails, it logs `ERROR: identity mismatch` and aborts without taking any action.
- **Automated Binding:** Using deterministic selectors, the script clicks "Grant Access", fills the service account email from the Playwright extraction payload, selects the `roles/compute.admin` chip, and clicks "Save". Every click is gated by explicit wait-for-element steps to prevent misfires on layout changes.
- **Fallback:** If IAM UI elements are missing or Google blocks automation (e.g., additional reCAPTCHA), the script exits with `EXIT_CODE=IAM_UI_BLOCKED` and prints manual instructions.

## Implementation Details

### Files to Change
- **UPDATE** `scripts/plugins/gcp.sh`:
  - Launch Chrome with `--remote-debugging-port` when needed and persist the port file.
  - Pass `GCP_CONSOLE_URL`, `GCP_USERNAME`, `GCP_PASSWORD`, and service-account email to the Node script via stdin JSON.
  - Handle `EXIT_CODE=IAM_UI_BLOCKED` by printing manual grant instructions.
- **NEW** `scripts/playwright/gcp_iam_grant.js`:
  - Attach to Chrome via the recorded CDP port, click consent dialogs, navigate to IAM, run identity guard, and perform the grant.
  - Exit with `IAM_GRANT=already_exists` when the binding is present.
- **UPDATE** `scripts/playwright/acg_credentials.js`: Emit `GCP_CONSOLE_URL` and the service-account email in the Playwright output payload (`GCP_SA_EMAIL`).

### Security Protocol
- **Stdin Injection:** Credentials will be passed to Node.js via `stdin` JSON to prevent exposure in the process table (`ps`).
- **Isolation:** All logic is provider-scoped to GCP; zero impact on existing AWS code paths.
- **Identity Guard logging:** Only log redacted email prefixes (e.g., `cloud_user_p_***@qwiklabs.net`). Never echo the full sandbox password in stdout/stderr.
- **Re-auth prompts:** If Google surfaces additional verification prompts, pause automation and instruct the user to complete them manually; never attempt to solve captchas.

## Verification Gates (DoD)
1. **Extraction:** `gcp_get_credentials` populates `GCP_CONSOLE_URL`.
2. **Identity Guard:** Script logs `DIAGNOSTIC: identity verified for cloud_user_p_*@qwiklabs.net` and aborts if the selector mismatch occurs.
3. **CDP Attach:** `scripts/plugins/gcp.sh` writes the Chrome debug port file and Playwright attaches successfully (log contains `DIAGNOSTIC: CDP attach OK`).
4. **IAM Grant:** First run logs `IAM_GRANT=applied`; second run logs `IAM_GRANT=already_exists`.
5. **Leak Check:** `ps aux` during run shows no credentials; log scrubber confirms no passwords/emails leaked.
6. **Fallback:** When IAM UI is blocked (simulated by feature flag), the script exits with `EXIT_CODE=IAM_UI_BLOCKED` and prints manual instructions.
