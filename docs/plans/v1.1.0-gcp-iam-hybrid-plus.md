# Plan: v1.1.0 — Secure GCP IAM Hybrid+ Automation

## Problem Statement
Full automation of the GCP IAM grant is blocked by Google's bot detection on login screens and ACG's API-level restrictions on `setIamPolicy`. Manual granting is tedious (10+ clicks). 

## Solution: The "Hybrid+" Workflow
A guided automation flow that combines human authentication with surgical robot interaction.

### 1. Automated Extraction (Gemini)
- Extract sandbox credentials (Username, Password, Project ID, and the specific "AddSession" Sandbox URL) from the ACG dashboard.

### 2. The Manual Bridge (User + Gemini)
- **Bypass Defaults:** The CLI will force-open the Sandbox URL in **Google Chrome** (using `open -a "Google Chrome"`) even if Safari is the system default.
- **Human Role:** User selects the profile and account (bypassing Google's bot detection).
- **Robot Role (Hybrid+):** Once the account is selected, Playwright latches on via CDP and automatically clicks the "Consent" interstitials:
    - "I understand" (New account welcome)
    - "Continue" (Sign in confirmation)
    - "Allow" (Google Cloud SDK access)

### 3. Surgical IAM Grant (Gemini)
- Once logged in, the script navigates the active tab to the IAM dashboard.
- **Identity Guard:** Explicitly verifies the active identity in the console header. Aborts immediately if it detects the user's personal account.
- **Automated Binding:** Performs the tedious clicks to grant `roles/compute.admin` to the Service Account.

## Implementation Details

### Files to Change
- **UPDATE** `scripts/plugins/gcp.sh`: Add logic to open Chrome and pipe credentials to the script.
- **NEW** `scripts/playwright/gcp_iam_grant.js`: The hybrid script that handles both the consent bridge and the IAM grant.
- **UPDATE** `scripts/playwright/acg_credentials.js`: Ensure robust extraction of the "Sandbox URL" field.

### Security Protocol
- **Stdin Injection:** Credentials will be passed to Node.js via `stdin` JSON to prevent exposure in the process table (`ps`).
- **Isolation:** All logic is provider-scoped to GCP; zero impact on existing AWS code paths.

## Verification Gates (DoD)
1. **Extraction:** `gcp_get_credentials` populates `GCP_CONSOLE_URL`.
2. **Identity Guard:** Script logs `DIAGNOSTIC: Identity verified` and never acts on personal accounts.
3. **Idempotency:** Second run reports `IAM_GRANT=already_exists`.
4. **Leak Check:** `ps aux` during run shows no credentials.
