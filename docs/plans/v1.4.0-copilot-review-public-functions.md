## What to Change

### File 1 (NEW): `scripts/plugins/copilot.sh`

Create this file with two public functions. No other files in `scripts/plugins/` need
to change — the dispatcher loads plugins on demand.

```bash
#!/usr/bin/env bash
# scripts/plugins/copilot.sh

function copilot_triage_pod() {
  local namespace="${1:-}"
  local pod="${2:-}"
  if [[ -z "$namespace" || -z "$pod" ]]; then
    _err "Usage: copilot_triage_pod <namespace> <pod-name>"
  fi

  local context
  context="$(
    echo "=== kubectl describe pod ==="
    kubectl describe pod -n "$namespace" "$pod" 2>&1 || true
    echo ""
    echo "=== last 100 log lines ==="
    kubectl logs -n "$namespace" "$pod" --previous --tail=100 2>&1 || \
      kubectl logs -n "$namespace" "$pod" --tail=100 2>&1 || true
  )"

  _copilot_review --prompt \
    "Diagnose this Kubernetes pod failure and suggest a fix. Do not run any commands.\n\n${context}"
}

function copilot_draft_spec() {
  local description="${1:-}"
  if [[ -z "$description" ]]; then
    _err "Usage: copilot_draft_spec '<short bug description>'"
  fi

  local context
  context="$(
    echo "=== recent git log ==="
    git log --oneline -10 2>/dev/null || true
    echo ""
    echo "=== recently changed files ==="
    git diff --name-only HEAD~3..HEAD 2>/dev/null || true
  )"

  _copilot_review --prompt \
    "Write a docs/bugs/ spec for the following bug in k3d-manager. Use this format: ## Root Cause, ## What to Change (with Before/After code blocks), ## Definition of Done (checkbox list). Do not run any commands.\n\nBug: ${description}\n\nContext:\n${context}"
}
```

### File 2: pre-commit hook — wire `AGENT_LINT_AI_FUNC`

Find the pre-commit hook file (check `.git/hooks/pre-commit` or the installed hook path).
Add these two exports **before** the `_agent_lint` call so `_agent_lint` picks up
`_copilot_review` as its AI function:

```bash
export AGENT_LINT_AI_FUNC="_copilot_review"
export K3DM_ENABLE_AI="${K3DM_ENABLE_AI:-0}"
```

**Important:** `K3DM_ENABLE_AI` defaults to `0` here — `_copilot_review` exits early
when it is `0`, so the AI lint step is opt-in. Users who want AI lint at commit time set
`K3DM_ENABLE_AI=1` in their environment. This means the pre-commit hook never fails for
users who don't have Copilot CLI installed.
