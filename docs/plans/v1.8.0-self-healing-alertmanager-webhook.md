# v1.8.0 — Self-healing: Alertmanager → k3dm-webhook remediation

## Context

Prometheus detects operational issues; Alertmanager routes fired alerts to the
`k3dm-webhook` server (already running on the laptop from v1.6.0). The webhook
executes the appropriate fix and logs the result. Repeated fires while an alert
is active are suppressed by a per-alert cooldown.

**Decision recorded:** 2026-05-29

**Prerequisites:**
- `k3dm-webhook` running as launchd agent (v1.6.0)
- kube-prometheus-stack deployed on OCI (v1.7.0)
- All three cluster kubeconfigs on the laptop (`oci`, `infra`, `acg`)

---

## Architecture

```
Prometheus fires PrometheusRule alert
  → Alertmanager
      → POST https://webhook.3ai-talk.org/api/v1/remediate
          {"alerts": [{"labels": {"alertname": "VaultSealed", "cluster": "infra"}, ...}]}
              → cooldown check (10 min per alertname+cluster)
              → dispatch to handler
              → log to /tmp/k3dm-remediation-jobs/<job-id>/
              → notify via notify.sh (info or critical)
```

Alertmanager sends the standard [webhook payload](https://prometheus.io/docs/alerting/latest/configuration/#webhook_config).
The webhook parses `alerts[].labels` to determine what to fix and where.

### Cluster label → kubectl context mapping

Each cluster's Prometheus is configured with `externalLabels.cluster`:

| `cluster` label | kubectl context | Description |
|-----------------|-----------------|-------------|
| `oci` | `${OCI_CONTEXT}` | OCI production (ARM64) |
| `infra` | `${INFRA_CONTEXT}` | Laptop hub cluster |
| `acg` | `${ACG_CONTEXT}` | ACG dev sandbox |

Contexts are read from `~/.k3dm/webhook-config` or environment variables
(same pattern as the existing webhook token).

### Cooldown

Before running any fix: check `/tmp/k3dm-remediation-cooldowns/<alertname>/<cluster>`.
If the file exists and was modified less than 10 minutes ago — skip (log and return 200).
After running a fix: write the timestamp file.

This prevents Alertmanager's repeat-fire from hammering the same fix every 30 seconds.

## Self-healing Use Cases

| Alert | Trigger condition | Remediation | Cooldown |
|-------|-------------------|-------------|----------|
| `ArgoCDAppOutOfSync` | App OutOfSync > 15m | `argocd app sync <app>` or refresh annotation | 10m |
| `VaultSealed` | Vault pod `vault_core_unsealed == 0` | `vault operator unseal` with key from `~/.k3dm/vault-unseal-key` | 30m |
| `ESOSecretStale` | ExternalSecret last sync age > 30m | Force-sync annotation | 10m |
| `PodCrashLooping` | `kube_pod_container_status_restarts_total` rate > 0.5/min for 5m | `kubectl rollout restart deployment/<name>` | 20m |
| `ACGClusterUnreachable` | Prometheus scrape fails for `cluster=acg` | POST to `/api/v1/argocd-upgrade` with `stage=acg` (triggers `make up`) | 60m |

**Not automated (human judgment required):**
- CloudFormation stack failures — too many failure modes
- Credential rotation — requires Playwright + human verification
- Any alert in `severity: page` that has fired 3+ times without clearing

## File 1: `scripts/etc/monitoring/alert-rules/platform-ops.yaml`

```yaml
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  name: platform-ops
  namespace: monitoring
  labels:
    release: kube-prometheus-stack
    managed-by: k3d-manager
spec:
  groups:
    - name: vault
      interval: 60s
      rules:
        - alert: VaultSealed
          expr: vault_core_unsealed == 0
          for: 2m
          labels:
            severity: critical
          annotations:
            summary: "Vault sealed on {{ $labels.cluster }}"
            description: "vault_core_unsealed=0 for >2m on cluster={{ $labels.cluster }}"

    - name: argocd
      interval: 60s
      rules:
        - alert: ArgoCDAppOutOfSync
          expr: |
            argocd_app_info{sync_status="OutOfSync"} == 1
          for: 15m
          labels:
            severity: warning
          annotations:
            summary: "ArgoCD app {{ $labels.name }} OutOfSync on {{ $labels.cluster }}"
            description: "App {{ $labels.name }} has been OutOfSync for >15m"

    - name: eso
      interval: 60s
      rules:
        - alert: ESOSecretStale
          expr: |
            (time() - externalsecret_sync_calls_total) > 1800
          for: 5m
          labels:
            severity: warning
          annotations:
            summary: "ExternalSecret {{ $labels.name }} not syncing on {{ $labels.cluster }}"
            description: "Last sync >30m ago"

    - name: pods
      interval: 60s
      rules:
        - alert: PodCrashLooping
          expr: |
            rate(kube_pod_container_status_restarts_total[5m]) * 60 > 0.5
          for: 5m
          labels:
            severity: warning
          annotations:
            summary: "Pod {{ $labels.pod }} crash-looping on {{ $labels.cluster }}"
            description: "Restart rate >0.5/min for 5m in namespace={{ $labels.namespace }}"
```

## File 2: `scripts/etc/monitoring/alertmanager-values.yaml`

Patch values for kube-prometheus-stack (merged with `kube-prometheus-values.yaml`
at deploy time via `--values`):

```yaml
alertmanager:
  config:
    global:
      resolve_timeout: 5m

    route:
      group_by: [alertname, cluster]
      group_wait: 30s
      group_interval: 5m
      repeat_interval: 1h
      receiver: k3dm-webhook
      routes:
        - matchers:
            - severity = critical
          receiver: k3dm-webhook
          repeat_interval: 30m
        - matchers:
            - severity = warning
          receiver: k3dm-webhook
          repeat_interval: 1h

    receivers:
      - name: k3dm-webhook
        webhook_configs:
          - url: https://webhook.3ai-talk.org/api/v1/remediate
            send_resolved: false
            http_config:
              authorization:
                type: Bearer
                credentials_file: /etc/alertmanager/secrets/webhook-token/token
```

The `credentials_file` path maps to a Kubernetes Secret `alertmanager-webhook-token`
mounted into the Alertmanager pod. Create it once:

```bash
kubectl create secret generic alertmanager-webhook-token \
  --from-literal=token="$(cat ~/.k3dm/webhook-token)" \
  --namespace monitoring
```

Also add to kube-prometheus-stack values:

```yaml
alertmanager:
  alertmanagerSpec:
    secrets:
      - alertmanager-webhook-token
```

### New constants

```python
COOLDOWN_DIR = Path("/tmp/k3dm-remediation-cooldowns")
COOLDOWN_SECONDS = {
    "VaultSealed":          1800,   # 30 min — unseal is sensitive
    "ArgoCDAppOutOfSync":   600,    # 10 min
    "ESOSecretStale":       600,    # 10 min
    "PodCrashLooping":      1200,   # 20 min
    "ACGClusterUnreachable": 3600,  # 60 min — make up is slow
}
DEFAULT_COOLDOWN = 600

CONTEXT_MAP_FILE = Path.home() / ".k3dm" / "webhook-config"
```

### New helper: `_load_context_map()`

```python
def _load_context_map():
    """Read cluster→context mapping from ~/.k3dm/webhook-config (KEY=VALUE lines)."""
    mapping = {}
    if CONTEXT_MAP_FILE.exists():
        for line in CONTEXT_MAP_FILE.read_text().splitlines():
            line = line.strip()
            if "=" in line and not line.startswith("#"):
                k, _, v = line.partition("=")
                mapping[k.strip()] = v.strip()
    return mapping
```

`~/.k3dm/webhook-config` format:

```
OCI_CONTEXT=k3s-oci
INFRA_CONTEXT=k3s-local
ACG_CONTEXT=k3s-acg
```

### New helper: `_cooldown_ok(alert_name, cluster)`

```python
def _cooldown_ok(alert_name, cluster):
    key = f"{alert_name}--{cluster}"
    path = COOLDOWN_DIR / key
    if path.exists():
        age = time.time() - path.stat().st_mtime
        limit = COOLDOWN_SECONDS.get(alert_name, DEFAULT_COOLDOWN)
        if age < limit:
            return False, int(limit - age)
    return True, 0


def _set_cooldown(alert_name, cluster):
    COOLDOWN_DIR.mkdir(parents=True, exist_ok=True)
    (COOLDOWN_DIR / f"{alert_name}--{cluster}").touch()
```

### New remediation handlers

```python
def _remediate_vault_sealed(cluster, context, labels, log):
    key_file = Path.home() / ".k3dm" / "vault-unseal-key"
    if not key_file.exists():
        log("VaultSealed: ~/.k3dm/vault-unseal-key not found — cannot unseal")
        return False
    key = key_file.read_text().strip()
    r = subprocess.run(
        ["kubectl", "exec", "-n", "vault", "vault-0",
         "--context", context, "--",
         "vault", "operator", "unseal", key],
        capture_output=True, text=True
    )
    log(r.stdout.strip() or r.stderr.strip())
    return r.returncode == 0


def _remediate_argocd_out_of_sync(cluster, context, labels, log):
    app = labels.get("name", "")
    if not app:
        log("ArgoCDAppOutOfSync: no app name in labels — skip")
        return False
    r = subprocess.run(
        ["kubectl", "annotate", "application", app,
         "-n", "cicd", "--context", context,
         "argocd.argoproj.io/refresh=normal", "--overwrite"],
        capture_output=True, text=True
    )
    log(f"Annotated app {app} for refresh: {r.stdout.strip() or r.stderr.strip()}")
    return r.returncode == 0


def _remediate_eso_stale(cluster, context, labels, log):
    name = labels.get("name", "")
    namespace = labels.get("namespace", "default")
    if not name:
        log("ESOSecretStale: no secret name in labels — skip")
        return False
    ts = str(int(time.time()))
    r = subprocess.run(
        ["kubectl", "annotate", "externalsecret", name,
         "-n", namespace, "--context", context,
         f"force-sync={ts}", "--overwrite"],
        capture_output=True, text=True
    )
    log(f"Force-sync ESO {namespace}/{name}: {r.stdout.strip() or r.stderr.strip()}")
    return r.returncode == 0


def _remediate_pod_crash_loop(cluster, context, labels, log):
    namespace = labels.get("namespace", "")
    pod = labels.get("pod", "")
    if not namespace or not pod:
        log("PodCrashLooping: missing namespace or pod label — skip")
        return False
    # Derive deployment name: strip last two segments (e.g. frontend-abc12-xyz99 → frontend)
    parts = pod.rsplit("-", 2)
    deployment = parts[0] if len(parts) == 3 else pod
    r = subprocess.run(
        ["kubectl", "rollout", "restart",
         f"deployment/{deployment}", "-n", namespace, "--context", context],
        capture_output=True, text=True
    )
    log(f"Rollout restart {namespace}/{deployment}: {r.stdout.strip() or r.stderr.strip()}")
    return r.returncode == 0


def _remediate_acg_unreachable(cluster, context, labels, log):
    # Reuse existing ACG upgrade job: stage=acg, version=current (no-op patch)
    # Effectively triggers make up if ACG is down
    import urllib.request
    token = _get_token()
    if not token:
        log("ACGClusterUnreachable: no webhook token — cannot trigger make up")
        return False
    payload = json.dumps({"chart_version": "current", "stage": "acg"}).encode()
    req = urllib.request.Request(
        "http://127.0.0.1:7443/api/v1/argocd-upgrade",
        data=payload,
        headers={"Authorization": f"Bearer {token}", "Content-Type": "application/json"},
        method="POST"
    )
    try:
        with urllib.request.urlopen(req, timeout=5) as resp:
            body = json.loads(resp.read())
            log(f"ACG make up triggered: job_id={body.get('job_id')}")
            return True
    except Exception as exc:
        log(f"ACGClusterUnreachable: self-POST failed: {exc}")
        return False
```

### Handler dispatch table and `_run_remediation()`

```python
_HANDLERS = {
    "VaultSealed":           _remediate_vault_sealed,
    "ArgoCDAppOutOfSync":    _remediate_argocd_out_of_sync,
    "ESOSecretStale":        _remediate_eso_stale,
    "PodCrashLooping":       _remediate_pod_crash_loop,
    "ACGClusterUnreachable": _remediate_acg_unreachable,
}


def _run_remediation(job_id, alert_name, cluster, context, labels):
    job_dir = JOB_DIR / job_id
    lines = []

    def _log(msg):
        lines.append(msg + "\n")

    def _finish(status):
        (job_dir / "status").write_text(status)
        (job_dir / "output").write_text("".join(lines))

    try:
        (job_dir / "status").write_text("running")

        ok, remaining = _cooldown_ok(alert_name, cluster)
        if not ok:
            _log(f"{alert_name}@{cluster}: cooldown active ({remaining}s remaining) — skip")
            _finish("skipped")
            return

        handler = _HANDLERS.get(alert_name)
        if not handler:
            _log(f"{alert_name}: no handler registered — skip")
            _finish("skipped")
            return

        _log(f"Remediating {alert_name} on {cluster} (context={context})")
        success = handler(cluster, context, labels, _log)
        if success:
            _set_cooldown(alert_name, cluster)
            _finish("success")
        else:
            _finish("failed")

    except Exception as exc:
        _log(f"ERROR: {exc}")
        _finish("failed")
```

### New `do_POST` branch for `/api/v1/remediate`

Add to `_Handler.do_POST` after the existing `/api/v1/argocd-upgrade` block:

```python
if self.path == "/api/v1/remediate":
    length = int(self.headers.get("Content-Length", 0))
    body = json.loads(self.rfile.read(length) or b"{}")
    alerts = body.get("alerts", [])
    jobs = []
    ctx_map = _load_context_map()

    for alert in alerts:
        if alert.get("status") != "firing":
            continue
        lbls = alert.get("labels", {})
        alert_name = lbls.get("alertname", "")
        cluster = lbls.get("cluster", "")
        context = ctx_map.get(f"{cluster.upper()}_CONTEXT", cluster)

        if not alert_name or not cluster:
            continue

        job_id = str(uuid.uuid4())[:8]
        (JOB_DIR / job_id).mkdir(parents=True, exist_ok=True)
        (JOB_DIR / job_id / "status").write_text("queued")
        threading.Thread(
            target=_run_remediation,
            args=(job_id, alert_name, cluster, context, lbls),
            daemon=True
        ).start()
        jobs.append({"job_id": job_id, "alert": alert_name, "cluster": cluster})

    self._json(202, {"jobs": jobs})
    return
```
