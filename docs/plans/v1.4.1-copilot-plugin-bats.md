## Purpose

`scripts/plugins/copilot.sh` has two public functions:

- `copilot_triage_pod <namespace> <pod>` — collects kubectl describe + logs, then calls `_ai_agent_review`
- `copilot_draft_spec '<description>'` — collects git log + diff, then calls `_ai_agent_review`

The lib-level tests in `k3d_manager_copilot.bats` cover `_ai_agent_review` itself.
This suite covers the plugin's own argument validation and context-collection behaviour.

**Test 5: invokes _ai_agent_review with pod context**
Stubs: `kubectl` stub that logs calls; `_ai_agent_review` stub that logs its `--prompt` argument to `$RUN_LOG`. Asserts the prompt contains `kubectl describe pod` and `last 100 log lines`.

```
@test "copilot_triage_pod invokes _ai_agent_review with kubectl context" {
  export K3DM_ENABLE_AI=1
  kubectl() { echo "describe pod -n default my-pod"; }
  _ai_agent_review() {
    printf '%s\n' "$*" >> "$RUN_LOG"
    return 0
  }
  export -f kubectl _ai_agent_review

  run copilot_triage_pod default my-pod
  [ "$status" -eq 0 ]
  grep -q -- '--prompt' "$RUN_LOG"
  grep -q 'kubectl describe pod' "$RUN_LOG"
}
```
