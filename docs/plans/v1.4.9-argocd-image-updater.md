## Problem

After every PR merge to `shopping-cart-product-catalog`, the CI workflow tries to commit
a new image SHA to `k8s/base/kustomization.yaml` and push directly to `main`. GitHub
branch protection blocks that push (`GH006`), so the kustomization tag is never updated
and the running pod keeps the stale image indefinitely.

**Fix:** Install ArgoCD Image Updater. It watches ghcr.io for digest changes on the
`latest` tag and updates the ArgoCD Application's kustomize image override directly
(write-back method `argocd`). No git writes, no branch protection conflict.

## Context

- ArgoCD runs in the `cicd` namespace on `k3d-k3d-cluster`
- The `services-git` ApplicationSet (in `scripts/etc/argocd/applicationsets/services-git.yaml`)
  discovers `services/*` directories in the k3d-manager repo and creates an Application for each
- For `services/shopping-cart-product-catalog/`, it creates an Application named
  `shopping-cart-product-catalog` that deploys kustomize resources to `ubuntu-k3s`
- The image is `ghcr.io/wilddog64/shopping-cart-product-catalog:latest` (private, requires auth)
- Credentials already exist: `ghcr-pull-secret` in `shopping-cart-apps` ns on `k3d-k3d-cluster`
  (type `kubernetes.io/dockerconfigjson`); image updater can reference it via `pullsecret:`

## Change 1 — New file: `scripts/etc/argocd/image-updater/kustomization.yaml`

Install ArgoCD Image Updater into the `cicd` namespace using upstream stable manifests with
a namespace override.

**Exact content:**

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

namespace: cicd

resources:
  - https://raw.githubusercontent.com/argoproj-labs/argocd-image-updater/stable/manifests/install.yaml

patches:
  - patch: |-
      - op: replace
        path: /spec/template/spec/containers/0/args/0
        value: run
    target:
      kind: Deployment
      name: argocd-image-updater
```

## Change 2 — New file: `scripts/etc/argocd/image-updater/registry-config.yaml`

ConfigMap telling image updater how to authenticate with ghcr.io. Uses the `pullsecret:`
reference to read credentials from the existing `ghcr-pull-secret` secret — no new
credentials needed.

**Exact content:**

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: argocd-image-updater-config
  namespace: cicd
data:
  registries.conf: |
    registries:
    - name: GitHub Container Registry
      api_url: https://ghcr.io
      prefix: ghcr.io
      credentials: pullsecret:shopping-cart-apps/ghcr-pull-secret
      defaultns: wilddog64
      insecure: false
```

## Change 3 — Modified: `scripts/etc/argocd/applicationsets/services-git.yaml`

Two changes:

### 3a — Add `ignoreApplicationDifferences`

Prevents the ApplicationSet reconciler from overwriting `spec.source.kustomize.images`
after image updater writes the digest override. Without this, ApplicationSet would reset
the image on every reconcile, undoing the image updater's work.

### 3b — Add conditional image updater annotations to template metadata

Only applies to `shopping-cart-product-catalog` using goTemplate conditional.
- `image-list`: identifies the image alias and registry path to watch
- `catalog.update-strategy: digest`: watches the `latest` tag's digest — when CI pushes a
  new image, the digest changes; updater pins the Application to the new digest
- `write-back-method: argocd`: stores the override in `spec.source.kustomize.images` on the
  Application object, not in git

**Exact old file content (full):**

```yaml
apiVersion: argoproj.io/v1alpha1
kind: ApplicationSet
metadata:
  name: services-git
  namespace: ${ARGOCD_NAMESPACE}
  labels:
    managed-by: k3d-manager
    app-type: services
spec:
  generators:
    - git:
        repoURL: https://github.com/wilddog64/k3d-manager
        revision: main
        directories:
          - path: services/*
          - path: services/shopping-cart-identity
            exclude: true
  goTemplate: true
  goTemplateOptions:
    - missingkey=error
  template:
    metadata:
      name: '{{.path.basename}}'
      labels:
        app-type: service
        discovered-from: git
    spec:
      project: platform
      destination:
        name: ubuntu-k3s
        namespace: shopping-cart-apps
      source:
        repoURL: https://github.com/wilddog64/k3d-manager
        targetRevision: main
        path: '{{.path.path}}'
      ignoreDifferences:
        - group: ""
          kind: Secret
          name: payment-db-credentials
          namespace: shopping-cart-payment
          jsonPointers:
            - /data/username
            - /data/password
      syncPolicy:
        automated:
          prune: true
          selfHeal: true
        syncOptions:
          - CreateNamespace=true
```

**Exact new file content (full):**

```yaml
apiVersion: argoproj.io/v1alpha1
kind: ApplicationSet
metadata:
  name: services-git
  namespace: ${ARGOCD_NAMESPACE}
  labels:
    managed-by: k3d-manager
    app-type: services
spec:
  ignoreApplicationDifferences:
    - jsonPointers:
        - /spec/source/kustomize/images
  generators:
    - git:
        repoURL: https://github.com/wilddog64/k3d-manager
        revision: main
        directories:
          - path: services/*
          - path: services/shopping-cart-identity
            exclude: true
  goTemplate: true
  goTemplateOptions:
    - missingkey=error
  template:
    metadata:
      name: '{{.path.basename}}'
      labels:
        app-type: service
        discovered-from: git
      annotations:
        argocd-image-updater.argoproj.io/image-list: >-
          {{- if eq .path.basename "shopping-cart-product-catalog"}}catalog=ghcr.io/wilddog64/shopping-cart-product-catalog:latest{{end}}
        argocd-image-updater.argoproj.io/catalog.update-strategy: >-
          {{- if eq .path.basename "shopping-cart-product-catalog"}}digest{{end}}
        argocd-image-updater.argoproj.io/write-back-method: >-
          {{- if eq .path.basename "shopping-cart-product-catalog"}}argocd{{end}}
    spec:
      project: platform
      destination:
        name: ubuntu-k3s
        namespace: shopping-cart-apps
      source:
        repoURL: https://github.com/wilddog64/k3d-manager
        targetRevision: main
        path: '{{.path.path}}'
      ignoreDifferences:
        - group: ""
          kind: Secret
          name: payment-db-credentials
          namespace: shopping-cart-payment
          jsonPointers:
            - /data/username
            - /data/password
      syncPolicy:
        automated:
          prune: true
          selfHeal: true
        syncOptions:
          - CreateNamespace=true
```

## Apply Steps

Run these in order after creating/modifying the files above:

```bash
# 1. Install ArgoCD Image Updater in cicd namespace
kubectl apply -k scripts/etc/argocd/image-updater/ --context k3d-k3d-cluster

# 2. Apply the registry config ConfigMap
kubectl apply -f scripts/etc/argocd/image-updater/registry-config.yaml \
  --context k3d-k3d-cluster

# 3. Re-apply the ApplicationSet (note: this file uses ${ARGOCD_NAMESPACE} template var)
#    Substitute the variable and apply:
ARGOCD_NAMESPACE=cicd envsubst < scripts/etc/argocd/applicationsets/services-git.yaml \
  | kubectl apply -f - --context k3d-k3d-cluster

# 4. Restart image updater to pick up config
kubectl rollout restart deploy/argocd-image-updater -n cicd --context k3d-k3d-cluster

# 5. Verify install
kubectl get deploy argocd-image-updater -n cicd --context k3d-k3d-cluster
kubectl get configmap argocd-image-updater-config -n cicd --context k3d-k3d-cluster

# 6. Check logs for first registry poll (wait ~30s after restart)
kubectl logs deploy/argocd-image-updater -n cicd --context k3d-k3d-cluster --tail=30
```
