# Bugfix: v1.6.5 — wire Azure credential extraction into bin/acg-up Step 1

**Branch:** `k3d-manager-v1.6.5`
**Files:** `bin/acg-up`, `scripts/etc/azure/azure-vars.sh`

---

## Problem

Two failures when running `make up` with `CLUSTER_PROVIDER=k3s-azure`:

**Bug 1 — `KV_NAME: unbound variable`**
`scripts/plugins/azure.sh` sources `scripts/etc/azure/azure-vars.sh` at plugin load time.
`azure-vars.sh` line 2 uses bare `${KV_NAME}` inside a command substitution with `set -u`
active, causing `KV_NAME: unbound variable` every time the plugin is sourced without
`KV_NAME` pre-set. This fires before Step 1 runs.

**Bug 2 — `AZ_RESOURCE_GROUP` unset / wrong tenant SP**
`bin/acg-up` Step 1 `k3s-azure` case is an empty passthrough (`k3s-azure) ;;`). No
credential extraction runs. `az` falls back to cached credentials from a previous sandbox
(wrong tenant), `az group list` returns nothing, causing:

```
ERROR: AADSTS700016: Application with identifier '...' was not found in the directory 'Pluralsight Cloud'.
ERROR: [k3s-azure] Could not determine resource group — set AZ_RESOURCE_GROUP
```

---

## Fix

### Change 1 — `scripts/etc/azure/azure-vars.sh`: guard `KV_ID` export against unbound `KV_NAME`

**Exact old block (line 2):**

```bash
export KV_ID="${KV_ID:-$(az keyvault show --name "${KV_NAME}" --query id -o tsv)}"
```

**Exact new block:**

```bash
export KV_ID="${KV_ID:-${KV_NAME:+$(az keyvault show --name "${KV_NAME}" --query id -o tsv)}}"
```

`${KV_NAME:+...}` expands to empty when `KV_NAME` is unset or empty — no API call, no
`set -u` error. When `KV_NAME` is set the behavior is identical to before.

---

### Change 2 — `bin/acg-up`: replace empty `k3s-azure` Step 1 case with credential extraction

**Exact old block:**

```bash
  k3s-azure)
    ;;
esac
```

**Exact new block:**

```bash
  k3s-azure)
    if _az_ok 2>/dev/null; then
      _info "[acg-up] Azure credentials already valid — skipping Playwright extraction"
    else
      _acg_cred_test_url="${sandbox_url:-${_ACG_SANDBOX_LIST_URL}}"
      _info "[acg-up] Extracting and validating Azure credentials..."
      _browser_launch
      "${REPO_ROOT}/scripts/lib/acg/bin/acg-credential-test" "${_acg_cred_test_url}" --provider azure || exit 1
      _info "[acg-up] Azure credentials extracted and validated."
    fi
    if [[ -z "${AZ_RESOURCE_GROUP:-}" ]]; then
      AZ_RESOURCE_GROUP=$(az group list --query '[0].name' -o tsv 2>/dev/null || true)
      export AZ_RESOURCE_GROUP
    fi
    ;;
esac
```
