## Problem

`bin/acg-up` Step 1 `k3s-azure` case is an empty passthrough (`k3s-azure) ;;`), introduced
in the acg-up azure wiring spec. No credentials are extracted, so `az` is unauthenticated
when `_provider_k3s_azure_deploy_cluster` runs. `_az_resource_group()` calls `az group list`
and gets nothing back, causing:

```
ERROR: [k3s-azure] Could not determine resource group â€” set AZ_RESOURCE_GROUP
```

**Prerequisite:** `lib-acg` commit for `fix(credentials): persist az login to ~/.azure after SP validation`
must be subtree-pulled into k3d-manager before this fix is committed.

---

## Fix

### Change 1 â€” `bin/acg-up`: replace empty `k3s-azure` Step 1 case with credential extraction

**Exact old block:**

```bash
  k3s-azure)
    ;;
esac
```

**Exact new block:**

```bash
  k3s-azure)
    if _az_ok 2>/dev/null; then
      _info "[acg-up] Azure credentials already valid â€” skipping Playwright extraction"
    else
      _acg_cred_test_url="${sandbox_url:-${_ACG_SANDBOX_LIST_URL}}"
      _info "[acg-up] Extracting and validating Azure credentials..."
      _browser_launch
      "${REPO_ROOT}/scripts/lib/acg/bin/acg-credential-test" "${_acg_cred_test_url}" --provider azure || exit 1
      _info "[acg-up] Azure credentials extracted and validated."
    fi
    if [[ -z "${AZ_RESOURCE_GROUP:-}" ]]; then
      AZ_RESOURCE_GROUP=$(az group list --query '[0].name' -o tsv 2>/dev/null || true)
      export AZ_RESOURCE_GROUP
    fi
    ;;
esac
```
