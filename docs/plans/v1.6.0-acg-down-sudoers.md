# v1.6.0 — acg-down sudoers NOPASSWD rule

## Problem

`bin/acg-down` unloads two system-level LaunchDaemons and removes their plist files.
These operations require `root` because they target the macOS `system` launchd domain
(`/Library/LaunchDaemons/`). When run from the Slack webhook (no TTY), `sudo` cannot
prompt for a password, so those cleanup steps silently fail and leave dangling daemons.

The four affected `_run_command` calls in `bin/acg-down`:

```bash
_run_command --prefer-sudo --soft -- launchctl bootout system "${_keycloak_browser_plist}"
_run_command --prefer-sudo --soft -- rm -f "${_keycloak_browser_plist}" "${_keycloak_browser_wrapper}"
_run_command --prefer-sudo --quiet --soft -- launchctl bootout system "${_argocd_browser_plist}"
_run_command --prefer-sudo --soft -- rm -f "${_argocd_browser_plist}" "${_argocd_browser_wrapper}"
```

Default values:
- `_keycloak_browser_plist` = `/Library/LaunchDaemons/com.k3d-manager.keycloak-browser-http.plist`
- `_keycloak_browser_wrapper` = `${HOME}/.local/share/k3d-manager/keycloak-browser-http.sh`
- `_argocd_browser_plist` = `/Library/LaunchDaemons/com.k3d-manager.argocd-browser-https.plist`
- `_argocd_browser_wrapper` = `${HOME}/.local/share/k3d-manager/bin/argocd-browser-https.sh`

## Solution

Install a `/etc/sudoers.d/k3d-manager` drop-in that grants `cliang` passwordless sudo
for exactly the four commands above. No other commands are granted.

Sudoers wildcards (`*`) are used to match the plist and wrapper paths. The rules use
`/bin/launchctl` and `/bin/rm` — the canonical paths verified below.

## Before You Start

1. `git pull origin k3d-manager-v1.6.0` in the k3d-manager repo
2. Read `bin/acg-down` lines 40–130 to confirm exact plist/wrapper paths
3. Run `ls -la /bin/launchctl /bin/rm` to confirm binary paths on this Mac
4. Confirm current user: `whoami` — must be `cliang`
5. Branch: `k3d-manager-v1.6.0`

## Target file

**System file (not in repo):** `/etc/sudoers.d/k3d-manager`

Create with mode `0440`, owned `root:wheel`. Use `visudo -c -f <file>` to validate
before installing — **an invalid sudoers file can lock out sudo entirely**.

## Exact content of `/etc/sudoers.d/k3d-manager`

```
# k3d-manager — allow launchd system-daemon cleanup without a password prompt.
# Required by bin/acg-down when invoked from the Slack webhook (no TTY).
cliang ALL=(root) NOPASSWD: /bin/launchctl bootout system /Library/LaunchDaemons/com.k3d-manager.*.plist
cliang ALL=(root) NOPASSWD: /bin/rm -f /Library/LaunchDaemons/com.k3d-manager.*.plist *
```

## Install steps

```bash
# 1. Write to a temp file
TMPF=$(mktemp)
cat > "${TMPF}" <<'EOF'
# k3d-manager — allow launchd system-daemon cleanup without a password prompt.
# Required by bin/acg-down when invoked from the Slack webhook (no TTY).
cliang ALL=(root) NOPASSWD: /bin/launchctl bootout system /Library/LaunchDaemons/com.k3d-manager.*.plist
cliang ALL=(root) NOPASSWD: /bin/rm -f /Library/LaunchDaemons/com.k3d-manager.*.plist *
EOF

# 2. Validate syntax before installing
visudo -c -f "${TMPF}"
# Must print: "${TMPF}: parsed OK"

# 3. Install
sudo install -m 0440 -o root -g wheel "${TMPF}" /etc/sudoers.d/k3d-manager
rm -f "${TMPF}"

# 4. Verify
sudo -n launchctl bootout system /Library/LaunchDaemons/com.k3d-manager.keycloak-browser-http.plist 2>&1 || true
echo "exit: $?"
# Should exit 0 or "No such file" error — NOT a password prompt
```

## Smoke test

After installation, run without any sudo cache active:

```bash
sudo -k   # invalidate existing sudo ticket
sudo -n launchctl bootout system /Library/LaunchDaemons/com.k3d-manager.keycloak-browser-http.plist 2>&1
```

Expected: command exits 0 (daemon unloaded) or prints `Could not find specified service` —
**not** `sudo: a password is required`.

## What NOT to Do

- Do NOT run `sudo visudo` without `-f` on a temp file first — editing the live sudoers
  directly risks a parse error that locks out sudo.
- Do NOT use `NOPASSWD: ALL` — grant only the two specific commands above.
- Do NOT create a PR — this is a system file change, not a repo commit.
- Do NOT skip `visudo -c -f` validation before `sudo install`.
- Do NOT modify files outside `/etc/sudoers.d/k3d-manager`.
- Do NOT commit to `main` — this spec is on `k3d-manager-v1.6.0` for reference only.
