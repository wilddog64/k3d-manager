# v1.6.0 — acg-down pre-auth removal + deploy-worker target

## Problem 1: acg-down always prompts for a password

`bin/acg-down` contains a pre-authorization block:

```bash
if _is_mac; then
  if [[ -t 0 && -t 1 ]] && ! sudo -n true 2>/dev/null; then
    _info "[acg-down] Pre-authorizing sudo for macOS host operations (launchd cleanup)..."
    _run_command --interactive-sudo --soft -- true
  fi
fi
```

This probes with `sudo -n true`. `/usr/bin/true` is NOT in the NOPASSWD rules, so the
probe always fails and the password prompt fires — even though the actual commands later
(`launchctl bootout`, `rm -f /Library/LaunchDaemons/...`) ARE NOPASSWD since May 31.

The pre-auth block was written before the NOPASSWD rules existed. It is now redundant
and causes every `make down` (especially from the Slack webhook where there is no TTY)
to prompt or silently fail.

**Fix:** Remove the entire pre-auth block.

## Problem 2: wrangler deploy loses WEBHOOK_TOKEN

Running `npx wrangler deploy` directly (bypassing `bin/k3dm-worker-setup`) deploys new
Worker code but leaves the `WEBHOOK_TOKEN` wrangler secret at whatever value was last
set — which may be stale if the webhook server was reinstalled. The fix is a lightweight
`deploy-worker` Makefile target that always syncs secrets before deploying.

## Change 1 — Remove pre-auth block from `bin/acg-down`

**File:** `bin/acg-down`

Remove this block entirely (exact lines — verify with grep before deleting):

```bash
if _is_mac; then
  if [[ -t 0 && -t 1 ]] && ! sudo -n true 2>/dev/null; then
    _info "[acg-down] Pre-authorizing sudo for macOS host operations (launchd cleanup)..."
    _run_command --interactive-sudo --soft -- true
  fi
fi
```

Nothing replaces it. The `_run_command --prefer-sudo --soft` calls later in the script
already handle elevation correctly via the NOPASSWD rules in `/etc/sudoers.d/k3d-manager`.

## Change 2 — Add `deploy-worker` target to `Makefile`

**File:** `Makefile`

Add to the `.PHONY` line (keep alphabetical order):
```
deploy-worker
```

Add the target after `setup-worker`:

```makefile
## Re-deploy Cloudflare Worker and sync secrets from Keychain (run after Worker code changes)
deploy-worker:
	@_cf=$$(security find-generic-password -s k3dm-cloudflare-api-token -a k3dm -w 2>/dev/null) && \
	_tok=$$(security find-generic-password -s k3dm-webhook-token -a k3dm -w 2>/dev/null) && \
	_sig=$$(security find-generic-password -s k3dm-slack-signing-secret -a k3dm -w 2>/dev/null) && \
	cd workers/slack-relay && \
	CLOUDFLARE_API_TOKEN="$$_cf" printf '%s' "$$_tok" | npx --yes wrangler secret put WEBHOOK_TOKEN && \
	CLOUDFLARE_API_TOKEN="$$_cf" printf '%s' "$$_sig" | npx --yes wrangler secret put SLACK_SIGNING_SECRET && \
	CLOUDFLARE_API_TOKEN="$$_cf" npx --yes wrangler deploy
```

Note: the Keychain service names are:
- `k3dm-cloudflare-api-token` — Cloudflare API token
- `k3dm-webhook-token` — webhook bearer token (set by `bin/k3dm-webhook-setup`)
- `k3dm-slack-signing-secret` — Slack signing secret
