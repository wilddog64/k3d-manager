## Problem

The Slack slash commands are named `/acg-*` (A Cloud Guru). The stack now also runs on
Hostinger and OCI, so the `acg` prefix advertises the wrong platform and no longer reflects
the current situation.

**Decision (user, 2026-06-14):** rename the five `/acg-*` commands to `/cluster-*` (aligns
with the just-completed `bin/cluster-*` rename and the webhook's internal `_run_cluster_*` /
`/api/v1/cluster*` naming). **Hard cutover** — remove the `/acg-*` names entirely; no aliases.
Keep `/hostinger-status` unchanged. Scope is **command tokens + the display labels that echo
them** — NOT the ACG environment narrative or any real k8s resource name.

| Old | New |
|-----|-----|
| `/acg-up`      | `/cluster-up` |
| `/acg-down`    | `/cluster-down` |
| `/acg-status`  | `/cluster-status` |
| `/acg-refresh` | `/cluster-refresh` |
| `/acg-resume`  | `/cluster-resume` |
| `/hostinger-status` | *(unchanged)* |

The HTTP relay endpoints (`/api/v1/cluster`, `/api/v1/cluster-status`,
`/api/v1/cluster-refresh`, `/api/v1/cluster-resume`) are **already** neutral — do NOT change them.
The `_run_cluster_*` function names are already neutral — do NOT change them.
