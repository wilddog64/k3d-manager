## Objective

Remove the imperative `kubectl apply` calls for PostgreSQL/Redis/RabbitMQ/MinIO from
`deploy_shopping_cart_data` in `scripts/plugins/shopping_cart.sh`, delegating ownership to
the existing ArgoCD `data-layer` Application already defined in
`shopping-cart-infra/argocd/applications/data-layer.yaml`.

`register_shopping_cart_apps` already applies every file under `argocd/applications/` (including
`data-layer.yaml`), so the Application object is already created during bootstrap. The only
changes are: (1) set `prune: false` in `data-layer.yaml` to protect PVCs from accidental deletion,
and (2) remove the duplicate `kubectl apply` lines from the Bash function.

### Change 2 ŌĆö `k3d-manager`: `scripts/plugins/shopping_cart.sh` (lines 96ŌĆō102)

Remove the five imperative `kubectl apply` lines. Keep the directory existence check, the info
log, and all readiness waits. ArgoCD owns the apply; the script still waits for readiness before
downstream steps run.

**Exact old block (lines 96ŌĆō102):**
```bash

  for pg_dir in orders payment products; do
    _run_command -- kubectl apply --context ubuntu-k3s -f "${infra_root}/postgresql/${pg_dir}/"
  done
  _run_command -- kubectl apply --context ubuntu-k3s -f "${infra_root}/redis/cart/"
  _run_command -- kubectl apply --context ubuntu-k3s -f "${infra_root}/rabbitmq/"
  _run_command -- kubectl apply --context ubuntu-k3s -f "${infra_root}/minio/"
```

**Exact new block:**
```bash

```

Everything from line 104 onward (the readiness waits) remains unchanged.
