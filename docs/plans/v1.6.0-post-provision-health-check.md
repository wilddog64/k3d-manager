# v1.6.0 — Post-provision health check + AI triage after acg-up

**Branch:** `k3d-manager-v1.6.0`
**File:** `bin/k3dm-webhook`

---

## Problem

`acg-up` exits 0 (reports success) but the cluster is not fully healthy:
- SSH tunnel may be DOWN (autossh died, EC2 not yet accepting connections)
- ArgoCD apps may be Degraded (ESO not synced, pods still starting, real failures)

The user currently has no automated signal distinguishing a transient post-startup
state from a real failure — they have to run `/acg-status` manually and interpret
the output themselves.

**Goal:** 90 seconds after a successful `make up`, automatically check the cluster,
identify degraded apps, call Claude Haiku for each one ("transient or real failure?"),
and post a consolidated triage report to Slack.

Two targeted additions to `bin/k3dm-webhook`. No other file is touched.

### Change 1 — Add `_run_post_provision_check` function before `_run_cluster_status`

Insert this entire function immediately before the line
`def _run_cluster_status(job_id, response_url):`:

```python
_POST_PROVISION_DELAY_SECS = 90  # ESO sync + pod startup grace period


def _run_post_provision_check(job_id, provider):
    """90s after acg-up: check tunnel + apps; AI-triage each degraded app to Slack."""
    import time
    time.sleep(_POST_PROVISION_DELAY_SECS)

    def _notify(text):
        url = SLACK_WEBHOOK_URL
        if not url:
            _ru_file = JOB_DIR / job_id / "response_url"
            if _ru_file.exists():
                url = _ru_file.read_text().strip()
        if url:
            _slack_post(url, text)

    lines = []

    # SSH tunnel
    tunnel = subprocess.run(
        ["curl", "-sf", "--max-time", "3", "http://localhost:6443"],
        capture_output=True, text=True, timeout=8
    )
    tunnel_ok = tunnel.returncode == 0 or tunnel.returncode == 22
    lines.append(f"*SSH tunnel (6443):* {'UP ✅' if tunnel_ok else 'DOWN ❌'}")

    if not tunnel_ok:
        lines.append("⚠️ Tunnel is down — ACG cluster unreachable; skipping app health check.")
        _notify(f"🏥 *Post-provision check* ({provider})\n" + "\n".join(lines))
        return

    # ArgoCD degraded apps
    apps = subprocess.run(
        ["kubectl", "get", "applications", "-n", "cicd",
         "--context", "k3d-k3d-cluster", "--no-headers",
         "-o", "custom-columns=NAME:.metadata.name,HEALTH:.status.health.status"],
        capture_output=True, text=True, timeout=10
    )
    if apps.returncode != 0 or not apps.stdout.strip():
        lines.append("_ArgoCD app status unavailable_")
        _notify(f"🏥 *Post-provision check* ({provider})\n" + "\n".join(lines))
        return

    app_lines = [l for l in apps.stdout.strip().splitlines() if l]
    degraded = [l.split()[0] for l in app_lines
                if len(l.split()) >= 2 and l.split()[1] == "Degraded"]
    healthy_count = len(app_lines) - len(degraded)
    lines.append(f"*ArgoCD:* {healthy_count}/{len(app_lines)} apps healthy")

    if not degraded:
        _notify(f"🏥 *Post-provision check* ({provider}) — all clear ✅\n" + "\n".join(lines))
        return

    lines.append(f"⚠️ Degraded: {', '.join(f'`{a}`' for a in degraded)}")
    _notify(f"🏥 *Post-provision check* ({provider})\n" + "\n".join(lines))

    # AI triage each degraded app
    for app_name in degraded[:5]:  # cap at 5 to limit API calls
        diag_parts = []

        desc = subprocess.run(
            ["kubectl", "describe", "application", app_name, "-n", "cicd",
             "--context", "k3d-k3d-cluster"],
            capture_output=True, text=True, timeout=15
        )
        if desc.returncode == 0 and desc.stdout.strip():
            diag_parts.append(f"ArgoCD Application:\n{desc.stdout[-2000:]}")

        pods = subprocess.run(
            ["kubectl", "get", "pods", "-A", "--context", "ubuntu-k3s",
             "--no-headers", "--request-timeout=8s",
             "--field-selector=status.phase!=Running"],
            capture_output=True, text=True, timeout=15
        )
        if pods.returncode == 0 and pods.stdout.strip():
            diag_parts.append(f"Non-running pods:\n{pods.stdout.strip()[:1000]}")

        if not diag_parts:
            continue

        prompt = (
            f"A Kubernetes ArgoCD application named '{app_name}' is Degraded "
            f"90 seconds after a fresh cluster provision.\n\n"
            f"Diagnostics:\n" + "\n\n".join(diag_parts) + "\n\n"
            "Is this a TRANSIENT issue (ESO not synced yet, pod still starting — "
            "will resolve in a few minutes) or a REAL FAILURE (misconfiguration, "
            "missing secret, crash loop — needs human action)?\n"
            "Reply with TRANSIENT or REAL FAILURE, then one sentence with the "
            "most likely cause and fix. Be brief."
        )
        analysis = _call_claude(prompt)
        _notify(f"🔍 *{app_name}* — {analysis}")


```

### Change 2 — Launch health check thread after successful `make up`

**Exact old block (lines 215–218):**
```python
        _log(f"{action} complete")
        if action == "up":
            _record_acg_state(provider)
        _finish("success")
```

**Exact new block:**
```python
        _log(f"{action} complete")
        if action == "up":
            _record_acg_state(provider)
            threading.Thread(
                target=_run_post_provision_check,
                args=(job_id, provider),
                daemon=True
            ).start()
        _finish("success")
```

## Behaviour

```
make up exits 0
  → _record_acg_state writes ConfigMap
  → _run_post_provision_check thread starts (daemon)
  → _finish("success") fires → Slack: "✅ acg-up (aws) — success in Nm"

90 seconds later (in background):
  → curl localhost:6443 (tunnel check)
  → kubectl get applications (degraded list)
  → Slack: "🏥 Post-provision check (aws) — SSH UP, 6/17 apps healthy ⚠️ Degraded: ..."
  → For each degraded app (max 5):
      kubectl describe application + kubectl get pods --field-selector!=Running
      → _call_claude (Haiku, ~400 tokens in, ~60 tokens out)
      → Slack: "🔍 shopping-cart-basket — TRANSIENT: ESO ExternalSecret not yet synced, wait 2 min"
```

The `_finish("success")` fires immediately so the user sees the ✅ right away.
The health report arrives 90s later as a follow-on Slack message in the same channel.
