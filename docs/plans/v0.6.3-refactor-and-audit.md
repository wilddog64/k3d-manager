# v0.6.3 Plan — Refactoring & The Digital Auditor

## Context
Following the successful AI integration in v0.6.2, this release focuses on cleaning up "AI Drift" and implementing deterministic architectural gates. We will eliminate the "Defensive Bloat" found in `core.sh` and `system.sh` and deploy the Digital Auditor to prevent its return.

---

## Components

### 1. Simplification & De-Bloat (Refactoring)
- **Goal**: Reduce the 400+ `if` blocks in core libraries.
- **Logic**:
  - Consolidate multi-stage permission checks into `_run_command --prefer-sudo`.
  - Move OS-specific logic from `core.sh` to specialized "footprint" helpers in `system.sh`.
  - Clean up state management (e.g., environmental overrides like `IP`).

### 2. `_agent_lint()` (The Digital Auditor)
- **Location**: `scripts/lib/agent_rigor.sh`
- **Logic**:
  - Uses `copilot-cli` to perform deterministic architectural audits based on Memory Bank focus areas.
  - **Gate**: Returns non-zero exit code on violation, blocking commits.

### 3. `_agent_audit()` (Post-Implementation Rigor)
- **Logic**:
  - Checks for test weakening (unauthorized deletions in BATS files).
  - Verifies logic simplicity.

## Acceptance Criteria
1. `scripts/lib/core.sh` line count reduced by > 15%.
2. `_agent_lint` correctly identifies a missing RabbitMQ listener pattern (simulated).
3. All BATS tests pass with zero regressions.
