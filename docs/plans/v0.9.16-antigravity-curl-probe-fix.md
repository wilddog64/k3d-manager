## Context

`_curl` calls `_run_command --quiet` without `--soft`. When curl returns a non-zero exit
code (e.g. port 9222 not yet listening), `_run_command` calls `exit 1` — killing the entire
script. The error is swallowed by `>/dev/null 2>&1`, making it look like a silent abort.

`_curl` must NOT be used as a boolean probe in an `if` statement. The correct pattern for
a probe is `_run_command --soft -- curl ...` which returns the exit code instead of calling
`exit 1`.

## Change

In `_antigravity_launch`, replace:

```bash
  if _curl -sf http://localhost:9222/json >/dev/null 2>&1; then
```

With:

```bash
  if _run_command --soft -- curl -sf http://localhost:9222/json >/dev/null 2>&1; then
```

That is the only change required.
