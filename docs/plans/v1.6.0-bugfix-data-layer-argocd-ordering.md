# Bugfix: v1.6.0 — Data-layer ArgoCD Application applied after wait loop

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/acg-up`

---

## Problem

`make up` always fails with:

```
ERROR: [shopping_cart] Timed out waiting for StatefulSet postgresql-orders to be created by ArgoCD
```

`deploy_shopping_cart_data` (Step 10b, line 563) waits up to 300 s for the
`postgresql-orders`, `postgresql-payment`, and `postgresql-products` StatefulSets to appear
in `shopping-cart-data` on `ubuntu-k3s`. Those StatefulSets are created by the ArgoCD
`data-layer` Application — but that Application is not applied until
`register_shopping_cart_apps` runs at Step 11b (line 1346). ArgoCD has nothing to act on,
so the wait loop always times out.

**Root cause:** `register_shopping_cart_apps` (which calls
`kubectl apply -f shopping-cart-infra/argocd/applications/`) runs ~800 lines after
`deploy_shopping_cart_data`, so the `data-layer` Application is never registered before the
wait loop starts.

**Confirmed by cluster state:**
- `kubectl get applications -A` → No resources found (zero Applications registered)
- `shopping-cart-data` namespace: empty (no StatefulSets, no PVCs)
- ArgoCD application-controller pod: healthy and running

---

## Reproduction

```
bin/acg-up --no-login-prompt
# Wait ~8 min
# Expected: make up completes successfully
# Actual: "Timed out waiting for StatefulSet postgresql-orders to be created by ArgoCD"
```

---

## Fix

Apply only the `data-layer` ArgoCD Application immediately before `deploy_shopping_cart_data`
so ArgoCD can start syncing the StatefulSets during the wait loop.

`register_shopping_cart_apps` at Step 11b stays where it is — it applies the full
applications set (identity, networking, shopping-cart-apps, etc.) at the appropriate time
after Vault and ESO are ready.

### Change 1 — `bin/acg-up`: apply data-layer Application before wait loop

**Exact old block (lines 562–563):**

```bash
_info "[acg-up] Step 10b/14 — Deploying shopping-cart data layer (PostgreSQL, Redis, RabbitMQ)..."
deploy_shopping_cart_data
```

**Exact new block:**

```bash
_info "[acg-up] Step 10b/14 — Deploying shopping-cart data layer (PostgreSQL, Redis, RabbitMQ)..."
_info "[acg-up] Applying data-layer ArgoCD Application so sync starts before wait loop..."
_run_command -- kubectl apply --context k3d-k3d-cluster \
  -f "${REPO_ROOT}/../shopping-carts/shopping-cart-infra/argocd/applications/data-layer.yaml"
deploy_shopping_cart_data
```
