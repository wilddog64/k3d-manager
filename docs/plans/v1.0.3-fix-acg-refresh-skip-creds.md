# v1.0.3 ‚Äî Fix `bin/acg-refresh` unconditional credential extraction

**Date**: 2026-04-04
**Branch**: k3d-manager-v1.0.3
**Status**: ASSIGNED to Codex

## Problem

`bin/acg-refresh` always calls `acg_get_credentials` even when AWS credentials are still
valid. When the Chrome CDP launchd agent is running, this causes a `SingletonLock` conflict
because Playwright tries to launch a second Chrome against the same profile dir. The tunnel
restart ‚Äî the actual purpose of `make refresh` ‚Äî never runs.

Issue: `docs/issues/2026-04-04-acg-refresh-unconditional-cred-extraction.md`

## Change ‚Äî `bin/acg-refresh`

Replace the unconditional `acg_get_credentials` call with a conditional check:

```bash
# Before
_info "[acg-refresh] Refreshing AWS credentials..."
acg_get_credentials ${sandbox_url:+"$sandbox_url"}
```

```bash
# After
_info "[acg-refresh] Checking AWS credentials..."
if _acg_check_credentials 2>/dev/null; then
  _info "[acg-refresh] Credentials still valid ‚Äî skipping extraction"
else
  _info "[acg-refresh] Credentials expired ‚Äî extracting..."
  acg_get_credentials ${sandbox_url:+"$sandbox_url"}
fi
```
