# Bugfix: v1.0.2 — `ghcr-pull-secret` Never Created in App Namespaces

**Branch:** `k3d-manager-v1.0.2`
**Files:** `bin/acg-up`

---

## Problem

All 5 app pods show `ImagePullBackOff` — `ghcr-pull-secret` is missing in every app namespace.
`bin/acg-up` step 4 silently skips every namespace with "namespace not found — skipping".

**Two bugs in the same block (lines 75–89 of `bin/acg-up`):**

1. **Wrong namespace names** — loop iterates `apps payment data` but the actual Kubernetes
   namespaces are `shopping-cart-apps`, `shopping-cart-payment`, `shopping-cart-data`.
   The checked namespaces will never exist.

2. **Timing gap** — even with correct names, the namespaces are created by ArgoCD on first
   sync, which happens *after* `acg-up` completes. The existence check (`kubectl get namespace`)
   guards against creation, so the secret is always skipped.

**Root cause:** Wrong namespace names + conditional skip on non-existent namespace = secret
is never created regardless of cluster state.

---

## Reproduction

```bash
bin/acg-up
# Step 4/5 output:
# [acg-up] Namespace apps not found — skipping (ArgoCD will create it on first sync)
# [acg-up] Namespace payment not found — skipping (ArgoCD will create it on first sync)
# [acg-up] Namespace data not found — skipping (ArgoCD will create it on first sync)
# Result: ghcr-pull-secret exists nowhere → ImagePullBackOff on all pods
```

---

## Fix

### Change 1 — `bin/acg-up`: fix namespace names and remove conditional skip

**Exact old block (lines 75–89):**

```bash
for ns in apps payment data; do
  if kubectl get namespace "$ns" --context ubuntu-k3s >/dev/null 2>&1; then
    kubectl create secret docker-registry ghcr-pull-secret \
      --docker-server=ghcr.io \
      --docker-username="${_github_user}" \
      --docker-password="${_ghcr_pat}" \
      --context ubuntu-k3s \
      -n "$ns" \
      --dry-run=client -o yaml \
      | kubectl apply --context ubuntu-k3s -f -
    _info "[acg-up] ghcr-pull-secret applied in namespace: ${ns}"
  else
    _info "[acg-up] Namespace ${ns} not found — skipping (ArgoCD will create it on first sync)"
  fi
done
```

**Exact new block:**

```bash
for ns in shopping-cart-apps shopping-cart-payment shopping-cart-data; do
  kubectl create namespace "$ns" --context ubuntu-k3s \
    --dry-run=client -o yaml \
    | kubectl apply --context ubuntu-k3s -f - >/dev/null
  kubectl create secret docker-registry ghcr-pull-secret \
    --docker-server=ghcr.io \
    --docker-username="${_github_user}" \
    --docker-password="${_ghcr_pat}" \
    --context ubuntu-k3s \
    -n "$ns" \
    --dry-run=client -o yaml \
    | kubectl apply --context ubuntu-k3s -f -
  _info "[acg-up] ghcr-pull-secret applied in namespace: ${ns}"
done
```

**What changed:**
- `apps payment data` → `shopping-cart-apps shopping-cart-payment shopping-cart-data`
- Removed `if kubectl get namespace ... else skip` — replaced with idempotent `kubectl create namespace ... | kubectl apply`
- Secret creation now always runs regardless of whether namespace existed before
