## Context

The `services-git` ApplicationSet templates one ArgoCD Application per
`services/<app>/` directory. Today the ArgoCD Image Updater annotations are
gated to **product-catalog only** (lines 36–42). product-catalog has run Image
Updater in production successfully — Phase 2 generalizes the exact same pattern
to **basket** and **order** as well.

Mechanism (do not change — just understand it):
- `write-back-method: argocd` makes Image Updater write the resolved digest to
  the Application's `spec.source.kustomize.images` parameter override.
- The top-level `ignoreApplicationDifferences: /spec/source/kustomize/images`
  (lines 10–12) stops self-heal from reverting that override.
- So the Image Updater override is authoritative over the base `newTag` — **no
  app-repo `kustomization.yaml` edit is needed in this phase.**

`.path.basename` equals the service image repo name exactly
(`shopping-cart-basket`, `shopping-cart-order`, `shopping-cart-product-catalog`),
so the image ref can be built as `ghcr.io/wilddog64/{{.path.basename}}:latest`.

The annotation uses a single alias `app` for all three services (each
Application manages exactly one image, so one alias suffices). This replaces the
old per-app alias `catalog`; the per-alias key therefore becomes
`app.update-strategy`.

## New test — `scripts/tests/plugins/argocd_image_updater_annotations.bats`

Mirror the `grep -F` style of `argocd_app_cluster_generator.bats`. Create the
file with exactly these contents:

```bash
#!/usr/bin/env bats

YAML="${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/services-git.yaml"

@test "image-updater: image-list guarded for basket" {
  run grep -F -- 'eq .path.basename "shopping-cart-basket"' "${YAML}"
  [ "${status}" -eq 0 ]
}

@test "image-updater: image-list guarded for order" {
  run grep -F -- 'eq .path.basename "shopping-cart-order"' "${YAML}"
  [ "${status}" -eq 0 ]
}

@test "image-updater: image-list guarded for product-catalog" {
  run grep -F -- 'eq .path.basename "shopping-cart-product-catalog"' "${YAML}"
  [ "${status}" -eq 0 ]
}

@test "image-updater: image ref built from basename with :latest" {
  run grep -F -- 'app=ghcr.io/wilddog64/{{.path.basename}}:latest' "${YAML}"
  [ "${status}" -eq 0 ]
}

@test "image-updater: digest update-strategy present" {
  run grep -F -- 'argocd-image-updater.argoproj.io/app.update-strategy' "${YAML}"
  [ "${status}" -eq 0 ]
}

@test "image-updater: write-back-method argocd present" {
  run grep -F -- 'argocd-image-updater.argoproj.io/write-back-method' "${YAML}"
  [ "${status}" -eq 0 ]
}
```
