# Spec: Safety gate audit — deploy_* no-args guard (v0.9.7)

**Branch:** `k3d-manager-v0.9.7`
**Work repo:** `k3d-manager`

---

## Context

Audit of all 16 `deploy_*` functions found that 15 are already safe:
- 10 have `-h|--help` handling on `$1`
- `deploy_ad` has an explicit `if [[ $arg_count -eq 0 ]]` guard
- `deploy_app_cluster` requires `--confirm` flag — errors without it
- `deploy_jenkins` is guarded by `ENABLE_JENKINS=1` env var
- `deploy_k3d_cluster` and `deploy_k3s_cluster` are wrappers that delegate to `deploy_cluster`

**Only gap:** `deploy_cluster` (scripts/lib/core.sh:684) has `--help` in its arg parser but
no no-args guard — calling it bare skips past the parser and proceeds to create a cluster.
`deploy_k3d_cluster` and `deploy_k3s_cluster` inherit the fix automatically via `"$@"`.

## The Change

One edit to `scripts/lib/core.sh` — nothing else.

Add a no-args guard immediately after the `show_help` block (line 731).

**Old (lines 721–733):**
```bash
   if (( show_help )); then
      cat <<'EOF'
Usage: deploy_cluster [options] [cluster_name]

Options:
  -f, --force-k3s     Skip the provider prompt and deploy using k3s.
  --provider <name>   Explicitly set the provider (k3d or k3s).
  -h, --help          Show this help message.
EOF
      return 0
   fi

   local platform="" platform_msg=""
```

**New:**
```bash
   if (( show_help )); then
      cat <<'EOF'
Usage: deploy_cluster [options] [cluster_name]

Options:
  -f, --force-k3s     Skip the provider prompt and deploy using k3s.
  --provider <name>   Explicitly set the provider (k3d or k3s).
  -h, --help          Show this help message.
EOF
      return 0
   fi

   if [[ ${#positional[@]} -eq 0 && -z "${provider_cli}" && "${force_k3s}" -eq 0 ]]; then
      cat <<'EOF'
deploy_cluster: no arguments given — run with -h for usage.
EOF
      return 1
   fi

   local platform="" platform_msg=""
```
