## Problem

`add_ubuntu_k3s_cluster()` in `scripts/plugins/shopping_cart.sh` fetches the Ubuntu k3s
kubeconfig into `${HOME}/.kube/k3s-ubuntu.yaml` but never merges it into `~/.kube/config`.

This forces Gemini to manually copy and merge the kubeconfig every session before running
`kubectl config use-context ubuntu-k3s`. The context should be set up once and persist.

---

## What to Change

**File:** `scripts/plugins/shopping_cart.sh`
**Function:** `add_ubuntu_k3s_cluster()`

After the connectivity verify block (after `fi` that checks `kubectl get nodes`) and before
the `local kube_context` block, insert a merge step that:

1. Checks if `ubuntu-k3s` context already exists in `~/.kube/config` — if yes, skip (idempotent).
2. Copies `${local_kubeconfig}` to a temp file in `${HOME}/.kube/`.
3. Renames the current-context in the temp file to `ubuntu-k3s` (using `kubectl config rename-context`) — skip rename if it's already named `ubuntu-k3s`.
4. Merges temp file into `~/.kube/config` using `KUBECONFIG=file1:file2 kubectl config view --flatten`.
5. Atomically replaces `~/.kube/config` with merged output, sets `chmod 600`.
6. Deletes the temp file.
7. Logs result via `_info`.

## Implementation

```bash
  _info "[shopping_cart] Merging ubuntu-k3s context into ~/.kube/config"
  if ! kubectl config get-contexts ubuntu-k3s &>/dev/null; then
    local _tmp_kube _tmp_merged
    _tmp_kube="${HOME}/.kube/ubuntu-k3s-tmp.yaml"
    _tmp_merged="${HOME}/.kube/config-merged-tmp.yaml"
    cp "${local_kubeconfig}" "${_tmp_kube}"
    chmod 600 "${_tmp_kube}"
    local _src_context
    if ! _src_context=$(KUBECONFIG="${_tmp_kube}" kubectl config current-context 2>/dev/null); then
      _src_context=""
    fi
    if [[ -n "${_src_context}" && "${_src_context}" != "ubuntu-k3s" ]]; then
      KUBECONFIG="${_tmp_kube}" kubectl config rename-context "${_src_context}" ubuntu-k3s
    fi
    KUBECONFIG="${HOME}/.kube/config:${_tmp_kube}" kubectl config view --flatten > "${_tmp_merged}"
    mv "${_tmp_merged}" "${HOME}/.kube/config"
    chmod 600 "${HOME}/.kube/config"
    rm -f "${_tmp_kube}"
    _info "[shopping_cart] ubuntu-k3s context merged into ~/.kube/config"
  else
    _info "[shopping_cart] Context 'ubuntu-k3s' already present in ~/.kube/config — skipping merge"
  fi
```
