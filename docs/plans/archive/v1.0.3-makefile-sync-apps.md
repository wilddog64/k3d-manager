## Problem

After `make up` + `make argocd-registration`, there is no Makefile shortcut to trigger
an ArgoCD sync of the data-layer app. Users must manually port-forward argocd-server,
log in, and run `argocd app sync`. This is needed to verify the full stack after every
fresh sandbox.

## Files Changed

1. `bin/acg-sync-apps` — new script: port-forward → login → sync data-layer → cleanup
2. `Makefile` — add `sync-apps` target + `.PHONY` entry + help text

## Change 1 — new `bin/acg-sync-apps`

Create this file exactly as shown:

```bash
#!/usr/bin/env bash
# bin/acg-sync-apps
#
# Trigger ArgoCD sync for data-layer, then show all remote pod status.
# Handles argocd-server port-forward and login automatically.
#
# Usage:
#   bin/acg-sync-apps
#
# Environment:
#   INFRA_CONTEXT  kubectl context for infra cluster (default: k3d-k3d-cluster)
#   APP_CONTEXT    kubectl context for app cluster   (default: ubuntu-k3s)
#   ARGOCD_NS      ArgoCD namespace                  (default: cicd)

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd "${SCRIPT_DIR}/.." && pwd)"
source "${REPO_ROOT}/scripts/lib/system.sh"

INFRA_CONTEXT="${INFRA_CONTEXT:-k3d-k3d-cluster}"
APP_CONTEXT="${APP_CONTEXT:-ubuntu-k3s}"
ARGOCD_NS="${ARGOCD_NS:-cicd}"

_pf_pid=""

_cleanup() {
  if [[ -n "${_pf_pid}" ]]; then
    kill "${_pf_pid}" 2>/dev/null || true
  fi
}
trap _cleanup EXIT

_info "[sync-apps] Starting argocd-server port-forward..."
kubectl port-forward svc/argocd-server -n "${ARGOCD_NS}" 8080:443 \
  --context "${INFRA_CONTEXT}" >/dev/null 2>&1 &
_pf_pid=$!
sleep 3

_argocd_pass=$(kubectl get secret argocd-initial-admin-secret \
  -n "${ARGOCD_NS}" --context "${INFRA_CONTEXT}" \
  -o jsonpath='{.data.password}' | base64 -d)

_info "[sync-apps] Logging in to ArgoCD..."
argocd login localhost:8080 --username admin --password "${_argocd_pass}" --insecure >/dev/null

_info "[sync-apps] Syncing data-layer..."
argocd app sync cicd/data-layer --timeout 180

_info "[sync-apps] Pod status (${APP_CONTEXT}):"
kubectl get pods -A --context "${APP_CONTEXT}"
```
