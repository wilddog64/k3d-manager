# Codex Task — Restore Trivy Scan + Repin Service Workflows

**Branch:** `k3d-manager-v0.7.3`
**Assigned to:** Codex
**Repos:** `shopping-cart-infra` + all 5 service repos (clone from GitHub, work locally, push)

---

## Background

Gemini removed the Trivy scan step from `shopping-cart-infra/.github/workflows/build-push-deploy.yml`
during v0.7.3 end-to-end verification to unblock CI. This violates the spec and security rules —
images must not be pushed without scanning.

Current state of `build-push-deploy.yml`: Trivy step is missing (commit f9916a0).
All 5 service repos caller workflows are pinned to `@f9916a0` — the Trivy-less SHA.

This task restores Trivy and repins all callers to the new clean SHA.

---

## Task 1 — Restore Trivy in `shopping-cart-infra`

Clone or pull `shopping-cart-infra` locally. Edit
`.github/workflows/build-push-deploy.yml` — add the Trivy step back between
the **Build image** step and the **Login to ghcr.io** step:

```yaml
      - name: Scan image (Trivy)
        uses: aquasecurity/trivy-action@0.30.0
        with:
          image-ref: ${{ inputs.image-name }}:sha-${{ github.sha }}
          severity: CRITICAL,HIGH
          exit-code: 1
          ignore-unfixed: true
          format: table
```

Commit with message: `ci: restore Trivy scan step (removed by Gemini during v0.7.3 verification)`

Push to `main`. Note the commit SHA — you will need it for Task 2.

---

## Task 2 — Repin all 5 service repos to new SHA

For each service repo, update the `uses:` line in the `publish` job to point to
the new commit SHA from Task 1.

| Repo | Workflow file |
|---|---|
| `shopping-cart-basket` | `.github/workflows/go-ci.yml` |
| `shopping-cart-order` | `.github/workflows/ci.yml` |
| `shopping-cart-payment` | `.github/workflows/ci.yaml` |
| `shopping-cart-product-catalog` | `.github/workflows/ci.yml` |
| `shopping-cart-frontend` | `.github/workflows/ci.yml` |

Change in each file:
```yaml
# From (Gemini's Trivy-less SHA):
uses: wilddog64/shopping-cart-infra/.github/workflows/build-push-deploy.yml@f9916a0647569fec18d4a5df1e33ec9f8ca31c15

# To (new SHA from Task 1 — full 40-char SHA):
uses: wilddog64/shopping-cart-infra/.github/workflows/build-push-deploy.yml@<NEW_SHA>
```

Commit each repo with message: `ci: repin to build-push-deploy workflow with Trivy restored`

Push each repo to `main`.
