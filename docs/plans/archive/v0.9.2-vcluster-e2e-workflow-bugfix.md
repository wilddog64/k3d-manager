## Bug 1 — Teardown step incorrectly recreates the k3d cluster

**Location:** `Teardown` step, first line of the `run` block.

**Current (wrong):**
```yaml
- name: Teardown
  if: inputs.action == 'teardown'
  env:
    RUN_ID: ${{ inputs.run-id }}
  run: |
    set -euo pipefail
    cluster="e2e-${RUN_ID}"
    k3d cluster create "$cluster" --no-lb --wait >/dev/null 2>&1 || true  # <-- BUG
    kubectl config use-context "k3d-${cluster}" || true
    ./scripts/k3d-manager vcluster_destroy "$cluster" || true
    k3d cluster delete "$cluster" || true
```

**Fix:** Remove the `k3d cluster create` line entirely. Teardown assumes the cluster
already exists (or handles gracefully if it doesn't via the `|| true` on delete).

**Correct teardown `run` block:**
```bash
set -euo pipefail
cluster="e2e-${RUN_ID}"
kubectl config use-context "k3d-${cluster}" || true
./scripts/k3d-manager vcluster_destroy "$cluster" || true
k3d cluster delete "$cluster" || true
```

## Bug 2 — Output wiring is broken

**Root cause:** The `Setup` step has no `id`, so GitHub Actions cannot reference its
`$GITHUB_OUTPUT` writes. The separate `Set outputs` step tries to read from
`steps.run.outputs` — but `run` is a job id, not a step id. No step with id `run`
exists. Result: `kubeconfig` and `vcluster-name` outputs are always empty.

**Current (wrong):**
```yaml
    outputs:
      kubeconfig: ${{ steps.set-outputs.outputs.kubeconfig }}   # broken
      vcluster-name: ${{ steps.set-outputs.outputs.vcluster-name }}  # broken

    steps:
      ...
      - name: Setup          # no id — outputs can't be referenced
        if: inputs.action == 'setup'
        run: |
          ...
          echo "kubeconfig=$kubeconfig" >> "$GITHUB_OUTPUT"
          echo "vcluster-name=$cluster" >> "$GITHUB_OUTPUT"

      - name: Set outputs    # reads from steps.run.outputs — step id 'run' doesn't exist
        id: set-outputs
        if: inputs.action == 'setup'
        run: |
          echo "kubeconfig=${{ steps.run.outputs.kubeconfig || '' }}" >> "$GITHUB_OUTPUT"
          echo "vcluster-name=${{ steps.run.outputs.vcluster-name || '' }}" >> "$GITHUB_OUTPUT"
```

**Fix:**
1. Add `id: setup` to the Setup step
2. Remove the `Set outputs` step entirely — it's redundant once Setup has an id
3. Update job `outputs` to reference `steps.setup.outputs.*`

**Correct:**
```yaml
    outputs:
      kubeconfig: ${{ steps.setup.outputs.kubeconfig }}
      vcluster-name: ${{ steps.setup.outputs.vcluster-name }}

    steps:
      ...
      - name: Setup
        id: setup            # add this
        if: inputs.action == 'setup'
        run: |
          ...
          echo "kubeconfig=$kubeconfig" >> "$GITHUB_OUTPUT"
          echo "vcluster-name=$cluster" >> "$GITHUB_OUTPUT"
      # remove the Set outputs step entirely
```
