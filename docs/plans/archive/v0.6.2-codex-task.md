## Batch 4 — Security Hardening

### 4a. `_safe_path()` helper

**Add to `scripts/lib/system.sh` near `_args_have_sensitive_flag` (after line ~138).**

Validates that PATH does not contain world-writable directories before critical operations.

```
Logic:
1. local dir
2. local IFS=':'
3. for dir in $PATH; do
     # Skip empty segments
     [[ -z "$dir" ]] && continue
     # Check if directory is world-writable
     if [[ -d "$dir" ]] && [[ -w "$dir" ]] && stat output shows world-writable; then
       Use: stat -f '%Sp' "$dir" (macOS) or stat -c '%A' "$dir" (Linux)
       Check if permissions string contains 'w' at position for 'other'
       _warn "PATH contains world-writable directory: $dir"
       return 1
     fi
   done
4. return 0
```

Platform note: `stat` syntax differs between macOS and Linux. Use `_is_mac` to branch.

### 4b. Stdin secret injection in `ldap-password-rotator.sh`

**File: `scripts/etc/ldap/ldap-password-rotator.sh` — around line 84-88**

Current (secrets visible in process listing):
```bash
kubectl exec -n "$VAULT_NAMESPACE" vault-0 -- \
    env VAULT_TOKEN="$vault_token" VAULT_ADDR="$VAULT_ADDR" \
    vault kv put "$vault_path" \
    username="$username" \
    password="$new_password" \
```

Change to stdin-based injection:
```bash
kubectl exec -i -n "$VAULT_NAMESPACE" vault-0 -- \
    env VAULT_TOKEN="$vault_token" VAULT_ADDR="$VAULT_ADDR" \
    sh -c 'read -r pw; vault kv put "'"$vault_path"'" username="'"$username"'" password="$pw"' \
    <<< "$new_password"
```

Key points:
- Add `-i` flag to `kubectl exec` for stdin passthrough
- Password arrives via stdin, not in command args
- `$vault_path` and `$username` are not secrets — OK in args
- Only `$new_password` must be protected from `/proc` sniffing
