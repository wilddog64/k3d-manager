### Step 2 — Inspect and patch ESO CRD to serve `v1beta1`

First inspect the current CRD version state:
```bash
kubectl --context ubuntu-k3s get crd externalsecrets.external-secrets.io \
  -o json | jq '[.spec.versions[] | {name: .name, served: .served, storage: .storage}]'
```

Report this output verbatim.

If `v1beta1` exists with `served: false`, find its index and patch:
```bash
# Find the index of v1beta1 in the versions array
INDEX=$(kubectl --context ubuntu-k3s get crd externalsecrets.external-secrets.io \
  -o json | jq '.spec.versions | to_entries[] | select(.value.name == "v1beta1") | .key')

echo "v1beta1 is at index: $INDEX"

# Patch served=true
kubectl --context ubuntu-k3s patch crd externalsecrets.external-secrets.io \
  --type=json \
  -p "[{\"op\":\"replace\",\"path\":\"/spec/versions/${INDEX}/served\",\"value\":true}]"
```

Also patch `secretstores.external-secrets.io` and `clustersecretstores.external-secrets.io`
with the same approach if they also have `v1beta1` unserved.

If `v1beta1` is NOT present in the CRD at all — report that and stop. Do not attempt
to add it; this requires ESO reinstallation which is out of scope for this session.

Verify after patching:
```bash
kubectl --context ubuntu-k3s get crd externalsecrets.external-secrets.io \
  -o json | jq '[.spec.versions[] | {name: .name, served: .served}]'
```

### Step 3 — Verify ArgoCD manager RBAC

```bash
kubectl --context ubuntu-k3s get serviceaccount argocd-manager -n kube-system 2>/dev/null \
  && echo "SA exists" || echo "SA missing"
kubectl --context ubuntu-k3s get clusterrolebinding argocd-manager 2>/dev/null \
  && echo "CRB exists" || echo "CRB missing"
```

If either is missing, create idempotently:
```bash
kubectl --context ubuntu-k3s create serviceaccount argocd-manager -n kube-system \
  --dry-run=client -o yaml | kubectl --context ubuntu-k3s apply -f -

kubectl --context ubuntu-k3s create clusterrolebinding argocd-manager \
  --clusterrole=cluster-admin \
  --serviceaccount=kube-system:argocd-manager \
  --dry-run=client -o yaml | kubectl --context ubuntu-k3s apply -f -
```

Note: `cluster-admin` is required for ArgoCD to manage workloads — this is the standard
ArgoCD pattern, not a security misconfiguration.

- `data-layer` pods (postgres, rabbitmq, redis): `1/1 Running`

If any app pod is not Running after sync: paste the pod description:
