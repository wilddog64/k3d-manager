# v0.9.4 — Full Stack Health

**Branch:** `k3d-manager-v0.9.4` (cut from main after v0.9.3 merges)
**Theme:** Everything end-to-end green — shopping cart apps Running, Playwright E2E passing in CI.
**Gate:** Playwright E2E green on a scheduled run = milestone done.

---

## Context

After v0.9.3 ships the cluster rebuild smoke test, the app cluster has all infra components
Running but shopping-cart apps are in `ImagePullBackOff`. Root cause: images have never been
pushed to ghcr.io since CI was stabilized in v0.7.3. The reusable workflow already handles
build + Trivy + ghcr.io push + kustomize manifest update — it just needs to run.

The `shopping-cart-e2e-tests` scheduled run was disabled (2026-03-16) to stop daily failure
noise. Re-enable is gated on this milestone completing.

---

## Tasks

### 1. Push Images to ghcr.io (Owner)

Trigger CI in each shopping cart repo. Options:
- `workflow_dispatch` from GitHub UI / `gh workflow run`
- Push a trivial commit to `main` in each repo

Repos that need image push:
- `shopping-cart-order`
- `shopping-cart-payment`
- `shopping-cart-product-catalog`
- `shopping-cart-frontend`
- `shopping-cart-data` (if image-based; verify — may be Helm-only)

Verify each image lands in GHCR:
```bash
gh api /orgs/wilddog64/packages?package_type=container | jq '.[].name'
# or per-repo:
gh api /repos/wilddog64/shopping-cart-order/packages?package_type=container
```

**Note:** The kustomize update step in the reusable workflow commits an image tag back to the
repo. Confirm that commit fires and ArgoCD detects it.

### 2. NVD API Key (Owner)

`shopping-cart-order` and `shopping-cart-payment` CI fails on CVE scan without NVD API key.
- Register at https://nvd.nist.gov/developers/request-an-api-key
- Add as `NVD_API_KEY` Actions secret to both repos:
  ```bash
  gh secret set NVD_API_KEY --repo wilddog64/shopping-cart-order
  gh secret set NVD_API_KEY --repo wilddog64/shopping-cart-payment
  ```

### 3. Verify App Cluster Health (Gemini — M2 Air)

After CI runs complete and kustomize manifests are updated:

```bash
# ArgoCD: all 5 apps Synced + Healthy
kubectl get applications -n cicd

# App cluster: no ImagePullBackOff
ssh ubuntu "kubectl get pods -A" | grep -v Running | grep -v Completed
```

Expected: all 5 shopping-cart apps Running on Ubuntu k3s.

### 4. Re-enable Scheduled E2E Tests (Claude)

In `shopping-cart-e2e-tests/.github/workflows/e2e-tests.yml`:
- Uncomment the `schedule: cron: '0 6 * * *'` block
- Remove the "re-enable once ImagePullBackOff resolved" comment

### 5. Playwright E2E Green in CI (CI / Gemini verify)

Confirm scheduled run passes:
```bash
gh run list --repo wilddog64/shopping-cart-e2e-tests --limit 5
gh run view <run-id> --repo wilddog64/shopping-cart-e2e-tests
```

Expected: `api-tests` + `flow-tests` both green.
