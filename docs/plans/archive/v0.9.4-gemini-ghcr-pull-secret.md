# Gemini Task: Create ghcr-pull-secret on Ubuntu k3s (v0.9.4)

## Before You Start

1. **First command:** `hostname && uname -n` — confirm you are on M2 Air
2. Ensure SSH tunnel is active: `ssh -fNL 0.0.0.0:6443:localhost:6443 ubuntu`
3. `export KUBECONFIG=~/.kube/ubuntu-k3s.yaml`
4. `kubectl get nodes` — confirm k3s node is Ready

---

## Context

All 5 shopping-cart images are in `ghcr.io/wilddog64/` (private).
ArgoCD syncs are failing with `ImagePullBackOff` because k3s pods have no credentials
to pull from ghcr.io. Fix: create a `ghcr-pull-secret` in each app namespace.

---

## Step 1 — Create ghcr-pull-secret in all 3 namespaces

You need a GitHub Personal Access Token (PAT) with `read:packages` scope.
The token is already stored in Vault — retrieve it:

```bash
# On M2 Air, infra cluster context
kubectl config use-context <infra-cluster-context>
vault kv get secret/github/pat-read-packages
```

If not in Vault, use the PAT you have locally. Do NOT commit the token anywhere.

```bash
export KUBECONFIG=~/.kube/ubuntu-k3s.yaml
export GITHUB_USER=wilddog64
export GITHUB_PAT=<token>

for ns in shopping-cart-apps shopping-cart-payment shopping-cart-data; do
  kubectl create secret docker-registry ghcr-pull-secret \
    --namespace="${ns}" \
    --docker-server=ghcr.io \
    --docker-username="${GITHUB_USER}" \
    --docker-password="${GITHUB_PAT}" \
    --dry-run=client -o yaml | kubectl apply -f -
done
```

Verify:
```bash
kubectl get secret ghcr-pull-secret -n shopping-cart-apps
kubectl get secret ghcr-pull-secret -n shopping-cart-payment
```

---

## Step 2 — Force ArgoCD to re-sync all 5 apps

```bash
# Switch back to infra cluster
kubectl config use-context <infra-cluster-context>

for app in shopping-cart-order shopping-cart-payment \
           shopping-cart-product-catalog shopping-cart-frontend \
           shopping-cart-basket; do
  argocd app sync "${app}" --force
done
```

## Step 3 — Verify pods Running

```bash
export KUBECONFIG=~/.kube/ubuntu-k3s.yaml

# Watch until all pods are Running (no ImagePullBackOff)
kubectl get pods -n shopping-cart-apps
kubectl get pods -n shopping-cart-payment
```

Expected: all 5 pods `Running`, `2/2` READY (2/2 = app + Istio sidecar).
If still `1/1`, Istio sidecar injection may not be active yet — that is acceptable for v0.9.4.
