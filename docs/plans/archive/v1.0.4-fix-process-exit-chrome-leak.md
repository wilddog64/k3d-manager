# v1.0.4 — Fix process.exit(0) Chrome Leak in acg_credentials.js

**Date**: 2026-04-03
**Branch**: k3d-manager-v1.0.2
**Status**: COMPLETE (`b06451b`)
**Issue**: `docs/issues/2026-04-03-process-exit-bypasses-finally-chrome-leak.md`

## Problem

`acg_credentials.js` called `process.exit(0)` on success inside a `try` block. In Node.js,
`process.exit()` bypasses all `finally` blocks, so `browserContext.close()` never ran and Chrome
was left open after every successful credential extraction.

## Fix Applied

One-line change in `scripts/playwright/acg_credentials.js`:

Old:
```js
      process.exit(0);
```

New:
```js
      return;
```

`return` lets the async function resolve normally → `finally` executes → `browserContext.close()`
runs → Chrome exits → Node event loop drains → process exits with code 0.

## Files Changed

- `scripts/playwright/acg_credentials.js` — line 255: `process.exit(0)` → `return`
