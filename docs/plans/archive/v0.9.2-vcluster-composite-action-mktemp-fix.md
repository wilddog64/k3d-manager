# Task: Fix Copilot Findings — mktemp + teardown dependency check

## Context

Two new Copilot findings after the `curl -fsSL` fix:

**Finding 1 (id: 2937310800)** — predictable `/tmp` path is a TOCTOU risk:
> `/tmp/k3d-install.sh` is a fixed, predictable filename in a world-writable
> directory. Another process in the same job could tamper with it between
> download and execution (symlink attack). Use `mktemp` for a unique temp file,
> set restrictive permissions, and remove it via `trap` after execution.

**Finding 2 (id: 2937310807)** — teardown assumes kubectl/k3d installed:
> If a caller runs teardown independently, or if setup fails before installing
> dependencies, `kubectl` and `k3d` commands will emit confusing
> `command not found` errors. Add explicit dependency checks with a clear
> error message so failures are easier to diagnose.

## Step 2 — Fix Install k3d step with mktemp

File: `.github/actions/vcluster-e2e-setup/action.yml`

Replace the `Install k3d` step `run` block:

**Before:**
```yaml
      run: |
        set -euo pipefail
        k3d_version="${{ inputs.k3d-version }}"
        if [[ ! "$k3d_version" =~ ^v[0-9]+\.[0-9]+\.[0-9]+$ ]]; then
          echo "Error: k3d-version must match vX.Y.Z (got: $k3d_version)" >&2
          exit 1
        fi
        base_url="https://raw.githubusercontent.com/k3d-io/k3d"
        installer="${base_url}/${k3d_version}/install.sh"
        curl -fsSL "$installer" -o /tmp/k3d-install.sh
        sudo bash /tmp/k3d-install.sh
```

**After:**
```yaml
      run: |
        set -euo pipefail
        k3d_version="${{ inputs.k3d-version }}"
        if [[ ! "$k3d_version" =~ ^v[0-9]+\.[0-9]+\.[0-9]+$ ]]; then
          echo "Error: k3d-version must match vX.Y.Z (got: $k3d_version)" >&2
          exit 1
        fi
        base_url="https://raw.githubusercontent.com/k3d-io/k3d"
        installer="${base_url}/${k3d_version}/install.sh"
        tmp_script="$(mktemp /tmp/k3d-install.XXXXXX)"
        trap 'rm -f "$tmp_script"' EXIT
        curl -fsSL "$installer" -o "$tmp_script"
        chmod 600 "$tmp_script"
        sudo bash "$tmp_script"
```

## Step 3 — Add dependency check to teardown action

File: `.github/actions/vcluster-e2e-teardown/action.yml`

Add a dependency check step before the teardown step:

**Before** (the single existing step):
```yaml
  steps:
    - name: Teardown vCluster and k3d cluster
      shell: bash
      env:
        RUN_ID: ${{ inputs.run-id }}
      run: |
        set -euo pipefail
        cluster="e2e-${RUN_ID}"
        kubectl config use-context "k3d-${cluster}" || true
        manager="${{ github.action_path }}"/../../../scripts/k3d-manager
        "$manager" vcluster_destroy "$cluster" || true
        k3d cluster delete "$cluster" || true
```

**After:**
```yaml
  steps:
    - name: Check dependencies
      shell: bash
      run: |
        set -euo pipefail
        for cmd in kubectl k3d; do
          if ! command -v "$cmd" &>/dev/null; then
            echo "Error: $cmd is not installed. This action must run in the same job as vcluster-e2e-setup." >&2
            exit 1
          fi
        done

    - name: Teardown vCluster and k3d cluster
      shell: bash
      env:
        RUN_ID: ${{ inputs.run-id }}
      run: |
        set -euo pipefail
        cluster="e2e-${RUN_ID}"
        kubectl config use-context "k3d-${cluster}" || true
        manager="${{ github.action_path }}"/../../../scripts/k3d-manager
        "$manager" vcluster_destroy "$cluster" || true
        k3d cluster delete "$cluster" || true
```
