## Problem

The remote k3s cluster (ubuntu-k3s, ESO 0.9.20) serves ExternalSecret and ClusterSecretStore
under `external-secrets.io/v1`. The manifests in `shopping-cart-infra/data-layer/secrets/`
use `external-secrets.io/v1beta1`, which is not served. ArgoCD sync for `data-layer` and
`product-catalog` apps fails with:

```
ExternalSecret version v1beta1 is missing from the remote cluster
```

Without the data-layer (PostgreSQL, Redis, RabbitMQ), all app pods CrashLoop.
