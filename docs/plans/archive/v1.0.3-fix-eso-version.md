# v1.0.3 — Fix ESO version in `bin/acg-up`

**Date**: 2026-04-04
**Branch**: k3d-manager-v1.0.3
**Status**: ASSIGNED to Codex

## Problem

`bin/acg-up` installs ESO v0.9.20 on the remote cluster by default. ESO v0.9.20 only
serves `external-secrets.io/v1alpha1` and `v1beta1`. The `shopping-cart-infra` manifests
(updated in PR #23) use `external-secrets.io/v1`, causing ArgoCD data-layer sync to fail
on every fresh sandbox. All app pods stay in `CrashLoopBackOff`.

Issue: `docs/issues/2026-04-04-eso-version-mismatch-v1alpha1.md`

## Files Changed

1. `bin/acg-up` — bump default `ESO_VERSION` from `0.9.20` to `0.14.0`
