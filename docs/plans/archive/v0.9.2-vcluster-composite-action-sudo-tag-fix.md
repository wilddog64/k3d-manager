# Task: Fix Copilot Findings — TAG env, sudo -n, teardown dependency message

## Context

Three new Copilot findings after the mktemp fix:

**Finding 1 (id: 2937365889)** — `k3d-version` not passed to installer:
> `k3d-version` is only used to build the URL. The installer selects the
> version via `TAG=...` env var. Without passing `TAG="$k3d_version"` the
> installer may install an unexpected version.

**Finding 2 (id: 2937365900)** — `sudo` without `-n` can hang:
> On self-hosted runners where sudo requires a password, `sudo bash` blocks
> indefinitely. Use `sudo -n bash` to fail fast with a clear error.

**Finding 3 (id: 2937365906)** — teardown dependency message ambiguous + missing `$manager` check:
> (a) Error text `run with the vcluster-e2e-setup job` is ambiguous — say
> explicitly this teardown must run in the same job as the setup action.
> (b) The `$manager` path (scripts/k3d-manager via github.action_path) is not
> validated — fail fast with a clear error if it's missing or not executable.

## Step 2 — Fix setup action

File: `.github/actions/vcluster-e2e-setup/action.yml`

In the `Install k3d` step, replace:

```yaml
        curl -fsSL "$installer" -o "$tmp_script"
        chmod 600 "$tmp_script"
        sudo bash "$tmp_script"
```

With:

```yaml
        curl -fsSL "$installer" -o "$tmp_script"
        chmod 600 "$tmp_script"
        sudo -n env TAG="$k3d_version" bash "$tmp_script"
```

This addresses both Finding 1 (passes `TAG` so installer pins the version) and
Finding 2 (`-n` fails fast instead of hanging for a password prompt).

## Step 3 — Fix teardown action

File: `.github/actions/vcluster-e2e-teardown/action.yml`

Replace the `Check dependencies` step `run` block:

**Before:**
```yaml
      run: |
        set -euo pipefail
        for cmd in kubectl k3d; do
          if ! command -v "$cmd" &>/dev/null; then
            echo "Error: $cmd missing; run with the vcluster-e2e-setup job." >&2
            exit 1
          fi
        done
```

**After:**
```yaml
      run: |
        set -euo pipefail
        for cmd in kubectl k3d; do
          if ! command -v "$cmd" &>/dev/null; then
            echo "Error: '$cmd' not found. This action must run in the same job as vcluster-e2e-setup." >&2
            exit 1
          fi
        done
        manager="${{ github.action_path }}"/../../../scripts/k3d-manager
        if [[ ! -x "$manager" ]]; then
          echo "Error: k3d-manager not found or not executable at: $manager" >&2
          exit 1
        fi
```
