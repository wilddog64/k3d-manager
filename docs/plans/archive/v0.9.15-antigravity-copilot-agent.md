# Spec: Antigravity × GitHub Copilot Coding Agent — Validation (v0.9.15)

## Goal

Validate whether Antigravity's browser automation surface can reliably trigger a GitHub
Copilot coding agent task, poll for completion, and extract the output. Run the test 3
times to assess determinism before building ACG automation on top of Antigravity in v0.9.16.

This milestone produces **no code changes** — only a findings doc and a determinism verdict.

---

## Background

- GitHub Copilot coding agent is web-UI only — no public REST/GraphQL API
- Tasks live at `github.com/wilddog64/k3d-manager/agents` (trigger) and
  `github.com/wilddog64/k3d-manager/tasks/<uuid>` (output)
- The `read:project` scope does not expose agent tasks via GraphQL
- Antigravity has a browser automation surface — the only programmatic path to this UI

---

## Before You Start

- Confirm Antigravity is running and browser surface is available
- Have `github.com` logged in as `wilddog64` in the Antigravity browser
- Read this spec in full before starting

---

## Test Procedure

### Run 1 — Trigger and extract

1. Navigate to `github.com/wilddog64/k3d-manager/agents`
2. Trigger a new Copilot coding agent task with prompt:
   ```
   Review the k3d-manager codebase for code quality, security, and architecture.
   Focus on: scripts/lib/system.sh, scripts/lib/system_overrides.sh,
   scripts/plugins/vault.sh, scripts/lib/providers/k3d.sh
   ```
3. Record the task UUID from the resulting URL (`/tasks/<uuid>`)
4. Poll `github.com/wilddog64/k3d-manager/tasks/<uuid>` until status = complete
5. Extract the full output text
6. Record: UUID, time-to-complete, output length (chars), top findings

### Run 2 — Repeat with same prompt

Same prompt as Run 1. Record: UUID, time-to-complete, output length, top findings.

### Run 3 — Repeat with same prompt

Same prompt as Run 1. Record: UUID, time-to-complete, output length, top findings.

---

## Determinism Criteria

After 3 runs, answer these questions:

| Question | Pass | Fail |
|---|---|---|
| Did all 3 runs complete without error? | Yes | Any run failed/timed out |
| Are the top findings consistent across runs? | Same P1/P2 findings in all 3 | Different findings each run |
| Is the output structure consistent? | Same sections/headings | Different structure |
| Time-to-complete variance | < 2× between fastest and slowest | > 3× variance |
| Can output be reliably extracted as text? | Yes, clean text every run | Partial/garbled output |
