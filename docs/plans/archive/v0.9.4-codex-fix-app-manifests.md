## Context

The ubuntu-k3s app cluster (EC2) was freshly reprovisioned. Three services fail to start
because the k8s manifests in the app repos have wrong credentials and configuration that
don't match the data-layer manifests in `shopping-cart-infra`.

Fixes identified by cross-referencing:
- `shopping-cart-infra/data-layer/postgresql/*/secret.yaml` — actual DB passwords
- `shopping-cart-infra/data-layer/rabbitmq/configmap.yaml` — actual RabbitMQ credentials
- `shopping-cart-infra/data-layer/rabbitmq/statefulset.yaml` — replicas: 3 too heavy for t3.medium
- Gemini live cluster verification — readiness probe path, Vault connectivity

## Repo 1 — `shopping-cart-order` (branch: `main`)

### File: `k8s/base/secret.yaml`

**Old:**
```yaml
  DB_USERNAME: "postgres"
  DB_PASSWORD: "CHANGE_ME"

  RABBITMQ_USERNAME: "guest"
  RABBITMQ_PASSWORD: "CHANGE_ME"
```

**New:**
```yaml
  DB_USERNAME: "postgres"
  DB_PASSWORD: "changeme456"

  RABBITMQ_USERNAME: "guest"
  RABBITMQ_PASSWORD: "guest"
```

### File: `k8s/base/configmap.yaml`

Add `SPRING_CLOUD_VAULT_ENABLED: "false"` after `VAULT_ENABLED: "true"`:

**Old:**
```yaml
  VAULT_ENABLED: "true"
  VAULT_ADDR: "http://vault.vault.svc.cluster.local:8200"
```

**New:**
```yaml
  VAULT_ENABLED: "true"
  SPRING_CLOUD_VAULT_ENABLED: "false"
  VAULT_ADDR: "http://vault.vault.svc.cluster.local:8200"
```

**Why:** Vault runs on the infra cluster (k3d on M2 Air), not on ubuntu-k3s. The app cluster
cannot reach `vault.vault.svc.cluster.local`. Spring Cloud Vault autoconfiguration must be
disabled or order-service fails at bean creation.

### File: `k8s/base/deployment.yaml`

Reduce resource requests (t3.medium has 4GB RAM, full stack needs headroom):

**Old:**
```yaml
        resources:
          requests:
            cpu: 200m
            memory: 512Mi
          limits:
            cpu: 1000m
            memory: 1Gi
```

**New:**
```yaml
        resources:
          requests:
            cpu: 100m
            memory: 256Mi
          limits:
            cpu: 500m
            memory: 512Mi
```
