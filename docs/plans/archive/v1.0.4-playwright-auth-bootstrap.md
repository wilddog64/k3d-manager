# v1.0.4 — Playwright Auth Bootstrap Detection

**Date**: 2026-04-03
**Branch**: k3d-manager-v1.0.2
**Status**: Ready for Codex

## Problem

After the Chrome `launchPersistentContext` refactor (commit `f7f15c5`), first-time runs use an
empty auth dir (`~/.local/share/k3d-manager/playwright-auth/`). The existing code opens Chrome
and enters the sign-in flow, but the 120s overall timeout (`acg_credentials.js`) and 90s timeout
(`acg_extend.js`) are not enough for a human to log in manually. Result: script times out and exits
with an error before credentials are extracted.

Additionally, `acg_extend.js` has no sign-in handling at all — if the auth dir is empty, it navigates
to the sandbox URL, finds it behind a login wall, and fails without a helpful error message.

Old:

New:

**Change 2 — add first-run detection at module level** (after `AUTH_DIR` const, before `async function extractCredentials()`):

Old:
```js
const AUTH_DIR = path.join(os.homedir(), '.local', 'share', 'k3d-manager', 'playwright-auth');

async function extractCredentials() {
```

New:
```js
const AUTH_DIR = path.join(os.homedir(), '.local', 'share', 'k3d-manager', 'playwright-auth');

function _isFirstRun() {
  try {
    return !fs.existsSync(AUTH_DIR) || fs.readdirSync(AUTH_DIR).length === 0;
  } catch {
    return true;
  }
}
const IS_FIRST_RUN = _isFirstRun();

async function extractCredentials() {
```

**Change 3 — print bootstrap banner when IS_FIRST_RUN** (at start of `extractCredentials()`, after the `if (!targetUrl)` guard block):

Old:
```js
  let browserContext;
  try {
    browserContext = await chromium.launchPersistentContext(AUTH_DIR, {
      headless: false,
      channel: 'chrome',
      args: ['--password-store=basic'],
    });
```

New:
```js
  if (IS_FIRST_RUN) {
    console.error('BOOTSTRAP: Auth dir is empty — first run detected.');
    console.error(`BOOTSTRAP: Auth dir: ${AUTH_DIR}`);
    console.error('BOOTSTRAP: Chrome will open. Please log in to Pluralsight when prompted.');
    console.error('BOOTSTRAP: The script will continue automatically after successful login (up to 300s).');
  }

  let browserContext;
  try {
    browserContext = await chromium.launchPersistentContext(AUTH_DIR, {
      headless: false,
      channel: 'chrome',
      args: ['--password-store=basic'],
    });
```

**Change 4 — extend overall timeout when IS_FIRST_RUN** (bottom of file, `OVERALL_TIMEOUT_MS` const):

Old:
```js

```

New:
```js

```

### `scripts/playwright/acg_extend.js`

**Change 1 — add `fs` import** (line 3, after `path` import):

Old:
```js
const { chromium } = require('playwright');

```

New:
```js
const { chromium } = require('playwright');
const fs = require('fs');
const os = require('os');
const path = require('path');
```

**Change 2 — add first-run guard at module level + early-exit in function** (after `AUTH_DIR` const):

Old:
```js
const AUTH_DIR = path.join(os.homedir(), '.local', 'share', 'k3d-manager', 'playwright-auth');

async function extendSandbox() {
  const targetUrl = process.argv[2];
```

New:
```js
const AUTH_DIR = path.join(os.homedir(), '.local', 'share', 'k3d-manager', 'playwright-auth');

function _isFirstRun() {
  try {
    return !fs.existsSync(AUTH_DIR) || fs.readdirSync(AUTH_DIR).length === 0;
  } catch {
    return true;
  }
}

async function extendSandbox() {
  if (_isFirstRun()) {
    console.error(`ERROR: Auth dir is empty (${AUTH_DIR}).`);
    console.error('ERROR: Run acg_get_credentials <sandbox-url> first to bootstrap the Pluralsight session.');
    process.exit(1);
  }

  const targetUrl = process.argv[2];
```

No timeout change needed for `acg_extend.js` — the early exit handles the first-run case.
