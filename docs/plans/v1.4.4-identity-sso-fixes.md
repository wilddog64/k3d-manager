## Problem

ArgoCD identity app sync fails and SSO does not work end-to-end. Four root causes found:

1. **Kustomizations wire in `secret.yaml` (plaintext `CHANGE_ME`) instead of ExternalSecrets** — Keycloak and LDAP pods never get real credentials from Vault.
2. **`frontend` OIDC client missing from `realm-shopping-cart.json`** — the React SPA can never redirect to Keycloak; `argocd` client also gets "Invalid redirect URL" because the realm is never imported.
3. **No ArgoCD Application for the identity stack in `shopping-cart-infra/argocd/applications/`** — the app-of-apps (`shopping-cart-apps.yaml`) only picks up files in that directory; identity is absent.
4. **`acg-up` has no step to import the Keycloak realm** — Keycloak runs `start --optimized` with no `--import-realm` flag; realm-shopping-cart.json is never loaded.

### k3d-manager — Branch: `k3d-manager-v1.4.4`

#### Change 6 — `bin/acg-up`: add Step 10c — deploy identity app + import Keycloak realm

Add the following block **after** the existing Step 10b (`deploy_shopping_cart_data`) block and **before** Step 11 (`Verifying ClusterSecretStore`).

**Exact insertion point — after line:**
```bash
_info "[acg-up] Step 10b/12 — Deploying shopping-cart data layer (PostgreSQL, Redis, RabbitMQ)..."
deploy_shopping_cart_data
```

**Insert this block:**
```bash

_info "[acg-up] Step 10c/12 — Deploying identity stack (Keycloak + LDAP) via ArgoCD..."
# Force ArgoCD to pick up the identity Application from shopping-cart-infra
kubectl apply --context k3d-k3d-cluster -f - <<'IDEOF'
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: shopping-cart-identity
  namespace: cicd
  annotations:
    argocd.argoproj.io/sync-wave: "-1"
  finalizers:
    - resources-finalizer.argocd.argoproj.io
spec:
  project: shopping-cart
  sources:
    - repoURL: https://github.com/wilddog64/shopping-cart-infra.git
      targetRevision: HEAD
      path: identity/keycloak
    - repoURL: https://github.com/wilddog64/shopping-cart-infra.git
      targetRevision: HEAD
      path: identity/ldap
  destination:
    server: https://kubernetes.default.svc
    namespace: identity
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
      - CreateNamespace=true
    retry:
      limit: 5
      backoff:
        duration: 10s
        factor: 2
        maxDuration: 3m
IDEOF

_info "[acg-up] Waiting for Keycloak pod to be Running (up to 5 min)..."
_kc_ready=false
for i in $(seq 1 30); do
  _kc_phase=$(kubectl get pods -n identity --context k3d-k3d-cluster \
    -l app.kubernetes.io/name=keycloak \
    -o jsonpath='{.items[0].status.phase}' 2>/dev/null || echo "")
  if [[ "${_kc_phase}" == "Running" ]]; then
    _kc_ready=true
    break
  fi
  _info "[acg-up] Keycloak not Running yet (attempt ${i}/30, phase=${_kc_phase}) — waiting 10s..."
  sleep 10
done
if [[ "${_kc_ready}" != "true" ]]; then
  _info "[acg-up] WARNING: Keycloak not Running after 5 min — skipping realm import. Run manually: bin/acg-up --import-realm"
else
  _info "[acg-up] Step 10d/12 — Importing Keycloak realm shopping-cart..."
  _kc_pf_port=18080
  kubectl port-forward svc/keycloak -n identity --context k3d-k3d-cluster "${_kc_pf_port}:80" \
    >>"${HOME}/.local/share/k3d-manager/keycloak-pf.log" 2>&1 &
  _kc_pf_pid=$!
  disown "${_kc_pf_pid}"
  sleep 3

  _kc_token=$(curl -sf -X POST \
    "http://localhost:${_kc_pf_port}/realms/master/protocol/openid-connect/token" \
    -d "client_id=admin-cli&grant_type=password&username=admin&password=${_kc_admin_pass}" \
    | grep -o '"access_token":"[^"]*"' | sed 's/"access_token":"//;s/"//') || _kc_token=""

  if [[ -z "${_kc_token}" ]]; then
    _info "[acg-up] WARNING: Could not get Keycloak admin token — skipping realm import"
    kill "${_kc_pf_pid}" 2>/dev/null || true
  else
    _realm_json=$(sed \
      -e "s/\${ARGOCD_CLIENT_SECRET}/${_argocd_client_secret}/g" \
      -e "s/\${ORDER_SERVICE_CLIENT_SECRET}/${_order_client_secret}/g" \
      -e "s/\${PRODUCT_CATALOG_CLIENT_SECRET}/${_product_client_secret}/g" \
      -e "s/\${GRAFANA_CLIENT_SECRET}/${_grafana_client_secret}/g" \
      -e "s/\${LDAP_BIND_CREDENTIAL}/${_ldap_admin_pass}/g" \
      "${REPO_ROOT}/../shopping-carts/shopping-cart-infra/identity/keycloak/realm-shopping-cart.json")

    _import_status=$(curl -sf -o /dev/null -w "%{http_code}" \
      -X POST \
      -H "Authorization: Bearer ${_kc_token}" \
      -H "Content-Type: application/json" \
      -d "${_realm_json}" \
      "http://localhost:${_kc_pf_port}/admin/realms" 2>/dev/null || echo "000")

    if [[ "${_import_status}" == "201" ]]; then
      _info "[acg-up] Keycloak realm 'shopping-cart' imported successfully"
    elif [[ "${_import_status}" == "409" ]]; then
      _info "[acg-up] Keycloak realm 'shopping-cart' already exists — skipping import"
    else
      _info "[acg-up] WARNING: Realm import returned HTTP ${_import_status} — check Keycloak logs"
    fi

    kill "${_kc_pf_pid}" 2>/dev/null || true
  fi
fi
```
