# Bugfix: v1.6.0 — acg-up re-run corrupts app state when nodes already Ready

**Branch:** `k3d-manager-v1.6.0`
**Files:** `scripts/lib/providers/k3s-aws.sh`

---

## Problem

When `make up` fails partway through (e.g. during app-layer deployment) and the user
triggers a second `/acg-up` from Slack, `_provider_k3s_aws_deploy_cluster` re-runs
`deploy_app_cluster --confirm` unconditionally — even when all 3 k3s nodes are already
Ready on the remote cluster. Re-running `deploy_app_cluster` on an already-provisioned
cluster re-installs k3s, tears down and recreates the agents, and leaves the cluster in
an inconsistent state that causes shopping-cart pods to CrashLoop.

CloudFormation is already idempotent (lines 52–68 detect stack status and skip or
recreate as needed). The node-install step has no equivalent guard.

**Root cause:** `deploy_app_cluster --confirm` is called unconditionally after the
CloudFormation check, with no pre-flight test of whether k3s nodes are already up.

---

## Reproduction

1. Run `make up` — let it succeed through the CloudFormation step and the k3s install,
   then fail somewhere in the app-layer (ArgoCD sync, shopping-cart deploy).
2. Run `make up` again without running `make down` first.
3. Observe: `deploy_app_cluster` reinstalls k3s, agents rejoin, pods enter CrashLoopBackOff.

---

## Fix

### Change 1 — `scripts/lib/providers/k3s-aws.sh`: skip `deploy_app_cluster` when nodes already Ready

Add a node-readiness pre-flight check immediately before the
`deploy_app_cluster --confirm` call. If ≥ 3 nodes are Ready in the
`ubuntu-k3s` context, skip the install and log it.

**Exact old block:**
```bash
  _info "[k3s-aws] Installing k3s server + joining agents..."
  UBUNTU_K3S_AGENT_HOSTS="ubuntu-1,ubuntu-2" deploy_app_cluster --confirm || return 1
```

**Exact new block:**
```bash
  local _ready_nodes
  _ready_nodes=$(kubectl get nodes --context ubuntu-k3s --no-headers 2>/dev/null \
    | grep -c " Ready" || true)
  if [[ "${_ready_nodes}" -ge 3 ]]; then
    _info "[k3s-aws] k3s nodes already Ready (${_ready_nodes}/3) — skipping deploy_app_cluster"
  else
    _info "[k3s-aws] Installing k3s server + joining agents..."
    UBUNTU_K3S_AGENT_HOSTS="ubuntu-1,ubuntu-2" deploy_app_cluster --confirm || return 1
  fi
```
