# v1.5.4 — Observability + Vulnerability Scanning (Local k3d)

## Goal

Add a lightweight observability stack and image vulnerability scanner to
the local k3d Hub cluster. Istio already emits metrics and traces — this
spec wires up the backends to receive them and surfaces dashboards via
Grafana.

**Scope: local k3d cluster only.** ACG clusters are ephemeral; observability
there is not worth the setup cost per session.

---

## Stack

| Component | Purpose | Memory target |
|---|---|---|
| kube-prometheus-stack | Prometheus + Grafana + AlertManager | ≤ 600 MB |
| Loki | Log aggregation (Promtail as DaemonSet) | ≤ 200 MB |
| Trivy Operator | Image vuln scanning, CIS benchmarks | ≤ 150 MB |

**Not included:** Tempo (traces) — deferred to OCI cluster when available.
Istio traces can be added later by pointing to a Tempo instance.

---

## Architecture

```
k3d Hub cluster (local)
├── monitoring/          ← new namespace
│   ├── prometheus       ← scrapes all cluster metrics incl. Istio
│   ├── grafana          ← dashboards: cluster, Istio, apps
│   └── alertmanager     ← disabled initially
├── logging/             ← new namespace
│   ├── loki             ← log backend
│   └── promtail         ← DaemonSet, tails container logs
└── trivy-system/        ← new namespace
    └── trivy-operator   ← scans images on pod admission + schedule
```

ArgoCD manages all three via a new `platform-helm` entry pattern —
ApplicationSets in `scripts/etc/argocd/applicationsets/`.

### C — New plugin `scripts/plugins/observability.sh`

```bash
#!/usr/bin/env bash
# scripts/plugins/observability.sh

function deploy_observability() {
  _info "[observability] Deploying observability stack to local k3d..."
  local _appset="${REPO_ROOT}/scripts/etc/argocd/applicationsets/observability.yaml"
  K3D_MANAGER_BRANCH="${K3D_MANAGER_BRANCH:-$(git -C "${REPO_ROOT}" rev-parse --abbrev-ref HEAD 2>/dev/null || echo main)}"
  export K3D_MANAGER_BRANCH
  if envsubst '$ARGOCD_NAMESPACE $K3D_MANAGER_BRANCH' < "${_appset}" | _kubectl apply -f -; then
    _info "[observability] ApplicationSet applied — ArgoCD will sync monitoring/logging/trivy-system"
  else
    _err "[observability] Failed to apply observability ApplicationSet"
    return 1
  fi
}

function observability_status() {
  for _ns in monitoring logging trivy-system; do
    _info "[observability] === ${_ns} ==="
    _kubectl get pods -n "${_ns}" --no-headers 2>/dev/null || true
  done
}

function trivy_scan_report() {
  _info "[observability] VulnerabilityReport summary:"
  _kubectl get vulnerabilityreports -A --no-headers 2>/dev/null \
    | awk '{print $1, $2, $6, $7, $8}' \
    | column -t \
    | sort -k4 -rn || true
}
```

### D — Register plugin in dispatcher

In `scripts/k3d-manager`, add `observability` to the plugin list so `deploy_observability`,
`observability_status`, and `trivy_scan_report` are callable.

### E — Makefile target

Add to `Makefile`:
```makefile
## Deploy observability stack (Prometheus, Grafana, Loki, Trivy) to local k3d
observability:
	./scripts/k3d-manager deploy_observability
```

Add `observability` to `.PHONY`.
