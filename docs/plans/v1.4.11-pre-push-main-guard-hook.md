# Spec: v1.4.11 — pre-push main-guard hook (all repos)

**Branch (spec repo):** `k3d-manager-v1.4.11`

---

## Problem

`push.default=upstream` + a feature branch tracking `origin/main` (set by Codex when
creating branches from `origin/main`) causes `git push origin <branch>` to silently push
to `main` instead of creating a remote feature branch. Incident: 2026-05-29, two commits
landed on `shopping-cart-order/main` directly.

---

## Fix

Add a `pre-push` hook to every repo that aborts any push from a non-`main` local branch
to `refs/heads/main`. Escape hatch: `ALLOW_MAIN_PUSH=1 git push` for intentional direct
pushes (e.g. emergency revert commits).

**Hook content (same for all repos — final, includes remote name via `$1`):**

```bash
#!/usr/bin/env bash
set -euo pipefail

remote="$1"

while IFS=' ' read -r local_ref _ remote_ref _; do
  if [[ "$remote_ref" == "refs/heads/main" ]] && \
     [[ "$local_ref" != "refs/heads/main" ]]; then
    if [[ "${ALLOW_MAIN_PUSH:-0}" == "1" ]]; then
      echo "pre-push: ALLOW_MAIN_PUSH=1 — bypassing main-guard for '$local_ref' → ${remote}/main" >&2
    else
      echo "pre-push: refusing to push '$local_ref' directly to ${remote}/main" >&2
      echo "  Use a feature branch + PR instead." >&2
      echo "  To override intentionally: ALLOW_MAIN_PUSH=1 git push ..." >&2
      exit 1
    fi
  fi
done
```
