# v1.6.0 — Cloudflare Worker deploy workflow

## Context

`workers/slack-relay/` was added in `695a04d1` but has no automated deploy step.
Currently requires `wrangler deploy` to be run manually.
This spec adds a GitHub Actions workflow that deploys the Worker on every push to `main`
when `workers/slack-relay/` changes.

| File | Action |
|------|--------|
| `.github/workflows/deploy-worker.yml` | **Create** |

## File 1: `.github/workflows/deploy-worker.yml`

```yaml
---
name: Deploy Cloudflare Worker

"on":
  push:
    branches:
      - main
    paths:
      - 'workers/slack-relay/**'

permissions:
  contents: read

concurrency:
  group: deploy-worker-${{ github.ref }}
  cancel-in-progress: true

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Deploy Slack relay Worker
        uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          workingDirectory: workers/slack-relay
          secrets: |
            WEBHOOK_TOKEN
            SLACK_SIGNING_SECRET
        env:
          WEBHOOK_TOKEN: ${{ secrets.K3DM_WEBHOOK_TOKEN }}
          SLACK_SIGNING_SECRET: ${{ secrets.SLACK_SIGNING_SECRET }}
```

## Secrets to add in GitHub (manual — after workflow is merged)

These must be set in the repo before the workflow can run successfully:

```bash
# CLOUDFLARE_API_TOKEN — create at dash.cloudflare.com → My Profile → API Tokens
# → Use template "Edit Cloudflare Workers" → scope to your account
gh secret set CLOUDFLARE_API_TOKEN

# SLACK_SIGNING_SECRET — from api.slack.com/apps → Basic Information → Signing Secret
gh secret set SLACK_SIGNING_SECRET

# K3DM_WEBHOOK_TOKEN — already set; confirm it exists
gh secret list | grep K3DM_WEBHOOK_TOKEN
```
