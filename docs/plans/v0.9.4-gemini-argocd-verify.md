## Tasks — Run in Order

### 1. Check ArgoCD app status

```bash
# Point kubectl at infra cluster
kubectl config use-context <infra-cluster-context>
kubectl get applications -n cicd
```

Expected: 5 apps listed — `shopping-cart-order`, `shopping-cart-payment`,
`shopping-cart-product-catalog`, `shopping-cart-frontend`, `shopping-cart-basket`.

### 2. Check app cluster pod status

```bash
# Switch to ubuntu k3s context (tunnel must be active)
kubectl config use-context ubuntu-k3s
kubectl get pods -n shopping-cart
```

Look for `ImagePullBackOff` or `ErrImagePull`. Note which apps are affected.

### 3. For any app still in ImagePullBackOff — check kustomization.yaml

```bash
# In shopping-cart-infra repo
cat k8s/base/kustomization.yaml
```

If `newTag` is stale (old SHA or missing `latest`), update it:

```bash
cd ~/src/gitrepo/personal/shopping-carts/shopping-cart-infra
git pull origin main
# Update each affected app's kustomization.yaml newTag to: latest
# Example:
sed -i "s|newTag:.*|newTag: latest|" apps/shopping-cart-order/k8s/base/kustomization.yaml
# Repeat for each affected app
git add -A
git commit -m "fix: update image tags to latest for ArgoCD sync"
git push origin main
```

### 4. Force sync ArgoCD apps

```bash
kubectl config use-context <infra-cluster-context>
for app in shopping-cart-order shopping-cart-payment shopping-cart-product-catalog shopping-cart-frontend shopping-cart-basket; do
  argocd app sync $app --force
done
```

Or via kubectl:
```bash
kubectl patch application shopping-cart-order -n cicd \
  --type merge -p '{"operation": {"initiatedBy": {"username": "gemini"}, "sync": {"syncStrategy": {"hook": {}}}}}'
```

### 5. Wait for all apps to be Synced + Healthy

```bash
kubectl get applications -n cicd -w
```

Expected for each app:
- `SYNC STATUS`: `Synced`
- `HEALTH STATUS`: `Healthy`

### 6. Verify pods Running on app cluster

```bash
kubectl config use-context ubuntu-k3s
kubectl get pods -n shopping-cart
```

Expected: all 5 app pods in `Running` state, no `ImagePullBackOff`.
