# Spec: Remove Redundant Antigravity Calls from `acg_get_credentials`

## Problem

`acg_get_credentials` calls three functions before running the Playwright script:

```bash
_ensure_antigravity        # checks gemini CLI — irrelevant to credential extraction
_antigravity_launch        # opens Chrome — may navigate away from current page
_antigravity_ensure_acg_session  # navigates to Pluralsight — disrupts CDP page state
```

These calls were written before `scripts/playwright/acg_credentials.js` existed.
The static Playwright script now handles everything:
- Finding the Pluralsight page in the running Chrome instance
- Sign-in detection and navigation
- Credential extraction

The three pre-calls actively break extraction: `_antigravity_ensure_acg_session`
navigates Chrome to a fixed URL, which can move the browser away from an already-open
credentials panel. Verified: running the Playwright script directly (without these calls)
works correctly.

The only pre-check needed is that Chrome CDP is accessible on port 9222.

---

## Change

**File:** `scripts/plugins/acg.sh`
**Function:** `acg_get_credentials`

### Old code (lines 211–214)

```bash
  _ensure_antigravity
  _antigravity_launch
  _antigravity_ensure_acg_session

  _info "[acg] Extracting AWS credentials from ${sandbox_url}..."
```

### New code

```bash
  if ! curl -sf http://localhost:9222/json >/dev/null 2>&1; then
    _info "[acg] Chrome CDP not available on port 9222 — launching Chrome..."
    _antigravity_launch
    _antigravity_browser_ready 30
  fi

  _info "[acg] Extracting AWS credentials from ${sandbox_url}..."
```

### Help text update (lines 178–192)

Replace references to "Antigravity browser automation" with "Chrome CDP automation":

#### Old

```bash
  cat <<HELP
Usage: acg_get_credentials [sandbox-url]

Extract AWS credentials from the Pluralsight Cloud Sandbox "Cloud Access" panel
via Antigravity browser automation and write to ~/.aws/credentials.

If sandbox-url is omitted, navigates to the sandbox listing page, starts a new
sandbox, waits for it to be ready, then extracts credentials.

Falls back to: pbpaste | acg_import_credentials

Arguments:
  sandbox-url   (optional) URL of a running sandbox, e.g.
                https://app.pluralsight.com/cloud-playground/cloud-sandboxes/<id>
                Defaults to ACG_SANDBOX_LIST_URL (listing page — auto-start flow)
HELP
```

#### New

```bash
  cat <<HELP
Usage: acg_get_credentials [sandbox-url]

Extract AWS credentials from the Pluralsight Cloud Sandbox "Cloud Access" panel
via Chrome CDP (Playwright) and write to ~/.aws/credentials.

Requires Chrome running with --remote-debugging-port=9222. If Chrome is not
already running, it will be launched automatically.

Falls back to: pbpaste | acg_import_credentials

Arguments:
  sandbox-url   (optional) URL of a running sandbox, e.g.
                https://app.pluralsight.com/hands-on/playground/cloud-sandboxes/<id>
                Defaults to ACG_SANDBOX_LIST_URL (listing page — auto-start flow)
HELP
```
