# Bugfix: v1.0.4 — Fix `acg_extend.js` extend button selectors

## Problem

`acg_extend.js` fails with "Extend button not found or not visible after multiple attempts".
Pluralsight's UI has changed since the selectors were last updated. None of the 12 selectors
in the `extendSelectors` array match any visible button on the current page.

**Issue:** `docs/issues/2026-04-06-acg-extend-selectors-stale.md`

## Step 1 — Diagnose: dump all buttons on the sandbox page

Run this one-off script (do NOT commit it):

```javascript
// scripts/playwright/debug_buttons.js — run once, do not commit
const { chromium } = require('playwright');
const os = require('os');
const path = require('path');

const AUTH_DIR = path.join(os.homedir(), '.local', 'share', 'k3d-manager', 'playwright-auth');
const URL = process.argv[2] || 'https://app.pluralsight.com/cloud-playground/cloud-sandboxes';

(async () => {
  const ctx = await chromium.launchPersistentContext(AUTH_DIR, {
    headless: false, channel: 'chrome', args: ['--password-store=basic'],
  });
  const page = ctx.pages()[0] || await ctx.newPage();
  await page.goto(URL, { waitUntil: 'domcontentloaded', timeout: 60000 });
  await page.waitForTimeout(5000);

  const buttons = await page.$$eval('button, [role="button"], a[class*="button"]', els =>
    els.map(el => ({
      text: el.innerText?.trim().slice(0, 80),
      ariaLabel: el.getAttribute('aria-label'),
      dataTestId: el.getAttribute('data-testid'),
      dataHeapId: el.getAttribute('data-heap-id'),
      className: el.className?.slice(0, 60),
      visible: el.offsetParent !== null,
    }))
  );
  console.log(JSON.stringify(buttons, null, 2));
  await ctx.close();
})();
```

Run: `node scripts/playwright/debug_buttons.js "https://app.pluralsight.com/cloud-playground/cloud-sandboxes" 2>/dev/null`

Look for entries with text or attributes containing: "extend", "renew", "+4", "add time", "Extend Session", "Extend Lab", or similar.
