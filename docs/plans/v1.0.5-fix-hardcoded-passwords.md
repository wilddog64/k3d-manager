# Task: Replace Hardcoded Passwords with CHANGE_ME Placeholders

**Branch (k3d-manager):** `k3d-manager-v1.0.5`
**Branch (all work repos):** `fix/k8s-base-secret-placeholder`
**Repos touched:** `k3d-manager`, `shopping-cart-infra`, `shopping-cart-order`, `shopping-cart-product-catalog`

---

## Context

Five files across three shopping-cart repos commit literal `changeme456`, `changeme123`,
`changeme`, and `guest` password values into git. This violates the security rule in CLAUDE.md:

> "Test credentials (`alice/password`, etc.) are dev-only — never reference in production config paths."

The `shopping-cart-payment` repo already follows the correct pattern (`CHANGE_ME` placeholder).
This task makes the other repos consistent with that pattern.

For the infra postgres admin secrets (`postgres-*-admin`), `bin/acg-up` already generates
random passwords (`_pg_pass_orders`, `_pg_pass_products`, `_pg_pass_payment`) and seeds them
into Vault KV. This task adds kubectl applies for the admin Secrets on the remote cluster
using those same generated passwords — so the StatefulSets continue to work. The CHANGE_ME
in git is overwritten at provision time before ArgoCD applies the StatefulSet.

## Change 1 — `k3d-manager/bin/acg-up`

After the Vault KV seeding block (currently ends with
`_info "[acg-up] Vault KV seeded (redis, postgres, payment, rabbitmq sandbox secrets)"`),
add the following block. Insert it BEFORE the `_info "[acg-up] Step 10/12 — Registering app cluster..."` line.

**Add after:**
```bash
_info "[acg-up] Vault KV seeded (redis, postgres, payment, rabbitmq sandbox secrets)"
```

**Insert:**
```bash
_info "[acg-up] Applying postgres admin + LDAP secrets on remote cluster..."
_ldap_admin_pass=$(openssl rand -base64 24 | tr -d '=+/')
_ldap_readonly_pass=$(openssl rand -base64 24 | tr -d '=+/')

kubectl create namespace shopping-cart-data --context ubuntu-k3s \
  --dry-run=client -o yaml | kubectl apply --context ubuntu-k3s -f - >/dev/null

kubectl apply --context ubuntu-k3s -f - <<EOF
apiVersion: v1
kind: Secret
metadata:
  name: postgres-orders-admin
  namespace: shopping-cart-data
type: Opaque
stringData:
  password: "${_pg_pass_orders}"
---
apiVersion: v1
kind: Secret
metadata:
  name: postgres-products-admin
  namespace: shopping-cart-data
type: Opaque
stringData:
  password: "${_pg_pass_products}"
---
apiVersion: v1
kind: Secret
metadata:
  name: postgres-payment-admin
  namespace: shopping-cart-data
type: Opaque
stringData:
  password: "${_pg_pass_payment}"
EOF
_info "[acg-up] postgres admin secrets applied on ubuntu-k3s"

kubectl create namespace identity --context ubuntu-k3s \
  --dry-run=client -o yaml | kubectl apply --context ubuntu-k3s -f - >/dev/null

kubectl apply --context ubuntu-k3s -f - <<EOF
apiVersion: v1
kind: Secret
metadata:
  name: ldap-secrets
  namespace: identity
type: Opaque
stringData:
  LDAP_ADMIN_PASSWORD: "${_ldap_admin_pass}"
  LDAP_READONLY_USER_PASSWORD: "${_ldap_readonly_pass}"
EOF
_info "[acg-up] ldap-secrets applied on ubuntu-k3s"
```

**Note:** `_pg_pass_orders`, `_pg_pass_products`, and `_pg_pass_payment` are already declared
earlier in the same function by the Vault KV seeding block. Do NOT re-declare them — reuse the
existing variables.
