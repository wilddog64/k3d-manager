# Task: Move Postgres Admin + LDAP Secrets to Vault → ExternalSecret

**Branch (k3d-manager):** `k3d-manager-v1.0.5`
**Branch (all work repos):** `fix/k8s-base-secret-placeholder`
**Repos touched:** `k3d-manager`, `shopping-cart-infra`, `shopping-cart-order`, `shopping-cart-product-catalog`

## Context

Static `kind: Secret` manifests with hardcoded passwords in git are a security violation.
`shopping-cart-infra` already uses Vault → ExternalSecret for all app-level credentials.
This task extends that pattern to the postgres admin Secrets and the LDAP secret.

**Current state (after first Codex pass):**
- `shopping-cart-infra` branch `fix/k8s-base-secret-placeholder` (`ed0cf82`) already replaced
  literal passwords with `CHANGE_ME` in `postgresql/orders/secret.yaml`,
  `postgresql/products/secret.yaml`, and `identity/ldap/deployment.yaml`.
  Those files are the starting point for this task — they will be fully replaced.

**Design decisions:**
- Postgres admin ExternalSecrets reuse the **existing** `secret/data/postgres/<instance>` Vault
  paths — no new Vault paths needed. The StatefulSet `POSTGRES_PASSWORD` and the app `DB_PASSWORD`
  will be the same credential, which is correct for this sandbox topology (single `postgres` user).
- LDAP gets a new Vault path `secret/data/ldap/admin` with `admin_password` + `readonly_password`.
  The `identity/ldap/deployment.yaml` embedded `kind: Secret` is removed; a standalone
  `ldap-secrets-externalsecret.yaml` replaces it.
- App repo `k8s/base/secret.yaml` files (order, product-catalog) remain `CHANGE_ME` placeholders —
  they are dead code in production because ExternalSecrets in shopping-cart-infra already create
  `order-service-secrets` and `product-catalog-secrets` with Vault values.

## Change 1 — `k3d-manager/bin/acg-up`

In the Vault KV seeding block (ends with
`_info "[acg-up] Vault KV seeded (redis, postgres, payment, rabbitmq sandbox secrets)"`),
add LDAP generation and seeding immediately **before** that `_info` line:

**Add before:**
```bash
_info "[acg-up] Vault KV seeded (redis, postgres, payment, rabbitmq sandbox secrets)"
```

**Insert:**
```bash
_ldap_admin_pass=$(openssl rand -base64 24 | tr -d '=+/')
_ldap_readonly_pass=$(openssl rand -base64 24 | tr -d '=+/')
_vault_kv_put "{\"admin_password\":\"${_ldap_admin_pass}\",\"readonly_password\":\"${_ldap_readonly_pass}\"}" ldap/admin
```

**Then update** the `_info` line immediately after to include ldap:

**Old:**
```bash
_info "[acg-up] Vault KV seeded (redis, postgres, payment, rabbitmq sandbox secrets)"
```

**New:**
```bash
_info "[acg-up] Vault KV seeded (redis, postgres, payment, rabbitmq, ldap sandbox secrets)"
```

**Note:** Do NOT add a kubectl apply block for postgres admin or LDAP — ESO will handle those
from Vault via the ExternalSecrets created in shopping-cart-infra.

## Change 2 — `shopping-cart-infra/data-layer/postgresql/orders/secret.yaml`

Replace the entire file content with:

```yaml
---
# ExternalSecret: postgres-orders-admin
# Syncs the postgres superuser password from Vault KV.
# Reuses the same secret/data/postgres/orders path as the app-level ExternalSecret
# so the StatefulSet POSTGRES_PASSWORD and the app DB_PASSWORD are always in sync.
apiVersion: external-secrets.io/v1
kind: ExternalSecret
metadata:
  name: postgres-orders-admin
  namespace: shopping-cart-data
  annotations:
    argocd.argoproj.io/sync-wave: "0"
  labels:
    app.kubernetes.io/name: external-secret
    app.kubernetes.io/instance: postgres-orders-admin
    app.kubernetes.io/component: database-credentials
    app.kubernetes.io/part-of: shopping-cart
spec:
  refreshInterval: 24h
  secretStoreRef:
    name: vault-backend
    kind: ClusterSecretStore
  target:
    name: postgres-orders-admin
    creationPolicy: Owner
  data:
    - secretKey: password
      remoteRef:
        key: secret/data/postgres/orders
        property: password
```

## Change 6 — NEW `shopping-cart-infra/identity/ldap/ldap-secrets-externalsecret.yaml`

Create this new file:

```yaml
---
# ExternalSecret: ldap-secrets
# Syncs LDAP admin and readonly passwords from Vault KV (secret/data/ldap/admin).
# Seeded by bin/acg-up at provision time.
# The ldap Deployment mounts this Secret via envFrom secretRef: ldap-secrets.
apiVersion: external-secrets.io/v1
kind: ExternalSecret
metadata:
  name: ldap-secrets
  namespace: identity
  annotations:
    argocd.argoproj.io/sync-wave: "0"
  labels:
    app.kubernetes.io/name: external-secret
    app.kubernetes.io/instance: ldap-secrets
    app.kubernetes.io/component: directory-credentials
    app.kubernetes.io/part-of: shopping-cart
spec:
  refreshInterval: 24h
  secretStoreRef:
    name: vault-backend
    kind: ClusterSecretStore
  target:
    name: ldap-secrets
    creationPolicy: Owner
  data:
    - secretKey: LDAP_ADMIN_PASSWORD
      remoteRef:
        key: secret/data/ldap/admin
        property: admin_password
    - secretKey: LDAP_READONLY_USER_PASSWORD
      remoteRef:
        key: secret/data/ldap/admin
        property: readonly_password
```
