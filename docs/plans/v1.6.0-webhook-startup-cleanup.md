# v1.6.0 — Webhook startup: clear stale `running` jobs

**Branch:** `k3d-manager-v1.6.0`
**File:** `bin/k3dm-webhook`

---

## Problem

When the webhook process is killed or restarts while a `make up/down` job is in flight,
the job's status file stays at `running` forever. The concurrent job guard in
`_running_cluster_job` will then block every subsequent `/acg-up` or `/acg-down` from
Slack until someone manually overwrites the status file.

The `_MAX_JOB_AGE_SECS = 10800` (3 hour) TTL in `_running_cluster_job` is the only
existing protection — but it only fires when a new request comes in. If the stale job is
less than 3 hours old (common after a crash or manual kill), every provision attempt is
silently rejected.

**Root cause:** `if __name__ == "__main__"` startup does not sweep `JOB_DIR` for orphaned
`running` jobs before serving requests.

**Goal:** on startup, mark every `running` job in `JOB_DIR` as `failed`. They are
orphaned by definition — if this webhook instance just started, it has no live subprocess
for any previously-running job.

### Change 1 — Add `_clear_stale_jobs()` call in the startup block

**Exact old block (lines 818–822):**
```python
if __name__ == "__main__":
    JOB_DIR.mkdir(parents=True, exist_ok=True)
    server = http.server.ThreadingHTTPServer(("127.0.0.1", PORT), _Handler)
    print(f"[k3dm-webhook] listening on 127.0.0.1:{PORT}")
    server.serve_forever()
```

**Exact new block:**
```python
if __name__ == "__main__":
    JOB_DIR.mkdir(parents=True, exist_ok=True)
    _clear_stale_jobs()
    server = http.server.ThreadingHTTPServer(("127.0.0.1", PORT), _Handler)
    print(f"[k3dm-webhook] listening on 127.0.0.1:{PORT}")
    server.serve_forever()
```

---

### Change 2 — Add `_clear_stale_jobs` function before `_running_cluster_job`

Insert this function immediately before the line
`_MAX_JOB_AGE_SECS = 10800  # 3 hours`:

**Exact insertion point — before this line:**
```python
_MAX_JOB_AGE_SECS = 10800  # 3 hours — any older "running" job is stale
```

**Insert this block immediately before it:**
```python
def _clear_stale_jobs():
    """On startup, mark any job stuck in 'running' as 'failed' — they are orphaned."""
    if not JOB_DIR.exists():
        return
    for entry in JOB_DIR.iterdir():
        status_file = entry / "status"
        try:
            if status_file.exists() and status_file.read_text().strip() == "running":
                status_file.write_text("failed")
                print(f"[k3dm-webhook] cleared stale job {entry.name}")
        except Exception:
            pass


_MAX_JOB_AGE_SECS = 10800  # 3 hours — any older "running" job is stale
```

---

## Behaviour

```
webhook starts
  → JOB_DIR created if missing
  → _clear_stale_jobs() scans all job dirs
  → any job with status == "running" → status = "failed"
  → prints "[k3dm-webhook] cleared stale job <id>" for each one
  → server starts normally

Next /acg-up from Slack:
  → _running_cluster_job() finds no running jobs → proceeds normally
```

If no stale jobs exist, `_clear_stale_jobs` is a no-op (iterates the dir, finds nothing
to fix, returns silently).
