## Problem

`bin/acg-up/down/refresh/status/sync-apps` are named for the ACG sandbox, but they are the
generic ephemeral cloud-sandbox provider implementation (`aws`/`gcp`/`az` — the `*` arm of the
`make` targets). The permanent providers (`k3s-hostinger`, `k3s-oci`) are deliberately routed
*away* from them. The `make` interface (`up`/`down`/`refresh`/`status`) is already provider-neutral
and stays the user contract; only the underlying binary names are misleading.

**Decision (user, 2026-06-14):** rename the 5 binaries to the `cluster-` prefix (aligns with the
webhook's existing `_run_cluster_*` / `/api/v1/cluster*` naming). Functional-only scope. Keep the
`make` targets and the Slack `/acg-*` command names unchanged.
