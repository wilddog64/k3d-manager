# Retrospective — v1.8.0

**Date:** 2026-06-26
**Milestone:** v1.8.0 — lib-acg absorption Phase 2 + agy retarget + Hostinger hardening
**PR:** #99 — merged to main (`dd54b653`)
**Participants:** Claude, Codex, Gemini, Copilot

## What Went Well

- **lib-acg absorption Phase 2 landed clean.** The standalone `scripts/lib/acg/` subtree (−33,655 lines, 224 files) was dropped and the code now lives solely in the lib-foundation subtree (synced to v0.4.0). Stubs (`acg.sh`, `gcp.sh`, `gemini.sh`) and `bin/cluster-{up,refresh}` repointed to `scripts/lib/foundation/scripts/lib/acg/`; the `lib-acg` git remote was removed. The 3-level subtree chain (foundation→lib-acg→k3d-manager) collapsed to one.
- **agy retarget was a surgical backend swap.** Google retired `@google/gemini-cli`; `gemini.sh` browser automation now drives the Go-based Antigravity CLI (`agy --dangerously-skip-permissions`) and `_ensure_gemini` provisions `agy`. Public `gemini_*` names were retained, so no caller churn.
- **Combining the absorption rewire + agy retarget into one Codex spec** (`docs/plans/v1.8.0-acg-absorb-phase2-agy-rewire.md`) kept the change atomic and avoided a half-migrated intermediate state.
- **Local validation held the line.** Codex's work passed `shellcheck -S warning`, `bash -n`, the antigravity/provider-contract/gcp BATS suites, `./scripts/k3d-manager test all` (1..388), and `_agent_audit` (exit 0) before the PR — so the unreviewable diff was already independently verified.
- **Hostinger hardening consolidated.** ~11 fixes (ghcr-pull-secret reconcile, CA-verified TLS + argocd-manager SA, provider-aware refresh/status, provider-state precedence, vcluster deregister, observability namespace race) shipped together with the ESO app-cluster ApplicationSet generator.

## What Went Wrong

- **Copilot could not review the PR.** The diff exceeded Copilot's 20,000-line limit because of mechanical subtree churn (the drop + the foundation sync). Resolution was to prove (via `git diff --shortstat` on subtree-excluded paths) that the overflow was 100% generated subtree movement, leaving ~49 hand-written files already reviewed line-by-line and locally verified — then get explicit user sign-off to proceed. A subtree-drop PR is structurally unreviewable by Copilot; splitting does not help because the drop is itself the oversized diff.
- **CHANGELOG was incomplete at PR-creation time.** `[Unreleased]` covered only 3 of ~25 branch changes; it had to be expanded to full v1.8.0 scope (`d5fcd471`) as a PR-gate blocker. The branch had accumulated changes since v1.7.0 without incremental CHANGELOG upkeep.
- **README releases table + `docs/releases.md` drift.** Both were stale — they still listed v1.6.5/6.4/6.3 as "most recent" and never gained v1.7.0 or v1.7.1 rows. Fixed this release (added v1.8.0/v1.7.1/v1.7.0, demoted the v1.6.x rows). The convention (update on the NEXT branch after shipping) had not been followed for two prior releases.
- **v1.7.1 never received a CHANGELOG section** — it was tagged and released (PR #96) but the CHANGELOG jumps from `[1.7.0]` straight to `[1.6.5]`. Still-open drift (not back-filled here to avoid scope creep).
- **Two upstream-first follow-ups remain.** `acg-credential-test:277` calls undefined `_sts_valid` (pre-existing, in the absorbed module — fix on lib-foundation); the drop commit added a now-dead `scripts/lib/acg/*` skip glob to top-level `scripts/lib/agent_rigor.sh` (reconcile/remove via lib-foundation).

## Decisions Made

- **agy retarget keeps `gemini_*` public function names** — this release is a backend-only swap; a public rename is deferred so callers and docs are untouched.
- **Proceed-to-merge despite Copilot size-decline is acceptable only for mechanical subtree churn** — and only after proving the overflow is generated movement and the hand-written delta was reviewed + locally verified. Recorded as the precedent for future absorption/sync PRs.
- **lib-acg absorption Phase 3 = archive the standalone `lib-acg` GitHub repo** — deferred (gravestone v0.1.9 already tagged; history preserved in the archived repo).
- **Next milestone (v1.9.0) = app-cluster Vault-auth portability Phase 3** — factor `configure_vault_app_auth` out of the Hostinger-pinned `_hostinger_register_cluster` into one kubecontext-keyed helper callable from every provider (AWS/Azure/OCI/EKS/AKS/home-lab). See `project_app_cluster_vault_auth_portability`.

## Theme

v1.8.0 is the release where the ACG automation stopped being a guest and became a resident: the standalone lib-acg repo, after a clean gravestone release, was fully absorbed into lib-foundation, and k3d-manager's two-subtree limbo collapsed to one. Riding along were a forced-by-Google backend swap (gemini-cli → Antigravity) and a batch of Hostinger hardening fixes that had piled up since v1.7.0. The hard lesson was process, not code: a subtree-drop PR is too large for Copilot to read, so trust had to come from local rigor (shellcheck, BATS, `_agent_audit`, line-by-line review) and an explicit human decision — and the CHANGELOG and releases tables, neglected across two releases, had to be reconstructed at the finish line instead of maintained along the way.
