**Date:** 2026-03-24
**Milestone:** if-count elimination — `_run_command` + `_ensure_node` in system.sh
**PR:** #50 — merged to main (`d317429b`)
**Participants:** Claude, Codex, Copilot, Copilot Coding Agent

## What Went Well

- if-count allowlist is now fully empty — both `_run_command` and `_ensure_node` drop from 9→7 ifs via clean helper extraction
- Subtree discipline enforced correctly: edits went to lib-foundation first (`b9fcbf6`), then subtree-pulled; pre-commit hook blocked a mistaken direct edit to `scripts/lib/foundation/`
- lib-foundation PR #13 and k3d-manager PR #50 sequenced correctly after user caught the ordering issue — cancel k3d-manager PR, wait for lib-foundation to merge, then re-create
- Copilot coding agent (new GitHub feature) produced a full-repo design review (7.8/10) at no quota cost — useful alternative when PR reviewer quota is exhausted
- Copilot findings on both PRs were minor (indentation, missing header comment, missing BATS status assertion) — no functional issues

## What Went Wrong

- k3d-manager PR #49 was created before lib-foundation PR #13 merged — had to close and re-create as PR #50; adds noise to PR history
- Attempted direct edit to `scripts/lib/foundation/scripts/tests/lib/system.bats` inside k3d-manager — blocked by pre-commit hook (correct), but the attempt itself was a process gap
- Copilot quiet-mode BATS test lacked a status assertion (`[ "$status" -eq 1 ]`) — should have been caught in the original spec review before writing the test

## Process Rules Added

| Rule | Context |
|---|---|
| lib-foundation PR must be merged to main **before** k3d-manager PR is created | Subtree discipline: k3d-manager consumer pulls from lib-foundation main, not feature branches |
| BATS tests for soft/quiet modes must assert **both** output content AND exit status | One assertion per observable dimension; output-only tests are incomplete |
| Direct edits inside `scripts/lib/foundation/` are blocked by pre-commit hook — fix upstream, subtree pull | The hook is working correctly; the rule is now explicit in the retro |

## Decisions Made

- **Copilot coding agent** — full-repo design review via GitHub's `/agents` UI; backed by GitHub Projects API (requires `read:project` scope); no public REST API yet — web UI only. Identified 9 findings (P2 + nits) for future milestones.
- **CodeRabbit** — evaluated as quota-gap alternative; free tier only on public repos, paid for private; not installed. Copilot coding agent fills the gap better for design-level review.
- **PR ordering discipline** — when a change spans lib-foundation + k3d-manager, always create lib-foundation PR first, merge it, then create k3d-manager PR. Never create both simultaneously.
