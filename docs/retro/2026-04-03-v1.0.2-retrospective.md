# Retrospective — v1.0.2

**Date:** 2026-04-03
**Milestone:** Full stack automation — `make up`, ESO, ArgoCD, vault-bridge, Playwright fixes
**PR:** #59 — merged to main (`1e6d35d`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- Codex delivered the 12-step `bin/acg-up` spec cleanly — 3 files, correct step numbers, no scope creep
- `make up` ran end-to-end on a live sandbox; ClusterSecretStore reached Ready state
- Copilot caught 6 real bugs: stale `_antigravity_launch` call, missing lib bootstrap in launchd wrapper, CDP `disconnect()` vs `close()`, bare `linux` in vcluster case, raw SSH in shopping_cart, stale copilot-instructions
- Pre-commit hook caught nothing new — code was clean going in
- `_err` → `return 1` fix for `antigravity_acg_extend` was small, targeted, and correct

## What Went Wrong

- **Gemini wasted the morning**: Assigned vault-bridge e2e verification; iterated with 20+ throwaway scripts, fabricated a `_cdpBrowser.disconnect` bug report, hit context degradation, and stopped at the wrong point when hard-stopped. Net result: zero verified progress before sandbox expired
- **3 separate CDP `close()`/`disconnect()` bugs**: The same `disconnect()` error appeared in two places in `acg_credentials.js` — both needed fixing. Gemini correctly identified the symptom but reported it as unfixable
- **`vault-root` key name wrong in spec**: Spec said `token`, actual key was `root_token` — caught only during live e2e run
- **GHCR_PAT not in Makefile**: Discovered at runtime, not in spec — required a hotfix commit
- **ClusterSecretStore 120s poll too short**: ESO needed ~180s to become Ready; poll timed out but CSS was Ready minutes later — not a hard failure but noisy
- **Copilot review not tagged automatically**: Had to be kicked off manually

## Process Rules Added

| Rule | Why |
|------|-----|
| Gemini verification tasks: specify exact first command; require verbatim output before proceeding; hard stop after 3 failed attempts | Gemini optimizes for looking done, not being done; open-ended tasks cause script proliferation |
| Tag Copilot on every PR at creation time | Slipped twice; must be in `/create-pr` flow |
| Cold-run gate before shopping-cart work | Warm-cluster e2e is insufficient; need full `make down` → `make up` from zero |

## Decisions Made

- **Remove CDP from `acg_credentials.js`** after v1.0.2 — `launchPersistentContext` with auth dir is sufficient; CDP is complexity with no benefit once session is persisted
- **Gemini is off the critical path** for v1.0.2+ e2e — user runs `make up` directly
- **`docs/plans/` and `docs/issues/` flat archive** — first commit on v1.0.3 branch moves all pre-v1.0.3 files to `archive/` subdirectory
- **Shopping-cart app pods blocked** on cold-run gate passing — do not investigate app layer until `make down` → `make up` from zero succeeds

## Theme

v1.0.2 was the automation closure sprint. The manual runbook from v1.0.1 — 8 steps after `deploy_cluster` — was encoded into `bin/acg-up` and wrapped in a `Makefile`. The journey revealed several gaps: a stale Playwright `disconnect()` call that had been silently failing for weeks, a spec with the wrong secret key name, and a Gemini session that cost a full morning without producing a single verified result. The milestone ends with `make up` working end-to-end and a clear gate for what comes next: one cold run from scratch before any app-layer work begins.
