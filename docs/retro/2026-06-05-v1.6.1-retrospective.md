# Retrospective — v1.6.1

**Date:** 2026-06-05
**Milestone:** ACG refresh self-healing + observability fixes
**PR:** #90 — merged to main (`d2a8615e282b9103a20649dac9a49c64886299fe`)
**Participants:** Claude, Codex, Copilot

## What Went Well
- `_system_daemon_install_if_missing` helper pattern is clean and reusable — same plist-regeneration logic works for all three system daemons
- Codex implemented all 5 bugfix specs cleanly with no regressions
- Pre-auth sudo guard (`sudo -n true`) is a nice small fix with high daily-driver impact
- Copilot caught `which` vs `command -v` — valid portability finding, fixed quickly
- ACG Prometheus datasource (`acg-prometheus`) was already wired in hub Grafana via `kube-prometheus-stack-values.yaml`; no new code needed to expose shopping-cart metrics

## What Went Wrong
- Three separate `_launchd_ensure` calls silently no-op when plist is missing — root cause was in `_launchd_ensure` design, but fix was applied at call sites (correct: least invasive)
- System daemon plists live in `/Library/LaunchDaemons/` (requires sudo) while user agents are in `~/Library/LaunchAgents/` — two separate self-heal code paths needed; could be unified in a future refactor
- CHANGELOG was `[Unreleased]` only at PR time — versioned entry not added; note for next release

## Process Rules Added
| Rule | Location |
|------|----------|
| Use `command -v` not `which` for binary path resolution | Copilot PR #90 finding |

## Decisions Made
- `_system_daemon_install_if_missing` copies plist via `_run_command --prefer-sudo` (not `--require-sudo`) so a missing sudo cache produces a warning, not a hard failure
- Stack vulnerability scan pipeline (`v1.6.1-stack-vuln-report.md`) deferred to v1.6.2 — it requires OCI CronJob infrastructure not yet proven stable
- frontend wrapper path checked at both `${_ACG_STATE_DIR}/bin/` and `${_ACG_STATE_DIR}/` root — covers both current and legacy `acg-up` output paths

## Theme
v1.6.1 was a resilience sprint: every fix makes `make refresh` more self-healing after an interrupted `acg-up`. The pattern — check if plist exists, regenerate from wrapper if missing, then call `_launchd_ensure` — is now applied to all four managed LaunchAgents (Vault, Keycloak port-forward, and the three system daemons). The cluster can now recover from a partial provision without any manual intervention.
