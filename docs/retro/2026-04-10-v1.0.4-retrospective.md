# Retrospective — v1.0.4

**Date:** 2026-04-10
**Milestone:** ACG extend hardening — button-first search, random passwords, sandbox-expired guidance
**PR:** #61 — merged to main (`bc9028fb`)
**Participants:** Claude, Codex, Gemini, Copilot

## What Went Well

- Copilot caught 4 real security and correctness bugs before merge — CodeQL URL check, credential leak, CDP browser close, ghost-state guard; all fixed in `4f7f273d`
- "Button-first" search eliminated the unnecessary TTL parse round-trip for the common case where the extend modal is already open
- CDP browser close fix (`finally` only closes context if we launched it; attached sessions left running) prevented accidental Chrome process termination
- `openssl rand -base64 24 | tr -d '=+/'` random password generation in `bin/acg-up` eliminated hardcoded sandbox credentials
- Sandbox-expired guidance in `_acg_check_credentials` gives clear next steps (start sandbox → `acg_get_credentials` → `make up`) instead of a generic `InvalidClientTokenId` error
- Midnight date-wrap fix prevented `remainingMins` from going negative when sandbox expires after midnight

## What Went Wrong

- Claude edited `rabbitmq-client-java/CHANGELOG.md` directly instead of writing a spec for Codex — scope violation; required force-push revert on two branches; root cause was context collapse during a long session
- `_antigravity_browser_ready` naming in issue doc `2026-04-06-acg-antigravity-false-dependency.md` was stale (function renamed to `n()` in lib-foundation v0.3.12); doc flagged for correction on v1.0.5

## Decisions Made

- **Ghost State recovery guard tightened** — `remainingMins !== null && remainingMins < 15` (not `null`) so TTL parse failure alone never triggers destructive delete→start→extend; only confirmed critical windows do
- **CDP browser close scoped to launched contexts** — `browser.close()` shuts down entire Chrome process when attached via CDP; correct pattern is `browserContext.close()` only when `!_cdpBrowser`
- **antigravity.sh refactor deferred to v1.0.5** — removing `acg.sh`'s `source antigravity.sh` dependency is architecture cleanup, not a bug; does not affect v1.0.4 correctness

## Theme

v1.0.4 was the "harden the extend loop" release. The Playwright automation that extends sandbox TTL had four latent issues exposed in a single Copilot review: a CodeQL URL substring check that could be bypassed, a `parentElement.innerText` log that could leak AWS credentials, a `browser.close()` call that nuked the user's entire Chrome session on CDP-attached runs, and a ghost-state guard that could trigger destructive sandbox deletion on a transient DOM flicker. All four were fixed before merge. On top of that, `bin/acg-up` now generates random passwords per sandbox instead of using hardcoded values, and expired-sandbox guidance gives operators a clear recovery path. Small scope, high security delta.
