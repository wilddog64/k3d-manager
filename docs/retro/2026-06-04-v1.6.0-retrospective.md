# Retrospective — v1.6.0

**Date:** 2026-06-04  
**Milestone:** Webhook Enhancement — Slack threads, AI triage, token rotation  
**PR:** #89 — merged to main (`eeee5c46`)  
**Participants:** Claude, Codex, Copilot, Gemini  

---

## What Went Well

- **Webhook foundation solid** — k3dm-webhook server (`bin/k3dm-webhook`, `bin/k3dm-webhook-setup`, plist template) provides stable HTTP interface for remote orchestration. Bearer token auth + concurrent job guard + stall detection passed 391/391 BATS tests.
- **Slack threading unified job visibility** — all job notifications (start, update, stall alert, failure analysis, cleanup, health check) now group into a single thread via `_notify_job()`. Thread-aware slash commands (`/kill`, `/diagnosis`, `/status`, `/logs`) reply directly in the job thread without polluting main channel.
- **AI triage at scale** — switched from Claude API (paid, slow) to Gemini CLI free tier (`gemini --model gemini-2.5-flash`). Failure analysis now runs asynchronously in daemon threads, enriched with live pod/app state and node reachability checks. Visual diagnosis via Claude Haiku vision API on Playwright failure screenshots.
- **Token rotation automation** — `bin/rotate-webhook-token` + LaunchAgent (StartInterval=21600) keeps the Slack relay Worker secret in sync without manual intervention. Webhook now verifies new token before committing to Keychain.
- **Cloudflare Worker relay** — slash command Request URLs fixed to point at `https://k3dm-slack-relay.k3dm.workers.dev` (Cloudflare Worker with auth) instead of direct tunnel endpoint. Eliminates 401 auth failures and prevents bypass of HMAC verification.
- **Python 3.13 SIGSEGV fix** — macOS 26.5.1 Beta Python 3.9 fork bug (SIGSEGV in multiprocessing) resolved by bumping webhook plist to `/opt/homebrew/bin/python3` (Python 3.13).
- **13 Copilot findings addressed** — Copilot review of PR #89 caught 13 valid findings (CodeQL security, shell injection patterns, incomplete app lists); all fixed in follow-up commit `9b34d546`.

## What Went Wrong

- **Python 3.9 SIGSEGV not caught early** — fork bug on macOS 26.5.1 Beta only appears when webhook restarts and imports multiprocessing. Root cause took several live iterations to isolate. Lesson: always pin Python version in plist template when possible; don't rely on system default or Homebrew `python3` symlink.
- **Slack slash command URL misconfiguration** — slash command Request URLs initially pointed at `webhook.3ai-talk.org/slack/command` (direct tunnel endpoint, bypassing Worker auth). Error surfaced in live Slack thread after Worker deployment. Issue doc: `docs/issues/2026-06-04-slack-slash-commands-wrong-url.md`.
- **lib-acg subtree doc fix still pending** — Playwright sign-in click regression (overlay intercepting pointer events) merged via lib-acg PR #33. fix committed to lib-acg subtree (`scripts/lib/acg/docs/bugs/2026-06-04-acg-credentials-signin-click-outside-viewport.md`) but original lib-acg fix needs a separate PR on lib-acg main to avoid accumulating unreleased fixes in k3d-manager. Tracked as subtree discipline debt.
- **KUBECONFIG not persisted across webhook restarts** — webhook plist template was missing `KUBECONFIG` env var, so context switching was lost after restart. Fixed in commit `ec5ee97c` by adding the env var to plist.

## Process Rules Added

| Rule | File | Reason |
|------|------|--------|
| **Python version pinning in plist** | webhook plist template | SIGSEGV on fork() in Python 3.9 on macOS 26.5.1 Beta required explicit pin to 3.13 |
| **Env vars in plist for persistence** | webhook plist template | KUBECONFIG was missing, causing context loss after restart |
| **lib-acg subtree PR discipline** | Memory/lib-foundation rules | Playwright fixes committed to k3d-manager subtree must also land upstream PR before merging to lib-acg main |
| **Slack relay Worker URL validation** | Memory/webhook setup | Validate slash command Request URLs point to Cloudflare Worker endpoint, not direct tunnel |

## Decisions Made

1. **Gemini CLI over Claude API** — switched failure analysis AI from `claude-3-sonnet-20240229` (paid, ~500ms latency) to `gemini --model gemini-2.5-flash` (free, <500ms). Reduced cost to zero while maintaining quality. Fallback: silent skip if Gemini unavailable (preserves webhook availability).

2. **Webhook as persistent LaunchAgent** — webhook lifecycle is now outside ACG lifetime. Restart via launchctl handles stale jobs (clear on startup), token rotation (every 6h), and all job orchestration without needing the cluster up.

3. **Worker relay for slash commands** — slash commands route through Cloudflare Worker (`k3dm-slack-relay.k3dm.workers.dev`) for auth verification instead of direct tunnel endpoint. Allows HMAC check before webhook sees the request.

4. **Post-provision health check async** — health check daemon thread runs after successful `make up`, waits 90s for cluster to stabilize, then triages degraded apps. Runs in background so provisioning returns immediately.

## Theme

**Webhook Maturity** — v1.6.0 shifted k3d-manager from a local dev CLI to a distributed system where the webhook acts as a persistent, autonomous orchestrator. Slack becomes the control plane; the webhook is the execution plane. Token rotation, failure analysis, and health checks now run on the laptop even when ACG is down, opening the door to self-healing and multi-cluster management (v1.7.0+).
