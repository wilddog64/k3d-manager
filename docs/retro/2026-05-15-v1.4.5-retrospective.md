# Retrospective — v1.4.5

**Date:** 2026-05-15
**Milestone:** k3d-manager v1.4.5 — ACG extend navigation fix + frontend SSO wiring
**PR:** #74 — merged to main (`8f93df25`)
**Participants:** Claude, Codex, Copilot

## What Went Well
- Copilot caught 5 real issues across 4 files: URL matching precision, PGPASSWORD exposure in command args, ESO dependency guard, and vault revoke error propagation
- All Copilot threads replied and resolved in a single pass
- Bug spec process worked well: spec written, verified, then implemented cleanly
- Fix commit (`6e5a050d`) addressed all 5 findings in one commit with clear per-finding messages

## What Went Wrong
- acg_extend.js: initial fix used `includes()` on the full URL — should have used `new URL().pathname` from the start; caught by Copilot
- keycloak.sh: credential injection pattern in kubectl exec args introduced without pre-commit detection; agent audit hook now catches this family of errors
- shopping-cart-frontend: Codex accidentally pushed the functional fix directly to `main`, then the revert (`2872a4cd`) left main without the fix; PR #15 diff showed only the issue doc — required a new spec to re-apply the fix explicitly
- Commit messages on shopping-cart repos deviated from the spec's exact wording (cosmetic, but spec said exact)

## Process Rules Added
- Copilot PR scope comment: for milestone branches, always add a Notes section to the PR body clarifying it is a milestone branch and naming the single functional change being reviewed
- Credential injection patterns now caught by agent audit pre-commit hook — use heredoc pattern for sensitive values instead
- After any accidental push to main + revert: check that the PR diff still shows the functional files; if not, write a spec to merge-main-then-re-apply

## Decisions Made
- `isOnSandboxPage` check uses `new URL().pathname` (not full URL includes) — more precise and matches the original `isPluralsight` pattern it replaced
- Vault revoke failures now return non-zero — callers should handle or log them; cert-not-found is still 0 (benign)
- ESO plugin is a hard dependency for ldap.sh — missing file or undefined function after source is fatal (exit 1)

## Theme
v1.4.5 closed out the ACG extend navigation bug (browser on wrong Pluralsight page caused silent extend failure) and wired up frontend SSO across two repos. The milestone also surfaced a cluster of security and robustness findings via Copilot: password exposure in kubectl args, swallowed error codes, and a missing dependency guard. Most of the work happened in the post-commit review phase rather than the implementation phase — a sign that specs and verification are working, but that security review rules need earlier enforcement.
