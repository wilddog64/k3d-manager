# Retrospective — v1.5.3

**Date:** 2026-06-01
**Milestone:** Observability ACG stack + infra bug fixes
**PR:** #88 — merged to main (`6e6663027b`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- All four planned fixes landed cleanly: Vault CA trust (bin/acg-up), ClusterSecretStore retry logic, ArgoCD app registration and sync, ACG observability stack (Alertmanager + Grafana)
- Prometheus memory degraded-health caught and fixed same session: pod memory usage 531Mi exceeded 512Mi cap; raised to 1Gi to accommodate federate-acg scrape job cardinality
- Copilot PR #88 findings were all valid and addressed in follow-up commits: curl header injection risk in credential extraction, context scope leak in copy-output, credential exposure in logs, Grafana password exposure in Makefile
- Comprehensive security review cycle: spec → implementation → Copilot findings → fixes → merge clean
- Observability stack now production-ready on ACG: Alertmanager with email routing (Vault-backed SMTP), Grafana dashboards (NodePort 30030), pruned alert rules for k3s

## What Went Wrong

- v1.5.2 was released without a git tag or GitHub release — slipped through post-merge housekeeping after PR #87 merge
- Prometheus memory limit set too low from initial deployment (512Mi): should have been sized against actual federation load before merge

## Process Rules Added or Clarified

1. **Pre-release tag verification:** Before creating a new PR, verify that the previous release version has a git tag and GitHub release; if missing, create retroactively before proceeding
2. **Memory resource planning:** Cluster-wide Prometheus federation (federate-acg scrape) increases cardinality significantly; baseline allocation for ACG observability stack: 1Gi

## Decisions Made

- Keycloak redirect URI for public domains: exact `/callback` path, not wildcard `/*`
- Prometheus memory limit baseline for clusters running `federate-acg`: 1Gi minimum (cardinality increases ~2-3x with remote write config)
- v1.5.2 retroactive tag acceptable for docs-only releases without impact on v1.5.3 timeline

## Theme

v1.5.3 closed out the observability ACG stack with Alertmanager and Grafana on the management cluster, fixed four infrastructure reliability issues that were causing intermittent failures in `acg-up` runs, and included comprehensive security hardening of ACG credential automation. The Prometheus memory fix was unplanned but came from a live cluster health check and resolved a long-standing degraded state. Copilot's PR review caught real security gaps (curl header injection, credential leaks in logs), reinforcing the effectiveness of the pre-merge review gate. Release is stable and ready for production deployment of k3s-oci provider.
