# Retrospective — v0.9.16

**Date:** 2026-03-27
**Milestone:** Antigravity IDE + CDP browser automation (v0.9.15 + v0.9.16)
**PR:** #51 — merged to main (`484354da`)
**Participants:** Claude, Gemini, Copilot

---

## What Went Well

- **Gemini's CDP insight was correct** — connecting to Antigravity via `chromium.connectOverCDP('http://localhost:9222')` works cleanly; gemini CLI as the driver is the right architecture
- **Copilot caught real bugs** — 12 findings total; 7 fixed in-PR, 5 deferred cleanly to lib-foundation v0.3.14 with proper tracking
- **Findings doc discipline held** — `docs/issues/2026-03-26-copilot-pr51-review-findings.md` captures all 12 findings with fix/defer status; deferred items landed in lib-foundation `memory-bank/progress.md` before session ended
- **Binary detection pattern** — `_ag_bin` try-agy-then-antigravity approach is the right model for `_ensure_antigravity_ide` upstream fix
- **Staged-only audit stayed** — Copilot suggested reverting bare-sudo scan to unstaged; rejected correctly; BATS test was wrong, not the implementation

## What Went Wrong

- **Session persistence spec was wrong** — Gemini proposed `storageState` JSON injection (headless browser pattern); Antigravity is persistent so cookies are already there; 3 spec files deleted before PR
- **`_antigravity_browser_ready` deferred but almost patched directly** — started editing `scripts/lib/foundation/scripts/lib/system.sh` directly before catching the subtree discipline violation; reverted immediately
- **lib-foundation memory-bank not updated after deferred finding reply** — user had to prompt; should be automatic whenever Copilot findings are deferred to a subtree-managed file

| Rule | File | Trigger |

## Decisions Made

- **`storageState` approach rejected** — Antigravity is a persistent browser; session cookies are already present after first login; no file-based session storage needed
- **`_antigravity_ensure_acg_session` → v0.9.17** — simple CDP login wait, mirrors `_antigravity_ensure_github_session`; Gemini implements, Codex writes BATS
- **lib-foundation v0.3.14 before k3d-manager v0.9.17** — 5 Copilot findings must ship upstream first; subtree pull required before k3d-manager can consume fixes
- **Both v0.9.15 and v0.9.16 shipped in a single PR (#51)** — scopes were small enough; both CHANGE.md entries versioned correctly

## Theme

v0.9.15 and v0.9.16 complete the Antigravity browser automation stack: gemini CLI as the agent brain, Playwright MCP over CDP as the browser surface, Antigravity IDE as the persistent browser. The session covered a full cycle — Gemini's initial CDP design validated, one wrong approach (storageState) corrected early, Copilot review caught 12 real issues, 7 fixed in-PR, 5 deferred upstream with proper tracking. The deferred items created a clean dependency: lib-foundation v0.3.14 goes first, then k3d-manager v0.9.17 continues.
