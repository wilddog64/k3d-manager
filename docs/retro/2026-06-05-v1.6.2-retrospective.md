# Retrospective — v1.6.2

**Date:** 2026-06-05
**Milestone:** ChatOps /acg-refresh + reliability fixes
**PR:** #91 — merged to main (SHA `74b320c37cddfaff561d74dd96972713edb559b0`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- `/acg-refresh` delivered end-to-end: Codex wired the webhook + worker, Claude fixed both headless failures
- Keycloak retry fix (Codex) was clean — exact spec, no surprises
- Same root cause pattern (ubuntu-k3s context missing in webhook kubeconfig) caught twice in one session — first in bin/acg-refresh summary line, then in ESO smoke checks; both fixed before merge
- Copilot caught the gcp.js username logging issue; correctly categorized as lib-acg upstream debt, not a v1.6.2 regression
- Prometheus port-forward LaunchAgent and cleanup LaunchAgent additions were straightforward and clean

## What Went Wrong

- Both bin/acg-refresh headless failures required two separate debug cycles (sudo pre-auth found first, then the summary kubectl); the pattern of "webhook has no ubuntu-k3s context" should have been caught in spec review
- ESO false-positive was the same pattern again — could have been caught by searching for all `--context ubuntu-k3s` calls in k3dm-webhook before shipping
- GEMINI_CLI_TRUST_WORKSPACE removal (duplicate-messages fix Change 1) was reverted after breaking headless — env var is required for Gemini CLI non-interactive mode; the revert was not documented in the spec until found during post-merge review

## Process Rules Added

None explicitly added to CLAUDE.md this milestone, but a pattern emerged:
- Before any webhook PR: grep all `--context ubuntu-k3s` calls in k3dm-webhook and bin/* — each one must be non-fatal or skip-guarded

## Decisions Made

- `/resume` Slack command (checkpoint-based pipeline re-entry) deferred to v1.6.3 — requires checkpoint file design and RESUME_FROM env var support in make up
- lib-acg `gcp.js` username logging fix is upstream debt — fix at source, then subtree pull; do not patch subtree files directly

## Theme

v1.6.2 was the "close the ChatOps loop" sprint. The `/acg-refresh` command moved daily credential refresh from a terminal task to a Slack slash command — the last of the four ChatOps commands (acg-up, acg-down, acg-status, acg-refresh) to be wired. Two bugs blocked headless execution and both traced to the same root cause: the webhook daemon runs with a different HOME and has no ubuntu-k3s context. The Keycloak retry was a separate reliability win for slow-start provision. Net result: ACG day-2 operations are now fully Slack-driven.
