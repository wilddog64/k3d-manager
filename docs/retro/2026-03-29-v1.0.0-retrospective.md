# Retrospective — v1.0.0

**Date:** 2026-03-29
**Milestone:** k3s-aws provider foundation
**PR:** #57 — merged to main (`807c0432`)
**Participants:** Claude, Codex, Gemini, Copilot

## What Went Well

- **Multi-agent pipeline held up under pressure** — Claude wrote 5 specs + 1 bugfix spec, Codex implemented all commits cleanly, Gemini ran 3 e2e smoke test runs. Each agent did exactly its role.
- **Gemini found real bugs before release** — Run 2 failed on keypair import noise + extend 404 abort. Without live smoke testing these would have shipped as known-bad behavior. The 3-run protocol proved its value.
- **Copilot caught 5 meaningful issues** — CSV column parsing fragility, missing BATS tests, awk regex injection, acg_watch PID lifecycle, and `_write_sensitive_file` security pattern. All actionable, all fixed.
- **`/bugfix` skill formalized the naming convention** — `bugfix-` prefix now enforced for fix specs vs `hotfix-`; committed to `wilddog64/claude-settings`.
- **`acg_watch` lifecycle design** — PID file management + deploy guard + destroy cleanup is a clean pattern; Copilot's lifecycle comment pushed this to the right design before it caused problems in production.
- **`_write_sensitive_file` adoption** — Copilot forced consistent use of the secure file-write helper; now `aws.sh` is consistent with `vcluster.sh` and `system.sh`.

## What Went Wrong

- **5-spec rule exceeded** — v1.0.0 accumulated 6 specs (5 features + 1 bugfix). Rule is correct for features; hotfix/bugfix exemption is reasonable but should be stated explicitly.
- **Gemini e2e run 2 failed** — Two bugs hit simultaneously: keypair import noisy error (misdiagnosed as fatal) and extend 404 abort (Playwright gave up on stale page). Both were fixable but cost a full sandbox TTL cycle.
- **GitGuardian false positive on test fixtures** — `AKIAIOSFODNN7EXAMPLE` / `wJalrXUtnFEMI/K7MDENG` (AWS docs example values) triggered GitGuardian in the new `aws.bats`. Should have used clearly invalid fixture values from the start.
- **CSV detection too broad initially** — First fix used `grep -qi 'access key id'` which matched the Pluralsight label format. Caught by BATS (test 4 failure) before push, but caused a debugging round.
- **`acg_watch` had no lifecycle management in the original spec** — The `--recreate` + `acg_watch` spec didn't include PID tracking. Copilot added it post-merge. Should be part of the spec for any background process.

## Process Rules Added

| Rule | Where | Trigger |
|------|--------|---------|
| Bugfix specs use `bugfix-` prefix, not `hotfix-` | `/bugfix` skill | This milestone |
| Test fixtures must use clearly invalid credential values (no AKIA*/wJalr* patterns) | CLAUDE.md candidate | GitGuardian false positive |
| Background process specs must include PID file lifecycle (start guard + stop in destroy) | spec template | Copilot finding on `acg_watch` |
| All credential file writes must use `_write_sensitive_file` (not raw redirect + chmod) | Copilot review guide | Copilot finding on `_aws_write_credentials` |
| awk host lookups must use `-v host=` + field comparison, never regex interpolation | Copilot review guide | Copilot finding on `shopping_cart.sh` |

## Decisions Made

- **`lib-agc` extraction plan** — `acg_*` stays in k3d-manager until 3 cloud providers ship (v1.0.1–v1.0.5), then extract to `wilddog64/lib-agc`. Prep steps threaded through v1.0.1/v1.0.2/v1.0.3.
- **`aws_provision` namespace** — `acg_*` public functions become private (`_acg_*`); `aws_provision` becomes the public API with ARN-based sandbox auto-detection. `acg_provision` deprecated alias.
- **Remote Control vs k3dm-mcp** — Claude Code Remote Control (`claude --remote-control`) covers mobile operator access. k3dm-mcp repositioned as non-interactive AI agent use case only.
- **`k3d-manager doctor`** — Operator-facing preflight check (inspired by `flutter doctor`) planned for v1.0.1 alongside bin/ wrapper scripts + Claude skills.
- **5-spec rule: bugfix exemption** — Bug fixes discovered during smoke testing must ship in the same release. The 5-spec cap applies to feature scope only; bugfix specs are exempt.

## Theme

v1.0.0 was the first real end-to-end release: a cloud provider that actually provisions infrastructure, installs k3s, and wires up the full lifecycle. The milestone arrived with more scope than planned — six specs instead of five, three smoke test runs instead of one — but it shipped clean. Codex implemented every spec exactly. Gemini found real bugs on a live sandbox. Copilot hardened the security and lifecycle design. The five-agent workflow (Claude + Codex + Gemini + Copilot + GitGuardian) caught every issue before merge. The k3s-aws provider is now a solid foundation for v1.0.1 multi-node work.
