# Retrospective — v0.1.5

**Date:** 2026-06-11
**Milestone:** Bidirectional sandbox switching (AWS ↔ Azure)
**PR:** #42 — merged to main (`180beb8`)
**Participants:** Claude, Codex, Copilot

## What Went Well
- Conflict warning fix (`5f704d1`) correctly identified root cause (broken "Auto Shutdown" selector) and fixed it cleanly
- CDP device code automation (`e9cc187`) fully automated Azure MFA sign-in — no manual browser prompts required
- v0.1.4 cross-reference confirmed provisioning issue was a new edge case, not a regression — accurate diagnosis
- Copilot caught device code leaking to logs (security finding) and three CHANGELOG inaccuracies before merge
- Branch cleanup reduced 44 branches to 5 in one pass

## What Went Wrong
- Cascading regression chain: 5 sequential fixes needed because each fix exposed the next latent bug beneath it; root trigger was `a34cd3c` (addLocatorHandler strict-mode) making the handler fire for the first time
- v0.1.4 baseline not cross-referenced upfront — would have revealed provisioning issue was new, not a regression, saving one iteration
- `24a89f1` device code fix required manual browser prompt (wrong approach); user had to correct to CDP automation
- CHANGELOG had three inaccurate bullets that Copilot caught post-PR-creation; should have been caught during authoring

## Process Rules Added

| Rule | Where |
|------|-------|
| Cross-reference the previous version branch before diagnosing regressions — identical code = new edge case, not regression | CLAUDE.md session notes |
| Spec docs: "Do NOT create a PR yourself — Claude handles PR creation after verifying the commit" | All spec/bug doc templates |

## Decisions Made
- Provisioning fix: use `document.body.innerText` check for "Hang tight" / "Finalizing your playground" text — survives DOM structure changes better than selector-based approach
- addLocatorHandler scoped to `h3, h2` heading elements (not broad text match) — more targeted than `.first()` approach on main; kept in conflict resolution

## Theme
v0.1.5 was a bug-fix cascade triggered by a single correct fix (addLocatorHandler strict-mode) that made a handler fire in production for the first time, revealing four layers of latent bugs underneath. Each fix was correct in isolation but exposed the next problem. The milestone closed when the final edge case — Azure sandbox provisioning auto-closing the credential panel — was detected and handled. The process held: spec-before-implement, Codex implements, Claude verifies, Copilot reviews. The main friction was not having a systematic baseline comparison protocol at the start.
