# Retrospective — v1.4.12

**Date:** 2026-05-29
**Milestone:** Reliability Hardening
**PR:** #83 — merged to main (`c5949cf47097d3a89ff485e01f68c2725644d327`)
**Participants:** Claude, Codex, Gemini, Copilot

## What Went Well

- **imagePullSecrets patch correctly identified root cause** — All named ServiceAccounts patched in one commit. ghcr.io 401 errors eliminated by ensuring non-default SAs have pull credentials.
- **ArgoCD ApplicationSets now visible via `make status`** — Good mitigation for v3.4.2 UI regression. Provides reliable troubleshooting path when UI sidebar is unavailable.
- **ExternalSecret SharedResourceWarning resolved cleanly** — Removed data-layer duplicate. Sole ownership correctly assigned to shopping-cart-infra cicd/product-catalog app.
- **sync-branch/sync-main targets give safe pre-merge testing** — New Make targets allow verifying k3d-manager branches against a live cluster before merging to main. Reduces merge risk significantly.
- **Copilot caught real shell injection risk** — Shell injection via branch/tag names in `kubectl patch` JSON commands. Missing app coverage in refresh loops also flagged before merge.

## What Went Wrong

- **sync-branch/sync-main initially incomplete** — Only 4 of 6 ApplicationSet-managed apps were in the refresh loop. Caught by Copilot review, not by local testing. Should have validated all service apps were included.
- **activeContext.md carry-forward items not cleared** — When resolution PR was merged in the same session, stale In Progress rows were not removed. Created noise for Copilot in the next PR review.

## Process Rules Added

| Rule | File | Rationale |
|------|------|-----------|
| Assign shell variables before JSON interpolation | CLAUDE.md code style | Shell injection via branch/tag names in kubectl patch JSON |
| Clear carry-forward items immediately when resolution PR merges | CLAUDE.md process | Prevents stale Copilot noise in next PR |

## Decisions Made

- **ArgoCD version policy:** OCI (`prod`) always gets latest ArgoCD; ACG AWS (`dev`) runs one version behind. Provides a canary buffer without a separate staging cluster.
- **platform-helm ApplicationSet for OCI:** `environment: prod` label will select OCI cluster; `environment: dev` selects ACG AWS. To be wired up in v1.0.8.
- **ArgoCD v3.4.2 UI regression:** ApplicationSets sidebar removed in v3.4.x. `make status` is now the primary visibility mechanism for ApplicationSet status and troubleshooting.

## Theme

v1.4.12 was a focused reliability hardening sprint closing out loose ends from v1.4.11. The milestone resolved three classes of issues: image pull auth (imagePullSecrets), ArgoCD visibility (ApplicationSets in make status), and ExternalSecret ownership conflicts. The sync-branch/sync-main targets were the only net-new capability — everything else was hardening. Copilot's review was the most valuable gate this cycle, catching the shell injection pattern and the incomplete app list before merge.
