# Retrospective — v1.2.0

**Date:** 2026-04-30
**Milestone:** lib-acg extraction + shopping-cart bootstrap + GHCR hardening
**PR:** #67 — merged to main (`f628c3cb`)
**Participants:** Claude, Codex, Gemini, Copilot

## What Went Well

- **lib-acg extraction landed cleanly** — P1–P5 all completed by Codex; the subtree pattern (`git subtree`) worked well for keeping lib-acg independently versioned while still embeddable in k3d-manager. Three subtree syncs happened during the milestone without conflict.
- **GHCR Vault-first strategy** — Gemini identified the OAuth token scope issue (`gho_` vs `ghp_`), documented it accurately, and the fix (fail-closed + Vault-first) prevents the whole class of `ImagePullBackOff` from OAuth fallback.
- **Vault sealed-state auto-recovery** — `acg-up` now handles sealed Vault at Step 4 instead of aborting; fixes a persistent pain point after Mac sleep.
- **ArgoCD launchd port-forward** — KeepAlive agent survives terminal close and Mac sleep restarts; much more reliable than the PID-file approach.
- **`deploy_shopping_cart_data()` automation** — full data layer provisioned idempotently on every sandbox rebuild; eliminates the multi-step manual bootstrap.
- **ApplicationSet branch variable** — `${K3D_MANAGER_BRANCH}` envsubst approach was cleaner than the runtime kubectl patch that was initially proposed; user caught the overengineering before it merged.

## What Went Wrong

- **Copilot 20k-line limit** — the branch accumulated 20,329 insertions (mostly lib-acg subtree squashes + docs), exceeding Copilot's review limit. PR #66 was created and closed because of this. Process rule needed: subtree squash commits dramatically inflate line counts — future extractions should either be done via a separate PR or noted as exempt from Copilot review upfront.
- **Branch scope too large** — 102 commits across 5 lib-acg extraction phases, 15+ shopping-cart fixes, GCP provider fixes, and infra hardening all in one branch. The CLAUDE.md 5-plan-docs limit was not enforced. The shopping-cart upstream fixes (Fix 1+2+3) were specced but not executed — they became v1.2.1 debt.
- **SharedResourceWarning only half-fixed** — the k3d-manager side (`services/shopping-cart-namespace/`) was done, but upstream repos (`shopping-cart-order`, `shopping-cart-product-catalog`) still define `Namespace/shopping-cart-apps`. ArgoCD still shows `shopping-cart-product-catalog` OutOfSync. The Codex spec exists but was not handed off this milestone.
- **Gemini verify tasks turned into fix tasks** — Gemini was supposed to verify shopping-cart sync state but applied patches instead, some of which didn't solve the root cause. Consistent with the known failure mode documented in memory.
- **Stage2 CI always fails in PR context** — the cluster health check requires a live OrbStack/k3d cluster on the self-hosted runner. This is structurally broken for PR CI — the check passes only when manually running tests. Should either gate behind a `[skip-cluster]` label or make stage2 optional in the workflow.
- **`${K3D_MANAGER_BRANCH}` envsubst left in** — the ApplicationSet template now uses a variable that should revert to hardcoded `main` after merge. This is a post-merge cleanup item on v1.3.0 — easy to forget.

## Process Rules Added

| Rule | Where |
|---|---|
| Subtree squash commits inflate Copilot line count — note as review-exempt upfront | CLAUDE.md |
| Never use `gh auth token` as a fallback for GHCR pull secrets — Vault PAT only | bin/acg-up, rotate-ghcr-pat |
| `${K3D_MANAGER_BRANCH}` must revert to `main` in services-git.yaml on v1.3.0 first commit | activeContext.md |

## Decisions Made

- **Option 2 for sandbox rebuild** — permanent fixes first (Codex), then cleanup gates, then merge to main. No ApplicationSet branch-pin. Documented dependency chain in activeContext.md.
- **SharedResourceWarning** — k3d-manager owns the namespace via `services/shopping-cart-namespace/`; upstream repos remove their `namespace.yaml`. This is cleaner than having ArgoCD `CreateNamespace=true` fight over ownership.
- **Copilot $10 PAYG model** — with the new credit model, use credits for targeted file reviews rather than whole-PR review. The 20k-line limit on squash-heavy branches is an accepted constraint.
- **shopping-cart upstream fixes → v1.2.1** — the three Codex fix branches (init SQL, SecurityConfig, shared namespace upstream) are a separate milestone, not part of this PR. The kustomize workarounds (TCP probes, `ddl-auto=update`) remain as acknowledged tech debt until those PRs merge.

## Theme

v1.2.0 was a debt-reduction milestone disguised as a feature branch. The primary goal — extracting ACG automation out of k3d-manager into `wilddog64/lib-acg` — required five Codex-driven phases, three subtree syncs, and a dozen smaller fixes along the way. The shopping-cart work was opportunistic: every sandbox rebuild exposed a new failure, and each fix landed as a kustomize patch or `acg-up` step. The branch closed with all 5 shopping-cart pods Running and a hardened GHCR provisioning path, but the upstream app fixes are still pending. The architecture is cleaner — lib-acg is its own repo, the data layer bootstraps automatically, and ArgoCD has a stable port-forward — but the v1.2.1 cleanup work is real. The lesson: subtree extractions need their own PR, not a lane inside a feature branch.
