## What Went Wrong

- **Missing README releases-table backfill** ŌĆö the v1.6.5 row was never added when the v1.7.0 branch opened. Project rule states: "first commit on the next branch backfills the just-shipped version." Caught only at PR time, not earlier.

- **Repeated failure-masking anti-pattern** ŌĆö "function ends on bare `_info`" appeared twice before Copilot flagged it. This should have been caught in the spec definition or the first implementation review.

- **External forcing function** ŌĆö ACG sandbox 72h abuse-block + free-tier squeeze (OCI A1 capacity-blocked / unobtainable) drove the unplanned Hostinger VPS purchase. This was not a planned v1.7.0 scope until the sandbox became unusable. Forces next planning to account for sandbox instability.

## Process Rules Added

**Copilot PR review rule** ŌĆö documented in `docs/issues/2026-06-13-copilot-pr95-review-findings.md`:

A function performing file/remote mutations (kubeconfig write, SSH command, etc.) must not end on a bare `_info` or return without checking exit codes. Mutating commands need explicit `|| return 1` or `exit_code=$(...)` capture before continuing. This prevents silent failure masking.

## Decisions Made

- **ACG demoted to occasional learning sandbox** ŌĆö no longer the production app cluster.
- **Permanent app cluster: Hostinger KVM2** ŌĆö $215/2yr, usable ~2028. Registered to the hub orbstack via mTLS cluster secret.
- **`APP_CLUSTER_NAME` envsubst parameter** ŌĆö the retarget switch. Default `ubuntu-hostinger`. Set to `ubuntu-k3s` to override for ACG (useful for local testing).
- **Webhook context-aware smoke test deferred to v1.7.1** ŌĆö v1.7.0 hit the 5-plan-doc limit after 5 specs (provider, argocd-register, applicationset-retarget, status-report, plus 5 bugfix docs in docs/bugs/). No more specs for this release.

## Theme

A milestone driven by an external constraint (ACG abuse-block + free-tier scarcity) turned into a durable win ŌĆö a cheap, permanent, owned app cluster reachable by the same plugin stack, registered to the laptop hub over mTLS, validated live end-to-end.

The provider pattern proved reusable: thin SSH wrapper + k3sup, delegates workload deployment to existing plugins. With the hub on orbstack and the app on Hostinger, k3d-manager now owns the full stack across two infrastructure layers without duplicating logic.
