# Retrospective — v1.0.1

**Date:** 2026-03-31
**Milestone:** Multi-node k3s-aws + CloudFormation + Playwright hardening
**PR:** #58 — merged to main (`a8b6c583`)
**Participants:** Claude, Codex, Gemini, Copilot

## What Went Well

- **CloudFormation parallel provisioning** — single CF stack (`acg-cluster.yaml`) deploys 1 server + 2 agents atomically; eliminates sequential EC2 loops and race conditions; teardown is idempotent via describe-stacks check
- **lib-foundation upstream discipline** — all `agent_rigor.sh` fixes went to lib-foundation first (PR #21 v0.3.15, PR #22 v0.3.16); subtree pull propagated correctly; pre-commit hook caught one direct-edit violation before it merged
- **Copilot coverage quality** — 11 findings across 6 files caught real issues: intra-VPC CIDR scope (10.0.0.0/8 → /16), email leak in INFO log, unbound array variable, `acg_provision` help text describing wrong architecture
- **BATS test isolation** — `[[ "${1:-}" == "--" ]] && shift` stub pattern correctly strips mode flags; all 5 k3s_aws_provider tests pass with clean isolation
- **Gemini e2e (3-node)** — full CloudFormation lifecycle verified on real ACG sandbox: provision → 3 nodes Ready → teardown; milestone gate passed before PR created

## What Went Wrong

- **Subtree violation by Codex** — Codex directly edited `scripts/lib/foundation/scripts/lib/agent_rigor.sh` in commit `abe149f` (CloudFormation spec). Pre-commit hook caught it at PR review time, not at commit time. Required upstream detour (v0.3.15 → v0.3.16) before PR could merge.
- **`grep -Fqx --` missed in original v0.3.15 scope** — The double-dash separator for `grep -Fqx` was a separate lib-foundation PR (#22 v0.3.16) rather than included in the original allowlist fix. Could have been caught earlier if the Copilot review of PR #21 had flagged it.
- **zsh associative array failure** — `declare -A replies=(...)` syntax works in bash but not zsh; caused runtime error during Copilot reply loop. Workaround: `for pair in "id:msg"` loop with `${pair%%:*}` / `${pair#*:}` splitting.
- **enforce_admins restore 422** — Sending `-f enabled=true` in the POST body returns 422; correct call is `POST` with no body. Wasted one round trip.
- **lib-foundation BATS run skipped initially** — Ad-hoc inline test used for v0.3.15 verification before user challenge prompted running the authoritative `bats scripts/tests/lib/agent_rigor.bats` suite. Should always run the full suite first.

## Process Rules Added

| Rule | File | Notes |
|------|------|-------|
| Subtree violation is a Codex failure mode | CLAUDE.md / AGENTS.md | Every spec must explicitly state: "do NOT edit `scripts/lib/foundation/` directly" |
| Always run authoritative BATS suite | CLAUDE.md | Ad-hoc inline tests are not a substitute for `bats <suite>.bats` |
| `grep -Fqx --` double-dash is required | lib-foundation AGENTS.md | Filenames beginning with `-` would be treated as flags without `--` |
| enforce_admins restore: POST with no body | /create-pr skill | `-f enabled=true` in body causes 422; omit body entirely |

## Decisions Made

- **`total_nodes=3` hardcoded** — `ACG_AGENT_COUNT` variable removed; CloudFormation template drives node count; bash code reflects the template, not vice versa. Prevents drift between CF stack and k3sup join logic.
- **VPC CIDR scoped to /16** — `10.0.0.0/8` in intra-VPC SG rule was overly broad (all RFC-1918 class A). Scoped to `10.0.0.0/16` matching the VPC CIDR; security improvement with no functional change.
- **Email not logged in INFO** — `PLURALSIGHT_EMAIL` value is user-supplied credentials; should never appear in structured log output even at INFO level.
- **lib-foundation v0.3.16 as separate PR** — The `grep -Fqx --` fix got its own PR (#22) rather than amending v0.3.15 post-merge. Cleaner history; avoids amending a tagged release commit.

## Theme

v1.0.1 crossed the multi-node threshold: a single `deploy_cluster` call now provisions a real 3-node cluster on ACG EC2 via CloudFormation and k3sup, resolving the t3.medium resource exhaustion that blocked `frontend`, `order-service`, and `payment-service` since v0.9.4. The milestone also tightened the library boundary — lib-foundation's `agent_rigor.sh` gained an IP allowlist feature (v0.3.15) and a separator fix (v0.3.16), both propagated cleanly into k3d-manager via subtree pull. Eleven Copilot findings across the PR caught a real CIDR security gap, a credential leak, and several documentation gaps. The milestone gate: Gemini verified 3 nodes Ready on a live ACG sandbox.
