# Retrospective — v1.4.10

**Date:** 2026-05-28
**Milestone:** ArgoCD Stability + Bootstrap Reliability
**PR:** #81 — merged to main (`f8bad52d`)
**Participants:** Claude, Gemini, Copilot

## What Went Well
- Gemini executed the CHANGELOG update and PR creation cleanly without manual correction
- Copilot identified 5 real bugs (security, reliability, supply-chain, docs accuracy) — all legitimate
- All 16 Copilot threads replied to and resolved programmatically in one pass
- CI stayed green throughout — fix commit required no re-run
- `set -euo pipefail` guard bug caught before it could silently eat a cluster error on fresh bring-up

## What Went Wrong
- SA imagePullSecrets patch was only applied to `shopping-cart-apps` — `shopping-cart-payment` and `shopping-cart-data` would have failed GHCR pulls silently on fresh cluster
- `curl -u user:token` pattern (PAT in argv) introduced in the refactor; repo PAT hygiene rules exist but were not applied at write time
- Helm installer was pinned to `main` branch — supply-chain rule exists in CLAUDE.md but was not caught in spec review
- `docs/api/functions.md` entry for `deploy_shopping_cart_data` was stale — not updated when function behavior changed

## Process Rules Added
| Rule | Where |
|------|-------|
| PAT validation must use `--netrc-file` or `GH_TOKEN` env — never `curl -u user:token` | CLAUDE.md Security Rules |
| `docs/api/functions.md` must be updated atomically with any public function behavior change | PR checklist |
| Helm/tool installers must pin to version tag + set `DESIRED_VERSION` | CLAUDE.md Supply Chain |
| SA `imagePullSecrets` patch must cover all namespaces where `ghcr-pull-secret` is provisioned | spec template note |

## Theme
v1.4.10 closed out the ArgoCD stabilization sprint that started when the shopping-cart stack started silently diverging on every fresh `make up`. The root causes — ApplicationSet overwriting manual project patches, hook finalizers blocking teardown, password reconciliation races, and missing ESO syncs — were all fixed and hardened into a single PR. Copilot caught four additional issues (PAT hygiene, SA imagePullSecrets completeness, supply-chain pin, stale docs) that the spec didn't anticipate, keeping the quality bar consistent with the rest of the codebase.
