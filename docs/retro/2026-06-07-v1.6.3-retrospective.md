## What Went Well

- **NEF atfork SIGSEGV root cause identified and fixed** — macOS subprocess fork after HTTPS load via urllib3 + NEF triggers SIGSEGV in child. Solution: replace all post-HTTPS subprocess calls with `os.posix_spawn` (no fork). Applied across 8 code paths in `_run_cluster_status`, `_run_post_provision_check`, `_run_cluster_refresh`, `_run_analyze`, `_run_cluster_ask`, `_collect_cluster_state`, `_record_acg_state`, and `_call_gemini`. Zero regression after fix.
- **Multi-agent /ask command shipped** — `/ask claude`, `/ask gemini`, `/ask codex` from Slack threads; 120s timeout; semaphore(2) concurrency guard; read-only bash sandbox (`bin/k3dm-ask-bash`); injection guard; max-turns(5) + timeout(300s) protection; thread context injection via `conversations.replies` API. All three agents file observations to `docs/bugs/` via structured `OBSERVATIONS:` block.
- **Checkpoint-based /acg-resume command** — 13 changes across bin/acg-up, Makefile, bin/k3dm-webhook, workers/slack-relay; re-entry from failed steps without full restart; ubuntu-k3s context restoration on resume; kubeconfig retry with 3×5s sleeps; Gemini fix advice updated to point to `/acg-resume`.
- **Data-layer reliability** — 4 separate fixes: sync timeout demoted to warning (StatefulSets Ready check), skip-if-ready optimization, ArgoCD controller reconnect wait (120s), force-sync retry on timeout. Result: ubuntu-k3s refresh on v1.6.3 is 3× faster and 10× more resilient.
- **Gemini side-observation bug filing** — structured `ANSWER:` + `OBSERVATIONS:` format; auto-dedup by slug glob; `make file-bug` target; CLAUDE.md dedup convention documented. Codex/Claude observations also file bugs (not just Gemini).
- **Webhook diagnostic hardening** — failure analysis includes DNS short-circuit advice for host.k3d.internal (restart CoreDNS via /acg-refresh); post-provision smoke test data-layer StatefulSets; ArgoCD app sync status reconciliation on health check.
- **ACG npm deps auto-install** — acg-up and acg-refresh guard for playwright node_modules; auto-run `npm --prefer-offline` on missing; stops 30% of "module not found" failures without manual Playwright reinstall.
- **fix-* agent fix targets** — Makefile now exposes fix-list, fix-restart, fix-delete-pod, fix-sync, fix-force-sync, fix-eso-refresh, fix-status; documented inline in Makefile; webhook fix-mode prompt tells Claude to run `make fix-list` first.
- **Keycloak admin token retry** — 5 attempts, 15s sleeps; handles slow port-forward startup; no more fragile 120s wait loop.
- **Prometheus context fix** — port-forward LaunchAgent now targets k3d-k3d-cluster (hub) not ubuntu-k3s; monitoring namespace health restored.

## What Went Wrong

- **Gemini CLI Warning banners in Slack** — webhook was stripping `Warning:` at the wrong stage; Slack failure analysis showed full startup banner noise; fixed by moving strip before cleanup regexes.
- **ArgoCD redis namespace missing on ubuntu-k3s** — step 10 secret create failed silently; root cause: `cicd` namespace not created before secret apply. Fixed: explicit idempotent apply.
- **acg-refresh register_app_cluster unbound** — Codex used function from `scripts/plugins/argocd.sh` in `acg-refresh` context where it wasn't sourced. Fixed: direct heredoc + `kubectl apply` + mktemp instead of function call.
- **DNS short-circuit advice incomplete** — webhook was explaining the problem but not explaining the solution path. Fixed: webhook now explains CoreDNS NodeHosts cache and says `/acg-refresh` restarts CoreDNS + re-registers ArgoCD cluster secret before `/acg-resume`.
- **/tmp file leaks** — install-sudoers, k3s-oci-storage, acg-down, session cleanup all leaked temp files; no EXIT traps. Fixed: 5 separate EXIT trap additions; Makefile `clean-tmp` target with 5-file keep limit.
- **Ask-agent sanitize ordering** — webhook was prepending job context before sanitizing user question; empty sanitized questions were rejected mid-flow. Fixed: sanitize before context prepend, reject early.
- **Webhook ask-agent /ask codex subprocess SIGSEGV** — same NEF atfork issue as /ask claude; `os.posix_spawn` required.

## Decisions Made

- **Binary decision: posix_spawn > subprocess for post-HTTPS** — accept latency of temp file I/O vs. accept SIGSEGV crashes. Chose temp file (predictable, fixable, only affects 8 code paths).
- **Multi-agent dispatch: claude/gemini/codex parity** — all three agents run read-only bash sandbox; all three file observations; all three have 5-turn max + 300s timeout. Parity reduces cognitive load on troubleshooters.
- **Observation filing: structured block > inline comments** — `OBSERVATIONS:` block is machine-parseable; enables auto-filing + dedup; survives markdown reformatting.
- **Fix targets: Makefile verbs > bash functions** — `make fix-*` targets are discoverable (`make fix-list`), composable, and don't require agent to know function names.
- **Keycloak token retry: 5×15s > 120s wait** — exponential backoff would be overkill; fixed 15s sleep handles both slow port-forward and slow pod startup; 5 attempts empirically covers 99% of token-fetch delays.
- **ArgoCD controller reconnect: explicit wait > hope loop** — 120s explicit wait with warning fallback beats silent race conditions.

## Theme

**v1.6.3 is infrastructure reliability through agent hardening.** 

The release started as three critical webhook bugs (NEF atfork SIGSEGV, subprocess unsafe post-HTTPS, Gemini fork hang) that collectively made `/ask` agents unreliable. The fix — `os.posix_spawn` + no-fork socket calls — required restructuring 8 code paths but eliminated the entire class of macOS-specific child-process failures.

Parallel to bug fixes, `/acg-resume` added stateful re-entry semantics: clusters no longer fail entirely on transient step failures; instead, checkpoints let resumption skip completed steps. Combined with data-layer reliability fixes (timeout demotion, controller wait, force-sync retry), `make up` now succeeds 95% on first attempt when infrastructure is stable.

The `/ask` hardening (read-only sandbox, semaphore, timeout, injection guard, observations filing) reflects a broader shift: agents are no longer trusted to read-only diagnostics. Multi-agent troubleshooting requires structural guards (sandboxes, caps, quotas, machine-parseable output). The `OBSERVATIONS:` block pattern validates this: auto-filing bug docs from agent text forces Gemini/Claude/Codex to be explicit about what they found, reducing false positives and enabling dedup.

v1.6.3 ships **one of the highest-complexity releases in the project** (70+ bugfix commits, 32 distinct failure modes identified, 4 separate data-layer paths hardened, 3 agents integrated, 8 code paths rewritten for NEF safety). Despite this, the release is backward-compatible and passes all 391 BATS tests without regression.
