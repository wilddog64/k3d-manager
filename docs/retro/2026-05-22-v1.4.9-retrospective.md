## What Went Well

1. **Subtree pulls from lib-acg stabilizing the pipeline** — Ghost State Delete Sandbox selector fixes (lib-acg commits `0248736`, `0b40f42`, `6a69232`) were pulled into k3d-manager and resolved Playwright locator drift. The `acg_check_ttl` regression recovery on `6a69232` was caught during the subtree pull, preventing silent breakage on `make up`.

2. **Credential extraction visibility fixes** — The ACG flow now properly skips extraction when credentials are valid (`7d2be2e6`), extends sandboxes before TTL expiration (`a89cc2e5`), and auto-restarts expired sandboxes (`0395cf8e`) without user intervention.

3. **CloudFormation race condition hardening** — Added polling loop after credential extraction to wait for CloudFormation stack visibility (`2e46a4a2`), preventing `Invalid template path` errors on fresh AWS credential flows.

4. **Keycloak + ArgoCD public URL wiring completed** — ACG provisioning now generates Cloudflare tunnel config from template (`d8f2ca47`), patches ArgoCD to use public URLs, and updates Keycloak redirectUri + frontendUrl. Cross-repo fixes in shopping-cart repos landed cleanly.

5. **CoreDNS host plugin patching** — Replaced the broken duplicate `hosts {}` block injection with a surgical `data.NodeHosts` patch (`5efe45e8`), eliminating CoreDNS ConfigMap apply failures on ubuntu-k3s.

## What Went Wrong

1. **Playwright credential-wait handler race condition persists** — The "Extend Your Session" modal continues to cause Playwright locator instability. Despite multiple handler fixes (adding `{ times: 1 }`, 1000ms settle, noWaitAfter, etc.), the handler is firing, but the modal re-appear during React re-render is still causing Start Sandbox button detachment in edge cases. This issue was documented in earlier cycles (`docs/bugs/2026-05-16-*`) and superseded by the handler-times-1-click-helper spec, but live verify still catches intermittent failures.

2. **Ghost State credential extraction timeout still flaky** — The `InvalidClientTokenId` error in `make credential-test` (recorded in `docs/issues/2026-05-20-lib-acg-credential-test-invalid-client-token.md`) suggests that the Playwright flow is timing out or not waiting long enough for ACG to fully render credentials after Ghost State recovery. The 10s deleteBtn timeout in `acg_extend.js` may be too aggressive.

3. **lib-foundation enforceAdmins restoration incomplete** — Branch protection API call for enforce_admins on lib-foundation PR #28 post-merge never completed (API call failed with 404). Blocking issue is that the exact API endpoint for turning enforce_admins back ON is unclear; GitHub REST docs show it as a GET/PATCH on `.../protection/enforce_admins` but `curl -i` tests returned 404.

## Process Rules Added

1. **Subtree regression protocol** — When pulling from a subtree branch (`feat/acg-gcp-credentials`, `fix/ghost-state-waitForSelector`, etc.), always verify that no prior functions regressed (e.g., `acg_check_ttl` on the `fix/ghost-state-waitForSelector` pull was found missing and had to be restored from `936fea39`).

2. **CloudFormation status polling** — AWS credential extraction followed by CloudFormation stack operations must include a polling loop; add 15s×12 retries to prevent `Invalid template path` from race condition.

3. **Playwright handler scope narrowing** — When `addLocatorHandler()` is catching modal appearances, always scope to the modal visibility state (`[data-open="true"]`) rather than global page state to prevent handler loops on unrelated DOM changes.

## Decisions Made

1. **v1.5.0 scope: sudo passwordless work** — Next release will focus on removing interactive sudo prompts from `acg-up` and `acg_extend` flows via polkit rules or passwordless sudoers entries on ubuntu-k3s.

2. **lib-foundation PR #28 enforce_admins debt** — Deferred to next maintenance cycle; manual GitHub UI restore is acceptable interim solution.

3. **Ghost State credential timeout** — Postponed extending the wait timer further until live verify fully characterizes the failure mode (timing vs. selector drift vs. modal race).

## Theme

**Infrastructure resilience through visibility and polling.** This cycle solidified the ACG sandbox lifecycle (TTL checks, auto-extend, auto-restart) and multi-cluster routing (Keycloak + ArgoCD public URLs). The remaining work is hardening transient AWS + Playwright failures via polling loops and handler scope narrowing — moving from "it usually works" to "it works with recovery."
