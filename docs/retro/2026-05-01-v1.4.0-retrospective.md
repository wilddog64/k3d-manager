# Retrospective — v1.4.0

**Date:** 2026-05-01
**Milestone:** Copilot CLI plugin + `_copilot_review` rename
**PR:** #69 — merged to main (`a805dee0`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- Codex implemented `scripts/plugins/copilot.sh` exactly to spec — zero diff from the plan doc; shellcheck + bash -n both passed first try
- Copilot found 9 real bugs on the PR (gate variable mismatch, missing `--help`, fail-fast ordering, CWD-dependent git, wrong lint-rules path) — all fixed in one commit before merge
- Lib-foundation subtree discipline held: `_copilot_review` rename went to lib-foundation first (`d24b457`), then subtree pull into k3d-manager — no direct edits to subtree files
- `AGENT_LINT_GATE_VAR` double-gate bug (Copilot finding) caught before merge — would have silently no-op'd the entire AI lint feature

## What Went Wrong

- **How-to doc used wrong lint-rules path** (`scripts/etc/lint-rules.md` vs actual `scripts/etc/agent/lint-rules.md`) — Copilot caught it; missed in Claude's own review pass
- **`--help` convention not followed** — docs/api/functions.md says all public functions support `-h/--help`; both new functions omitted it; Copilot caught it
- **Fail-fast ordering** — context collection (kubectl/git) ran before `K3DM_ENABLE_AI` check; Copilot caught it; should be standard template in specs going forward
- **Naming decided after implementation** — the `_ai_agent_review` / `AI_REVIEW_FUNC` design was identified after #69 was already open; means docs shipped with placeholder naming that needs a follow-on refactor (`k3d-manager-v1.4.1`)
- **BATS suite deferred** — `scripts/tests/plugins/copilot.bats` not written; Copilot flagged it; accepted as follow-on

## Process Rules Added

| Rule | Where to enforce |
|---|---|
| Public plugin functions must check `K3DM_ENABLE_AI` (or equivalent gate) before any external command (kubectl/git/curl) | Spec template `## What to Change` section |
| Public functions must implement `-h\|--help` returning 0 | Codex spec `## Rules` section |
| Every new plugin must have a corresponding BATS file in `scripts/tests/plugins/` | `## Definition of Done` checklist |
| How-to docs must reference lint-rules at `scripts/etc/agent/lint-rules.md` | How-to template |

## Decisions Made

- **`_ai_agent_review` + `AI_REVIEW_FUNC` + `AI_REVIEW_MODEL`** — generic AI dispatch abstraction goes in lib-foundation; no `K3DM_` prefix since lib-foundation is shared. Default backend: `copilot`. Default model: `gpt-5.4-mini`. Refactor target: `k3d-manager-v1.4.1`.
- **`_copilot_review` stays as internal impl** — not renamed; `_ai_agent_review` becomes the stable public surface that routes to it.
- **`K3DM_ENABLE_AI` stays** — it is a k3d-manager feature gate (separate concern from AI function selection).

## Theme

v1.4.0 was a short, focused sprint: extract a daily-use interface on top of `_copilot_review` and confirm the pattern works. It did — pod triage and spec drafting are live, and the pre-commit AI lint gate is wired. The milestone surfaced a naming gap (function and env var don't match the generic-AI vision) and a BATS gap, both accepted as clean follow-on work. Copilot review quality was high — 9 findings, all legitimate, all fixed before merge. The subtree discipline held for the first time without incident.
