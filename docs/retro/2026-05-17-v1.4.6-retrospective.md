## What Went Well

- Root cause investigation was systematic: live cluster kubectl sessions identified exact failure points (UUID PK mismatch, missing customerId param, field name mismatches)
- Spec-before-implement discipline held: all 3 fix areas captured in docs/bugs/ specs before Codex touched code
- Codex implemented all 3 specs correctly in one pass — branch discipline, shellcheck gate, tsc --noEmit gate all passed
- Copilot caught a real bug: `Number(p.price)` producing NaN — fixed in Phase 2 with `?? 0` default
- Multi-repo PR workflow (Phase 1 Haiku → Phase 2 Sonnet → Phase 3 Haiku) handled 5 repos cleanly

## What Went Wrong

- ArgoCD self-heal reverted a manual configmap patch mid-session — wasted investigation time; permanent spec fix was the right call
- Keycloak LDAP federation was accidentally patched to wrong instance (dc=home,dc=org) in an earlier session — required manual revert
- product-catalog port confusion (8080 vs 8082) — app listens on 8080; service routes 8082→8080; easy to confuse when testing directly inside pod

## Process Rules Added

None added this milestone.

## Decisions Made

- **SSH reverse tunnel + iptables DNAT + CoreDNS patch pattern** — chosen over VPN/overlay because: no new infrastructure, idempotent, failure is non-fatal (warn not error), works with existing ssh ubuntu alias
- **Frontend-only adapter pattern** — all response field mapping done in service layer, not in components; backend contract is immutable from frontend
- **SQLAlchemy owns products schema** — init-db.sql must not compete with ORM for DDL ownership; CREATE TABLE IF NOT EXISTS guard is the anti-pattern to avoid

## Theme

This milestone closed out three compounding failures in the shopping-cart stack: a DNS unreachability bug blocking JWT auth across clusters, a schema ownership conflict silently breaking product serialization on fresh clusters, and frontend service functions that assumed a backend contract that was never true. All three were discovered through live kubectl investigation rather than spec review — a reminder that the cluster is the source of truth, not the manifests.
