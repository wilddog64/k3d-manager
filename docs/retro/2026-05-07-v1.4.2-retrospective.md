# Retrospective — v1.4.2

**Date:** 2026-05-07
**Milestone:** _ai_agent_review abstraction + ArgoCD bootstrap hardening + lib-acg cdp.sh fixes
**PR:** #71 — merged to main (`ad8df98c`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- **`_ai_agent_review` abstraction landed cleanly** — the indirection layer (backend via `AI_REVIEW_FUNC`, model via `AI_REVIEW_MODEL`) gives consumers a stable surface regardless of which AI tool we switch to next.
- **BATS coverage expanded** — `scripts/tests/plugins/copilot.bats` added 8 targeted tests covering the triage and spec functions including the AI guard; test helpers (`RUN_LOG`, stub functions) followed the established pattern without deviation.
- **lib-acg subtree pull was mechanical** — PR #10 changes pulled cleanly (`0038ce4d`), no merge conflicts; `launchctl bootout` replacement and dead Linux else-block removal were self-contained.
- **Copilot caught real issues** — 8 findings across two review rounds, all substantive: `$@` passthrough scope bug, CHANGE.md timeout value mismatch (300s vs 600s), misleading `K3DM_ENABLE_AI=1 source` pattern in docs, unreliable BATS `case` patterns with embedded `$SCRIPT_DIR`.
- **Subtree discipline held** — pre-commit hook blocked the attempted direct edit to `scripts/lib/foundation/`; the fix was correctly deferred to a lib-foundation upstream PR rather than bypassing the hook.

## What Went Wrong

- **PR #70 (v1.4.1) was closed without merging** — required accumulating v1.4.1 + v1.4.2 changes in a single PR, complicating the CHANGE.md story and making it harder to bisect if regressions appear.
- **CHANGE.md timeout value error** — stated `default 300s` when the code default is `600s`; Copilot caught it in two separate findings before it was fixed. Root cause: value was copied from memory rather than read from the source.
- **`deploy_argocd_bootstrap "$@"` decision thrash** — user feedback was contradictory mid-session (keep vs. remove), requiring multiple back-and-forth commits. Final resolution (remove `$@`, defer proper fix to next release via lib-foundation) was clear but arrived late.
- **Push rejected after `git reset --hard`** — the no-args commit (`a7593636`) had partially materialized on the remote during an apparently rejected tool call, requiring a `git pull --rebase` to reconcile.
- **lib-foundation subtree doc not fixed this milestone** — `scripts/lib/foundation/docs/api/functions.md` still has a `K3DM_ENABLE_AI=1` in the usage snippet that is k3d-manager-specific. Deferred to next release upstream PR.

## Decisions Made

- **`deploy_argocd_bootstrap "$@"` removed for v1.4.2**: The passthrough was leaking `deploy_argocd` flags (e.g. `--enable-ldap`) into bootstrap which only accepts `--skip-applicationsets`/`--skip-appproject`. Correct fix is to restore it via lib-foundation with explicit flag filtering. Tracked for v1.4.3.
- **`K3DM_ENABLE_AI` gate stays in callers**: lib-foundation functions must not check consumer-specific env vars; `copilot_triage_pod` and `copilot_draft_spec` already have the gate and that is the correct layer.
- **BATS `case` patterns use `*` glob for path segments**: `"$SCRIPT_DIR"` embedded in `case "$*" in` patterns is unreliable because `$*` carries no literal quotes; `*` wildcards are the correct form when the path segment value is dynamic.

## Theme

v1.4.2 was a consolidation milestone — two releases worth of work in one PR because v1.4.1 never made it to main. The AI dispatch abstraction (`_ai_agent_review`) is the headline feature: it decouples the plugin surface from the Copilot backend and makes the model switchable by env var. The session demonstrated the subtree discipline working as designed — the pre-commit hook blocked a shortcut that would have put k3d-manager-specific code in lib-foundation. The tension around `"$@"` passthrough is a design debt item: bootstrap needs to be a first-class caller-aware interface, not a flag absorber. That's the lead story for v1.4.3.
