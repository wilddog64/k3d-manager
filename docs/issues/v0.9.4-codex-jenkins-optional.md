# Codex Task: Make Jenkins an Optional Component

## Context

Jenkins has no active use cases in this environment — all CI/CD runs on GitHub Actions,
all GitOps runs on ArgoCD. Jenkins is deployed by default but consumes ~512MB RAM on the
infra cluster unnecessarily. It should be opt-in via `ENABLE_JENKINS=1`.

## Before You Start

1. `hostname && uname -n` — confirm you are on the correct machine
2. `git checkout k3d-manager-v0.9.4 && git pull origin k3d-manager-v0.9.4`
3. Read `memory-bank/activeContext.md` and `memory-bank/progress.md`

## Files to Change

### 1. `scripts/plugins/jenkins.sh` — early-exit gate in `deploy_jenkins()`

Add at the very top of the `deploy_jenkins()` function body (after the opening `{`),
before any local variable declarations:

```bash
if [[ "${ENABLE_JENKINS:-0}" != "1" ]]; then
   _info "[jenkins] skipped — set ENABLE_JENKINS=1 to deploy"
   return 0
fi
```

### 2. `scripts/plugins/ldap.sh` — gate jenkins user/group LDIF seeding

In the function that seeds LDAP (search for `jenkins_user_dn`), wrap the jenkins-specific
LDIF blocks behind `ENABLE_JENKINS`. Specifically:

- The `jenkins_password` lookup from Vault (lines ~398–402) — skip if `ENABLE_JENKINS!=1`
- The `group_members` append for jenkins (line ~420) — skip if `ENABLE_JENKINS!=1`
- The `cn=jenkins-admins` group entry in the LDIF (line ~441) — skip if `ENABLE_JENKINS!=1`
- The `jenkins_user_dn` entry append (lines ~449–462) — skip if `ENABLE_JENKINS!=1`

Pattern to use:

```bash
if [[ "${ENABLE_JENKINS:-0}" == "1" ]]; then
   # jenkins-specific LDIF block
fi
```

### 3. `scripts/plugins/vault.sh` — gate jenkins Vault policy in `_enable_kv2_k8s_auth()`

In `_enable_kv2_k8s_auth()`, the last line calls `_vault_set_eso_init_jenkins_writer`.
Wrap it:

```bash
if [[ "${ENABLE_JENKINS:-0}" == "1" ]]; then
   _vault_set_eso_init_jenkins_writer "$ns" "$release" "$eso_sa" "$eso_ns"
fi
```

```bash
@test "deploy_jenkins skips when ENABLE_JENKINS is unset" {
  run env -i HOME="$HOME" PATH="$PATH" \

}
```
