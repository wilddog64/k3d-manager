# Issue: services-git ApplicationSet stamps shopping-cart-identity with wrong destination

**Date:** 2026-05-09
**Severity:** High — blocks Keycloak deployment, ArgoCD SSO unavailable
**Repo:** k3d-manager
**Branch affected:** main

---

## Symptom

After `make up` and shopping-cart-infra PRs merge, the `shopping-cart-identity` ArgoCD
Application fails to sync with:

```
namespace identity is not permitted in project 'platform'
```

Keycloak, OpenLDAP, and all associated ExternalSecrets never deploy. ArgoCD SSO is
unavailable.

---

## Root Cause

`scripts/etc/argocd/applicationsets/services-git.yaml` scans `services/*` with no
exclusions and stamps every discovered directory with:

```yaml
destination:
  name: ubuntu-k3s        # remote k3s cluster
  namespace: shopping-cart-apps
```

`services/shopping-cart-identity` falls within `services/*`, so ArgoCD generates an
Application for it with the remote-cluster destination. But the identity stack
(Keycloak + LDAP) deploys to the **local** hub cluster (`https://kubernetes.default.svc`)
in namespace `identity`.

The `platform` AppProject permits `identity` namespace only on the local cluster — not on
`ubuntu-k3s`. The destination mismatch causes every sync attempt to fail with a project
permission error.

The Application generated by the ApplicationSet controller also **overwrites** the correct
Application that `acg-up` Step 10c applies via its heredoc (which has the right destination),
because `selfHeal: true` continuously reconciles back to the broken template.

---

## Impact

- `shopping-cart-identity` Application: `Sync Status: OutOfSync`, `Health: Missing`
- Keycloak: never deployed
- OpenLDAP ExternalSecret: never synced
- ArgoCD SSO: unavailable
- `acg-up` Step 10c Application overwritten on every ApplicationSet reconcile cycle

---

## Fix

Exclude `services/shopping-cart-identity` from the `services-git` ApplicationSet generator.
See `docs/bugs/v1.4.5-bugfix-services-git-identity-exclude.md`.

The identity stack is a platform concern managed by `acg-up` Step 10c; it must not be
generated by the services ApplicationSet.
