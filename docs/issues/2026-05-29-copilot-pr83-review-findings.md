## Finding 1 — Shell injection in `sync-branch` BRANCH interpolation

**File:** `Makefile:87`  
**What Copilot flagged:** `$(BRANCH)` interpolated directly into single-quoted JSON string. A branch name containing a single quote could break out of the JSON string and execute arbitrary shell commands.

**Fix applied:**

Before:
```makefile
'[{"op":"replace",...,"value":"$(BRANCH)"}]'
```

After:
```makefile
@_b='$(BRANCH)'; \
kubectl patch ... "[{\"op\":\"replace\",...,\"value\":\"$$_b\"}]"
```

**Root cause:** Make variable expansion happens before the shell sees the string. Assigning to a shell variable with single quotes first prevents metacharacter injection.

**Process note:** Any Makefile target that accepts a user-supplied variable and uses it in a shell string must assign to a quoted shell variable before interpolation.
