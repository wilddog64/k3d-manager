# Bugfix: v1.6.3 — acg-refresh section 2b calls unbound `register_app_cluster`

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/acg-refresh`

---

## Problem

Section 2b (added in `b46391f1`) calls `register_app_cluster` inside `bin/acg-refresh`,
but that function is defined in `scripts/plugins/argocd.sh` which is never sourced by
`acg-refresh`. At runtime this fails immediately with:

```
bin/acg-refresh: line 197: register_app_cluster: command not found
```

**Root cause:** Codex replaced the spec's direct `kubectl apply` heredoc with a
`register_app_cluster` call, which is not in scope in `acg-refresh`.

---

## Fix

Replace the `else` branch (lines 195–198) with the direct heredoc + `kubectl apply`
approach the original spec required. Also add xtrace-save/restore around token
extraction and heredoc generation to prevent the bearer token appearing in shell trace.

**Exact old block (lines 188–199):**

```bash
# ── 2b. ArgoCD cluster secret ────────────────────────────────────────────────
_info "[acg-refresh] Refreshing ArgoCD cluster secret..."
_argocd_sa_token=$(kubectl get secret argocd-manager-token -n kube-system --context ubuntu-k3s \
  -o jsonpath='{.data.token}' 2>/dev/null | base64 -d | tr -d '\n')
if [[ -z "${_argocd_sa_token}" ]]; then
  _warn "[acg-refresh] argocd-manager-token secret not yet populated — skipping cluster secret refresh"
else
  kubectl config use-context k3d-k3d-cluster >/dev/null
  ARGOCD_APP_CLUSTER_TOKEN="${_argocd_sa_token}" \
    register_app_cluster
  _info "[acg-refresh] ArgoCD cluster registered: ubuntu-k3s → https://host.k3d.internal:6443"
fi
```

**Exact new block:**

```bash
# ── 2b. ArgoCD cluster secret ────────────────────────────────────────────────
_info "[acg-refresh] Refreshing ArgoCD cluster secret (ubuntu-k3s SA token)..."
_argocd_tracing=0; case $- in *x*) _argocd_tracing=1; set +x;; esac
_argocd_token=$(kubectl get secret argocd-manager-token -n kube-system \
  --context ubuntu-k3s -o jsonpath='{.data.token}' 2>/dev/null | base64 -d | tr -d '\n' || echo "")
(( _argocd_tracing )) && set -x || true
if [[ -n "${_argocd_token}" ]]; then
  _argocd_server="${ARGOCD_APP_CLUSTER_SERVER:-https://host.k3d.internal:6443}"
  _argocd_patch="$(mktemp -t argocd-cluster-patch.XXXXXX.yaml)"
  _argocd_tracing=0; case $- in *x*) _argocd_tracing=1; set +x;; esac
  cat > "${_argocd_patch}" <<PATCH
apiVersion: v1
kind: Secret
metadata:
  name: cluster-ubuntu-k3s
  namespace: cicd
stringData:
  server: "${_argocd_server}"
  config: |
    {"bearerToken":"${_argocd_token}","tlsClientConfig":{"insecure":true}}
PATCH
  (( _argocd_tracing )) && set -x || true
  kubectl apply -f "${_argocd_patch}" --context k3d-k3d-cluster >/dev/null
  rm -f "${_argocd_patch}"
  _info "[acg-refresh] ArgoCD cluster secret refreshed: ${_argocd_server}"
else
  _warn "[acg-refresh] argocd-manager-token not readable on ubuntu-k3s — cluster secret not refreshed"
fi
```
