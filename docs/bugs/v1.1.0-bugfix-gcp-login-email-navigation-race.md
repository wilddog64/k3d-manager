## Problem

After clicking "Use another account" in the Google account chooser, `gcp_login.js` still times out waiting for `input[type="email"]` despite the `waitForLoadState('domcontentloaded')` added in `c7930b93`.

**Root cause:** `waitForLoadState('domcontentloaded')` called *after* `click()` resolves immediately against the already-loaded current page — it does not wait for the navigation triggered by the click. The navigation event fires and completes before the listener is registered, so the script proceeds to look for the email input on the wrong page state.

---

## Reproduction

```
make up CLUSTER_PROVIDER=k3s-gcp
# observe:
# INFO: Clicking "Use another account"
# ERROR: locator.waitFor: Timeout 15000ms exceeded.
# Call log: waiting for locator('input[type="email"]').first() to be visible
```

---

## Fix

### Change 1 — `scripts/playwright/gcp_login.js`: replace post-click `waitForLoadState` with `Promise.all` navigation guard

The navigation listener **must** be registered before `click()` is called. The Playwright idiom for click-triggered navigations is `Promise.all([waitForNavigation, click])`.

**Exact old block (lines 65–72):**

```js
      if (useAnotherVisible) {
        console.error('INFO: Clicking "Use another account"');
        await useAnother.click();
        await page.waitForLoadState('domcontentloaded');
      }
      // Fill email
      const emailInput = page.locator('input[type="email"]').first();
      await emailInput.waitFor({ timeout: 15000 });
```

**Exact new block:**

```js
      if (useAnotherVisible) {
        console.error('INFO: Clicking "Use another account"');
        await Promise.all([
          page.waitForNavigation({ waitUntil: 'domcontentloaded', timeout: 20000 }),
          useAnother.click(),
        ]);
      }
      // Fill email
      const emailInput = page.locator('input[type="email"]').first();
      await emailInput.waitFor({ timeout: 20000 });
```
