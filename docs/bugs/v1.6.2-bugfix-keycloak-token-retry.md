# Bugfix: v1.6.2 — Retry Keycloak admin token fetch on slow start

**Branch:** `k3d-manager-v1.6.2`
**Files:** `bin/acg-up`

---

3. Read `bin/acg-up` lines 668–684 (port-forward + token fetch block)

---

## Problem

After the Keycloak deployment becomes `Available`, `acg-up` starts a port-forward and
waits `sleep 3` before attempting a single curl to fetch an admin token. If Keycloak's
HTTP API is not yet accepting connections (typically 30–60s after the pod becomes
`Available`), the curl fails silently, `_kc_token` is empty, and the provision aborts:

```
[acg-up] Could not get Keycloak admin token — realm import is required for SSO and cannot be skipped
```

**Root cause:** `sleep 3` is too short; there is no retry, so any startup lag causes a
hard failure that requires a full re-run.

---

## Reproduction

1. Run `acg-up` on a fresh sandbox where Keycloak is starting cold.
2. Observe: provision fails at Step 10d with `Could not get Keycloak admin token`.
3. Gemini triage reports `TRANSIENT` — re-running `make up` succeeds because Keycloak
   is already warm.

---

## Fix

### Change — `bin/acg-up`: replace `sleep 3` + single token fetch with retry loop

Replace the `sleep 3` and single-attempt curl block with a retry loop (5 attempts,
15s between retries). The port-forward is started once before the loop.

**Exact old block (lines 670–683):**

```bash
  kubectl port-forward svc/keycloak -n identity --context k3d-k3d-cluster "${_kc_pf_port}:80" \
    >>"${_ACG_STATE_DIR}/logs/keycloak-pf.log" 2>&1 &
  _kc_pf_pid=$!
  disown "${_kc_pf_pid}"
  sleep 3

  _kc_token=$(curl -sf -X POST \
    "http://localhost:${_kc_pf_port}/realms/master/protocol/openid-connect/token" \
    -d "client_id=admin-cli&grant_type=password&username=admin&password=${_kc_admin_pass}" \
    | grep -o '"access_token":"[^"]*"' | sed 's/"access_token":"//;s/"//') || _kc_token=""

  if [[ -z "${_kc_token}" ]]; then
    kill "${_kc_pf_pid}" 2>/dev/null || true
    _acg_fail "[acg-up] Could not get Keycloak admin token — realm import is required for SSO and cannot be skipped"
  else
```

**Exact new block:**

```bash
  kubectl port-forward svc/keycloak -n identity --context k3d-k3d-cluster "${_kc_pf_port}:80" \
    >>"${_ACG_STATE_DIR}/logs/keycloak-pf.log" 2>&1 &
  _kc_pf_pid=$!
  disown "${_kc_pf_pid}"

  _kc_token=""
  for _kc_attempt in 1 2 3 4 5; do
    sleep 15
    _kc_token=$(curl -sf -X POST \
      "http://localhost:${_kc_pf_port}/realms/master/protocol/openid-connect/token" \
      -d "client_id=admin-cli&grant_type=password&username=admin&password=${_kc_admin_pass}" \
      | grep -o '"access_token":"[^"]*"' | sed 's/"access_token":"//;s/"//') || _kc_token=""
    if [[ -n "${_kc_token}" ]]; then
      break
    fi
    _info "[acg-up] Keycloak admin token not ready (attempt ${_kc_attempt}/5) — retrying in 15s..."
  done

  if [[ -z "${_kc_token}" ]]; then
    kill "${_kc_pf_pid}" 2>/dev/null || true
    _acg_fail "[acg-up] Could not get Keycloak admin token after 5 attempts — realm import is required for SSO and cannot be skipped"
  else
```
