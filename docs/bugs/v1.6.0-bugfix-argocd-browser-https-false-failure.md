# Bugfix: v1.6.0 — argocd-browser-https wait prints false failure in headless context

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/acg-up`

---

## Problem

In the webhook/headless context (no sudo, no TTY), `bin/acg-up` skips the
`argocd-browser-https` LaunchDaemon install but then calls
`_argocd_wait_for_browser_https` anyway. That function times out, tails the old
socat log to stderr, and prints `ERROR: [argocd] Argo CD did not become reachable`.
The Slack relay forwards this stderr to the Slack notification, making the job look
like it failed — even though the script continues and ultimately succeeds.

**Root cause:** The headless skip guard on the install block sets no flag, so the
wait block runs unconditionally, triggering a 120-second timeout and a misleading
error dump every time the install is skipped in headless context.

---

## Reproduction

1. Trigger `/acg-up` via Slack (k3dm-webhook context — no sudo cached, no TTY)
2. Observe Slack notification showing socat errors and `ERROR: [argocd] Argo CD
   did not become reachable` mid-run while the job keeps going

---

## Fix

### Change 1 — `bin/acg-up`: set skip flag, check before wait

**Exact old block (install if-chain):**

```bash
    if [[ -f "${_argocd_browser_plist}" ]] && diff -q "${_argocd_browser_plist_tmp}" "${_argocd_browser_plist}" >/dev/null 2>&1; then
      rm -f "${_argocd_browser_plist_tmp}"
      _info "[acg-up] ArgoCD browser HTTPS listener LaunchDaemon unchanged — skipping reinstall"
    elif ! sudo -n true 2>/dev/null && ! [[ -t 0 && -t 1 ]]; then
      rm -f "${_argocd_browser_plist_tmp}"
      _warn "[acg-up] no sudo in headless context — skipping argocd-browser-https LaunchDaemon install (run interactively or install /etc/sudoers.d/k3d-manager)"
    else
```

**Exact new block:**

```bash
    _argocd_browser_install_skipped=0
    if [[ -f "${_argocd_browser_plist}" ]] && diff -q "${_argocd_browser_plist_tmp}" "${_argocd_browser_plist}" >/dev/null 2>&1; then
      rm -f "${_argocd_browser_plist_tmp}"
      _info "[acg-up] ArgoCD browser HTTPS listener LaunchDaemon unchanged — skipping reinstall"
    elif ! sudo -n true 2>/dev/null && ! [[ -t 0 && -t 1 ]]; then
      rm -f "${_argocd_browser_plist_tmp}"
      _warn "[acg-up] no sudo in headless context — skipping argocd-browser-https LaunchDaemon install (run interactively or install /etc/sudoers.d/k3d-manager)"
      _argocd_browser_install_skipped=1
    else
```

**Exact old wait block:**

```bash
    if _argocd_wait_for_browser_https "${_argocd_browser_log}" "${ARGOCD_BROWSER_LISTENER_WAIT_TIMEOUT:-120}"; then
      _info "[acg-up] ArgoCD reachable at https://${ARGOCD_BROWSER_HOST:-argocd.shopping-cart.local} (launchd: ${_argocd_browser_label})"
      _argocd_browser_ready=1
    elif ! sudo -n true 2>/dev/null && ! [[ -t 0 && -t 1 ]]; then
      _warn "[acg-up] ArgoCD browser HTTPS listener not ready in headless context — continuing (Cloudflare tunnel path unaffected)"
      _argocd_browser_ready=0
    else
      exit 1
    fi
```

**Exact new wait block:**

```bash
    if [[ "${_argocd_browser_install_skipped}" -eq 1 ]]; then
      _warn "[acg-up] ArgoCD browser HTTPS listener not installed in headless context — skipping wait (Cloudflare tunnel path unaffected)"
      _argocd_browser_ready=0
    elif _argocd_wait_for_browser_https "${_argocd_browser_log}" "${ARGOCD_BROWSER_LISTENER_WAIT_TIMEOUT:-120}"; then
      _info "[acg-up] ArgoCD reachable at https://${ARGOCD_BROWSER_HOST:-argocd.shopping-cart.local} (launchd: ${_argocd_browser_label})"
      _argocd_browser_ready=1
    else
      exit 1
    fi
```
