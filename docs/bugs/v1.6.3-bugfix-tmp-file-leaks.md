# Bugfix: v1.6.3 — /tmp file leaks from install-sudoers, k3s-oci-storage, and session teardown

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/install-sudoers.sh`, `scripts/lib/providers/k3s-oci-storage.sh`, `bin/acg-down`, `bin/k3dm-cleanup`, `Makefile`

---

## Problem

Five categories of `/tmp` files accumulate and are never cleaned up:

1. **`/tmp/k3d-manager-sudoers.*`** — `bin/install-sudoers.sh` writes a temp file then calls `_run_command --interactive-sudo`. If sudo fails or is interrupted, `set -e` exits before the `rm -f` reaches. Two files are left even in `--dry-run` if `visudo` exits non-zero.

2. **`/tmp/k3s-etcd-*.db`** — `k3s-oci-storage.sh` downloads the etcd snapshot to `/tmp`, then calls `_oci_storage_upload`. If that upload call fails, `return 1` fires without the `rm -f` cleanup that follows on the happy path.

3. **`/tmp/k3d-manager-tunnel.{out,err}`** and **`/tmp/k3d-manager-acg-watch.{out,err}`** — log files for the SSH tunnel and acg-watch LaunchAgents, defined in their plist templates. `acg-down` tears down both LaunchAgents but doesn't remove the log files.

4. **`/tmp/k3dm-acg-screenshot-*.png`** — failure screenshots written by Playwright scripts. `k3dm-cleanup` only prunes `acg-extend-failure-*.png` in `~/.local/share/k3d-manager`; the `/tmp` copies are never touched.

5. **`/tmp/playwright-artifacts-*/`** — Playwright internal temp dirs created per browser session. Never removed after a crash.

**Out of scope (lib-acg subtree):** `gcp.sh` already has `rm -f "${creds_tmp}"` on all code paths; the leak only occurs on SIGKILL of the `node` child. Fix requires a subshell trap inside the function — defer to a lib-acg PR.

---

## Fix

### Change 1 — `bin/install-sudoers.sh`: add EXIT trap for `_tmpfile`

Add `_tmpfile=""` and a `trap` immediately after `source` so both `dry-run` and `install` modes clean up on any exit.

**Exact old block (lines 11–17):**

```bash
set -euo pipefail

REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
# shellcheck source=/dev/null
source "${REPO_ROOT}/scripts/lib/system.sh"

_SUDOERS_FILE="/etc/sudoers.d/k3d-manager"
```

**Exact new block:**

```bash
set -euo pipefail

REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
# shellcheck source=/dev/null
source "${REPO_ROOT}/scripts/lib/system.sh"

_tmpfile=""
trap 'rm -f "${_tmpfile:-}"' EXIT

_SUDOERS_FILE="/etc/sudoers.d/k3d-manager"
```

### Change 4 — `bin/k3dm-cleanup`: prune /tmp screenshots and playwright-artifacts

**Exact old block (lines 38–43):**

```bash
_prune_pngs "${K3DM_STATE_DIR}"
_prune_pngs "${K3DM_STATE_DIR}/logs"
_prune_chromium_cache "${K3DM_STATE_DIR}/profile"

```

**Exact new block:**

```bash
_prune_pngs "${K3DM_STATE_DIR}"
_prune_pngs "${K3DM_STATE_DIR}/logs"
_prune_chromium_cache "${K3DM_STATE_DIR}/profile"
_prune_chromium_cache "${K3DM_STATE_DIR}/playwright-auth"

# /tmp screenshots — keep most recent 5; delete older ones
_tmp_ss_count=$(ls -1t /tmp/k3dm-acg-screenshot-*.png 2>/dev/null | wc -l | tr -d ' ')
if [[ "${_tmp_ss_count}" -gt 5 ]]; then
  ls -1t /tmp/k3dm-acg-screenshot-*.png | tail -n "+6" | xargs rm -f
  echo "k3dm-cleanup: pruned $(( _tmp_ss_count - 5 )) screenshot(s) from /tmp"
fi

# Playwright artifact dirs — delete any older than 1 day
find /tmp -maxdepth 1 -name 'playwright-artifacts-*' -type d -mtime +1 -exec rm -rf {} + 2>/dev/null || true
echo "k3dm-cleanup: pruned stale playwright-artifacts dirs from /tmp"

echo "k3dm-cleanup: done"
```
