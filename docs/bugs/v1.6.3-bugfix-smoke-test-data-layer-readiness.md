# Bugfix: v1.6.3 — smoke test misses data-layer StatefulSet readiness

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/k3dm-webhook`

---

## Problem

`_smoke_test_services` checks HTTP endpoints (ArgoCD, Frontend, Keycloak, Prometheus,
Grafana) and ESO secret sync, but none of these depend on the shopping-cart data layer
(`postgresql-orders`, `postgresql-payment`, `postgresql-products`, `minio` on `ubuntu-k3s`).
While those StatefulSets are still coming up after `acg-up`, the smoke test reports all
green — giving a false healthy signal when the apps cannot yet serve requests.

**Root cause:** The smoke test has no check for data-layer pod readiness. The HTTP endpoints
it checks are served independently of the databases.

**Subprocess constraint:** `subprocess.run` inside `_smoke_test_services` triggers the macOS
NEF atfork SIGSEGV. The fix uses `_posix_spawn_capture` (already in the file) which calls
`os.posix_spawn` — safe after NEF is loaded.

---

Insert after the ESO block (after the `return results` line is NOT the place — insert before

line 1284).

**Exact old block (line 1284–1286):**

```python
        except Exception as _exc:

```

**Exact new block:**

```python
        except Exception as _exc:
            results.append(("ESO ExternalSecrets", False, str(_exc)[:200]))

    # Data-layer StatefulSet readiness (ubuntu-k3s) — uses posix_spawn, safe after NEF load
    _dl_names = ["postgresql-orders", "postgresql-payment", "postgresql-products", "minio"]
    _dl_ready = 0
    _dl_not_ready = []
    for _ss in _dl_names:
        _dl_out, _dl_timeout = _posix_spawn_capture(
            ["kubectl", "get", "statefulset", _ss,
             "-n", "shopping-cart-data", "--context", "ubuntu-k3s",
             "-o", "jsonpath={.status.readyReplicas}",
             "--request-timeout=5s"],
            timeout=8,
        )
        if not _dl_timeout and _dl_out.strip().isdigit() and int(_dl_out.strip()) >= 1:
            _dl_ready += 1
        else:
            _dl_not_ready.append(_ss)
    _dl_ok = _dl_ready == len(_dl_names)
    _dl_detail = (
        f"{_dl_ready}/{len(_dl_names)} ready"
        if _dl_ok
        else f"{len(_dl_not_ready)} not ready: {', '.join(_dl_not_ready[:2])}"
    )
    results.append(("Data layer", _dl_ok, _dl_detail))

    return results
```
