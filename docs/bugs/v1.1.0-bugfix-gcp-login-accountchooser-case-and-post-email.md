## Problem

### Part A — URL case mismatch in polling loop

The polling loop checks `page.url().includes('AccountChooser')` (capital C). Google's actual URL uses `accountchooser` (all lowercase): `accounts.google.com/v3/signin/accountchooser?...`. JavaScript `String.includes()` is case-sensitive, so the branch never fires. The loop burns 60s doing nothing, then times out.

Confirmed from `~/.local/share/k3d-manager/gcp_login_pw.log`:
```
URL before click: https://accounts.google.com/v3/signin/accountchooser?...
```

### Part B — No wait or check after email Next click

After `emailInput.fill(username)` + `nextBtn.click()`, the code immediately checks for a password input (5s). When Google recognises the email from session cookies it redirects to `accountchooser` rather than showing a password prompt — the redirect takes longer than 5s in some flows, so the password check returns false and we log "session already authenticated" incorrectly. There is no URL log or account chooser check after the email Next click to handle this redirect.

**Fix:** After `nextBtn.click()`, add a 2s wait, log the URL, then check for `div[data-identifier]` with username (post-email account chooser). If found, click it before proceeding to the password step.

## Fix

### Change 1 — polling loop: fix AccountChooser URL case check

**Exact old line:**

```js
      if (page.url().includes('AccountChooser')) {
```

**Exact new line:**

```js
      if (page.url().toLowerCase().includes('accountchooser')) {
```

---

### Change 2 — initial email flow: add post-Next wait, URL log, and account chooser check

**Exact old block:**

```js
      // Fill email — SPA transition; wait directly for input
      const emailInput = page.locator('input[type="email"], input#identifierId').first();
      await emailInput.waitFor({ timeout: 30000 });
      await emailInput.fill(username);
      console.error(`INFO: Filled email ${username}`);
      const nextBtn = page.locator('button:has-text("Next")').first();
      await nextBtn.click();
    }
```

**Exact new block:**

```js
      // Fill email — SPA transition; wait directly for input
      const emailInput = page.locator('input[type="email"], input#identifierId').first();
      await emailInput.waitFor({ timeout: 30000 });
      await emailInput.fill(username);
      console.error(`INFO: Filled email ${username}`);
      const nextBtn = page.locator('button:has-text("Next")').first();
      await nextBtn.click();
      await page.waitForTimeout(2000);
      console.error(`INFO: URL after email Next: ${page.url()}`);
      // Post-email account chooser — Google may redirect to accountchooser after recognising the email
      const postEmailChooser = page.locator('div[data-identifier]').filter({ hasText: username });
      const postEmailChooserVisible = await postEmailChooser.first().isVisible({ timeout: 3000 }).catch(() => false);
      if (postEmailChooserVisible) {
        console.error(`INFO: Post-email AccountChooser detected — clicking ${username}`);
        await postEmailChooser.first().click({ force: true });
        await page.waitForTimeout(2000);
        console.error(`INFO: URL after post-email chooser: ${page.url()}`);
      }
    }
```
