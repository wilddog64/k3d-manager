# Bugfix: v1.6.1 — /acg-status stale note breaks code fence rendering

**Branch:** `k3d-manager-v1.6.1`
**File:** `bin/k3dm-webhook`
**Discovered:** 2026-06-05
**Follows:** `docs/bugs/v1.6.1-bugfix-acg-status-stale-argocd-display.md`

---

## Problem

After the stale-caveat fix (commit `6651a192`), the `/acg-status` Slack output renders:

```
ACG ArgoCD apps: (cached — ACG cluster was unreachable)acg-kube-prometheus-stack   Synced   Healthy
```

The stale note is visible but runs directly into the first app row with no code fence.

---

## Root Cause

`_log` appends `msg + "\n"` and the final message is `"".join(lines)`. When
`_(cached — ACG cluster was unreachable)_` italic markup is on the **same line** as
the `*ACG ArgoCD apps:*` bold header, Slack's mrkdwn parser does not cleanly close the
inline formatting before the next ` ``` ` fence. As a result, the fence is not
interpreted as a code block and the app table renders as plain inline text.

---

## Fix

### Change 1 — `bin/k3dm-webhook` ~line 877: separate stale note into its own `_log` call

**Exact old block:**
```python
            if acg_apps:
                stale_note = " _(cached — ACG cluster was unreachable)_" if not acg_reachable else ""
                _log(f"\n*ACG ArgoCD apps:*{stale_note}")
                _log(f"```{chr(10).join(acg_apps)}```")
```

**Exact new block:**
```python
            if acg_apps:
                _log(f"\n*ACG ArgoCD apps:*")
                if not acg_reachable:
                    _log("_(cached — ACG cluster was unreachable)_")
                _log(f"```{chr(10).join(acg_apps)}```")
```
