## Problem

After clicking "Use another account", `gcp_login.js` fails to find `input[type="email"]`.

Two prior fixes both failed for the same reason: they assumed clicking "Use another account" triggers a Playwright navigation event. It does not. Google's account chooser is a SPA â€” the transition re-renders the form in-place with no navigation event. `waitForLoadState` resolves immediately (page already loaded), and `waitForNavigation` times out (no event fires).

**Root cause:** The email input appears via a SPA re-render, not a navigation. The fix is to click and then wait directly for the email input with a generous timeout. No navigation guard is needed or possible.

---

```
make up CLUSTER_PROVIDER=k3s-gcp
# prior fix produced:
# INFO: Clicking "Use another account"
# ERROR: page.waitForNavigation: Timeout 20000ms exceeded.
# waiting for navigation until "domcontentloaded"
```

## Fix

### Change 1 â€” `scripts/playwright/gcp_login.js`: remove navigation guard; wait directly for email input

**Exact old block (lines 64â€“72):**

```js
      if (useAnotherVisible) {
        console.error('INFO: Clicking "Use another account"');
        await Promise.all([
          page.waitForNavigation({ waitUntil: 'domcontentloaded', timeout: 20000 }),
          useAnother.click(),
        ]);
      }
      // Fill email
      const emailInput = page.locator('input[type="email"]').first();
      await emailInput.waitFor({ timeout: 20000 });
```

**Exact new block:**

```js
      if (useAnotherVisible) {
        console.error('INFO: Clicking "Use another account"');
        await useAnother.click();
      }
      // Fill email â€” SPA transition, no navigation event; wait directly for input
      const emailInput = page.locator('input[type="email"], input#identifierId').first();
      await emailInput.waitFor({ timeout: 30000 });
```
