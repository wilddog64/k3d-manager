# Bugfix: v1.6.5 — Codex repeating same answer on thread follow-up

**Branch:** `k3d-manager-v1.6.5`
**Files:** `bin/k3dm-webhook`

---

## Problem

When a user sends a second `/codex <question>` (or `codex <question>`) reply into a Slack
thread, codex echoes the answer it gave in the previous reply instead of answering the new
question.

**Root cause:** `_handle_thread_command` creates an ask sub-job and writes the original
`thread_ts` into `new_job_dir / "thread_ts"` (lines 738–739). On the next thread reply,
`_find_job_by_thread_ts` iterates `webhook-jobs/*/thread_ts` files and may return this ask
sub-job instead of the original anchor/up job. That causes lines 715–728 to inject the ask
sub-job's output file (the previous codex answer) as "job output context", and codex then
echoes it as its answer.

---

## Reproduction

1. In Slack, type `/acg-up` — an up job starts and posts updates in a thread.
2. Reply in that thread: `codex how does the shopping-cart deploy work?`
   → codex gives a real answer (e.g., "It uses ArgoCD ApplicationSets…").
3. Reply again: `codex what is the Vault PKI role TTL?`
   → codex repeats the SAME answer from step 2 instead of answering the new question.

## Fix

### Change 1 — `bin/k3dm-webhook` lines 735–739: mark ask sub-jobs with `action=ask`

**Exact old block (lines 734–739):**

```python
        new_job_id = str(uuid.uuid4())[:8]
        new_job_dir = JOB_DIR / new_job_id
        new_job_dir.mkdir(parents=True, exist_ok=True)
        (new_job_dir / "status").write_text("queued")
        if thread_ts:

```

**Exact new block:**

```python
        new_job_id = str(uuid.uuid4())[:8]
        new_job_dir = JOB_DIR / new_job_id
        new_job_dir.mkdir(parents=True, exist_ok=True)
        (new_job_dir / "status").write_text("queued")
        (new_job_dir / "action").write_text("ask")
        if thread_ts:
            (new_job_dir / "thread_ts").write_text(thread_ts)
```

### Change 2 — `bin/k3dm-webhook` lines 530–540: skip ask sub-jobs in `_find_job_by_thread_ts`

**Exact old block (lines 530–540):**

```python
def _find_job_by_thread_ts(thread_ts):
    """Return job_id whose thread_ts file matches, or None."""
    if not JOB_DIR.exists():
        return None
    for ts_file in JOB_DIR.glob("*/thread_ts"):
        try:
            if ts_file.read_text().strip() == thread_ts:
                return ts_file.parent.name
        except OSError:
            pass
    return None
```

**Exact new block:**

```python
def _find_job_by_thread_ts(thread_ts):
    """Return job_id whose thread_ts file matches, or None. Skips ask sub-jobs."""
    if not JOB_DIR.exists():
        return None
    for ts_file in JOB_DIR.glob("*/thread_ts"):
        try:
            action_file = ts_file.parent / "action"
            if action_file.exists() and action_file.read_text().strip() == "ask":
                continue
            if ts_file.read_text().strip() == thread_ts:
                return ts_file.parent.name
        except OSError:
            pass
    return None
```
