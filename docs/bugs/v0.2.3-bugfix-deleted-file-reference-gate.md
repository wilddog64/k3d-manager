# Bugfix: v0.2.3 — pre-commit gate for deleted files still referenced in code

**Branch:** `fix/deleted-file-reference-gate`
**Files:** `.git/hooks/pre-commit` (the installed hook), `scripts/hooks/pre-commit` (source)

---

## Problem

The v0.2.0 fix deleted `scripts/etc/acg-cluster.yaml` from lib-acg without updating
the reference in `scripts/plugins/acg.sh` (line 135):

```bash
--template-file "${_LIB_ACG_ROOT}/scripts/etc/acg-cluster.yaml" \
```

The pre-commit hook did not catch this. `make up` failed with `Invalid template path`
only after the file was removed in the next subtree pull into k3d-manager.

**Root cause:** No gate exists to detect when a file staged for deletion is still
referenced by name in other `.sh` or `.js` files in the repo.

---

## Fix

### Change 1 — Add dangling-reference check to `scripts/hooks/pre-commit`

Insert this block **after** the `node --check` block and **before** the final line:

**Exact new block (append before `pre-commit` at end of file):**

```bash
# Dangling reference check: fail if a deleted file is still referenced by name
# in any .sh or .js file that remains in the index after this commit.
while IFS= read -r -d '' _deleted; do
  _base="$(basename "$_deleted")"
  _refs="$(git grep --cached -l "$_base" -- '*.sh' '*.js' 2>/dev/null || true)"
  if [[ -n "$_refs" ]]; then
    printf 'Pre-commit: deleting %s but it is still referenced in:\n' "$_deleted" >&2
    printf '  %s\n' $_refs >&2
    exit 1
  fi
done < <(git diff --cached --name-only -z --diff-filter=D)
```

**Why `git grep --cached`:** Searches the staged index (what will be committed), not
the working tree. Files staged for deletion are absent from the index, so results
only include files that survive the commit — exactly the set we need to check.
