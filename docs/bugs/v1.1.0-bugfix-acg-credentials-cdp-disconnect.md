# Bugfix: v1.1.0 — acg_credentials.js uses close() instead of disconnect() for CDP sessions

**Branch:** `k3d-manager-v1.1.0`
**Files:** `scripts/playwright/acg_credentials.js`

3. Read `scripts/playwright/acg_credentials.js` lines 162–165 and lines 431–434

---

## Problem

When `acg_credentials.js` connects to the user's running Chrome via CDP
(`chromium.connectOverCDP`), the `_cdpBrowser` variable holds a Playwright
`Browser` object that wraps the entire Chrome process — not a launched child.

Two places call `_cdpBrowser.close()` on this object:

1. **Line 163** — inside the CDP setup block, when no browser contexts are found
   (fallback path before `launchPersistentContext`).
2. **Line 433** — inside the `finally` block that runs after every extraction attempt.

In Playwright, calling `.close()` on a CDP-connected browser attempts to terminate
the entire Chrome process. On macOS, if other windows are open, this hangs for the
full Playwright timeout (up to 300s). It also destroys the user's active browser
session.

The correct call for CDP sessions is `.disconnect()`, which cleanly severs the
Playwright connection without touching the Chrome process.

---

## Fix

### Change 1 — CDP setup fallback (line 163)

**Exact old line:**

```js
        try { await _cdpBrowser.close(); } catch {}
```

**Exact new line:**

```js
        try { await _cdpBrowser.disconnect(); } catch {}
```

*(This is the `if (!browserContext)` fallback inside the inner try — when CDP
connected but found no contexts. Replace only this occurrence.)*

### Change 2 — finally block (line 433)

**Exact old line:**

```js
      try { await _cdpBrowser.close(); } catch {}
```

**Exact new line:**

```js
      try { await _cdpBrowser.disconnect(); } catch {}
```

*(This is inside the `finally` block's `if (_cdpBrowser)` branch.)*
