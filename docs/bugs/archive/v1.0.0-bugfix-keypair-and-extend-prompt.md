## Problems

### Bug 1 — `aws ec2 import-key-pair` prints noisy error on re-run

`_acg_provision_stack` calls `_run_command -- aws ec2 import-key-pair ... || true`.
The `|| true` prevents termination but `_run_command` without `--soft` still prints
its own error message when the key already exists. This caused Gemini to misdiagnose
the failure as fatal during the v1.0.0 e2e smoke test (run 2).

Fix: replace `_run_command --` with `_run_command --soft --` and drop `|| true`.
`--soft` makes the call non-fatal AND suppresses the error message. Behaviour is
identical but silent on duplicate key.

### Bug 2 — `antigravity_acg_extend` Gemini prompt gives up on 404 page

The prompt says "Navigate to ${sandbox_url}" but does not instruct the model to call
`page.goto()` unconditionally. When the browser is on a stale/404 page, the Gemini-
generated Playwright script treats the 404 as a navigation failure and aborts instead
of navigating away from it.

Fix: make step 3 explicit — "Call `await page.goto('${sandbox_url}', {waitUntil: 'networkidle'})` unconditionally, regardless of the current page state."

### Change 1 — `scripts/plugins/acg.sh`: keypair import idempotency

**Exact old block (lines 159–160):**

```bash
  _run_command -- aws ec2 import-key-pair --region "${ACG_REGION}" --key-name "${_ACG_KEY_NAME}" \
    --public-key-material "fileb://${_ACG_KEY_PEM%.pem}.pub" >/dev/null 2>&1 || true
```

```bash

    --public-key-material "fileb://${_ACG_KEY_PEM%.pem}.pub" >/dev/null 2>&1
```

### Change 2 — `scripts/plugins/antigravity.sh`: force navigation in extend prompt

**Exact old block (lines 221–232):**

```bash
  gemini_prompt="You are a browser automation agent. Use Playwright (Node.js) to do the following:

1. Connect to the running Antigravity browser via CDP: const browser = await chromium.connectOverCDP('http://localhost:9222');
2. Use the first browser context and page (do NOT launch a new browser).
3. Navigate to ${sandbox_url}
4. Find the sandbox TTL extend button (look for 'Extend', '+4 hours', or similar).
5. Click to extend by ${hours} hours.
6. Confirm the new TTL is shown on the page.
7. Print the new sandbox expiry time.

Write the Playwright script to ${HOME}/.gemini/tmp/k3d-manager/ag_acg_extend.js, execute with node, print the result.
Exit code 1 if the extend button is not found or the action fails."
```

**Exact new block:**

```bash
  gemini_prompt="You are a browser automation agent. Use Playwright (Node.js) to do the following:

1. Connect to the running Antigravity browser via CDP: const browser = await chromium.connectOverCDP('http://localhost:9222');
2. Use the first browser context and page (do NOT launch a new browser).
3. Call \`await page.goto('${sandbox_url}', {waitUntil: 'networkidle'})\` unconditionally — regardless of the current page state, always navigate.
4. Find the sandbox TTL extend button (look for 'Extend', '+4 hours', or similar).
5. Click to extend by ${hours} hours.
6. Confirm the new TTL is shown on the page.
7. Print the new sandbox expiry time.

Write the Playwright script to ${HOME}/.gemini/tmp/k3d-manager/ag_acg_extend.js, execute with node, print the result.
Exit code 1 if the extend button is not found or the action fails."
```
