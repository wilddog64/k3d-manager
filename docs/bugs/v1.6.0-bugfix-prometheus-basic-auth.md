# Bugfix: v1.6.0 â€” Prometheus exposed without authentication via Cloudflare tunnel

**Branch:** `k3d-manager-v1.6.0`
**Files:** `scripts/plugins/observability.sh`, `scripts/etc/helm/observability/kube-prometheus-stack-acg-values.yaml`

---

## Problem

`prometheus.3ai-talk.org` is reachable by anyone without credentials. The
kube-prometheus-stack Helm values do not configure a web config file, so
the Prometheus HTTP server runs with no authentication. The Cloudflare tunnel
routes directly to the Prometheus NodePort with no intermediate auth layer.

**Root cause:** `kube-prometheus-stack-acg-values.yaml` is missing
`prometheus.prometheusSpec.additionalArgs` and `secrets` entries that would
enable the Prometheus native basic-auth web config.

---

## Reproduction

1. Open `https://prometheus.3ai-talk.org` in any browser (no credentials required)
2. Prometheus UI loads unauthenticated

---

## Fix

### Prerequisite â€” store bcrypt hash in Vault

Before running `acg-up`, store the password in Vault:

```bash
# Generate bcrypt hash (requires htpasswd from apache2-utils or httpd-tools)
PROM_HASH=$(htpasswd -bnBC 12 '' '<your-password>' | tr -d ':\n')

vault kv put secret/k3d-manager/prometheus-basic-auth \
  user=admin \
  password_bcrypt="${PROM_HASH}"
```

### Change 1 â€” `scripts/plugins/observability.sh`: add helper + call from `deploy_observability_acg`

**Add the helper function after the closing `}` of `deploy_observability_acg` (after line 141):**

```bash
function _prometheus_acg_web_config_secret() {
  local _vault_addr="http://127.0.0.1:18200"
  local _vault_token
  _vault_token=$(_kubectl get secret vault-root -n secrets \
    --context k3d-k3d-cluster -o jsonpath='{.data.root_token}' | base64 -d)

  local _vault_hdr
  _vault_hdr=$(mktemp)
  printf 'X-Vault-Token: %s\n' "${_vault_token}" > "${_vault_hdr}"

  local _prom_creds
  if ! _prom_creds=$(curl -sf \
      --header "@${_vault_hdr}" \
      "${_vault_addr}/v1/secret/data/k3d-manager/prometheus-basic-auth" 2>/dev/null \
      | python3 -c "import json,sys; d=json.load(sys.stdin)['data']['data']; \
        print(d['user']+'|'+d['password_bcrypt'])" 2>/dev/null); then
    _prom_creds=""
  fi
  rm -f "${_vault_hdr}"

  if [[ -z "${_prom_creds}" ]]; then
    _warn "[observability] Prometheus basic auth Vault secret not found â€” Prometheus is unauthenticated"
    _warn "[observability] Store hash: vault kv put secret/k3d-manager/prometheus-basic-auth user=admin password_bcrypt='<bcrypt>'"
    return 0
  fi

  local _prom_user _prom_hash
  _prom_user="${_prom_creds%%|*}"
  _prom_hash="${_prom_creds#*|}"

  local _web_config _tmpfile
  _web_config=$(printf 'basic_auth_users:\n  %s: %s\n' "${_prom_user}" "${_prom_hash}")
  _tmpfile=$(mktemp)
  printf '%s' "${_web_config}" > "${_tmpfile}"
  _kubectl create secret generic prometheus-web-config \
    --context ubuntu-k3s \
    -n monitoring \
    --from-file=web.yml="${_tmpfile}" \
    --dry-run=client -o yaml | _kubectl apply --context ubuntu-k3s -f -
  rm -f "${_tmpfile}"
  _info "[observability] Prometheus basic auth secret applied (monitoring/prometheus-web-config)"
}
```
