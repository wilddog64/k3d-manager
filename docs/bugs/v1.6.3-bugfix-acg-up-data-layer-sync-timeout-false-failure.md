# Bugfix: v1.6.3 — `acg-up` reports failure when data-layer sync times out but pods are healthy

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/acg-up`

---

## Problem

When the data-layer ArgoCD sync times out (300s initial + 180s retry), `acg-up` calls
`_err` and exits with code 1. The webhook's failure analysis fires and posts an
`APP_FAILURE / TRANSIENT` message to Slack, giving the impression the deployment is broken.

In practice the data-layer StatefulSets (`postgresql-orders`, `postgresql-payment`,
`postgresql-products`, `minio`) are actually running — ArgoCD's sync status just didn't
update in time. `make status` subsequently reports everything healthy.

**Root cause:** The hard `_err + exit 1` at the end of the 180s retry loop (lines 657–658)
fires on sync timeout without checking whether the actual pods are running. A slow ArgoCD
controller is not a deployment failure.

---

## Fix

### Change 1 — `bin/acg-up`: check pod readiness before erroring on sync timeout

Add a StatefulSet readiness check at the timeout site. If all 4 pods are Ready, demote
to `_warn` + write checkpoint + continue. Only call `_err + exit 1` if pods are also not ready.

**Exact old block (lines 653–662):**

```bash
        _dl_retry_deadline=$(( $(date +%s) + 180 ))
        until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
            -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
          if [[ $(date +%s) -ge ${_dl_retry_deadline} ]]; then
            _err "[acg-up] data-layer ArgoCD Application did not reach Synced after force-sync + 180s retry — check: kubectl get application data-layer -n cicd --context k3d-k3d-cluster"
            exit 1
          fi
          _info "[acg-up] data-layer not yet Synced after force-sync — waiting..."
          sleep 10
        done
        break
```

**Exact new block:**

```bash
        _dl_retry_deadline=$(( $(date +%s) + 180 ))
        until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
            -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
          if [[ $(date +%s) -ge ${_dl_retry_deadline} ]]; then
            _dl_pods_ready=0
            for _ss in postgresql-orders postgresql-payment postgresql-products minio; do
              _rr=$(kubectl get statefulset/"${_ss}" -n shopping-cart-data --context ubuntu-k3s \
                -o jsonpath='{.status.readyReplicas}' 2>/dev/null || echo "0")
              [[ "${_rr:-0}" -ge 1 ]] && (( _dl_pods_ready++ )) || true
            done
            if [[ ${_dl_pods_ready} -eq 4 ]]; then
              _warn "[acg-up] data-layer ArgoCD sync timed out but all 4 StatefulSets are Ready — writing checkpoint and continuing"
              break
            fi
            _err "[acg-up] data-layer ArgoCD Application did not reach Synced after force-sync + 180s retry — check: kubectl get application data-layer -n cicd --context k3d-k3d-cluster"
            exit 1
          fi
          _info "[acg-up] data-layer not yet Synced after force-sync — waiting..."
          sleep 10
        done
        break
```
