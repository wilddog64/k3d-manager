# Bugfix: v1.6.0 — failure analysis missing SSH tunnel check

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/k3dm-webhook`

---

## Problem

`_collect_cluster_state()` runs `kubectl --context ubuntu-k3s` without first verifying
the SSH tunnel is up. When the tunnel is down, kubectl times out and the function
reports `UNREACHABLE`. Gemini then sees "UNREACHABLE" alongside "ArgoCD sync timed out"
in the job log and concludes `APP_FAILURE / PERMANENT` — the wrong verdict. The real
failure is the tunnel being down, which would resolve with a re-run.

**Root cause:** `_collect_cluster_state()` has no tunnel health check; the probe
(`curl -sf --max-time 3 http://localhost:6443`) exists in `_run_post_provision_check`
and `_run_cluster_status` but was never added to `_collect_cluster_state`.

---

## Reproduction

1. Bring up the ACG cluster, then kill the SSH tunnel.
2. Trigger a webhook `acg-up` job that fails during the app-deploy phase.
3. Observe the Slack failure analysis: it reports `APP_FAILURE / PERMANENT` and
   recommends inspecting ArgoCD, when the actual fix is just re-running acg-up.

---

## Fix

### Change 1 — `bin/k3dm-webhook`: add tunnel check as first section in `_collect_cluster_state`

**Exact old block:**
```python
def _collect_cluster_state():
    """Collect live cluster state to enrich failure analysis. Returns a string or empty string."""
    sections = []

    try:
        result = subprocess.run(
            ["kubectl", "get", "nodes", "--context", "ubuntu-k3s", "--no-headers"],
            capture_output=True, text=True, timeout=15,
        )
        if result.returncode == 0 and result.stdout.strip():
            sections.append("Remote cluster nodes (ubuntu-k3s):\n" + result.stdout.strip())
        else:
            sections.append("Remote cluster nodes (ubuntu-k3s): UNREACHABLE — cluster may not be provisioned")
    except Exception:
        sections.append("Remote cluster nodes (ubuntu-k3s): UNREACHABLE — cluster may not be provisioned")
```

**Exact new block:**
```python
def _collect_cluster_state():
    """Collect live cluster state to enrich failure analysis. Returns a string or empty string."""
    sections = []

    tunnel = subprocess.run(
        ["curl", "-sf", "--max-time", "3", "http://localhost:6443"],
        capture_output=True,
    )
    tunnel_ok = tunnel.returncode == 0 or tunnel.returncode == 22
    sections.append(f"SSH tunnel (localhost:6443): {'UP' if tunnel_ok else 'DOWN — remote kubectl calls will be unreachable'}")

    if not tunnel_ok:
        return "\n\n".join(sections)

    try:
        result = subprocess.run(
            ["kubectl", "get", "nodes", "--context", "ubuntu-k3s", "--no-headers"],
            capture_output=True, text=True, timeout=15,
        )
        if result.returncode == 0 and result.stdout.strip():
            sections.append("Remote cluster nodes (ubuntu-k3s):\n" + result.stdout.strip())
        else:
            sections.append("Remote cluster nodes (ubuntu-k3s): UNREACHABLE — cluster may not be provisioned")
    except Exception:
        sections.append("Remote cluster nodes (ubuntu-k3s): UNREACHABLE — cluster may not be provisioned")
```
