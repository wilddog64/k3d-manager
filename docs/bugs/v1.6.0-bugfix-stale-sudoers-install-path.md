# Bugfix: v1.6.0 — Stale sudoers install path causes acg-up to fail

**Branch:** `k3d-manager-v1.6.0`
**Files:** none (operational fix — re-run `bin/install-sudoers.sh`)

---

## Problem

`acg-up` fails with `make: *** [up] Error 1` when installing the ArgoCD browser HTTPS
LaunchDaemon plist into `/Library/LaunchDaemons/`.

Visible output:
```
Boot-out failed: 1: Operation not permitted
WARN: [acg-up] existing ArgoCD browser HTTPS listener was not loaded; continuing
make: *** [up] Error 1
```

The `Boot-out` failure is a red herring (handled as WARN, continues). The real failure is
`install -m 644 <plist_tmp> /Library/LaunchDaemons/com.k3d-manager.argocd-browser-https.plist`
at `bin/acg-up:437` — sudo refuses because the binary path does not match the sudoers rule.

**Root cause:** the deployed `/etc/sudoers.d/k3d-manager` was generated by an older version
of `bin/install-sudoers.sh` that used the Homebrew coreutils path:

```
K3DM_INSTALL = /opt/homebrew/opt/coreutils/libexec/gnubin/install
```

The current `bin/install-sudoers.sh` (line 42) uses `/usr/bin/install` — the system binary
that is active in PATH in the webhook process. The deployed file was never refreshed after
this change, so `sudo install …` silently fails (path mismatch → sudo denies NOPASSWD).

---

## Reproduction

```bash
# Trigger acg-up via Slack webhook → make: *** [up] Error 1
# Confirm deployed rule uses old path:
sudo cat /etc/sudoers.d/k3d-manager | grep install
# Shows: /opt/homebrew/opt/coreutils/libexec/gnubin/install  ← stale
```

---

## Fix

No code change required — `bin/install-sudoers.sh` already has the correct path.
Re-deploy the sudoers file:

```bash
sudo bin/install-sudoers.sh
```

Verify the deployed rule now matches:
```bash
sudo cat /etc/sudoers.d/k3d-manager | grep install
# Should show: /usr/bin/install
```

---

## Files Changed

None. This is a one-time operational fix — re-running `install-sudoers.sh` updates
`/etc/sudoers.d/k3d-manager` in-place.

---

## Definition of Done

- [ ] `sudo cat /etc/sudoers.d/k3d-manager` shows `/usr/bin/install`, not the Homebrew path
- [ ] `acg-up` runs past the ArgoCD browser HTTPS listener install step without error
- [ ] No spec or code commit required

---

## Prevention

After any OS upgrade or Homebrew coreutils update: re-run `sudo bin/install-sudoers.sh`
to ensure the deployed sudoers matches the current script.

Consider adding a sudoers staleness check to `acg-up` pre-flight in a future milestone:
diff deployed file against `bin/install-sudoers.sh --dry-run` output and warn if they diverge.
