# Bugfix: v1.7.1 — `make refresh` cannot recover a dead argocd-browser-https listener

**Branch:** `k3d-manager-v1.7.1`
**Files:** `bin/cluster-refresh`

---

## Problem

The ArgoCD UI at `https://argocd.shopping-cart.local` becomes unreachable ("This site can't
be reached") even though the hub `argocd-server` pod is healthy. The local access path is a
launchd system daemon (`com.k3d-manager.argocd-browser-https`) running `socat`, which
terminates TLS on `127.0.0.1:443` with a Vault-PKI-issued cert and forwards plaintext to
`127.0.0.1:8080` (the argocd-server port-forward; argocd-server runs `--insecure`).

When the daemon's TLS material **or** its wrapper script goes missing, `socat` crash-loops:

```
[argocd-browser] starting HTTPS listener: 127.0.0.1:443 -> 127.0.0.1:8080
socat E SSL_CTX_use_certificate_file(): error:80000002:system library::No such file or directory
[argocd-browser] listener exited before healthz became reachable — restarting
```

…until launchd throttles and drops it (gone from `launchctl list`).

**`make refresh` — the daily-driver recovery command — cannot fix this:**

1. `_system_daemon_install_if_missing` short-circuits on `[[ -f "${plist_dst}" ]] && return 0`.
   The plist still exists, so it regenerates nothing.
2. `_launchd_ensure` then bootstraps that plist → launchd runs the wrapper at
   `${_ACG_STATE_DIR}/bin/argocd-browser-https.sh`, **which does not exist** (the only copy is a
   stale one at the share-dir root), and the cert dir `${_ACG_STATE_DIR}/argocd-browser-https-tls/`
   is **empty**.
3. `bin/cluster-refresh` does not `source scripts/plugins/argocd.sh`, so it has no access to
   `_argocd_issue_browser_tls_material` / `_argocd_write_browser_https_wrapper` — the functions
   that regenerate the cert and the wrapper. Those run **only** in `bin/cluster-up`.

**Root cause:** `bin/cluster-refresh` re-bootstraps the argocd-browser-https daemon without first
ensuring its TLS material and wrapper script exist. It can never self-heal a missing/expired cert
or a missing wrapper; it just re-bootstraps a dead plist. Recovery today requires a full
`make up`.

---

## Reproduction

```bash
# Delete the listener's TLS material (simulates expiry/cleanup):
rm -f ~/.local/share/k3d-manager/argocd-browser-https-tls/*

# Refresh — does NOT recover:
make refresh
curl -s -o /dev/null -w "%{http_code}\n" https://argocd.shopping-cart.local/   # -> 000

# Daemon is not running:
launchctl list | grep argocd-browser    # -> (nothing)
tail ~/.local/share/k3d-manager/logs/argocd-browser-https.log
#   -> SSL_CTX_use_certificate_file(): ... No such file or directory  (crash loop)
```

Expected after fix: `make refresh` re-issues the cert from Vault PKI, regenerates the wrapper at
the canonical path, and the daemon comes back Healthy (`curl ... /healthz -> 200`).

---

## Fix

Make `bin/cluster-refresh` self-heal the argocd-browser-https listener before bootstrapping it:
source the argocd plugin, then re-issue the TLS material and rewrite the wrapper if either is
missing — mirroring the proven sequence in `bin/cluster-up` (lines 510 and 523).

### Change 1 — `bin/cluster-refresh`: source the argocd plugin

**Exact old block (lines 23–26):**

```bash
source "${REPO_ROOT}/scripts/plugins/gemini.sh"
source "${REPO_ROOT}/scripts/plugins/aws.sh"
source "${REPO_ROOT}/scripts/plugins/acg.sh"
source "${REPO_ROOT}/scripts/plugins/tunnel.sh"
```

**Exact new block:**

```bash
source "${REPO_ROOT}/scripts/plugins/gemini.sh"
source "${REPO_ROOT}/scripts/plugins/aws.sh"
source "${REPO_ROOT}/scripts/plugins/acg.sh"
source "${REPO_ROOT}/scripts/plugins/tunnel.sh"
source "${REPO_ROOT}/scripts/plugins/argocd.sh"
```

### Change 2 — `bin/cluster-refresh`: regenerate cert + wrapper before bootstrapping

Insert a self-heal guard immediately after the
`_info "[acg-refresh] Checking launchd system daemons..."` line and before the
`_system_daemon_install_if_missing "com.k3d-manager.argocd-browser-https" \` call.

**Exact old block:**

```bash
  _info "[acg-refresh] Checking launchd system daemons..."

  _system_daemon_install_if_missing \
    "com.k3d-manager.argocd-browser-https" \
```

**Exact new block:**

```bash
  _info "[acg-refresh] Checking launchd system daemons..."

  # Self-heal the argocd-browser-https listener before (re)bootstrapping it. The daemon
  # socat-terminates TLS on :443 using a Vault-issued cert; if the cert material or the
  # wrapper script is missing, socat crash-loops until launchd drops it, and a plain
  # _launchd_ensure cannot recover it. Regenerate both here (mirrors bin/cluster-up).
  _argocd_browser_tls_dir="${ARGOCD_BROWSER_TLS_DIR:-${_ACG_STATE_DIR}/argocd-browser-https-tls}"
  _argocd_browser_tls_cert="${ARGOCD_BROWSER_TLS_CERT_FILE:-${_argocd_browser_tls_dir}/fullchain.crt}"
  _argocd_browser_tls_key="${ARGOCD_BROWSER_TLS_KEY_FILE:-${_argocd_browser_tls_dir}/tls.key}"
  _argocd_browser_wrapper="${ARGOCD_BROWSER_LISTENER_WRAPPER:-${_ACG_STATE_DIR}/bin/argocd-browser-https.sh}"
  _argocd_browser_log="${ARGOCD_BROWSER_LISTENER_LOG:-${_ACG_STATE_DIR}/logs/argocd-browser-https.log}"
  if [[ ! -s "${_argocd_browser_tls_cert}" || ! -s "${_argocd_browser_tls_key}" ]]; then
    _info "[acg-refresh] argocd-browser-https TLS material missing — re-issuing from Vault PKI..."
    _argocd_issue_browser_tls_material "${_argocd_browser_tls_dir}" \
      || _warn "[acg-refresh] failed to re-issue argocd-browser TLS material — listener will not start"
  fi
  if [[ ! -f "${_argocd_browser_wrapper}" ]]; then
    _info "[acg-refresh] argocd-browser-https wrapper missing — regenerating..."
    _argocd_write_browser_https_wrapper \
      "${_argocd_browser_wrapper}" "${_argocd_browser_log}" \
      "$(command -v socat)" "$(command -v curl)" \
      "127.0.0.1" "${ARGOCD_BROWSER_PORT:-443}" "127.0.0.1" "8080" \
      "${_argocd_browser_tls_cert}" "${_argocd_browser_tls_key}" \
      "https://127.0.0.1:${ARGOCD_BROWSER_PORT:-443}/healthz"
  fi

  _system_daemon_install_if_missing \
    "com.k3d-manager.argocd-browser-https" \
```
