# Bug: deploy_eso installs ESO Helm chart without a pinned version

**Branch:** `k3d-manager-v1.4.5`
**Severity:** Medium — unpinned Helm chart install silently upgrades ESO on the Hub cluster, risking CRD schema drift and broken ExternalSecret syncing

---

## Problem

`deploy_eso` (in `scripts/plugins/eso.sh`) calls `_eso_install_chart` without passing a
`--version` flag to `helm upgrade --install`. This violates the CLAUDE.md security rule:

> New Helm chart installations must pin chart versions explicitly — no floating `latest`.

The remote ubuntu-k3s cluster **does** pin the version — `acg-up` uses `--version ${_eso_version}`
which defaults to `1.0.0`. The k3d Hub cluster does not, so the two clusters can end up with
different ESO operator versions, different CRD schemas, and incompatible behaviour.

---

## Reproduction

```bash
grep -n "_eso_install_chart\|--version" scripts/plugins/eso.sh
# Actual: no --version flag in _eso_install_chart
grep -n "_eso_version" bin/acg-up
# Remote cluster: _eso_version="${ESO_VERSION:-1.0.0}" — pinned
```

On any `deploy_ldap --confirm` / `make up` on a fresh k3d cluster, ESO resolves to the
latest chart version in the Helm repo at that moment.

---

## Fix

### File: `scripts/plugins/eso.sh`

**Add `ESO_HELM_CHART_VERSION` default near the top of `deploy_eso`, then pass it
to `_eso_install_chart`.**

**Exact old lines (in `deploy_eso`, lines 125–128):**
```bash
  local helm_repo_url_default="https://charts.external-secrets.io"
  local helm_repo_url="${ESO_HELM_REPO_URL:-$helm_repo_url_default}"
  local helm_chart_ref_default="external-secrets/external-secrets"
  local helm_chart_ref="${ESO_HELM_CHART_REF:-$helm_chart_ref_default}"
```

**Exact new lines:**
```bash
  local helm_repo_url_default="https://charts.external-secrets.io"
  local helm_repo_url="${ESO_HELM_REPO_URL:-$helm_repo_url_default}"
  local helm_chart_ref_default="external-secrets/external-secrets"
  local helm_chart_ref="${ESO_HELM_CHART_REF:-$helm_chart_ref_default}"
  local helm_chart_version="${ESO_HELM_CHART_VERSION:-1.0.0}"
```

**Exact old line (in `_eso_install_chart`, the `_helm upgrade` call):**
```bash
  _helm upgrade --install -n "$ns" external-secrets "$helm_chart_ref" \
      --create-namespace \
      --set installCRDs=true
```

**Exact new lines:**
```bash
  _helm upgrade --install -n "$ns" external-secrets "$helm_chart_ref" \
      --create-namespace \
      --set installCRDs=true \
      ${helm_chart_version:+--version "$helm_chart_version"}
```

Wait — `helm_chart_version` is a local in `deploy_eso`, not in `_eso_install_chart`.
Pass it as a 4th argument instead:

**Revised — also update the `_eso_install_chart` signature and its call site:**

`_eso_install_chart` signature change:
```bash
# Old
function _eso_install_chart() {
  local ns="$1"
  local helm_chart_ref="$2"
  local helm_repo_url="$3"

# New
function _eso_install_chart() {
  local ns="$1"
  local helm_chart_ref="$2"
  local helm_repo_url="$3"
  local helm_chart_version="${4:-}"
```

`_helm upgrade` call change (inside `_eso_install_chart`):
```bash
# Old
  _helm upgrade --install -n "$ns" external-secrets "$helm_chart_ref" \
      --create-namespace \
      --set installCRDs=true

# New
  local -a version_arg=()
  [[ -n "$helm_chart_version" ]] && version_arg=(--version "$helm_chart_version")
  _helm upgrade --install -n "$ns" external-secrets "$helm_chart_ref" \
      --create-namespace \
      --set installCRDs=true \
      "${version_arg[@]}"
```

Call site change (inside `deploy_eso`):
```bash
# Old
  _eso_install_chart "$ns" "$helm_chart_ref" "$helm_repo_url"

# New
  _eso_install_chart "$ns" "$helm_chart_ref" "$helm_repo_url" "$helm_chart_version"
```
