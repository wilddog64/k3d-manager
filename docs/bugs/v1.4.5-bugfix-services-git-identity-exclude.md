# Bugfix: v1.4.5 — services-git ApplicationSet must exclude shopping-cart-identity

**Branch:** `k3d-manager-v1.4.5`
**Files:** `scripts/etc/argocd/applicationsets/services-git.yaml`

---

## Problem

The `services-git` ApplicationSet generator scans `services/*` without exclusions and
stamps every directory — including `services/shopping-cart-identity` — with destination
`ubuntu-k3s/shopping-cart-apps`. The identity stack targets the local cluster in namespace
`identity`, so every sync attempt fails with a project permission error and Keycloak never
deploys.

**Root cause:** `services-git.yaml` `spec.generators[0].git.directories` has only one
entry (`services/*`) with no `exclude: true` guard for `shopping-cart-identity`.

---

## Reproduction

```bash
# After make up, check the identity Application's destination and sync status:
kubectl get app shopping-cart-identity -n cicd \
  -o jsonpath='{.spec.destination} {.status.sync.status}' --context k3d-k3d-cluster
# Actual:   {"name":"ubuntu-k3s","namespace":"shopping-cart-apps"} OutOfSync
# Expected: {"server":"https://kubernetes.default.svc","namespace":"identity"} Synced
```

---

## Fix

### Change 1 — `scripts/etc/argocd/applicationsets/services-git.yaml` lines 14–15: add exclude entry

**Exact old block (lines 13–15):**

```yaml
        directories:
          - path: services/*
```

**Exact new block:**

```yaml
        directories:
          - path: services/*
          - path: services/shopping-cart-identity
            exclude: true
```
