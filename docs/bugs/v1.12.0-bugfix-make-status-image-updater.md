# Bugfix: `make status` is blind to ArgoCD Image Updater activity

## Context / Why

`make status` → `bin/cluster-status` already prints **ArgoCD Apps** and
**ApplicationSets**, but shows nothing about **ArgoCD Image Updater** — the
component that watches registries for new image tags and writes them back so
ArgoCD re-syncs. Today the only way to see whether image-updater is detecting
new images, performing upgrades, or stuck flapping is to hand-grep the pod logs
(`kubectl logs -n cicd deploy/argocd-image-updater`).

This was confirmed on the live hub cluster during the metrics-ServiceMonitor
validation (2026-06-29): `shopping-cart-product-catalog` is being updated on
nearly every 2-minute cycle (non-converging / flapping), and that signal is
completely invisible in `make status`.

This spec adds a read-only **ArgoCD Image Updater** section to
`bin/cluster-status`. It is **log/annotation based** (plain `kubectl`), NOT
Prometheus based — `make status` deliberately avoids port-forwards, and the
Grafana dashboard (commit `60cd500c`) already owns the time-series view. Two
different surfaces, each with the right tool.

**Classification:** observability bugfix in `docs/bugs/` (exempt from the
max-5-plan limit). Keeps the v1.12.0 plan-doc count unchanged.

**Find this exact text:**

```bash
if [[ "${CLUSTER_PROVIDER}" == "k3s-aws" ]]; then
```

**Replace with (section inserted before the `if`):**

```bash
kubectl get applicationsets.argoproj.io -A --context "${INFRA_CONTEXT}" 2>/dev/null \
  || echo "Cannot reach Hub cluster or ArgoCD CRDs not installed"

echo ""
echo "=== ArgoCD Image Updater ==="
_iu_ready="$(kubectl get deploy argocd-image-updater -n "${ARGOCD_NAMESPACE}" \
  --context "${INFRA_CONTEXT}" -o jsonpath='{.status.readyReplicas}/{.status.replicas}' 2>/dev/null || true)"
if [[ -z "${_iu_ready}" || "${_iu_ready}" == "/" ]]; then
  echo "Not installed (no argocd-image-updater deployment in ${ARGOCD_NAMESPACE})"
else
  echo "Deployment: ${_iu_ready} ready"

  _iu_logs="$(kubectl logs deploy/argocd-image-updater -n "${ARGOCD_NAMESPACE}" \
    --context "${INFRA_CONTEXT}" --tail=200 2>/dev/null || true)"

  _iu_cycle="$(printf '%s\n' "${_iu_logs}" | grep -F 'Processing results:' | tail -1 \
    | sed 's/.*Processing results: //; s/"$//' || true)"
  if [[ -n "${_iu_cycle}" ]]; then
    echo "Last cycle: ${_iu_cycle}"
  else
    echo "Last cycle: (no completed cycle in recent logs)"
  fi

  _iu_updates="$(printf '%s\n' "${_iu_logs}" | grep -c 'images_updated=[1-9]' || true)"
  if [[ "${_iu_updates}" -ge 3 ]]; then
    echo "WARN Flapping: ${_iu_updates} recent cycles wrote an update — an app may not be converging"
  fi

  echo "Watching:"
  _iu_watch="$(kubectl get applications.argoproj.io -n "${ARGOCD_NAMESPACE}" \
    --context "${INFRA_CONTEXT}" --no-headers \
    -o custom-columns='N:.metadata.name,IL:.metadata.annotations.argocd-image-updater\.argoproj\.io/image-list' 2>/dev/null \
    | awk '$2 != "<none>" && $2 != "" { printf "  %s -> %s\n", $1, $2 }' || true)"
  if [[ -n "${_iu_watch}" ]]; then
    printf '%s\n' "${_iu_watch}"
  else
    echo "  (no applications carry an image-list annotation)"
  fi
fi

if [[ "${CLUSTER_PROVIDER}" == "k3s-aws" ]]; then
```

Notes (do not add as comments to the script):
- `|| true` on every command substitution is required because `set -euo
  pipefail` + `pipefail` would otherwise abort on `grep` "no match" (exit 1).
- `grep -c` prints `0` and exits 1 when there is no match; `|| true` keeps the
  count string as `0` so the `-ge 3` test is safe.
- Use a plain ASCII `WARN` prefix (no emoji) to stay consistent with the rest of
  the file.

## Change 2 — `scripts/tests/bin/cluster_status_image_updater.bats` (new)

```bash
#!/usr/bin/env bats

@test "cluster-status surfaces ArgoCD Image Updater health and activity" {
  run grep -nF '=== ArgoCD Image Updater ===' bin/cluster-status
  [ "$status" -eq 0 ]

  run grep -nF 'deploy/argocd-image-updater' bin/cluster-status
  [ "$status" -eq 0 ]

  run grep -nF 'Processing results:' bin/cluster-status
  [ "$status" -eq 0 ]

  run grep -nF 'argocd-image-updater\.argoproj\.io/image-list' bin/cluster-status
  [ "$status" -eq 0 ]

  run grep -nF 'Flapping' bin/cluster-status
  [ "$status" -eq 0 ]
}
```
