# Bugfix: v1.6.3 — data-layer step 10b re-runs sync wait when StatefulSets already ready

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/acg-up`

---

## Problem

On every `acg-resume` after a partial failure, step 10b re-runs the full ArgoCD sync wait
(300s + force-sync + 180s retry) even when all data-layer StatefulSets are already running
and healthy on `ubuntu-k3s`. This happens because `_cp_write "step-10b-data-layer"` is
only written at the very end of the block — any failure before that point leaves the
checkpoint unset, so `_cp_skip` never fires on resume.

**Root cause:** The checkpoint guard (`_cp_skip`) is a binary written/not-written flag. When
the block partially succeeds (StatefulSets running, PostSync hooks incomplete), the
checkpoint is not written. `acg-resume` re-enters the full sync loop, which stalls on
`status.sync.status != Synced` because ArgoCD's sync status does not always reflect pod
health (Healthy ≠ Synced).

---

## Reproduction

```
bin/acg-up   # fails at or after step 10b StatefulSets ready, before _cp_write
bin/acg-resume
# → spins in "data-layer not yet Synced" for 300s even though pods are running
# make status shows all services green the entire time
```

---

## Fix

### Change 1 — `bin/acg-up`: add StatefulSet readiness pre-check inside the `else` block

At the start of the `else` body, check if all four data-layer StatefulSets
(`postgresql-orders`, `postgresql-payment`, `postgresql-products`, `minio`) already have
`readyReplicas >= 1` on `ubuntu-k3s`. If all four are ready, write the checkpoint and skip
the entire sync wait + PostSync force-sync.

**Exact old block (lines 624–668):**

```bash
_info "[acg-up] Step 10b/14 — Deploying shopping-cart data layer (PostgreSQL, Redis, RabbitMQ)..."
if _cp_skip "step-10b-data-layer"; then
  _info "[acg-up] Step 10b/14 — data layer already deployed (checkpoint), skipping"
else
  _info "[acg-up] Applying data-layer ArgoCD Application so sync starts before wait loop..."
  _run_command -- kubectl apply --context k3d-k3d-cluster \
    -f "${REPO_ROOT}/../shopping-carts/shopping-cart-infra/argocd/applications/data-layer.yaml"
  _info "[acg-up] Waiting for ArgoCD to sync data-layer Application (max 300s)..."
  _dl_sync_deadline=$(( $(date +%s) + 300 ))
  until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
      -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
    if [[ $(date +%s) -ge ${_dl_sync_deadline} ]]; then
      _warn "[acg-up] data-layer did not reach Synced within 300s — force-syncing and retrying (one attempt)..."
      argocd app sync data-layer \
        --server localhost:8080 --insecure \
        --force \
        --timeout 180 \
        2>/dev/null \
        || true
      _dl_retry_deadline=$(( $(date +%s) + 180 ))
      until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
          -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
        if [[ $(date +%s) -ge ${_dl_retry_deadline} ]]; then
          _err "[acg-up] data-layer ArgoCD Application did not reach Synced after force-sync + 180s retry — check: kubectl get application data-layer -n cicd --context k3d-k3d-cluster"
          exit 1
        fi
        _info "[acg-up] data-layer not yet Synced after force-sync — waiting..."
        sleep 10
      done
      break
    fi
    _info "[acg-up] data-layer not yet Synced — waiting..."
    sleep 10
  done
  _info "[acg-up] data-layer Application Synced — proceeding to StatefulSet readiness check"
  deploy_shopping_cart_data
  _info "[acg-up] Force-syncing data-layer to trigger PostSync hooks (MinIO bucket init + image upload)..."
  argocd app sync data-layer \
    --server localhost:8080 --insecure \
    --force \
    --timeout 180 \
    2>/dev/null \
    || _info "[acg-up] data-layer force-sync returned non-zero — PostSync hooks may still complete"
  _cp_write "step-10b-data-layer"
fi
```

**Exact new block:**

```bash
_info "[acg-up] Step 10b/14 — Deploying shopping-cart data layer (PostgreSQL, Redis, RabbitMQ)..."
if _cp_skip "step-10b-data-layer"; then
  _info "[acg-up] Step 10b/14 — data layer already deployed (checkpoint), skipping"
else
  _dl_ready=0
  for _ss in postgresql-orders postgresql-payment postgresql-products minio; do
    _rr=$(kubectl get statefulset/"${_ss}" -n shopping-cart-data --context ubuntu-k3s \
      -o jsonpath='{.status.readyReplicas}' 2>/dev/null || echo "0")
    [[ "${_rr:-0}" -ge 1 ]] && (( _dl_ready++ )) || true
  done
  if [[ ${_dl_ready} -eq 4 ]]; then
    _info "[acg-up] data-layer StatefulSets already ready on ubuntu-k3s — writing checkpoint and skipping sync wait"
    _cp_write "step-10b-data-layer"
  else
    _info "[acg-up] Applying data-layer ArgoCD Application so sync starts before wait loop..."
    _run_command -- kubectl apply --context k3d-k3d-cluster \
      -f "${REPO_ROOT}/../shopping-carts/shopping-cart-infra/argocd/applications/data-layer.yaml"
    _info "[acg-up] Waiting for ArgoCD to sync data-layer Application (max 300s)..."
    _dl_sync_deadline=$(( $(date +%s) + 300 ))
    until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
        -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
      if [[ $(date +%s) -ge ${_dl_sync_deadline} ]]; then
        _warn "[acg-up] data-layer did not reach Synced within 300s — force-syncing and retrying (one attempt)..."
        argocd app sync data-layer \
          --server localhost:8080 --insecure \
          --force \
          --timeout 180 \
          2>/dev/null \
          || true
        _dl_retry_deadline=$(( $(date +%s) + 180 ))
        until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
            -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
          if [[ $(date +%s) -ge ${_dl_retry_deadline} ]]; then
            _err "[acg-up] data-layer ArgoCD Application did not reach Synced after force-sync + 180s retry — check: kubectl get application data-layer -n cicd --context k3d-k3d-cluster"
            exit 1
          fi
          _info "[acg-up] data-layer not yet Synced after force-sync — waiting..."
          sleep 10
        done
        break
      fi
      _info "[acg-up] data-layer not yet Synced — waiting..."
      sleep 10
    done
    _info "[acg-up] data-layer Application Synced — proceeding to StatefulSet readiness check"
    deploy_shopping_cart_data
    _info "[acg-up] Force-syncing data-layer to trigger PostSync hooks (MinIO bucket init + image upload)..."
    argocd app sync data-layer \
      --server localhost:8080 --insecure \
      --force \
      --timeout 180 \
      2>/dev/null \
      || _info "[acg-up] data-layer force-sync returned non-zero — PostSync hooks may still complete"
    _cp_write "step-10b-data-layer"
  fi
fi
```
