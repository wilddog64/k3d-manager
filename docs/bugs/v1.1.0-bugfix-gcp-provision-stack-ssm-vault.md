Running `make provision CLUSTER_PROVIDER=k3s-gcp GHCR_PAT=<pat>` fails with two errors:

1. Brew installs `session-manager-plugin` (AWS-only tool) even though `CLUSTER_PROVIDER=k3s-gcp`

   ```
   /scripts/plugins/vault.sh: line 901: $1: unbound variable
   ```

## Root Causes

### Bug 1 ‚Äî Makefile `provision: ssm` unconditional prerequisite

`scripts/plugins/vault.sh:900`

`Makefile` declares `provision: ssm`, so `ssm` always runs before the `case` dispatch ‚Äî even for GCP. `ssm` installs `session-manager-plugin`, which is AWS SSM only.

**Fix:** Remove `ssm` as a top-level prerequisite. Move `$(MAKE) --no-print-directory ssm` inside the `*)` (non-GCP) case so it only runs for AWS.

**Before:**
```makefile
provision: ssm
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-gcp) ... gcp_provision_stack ;; \
	  *)       ... acg_provision --confirm ;; \
	esac
```

**After:**
```makefile
provision:
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-gcp) echo "[make] CLUSTER_PROVIDER=k3s-gcp ‚Äî running gcp_provision_stack..."; \
	           CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" \
	           GHCR_PAT="$(GHCR_PAT)" \
	           scripts/k3d-manager gcp_provision_stack ;; \
	  *)       $(MAKE) --no-print-directory ssm; \
	           echo "[make] CLUSTER_PROVIDER=$(CLUSTER_PROVIDER) ‚Äî running acg_provision..."; \
	           K3S_AWS_SSM_ENABLED=true scripts/k3d-manager acg_provision --confirm ;; \
	esac
```

### Bug 2 ‚Äî `deploy_vault` bare `$1` with `set -u`

`scripts/plugins/vault.sh` line 901:

```bash
if [[ "$1" == "-h" || "$1" == "--help" ]]; then
```

`gcp_provision_stack` calls `deploy_vault` with no arguments (`CLUSTER_ROLE=infra deploy_vault`). With `set -euo pipefail` active, bare `$1` triggers `unbound variable`.

**Fix:** Add default to both references on line 901:

**Before:**
```bash
if [[ "$1" == "-h" || "$1" == "--help" ]]; then
```

**After:**
```bash
if [[ "${1:-}" == "-h" || "${1:-}" == "--help" ]]; then
```
