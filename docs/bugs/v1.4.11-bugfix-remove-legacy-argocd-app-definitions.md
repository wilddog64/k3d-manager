## Problem

Two competing ArgoCD app layers manage the same K8s resources in `shopping-cart-apps`
namespace, causing `SharedResourceWarning` and `SyncError` on all `shopping-cart-*` apps.

**Layer 1 — `shopping-cart-apps` parent (shopping-cart-infra `argocd/applications/`)**
Creates 5 legacy apps (`basket-service`, `frontend`, `order-service`, `payment-service`,
`product-catalog`) pointing directly to each service repo's `k8s/base`.

**Layer 2 — `services-git` ApplicationSet (k3d-manager)**
Scans `services/*` and creates 5 canonical apps (`shopping-cart-basket`,
`shopping-cart-frontend`, `shopping-cart-order`, `shopping-cart-payment`,
`shopping-cart-product-catalog`) pointing to kustomize wrappers in k3d-manager.

Both layers deploy the same `Deployment`/`Service`/`ConfigMap` objects to
`shopping-cart-apps` → ArgoCD raises `SharedResourceWarning` and refuses to sync the
`shopping-cart-*` apps.

**Root cause:** The 5 legacy yaml files in `shopping-cart-infra/argocd/applications/` were
never removed when the `services-git` ApplicationSet was introduced in k3d-manager.

**Pre-condition already handled (Claude, 2026-05-29):** The cascade-delete finalizer
(`resources-finalizer.argocd.argoproj.io`) has been stripped from all 5 legacy apps on the
live cluster. ArgoCD will prune them as non-cascade when `shopping-cart-apps` next syncs.

## Fix

### Change 1 — delete 5 legacy app definition files from `argocd/applications/`

Delete the following files entirely (do NOT modify, do NOT leave empty):

- `argocd/applications/basket-service.yaml`
- `argocd/applications/frontend.yaml`
- `argocd/applications/order-service.yaml`
- `argocd/applications/payment-service.yaml`
- `argocd/applications/product-catalog.yaml`

The remaining files in `argocd/applications/` (`data-layer.yaml`, `identity.yaml`,
`networking.yaml`, `shopping-cart-apps.yaml`) are NOT touched.
