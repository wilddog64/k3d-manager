## Problem

When an ACG GCP sandbox expires and a new one is provisioned, the old
`cloud_user_p_xxx@linuxacademygclabs.com` account is deleted by GCP. Any subsequent
gcloud call that tries to refresh the OAuth token for that account returns:

```
ERROR: (gcloud.compute.instances.describe) There was a problem refreshing your current
auth tokens: ('invalid_grant: Account has been deleted', {'error': 'invalid_grant', ...})
```

Two failure points:

### Failure 1 — `_provider_k3s_gcp_status` calls gcloud without auth guard

`_provider_k3s_gcp_status` reads the project from the cached SA key on disk, logs
`INFO: [k3s-gcp] Instance ...`, then calls `gcloud compute instances describe` directly —
**no `_gcp_load_credentials`, no `gcp_login`**. When the sandbox has rotated the stale
gcloud token is used and `invalid_grant` is printed to stderr. `_run_command --soft`
makes it non-fatal, but the raw gcloud error is confusing.

### Failure 2 — `_gcp_load_credentials` does not revoke stale account before re-auth

When `gcloud projects describe "${_cached_project}"` fails (stale auth), the else branch
(lines 68–70) runs `gcp_get_credentials` to re-extract. But the stale deleted account
is **still present in gcloud's auth list as ACTIVE**. This can cause token-refresh
interference during the subsequent `gcp_login` re-auth flow.

---

## Reproduction

```bash
# 1. Run make up CLUSTER_PROVIDER=k3s-gcp — succeeds
# 2. Let the ACG sandbox expire (or open a new one)
# 3. Run make status CLUSTER_PROVIDER=k3s-gcp OR make up CLUSTER_PROVIDER=k3s-gcp
# → ERROR: (gcloud.compute.instances.describe) invalid_grant: Account has been deleted
```

---

## Fix

### Change 1 — `_gcp_load_credentials`: revoke stale accounts before re-extracting

**Exact old block (lines 67–70):**

```bash
    else
      _info "[k3s-gcp] Cached project ${_cached_project} not accessible — sandbox may have changed; re-extracting credentials..."
      rm -f "${_default_key}"
      gcp_get_credentials "${sandbox_url}" || return 1
    fi
```

**Exact new block:**

```bash
    else
      _info "[k3s-gcp] Cached project ${_cached_project} not accessible — sandbox may have changed; re-extracting credentials..."
      rm -f "${_default_key}"
      gcloud auth revoke --all --quiet 2>/dev/null || true
      gcp_get_credentials "${sandbox_url}" || return 1
    fi
```

---

### Change 2 — `_provider_k3s_gcp_status`: guard gcloud call with auth check

**Exact old block (lines 117–125):**

```bash
  if [[ -n "${project}" ]] && _command_exist gcloud; then
    _info "[k3s-gcp] Instance ${_GCP_INSTANCE_NAME} (project ${project}, zone ${_GCP_ZONE})"
    _run_command --soft -- gcloud compute instances describe "${_GCP_INSTANCE_NAME}" \
      --project="${project}" \
      --zone="${_GCP_ZONE}" \
      --format='table(name,status,networkInterfaces[0].accessConfigs[0].natIP)' || true
  else
    _info "[k3s-gcp] Skipping gcloud instance describe (set GCP_PROJECT and ensure gcloud is installed)"
  fi
```

**Exact new block:**

```bash
  if [[ -n "${project}" ]] && _command_exist gcloud; then
    local _status_auth
    _status_auth=$(gcloud auth list --filter="status:ACTIVE" --format="value(account)" 2>/dev/null || true)
    if [[ -z "${_status_auth}" ]]; then
      _info "[k3s-gcp] No active gcloud account — run: make up CLUSTER_PROVIDER=k3s-gcp to re-authenticate"
    else
      _info "[k3s-gcp] Instance ${_GCP_INSTANCE_NAME} (project ${project}, zone ${_GCP_ZONE})"
      _run_command --soft -- gcloud compute instances describe "${_GCP_INSTANCE_NAME}" \
        --project="${project}" \
        --zone="${_GCP_ZONE}" \
        --format='table(name,status,networkInterfaces[0].accessConfigs[0].natIP)' || true
    fi
  else
    _info "[k3s-gcp] Skipping gcloud instance describe (set GCP_PROJECT and ensure gcloud is installed)"
  fi
```
