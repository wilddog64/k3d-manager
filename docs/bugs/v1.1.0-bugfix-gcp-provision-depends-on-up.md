## Problem

`make provision CLUSTER_PROVIDER=k3s-gcp` runs `gcp_provision_stack` directly without
first ensuring the k3s-gcp cluster exists and is reachable. Running provision against
a stopped, not-yet-created, or IP-changed cluster fails mid-run with cryptic errors
(Vault deploy fails, kubectl times out, etc.).

The correct workflow is always:
```
make up CLUSTER_PROVIDER=k3s-gcp      # cluster node up + k3s installed
make provision CLUSTER_PROVIDER=k3s-gcp  # full plugin stack
```

`make up` is idempotent — it skips instance creation if the instance already exists and
re-runs k3sup only if needed. Calling it unconditionally at the start of provision adds
no risk and guarantees the cluster is in a known-good state before the stack deploys.

**Root cause:** `Makefile` `provision:` target (lines 107–116) has no dependency on
`make up` for the k3s-gcp provider.

---

## Reproduction

```bash
# cluster down or IP changed — run provision directly
make provision CLUSTER_PROVIDER=k3s-gcp GHCR_PAT=<pat>
# → deploy_vault: kubectl timeout / x509 error
# → no clear "cluster is not up" message
```

---

## Fix

### Change 1 — `Makefile`: call `make up` before `gcp_provision_stack` in the k3s-gcp provision case

**Exact old block (lines 108–116):**

```makefile
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-gcp) echo "[make] CLUSTER_PROVIDER=k3s-gcp — running gcp_provision_stack..."; \
	           CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" \
	           GHCR_PAT="$(GHCR_PAT)" \
	           scripts/k3d-manager gcp_provision_stack ;; \
	  *)       $(MAKE) --no-print-directory ssm; \
	           echo "[make] CLUSTER_PROVIDER=$(CLUSTER_PROVIDER) — running acg_provision..."; \
	           K3S_AWS_SSM_ENABLED=true scripts/k3d-manager acg_provision --confirm ;; \
	esac
```

**Exact new block:**

```makefile
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-gcp) echo "[make] CLUSTER_PROVIDER=k3s-gcp — ensuring cluster is up..."; \
	           $(MAKE) --no-print-directory up CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" URL="$(URL)"; \
	           echo "[make] CLUSTER_PROVIDER=k3s-gcp — running gcp_provision_stack..."; \
	           CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" \
	           GHCR_PAT="$(GHCR_PAT)" \
	           scripts/k3d-manager gcp_provision_stack ;; \
	  *)       $(MAKE) --no-print-directory ssm; \
	           echo "[make] CLUSTER_PROVIDER=$(CLUSTER_PROVIDER) — running acg_provision..."; \
	           K3S_AWS_SSM_ENABLED=true scripts/k3d-manager acg_provision --confirm ;; \
	esac
```
