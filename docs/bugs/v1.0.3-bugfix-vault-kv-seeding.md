## Problem

After `make up`, all ExternalSecrets in `shopping-cart-payment` show `SecretSyncedError:
could not get secret data from provider`. The `ClusterSecretStore vault-backend` is Ready,
but Vault KV has no secrets at the paths the ExternalSecrets expect.

**Root cause:** `bin/acg-up` applies the ClusterSecretStore but never seeds Vault with the
static KV secrets the ExternalSecrets reference. The companion fix in shopping-cart-infra
(`docs/plans/bugfix-eso-externalsecret-storeref.md`) switches postgres ExternalSecrets from
dynamic DB engine paths (`database/creds/…`) to static KV paths (`secret/data/postgres/…`).
This fix seeds all required KV paths so both redis and postgres ExternalSecrets sync.

Vault is accessible at `http://localhost:8200` after step 4 (port-forward already running).
`_vault_root_token` is already set at step 9. The seeding must happen after the vault-token
Secret is applied and before step 10 (ArgoCD registration).

---

## Fix

### Change 1 — `bin/acg-up`: add Vault KV seeding after ClusterSecretStore apply

**Exact old block (lines 250–253):**
```bash
CSSEOF
_info "[acg-up] vault-token secret + ClusterSecretStore applied"

_info "[acg-up] Step 10/12 — Registering app cluster with ArgoCD..."
```

**Exact new block:**
```bash
CSSEOF
_info "[acg-up] vault-token secret + ClusterSecretStore applied"

_info "[acg-up] Seeding Vault KV with sandbox static secrets..."
VAULT_ADDR="http://localhost:8200" VAULT_TOKEN="${_vault_root_token}" \
  vault kv put secret/redis/cart password="redis-cart-dev-pass" >/dev/null
VAULT_ADDR="http://localhost:8200" VAULT_TOKEN="${_vault_root_token}" \
  vault kv put secret/redis/orders-cache password="redis-orders-cache-dev-pass" >/dev/null
VAULT_ADDR="http://localhost:8200" VAULT_TOKEN="${_vault_root_token}" \
  vault kv put secret/postgres/orders username="postgres" password="changeme456" >/dev/null
VAULT_ADDR="http://localhost:8200" VAULT_TOKEN="${_vault_root_token}" \
  vault kv put secret/postgres/products username="postgres" password="changeme123" >/dev/null
VAULT_ADDR="http://localhost:8200" VAULT_TOKEN="${_vault_root_token}" \
  vault kv put secret/postgres/payment username="postgres" password="CHANGE_ME" >/dev/null
VAULT_ADDR="http://localhost:8200" VAULT_TOKEN="${_vault_root_token}" \
  vault kv put secret/payment/encryption key="dmF1bHQtZGV2LXNhbmRib3gtZW5jcnlwdGlvbg==" >/dev/null
VAULT_ADDR="http://localhost:8200" VAULT_TOKEN="${_vault_root_token}" \
  vault kv put secret/payment/stripe \
    api_key="sk_test_placeholder" \
    webhook_secret="whsec_placeholder" >/dev/null
VAULT_ADDR="http://localhost:8200" VAULT_TOKEN="${_vault_root_token}" \
  vault kv put secret/payment/paypal \
    client_id="paypal_sandbox_client_id" \
    client_secret="paypal_sandbox_client_secret" >/dev/null
_info "[acg-up] Vault KV seeded (redis, postgres, payment sandbox secrets)"

_info "[acg-up] Step 10/12 — Registering app cluster with ArgoCD..."
```

**Notes on values:**
- `postgres/orders password=changeme456` — matches `postgres-orders-admin` secret in shopping-cart-infra
- `postgres/products password=changeme123` — matches `postgres-products-admin` secret
- `postgres/payment password=CHANGE_ME` — matches `postgres-payment-admin` secret
- Redis and payment values are dev-only sandbox placeholders
