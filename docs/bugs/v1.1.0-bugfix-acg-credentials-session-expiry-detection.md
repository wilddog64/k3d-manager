## Problem

When the Pluralsight browser session expires, Pluralsight redirects mid-page-load
to `app.pluralsight.com/id` (its auth gateway, title: "Just a moment...").
The script does not detect this redirect. It runs the 30s `waitForFunction` against
the `/id` page — which has no sandbox buttons — times out, and then fails on
`text=Username` (15s timeout inside `_extractGcpCredentials`). The user sees:

```
WARN: Timed out waiting for sandbox buttons or credentials
DIAG: URL at timeout: https://app.pluralsight.com/id
DIAG: Title at timeout: Just a moment...
DIAG: Visible buttons: []
...
ERROR: page.waitForSelector: Timeout 15000ms exceeded.
  waiting for locator('text=Username') to be visible
```

The real cause (session expiry) is completely hidden. The fix requires the user
to delete the auth dir and re-run — but they have no idea that is the fix.

---

## Fix

After the aria-busy wait (line 239), add a 1-second stabilization pause and a URL
check for `/id`. If detected, throw immediately with an actionable error message.

**Exact old block (lines 239–241 of `acg_credentials.js`):**

```js
    ).catch(() => console.error('WARN: Skeleton loaders did not clear within 30s — proceeding anyway'));

    // 2b. Handle unauthenticated state — sign in via Google Password Manager if needed
```

**Exact new block:**

```js
    ).catch(() => console.error('WARN: Skeleton loaders did not clear within 30s — proceeding anyway'));

    // Detect Pluralsight session expiry — /id redirect means cookies are invalid
    await page.waitForTimeout(1000);
    if (page.url().includes('/id')) {
      throw new Error(
        'Pluralsight session expired (redirected to /id auth gateway).\n' +
        'Fix: rm -rf ~/.local/share/k3d-manager/playwright && re-run make up to trigger manual re-login.'
      );
    }

    // 2b. Handle unauthenticated state — sign in via Google Password Manager if needed
```
