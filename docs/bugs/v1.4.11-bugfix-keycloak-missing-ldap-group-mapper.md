# Bugfix: v1.4.11 — Keycloak LDAP federation missing group mapper

**Branch:** `k3d-manager-v1.4.11`
**Repos:** `shopping-cart-infra` (permanent fix), `k3d-manager` (`bin/acg-up` reconcile step)

---

## Problem

ArgoCD SSO users land on `policy.default: role:readonly` even though LDAP groups exist
and RBAC policy maps those groups to ArgoCD roles. Users can view apps but cannot sync —
`permission denied: applications, sync, shopping-cart/<app>`.

**Root cause:** The Keycloak LDAP federation (`ee968f21`) has no `group-ldap-mapper`
component. Keycloak syncs user attributes but never syncs LDAP group memberships, so
all users have zero Keycloak group memberships and the groups claim in the JWT is empty.
ArgoCD receives an empty `groups` array and falls through to the default readonly role.

---

## Reproduction

1. Log into ArgoCD via SSO (any LDAP user)
2. Try to sync an application
3. Error: `Unable to sync: permission denied: applications, sync, shopping-cart/<app>`
4. Confirm via Keycloak admin: GET `/admin/realms/shopping-cart/users/<sub>/groups` → `[]`

---

## Fix

### Part 1 — `shopping-cart-infra`: Add `group-ldap-mapper` to realm import

**File:** `identity/keycloak/keycloak-reconcile-hook-job.yaml`

Add a `group-ldap-mapper` component creation step after the LDAP federation is configured:

```bash
# Add group-ldap-mapper to LDAP federation
MAPPER_RESP=$(curl -s -w '\n%{http_code}' -X POST \
  "${KEYCLOAK_URL}/admin/realms/shopping-cart/components" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H 'Content-Type: application/json' \
  -d "{
    \"parentId\": \"${LDAP_COMPONENT_ID}\",
    \"providerType\": \"org.keycloak.storage.ldap.mappers.LDAPStorageMapper\",
    \"providerId\": \"group-ldap-mapper\",
    \"name\": \"group-mapper\",
    \"config\": {
      \"groups.dn\": [\"ou=groups,dc=shopping-cart,dc=local\"],
      \"group.name.ldap.attribute\": [\"cn\"],
      \"group.object.classes\": [\"groupOfNames\"],
      \"preserve.group.inheritance\": [\"true\"],
      \"ignore.missing.groups\": [\"false\"],
      \"membership.ldap.attribute\": [\"member\"],
      \"membership.attribute.type\": [\"DN\"],
      \"membership.user.ldap.attribute\": [\"uid\"],
      \"mode\": [\"READ_ONLY\"],
      \"user.roles.retrieve.strategy\": [\"LOAD_GROUPS_BY_MEMBER_ATTRIBUTE\"],
      \"memberof.ldap.attribute\": [\"memberOf\"],
      \"drop.non.existing.groups.during.sync\": [\"false\"]
    }
  }")
```

Then trigger a group sync:
```bash
curl -s -X POST \
  "${KEYCLOAK_URL}/admin/realms/shopping-cart/user-storage/${LDAP_COMPONENT_ID}/mappers/${MAPPER_ID}/sync?direction=fedToKeycloak" \
  -H "Authorization: Bearer ${TOKEN}"
```

### Part 2 — `bin/acg-up`: Add Step 10d.7 — group mapper reconciliation

After Step 10d.6 (LDAP bind credential reconciliation), add:

```bash
_info "[acg-up] Step 10d.7/14 — Reconciling Keycloak LDAP group mapper..."
_kc_token=$(curl -s -X POST "http://localhost:${_kc_port}/realms/master/protocol/openid-connect/token" \
  -d "client_id=admin-cli&grant_type=password&username=admin&password=${_kc_admin_pass}" \
  | python3 -c 'import sys,json; print(json.load(sys.stdin).get("access_token",""))' 2>/dev/null || true)

if [[ -z "${_kc_token}" ]]; then
  _warn "[acg-up] Could not get Keycloak token — skipping group mapper reconciliation"
else
  _ldap_component_id=$(curl -s \
    "http://localhost:${_kc_port}/admin/realms/shopping-cart/components?type=org.keycloak.storage.UserStorageProvider" \
    -H "Authorization: Bearer ${_kc_token}" \
    | python3 -c 'import sys,json; items=json.load(sys.stdin); [print(i["id"]) for i in items if i.get("providerId")=="ldap"]' 2>/dev/null | head -1 || true)

  # Check if group mapper already exists
  _existing_mapper=$(curl -s \
    "http://localhost:${_kc_port}/admin/realms/shopping-cart/components?parentId=${_ldap_component_id}&search=group-mapper" \
    -H "Authorization: Bearer ${_kc_token}" \
    | python3 -c 'import sys,json; items=json.load(sys.stdin); print(items[0]["id"] if items else "")' 2>/dev/null || true)

  if [[ -z "${_existing_mapper}" ]]; then
    _existing_mapper=$(curl -s -X POST \
      "http://localhost:${_kc_port}/admin/realms/shopping-cart/components" \
      -H "Authorization: Bearer ${_kc_token}" \
      -H 'Content-Type: application/json' \
      -d "{\"parentId\":\"${_ldap_component_id}\",\"providerType\":\"org.keycloak.storage.ldap.mappers.LDAPStorageMapper\",\"providerId\":\"group-ldap-mapper\",\"name\":\"group-mapper\",\"config\":{\"groups.dn\":[\"ou=groups,dc=shopping-cart,dc=local\"],\"group.name.ldap.attribute\":[\"cn\"],\"group.object.classes\":[\"groupOfNames\"],\"preserve.group.inheritance\":[\"true\"],\"ignore.missing.groups\":[\"false\"],\"membership.ldap.attribute\":[\"member\"],\"membership.attribute.type\":[\"DN\"],\"membership.user.ldap.attribute\":[\"uid\"],\"mode\":[\"READ_ONLY\"],\"user.roles.retrieve.strategy\":[\"LOAD_GROUPS_BY_MEMBER_ATTRIBUTE\"],\"memberof.ldap.attribute\":[\"memberOf\"],\"drop.non.existing.groups.during.sync\":[\"false\"]}}" \
      -w '%{http_code}' -o /dev/null 2>/dev/null || true)
    [[ "${_existing_mapper}" == "201" ]] && _info "[acg-up] Group mapper created" || _warn "[acg-up] Group mapper creation returned ${_existing_mapper}"
    # Re-fetch mapper ID for sync
    _existing_mapper=$(curl -s \
      "http://localhost:${_kc_port}/admin/realms/shopping-cart/components?parentId=${_ldap_component_id}&search=group-mapper" \
      -H "Authorization: Bearer ${_kc_token}" \
      | python3 -c 'import sys,json; items=json.load(sys.stdin); print(items[0]["id"] if items else "")' 2>/dev/null || true)
  fi

  if [[ -n "${_existing_mapper}" ]]; then
    curl -s -X POST \
      "http://localhost:${_kc_port}/admin/realms/shopping-cart/user-storage/${_ldap_component_id}/mappers/${_existing_mapper}/sync?direction=fedToKeycloak" \
      -H "Authorization: Bearer ${_kc_token}" -o /dev/null 2>/dev/null && \
      _info "[acg-up] LDAP group sync complete" || _warn "[acg-up] LDAP group sync failed"
  fi
fi
```
