## Problem

`acg_credentials.js` has four flaws that block the unified AWS+GCP story:

1. **`_cdpBrowser.close()` on lines 61 and 302** — for a CDP connection, calling
   `close()` on the remote `Browser` object tears down the user's Chrome. The
   correct call for a CDP-attached session is `.disconnect()`, which detaches
   the Playwright client and leaves the browser running.

2. **Hard-coded `localhost`** — line 47 uses `http://localhost:9222`. On macOS,
   `localhost` resolves to `::1` (IPv6) first; Chrome binds CDP on
   `127.0.0.1:9222` (IPv4) by default. The connect occasionally fails with
   `ECONNREFUSED` before the IPv6→IPv4 fallback kicks in.

3. **No provider switch** — the script only extracts AWS credentials. GCP
   sandboxes expose a different copyable-input layout (`Username`, `Password`,
   `Service Account Key JSON`) and require writing the SA JSON to disk.

4. **Stray timer keeps Node alive** — lines 309–318 use `Promise.race` with a
   `setTimeout` whose handle is never cleared on success. After
   `extractCredentials()` resolves, Node stays up until the 2- or 5-minute timer
   fires, so `make up` appears to hang for minutes after
   `INFO: Detached from Chrome session.` is printed.

  _timeoutHandle = setTimeout(
    () => reject(new Error(`Script timed out after ${OVERALL_TIMEOUT_MS / 1000}s`)),
    OVERALL_TIMEOUT_MS
  );
});

Promise.race([extractCredentials(), _timeoutPromise])
  .then(() => {
    clearTimeout(_timeoutHandle);
    process.exit(0);
  })
  .catch(err => {
    clearTimeout(_timeoutHandle);
    console.error(`ERROR: ${err.message}`);
    process.exit(1);
  });

### What changed

| | Old | New |
|---|---|---|
| CDP host | `http://localhost:9222` literal | `CDP_URL` from `PLAYWRIGHT_CDP_HOST`/`_PORT` env (default 127.0.0.1:9222) |
| Auth dir | hard-coded | env override via `PLAYWRIGHT_AUTH_DIR` with same default |
| CDP teardown | `_cdpBrowser.close()` (closes remote Chrome) | `_cdpBrowser.disconnect()` (detaches only) |
| Provider switch | none (AWS-only) | `--provider aws\|gcp` argv flag |
| GCP extraction | — | writes SA JSON to `~/.local/share/k3d-manager/gcp-service-account.json`, emits `GCP_PROJECT` / `GOOGLE_APPLICATION_CREDENTIALS` |
| Hang after success | setTimeout keeps Node alive | `clearTimeout` + explicit `process.exit(0)` |
