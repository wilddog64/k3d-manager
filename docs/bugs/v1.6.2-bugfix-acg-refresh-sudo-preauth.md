# Bugfix: v1.6.2 — Remove pre-auth sudo block from bin/acg-refresh

**Branch:** `k3d-manager-v1.6.2`
**Files:** `bin/acg-refresh`

---

## Problem

When the Slack webhook runs `/acg-refresh`, the script is headless (no TTY). `bin/acg-refresh`
contains an unconditional pre-auth block that calls `sudo -v`, which requires a TTY to read
the password. The webhook has no TTY, so the command fails immediately:

```
sudo: a terminal is required to read the password; either use the -S option to read from
standard input or configure an askpass helper
sudo: a password is required
```

**Root cause:** The pre-auth block at line 145 calls `sudo -v` directly without a TTY guard.
It predates the NOPASSWD rules in `/etc/sudoers.d/k3d-manager`, which already grant
passwordless elevation for the specific commands `acg-refresh` needs. The block is redundant.

---

## Reproduction

1. Run `bin/acg-refresh --no-login-prompt` from the webhook (no TTY).
2. Observe: immediate exit 1 with `sudo: a terminal is required`.

---

## Fix

### Change — `bin/acg-refresh`: remove pre-auth sudo block (lines 145–148)

**Exact old block:**

```bash
if _is_mac && ! sudo -n true 2>/dev/null; then
  _info "[acg-refresh] Pre-authorizing sudo for macOS host operations..."
  sudo -v
fi
```

**Exact new block:** *(delete entirely — nothing replaces it)*

The `_run_command --prefer-sudo` and `_run_command --interactive-sudo` calls later in the
script already handle elevation correctly via the NOPASSWD rules in
`/etc/sudoers.d/k3d-manager`.
