## Problem

Five issues found during post-commit review of `1c620795`:

1. **`--kubeconfig` is not a valid k3sup flag** — the correct flag is `--local-path`. k3sup will error or write the kubeconfig to an unexpected location.
2. **`--image-family` and `--image-project` missing from `gcloud compute instances create`** — GCE will use its default image (Debian) instead of Ubuntu 22.04 LTS. k3sup and k3d-manager assume Ubuntu.
3. **`--k3s-extra-args="--disable=traefik"` missing from `k3sup install`** — Traefik is installed alongside k3s, competing with Istio IngressGateway for ports 80/443.
4. **`destroy_cluster` does not clean up the kubeconfig context** — `kubectl config delete-context k3s-gcp`, `delete-cluster k3s-gcp`, and `rm kubeconfig` are missing. `make down` leaves a stale context.
5. **`gcp.sh` assigns directly to globals before validating** — if Playwright output is partial, empty values are silently exported. Locals should be used for parsing; validation should run before `export`.

### Change 6 — `scripts/lib/providers/k3s-gcp.sh`: add pre-flight compute permission check

**Context:** Gemini's investigation confirmed the ACG sandbox SA (`cli-service-account-1@<project>.iam.gserviceaccount.com`) lacks `compute.zones.list` and all Compute IAM permissions. Without a pre-flight check, `deploy_cluster` silently proceeds and fails deep in the flow with a cryptic `PERMISSION_DENIED` from `gcloud compute instances describe` or `gcloud compute instances create`. The user has no clear indication of what permission is missing or how to fix it.

**Step 1 — add the helper function** after `_gcp_ssh_config_remove` (after line 52, before `_provider_k3s_gcp_deploy_cluster`):

```bash
function _gcp_preflight_check_compute() {
  local project="$1"
  local key_file="$2"
  local sa_email
  sa_email=$(jq -r '.client_email' "${key_file}" 2>/dev/null || true)
  if ! gcloud compute instances list \
      --project="${project}" \
      --limit=1 \
      --quiet >/dev/null 2>&1; then
    _err "[k3s-gcp] SA lacks Compute permissions on project ${project}."
    _err "[k3s-gcp] Grant roles/compute.admin to: ${sa_email:-<service-account>}"
    _err "[k3s-gcp] Via: GCP Console → IAM & Admin → IAM → Grant Access"
    _err "[k3s-gcp] Then retry: make up CLUSTER_PROVIDER=k3s-gcp"
    return 1
  fi
}
```

**Step 2 — call it in `_provider_k3s_gcp_deploy_cluster`**, immediately after `unset _gcp_access_token` (currently line 114) and before the existing instance-describe block:

**Exact old block (lines 114–116):**

```bash
  unset _gcp_access_token

  _info "[k3s-gcp] Checking for existing instance ${_GCP_INSTANCE_NAME}..."
```

**Exact new block:**

```bash
  unset _gcp_access_token

  _info "[k3s-gcp] Pre-flight: checking Compute IAM permissions..."
  _gcp_preflight_check_compute "${project}" "${key_file}" || return 1

  _info "[k3s-gcp] Checking for existing instance ${_GCP_INSTANCE_NAME}..."
```
