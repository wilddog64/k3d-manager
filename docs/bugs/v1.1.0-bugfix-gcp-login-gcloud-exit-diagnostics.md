## Problem

After Playwright extracts the auth code and feeds it to gcloud via FIFO, gcloud exits 1. The current code deletes `_auth_log` **before** checking the exit code, so gcloud's error message is lost:

```bash
# lines 208–216 (current)
printf '%s\n' "${_auth_code}" >"${_auth_fifo}"
wait "${_gcloud_pid}"
local _exit=$?
kill "${_fifo_writer_pid}" 2>/dev/null || true
rm -f "${_auth_log}" "${_auth_fifo}"          # <-- log deleted here

if [[ "${_exit}" -ne 0 ]]; then
  _err "[gcp] gcp_login: gcloud auth login failed (exit ${_exit})"  # no log available
  return 1
fi
```

We cannot diagnose why gcloud rejects the auth code without seeing its output.

---

## Fix

### Change 1 — `scripts/plugins/gcp.sh`: log `_auth_log` contents on gcloud failure

Move `rm -f "${_auth_log}" "${_auth_fifo}"` to after the exit check. On failure, log gcloud's full output before deleting the file.

**Exact old block (lines 208–217 in current file):**

```bash
  # Feed the auth code to gcloud's stdin; kill sleeper after gcloud reads it
  printf '%s\n' "${_auth_code}" >"${_auth_fifo}"
  wait "${_gcloud_pid}"
  local _exit=$?
  kill "${_fifo_writer_pid}" 2>/dev/null || true
  rm -f "${_auth_log}" "${_auth_fifo}"

  if [[ "${_exit}" -ne 0 ]]; then
    _err "[gcp] gcp_login: gcloud auth login failed (exit ${_exit})"
    return 1
  fi
```

**Exact new block:**

```bash
  # Feed the auth code to gcloud's stdin; kill sleeper after gcloud reads it
  printf '%s\n' "${_auth_code}" >"${_auth_fifo}"
  wait "${_gcloud_pid}"
  local _exit=$?
  kill "${_fifo_writer_pid}" 2>/dev/null || true

  if [[ "${_exit}" -ne 0 ]]; then
    _err "[gcp] gcp_login: gcloud auth login failed (exit ${_exit})"
    _err "[gcp] gcloud output: $(cat "${_auth_log}" 2>/dev/null || echo '<empty>')"
    rm -f "${_auth_log}" "${_auth_fifo}"
    return 1
  fi
  rm -f "${_auth_log}" "${_auth_fifo}"
```
