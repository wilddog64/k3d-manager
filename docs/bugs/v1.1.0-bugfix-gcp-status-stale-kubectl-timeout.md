## Problem

When `gcloud auth print-access-token` returns empty (stale sandbox credentials),
`_provider_k3s_gcp_status` logs the stale-credentials message and then **falls
through to the kubectl section unconditionally**. The kubeconfig still points to
the old sandbox VM's IP, so `kubectl get nodes` hangs for ~3 minutes and exits 1,
causing `make status` to fail:

```
INFO: [k3s-gcp] Stale gcloud credentials (sandbox rotated) — run: make up CLUSTER_PROVIDER=k3s-gcp to re-authenticate
INFO: [k3s-gcp] kubectl get nodes (context k3s-gcp)
E0418 ... dial tcp 34.70.64.111:6443: i/o timeout
Unable to connect to the server: dial tcp 34.70.64.111:6443: i/o timeout
make: *** [status] Error 1
```

Two fixes are needed:

1. **Early return** — when credentials are stale, `return 0` immediately after the
   info message; there is no point querying a cluster whose endpoint is invalid.
2. **Soften kubectl** — change `|| return 1` to `|| true` on both kubectl calls so
   that a transient cluster-unreachable error in the non-stale case does not fail
   the whole `make status`.

## Fix

### Change 1 — early return on stale credentials

**Exact old block (lines 118–130):**

```bash
  if [[ -n "${project}" ]] && _command_exist gcloud; then
    local _gcloud_token
    _gcloud_token=$(gcloud auth print-access-token 2>/dev/null || true)
    if [[ -z "${_gcloud_token}" ]]; then
      _info "[k3s-gcp] Stale gcloud credentials (sandbox rotated) — run: make up CLUSTER_PROVIDER=k3s-gcp to re-authenticate"
    else
      _info "[k3s-gcp] Instance ${_GCP_INSTANCE_NAME} (project ${project}, zone ${_GCP_ZONE})"
      _run_command --soft -- gcloud compute instances describe "${_GCP_INSTANCE_NAME}" \
        --project="${project}" \
        --zone="${_GCP_ZONE}" \
        --format='table(name,status,networkInterfaces[0].accessConfigs[0].natIP)' || true
    fi
  else
    _info "[k3s-gcp] Skipping gcloud instance describe (set GCP_PROJECT and ensure gcloud is installed)"
  fi
```

**Exact new block:**

```bash
  if [[ -n "${project}" ]] && _command_exist gcloud; then
    local _gcloud_token
    _gcloud_token=$(gcloud auth print-access-token 2>/dev/null || true)
    if [[ -z "${_gcloud_token}" ]]; then
      _info "[k3s-gcp] Stale gcloud credentials (sandbox rotated) — run: make up CLUSTER_PROVIDER=k3s-gcp to re-authenticate"
      return 0
    fi
    _info "[k3s-gcp] Instance ${_GCP_INSTANCE_NAME} (project ${project}, zone ${_GCP_ZONE})"
    _run_command --soft -- gcloud compute instances describe "${_GCP_INSTANCE_NAME}" \
      --project="${project}" \
      --zone="${_GCP_ZONE}" \
      --format='table(name,status,networkInterfaces[0].accessConfigs[0].natIP)' || true
  else
    _info "[k3s-gcp] Skipping gcloud instance describe (set GCP_PROJECT and ensure gcloud is installed)"
  fi
```

---

### Change 2 — soften kubectl calls

**Exact old lines (lines 134–138):**

```bash
  _info "[k3s-gcp] kubectl get nodes (context k3s-gcp)"
  KUBECONFIG="${kubeconfig}" kubectl get nodes || return 1

  _info "[k3s-gcp] kubectl get pods --all-namespaces -o wide"
  KUBECONFIG="${kubeconfig}" kubectl get pods --all-namespaces -o wide || return 1
```

**Exact new lines:**

```bash
  _info "[k3s-gcp] kubectl get nodes (context k3s-gcp)"
  KUBECONFIG="${kubeconfig}" kubectl get nodes || true

  _info "[k3s-gcp] kubectl get pods --all-namespaces -o wide"
  KUBECONFIG="${kubeconfig}" kubectl get pods --all-namespaces -o wide || true
```
