# Bugfix: v1.1.0 — gcp CDP launch must not restart Chrome; error with instructions instead

**Branch:** `k3d-manager-v1.1.0`
**Files:** `scripts/plugins/gcp.sh`

---

## Before You Start

1. `git pull origin k3d-manager-v1.1.0`
2. Read `scripts/plugins/gcp.sh` lines 81–115 (the `_gcp_launch_cdp_chrome` function)

---

## Problem

`_gcp_launch_cdp_chrome` kills Chrome when it detects Chrome is running without CDP,
then relaunches it. This is wrong for two reasons:

1. **It kills the user's Chrome session** — all open tabs are lost even with a 5s countdown
2. **It times out** — 15 seconds is not enough for Chrome to fully start and expose the
   CDP port on macOS; `make up` fails with `ERROR: Timed out waiting for Chrome CDP`

The correct behavior when Chrome is running without CDP is to **print a clear error and stop**.
The user restarts Chrome manually once, and every subsequent `make up` reuses the existing
CDP session with no restart, no Playwright login trigger, and no disruption.

`--remote-debugging-port=9222` is a launch-time flag. It cannot be added to a running
Chrome instance by any means. Auto-restart is not a valid workaround.

## Fix

### Change — `_gcp_launch_cdp_chrome` (lines 81–115 in `scripts/plugins/gcp.sh`)

**Exact old block:**

```bash
function _gcp_launch_cdp_chrome() {
  if [[ "$(uname)" == "Darwin" ]]; then
    if pgrep -x "Google Chrome" >/dev/null 2>&1; then
      _info "[gcp] Chrome is running without CDP. It must be restarted to enable --remote-debugging-port=9222."
      _info "[gcp] Chrome will restore your tabs via session restore."
      _info "[gcp] Restarting Chrome in 5 seconds — Ctrl+C to abort..."
      sleep 5
      pkill -x "Google Chrome" 2>/dev/null || true
      sleep 1
    fi
    _info "[gcp] Launching Chrome with CDP port 9222 and your default profile..."
    open -a "Google Chrome" --args --remote-debugging-port=9222
    return 0
  fi

  local chrome_bin
  chrome_bin=$(command -v google-chrome 2>/dev/null || command -v google-chrome-stable 2>/dev/null || command -v chromium-browser 2>/dev/null || command -v chromium 2>/dev/null || true)
  if [[ -z "${chrome_bin}" ]]; then
    _err "[gcp] Chrome/Chromium not found — install google-chrome or chromium"
    return 1
  fi
  if pgrep -x "google-chrome" >/dev/null 2>&1 || pgrep -x "google-chrome-stable" >/dev/null 2>&1 || pgrep -x "chromium-browser" >/dev/null 2>&1 || pgrep -x "chromium" >/dev/null 2>&1; then
    _info "[gcp] Chrome is running without CDP. It must be restarted to enable --remote-debugging-port=9222."
    _info "[gcp] Chrome will restore your tabs via session restore."
    _info "[gcp] Restarting Chrome in 5 seconds — Ctrl+C to abort..."
    sleep 5
    pkill -x "google-chrome" 2>/dev/null || true
    pkill -x "google-chrome-stable" 2>/dev/null || true
    pkill -x "chromium-browser" 2>/dev/null || true
    pkill -x "chromium" 2>/dev/null || true
    sleep 1
  fi
  _info "[gcp] Launching Chrome with CDP port 9222..."
  "${chrome_bin}" --remote-debugging-port=9222 &
}
```

**Exact new block:**

```bash
function _gcp_launch_cdp_chrome() {
  if [[ "$(uname)" == "Darwin" ]]; then
    if pgrep -x "Google Chrome" >/dev/null 2>&1; then
      _err "[gcp] Chrome is running but CDP is not enabled on port 9222."
      _err "[gcp] Quit Chrome, then relaunch it with:"
      _err "[gcp]   open -a \"Google Chrome\" --args --remote-debugging-port=9222"
      _err "[gcp] Then rerun: make up CLUSTER_PROVIDER=k3s-gcp"
      return 1
    fi
    _info "[gcp] Chrome not running — launching with CDP port 9222 and your default profile..."
    open -a "Google Chrome" --args --remote-debugging-port=9222
    return 0
  fi

  local chrome_bin
  chrome_bin=$(command -v google-chrome 2>/dev/null || command -v google-chrome-stable 2>/dev/null || command -v chromium-browser 2>/dev/null || command -v chromium 2>/dev/null || true)
  if [[ -z "${chrome_bin}" ]]; then
    _err "[gcp] Chrome/Chromium not found — install google-chrome or chromium"
    return 1
  fi
  if pgrep -x "google-chrome" >/dev/null 2>&1 || pgrep -x "google-chrome-stable" >/dev/null 2>&1 || pgrep -x "chromium-browser" >/dev/null 2>&1 || pgrep -x "chromium" >/dev/null 2>&1; then
    _err "[gcp] Chrome is running but CDP is not enabled on port 9222."
    _err "[gcp] Quit Chrome, then relaunch it with:"
    _err "[gcp]   ${chrome_bin} --remote-debugging-port=9222"
    _err "[gcp] Then rerun: make up CLUSTER_PROVIDER=k3s-gcp"
    return 1
  fi
  _info "[gcp] Chrome not running — launching with CDP port 9222..."
  "${chrome_bin}" --remote-debugging-port=9222 &
}
```

### What changed

- All `pkill`, `sleep`, and auto-restart logic removed entirely
- Chrome running without CDP → 4 `_err` lines with exact relaunch command + `return 1`
- Chrome not running → launch with CDP (safe, no session to disrupt)
- Once the user launches Chrome with CDP once, all future `make up` runs skip this
  function entirely (the outer `curl` guard in `gcp_get_credentials` passes immediately)
