# Bug: missing `register_shopping_cart_apps` in `acg-up` + bare `sudo` in `acg-down`

**Branch:** `k3d-manager-v1.5.3`
**Files:** `bin/acg-up:1291`, `bin/acg-down:47–51`
**Severity:** P2 — ArgoCD Applications not registered on every cluster rebuild; `sudo` called directly in violation of CLAUDE.md

---

## Symptom A — ArgoCD Applications not registered

After `make up` completes, no ArgoCD Application resources exist in the cluster:

```
$ kubectl get applications --all-namespaces --context ubuntu-k3s
No resources found
```

`shopping-cart-rules`, `data-layer`, `shopping-cart-networking`, `shopping-cart-apps`, and
`shopping-cart-identity` must be applied manually via:

```
./scripts/k3d-manager register_shopping_cart_apps
```

The ArgoCD UI shows "Resource not found in cluster: argoproj.io/v1alpha1/Application:shopping-cart-rules"
until the manual step is run.

## Symptom B — Password prompt on every `make down`

```
INFO: [acg-down] keep-hub=false hub-cluster=k3d-cluster
INFO: [acg-down] Pre-authorizing sudo for macOS host operations (launchd cleanup)...
Password:
```

User is prompted for sudo password on every `make down` invocation when the sudo session has expired.

---

## Root Cause A

`register_shopping_cart_apps` is defined in `scripts/plugins/shopping_cart.sh` and exposed via the
dispatcher, but is never called in `bin/acg-up`. The provisioning flow reaches Step 12 (TTL watcher)
without registering any ArgoCD Applications.

**Correct insertion point:** after `shopping_cart_reconcile_product_catalog` (line 1291), before
Step 12 — all Vault secrets and ESO syncs are confirmed by this point.

## Root Cause B

`bin/acg-down:50` calls `sudo -v` directly. CLAUDE.md requires routing all privileged commands
through `_run_command`. The correct replacement is `_run_command --interactive-sudo --soft -- true`,
which prompts for password (when not cached) and refreshes the sudo credential cache identically
to `sudo -v`, but without calling `sudo` directly.

---

## Fix A — `bin/acg-up`

**After line 1291** (`shopping_cart_reconcile_product_catalog`), before the `_info "[acg-up] Step 12/14` line, insert:

```bash
_info "[acg-up] Step 11b/14 — Registering shopping-cart ArgoCD Applications..."
register_shopping_cart_apps
```

The surrounding context (exact lines to insert after):

```bash
shopping_cart_reconcile_order_service
shopping_cart_reconcile_product_catalog

_info "[acg-up] Step 11b/14 — Registering shopping-cart ArgoCD Applications..."
register_shopping_cart_apps

_info "[acg-up] Step 12/14 — Installing sandbox TTL watcher (launchd)..."
```

## Fix B — `bin/acg-down`

**Replace lines 47–51** with:

```bash
if _is_mac; then
  if [[ -t 0 && -t 1 ]] && ! sudo -n true 2>/dev/null; then
    _info "[acg-down] Pre-authorizing sudo for macOS host operations (launchd cleanup)..."
    _run_command --interactive-sudo --soft -- true
  fi
fi
```

## Behaviour After Fix

| Scenario | Before | After |
|----------|--------|-------|
| `make up` — ArgoCD Applications | Not registered; manual step required | Registered at Step 11b automatically |
| `make down` — sudo call | `sudo -v` (direct, violates CLAUDE.md) | `_run_command --interactive-sudo --soft -- true` (compliant) |
| `make down` — no cached session, TTY | Password prompt via raw `sudo -v` | Password prompt via `_run_command` |
| `make down` — no TTY | Skipped (TTY guard intact) | Same |
