## Problem

The fix in `b28b8925` replaced `_run_command --soft` with a `gcloud auth list` guard,
but `gcloud auth list --filter="status:ACTIVE"` reports whether an account is
**configured as active** â€” not whether its tokens are valid. A deleted sandbox account
remains ACTIVE in gcloud's config even after its GCP project and OAuth tokens are
revoked. The check passes, `gcloud compute instances describe` runs, and `invalid_grant`
still fires:

```
ERROR: (gcloud.compute.instances.describe) There was a problem refreshing your current
auth tokens: ('invalid_grant: Account has been deleted', ...)
gcloud command failed (1): gcloud compute instances describe k3s-gcp-server ...
```

## Fix

Replace `gcloud auth list` (config check only) with `gcloud auth print-access-token`
(actual token refresh attempt). When tokens are stale the command returns a non-zero
exit code and empty stdout; the `2>/dev/null` suppresses gcloud's own `invalid_grant`
error message.

**Exact old block (lines 119â€“129):**

```bash
    local _status_auth
    _status_auth=$(gcloud auth list --filter="status:ACTIVE" --format="value(account)" 2>/dev/null || true)
    if [[ -z "${_status_auth}" ]]; then
      _info "[k3s-gcp] No active gcloud account â€” run: make up CLUSTER_PROVIDER=k3s-gcp to re-authenticate"
    else
      _info "[k3s-gcp] Instance ${_GCP_INSTANCE_NAME} (project ${project}, zone ${_GCP_ZONE})"
      _run_command --soft -- gcloud compute instances describe "${_GCP_INSTANCE_NAME}" \
        --project="${project}" \
        --zone="${_GCP_ZONE}" \
        --format='table(name,status,networkInterfaces[0].accessConfigs[0].natIP)' || true
    fi
```

**Exact new block:**

```bash
    local _gcloud_token
    _gcloud_token=$(gcloud auth print-access-token 2>/dev/null || true)
    if [[ -z "${_gcloud_token}" ]]; then
      _info "[k3s-gcp] Stale gcloud credentials (sandbox rotated) â€” run: make up CLUSTER_PROVIDER=k3s-gcp to re-authenticate"
    else
      _info "[k3s-gcp] Instance ${_GCP_INSTANCE_NAME} (project ${project}, zone ${_GCP_ZONE})"
      _run_command --soft -- gcloud compute instances describe "${_GCP_INSTANCE_NAME}" \
        --project="${project}" \
        --zone="${_GCP_ZONE}" \
        --format='table(name,status,networkInterfaces[0].accessConfigs[0].natIP)' || true
    fi
```
