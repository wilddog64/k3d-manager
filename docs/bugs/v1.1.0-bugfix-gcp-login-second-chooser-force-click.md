## Problem

After the second account chooser block was added, the live test shows:

```
- locator resolved to <div role="link" ... data-identifier="cloud_user_p_...">
- element is visible, enabled and stable
- <div tabindex="0" class="dKGsO" jsname="OQ2Y6"></div> intercepts pointer events
- retrying click action
- element was detached from the DOM, retrying
- waiting for "https://accounts.google.com/signin/oauth/id?authuser=0&..." navigation to finish...
- navigated to "https://accounts.google.com/signin/oauth/id?authuser=0&..."
ERROR: locator.click: Timeout 30000ms exceeded.
```

**Root cause:** A Google overlay div (`class="dKGsO" jsname="OQ2Y6"`) intercepts pointer events, preventing the normal click from landing. Playwright retries for 30s and times out. The element detaches and the page re-navigates to the same `signin/oauth/id` URL — the Allow button should appear after this re-render, but the click action never completes because of the overlay.

**Fix:** Use `click({ force: true })` to bypass the intercepting overlay. Also increase the post-click settle wait from 1000ms to 2000ms to allow the page re-render to complete before the Allow button wait.

## Fix

### Change 1 — `scripts/playwright/gcp_login.js`: force-click second account chooser

**Exact old block (inside the `if (secondChooserVisible)` block):**

```js
    if (secondChooserVisible) {
      console.error(`INFO: Second account chooser detected — clicking ${username}`);
      await secondChooser.first().click();
      await page.waitForTimeout(1000);
      console.error(`INFO: URL after second chooser: ${page.url()}`);
    }
```

**Exact new block:**

```js
    if (secondChooserVisible) {
      console.error(`INFO: Second account chooser detected — clicking ${username}`);
      await secondChooser.first().click({ force: true });
      await page.waitForTimeout(2000);
      console.error(`INFO: URL after second chooser: ${page.url()}`);
    }
```
