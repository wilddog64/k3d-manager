# Bugfix: v1.6.4 — Grafana tunnel routed to frontend (port 3000 collision)

**Branch:** `k3d-manager-v1.6.4`
**Files:** `bin/acg-up`, `bin/acg-refresh`, `scripts/etc/cloudflared/config.yml`

---

## Problem

`grafana.3ai-talk.org` serves the shopping-cart frontend instead of Grafana dashboards.

**Root cause:** `scripts/etc/cloudflared/config.yml` routes `grafana.3ai-talk.org` to
`http://localhost:3000`, but `localhost:3000` is the frontend port-forward
(`com.k3d-manager.frontend-port-forward`). No Grafana port-forward LaunchAgent exists —
`acg-up` never installs one. Grafana traffic is sent to the frontend process.

---

## Reproduction

1. Run `acg-up` — Cloudflare tunnel starts, frontend port-forward starts on `localhost:3000`
2. Open `https://grafana.3ai-talk.org` — shopping-cart frontend loads instead of Grafana

---

## Fix

### Change 1 — `scripts/etc/cloudflared/config.yml`: route Grafana to port 3001

**Exact old block:**
```yaml
  - hostname: grafana.3ai-talk.org
    service: http://localhost:3000
```

**Exact new block:**
```yaml
  - hostname: grafana.3ai-talk.org
    service: http://localhost:3001
```

### Change 2 — `bin/acg-up`: add Grafana port-forward LaunchAgent after step 10g.10

Insert immediately after the `_info "[acg-up] Frontend port-forward agent loaded..."` line
(after line 1411):

**Exact old block:**
```bash
  _info "[acg-up] Frontend port-forward agent loaded (launchd: ${_frontend_pf_label})"
fi

if _is_mac; then
  _info "[acg-up] Step 10h/14 — Installing Cloudflare named tunnel
```

**Exact new block:**
```bash
  _info "[acg-up] Frontend port-forward agent loaded (launchd: ${_frontend_pf_label})"
fi

if _is_mac; then
  _info "[acg-up] Step 10g.11/14 — Installing Grafana port-forward launchd agent (localhost:3001 → ubuntu-k3s svc/acg-kube-prometheus-stack-grafana, auto-restart)..."
  _grafana_pf_label="com.k3d-manager.grafana-port-forward"
  _grafana_pf_plist="${HOME}/Library/LaunchAgents/${_grafana_pf_label}.plist"
  _grafana_pf_log="${_ACG_STATE_DIR}/logs/grafana-pf.log"
  _grafana_pf_kubectl="$(command -v kubectl)"
  _grafana_pf_kubeconfig="${HOME}/.kube/config:${HOME}/.kube/ubuntu-k3s.yaml:${HOME}/.kube/k3s-ubuntu.yaml"
  mkdir -p "$(dirname "${_grafana_pf_log}")"
  cat > "${_grafana_pf_plist}" <<PLIST
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>${_grafana_pf_label}</string>
  <key>ProgramArguments</key>
  <array>
    <string>${_grafana_pf_kubectl}</string>
    <string>port-forward</string>
    <string>svc/acg-kube-prometheus-stack-grafana</string>
    <string>--namespace</string>
    <string>monitoring</string>
    <string>--context</string>
    <string>ubuntu-k3s</string>
    <string>3001:80</string>
  </array>
  <key>EnvironmentVariables</key>
  <dict>
    <key>KUBECONFIG</key>
    <string>${_grafana_pf_kubeconfig}</string>
  </dict>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>${_grafana_pf_log}</string>
  <key>StandardErrorPath</key>
  <string>${_grafana_pf_log}</string>
</dict>
</plist>
PLIST
  if launchctl list "${_grafana_pf_label}" >/dev/null 2>&1; then
    launchctl unload "${_grafana_pf_plist}" 2>/dev/null || true
  fi
  launchctl load "${_grafana_pf_plist}"
  _info "[acg-up] Grafana port-forward agent loaded (launchd: ${_grafana_pf_label})"
fi

if _is_mac; then
  _info "[acg-up] Step 10h/14 — Installing Cloudflare named tunnel
```
