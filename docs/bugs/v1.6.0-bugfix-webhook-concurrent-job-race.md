# Bugfix: v1.6.0 — webhook concurrent job TOCTOU race causes duplicate acg-up timers

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/k3dm-webhook`

---

## Problem

Two concurrent `acg-up` jobs run simultaneously, producing interleaved Slack elapsed-time
messages with non-monotonic counts (e.g. 102m / 78m / 104m / 80m alternating). One job
succeeds; the other fails with `make: *** [up] Error 1`. Both emit progress notifications
every 2 minutes, making it appear the elapsed counter is broken.

**Root cause:** TOCTOU race in the HTTP request handler. The 409 concurrent-job guard
calls `_running_cluster_job()` which checks for `status == "running"`. However, the
handler writes `status = "queued"` before starting the thread, and the thread writes
`status = "running"` only after it starts executing (line 170 of `_run_cluster`). If a
second Slack trigger arrives in the window between the handler writing `"queued"` and the
thread writing `"running"`, `_running_cluster_job()` finds no `"running"` job and allows
the second job to start. Both threads then run `make up` concurrently.

---

## Reproduction

1. Trigger `/acg-up` via Slack
2. Trigger `/acg-up` again within ~1 second (e.g. double-click or rapid Slack retry)
3. Observe two jobs running simultaneously; Slack shows interleaved progress messages
   with different elapsed counts

---

## Fix

### Change 1 — `bin/k3dm-webhook`: write `"running"` in the HTTP handler, not the thread

**Exact old block (HTTP handler, cluster action, lines ~574–585):**

```python
            job_id = str(uuid.uuid4())[:8]
            (JOB_DIR / job_id).mkdir(parents=True, exist_ok=True)
            (JOB_DIR / job_id / "status").write_text("queued")
            (JOB_DIR / job_id / "action").write_text(action)
            if response_url:
                (JOB_DIR / job_id / "response_url").write_text(response_url)
            threading.Thread(
                target=_run_cluster,
                args=(job_id, action, provider),
                daemon=True
            ).start()
            self._json(202, {"job_id": job_id, "status": "queued"})
```

**Exact new block:**

```python
            job_id = str(uuid.uuid4())[:8]
            (JOB_DIR / job_id).mkdir(parents=True, exist_ok=True)
            (JOB_DIR / job_id / "status").write_text("running")
            (JOB_DIR / job_id / "action").write_text(action)
            if response_url:
                (JOB_DIR / job_id / "response_url").write_text(response_url)
            threading.Thread(
                target=_run_cluster,
                args=(job_id, action, provider),
                daemon=True
            ).start()
            self._json(202, {"job_id": job_id, "status": "queued"})
```

### Change 2 — `bin/k3dm-webhook`: remove redundant `"running"` write from `_run_cluster`

**Exact old block (top of `try` block in `_run_cluster`, line ~169–170):**

```python
    try:
        (job_dir / "status").write_text("running")
        if action == "up":
```

**Exact new block:**

```python
    try:
        if action == "up":
```
