# Bugfix: v1.1.0 — `deploy_eso` unbound `$1` crashes GCP provision stack

**Branch:** `k3d-manager-v1.1.0`
**Files:** `scripts/plugins/eso.sh`

---

## Problem

`make provision CLUSTER_PROVIDER=k3s-gcp` fails at Step 3 (Deploy ESO) with:

```
scripts/plugins/eso.sh: line 12: $1: unbound variable
```

`gcp_provision_stack` calls `deploy_eso` with no arguments. With `set -euo pipefail` active, the bare `$1` reference in the help guard triggers an unbound variable error before the function body runs.

**Root cause:** `deploy_eso` uses bare `"$1"` (not `"${1:-}"`) on line 12 for its help flag check.

---

## Reproduction

```bash
CLUSTER_PROVIDER=k3s-gcp make provision GHCR_PAT=any
# → scripts/plugins/eso.sh: line 12: $1: unbound variable
```

Or directly:

```bash
source scripts/k3d-manager
deploy_eso
# → bash: $1: unbound variable
```

---

## Fix

### Change 1 — `scripts/plugins/eso.sh` line 12: guard bare `$1`

**Exact old block (line 12):**

```bash
  if [[ "$1" == "-h" || "$1" == "--help" ]]; then
```

**Exact new block:**

```bash
  if [[ "${1:-}" == "-h" || "${1:-}" == "--help" ]]; then
```
