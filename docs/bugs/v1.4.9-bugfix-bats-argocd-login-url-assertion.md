# Bugfix: v1.4.9 — BATS assertion expects stale ArgoCD login URL log line

**Branch:** `k3d-manager-v1.4.9`
**File:** `scripts/tests/bin/acg_up.bats`

---

## Problem

`bats scripts/tests/bin/acg_up.bats` fails on test 1 with exit code 1 at the grep for
`ArgoCD browser login URL: https://argocd.shopping-cart.local`. That literal string no longer
exists in `bin/acg-up` — a prior commit changed the log line.

**Root cause:** `acg_up.bats` line 104 greps for the old log phrase; `bin/acg-up` line 362
now emits `ArgoCD reachable at https://${ARGOCD_BROWSER_HOST:-argocd.shopping-cart.local} (launchd: ${_argocd_browser_label})`.

---

## Reproduction

```bash
bats scripts/tests/bin/acg_up.bats
# not ok 1 acg-up sources the Argo CD plugin before readiness checks
# (in test file scripts/tests/bin/acg_up.bats, line 105)
# `[ "$status" -eq 0 ]' failed
```

---

## Fix

### Change 1 — `scripts/tests/bin/acg_up.bats` lines 104–106

**Exact old block:**

```bash
  run grep -nF 'ArgoCD browser login URL: https://argocd.shopping-cart.local' bin/acg-up
  [ "$status" -eq 0 ]
  [[ "$output" == *"ArgoCD browser login URL"* ]]
```

**Exact new block:**

```bash
  run grep -nF '_info "[acg-up] ArgoCD reachable at https://${ARGOCD_BROWSER_HOST:-argocd.shopping-cart.local} (launchd: ${_argocd_browser_label})"' bin/acg-up
  [ "$status" -eq 0 ]
  [[ "$output" == *"ArgoCD reachable at"* ]]
```
