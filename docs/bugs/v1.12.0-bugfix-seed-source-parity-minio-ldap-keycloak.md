## Problem

Commit `176ec5a6` wired `_seed_source_data` (copy-from-canonical-source) into the six
single-password generate-if-absent branches (`redis/cart`, `redis/orders-cache`,
`postgres/{orders,products,payment}`, `rabbitmq/default`) and hardened the KV helpers to
`curl --config`. The four **multi-field** branches were deliberately left out of that change for
reviewability: `minio/credentials`, `ldap/admin`, `keycloak/admin`, `keycloak/clients` still
**generate fresh random secrets when absent in the target** instead of copying from the canonical
source Vault. This means a fresh seed against an empty target diverges from the source of truth for
those four keys.

**Root cause:** the four multi-field else-branches were never updated to call `_seed_source_data`
before generating.

This is a latent parity gap, **not a regression** — the active cutover path is
`vault.sh:vault_seed_hub_into_context`, and in the default config `SEED_VAULT_SOURCE_*` resolves to
the same Vault. The fix brings these four keys to the same source-of-truth behavior the other six
already have.

---

## Reproduction

With a populated canonical source Vault and an empty target Vault, run
`shopping_cart_seed_sandbox_vault_kv`. Expected: `minio/credentials`, `ldap/admin`,
`keycloak/admin`, `keycloak/clients` are copied from the source. Actual: they are regenerated with
fresh random values, diverging from the source.

---

## Scope

In scope: the four multi-field else-branches + BATS coverage for the multi-field copy path.
The helper hardening (`curl --config`) and `_src_json` local declaration already landed in
`176ec5a6` — **do not touch them.** Reuse the existing `_src_json` local.

## Fix

The pattern mirrors `176ec5a6`: in the else-branch, try `_seed_source_data "<path>"`; if it returns
a non-empty object, `_vault_kv_put` the whole object and extract each field from `${_src_json}`;
otherwise fall back to the existing generate code (now nested in an inner `else`).

### Change 1 — `scripts/plugins/shopping_cart.sh`: `minio/credentials` else-branch

**Exact old block:**

```bash
  else
    _minio_root_user="minioadmin"
    _minio_root_password=$(openssl rand -base64 24 | tr -d '=+/')
    _vault_kv_put "{\"root-user\":\"${_minio_root_user}\",\"root-password\":\"${_minio_root_password}\"}" minio/credentials
  fi
```

**Exact new block:**

```bash
  else
    _src_json=$(_seed_source_data "minio/credentials")
    if [[ -n "${_src_json}" ]]; then
      _info "[acg-up] Copying minio/credentials from canonical source Vault"
      _vault_kv_put "${_src_json}" minio/credentials
      _minio_root_user=$(printf '%s' "${_src_json}" | jq -r '.["root-user"] // empty')
      _minio_root_password=$(printf '%s' "${_src_json}" | jq -r '.["root-password"] // empty')
    else
      _minio_root_user="minioadmin"
      _minio_root_password=$(openssl rand -base64 24 | tr -d '=+/')
      _vault_kv_put "{\"root-user\":\"${_minio_root_user}\",\"root-password\":\"${_minio_root_password}\"}" minio/credentials
    fi
  fi
```

> Note: `root-user`/`root-password` contain a hyphen, so the jq filter MUST use
> `.["root-user"]` bracket form — `.root-user` is invalid jq.
