# Bugfix: v1.10.0 — data-layer permanent OutOfSync (controller-injected volumeClaimTemplates fields)

**Branch:** `feat/v1.10.0-vault-auth-portable`
**Files:** `scripts/etc/argocd/applicationsets/data-git.yaml`, `scripts/tests/plugins/argocd_app_cluster_generator.bats`

---

## Problem

The ArgoCD `data-layer` Application (generated by the `data-git` ApplicationSet) is permanently
`OutOfSync` on the `ubuntu-hostinger` app cluster: all 7 StatefulSets show `OutOfSync` even
though every pod is `1/1 Ready` and the app is `Healthy` and serving. `selfHeal` cannot clear
it because `.spec.volumeClaimTemplates` is an immutable field — ArgoCD can never patch it, so
the diff never resolves.

**Root cause:** ArgoCD diffs the live StatefulSet's `.spec.volumeClaimTemplates[]` against git.
The Kubernetes StatefulSet controller injects fields that do not exist in the git manifest:
`.status` (e.g. `{"phase":"Pending"}`), `.apiVersion` (`v1`), and `.kind`
(`PersistentVolumeClaim`). The existing `ignoreDifferences` block only covers
`.spec.volumeClaimTemplates[].spec.storageClassName` and `.spec.volumeClaimTemplates[].spec.volumeMode`,
so these controller-injected fields produce a permanent, cosmetic diff. It is benign today but
masks any future real drift in the data-layer manifests.

This is distinct from `docs/bugs/2026-06-01-argocd-respectignoredifferences-statefulset.md`
(which targets the legacy per-app `argocd/applications/data-layer.yaml` in shopping-cart-infra,
and the `spec.*` immutable fields). The live `data-layer` app is generated by the ApplicationSet
`data-git.yaml` **in k3d-manager** — that is the authoritative `ignoreDifferences` source and
the file fixed here.

---

## Reproduction

```bash
# hub ArgoCD manages ubuntu-hostinger as a remote destination
kubectl --context ubuntu-hostinger -n shopping-cart-data get statefulset
# → all StatefulSets Ready, pods 1/1, but:
argocd app get data-layer            # → Sync Status: OutOfSync (StatefulSets), Health: Healthy
```

Expected: `Synced` once controller-injected volumeClaimTemplates fields are ignored.
Actual: permanent `OutOfSync` on the StatefulSet resources.

---

## Fix

### Change 1 — `scripts/etc/argocd/applicationsets/data-git.yaml`: extend `jqPathExpressions`

**Exact old block (lines 35–40):**

```yaml
      ignoreDifferences:
        - group: apps
          kind: StatefulSet
          jqPathExpressions:
            - .spec.volumeClaimTemplates[].spec.storageClassName
            - .spec.volumeClaimTemplates[].spec.volumeMode
```

**Exact new block:**

```yaml
      ignoreDifferences:
        - group: apps
          kind: StatefulSet
          jqPathExpressions:
            - .spec.volumeClaimTemplates[].spec.storageClassName
            - .spec.volumeClaimTemplates[].spec.volumeMode
            - .spec.volumeClaimTemplates[].status
            - .spec.volumeClaimTemplates[].apiVersion
            - .spec.volumeClaimTemplates[].kind
```

### Change 2 — `scripts/tests/plugins/argocd_app_cluster_generator.bats`: assert the new ignore paths

**Exact old block (lines 20–27):**

```bash
@test "argocd app cluster generator: data-git exists and targets shopping-cart-data" {
  run test -f "${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/data-git.yaml"
  [ "$status" -eq 0 ]
  run grep -F -- "k3d-manager/role: app-cluster" "${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/data-git.yaml"
  [ "$status" -eq 0 ]
  run grep -F -- "namespace: shopping-cart-data" "${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/data-git.yaml"
  [ "$status" -eq 0 ]

```

**Exact new block:**

```bash
@test "argocd app cluster generator: data-git exists and targets shopping-cart-data" {
  run test -f "${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/data-git.yaml"
  [ "$status" -eq 0 ]
  run grep -F -- "k3d-manager/role: app-cluster" "${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/data-git.yaml"
  [ "$status" -eq 0 ]
  run grep -F -- "namespace: shopping-cart-data" "${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/data-git.yaml"
  [ "$status" -eq 0 ]
}

@test "argocd app cluster generator: data-git ignores controller-injected volumeClaimTemplates fields" {
  run grep -F -- ".spec.volumeClaimTemplates[].status" "${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/data-git.yaml"
  [ "$status" -eq 0 ]
  run grep -F -- ".spec.volumeClaimTemplates[].apiVersion" "${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/data-git.yaml"
  [ "$status" -eq 0 ]
  run grep -F -- ".spec.volumeClaimTemplates[].kind" "${BATS_TEST_DIRNAME}/../../etc/argocd/applicationsets/data-git.yaml"
  [ "$status" -eq 0 ]
}
```
