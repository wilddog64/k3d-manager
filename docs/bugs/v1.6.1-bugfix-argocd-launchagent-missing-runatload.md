# Bugfix: v1.6.1 — ArgoCD port-forward LaunchAgent missing RunAtLoad

**Branch:** `k3d-manager-v1.6.1`
**Discovered:** 2026-06-05
**File:** `~/Library/LaunchAgents/com.k3d-manager.argocd-port-forward.plist`

---

## Problem

`argocd.3ai-talk.org` returns Cloudflare 502 Bad Gateway intermittently.
Cloudflare is healthy; the origin host is the error. The root cause is that
`localhost:8080` (the ArgoCD port-forward target in `scripts/etc/cloudflared/config.yml`)
goes dark after a mac sleep, wake, or OS session restart.

**Root cause:** `com.k3d-manager.argocd-port-forward.plist` had `KeepAlive: true` but
no `RunAtLoad: true`. macOS launchd only auto-starts a LaunchAgent on boot/login if
`RunAtLoad` is set. Without it, the agent stays registered in `~/Library/LaunchAgents/`
but is not bootstrapped into the active launchd session after a restart — so it never
starts and `KeepAlive` never kicks in.

**Suspected trigger:** macOS upgrade or user session restart unloaded the agent from
the gui domain. The plist survived on disk but was no longer active in launchctl
(`launchctl list | grep argocd` returned nothing, `launchctl print gui/<uid>/com.k3d-manager.argocd-port-forward`
returned "Could not find service").

---

## Symptoms

1. `argocd.3ai-talk.org` → Cloudflare 502
2. `lsof -i :8080` returns empty (nothing listening)
3. `pgrep -f argocd-port-forward` returns nothing (or a stale PID with no child kubectl process)
4. `launchctl list | grep argocd` returns empty

---

## Fix Applied (2026-06-05, manually)

Added `RunAtLoad: true` to the installed plist and re-bootstrapped:

```xml
<!-- before -->
<key>KeepAlive</key>
<true/>

<!-- after -->
<key>RunAtLoad</key>
<true/>
<key>KeepAlive</key>
<true/>
```

```bash
launchctl bootout "gui/$(id -u)/com.k3d-manager.argocd-port-forward" 2>/dev/null || true
launchctl bootstrap "gui/$(id -u)" ~/Library/LaunchAgents/com.k3d-manager.argocd-port-forward.plist
```

---

## Recovery Steps (if 502 recurs)

```bash
# 1. Confirm the agent is not loaded
launchctl list | grep argocd

# 2. If empty — bootstrap it
launchctl bootstrap "gui/$(id -u)" ~/Library/LaunchAgents/com.k3d-manager.argocd-port-forward.plist

# 3. Verify
curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/healthz
# expected: 200

# 4. If plist is missing RunAtLoad, add it (edit file, then re-bootstrap)
```

---

## Follow-On Work

The installed plist at `~/Library/LaunchAgents/com.k3d-manager.argocd-port-forward.plist`
was edited directly (no template source for this agent — it was generated by an earlier
install step). The `scripts/etc/launchd/` directory does not contain a template for
this agent. If the agent is ever reinstalled from scratch, `RunAtLoad: true` must be
included in the generated plist.

Consider adding an `install-argocd-port-forward` Makefile target (same pattern as
`install-vault-port-forward` shipped in this release) so the plist can be regenerated
reproducibly with the correct keys.
