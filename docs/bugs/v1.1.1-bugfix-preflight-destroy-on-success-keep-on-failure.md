# Bugfix: v1.1.1 — preflight `--destroy` must keep the vCluster on failure

**Branch:** `k3d-manager-v1.7.1`
**File:** `bin/cluster-preflight`

---

## Problem

The `--destroy` cleanup trap (`_cluster_preflight_cleanup`) tears down the vCluster and prunes
its ArgoCD resources **unconditionally** — it captures `rc=$?` at trap entry but only checks
`PREFLIGHT_MODE == "--destroy"`, never `rc`. So a **failed** preflight run (e.g. a deploy that
never reaches Synced/Healthy) is destroyed along with all evidence, leaving nothing to
investigate.

The required CI lifecycle policy is: **delete on success, keep on failure for investigation.**

**Root cause:** the teardown `if` in the cleanup trap is gated only on `PREFLIGHT_MODE`, not on
the captured exit code `rc`.

---

## Reproduction

```bash
make preflight NAME=pr99 MODE=--destroy
# If the deploy fails (wait timeout / unhealthy apps), the script exits non-zero AND the
# trap still destroys pr99 — the failed cluster is gone, nothing left to debug.
# Desired: a failed --destroy run keeps pr99; a successful one deletes it.
```

---

## Fix

> Both edits are in `bin/cluster-preflight`. Line numbers are approximate — match on the
> literal blocks. `rc` (`local rc=$?`) and `REPO_ROOT` (global) are both in scope inside
> `_cluster_preflight_cleanup`.

### Change 1 — cleanup trap: gate teardown on `rc`, keep on failure

**Exact old block (inside `_cluster_preflight_cleanup`, lines ~74–75):**

```bash
  if [[ "${PREFLIGHT_MODE}" == "--destroy" ]]; then
    _info "[preflight] Destroying ${PREFLIGHT_NAME} and pruning preflight ArgoCD resources..."
```

**Exact new block:**

```bash
  if [[ "${PREFLIGHT_MODE}" == "--destroy" && "${rc}" -ne 0 ]]; then
    _info "[preflight] --destroy: run FAILED (rc=${rc}) — keeping ${PREFLIGHT_NAME} and its ArgoCD resources for investigation. Clean up manually with: ${REPO_ROOT}/scripts/k3d-manager vcluster_destroy ${PREFLIGHT_NAME}"
  elif [[ "${PREFLIGHT_MODE}" == "--destroy" ]]; then
    _info "[preflight] Destroying ${PREFLIGHT_NAME} and pruning preflight ArgoCD resources..."
```

> Only the opening guard + the first `_info` change. The entire teardown body
> (`_cluster_preflight_remove_platform_destination`, the `_kubectl delete ...` lines, the
> `vcluster_destroy` call) and the closing `fi` stay exactly as they are — they now live under
> the `elif` (success) branch unchanged.

### Change 2 — usage string: document the new `--destroy` semantics

**Exact old block (inside `_cluster_preflight_usage`, line ~17):**

```bash
  --destroy  create, deploy, then tear down + prune ArgoCD resources on exit
```

**Exact new block:**

```bash
  --destroy  create, deploy, then tear down on success; keep on failure for investigation
```
