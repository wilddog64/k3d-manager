# Bugfix: v1.1.0 — kubectl label fires before k3s API server is ready

**Branch:** `k3d-manager-v1.1.0`
**Files:** `scripts/lib/providers/k3s-gcp.sh`

---

## Problem

After `k3sup install` returns, `_provider_k3s_gcp_deploy_cluster` immediately runs
`kubectl label nodes` — but the k3s API server (port 6443) may not be accepting
connections yet. k3sup reports success as soon as the k3s service starts, not when
the API server is fully ready.

**Observed failure:**

```
INFO: [k3s-gcp] Labeling node...
E0413 04:57:23 memcache.go:265] couldn't get current server API group list:
  Get "https://35.192.54.48:6443/api?timeout=32s": dial tcp 35.192.54.48:6443: i/o timeout
E0413 04:57:53 memcache.go:265] ... i/o timeout
E0413 04:58:23 memcache.go:265] ... i/o timeout
```

kubectl retries internally for ~90s before giving up, causing `make up` to fail.

**Root cause:** No API server readiness probe between `k3sup install` and `kubectl label`
(line 217–218 of `scripts/lib/providers/k3s-gcp.sh`).

---

## Reproduction

```bash
make up CLUSTER_PROVIDER=k3s-gcp URL=<sandbox-url>
# → k3sup install succeeds
# → kubectl label nodes ...
# → i/o timeout (API server not ready yet)
```

---

## Fix

### Change 1 — `scripts/lib/providers/k3s-gcp.sh`: add API server readiness probe before kubectl label

Poll port 6443 every 10 seconds for up to 5 minutes (30 retries), same pattern as
the existing SSH readiness probe.

**Exact old block (lines 216–218):**

```bash
  chmod 600 "${HOME}/.kube/config" "${_GCP_KUBECONFIG}"

  _info "[k3s-gcp] Labeling node..."
  kubectl --context k3s-gcp label nodes --all k3d-manager/node-type=server --overwrite || return 1
```

**Exact new block:**

```bash
  chmod 600 "${HOME}/.kube/config" "${_GCP_KUBECONFIG}"

  _info "[k3s-gcp] Waiting for k3s API server on ${external_ip}:6443..."
  local _api_retries=30
  local _api_i=0
  while (( _api_i < _api_retries )); do
    if nc -z -w 5 "${external_ip}" 6443 2>/dev/null; then
      _info "[k3s-gcp] k3s API server ready on ${external_ip}:6443"
      break
    fi
    _api_i=$(( _api_i + 1 ))
    _info "[k3s-gcp] API server not ready yet (${_api_i}/${_api_retries}) — retrying in 10s..."
    sleep 10
  done
  if (( _api_i >= _api_retries )); then
    _err "[k3s-gcp] k3s API server not available on ${external_ip}:6443 after $(( _api_retries * 10 ))s"
    return 1
  fi

  _info "[k3s-gcp] Labeling node..."
  kubectl --context k3s-gcp label nodes --all k3d-manager/node-type=server --overwrite || return 1
```
