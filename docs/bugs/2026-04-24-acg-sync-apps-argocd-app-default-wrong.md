# Bug: `bin/acg-sync-apps` default ARGOCD_APP is 'data-layer' — app never exists

**Date:** 2026-04-24
**Status:** OPEN
**Severity:** HIGH (make sync-apps exits 1 on every clean run — app not found)
**Branch:** `k3d-manager-v1.1.0`

## Summary

`bin/acg-sync-apps` defaults to `ARGOCD_APP=data-layer`. After `deploy_argocd_bootstrap`
runs, the available apps in ArgoCD are:

```
cicd/rollout-demo-default
cicd/rollout-demo-staging
```

The `data-layer` app never exists:

```
INFO: [sync-apps] ERROR: ArgoCD app 'data-layer' not found — is bootstrap complete?
INFO: [sync-apps] Available apps (set ARGOCD_APP=<name> to target a different one):
cicd/rollout-demo-default
cicd/rollout-demo-staging
make: *** [sync-apps] Error 1
```

## Root Cause

Three ApplicationSets are deployed by `_argocd_deploy_applicationsets`:

| ApplicationSet | Generator | Apps generated |
|---|---|---|
| `demo-rollout.yaml` | list (static) | `rollout-demo-default`, `rollout-demo-staging` — always present |
| `services-git.yaml` | git `services/*` | zero — no `services/` directory exists in the repo |
| `platform-helm.yaml` | clusters with `environment: dev` label | zero — no cluster has that label at bootstrap time |

Only `rollout-demo-default` and `rollout-demo-staging` are ever created after a clean
`deploy_argocd_bootstrap`. The `data-layer` default was a stale reference to a planned
but never-implemented service layout.

## Fix

Change the default `ARGOCD_APP` in `bin/acg-sync-apps` from `data-layer` to
`rollout-demo-default` — the canonical post-bootstrap app that is always present.

**File:** `bin/acg-sync-apps`

**Old (line 24):**
```bash
ARGOCD_APP="${ARGOCD_APP:-data-layer}"
```

**New (line 24):**
```bash
ARGOCD_APP="${ARGOCD_APP:-rollout-demo-default}"
```

One word changed. No other lines change.
