# Bugfix: v1.1.1 — preflight vcluster control plane crashloops (heavy distro + 500m limit)

**Branch:** `k3d-manager-v1.7.1`
**Files:** `scripts/etc/vcluster/values-preflight.yaml`

---

## Problem

After the isolation + cluster-name-label fixes let the platform clone install, the preflight
still never converges: the vcluster's own control plane crashloops, so every workload that talks
to the vcluster API (`argocd-pf1-server`, etc.) crashloops too, and the run times out (rc=1).

**Root cause (pinned live on `pf1`, 2026-06-16):** `values-preflight.yaml` caps
`controlPlane.statefulSet.resources.limits` at **`cpu: 500m` / `memory: 512Mi`**, and **no
`distro:` is set**, so vcluster 0.32.1 runs its heavy default **k8s distro** (embedded etcd +
standalone `kube-controller-manager`; images `loft-sh/vcluster-pro:0.32.1` +
`loft-sh/kubernetes:v1.35.0`). The 500m CPU limit throttles that control plane so lease
renewals exceed the 5s deadline:

```
Failed to renew lease ... Put "https://127.0.0.1:6443/.../leases/kube-controller-manager?timeout=5s": context deadline exceeded
error running controller-manager: error running command /binaries/kube-controller-manager: exit status 1
syncer Liveness/Readiness probe failed: .../healthz: context deadline exceeded
```

→ `kube-controller-manager` exits → the `syncer` `/healthz` times out → kubelet kills the
`pf1-0` syncer (liveness) → CrashLoop (8 restarts) → vcluster API (`kubernetes.default.svc`
`10.43.120.152:443`) flaps → `argocd-pf1-server` crashloops → cascade.

**This is NOT host capacity** — the node sits at 21% CPU / 34% memory (2 vCPU / 7.7Gi). It is
the **pod limit + heavy distro**. Two levers fix it: switch to the lightweight **k3s distro**
(single binary, sqlite — no etcd, far less disk/CPU pressure on the VPS) and raise the limit so
the control plane is not throttled during startup.

## Reproduction / evidence

```bash
kubectl --context ubuntu-hostinger -n vclusters get pod pf1-0 \
  -o jsonpath='{.status.containerStatuses[?(@.name=="syncer")].restartCount}'   # → 8, CrashLoop
kubectl --context ubuntu-hostinger -n vclusters get pod pf1-0 \
  -o jsonpath='{range .spec.initContainers[*]}{.name}{" -> "}{.image}{"\n"}{end}'
#   kubernetes -> ghcr.io/loft-sh/kubernetes:v1.35.0   (heavy k8s distro, default)
kubectl --context ubuntu-hostinger top node     # → ~21% CPU / ~34% mem — node has headroom
```

## Fix

### Change 1 — `scripts/etc/vcluster/values-preflight.yaml`: k3s distro + raised limits

**Exact old file contents:**

```yaml
controlPlane:
  service:
    spec:
      type: NodePort
  statefulSet:
    resources:
      requests:
        cpu: 200m
        memory: 256Mi
      limits:
        cpu: 500m
        memory: 512Mi
```

**Exact new file contents:**

```yaml
controlPlane:
  distro:
    k3s:
      enabled: true
  service:
    spec:
      type: NodePort
  statefulSet:
    resources:
      requests:
        cpu: 300m
        memory: 384Mi
      limits:
        cpu: 1500m
        memory: 1536Mi
```
