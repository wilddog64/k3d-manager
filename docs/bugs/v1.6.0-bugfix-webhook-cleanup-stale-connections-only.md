# Bugfix: v1.6.0 — webhook cleanup should only remove stale connections

**Branch:** `k3d-manager-v1.6.0`
**File:** `bin/k3dm-webhook`

---

## Problem

`_run_cleanup()` always runs `make down KEEP_LOCAL=1` after any failed `acg-up` job. When
the failure is at credential capture (before CloudFormation is invoked), there are no AWS
resources to tear down — the command runs `acg_teardown`, says "AWS resources removed", and
is both misleading and potentially dangerous (could delete a separately-provisioned cluster).

**Root cause:** `_run_cleanup` has no guard — it always runs the full `make down` teardown
regardless of which stage the failure occurred at.

---

## Reproduction

1. Run `/acg-up` with an expired Pluralsight session.
2. Observe Slack: failure analysis fires, then "Cleanup triggered — running `make down KEEP_LOCAL=1`"
   followed by "✅ Cleanup complete — AWS resources removed" — but no AWS resources were ever created.

---

## Fix

### Change 1 — `bin/k3dm-webhook`: replace `make down` with lightweight stale-connection cleanup

**Exact old block:**
```python
def _run_cleanup(job_id, provider):
    """Run make down KEEP_LOCAL=1 in the background after a failed acg-up."""
    _notify_job(job_id, f"🧹 *Cleanup triggered* ({provider}) — running `make down KEEP_LOCAL=1`")
    try:
        cmd = ["make", "down", "KEEP_LOCAL=1"]
        proc = subprocess.Popen(
            cmd,
            cwd=REPO_ROOT,
            stdout=subprocess.PIPE,
            stderr=subprocess.STDOUT,
            text=True,
            preexec_fn=os.setsid,
        )
        pgid = os.getpgid(proc.pid)
        out_lines = []

        def _drain():
            for line in proc.stdout:
                out_lines.append(line)

        reader = threading.Thread(target=_drain, daemon=True)
        reader.start()
        proc.wait()
        try:
            os.killpg(pgid, signal.SIGTERM)
        except (ProcessLookupError, OSError):
            pass
        reader.join(timeout=5)
        if proc.returncode == 0:
            _notify_job(job_id, f"✅ *Cleanup complete* ({provider}) — AWS resources removed")
        else:
            _notify_job(job_id, f"⚠️ *Cleanup failed* ({provider}) — `make down` exited {proc.returncode}; manual teardown may be needed")
    except Exception as exc:
        _notify_job(job_id, f"⚠️ *Cleanup error* ({provider}): {exc}")
```

**Exact new block:**
```python
def _run_cleanup(job_id, provider):
    """Remove stale SSH tunnel and kubeconfig context after a failed acg-up."""
    _notify_job(job_id, f"🧹 *Cleanup triggered* ({provider}) — removing stale SSH tunnel and kubeconfig context")
    try:
        subprocess.run(
            ["pkill", "-f", "autossh"],
            capture_output=True,
        )
        subprocess.run(
            ["kubectl", "config", "delete-context", "ubuntu-k3s"],
            capture_output=True,
        )
        _notify_job(job_id, f"✅ *Cleanup complete* ({provider}) — stale connections removed")
    except Exception as exc:
        _notify_job(job_id, f"⚠️ *Cleanup error* ({provider}): {exc}")
```
