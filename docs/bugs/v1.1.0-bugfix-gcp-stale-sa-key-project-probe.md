# Bugfix: v1.1.0 — Stale SA key causes pre-flight failure on new GCP sandbox

**Branch:** `k3d-manager-v1.1.0`
**Files:** `scripts/lib/providers/k3s-gcp.sh`

---

## Problem

`make up CLUSTER_PROVIDER=k3s-gcp` fails with:

```
ERROR: [k3s-gcp] Compute access check failed on project playground-s-11-<old-id>.
```

This happens when a new ACG sandbox is started after a previous one has expired. The old
SA key file (`~/.local/share/k3d-manager/gcp-service-account.json`) is still on disk with
the expired project's `project_id`. `_gcp_load_credentials` sees the file as valid and
skips Playwright extraction, so `GCP_PROJECT` points to the dead project. Every subsequent
gcloud call fails because the project no longer exists.

**Root cause:** `_gcp_load_credentials` trusts the cached SA key if the file exists and
has a non-null `project_id`, but never verifies the project is still accessible before
skipping Playwright.

---

## Reproduction

```bash
# Start a new ACG GCP sandbox (different project from last time)
# Old key still on disk: ~/.local/share/k3d-manager/gcp-service-account.json
make up CLUSTER_PROVIDER=k3s-gcp
# → INFO: [k3s-gcp] SA key valid on disk — skipping Playwright extraction
# → ERROR: [k3s-gcp] Compute access check failed on project playground-s-11-<old-id>.
```

Manual workaround (before fix):
```bash
rm ~/.local/share/k3d-manager/gcp-service-account.json
make up CLUSTER_PROVIDER=k3s-gcp URL=<new-sandbox-url>
```

---

## Fix

### Change 1 — `scripts/lib/providers/k3s-gcp.sh`: probe project before trusting cache

Replace the unconditional cache hit in `_gcp_load_credentials` (lines 59–68) with a
`gcloud projects describe` probe. If the project is gone (sandbox expired), delete the
stale key and fall through to Playwright extraction.

**Exact old block (lines 59–68):**

```bash
  if [[ -f "${_default_key}" && -n "${_cached_project}" && "${_cached_project}" != "null" ]]; then
    _info "[k3s-gcp] SA key valid on disk — skipping Playwright extraction"
    export GCP_PROJECT="${_cached_project}"
    export GOOGLE_APPLICATION_CREDENTIALS="${_default_key}"
    local _active_account
    _active_account=$(gcloud auth list --filter="status:ACTIVE" --format="value(account)" 2>/dev/null || true)
    [[ -n "${_active_account}" ]] && export GCP_USERNAME="${_active_account}"
  else
    _info "[k3s-gcp] Extracting GCP sandbox credentials..."
    gcp_get_credentials "${sandbox_url}" || return 1
  fi
```

**Exact new block:**

```bash
  if [[ -f "${_default_key}" && -n "${_cached_project}" && "${_cached_project}" != "null" ]]; then
    if gcloud projects describe "${_cached_project}" --quiet >/dev/null 2>&1; then
      _info "[k3s-gcp] SA key valid on disk — skipping Playwright extraction"
      export GCP_PROJECT="${_cached_project}"
      export GOOGLE_APPLICATION_CREDENTIALS="${_default_key}"
      local _active_account
      _active_account=$(gcloud auth list --filter="status:ACTIVE" --format="value(account)" 2>/dev/null || true)
      [[ -n "${_active_account}" ]] && export GCP_USERNAME="${_active_account}"
    else
      _info "[k3s-gcp] Cached project ${_cached_project} not accessible — sandbox may have changed; re-extracting credentials..."
      rm -f "${_default_key}"
      gcp_get_credentials "${sandbox_url}" || return 1
    fi
  else
    _info "[k3s-gcp] Extracting GCP sandbox credentials..."
    gcp_get_credentials "${sandbox_url}" || return 1
  fi
```
