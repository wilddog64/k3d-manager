# Bugfix: v1.4.11 — ESO sync saturation timeout in shopping_cart_sync_vault_backed_secrets

**Branch:** `k3d-manager-v1.4.11`
**File:** `scripts/plugins/shopping_cart.sh`

---

## Problem

`make up` intermittently fails with:

```
timed out waiting for the condition on externalsecrets/postgres-products-app
make: *** [up] Error 1
```

14 ExternalSecrets are applied simultaneously with `kubectl apply -f secrets/`. The current
loop annotates each ESO with `force-sync` and then immediately waits 180s before moving to
the next. While one ESO is being waited on, the others receive no `force-sync` annotation —
the ESO controller sees all 14 objects at once and begins syncing all of them. By the time
the 4th ESO (`postgres-products-app`) is annotated and waited on, the controller is saturated
with concurrent Vault fetches; the 180s window expires before it processes that ESO's sync.

**Root cause:** The annotate and wait operations are interleaved in a single loop, so each
ESO only gets its `force-sync` annotation *after* the previous ESO's 180s wait completes.
The ESO controller is racing against a per-resource timeout instead of getting all sync
triggers up front.

---

## Reproduction

Run `make up` against a cold cluster. `postgres-products-app` (4th ESO in the list) times
out even though Vault has valid data at `secret/data/postgres/products` and the ESO becomes
`Ready` several seconds after the 180s deadline.

---

## Fix

### Change 1 — `scripts/plugins/shopping_cart.sh`: split annotate loop from wait loop

**Exact old block (lines 176–189):**

```bash
  local externalsecret namespace name _es_sync_ts
  _es_sync_ts=$(date +%s)
  for externalsecret in "${externalsecrets[@]}"; do
    namespace="${externalsecret%%/*}"
    name="${externalsecret##*/}"
    if ! kubectl get externalsecret "${name}" -n "${namespace}" --context ubuntu-k3s >/dev/null 2>&1; then
      _warn "[shopping_cart] ExternalSecret ${namespace}/${name} not found — skipping wait"
      continue
    fi
    kubectl annotate externalsecret "${name}" -n "${namespace}" --context ubuntu-k3s \
      force-sync="${_es_sync_ts}" --overwrite >/dev/null
    kubectl wait --for=condition=Ready --timeout=180s \
      externalsecret/"${name}" -n "${namespace}" --context ubuntu-k3s
  done
```

**Exact new block:**

```bash
  local externalsecret namespace name _es_sync_ts
  _es_sync_ts=$(date +%s)
  local -a _existing_es=()
  for externalsecret in "${externalsecrets[@]}"; do
    namespace="${externalsecret%%/*}"
    name="${externalsecret##*/}"
    if ! kubectl get externalsecret "${name}" -n "${namespace}" --context ubuntu-k3s >/dev/null 2>&1; then
      _warn "[shopping_cart] ExternalSecret ${namespace}/${name} not found — skipping wait"
      continue
    fi
    _existing_es+=("${externalsecret}")
    kubectl annotate externalsecret "${name}" -n "${namespace}" --context ubuntu-k3s \
      force-sync="${_es_sync_ts}" --overwrite >/dev/null
  done

  for externalsecret in "${_existing_es[@]}"; do
    namespace="${externalsecret%%/*}"
    name="${externalsecret##*/}"
    kubectl wait --for=condition=Ready --timeout=300s \
      externalsecret/"${name}" -n "${namespace}" --context ubuntu-k3s
  done
```
