## Problem

The Allow button times out (30000ms) with page body showing account chooser content:

```
window.IJ_values = [[false,[["/AccountChooser",null,...]],'',...
```

The `''` second element means no account is selected — the page is at an account chooser state, not the OAuth consent page. The visible button scan returns nothing because account chooser rows are `div[data-identifier]` (role="link"), not `<button>` elements.

**Root cause:** The `secondChooserVisible` block uses a one-time 5s check. In some runs the second account chooser appears after the 5s window (e.g. due to page load delay after Continue), so the check is skipped and the flow jumps to the Allow wait — but the account chooser is still blocking.

**Fix:** Add a second pass of the account-chooser check immediately before the Allow `waitFor`. This runs regardless of whether `secondChooserVisible` was true, catching any late-appearing chooser.

## Reproduction

```
make up CLUSTER_PROVIDER=k3s-gcp
# observe:
# INFO: visible button: (none)
# INFO: page body on Allow timeout: ...AccountChooser...IJ_values...''...
# ERROR: locator.waitFor: Timeout 30000ms exceeded.
```

## Fix

### Change 1 — `scripts/playwright/gcp_login.js`: add late account chooser check before Allow wait

**Exact old block (between the secondChooserVisible block and the Allow locator):**

```js
    // Try Allow variants — Google sometimes uses different labels
    const allowBtn = page.locator(
      'button:has-text("Allow"), button:has-text("Grant access"), button:has-text("Yes, I\'m in")'
    ).first();
```

**Exact new block:**

```js
    // Late account chooser check — runs unconditionally in case the chooser appeared
    // after the secondChooserVisible 5s window (e.g. delayed page load after Continue)
    const lateChooser = page.locator('div[data-identifier]').filter({ hasText: username });
    const lateChooserVisible = await lateChooser.first().isVisible({ timeout: 5000 }).catch(() => false);
    if (lateChooserVisible) {
      console.error(`INFO: Late account chooser detected — clicking ${username}`);
      await lateChooser.first().click({ force: true });
      await page.waitForTimeout(2000);
      console.error(`INFO: URL after late chooser: ${page.url()}`);
      // Post-late-chooser Continue
      const lateContinueBtn = page.locator('button:has-text("Continue")').first();
      const lateContinueVisible = await lateContinueBtn.isVisible({ timeout: 5000 }).catch(() => false);
      if (lateContinueVisible) {
        console.error('INFO: Clicking "Continue" after late chooser');
        await lateContinueBtn.click();
        await page.waitForTimeout(1000);
        console.error(`INFO: URL after late-chooser Continue: ${page.url()}`);
      }
    }

    // Try Allow variants — Google sometimes uses different labels
    const allowBtn = page.locator(
      'button:has-text("Allow"), button:has-text("Grant access"), button:has-text("Yes, I\'m in")'
    ).first();
```
