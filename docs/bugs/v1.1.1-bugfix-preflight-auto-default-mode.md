# Bugfix: v1.1.1 — preflight `--auto` smart teardown becomes the default mode

**Branch:** `k3d-manager-v1.7.1`
**Files:** `bin/cluster-preflight`, `Makefile`

---

## Problem

The preflight default mode is `--keep` — every run leaves the vCluster running, so CI would
accumulate clusters and a human must decide what to tear down. The intended CI lifecycle is:
**a successful run cleans itself up; a failed run is kept for investigation.** That smart
behavior currently only exists behind the explicitly-typed `--destroy` flag, and `--destroy`
is a misleading name because it does NOT always destroy (it keeps on failure).

**Root cause:** the smart teardown logic is gated on the `--destroy` flag and is not the
default; the default (`${2:---keep}`) is unconditional keep.

**Fix:** rename the smart flag `--destroy` → `--auto`, make `--auto` the **default** mode, and
keep `--keep` as the explicit override for "always keep, even on success." `--reuse` is
unchanged. The smart success/failure logic itself is already correct (shipped in `75c8103c`) —
this change only renames the flag and flips the default.

### Final mode semantics

| Mode | Behavior |
|------|----------|
| `--auto` (default) | create, deploy; **delete on success, keep on failure** for investigation |
| `--keep` | create, deploy, always leave running (even on success) |
| `--reuse` | skip create/register; deploy into an existing vCluster, keep it |

---

## Reproduction

```bash
make preflight NAME=pr99
# Current: runs in --keep — vCluster is left running even on a clean success.
# Desired: runs in --auto — deletes pr99 on success, keeps it on failure.
```

> All edits except Change 6 are in `bin/cluster-preflight`. Match on the literal blocks; line

### Change 1 — `bin/cluster-preflight`: usage string (lines ~15–18)

**Exact old block:**

```bash
Usage: cluster-preflight <name> [--keep|--destroy|--reuse]
  --keep     create the vCluster, deploy the stack, leave it running (default)
  --destroy  create, deploy, then tear down on success; keep on failure for investigation
  --reuse    skip create/register; deploy into an already-registered vCluster, keep it
```

**Exact new block:**

```bash
Usage: cluster-preflight <name> [--auto|--keep|--reuse]
  --auto     create, deploy, then tear down on success; keep on failure for investigation (default)
  --keep     create the vCluster, deploy the stack, always leave it running
  --reuse    skip create/register; deploy into an already-registered vCluster, keep it
```

### Change 4 — `bin/cluster-preflight`: cleanup trap teardown guard (lines ~74–76)

**Exact old block:**

```bash
  if [[ "${PREFLIGHT_MODE}" == "--destroy" && "${rc}" -ne 0 ]]; then
    _info "[preflight] --destroy: run FAILED (rc=${rc}) — keeping ${PREFLIGHT_NAME} and its ArgoCD resources for investigation. Clean up manually with: ${REPO_ROOT}/scripts/k3d-manager vcluster_destroy ${PREFLIGHT_NAME}"
```

**Exact new block:**

```bash

  elif [[ "${PREFLIGHT_MODE}" == "--auto" ]]; then
    _info "[preflight] Destroying ${PREFLIGHT_NAME} and pruning preflight ArgoCD resources..."
```

> Only the two guard lines + the first `_info` change. The entire teardown body
> (`_cluster_preflight_remove_platform_destination`, the `_kubectl delete ...` lines, the
> `vcluster_destroy` call), the closing `fi`, and the trap's final `return "${rc}"` stay exactly
> as they are.
