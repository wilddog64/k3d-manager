# Bugfix: v1.4.11 — acg-down prompts for password on macOS Tahoe due to --interactive-sudo

**Branch:** `k3d-manager-v1.4.11`
**File:** `bin/acg-down`

---

## Problem

`make down` (or `acg-down`) prompts `Password:` and also emits
`sudo: unable to allocate pty: Device not configured` on macOS Tahoe (Darwin 25).

The mac block in `acg-down` that stops the Keycloak and ArgoCD browser listener
LaunchDaemons uses `--interactive-sudo` on 5 `_run_command` calls. `--interactive-sudo`
passes no `-n` flag to `sudo`, allowing sudo to prompt for a password and attempt PTY
allocation. On macOS Tahoe, PTY allocation fails with "unable to allocate pty: Device not
configured", and when the sudo credential cache is cold, the terminal shows `Password:`.

`--prefer-sudo` is the correct flag: it uses `sudo -n` (non-interactive, no password prompt,
no PTY). `sudo -n launchctl bootout system` is already confirmed to work without a password
(NOPASSWD or warm cache). For the `rm -f` calls that may not have NOPASSWD coverage,
`--soft` already ensures graceful failure.

**Root cause:** 5 `_run_command` calls in `bin/acg-down` lines 112, 119, 131, 138, 139
use `--interactive-sudo` instead of `--prefer-sudo`.

---

## Reproduction

Run `make down` on macOS Tahoe with an expired sudo credential cache. Observe:
```
sudo: unable to allocate pty: Device not configured
launchctl command failed (1): sudo launchctl bootout system /Library/LaunchDaemons/com.k3d-...
Password:
```

---

## Fix

### Change 1 — `bin/acg-down` lines 112–139: replace `--interactive-sudo` with `--prefer-sudo`

**Exact old block (lines 112–139):**

```bash
  if _run_command --interactive-sudo --soft -- launchctl bootout system "${_keycloak_browser_plist}" \
      >"${_keycloak_browser_launchctl_log}" 2>&1; then
    _info "[acg-down] Keycloak browser HTTP listener launchd daemon unloaded"
  elif [[ -s "${_keycloak_browser_launchctl_log}" ]]; then
    tail -n 20 "${_keycloak_browser_launchctl_log}" >&2 || true
    _warn "[acg-down] existing Keycloak browser HTTP listener was not loaded; continuing"
  fi
  _run_command --interactive-sudo --soft -- rm -f "${_keycloak_browser_plist}" "${_keycloak_browser_wrapper}" >/dev/null 2>&1 || true

  _info "[acg-down] Stopping ArgoCD browser HTTPS listener launchd daemon..."
  _argocd_browser_label="${ARGOCD_BROWSER_LISTENER_LABEL:-com.k3d-manager.argocd-browser-https}"
  _argocd_browser_plist="${ARGOCD_BROWSER_LISTENER_PLIST:-/Library/LaunchDaemons/${_argocd_browser_label}.plist}"
  _argocd_browser_wrapper="${ARGOCD_BROWSER_LISTENER_WRAPPER:-${HOME}/.local/share/k3d-manager/argocd-browser-https.sh}"
  _argocd_browser_tls_dir="${ARGOCD_BROWSER_TLS_DIR:-${HOME}/.local/share/k3d-manager/argocd-browser-https-tls}"
  _argocd_browser_tls_cert="${ARGOCD_BROWSER_TLS_CERT_FILE:-${_argocd_browser_tls_dir}/fullchain.crt}"
  _argocd_browser_tls_key="${ARGOCD_BROWSER_TLS_KEY_FILE:-${_argocd_browser_tls_dir}/tls.key}"
  _argocd_browser_tls_ca="${ARGOCD_BROWSER_TLS_CA_FILE:-${_argocd_browser_tls_dir}/ca.crt}"
  _argocd_browser_launchctl_log="${HOME}/.local/share/k3d-manager/argocd-browser-https-launchctl.log"
  : > "${_argocd_browser_launchctl_log}"
  if _run_command --interactive-sudo --quiet --soft -- launchctl bootout system "${_argocd_browser_plist}" \
      >"${_argocd_browser_launchctl_log}" 2>&1; then
    _info "[acg-down] ArgoCD browser HTTPS listener launchd daemon unloaded"
  elif [[ -s "${_argocd_browser_launchctl_log}" ]]; then
    tail -n 20 "${_argocd_browser_launchctl_log}" >&2 || true
    _warn "[acg-down] existing ArgoCD browser HTTPS listener was not loaded; continuing"
  fi
  _run_command --interactive-sudo --soft -- rm -f "${_argocd_browser_plist}" "${_argocd_browser_wrapper}" >/dev/null 2>&1 || true
  _run_command --interactive-sudo --soft -- rm -f "${_argocd_browser_tls_cert}" "${_argocd_browser_tls_key}" "${_argocd_browser_tls_ca}" "${_argocd_browser_tls_dir}/tls.crt" >/dev/null 2>&1 || true
```
