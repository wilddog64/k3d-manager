# Bug: acg-up Docker preflight fails because OrbStack VM is stopped — open -a is a no-op

**Date:** 2026-06-11
**Branch:** k3d-manager-v1.6.5
**File:** `bin/acg-up`

---

## Symptom

`make up` fails at the Docker preflight step even with the fix from `4adf5a1a`:

```
INFO: [acg-up] Docker daemon not responding — dismissing any OrbStack update dialogs...
INFO: [acg-up] Docker still unavailable — launching OrbStack...
ERROR: [acg-up] Docker daemon unavailable after recovery — ...
make: *** [up] Error 1
```

OrbStack shows "Starting containers" in its UI. Restarting OrbStack from the menu bar
does not resolve it — the VM stays stopped.

---

## Root Cause

Diagnosed via:

```
$ ls -la /var/run/docker.sock
/var/run/docker.sock -> /Users/cliang/.orbstack/run/docker.sock  # symlink exists

$ ls ~/.orbstack/run/docker.sock
No such file or directory   # socket target MISSING — VM not running

$ orb status
Stopped
```

**The OrbStack Linux VM is stopped.** The macOS OrbStack app is running (visible in the
menu bar / showing UI), but its Linux VM is not. The `~/.orbstack/run/docker.sock` socket
is only created by the VM — so `docker info` always fails until the VM starts.

The `open -a OrbStack` fallback in `4adf5a1a` is a no-op: the macOS app is already open.
It does not start the VM. `orbctl start` is the correct command.

Additionally, polling `docker info` is insufficient — the real signal is whether
`~/.orbstack/run/docker.sock` exists (VM socket created) combined with `docker info`
succeeding (daemon ready).

---

## Fix

Replace the fallback block in the Docker preflight section of `bin/acg-up`. The correct
recovery sequence:

1. `docker info` — fast path, common case, skip entire block.
2. `osascript` dismiss — handles the update-dialog case (dialog present, daemon blocked).
3. 30×2s poll — if a dialog was dismissed, 60s is enough to recover.
4. Still failing? Check `orb status` (OrbStack CLI, `which orbctl`).
   - If `Stopped` → `orbctl start` to start the VM.
   - Regardless → 60×2s poll (`docker info` + socket existence together).
5. If still failing → `_err` with clear message.

### Change 1 — replace the fallback block (lines ~225–241 as of `4adf5a1a`)

**Exact old block:**

```bash
  if ! docker info >/dev/null 2>&1; then
    _docker_app=""
    if [[ -d "/Applications/OrbStack.app" ]]; then
      _docker_app="OrbStack"
    elif [[ -d "/Applications/Docker.app" ]]; then
      _docker_app="Docker"
    fi
    if [[ -n "${_docker_app}" ]]; then
      _info "[acg-up] Docker still unavailable — launching ${_docker_app}..."
      open -a "${_docker_app}"
      _docker_wait=0
      until docker info >/dev/null 2>&1 || [[ "${_docker_wait}" -ge 30 ]]; do
        sleep 2
        ((_docker_wait++)) || true
      done
    fi
  fi
```

**Exact new block:**

```bash
  if ! docker info >/dev/null 2>&1; then
    _orbstack_sock="${HOME}/.orbstack/run/docker.sock"
    if command -v orbctl >/dev/null 2>&1; then
      _orb_status=$(orbctl status 2>/dev/null || true)
      if [[ "${_orb_status}" == "Stopped" ]]; then
        _info "[acg-up] OrbStack VM is stopped — starting it now..."
        orbctl start 2>/dev/null || true
      fi
    elif [[ -d "/Applications/OrbStack.app" ]]; then
      _info "[acg-up] orbctl not found — opening OrbStack.app..."
      open -a "OrbStack"
    elif [[ -d "/Applications/Docker.app" ]]; then
      _info "[acg-up] orbctl not found — opening Docker.app..."
      open -a "Docker"
    fi
    _docker_wait=0
    until (docker info >/dev/null 2>&1 && [[ -S "${_orbstack_sock}" ]]) || [[ "${_docker_wait}" -ge 60 ]]; do
      sleep 2
      ((_docker_wait++)) || true
    done
  fi
```
