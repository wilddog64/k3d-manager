# Bug: acg-up Docker preflight times out when OrbStack is starting containers

**Date:** 2026-06-11
**Branch:** k3d-manager-v1.6.5
**File:** `bin/acg-up`

---

## Symptom

`make up` still fails at the Docker preflight step even after the fix in `4adf5a1a`:

```
INFO: [acg-up] Docker daemon not responding — dismissing any OrbStack update dialogs...
INFO: [acg-up] Docker still unavailable — launching OrbStack...
ERROR: [acg-up] Docker daemon unavailable after recovery — ...
make: *** [up] Error 1
```

OrbStack shows "Starting containers" in its UI — the app is running and initializing,
but the Docker daemon is not yet ready.

---

## Root Cause

Two problems in the preflight block added by `4adf5a1a`:

1. **First poll too short** — 30 × 2s = 60s is not enough for OrbStack to finish its
   "Starting containers" initialization phase (Linux VM up → Docker daemon up → containers
   restored). The poll expires before the daemon is ready.

2. **`open -a` fires on an already-running app** — the `open -a` fallback has no guard for
   the case where the app process already exists. `open -a OrbStack` on a running OrbStack
   is a no-op (it just brings the window to focus); the second 60s poll then also expires
   with no improvement.

---

## Fix

Two targeted changes to the preflight block in `bin/acg-up` (lines 202–247 as of `4adf5a1a`).

### Change 1 — increase first poll from 30 to 90 iterations

3 minutes (90 × 2s) covers OrbStack's full startup sequence including "Starting containers".

**Exact old line (line 221):**

```bash
  until docker info >/dev/null 2>&1 || [[ "${_docker_wait}" -ge 30 ]]; do
```

**Exact new line:**

```bash
  until docker info >/dev/null 2>&1 || [[ "${_docker_wait}" -ge 90 ]]; do
```

---

### Change 2 — guard `open -a` with `pgrep` check

Only launch the app if it is not already running. If it IS running but the daemon
isn't ready, just extend the wait instead of calling `open -a`.

**Exact old block (lines 232–240):**

```bash
    if [[ -n "${_docker_app}" ]]; then
      _info "[acg-up] Docker still unavailable — launching ${_docker_app}..."
      open -a "${_docker_app}"
      _docker_wait=0
      until docker info >/dev/null 2>&1 || [[ "${_docker_wait}" -ge 30 ]]; do
        sleep 2
        ((_docker_wait++)) || true
      done
    fi
```

**Exact new block:**

```bash
    if [[ -n "${_docker_app}" ]]; then
      if pgrep -x "${_docker_app}" >/dev/null 2>&1; then
        _info "[acg-up] ${_docker_app} is starting — waiting for Docker daemon..."
      else
        _info "[acg-up] Docker still unavailable — launching ${_docker_app}..."
        open -a "${_docker_app}"
      fi
      _docker_wait=0
      until docker info >/dev/null 2>&1 || [[ "${_docker_wait}" -ge 60 ]]; do
        sleep 2
        ((_docker_wait++)) || true
      done
    fi
```
