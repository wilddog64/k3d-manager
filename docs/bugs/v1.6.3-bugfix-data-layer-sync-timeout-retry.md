# Bugfix: v1.6.3 — data-layer ArgoCD sync timeout auto-retry

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/acg-up`

---

## Problem

Step 10b of `bin/acg-up` waits up to 300s for the `data-layer` ArgoCD Application to reach
`Synced`. When ArgoCD's initial sync is slow (network latency, resource pressure, scheduler
backlog), it hits the deadline and calls `_err` / `exit 1`, producing:

```
APP_FAILURE / PERMANENT
The ArgoCD data-layer application failed to sync within the timeout...
```

This is a transient timing issue — a force-sync clears it every time, but the user must
manually send `/acg-resume` to recover.

**Root cause:** The timeout handler calls `_err` + `exit 1` immediately with no retry. A
force-sync attempt before giving up would recover the transient case automatically.

---

## Fix

### Change 1 — `bin/acg-up`: force-sync + retry loop on first timeout

Replace the timeout hard-fail with a one-shot `argocd app sync --force` followed by a
180s retry wait. Only `_err` after the retry is also exhausted.

**Exact old block (lines 620–631):**

```bash
  _info "[acg-up] Waiting for ArgoCD to sync data-layer Application (max 300s)..."
  _dl_sync_deadline=$(( $(date +%s) + 300 ))
  until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
      -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
    if [[ $(date +%s) -ge ${_dl_sync_deadline} ]]; then
      _err "[acg-up] data-layer ArgoCD Application did not reach Synced within 300s — check: kubectl get application data-layer -n cicd --context k3d-k3d-cluster"
      exit 1
    fi
    _info "[acg-up] data-layer not yet Synced — waiting..."
    sleep 10
  done
  _info "[acg-up] data-layer Application Synced — proceeding to StatefulSet readiness check"
```

**Exact new block:**

```bash
  _info "[acg-up] Waiting for ArgoCD to sync data-layer Application (max 300s)..."
  _dl_sync_deadline=$(( $(date +%s) + 300 ))
  until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
      -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
    if [[ $(date +%s) -ge ${_dl_sync_deadline} ]]; then
      _warn "[acg-up] data-layer did not reach Synced within 300s — force-syncing and retrying (one attempt)..."
      argocd app sync data-layer \
        --server localhost:8080 --insecure \
        --force \
        --timeout 180 \
        2>/dev/null \
        || true
      _dl_retry_deadline=$(( $(date +%s) + 180 ))
      until kubectl get application data-layer -n cicd --context k3d-k3d-cluster \
          -o jsonpath='{.status.sync.status}' 2>/dev/null | grep -q "Synced"; do
        if [[ $(date +%s) -ge ${_dl_retry_deadline} ]]; then
          _err "[acg-up] data-layer ArgoCD Application did not reach Synced after force-sync + 180s retry — check: kubectl get application data-layer -n cicd --context k3d-k3d-cluster"
          exit 1
        fi
        _info "[acg-up] data-layer not yet Synced after force-sync — waiting..."
        sleep 10
      done
      break
    fi
    _info "[acg-up] data-layer not yet Synced — waiting..."
    sleep 10
  done
  _info "[acg-up] data-layer Application Synced — proceeding to StatefulSet readiness check"
```
