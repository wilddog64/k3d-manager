# Spec: v0.3.0 — Make CloudFormation template path configurable

**Branch:** `fix/cfn-template-path-configurable`
**Files:** `scripts/plugins/acg.sh`, `scripts/vars.sh` (or equivalent vars block)

---

## Problem

`acg.sh` hard-codes the CloudFormation template path relative to `_LIB_ACG_ROOT`:

```bash
--template-file "${_LIB_ACG_ROOT}/scripts/etc/acg-cluster.yaml"
```

`acg-cluster.yaml` is a deployment artifact that belongs in k3d-manager (where the
cluster is actually provisioned), not in lib-acg (a browser-automation utility library).
Having it in lib-acg:
- Couples infrastructure topology to the browser-automation library
- Forces lib-acg to carry a file that is only meaningful when embedded in k3d-manager
- Caused the v0.2.0 bug: the file was deleted here because it "doesn't belong," but
  `acg.sh` still referenced it — no gate caught the dangling reference until `make up` failed

---

## Fix

### Change 1 — Add `ACG_CLUSTER_TEMPLATE` env var with fallback default

In `scripts/plugins/acg.sh`, replace the hard-coded path with a configurable variable:

```bash
# Before (line 135)
--template-file "${_LIB_ACG_ROOT}/scripts/etc/acg-cluster.yaml" \

# After
--template-file "${ACG_CLUSTER_TEMPLATE:-${_LIB_ACG_ROOT}/scripts/etc/acg-cluster.yaml}" \
```

**Why the fallback:** lib-acg ships `scripts/etc/acg-cluster.yaml` as a reference template.
Callers who want to use their own template (e.g. k3d-manager) set `ACG_CLUSTER_TEMPLATE`
to their copy. Callers who don't set it get the bundled template — no behavior change for
existing users.

### Change 2 — Document `ACG_CLUSTER_TEMPLATE` in vars.sh

Add to the vars block (wherever other `ACG_*` variables are documented):

```bash
# Path to the CloudFormation template for cluster provisioning.
# Default: ${_LIB_ACG_ROOT}/scripts/etc/acg-cluster.yaml (bundled reference template).
# Override to use a caller-managed template (e.g. from k3d-manager).
# ACG_CLUSTER_TEMPLATE=
```

The variable is intentionally commented out — callers set it in their environment
(shell profile, Makefile, or `.envrc`), not in this file.

## What NOT to Do

- Do NOT remove `scripts/etc/acg-cluster.yaml` from lib-acg — it stays as the bundled
  reference template and the default fallback
- Do NOT change the template content — only the path reference in `acg.sh`
- Do NOT add `ACG_CLUSTER_TEMPLATE` to `.envrc` or any committed env file — it is a
  caller-supplied override, not a lib-acg default
