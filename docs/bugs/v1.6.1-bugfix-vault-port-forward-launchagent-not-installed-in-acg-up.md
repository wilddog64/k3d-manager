# Bugfix: v1.6.1 — Vault port-forward LaunchAgent not installed by acg-up

**Branch:** `k3d-manager-v1.6.1`
**Files:** `bin/acg-up`, `bin/acg-down`
**Discovered:** 2026-06-05

---

## Problem

After every cluster rebuild, the Vault port-forward LaunchAgent
(`com.k3d-manager.vault-port-forward`) is absent from
`~/Library/LaunchAgents/`. It must be installed manually with
`make install-vault-port-forward`, but this step is not part of
`bin/acg-up`. If it is missed, the entire data layer fails silently:

```
Vault port-forward DOWN (18200 not listening)
→ vault-bridge.secrets.svc:8201 unreachable from ubuntu-k3s
→ ClusterSecretStore vault-backend: InvalidProviderConfig
→ All ExternalSecrets: "could not get secret data from provider"
→ data-layer StatefulSets not created
→ basket / order / payment / product-catalog: CrashLoopBackOff
```

**Root cause:** `bin/acg-up` Step 4 starts the Vault port-forward as a
bare background process (`kubectl port-forward … &` with PID tracking).
This process survives only the current session. The LaunchAgent template
exists at `scripts/etc/launchd/com.k3d-manager.vault-port-forward.plist.tmpl`
and the `make install-vault-port-forward` target is correct, but neither
is invoked from `bin/acg-up`.

## Fix

Two changes: one in `bin/acg-up` (install), one in `bin/acg-down` (teardown).

---

### Change 1 — `bin/acg-up` ~line 248: install LaunchAgent after Vault confirmed reachable

**Anchor — insert AFTER this line:**
```bash
_info "[acg-up] Vault is unsealed and reachable."
```

**Block to insert:**
```bash

if _is_mac; then
  _info "[acg-up] Step 4.1/12 — Installing Vault port-forward LaunchAgent (localhost:18200, auto-restart)..."
  _vault_pf_label="com.k3d-manager.vault-port-forward"
  _vault_pf_plist="${HOME}/Library/LaunchAgents/${_vault_pf_label}.plist"
  _vault_pf_tmpl="${REPO_ROOT}/scripts/etc/launchd/${_vault_pf_label}.plist.tmpl"
  _vault_pf_kubectl="$(command -v kubectl)"
  mkdir -p "$(dirname "${_vault_pf_plist}")"
  sed \
    -e "s|{{KUBECTL_PATH}}|${_vault_pf_kubectl}|g" \
    -e "s|{{HOME}}|${HOME}|g" \
    "${_vault_pf_tmpl}" > "${_vault_pf_plist}"
  launchctl bootout "gui/$(id -u)/${_vault_pf_label}" 2>/dev/null || true
  launchctl bootstrap "gui/$(id -u)" "${_vault_pf_plist}"
  _info "[acg-up] Vault port-forward LaunchAgent installed (launchd: ${_vault_pf_label})"
fi
```

---

### Change 2 — `bin/acg-down` ~line 82: unload LaunchAgent on teardown

**Exact old block:**
```bash
_info "[acg-down] Stopping Vault port-forward..."
_vault_pf_pid_file="${_ACG_STATE_DIR}/run/vault-pf.pid"
if [[ -f "${_vault_pf_pid_file}" ]]; then
  kill "$(cat "${_vault_pf_pid_file}")" 2>/dev/null || true
  rm -f "${_vault_pf_pid_file}"
  _info "[acg-down] Vault port-forward stopped"
fi
```

**Exact new block:**
```bash
_info "[acg-down] Stopping Vault port-forward..."
_vault_pf_pid_file="${_ACG_STATE_DIR}/run/vault-pf.pid"
if [[ -f "${_vault_pf_pid_file}" ]]; then
  kill "$(cat "${_vault_pf_pid_file}")" 2>/dev/null || true
  rm -f "${_vault_pf_pid_file}"
  _info "[acg-down] Vault port-forward stopped"
fi
if _is_mac; then
  _vault_pf_label="com.k3d-manager.vault-port-forward"
  _vault_pf_plist="${HOME}/Library/LaunchAgents/${_vault_pf_label}.plist"
  if launchctl list "${_vault_pf_label}" >/dev/null 2>&1; then
    launchctl unload "${_vault_pf_plist}" 2>/dev/null || true
    _info "[acg-down] Vault port-forward LaunchAgent unloaded"
  fi
  rm -f "${_vault_pf_plist}"
fi
```
