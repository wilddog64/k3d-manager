# Bug: bin/acg-up fails mid-run when OrbStack update dialog blocks Docker daemon

**Date:** 2026-06-11
**Branch:** k3d-manager-v1.6.5
**File:** `bin/acg-up`

---

## Symptom

`make up` fails at step 3.5 with:

```
ERRO[0000] Failed to create cluster >>> Rolling Back
Cannot connect to the Docker daemon at unix:///var/run/docker.sock. Is the docker daemon running?
WARN: [acg-up] failed (exit 1) — cleaning up local processes...
make: *** [up] Error 1
```

The script runs credential acquisition (steps 1–3), starts the SSH tunnel, and only then
discovers Docker is unavailable — 3+ minutes in.

---

## Root Cause

OrbStack is a macOS startup item and is always running. However, two OrbStack update dialog
states can make the Docker daemon temporarily unavailable or unresponsive:

1. **"A new version of OrbStack is available!"** — Update notification dialog with buttons
   "Skip This Version", "Remind Me Later", "Install Update". The daemon is running; this dialog
   is an interruption that does not require a start.

2. **"Update Error!"** — "An error occurred while running the updater. Please try again later."
   with a single "Cancel Update" button. OrbStack attempted an in-place update, the daemon
   went offline mid-install, and the update failed. The daemon does not restart until the
   error is dismissed.

`bin/acg-up` has no preflight for Docker daemon health and no mechanism to dismiss these
dialogs automatically, so any update state causes a confusing k3d error 3+ minutes in.

---

## Fix

Add a Docker daemon preflight block **immediately before step 3.5** (Hub cluster creation,
~line 202), after all lib sources have been loaded so `_info`, `_warn`, and `_err` are
available.

### Logic

1. `docker info` — if OK, skip the entire block (OrbStack is healthy, common case).
2. If not OK: run `osascript` to dismiss any visible OrbStack update dialog:
   - Click "Remind Me Later" if the version-available dialog is open.
   - Click "Cancel Update" if the update-error dialog is open.
   - Wrapped `|| true` — best-effort, never a hard failure.
3. Wait up to 30 × 2s for `docker info` to succeed.
4. If still failing and `OrbStack.app` or `Docker.app` exists, `open -a <app>` as a last
   resort (handles the rare case where OrbStack is not running).
5. Wait another 30 × 2s.
6. If still failing: `_err` with a clear message.

Inserting after the sources means `_info`, `_warn`, `_err` are fully available — no need
for bare `printf` calls. `osascript` is macOS-native scripting — same level as `open -a`,
not a deep API binding. No Accessibility permission is required to *attempt* it; it silently
no-ops if denied.

### Change 1 — `bin/acg-up`: add Docker daemon preflight before step 3.5

**Exact old block (lines 202–204):**

```bash
_HUB_CLUSTER_CTX="k3d-${HUB_CLUSTER_NAME:-k3d-cluster}"
_hub_newly_created=0
_info "[acg-up] Step 3.5/12 — Verifying local Hub cluster (create if missing)..."
```

**Exact new block:**

```bash
if ! docker info >/dev/null 2>&1; then
  _info "[acg-up] Docker daemon not responding — dismissing any OrbStack update dialogs..."
  osascript 2>/dev/null <<'APPLESCRIPT' || true
tell application "System Events"
  if exists process "OrbStack" then
    tell process "OrbStack"
      repeat with w in windows
        if exists button "Remind Me Later" of w then
          click button "Remind Me Later" of w
        end if
        if exists button "Cancel Update" of w then
          click button "Cancel Update" of w
        end if
      end repeat
    end tell
  end if
end tell
APPLESCRIPT
  _docker_wait=0
  until docker info >/dev/null 2>&1 || [[ $_docker_wait -ge 30 ]]; do
    sleep 2
    ((_docker_wait++)) || true
  done
  if ! docker info >/dev/null 2>&1; then
    _docker_app=""
    if [[ -d "/Applications/OrbStack.app" ]]; then
      _docker_app="OrbStack"
    elif [[ -d "/Applications/Docker.app" ]]; then
      _docker_app="Docker"
    fi
    if [[ -n "$_docker_app" ]]; then
      _info "[acg-up] Docker still unavailable — launching ${_docker_app}..."
      open -a "$_docker_app"
      _docker_wait=0
      until docker info >/dev/null 2>&1 || [[ $_docker_wait -ge 30 ]]; do
        sleep 2
        ((_docker_wait++)) || true
      done
    fi
  fi
  if ! docker info >/dev/null 2>&1; then
    _err "[acg-up] Docker daemon unavailable after recovery — check OrbStack/Docker Desktop status and dismiss any update dialogs, then re-run"
  fi
  _info "[acg-up] Docker daemon is up."
fi
_HUB_CLUSTER_CTX="k3d-${HUB_CLUSTER_NAME:-k3d-cluster}"
_hub_newly_created=0
_info "[acg-up] Step 3.5/12 — Verifying local Hub cluster (create if missing)..."
```
