# Bugfix: v1.4.11 — order-service Lombok annotation processor missing from maven-compiler-plugin

**Branch (spec repo):** `k3d-manager-v1.4.11`
**Branch (work repo):** `fix/order-actuator-security-npe` (same branch as actuator NPE fix)
**File:** `pom.xml`

---

## Problem

`mvn compile` fails with 9 `cannot find symbol: variable log` errors across
`RateLimitFilter.java`, `OrderService.java`, and `OrderEventPublisher.java`.
All three classes use Lombok's `@Slf4j` annotation to generate the `log` field.

**Root cause:** Lombok is declared as an `<optional>true</optional>` dependency but is
never configured as an annotation processor in `maven-compiler-plugin`. Without
`annotationProcessorPaths`, the Java compiler does not invoke Lombok's APT processor,
so `@Slf4j` generates nothing and `log` is undefined.

---

## Fix

### Change 1 — `pom.xml`: add `maven-compiler-plugin` with Lombok annotation processor

**Insert before the closing `</plugins>` tag (line 211):**

```xml
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <configuration>
                    <annotationProcessorPaths>
                        <path>
                            <groupId>org.projectlombok</groupId>
                            <artifactId>lombok</artifactId>
                        </path>
                    </annotationProcessorPaths>
                </configuration>
            </plugin>
```

No version pin needed — Spring Boot BOM manages the compiler plugin version.
