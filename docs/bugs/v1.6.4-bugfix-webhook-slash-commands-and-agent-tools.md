# Bugfix: v1.6.4 — Webhook slash command aliases + agent diagnostic tool access

**Branch:** `k3d-manager-v1.6.4`
**Files:** `bin/k3dm-webhook`, `bin/k3dm-ask-bash`

---

## Problem

Two related usability gaps in the webhook ChatOps interface:

1. **No slash command aliases** — Users must type `ask gemini <q>` or `ask claude <q>`.
   Direct `/gemini <q>`, `/claude <q>`, `/codex <q>` commands are unrecognised, returning
   "Unknown command". Expected by teams accustomed to ChatOps `/bot <question>` patterns.

2. **Agents refuse or lack access to diagnostic tools** — The Claude/Gemini system prompt
   says "Read-only commands only" without naming what IS allowed. Agents over-refuse sed,
   grep, awk, curl (GET), and network tools (dig, ping, ss). Additionally, `k3dm-ask-bash`
   scope-blocks absolute paths outside the two repos, preventing agents from reading
   `/etc/hosts`, `~/.kube/config`, LaunchDaemon plists, and ACG state logs — all needed
   for cluster diagnosis.

---

## Reproduction

1. Post `@k3dm /gemini why is the order-service crashing?` in Slack thread → "Unknown command"
2. Ask `/ask claude what does /etc/hosts look like` → Claude refuses or sandbox blocks path
3. Ask `/ask claude show me grep output of the pod log` → Claude declines citing "read-only"

### Change 4 — `bin/k3dm-ask-bash`: expand allowed read paths for diagnosis

**Exact old block (lines near the scope check — the `for arg in "$@"` loop):**

```bash
# Block file access outside allowed repos (absolute path arguments only)
for arg in "$@"; do
  if [[ "$arg" == /* ]]; then
    real_arg="$(realpath -m "$arg" 2>/dev/null || echo "$arg")"
    if [[ "$real_arg" != "${K3DM_REPO}"* && "$real_arg" != "${SHOPPING_CARTS_ROOT}"* \
       && "$real_arg" != /tmp/* && "$real_arg" != /var/tmp/* \
       && "$real_arg" != /proc/* && "$real_arg" != /sys/* ]]; then
      _deny_scope "$arg"
    fi
  fi
done
```

**Exact new block:**

```bash
# Allowed read-only system paths for cluster diagnosis
_DIAG_PATHS=(
  /etc
  /Library/LaunchDaemons
  /Library/LaunchAgents
  "${HOME}/.kube"
  "${HOME}/.local/share/k3d-manager"
  "${HOME}/.cloudflared"
  /var/log
  /usr/local/bin
  /usr/bin
  /bin
  /opt/homebrew
)

# Block file access outside allowed repos and diagnosis paths (absolute path arguments only)
for arg in "$@"; do
  if [[ "$arg" == /* ]]; then
    real_arg="$(realpath -m "$arg" 2>/dev/null || echo "$arg")"
    _allowed=0
    for _p in "${K3DM_REPO}" "${SHOPPING_CARTS_ROOT}" /tmp /var/tmp /proc /sys "${_DIAG_PATHS[@]}"; do
      if [[ "$real_arg" == "${_p}"* ]]; then
        _allowed=1
        break
      fi
    done
    if [[ "$_allowed" -eq 0 ]]; then
      _deny_scope "$arg"
    fi
  fi
done
```
