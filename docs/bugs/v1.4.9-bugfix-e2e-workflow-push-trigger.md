# Bugfix: v1.4.9 — e2e-tests CI fails on every push to main

**Branch:** `k3d-manager-v1.4.9`
**Work repo:** `shopping-cart-e2e-tests`
**Work branch:** `fix/e2e-workflow-push-trigger` (create from main)
**Files:** `.github/workflows/e2e-tests.yml`

---

## Problem

The `api-tests` and `flow-tests` jobs in `.github/workflows/e2e-tests.yml` run on every
`push` and `pull_request` to main. Both jobs connect to live services at
`localhost:8083` (basket), `localhost:8000` (product catalog), `localhost:8080` (order),
which do not exist in GitHub Actions CI. Every push to main fails immediately with
`ECONNREFUSED`.

The `all-tests` job is correctly gated with
`if: github.event_name == 'schedule' || github.event_name == 'workflow_dispatch'` — but
`api-tests` and `flow-tests` are not.

**Root cause:** The `push` and `pull_request` triggers at lines 17-19 fire all jobs that
lack a condition gate. The `api-tests` and `flow-tests` jobs have no `if:` condition.

---

## Reproduction

Push any commit to `wilddog64/shopping-cart-e2e-tests` main → CI fails in ~5 minutes 39 seconds
with `ECONNREFUSED ::1:8083`.

---

## Fix

### Change 1 — `.github/workflows/e2e-tests.yml`: remove push/PR triggers

**Exact old block (lines 3-22):**

```yaml
on:
  workflow_dispatch:
    inputs:
      environment:
        description: 'Environment to test against'
        required: true
        default: 'staging'
        type: choice
        options:
          - staging
          - production
  # schedule disabled — re-enable once ImagePullBackOff resolved and app cluster healthy
  # schedule:
  #   - cron: '0 6 * * *'
  push:
    branches:
      - main
  pull_request:
    branches:
      - main
```

**Exact new block:**

```yaml
on:
  workflow_dispatch:
    inputs:
      environment:
        description: 'Environment to test against'
        required: true
        default: 'staging'
        type: choice
        options:
          - staging
          - production
  # schedule disabled — re-enable once ImagePullBackOff resolved and app cluster healthy
  # schedule:
  #   - cron: '0 6 * * *'
  # push/pull_request disabled — tests require live services (basket:8083, catalog:8000, order:8080)
  # Re-enable only after adding service containers or a staging environment secret
```
