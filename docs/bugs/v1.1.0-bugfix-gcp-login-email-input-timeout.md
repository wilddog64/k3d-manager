## Problem

After clicking "Use another account" (line 67), `gcp_login.js` immediately waits
for `input[type="email"]` with a 10-second timeout. Google's account chooser triggers
a full page navigation to the sign-in page — the script does not await that navigation
before checking for the email input, so the locator fires against the old DOM and times
out.

**Observed error:**

```
INFO: Clicking "Use another account"
ERROR: locator.waitFor: Timeout 10000ms exceeded.
Call log:
  - waiting for locator('input[type="email"]').first() to be visible
ERROR: [gcp] gcp_login: Playwright auth failed
```

**Root cause:** No `waitForLoadState` (or equivalent) between the `useAnother.click()`
and `emailInput.waitFor()`. The email input check runs against the still-loading or
pre-navigation DOM.

---

## Reproduction

```bash
make up CLUSTER_PROVIDER=k3s-gcp
# GCP credentials extracted OK
# → INFO: Clicking "Use another account"
# → ERROR: locator.waitFor: Timeout 10000ms exceeded.
```

---

## Fix

### Change 1 — `scripts/playwright/gcp_login.js`: await navigation + increase email timeout

**Exact old block (lines 65–71):**

```js
      if (useAnotherVisible) {
        console.error('INFO: Clicking "Use another account"');
        await useAnother.click();
      }
      // Fill email
      const emailInput = page.locator('input[type="email"]').first();
      await emailInput.waitFor({ timeout: 10000 });
```

**Exact new block:**

```js
      if (useAnotherVisible) {
        console.error('INFO: Clicking "Use another account"');
        await useAnother.click();
        await page.waitForLoadState('domcontentloaded');
      }
      // Fill email
      const emailInput = page.locator('input[type="email"]').first();
      await emailInput.waitFor({ timeout: 15000 });
```

Two changes:
1. Add `await page.waitForLoadState('domcontentloaded')` immediately after the click so the new sign-in page settles before the email locator runs.
2. Increase email input timeout from 10000 → 15000ms to absorb slow Google sign-in page loads.
