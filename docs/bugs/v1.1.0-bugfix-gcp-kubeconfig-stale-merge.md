## Problem

When the GCP VM's ephemeral IP changes (instance stopped/restarted) and `make up` is
re-run, `k3sup install` writes fresh credentials to `~/.kube/k3s-gcp.yaml` for the
new IP — but the kubeconfig merge puts `~/.kube/config` first:

```bash
KUBECONFIG="${HOME}/.kube/config:${_GCP_KUBECONFIG}" \
  kubectl config view --flatten > "${HOME}/.kube/config.tmp"
```

`kubectl config view --flatten` gives precedence to the first file. If `~/.kube/config`
already contains a `k3s-gcp` cluster/user from the previous installation (old IP, old
client cert), those stale entries win and the fresh entries in `k3s-gcp.yaml` are
silently discarded.

**Result:** `kubectl --context k3s-gcp get nodes` fails with:
```
x509: certificate signed by unknown authority
```
or the wrong server IP, even though the cluster is running fine.

**Root cause:** `_provider_k3s_gcp_deploy_cluster` (lines 211–215) does not purge stale
`k3s-gcp` context/cluster/user entries from `~/.kube/config` before the merge.

---

## Reproduction

```bash
# First run — creates instance at IP A
make up CLUSTER_PROVIDER=k3s-gcp URL=<sandbox-url>

# Stop the instance (IP changes to IP B on restart)
gcloud compute instances stop k3s-gcp-server --zone=us-central1-a --project=<project>
gcloud compute instances start k3s-gcp-server --zone=us-central1-a --project=<project>

# Second run — finds existing instance, re-runs k3sup with new IP
make up CLUSTER_PROVIDER=k3s-gcp URL=<sandbox-url>

# kubectl still uses old IP / old client cert
kubectl --context k3s-gcp get nodes
# → x509: certificate signed by unknown authority
```

---

## Fix

### Change 1 — `scripts/lib/providers/k3s-gcp.sh`: purge stale entries before merge

**Exact old block (lines 211–215):**

```bash
  mkdir -p "${HOME}/.kube"
  KUBECONFIG="${HOME}/.kube/config:${_GCP_KUBECONFIG}" \
    kubectl config view --flatten > "${HOME}/.kube/config.tmp" && \
    mv "${HOME}/.kube/config.tmp" "${HOME}/.kube/config"
  chmod 600 "${HOME}/.kube/config" "${_GCP_KUBECONFIG}"
```

**Exact new block:**

```bash
  mkdir -p "${HOME}/.kube"
  kubectl config delete-context k3s-gcp 2>/dev/null || true
  kubectl config delete-cluster k3s-gcp 2>/dev/null || true
  kubectl config delete-user k3s-gcp 2>/dev/null || true
  KUBECONFIG="${HOME}/.kube/config:${_GCP_KUBECONFIG}" \
    kubectl config view --flatten > "${HOME}/.kube/config.tmp" && \
    mv "${HOME}/.kube/config.tmp" "${HOME}/.kube/config"
  chmod 600 "${HOME}/.kube/config" "${_GCP_KUBECONFIG}"
```
