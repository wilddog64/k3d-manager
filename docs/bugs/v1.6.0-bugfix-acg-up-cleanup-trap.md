# Bugfix: v1.6.0 — acg-up leaves orphaned processes on failure

**Branch:** `k3d-manager-v1.6.0`
**Files:** `bin/acg-up`

---

## Problem

When `bin/acg-up` fails mid-run (from Slack/webhook or interactively), it exits without
cleaning up any locally-started processes. Port-forwards (Vault, frontend, ArgoCD, Prometheus),
the cloudflared LaunchAgent, and the ArgoCD port-forward LaunchAgent all remain running,
holding ports and consuming memory until the next explicit `acg-down`.

**Root cause:** `bin/acg-up` has no `trap` handler; `set -euo pipefail` exits on error
but nothing unloads agents or kills PID-file processes.

---

## Fix

### Change 1 — `bin/acg-up`: add `_acg_up_cleanup` function and `trap EXIT`

Insert after the `_acg_fail` function (after line 87) and before `login_prompt=0`:

**Exact new block:**

```bash
function _acg_up_cleanup() {
  local _exit_code=$?
  [[ "${_exit_code}" -eq 0 ]] && return
  _warn "[acg-up] failed (exit ${_exit_code}) — cleaning up local processes..."
  for _pid_file in vault-pf frontend-pf acg-prom-pf; do
    local _f="${_ACG_STATE_DIR}/run/${_pid_file}.pid"
    if [[ -f "${_f}" ]]; then
      kill "$(cat "${_f}")" 2>/dev/null || true
      rm -f "${_f}"
    fi
  done
  launchctl unload "${HOME}/Library/LaunchAgents/com.k3d-manager.argocd-port-forward.plist" 2>/dev/null || true
  launchctl bootout "gui/$(id -u)" "${HOME}/Library/LaunchAgents/com.k3d-manager.cloudflare-tunnel.plist" 2>/dev/null || true
}
trap _acg_up_cleanup EXIT
```

Insert between the closing `}` of `_acg_fail` (line 87) and `login_prompt=0` (line 89).

## Scope

- Does NOT tear down LaunchDaemons (argocd-browser-https, keycloak-browser-http, frontend-browser-http) — those require sudo
- Does NOT touch CloudFormation or the remote cluster — user must run `bin/acg-down --confirm` explicitly
