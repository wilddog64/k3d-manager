# Bug: acg-up Azure block calls az group list before _az_ok — triggers device code auth

**Date:** 2026-06-11
**Branch:** k3d-manager-v1.6.5
**File:** `bin/acg-up`

---

## Symptom

`make up CLUSTER_PROVIDER=k3s-azure` opens a "Pick an account" Microsoft device code auth
browser window on first run (when no Azure credentials exist).

```
You're signing in to Microsoft Azure Cross-platform Command Line Interface
on another device located in United States.
```

---

## Root Cause

Commit `1f7e3ef3` moved `az group list` to run BEFORE `_az_ok` (lines 185–201):

```bash
  k3s-azure)
    _az_existing_rg=$(az group list --query '[0].name' -o tsv 2>/dev/null || true)
    if _az_ok 2>/dev/null && [[ -n "${_az_existing_rg}" ]]; then
```

When there are no Azure credentials, `az group list` invokes interactive device code auth
— it ignores `2>/dev/null` for the browser prompt and opens the Microsoft sign-in page.
In v1.5.0, `_az_ok` was the outermost guard; `az group list` only ran if credentials were
already valid.

---

## Fix

### Change 1 — `bin/acg-up`: gate `az group list` behind `_az_ok`

Initialize `_az_existing_rg` to empty, run `az group list` only when `_az_ok` passes,
then use `[[ -n "${_az_existing_rg}" ]]` as the sole fast-path condition.

**Exact old block (lines 185–201):**

```bash
  k3s-azure)
    _az_existing_rg=$(az group list --query '[0].name' -o tsv 2>/dev/null || true)
    if _az_ok 2>/dev/null && [[ -n "${_az_existing_rg}" ]]; then
      _info "[acg-up] Azure credentials already valid — skipping Playwright extraction"
      AZ_RESOURCE_GROUP="${_az_existing_rg}"
      export AZ_RESOURCE_GROUP
    else
      _acg_cred_test_url="${sandbox_url:-${_ACG_SANDBOX_LIST_URL}}"
      _info "[acg-up] Extracting and validating Azure credentials..."
      _browser_launch
      "${REPO_ROOT}/scripts/lib/acg/bin/acg-credential-test" "${_acg_cred_test_url}" --provider azure || exit 1
      _info "[acg-up] Azure credentials extracted and validated."
      AZ_RESOURCE_GROUP=$(az group list --query '[0].name' -o tsv 2>/dev/null || true)
      export AZ_RESOURCE_GROUP
    fi
    [[ -n "${AZ_RESOURCE_GROUP:-}" ]] || { _err "[acg-up] Could not determine AZ_RESOURCE_GROUP after credential extraction — aborting"; exit 1; }
    ;;
```

**Exact new block:**

```bash
  k3s-azure)
    _az_existing_rg=""
    if _az_ok 2>/dev/null; then
      _az_existing_rg=$(az group list --query '[0].name' -o tsv 2>/dev/null || true)
    fi
    if [[ -n "${_az_existing_rg}" ]]; then
      _info "[acg-up] Azure credentials already valid — skipping Playwright extraction"
      AZ_RESOURCE_GROUP="${_az_existing_rg}"
      export AZ_RESOURCE_GROUP
    else
      _acg_cred_test_url="${sandbox_url:-${_ACG_SANDBOX_LIST_URL}}"
      _info "[acg-up] Extracting and validating Azure credentials..."
      _browser_launch
      "${REPO_ROOT}/scripts/lib/acg/bin/acg-credential-test" "${_acg_cred_test_url}" --provider azure || exit 1
      _info "[acg-up] Azure credentials extracted and validated."
      AZ_RESOURCE_GROUP=$(az group list --query '[0].name' -o tsv 2>/dev/null || true)
      export AZ_RESOURCE_GROUP
    fi
    [[ -n "${AZ_RESOURCE_GROUP:-}" ]] || { _err "[acg-up] Could not determine AZ_RESOURCE_GROUP after credential extraction — aborting"; exit 1; }
    ;;
```
