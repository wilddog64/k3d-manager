# Bugfix: v1.6.3 — argocd-redis secret fails on ubuntu-k3s due to missing cicd namespace

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/acg-up`

---

## Problem

Step 10 of `bin/acg-up` creates the `cicd` namespace on `k3d-k3d-cluster` (line 584),
then immediately tries to create `argocd-redis` in `cicd` on `ubuntu-k3s` (lines 597–611).
On a fresh run, `cicd` does not exist on `ubuntu-k3s` at that point — ArgoCD has just been
registered with the app cluster but has not synced any applications yet, so the namespace
has never been created there.

A 120s wait loop polls for `cicd` on `ubuntu-k3s`, but the namespace never appears during
that window, so the loop exits on deadline and the `kubectl create secret` call fails:

```
Error from server (NotFound): namespaces "cicd" not found
```

**Root cause:** The script assumes ArgoCD will create `cicd` on `ubuntu-k3s` before the
secret step runs. ArgoCD cannot do this — no ApplicationSet has been deployed to
`ubuntu-k3s` yet at this point in the sequence. The namespace must be created explicitly.

---

## Reproduction

Fresh ACG provision (`/acg-up`) on a new sandbox. Step 10 fails with:
```
Error from server (NotFound): namespaces "cicd" not found
```
after the 120s deadline expires.

---

## Fix

### Change 1 — `bin/acg-up`: create `cicd` on `ubuntu-k3s` explicitly; remove wait loop

Replace the fragile 120s poll with a direct idempotent namespace create, matching the
same pattern used for `k3d-k3d-cluster` at line 584.

**Exact old block (lines 597–611):**

```bash
_info "[acg-up] Ensuring argocd-redis secret on ubuntu-k3s (Helm chart hook skipped when ArgoCD manages ArgoCD)..."
_redis_deadline=$(( $(date +%s) + 120 ))
while (( $(date +%s) < _redis_deadline )); do
  if kubectl get namespace cicd --context ubuntu-k3s >/dev/null 2>&1; then
    break
  fi
  sleep 5
done
if ! kubectl get secret argocd-redis -n cicd --context ubuntu-k3s >/dev/null 2>&1; then
  kubectl create secret generic argocd-redis -n cicd --context ubuntu-k3s \
    --from-literal=auth="$(openssl rand -base64 32)"
  _info "[acg-up] argocd-redis secret created on ubuntu-k3s"
else
  _info "[acg-up] argocd-redis secret already exists on ubuntu-k3s"
fi
```

**Exact new block:**

```bash
_info "[acg-up] Ensuring argocd-redis secret on ubuntu-k3s (Helm chart hook skipped when ArgoCD manages ArgoCD)..."
kubectl create namespace cicd --context ubuntu-k3s \
  --dry-run=client -o yaml | kubectl apply --context ubuntu-k3s -f - >/dev/null
if ! kubectl get secret argocd-redis -n cicd --context ubuntu-k3s >/dev/null 2>&1; then
  kubectl create secret generic argocd-redis -n cicd --context ubuntu-k3s \
    --from-literal=auth="$(openssl rand -base64 32)"
  _info "[acg-up] argocd-redis secret created on ubuntu-k3s"
else
  _info "[acg-up] argocd-redis secret already exists on ubuntu-k3s"
fi
```
