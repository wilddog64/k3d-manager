# Bugfix: v1.4.11 — Keycloak browser flow REQUIRED/ALTERNATIVE mix blocks SSO login

**Branch:** `k3d-manager-v1.4.11`
**Files:** `identity/keycloak/keycloak-reconcile-hook-job.yaml` (shopping-cart-infra)

---

## Problem

ArgoCD SSO login fails with "Invalid username or password" even when LDAP credentials are
correct. Keycloak logs show:

```
WARN REQUIRED and ALTERNATIVE elements at same level!
Those alternative executions will be ignored: [auth-cookie, identity-provider-redirector, null]
```

**Root cause:** The keycloak-reconcile-hook-job added `otp-conditional-subflow` as a
top-level (level 0) CONDITIONAL step inside `browser-with-conditional-otp`. This caused
Keycloak to apply the rule: when REQUIRED/CONDITIONAL steps coexist with ALTERNATIVE steps
at the same level, all ALTERNATIVE steps are ignored. The `browser-with-conditional-otp forms`
subflow (which contains Username Password Form) was ALTERNATIVE — so it was silently skipped,
preventing any user from logging in.

---

## Reproduction

1. Provision cluster with `make up`
2. Keycloak reconcile job runs and adds `otp-conditional-subflow` to the browser flow
3. Navigate to ArgoCD → login → Keycloak SSO → enter valid credentials
4. "Invalid username or password"

---

## Fix Applied (manual, live cluster)

1. Set `browser-with-conditional-otp forms` from ALTERNATIVE → REQUIRED
2. Deleted the orphaned top-level `otp-conditional-subflow` execution (execution ID `76a54bd1`)

**Correct flow structure:**
```
Cookie                                    ALTERNATIVE  level:0
Kerberos                                  DISABLED     level:0
Identity Provider Redirector              ALTERNATIVE  level:0
browser-with-conditional-otp forms        REQUIRED     level:0   ← was ALTERNATIVE
  Username Password Form                  REQUIRED     level:1
  Browser - Conditional OTP              CONDITIONAL  level:1
    Condition - user configured           REQUIRED     level:2
    OTP Form                             REQUIRED     level:2
```

**Permanent fix needed:** The reconcile hook job must not add `otp-conditional-subflow` as
a top-level execution. It should either be nested inside `browser-with-conditional-otp forms`
or the `browser-with-conditional-otp forms` step must be set to REQUIRED before adding peers.
