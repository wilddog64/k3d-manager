# Bugfix: v1.6.3 — data-layer sync races ArgoCD controller reconnect to ubuntu-k3s

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/acg-up`

---

## Problem

Step 10b immediately applies and syncs `data-layer` after the ArgoCD application controller
rollout restart at step 10. `kubectl rollout status` confirms the pod is running, but the
controller hasn't yet re-established its connection to the `ubuntu-k3s` cluster. ArgoCD
attempts to sync resources on `ubuntu-k3s` before the cluster connection is live, causing
the sync to stall and eventually timeout.

**Root cause:** No wait for ArgoCD cluster connectivity to `ubuntu-k3s` between the
controller restart (line 592) and the `data-layer` apply (line 612). `rollout status`
only confirms the pod is scheduled — not that cluster watches are re-established.

---

## Reproduction

```
bin/acg-up   # fresh run or acg-resume hitting step 10 → 10b
# controller restarts at step 10
# data-layer apply fires immediately at step 10b
# ArgoCD shows data-layer OutOfSync or sync stalled
# → _err "[acg-up] data-layer ArgoCD Application did not reach Synced after force-sync + 180s retry"
```

---

## Fix

### Change 1 — `bin/acg-up`: wait for ArgoCD cluster connection to ubuntu-k3s before step 10b

Add a readiness loop between the controller `rollout status` (line 593) and the step 10b
checkpoint check (line 608). Poll `argocd cluster list` JSON output until the `ubuntu-k3s`
cluster shows `connectionState.status == "Successful"`, with a 120s deadline.

**Exact old block (lines 592–607):**

```bash
kubectl rollout status statefulset/argocd-application-controller \
  -n cicd --context k3d-k3d-cluster --timeout=300s
_info "[acg-up] ArgoCD cluster registered: ubuntu-k3s → https://host.k3d.internal:6443"

_info "[acg-up] Ensuring argocd-redis secret on ubuntu-k3s (Helm chart hook skipped when ArgoCD manages ArgoCD)..."
kubectl create namespace cicd --context ubuntu-k3s \
  --dry-run=client -o yaml | kubectl apply --context ubuntu-k3s -f - >/dev/null
if ! kubectl get secret argocd-redis -n cicd --context ubuntu-k3s >/dev/null 2>&1; then
  kubectl create secret generic argocd-redis -n cicd --context ubuntu-k3s \
    --from-literal=auth="$(openssl rand -base64 32)"
  _info "[acg-up] argocd-redis secret created on ubuntu-k3s"
else
  _info "[acg-up] argocd-redis secret already exists on ubuntu-k3s"
fi

_info "[acg-up] Step 10b/14 — Deploying shopping-cart data layer (PostgreSQL, Redis, RabbitMQ)..."
```

**Exact new block:**

```bash
kubectl rollout status statefulset/argocd-application-controller \
  -n cicd --context k3d-k3d-cluster --timeout=300s
_info "[acg-up] ArgoCD cluster registered: ubuntu-k3s → https://host.k3d.internal:6443"

_info "[acg-up] Waiting for ArgoCD controller to reconnect to ubuntu-k3s (max 120s)..."
_argocd_conn_deadline=$(( $(date +%s) + 120 ))
until argocd cluster list -o json \
    --server localhost:8080 --insecure 2>/dev/null \
    | python3 -c "import sys,json; clusters=json.load(sys.stdin); \
      print(next((c['connectionState']['status'] for c in clusters \
        if 'ubuntu-k3s' in c.get('name','') or 'host.k3d.internal' in c.get('server','')), \
        'Unknown'))" 2>/dev/null | grep -q "Successful"; do
  if [[ $(date +%s) -ge ${_argocd_conn_deadline} ]]; then
    _warn "[acg-up] ArgoCD controller did not reconnect to ubuntu-k3s within 120s — proceeding anyway"
    break
  fi
  _info "[acg-up] ArgoCD controller not yet connected to ubuntu-k3s — waiting..."
  sleep 5
done
_info "[acg-up] ArgoCD controller connected to ubuntu-k3s — proceeding"

_info "[acg-up] Ensuring argocd-redis secret on ubuntu-k3s (Helm chart hook skipped when ArgoCD manages ArgoCD)..."
kubectl create namespace cicd --context ubuntu-k3s \
  --dry-run=client -o yaml | kubectl apply --context ubuntu-k3s -f - >/dev/null
if ! kubectl get secret argocd-redis -n cicd --context ubuntu-k3s >/dev/null 2>&1; then
  kubectl create secret generic argocd-redis -n cicd --context ubuntu-k3s \
    --from-literal=auth="$(openssl rand -base64 32)"
  _info "[acg-up] argocd-redis secret created on ubuntu-k3s"
else
  _info "[acg-up] argocd-redis secret already exists on ubuntu-k3s"
fi

_info "[acg-up] Step 10b/14 — Deploying shopping-cart data layer (PostgreSQL, Redis, RabbitMQ)..."
```
