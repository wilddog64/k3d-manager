# Bug: `make up` fails with `sudo: unable to allocate pty` when trusting Vault CA cert

**Branch:** `k3d-manager-v1.5.0`
**File:** `bin/acg-up:297`
**Severity:** P2 — blocks `make up` after a fresh sandbox start

---

## Symptom

```
INFO: [acg-up] Trusting Vault CA cert in macOS System Keychain (may prompt for sudo password)...
sudo: unable to allocate pty: Device not configured
make: *** [up] Error 1
```

`make up` exits non-zero at the Vault CA trust step. ArgoCD browser HTTPS setup never completes.

---

## Root Cause

`make` spawns `bin/acg-up` in a subshell with no controlling TTY. When `_run_command
--interactive-sudo` calls `sudo security add-trusted-cert`, sudo tries to allocate a PTY
for the password prompt and fails because the process has no terminal.

The call at `bin/acg-up:297` is:

```bash
_run_command --interactive-sudo --soft -- \
  security add-trusted-cert -d -r trustRoot \
    -k /Library/Keychains/System.keychain \
    "${_argocd_browser_tls_dir}/ca.crt"
```

The `--soft` flag suppresses the error, but `make` still sees the non-zero exit code from
sudo and aborts.

---

## Fix

Add an idempotency check before calling `sudo`. If the Vault CA cert is already trusted in
the System Keychain, skip the `add-trusted-cert` call entirely — no sudo needed, no PTY
needed.

`security verify-cert -c <cert>` returns 0 if the cert is already trusted.

**`bin/acg-up:295–301` — replace with:**

```bash
if [[ "$(uname)" == "Darwin" ]]; then
  if ! security verify-cert -c "${_argocd_browser_tls_dir}/ca.crt" >/dev/null 2>&1; then
    _info "[acg-up] Trusting Vault CA cert in macOS System Keychain (may prompt for sudo password)..."
    _run_command --interactive-sudo --soft -- \
      security add-trusted-cert -d -r trustRoot \
        -k /Library/Keychains/System.keychain \
        "${_argocd_browser_tls_dir}/ca.crt"
  else
    _info "[acg-up] Vault CA cert already trusted in System Keychain — skipping"
  fi
fi
```

---

## Behaviour After Fix

| Scenario | Before | After |
|----------|--------|-------|
| First `make up` after fresh sandbox (cert not yet trusted) | PTY error, `make` aborts | Prompts for sudo password once in terminal |
| Subsequent `make up` (cert already trusted) | PTY error, `make` aborts | Skips sudo entirely — silent |
| Running `make up` directly in terminal (TTY present) | Works | Works (unchanged) |
