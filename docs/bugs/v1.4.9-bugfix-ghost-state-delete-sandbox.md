# Bugfix: v1.4.9 — Ghost State Delete Sandbox never clicks

**Branch:** `k3d-manager-v1.4.9`
**File:** `scripts/lib/acg/playwright/acg_extend.js`

---

## Problem

In the Ghost State recovery path (TTL < 15 min, extend button not found), the script
re-navigates to the listing page and immediately calls `deleteBtn.isVisible({ timeout: 30000 })`.
The wait between re-navigation and the visibility check is `waitForFunction(() => !document.querySelector('[aria-busy="true"]'))`,
which resolves instantly because Pluralsight does not set `aria-busy` on its skeleton loaders.
React has not rendered the sandbox cards yet when `isVisible` runs, so the timeout expires
and the Delete Sandbox click never happens.

**Root cause:** `waitForFunction(aria-busy)` is not a reliable React render signal for
the Pluralsight SPA. The "Delete Sandbox" button appears only after React renders the sandbox
card, which happens after a network-fetched data response — not at `domcontentloaded`.
Pluralsight does not set `aria-busy` on its card skeleton loaders, so the `waitForFunction`
resolves immediately (no element matching `[aria-busy="true"]` exists).

**Prior fix (insufficient):** lib-acg PR #18 (`0b40f425`) added the `waitForFunction(aria-busy)`
wait and extended `deleteBtn.isVisible` from 5s to 30s. The `aria-busy` wait still resolves
immediately. The real fix is to wait for the actual sandbox card buttons to appear in the DOM.

---

## Reproduction

1. Let sandbox TTL drop to < 15 minutes
2. Cause Ghost State (navigate away so extend panel button is not visible)
3. Run `acg_extend_playwright <sandbox_url>` — script exits without clicking Delete Sandbox
4. Browser shows Delete Sandbox button clearly visible on listing page

---

## Fix

### Change 1 — `scripts/lib/acg/playwright/acg_extend.js`: add `waitForSelector` after re-navigation

**Exact old block (lines 295–302):**

```javascript
      // Wait for SPA to render sandbox cards after navigation (domcontentloaded fires before React renders)
      await page.waitForFunction(
        () => !document.querySelector('[aria-busy="true"]'),
        { timeout: 30000 }
      ).catch(() => console.error('WARN: Skeleton loaders did not clear after Ghost State re-navigation — proceeding'));

      const deleteBtn = page.locator('button:has-text("Delete Sandbox")').first();
      if (await deleteBtn.isVisible({ timeout: 30000 }).catch(() => false)) {
```

**Exact new block:**

```javascript
      // Wait for SPA to render sandbox cards after navigation (domcontentloaded fires before React renders)
      await page.waitForFunction(
        () => !document.querySelector('[aria-busy="true"]'),
        { timeout: 30000 }
      ).catch(() => console.error('WARN: Skeleton loaders did not clear after Ghost State re-navigation — proceeding'));

      // Wait for sandbox card buttons — aria-busy alone is not a reliable render signal for Pluralsight SPA
      await page.waitForSelector(
        'button:has-text("Delete Sandbox"), button:has-text("Start Sandbox"), button:has-text("Open Sandbox")',
        { timeout: 30000 }
      ).catch(() => console.error('WARN: Sandbox card controls did not appear after re-navigation — attempting anyway'));

      const deleteBtn = page.locator('button:has-text("Delete Sandbox")').first();
      if (await deleteBtn.isVisible({ timeout: 10000 }).catch(() => false)) {
```
