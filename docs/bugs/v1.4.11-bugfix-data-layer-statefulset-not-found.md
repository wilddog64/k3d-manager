# Bugfix: v1.4.11 — deploy_shopping_cart_data: StatefulSet not found on fresh cluster

**Branch:** `k3d-manager-v1.4.11`
**Files:** `scripts/plugins/shopping_cart.sh`

---

## Problem

`make up` fails at Step 10b with:

```
Error from server (NotFound): statefulsets.apps "postgresql-orders" not found
```

`deploy_shopping_cart_data()` calls `kubectl rollout status statefulset/postgresql-orders`
immediately after ArgoCD registers the cluster. ArgoCD has not yet synced the `data-layer`
app and created the StatefulSets, so kubectl returns NotFound and exits non-zero, which
fails the step.

**Root cause:** Lines 98–102 of `shopping_cart.sh` call `kubectl rollout status` with no
existence check — the StatefulSets must be created by ArgoCD before `rollout status` is valid.

---

## Reproduction

1. `make up` on a fresh cluster (no prior data-layer StatefulSets)
2. Step 10b exits non-zero immediately after `kubectl rollout status` is called

---

## Fix

### Change 1 — `scripts/plugins/shopping_cart.sh` (before line 98)

Add a per-StatefulSet existence poll (60s deadline each) before the `rollout status` loop.

**Exact old block (lines 96–102):**
```bash
  _info "[shopping_cart] Data layer managed by ArgoCD — waiting for readiness..."

  _info "[shopping_cart] Waiting for PostgreSQL instances to be Ready..."
  for pg in postgresql-orders postgresql-payment postgresql-products; do
    kubectl rollout status statefulset/"${pg}" \
      -n shopping-cart-data --context ubuntu-k3s --timeout=120s
  done
```

**Exact new block:**
```bash
  _info "[shopping_cart] Data layer managed by ArgoCD — waiting for readiness..."

  _info "[shopping_cart] Waiting for ArgoCD to create data-layer StatefulSets..."
  local _sts_deadline
  for pg in postgresql-orders postgresql-payment postgresql-products; do
    _sts_deadline=$(( $(date +%s) + 300 ))
    until kubectl get statefulset/"${pg}" \
        -n shopping-cart-data --context ubuntu-k3s >/dev/null 2>&1; do
      if [[ $(date +%s) -ge ${_sts_deadline} ]]; then
        _err "[shopping_cart] Timed out waiting for StatefulSet ${pg} to be created by ArgoCD"
        return 1
      fi
      _info "[shopping_cart] StatefulSet ${pg} not yet created — waiting..."
      sleep 10
    done
  done

  _info "[shopping_cart] Waiting for PostgreSQL instances to be Ready..."
  for pg in postgresql-orders postgresql-payment postgresql-products; do
    kubectl rollout status statefulset/"${pg}" \
      -n shopping-cart-data --context ubuntu-k3s --timeout=120s
  done
```
