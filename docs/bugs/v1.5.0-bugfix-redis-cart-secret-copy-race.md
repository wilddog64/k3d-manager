# Bug: `redis-cart-secret` copy fails — secret not yet created when copy runs

**Branch:** `k3d-manager-v1.5.0`
**File:** `scripts/plugins/shopping_cart.sh:120–123`
**Severity:** P2 — blocks `make up` on fresh sandbox

---

## Symptom

```
Error from server (NotFound): secrets "redis-cart-secret" not found
json.decoder.JSONDecodeError: Expecting value: line 1 column 1 (char 0)
error: no objects passed to apply
make: *** [up] Error 1
```

---

## Root Cause

Line 98 applies the Redis cart manifests:
```bash
_run_command -- kubectl apply --context ubuntu-k3s -f "${infra_root}/redis/cart/"
```

Line 121 immediately tries to copy `redis-cart-secret` from `shopping-cart-data`:
```bash
kubectl get secret redis-cart-secret -n shopping-cart-data --context ubuntu-k3s -o json \
  | python3 -c "..." \
  | kubectl apply --context ubuntu-k3s -f -
```

There is no wait between the two. The Redis StatefulSet and its associated Secret are not
yet created when the copy runs — `kubectl get secret` returns NotFound, Python receives
empty stdin, and the pipeline fails.

---

## Fix

Add a `kubectl wait` for the `redis-cart-secret` before the copy step.
`redis-cart-secret` is created by an ExternalSecret or the Redis operator once the
StatefulSet is Running — wait for the secret to exist, then copy.

**`scripts/plugins/shopping_cart.sh:120` — replace with:**

```bash
  _info "[shopping_cart] Waiting for redis-cart-secret to be available in shopping-cart-data (up to 120s)..."
  kubectl wait secret/redis-cart-secret \
    -n shopping-cart-data --context ubuntu-k3s \
    --for=jsonpath='{.metadata.name}'=redis-cart-secret \
    --timeout=120s 2>/dev/null \
  || {
    # kubectl wait --for=jsonpath not available in older k8s — fall back to poll
    local _i=0
    until kubectl get secret redis-cart-secret -n shopping-cart-data --context ubuntu-k3s \
        >/dev/null 2>&1; do
      (( _i++ )); (( _i >= 24 )) && { _err "[shopping_cart] redis-cart-secret not found after 120s"; return 1; }
      sleep 5
    done
  }

  _info "[shopping_cart] Copying redis-cart-secret to shopping-cart-apps namespace..."
  kubectl get secret redis-cart-secret -n shopping-cart-data --context ubuntu-k3s -o json \
    | python3 -c "import sys,json; d=json.load(sys.stdin); d['metadata']={'name':'redis-cart-secret','namespace':'shopping-cart-apps'}; print(json.dumps(d))" \
    | kubectl apply --context ubuntu-k3s -f -
```

---

## Behaviour After Fix

| Scenario | Before | After |
|----------|--------|-------|
| Fresh sandbox — Redis secret not yet ready | JSONDecodeError, `make` aborts | Waits up to 120s, then copies |
| Secret already exists (idempotent re-run) | Copies immediately | Copies immediately (wait returns at once) |
| Secret never appears (misconfigured) | Silent failure / bad error | Clear timeout error after 120s |
