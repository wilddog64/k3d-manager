# Bugfix: v1.4.5 — shopping-cart-identity Application references non-existent AppProject

**Branch:** `k3d-manager-v1.4.5`
**Files:** `bin/acg-up`, `services/shopping-cart-identity/kustomization.yaml`

---

## Problem

After every cluster rebuild, `shopping-cart-identity` appears in the `cicd` namespace with
`Unknown / Unknown` health and sync status. The Application is stuck with:

```
Application referencing project shopping-cart which does not exist
```

Only the `default` and `platform` AppProjects are created during bootstrap. There is no
`shopping-cart` AppProject.

**Root cause:** `bin/acg-up` step 10c hard-applies a `shopping-cart-identity` Application
inline via `kubectl apply` with `spec.project: shopping-cart`. This Application is applied
on every rebuild and can never sync because the referenced AppProject does not exist.

Secondary issue: `services/shopping-cart-identity/kustomization.yaml` is dead code. It was
committed to the `services/` directory but the `services-git` ApplicationSet excludes
`services/shopping-cart-identity` (fixed in the previous bugfix). Nothing reads this file.

---

## Reproduction

```bash
# After make up / acg-up:
kubectl get app shopping-cart-identity -n cicd \
  -o jsonpath='{.status.conditions[0].message}' --context k3d-k3d-cluster
# Actual:   Application referencing project shopping-cart which does not exist
# Expected: (no conditions — app syncs cleanly under platform project)
```

---

## Fix

### Change 1 — `bin/acg-up` line 509: change project reference

**Old:**
```yaml
  project: shopping-cart
```

**New:**
```yaml
  project: platform
```

The `platform` AppProject already permits:
- `sourceRepos: '*'`
- destination `namespace: identity` at `server: https://kubernetes.default.svc`

No AppProject changes needed.

### Change 2 — delete `services/shopping-cart-identity/kustomization.yaml`

The file is excluded from the `services-git` ApplicationSet and is not referenced by any
script or plugin. Removing it eliminates confusion about whether ArgoCD manages it.

```bash
git rm services/shopping-cart-identity/kustomization.yaml
rmdir services/shopping-cart-identity
```
