# Bugfix: v1.6.1 — LaunchAgent plists missing RunAtLoad in acg-up

**Branch:** `k3d-manager-v1.6.1`
**File:** `bin/acg-up`

---

## Problem

After a macOS upgrade, sleep/wake cycle, or user session restart, two user-domain
LaunchAgents generated by `bin/acg-up` are not automatically started by launchd:

- `com.k3d-manager.argocd-port-forward` (port 8080 → k3d ArgoCD)
- `com.k3d-manager.keycloak-port-forward` (port 8880 → Keycloak)

**Symptom:** `argocd.3ai-talk.org` (and Keycloak) returns Cloudflare 502 Bad Gateway.
`lsof -i :8080` and `lsof -i :8880` return empty. `launchctl list | grep argocd` returns
nothing.

**Root cause:** Both plists are generated by `bin/acg-up` heredocs with `KeepAlive: true`
but no `RunAtLoad: true`. macOS launchd only auto-starts a LaunchAgent on boot/login when
`RunAtLoad` is present. Without it, the plist is installed on disk but launchd never
starts the process after a session restart — `KeepAlive` never activates because the
agent never ran in the first place.

All five LaunchDaemon plists generated by the same script have `RunAtLoad: true` already.
Only the two LaunchAgent (user-domain) plists are missing it.

**Discovered:** 2026-06-05 after suspected macOS upgrade unloaded the ArgoCD agent.
Temporary fix applied manually (added `RunAtLoad` to the installed plist and re-bootstrapped).
This spec makes it permanent by fixing the generator.

## Fix

Two identical two-line insertions in `bin/acg-up` — one for the ArgoCD plist heredoc,
one for the Keycloak plist heredoc.

### Change 1 — `bin/acg-up` line 309: ArgoCD port-forward plist

**Exact old block:**
```bash
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>${_argocd_pf_log}</string>
```

**Exact new block:**
```bash
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>${_argocd_pf_log}</string>
```

### Change 2 — `bin/acg-up` line 1195: Keycloak port-forward plist

**Exact old block:**
```bash
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>${_kc_pf_log}</string>
```

**Exact new block:**
```bash
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>${_kc_pf_log}</string>
```
