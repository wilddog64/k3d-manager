# Bugfix: v1.6.1 — LaunchAgent plists missing RunAtLoad in acg-up

**Branch:** `k3d-manager-v1.6.1`
**File:** `bin/acg-up`

---

## Problem

After a macOS upgrade, sleep/wake cycle, or user session restart, three user-domain
LaunchAgents are not automatically started by launchd:

- `com.k3d-manager.argocd-port-forward` (port 8080 → k3d ArgoCD)
- `com.k3d-manager.keycloak-port-forward` (port 8880 → Keycloak)
- `com.k3d-manager.frontend-port-forward` (port 3000 → shopping-cart frontend, ubuntu-k3s)

**Symptom:** `argocd.3ai-talk.org`, `keycloak.3ai-talk.org` return Cloudflare 502.
`lsof -i :8080`, `lsof -i :8880`, `lsof -i :3000` all return empty.

**Root cause — ArgoCD + Keycloak:** Both plists are generated by `bin/acg-up` heredocs
with `KeepAlive: true` but no `RunAtLoad: true`. macOS launchd only auto-starts a
LaunchAgent on boot/login when `RunAtLoad` is present. Without it, the plist is
installed on disk but launchd never starts the process after a session restart.

**Root cause — frontend:** The `frontend-port-forward` plist was hand-crafted and has
no generator in the codebase. It is missing both `RunAtLoad: true` and a `KUBECONFIG`
`EnvironmentVariables` entry (required because `ubuntu-k3s` context lives in
`~/.kube/k3s-ubuntu.yaml`, not in the default `~/.kube/config`). Without KUBECONFIG set,
kubectl cannot find the context and exits immediately — `KeepAlive` loops forever.

All five LaunchDaemon plists (system domain) generated by the same script already have
`RunAtLoad: true`. Only the three user-domain LaunchAgent plists are affected.

**Discovered:** 2026-06-05 after macOS upgrade unloaded all agents from the gui domain.
Temporary fixes applied manually to the installed argocd and keycloak plists.
This spec makes it permanent by fixing all three generators.

### Change 3 — `bin/acg-up` ~line 1209: Add frontend-port-forward generator

Insert the following block immediately after the closing `fi` of the Keycloak
port-forward section (the line that reads `_info "[acg-up] Keycloak port-forward agent loaded..."`).

**Anchor — insert AFTER this line:**
```bash
  _info "[acg-up] Keycloak port-forward agent loaded (launchd: ${_kc_pf_label})"
fi
```

**Block to insert:**
```bash

if _is_mac; then
  _info "[acg-up] Step 10g.10/14 — Installing frontend port-forward launchd agent (localhost:3000 → ubuntu-k3s svc/frontend, auto-restart)..."
  _frontend_pf_label="com.k3d-manager.frontend-port-forward"
  _frontend_pf_plist="${HOME}/Library/LaunchAgents/${_frontend_pf_label}.plist"
  _frontend_pf_log="${_ACG_STATE_DIR}/logs/frontend-pf.log"
  _frontend_pf_kubectl="$(command -v kubectl)"
  _frontend_pf_kubeconfig="${HOME}/.kube/config:${HOME}/.kube/ubuntu-k3s.yaml:${HOME}/.kube/k3s-ubuntu.yaml"
  mkdir -p "$(dirname "${_frontend_pf_log}")"
  cat > "${_frontend_pf_plist}" <<PLIST
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>${_frontend_pf_label}</string>
  <key>ProgramArguments</key>
  <array>
    <string>${_frontend_pf_kubectl}</string>
    <string>port-forward</string>
    <string>svc/frontend</string>
    <string>--namespace</string>
    <string>shopping-cart-apps</string>
    <string>--context</string>
    <string>ubuntu-k3s</string>
    <string>3000:80</string>
  </array>
  <key>EnvironmentVariables</key>
  <dict>
    <key>KUBECONFIG</key>
    <string>${_frontend_pf_kubeconfig}</string>
  </dict>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>${_frontend_pf_log}</string>
  <key>StandardErrorPath</key>
  <string>${_frontend_pf_log}</string>
</dict>
</plist>
PLIST
  if launchctl list "${_frontend_pf_label}" >/dev/null 2>&1; then
    launchctl unload "${_frontend_pf_plist}" 2>/dev/null || true
  fi
  launchctl load "${_frontend_pf_plist}"
  _info "[acg-up] Frontend port-forward agent loaded (launchd: ${_frontend_pf_label})"
fi
```

**Important heredoc note:** The `PLIST` heredoc delimiter is unquoted, so variables like
`${_frontend_pf_label}`, `${_frontend_pf_kubectl}`, `${_frontend_pf_kubeconfig}`, and
`${_frontend_pf_log}` expand at shell runtime and are baked into the generated plist file.
This is intentional and matches the ArgoCD and Keycloak heredoc pattern already in the file.
