# Bugfix: v1.6.3 — `ask gemini/codex what happened` rejected when job context exists

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/k3dm-webhook`

---

## Problem

When a user types `ask gemini what happened` (or `ask codex ...`) in a Slack thread where
a job previously ran, the response is:

```
❌ Question rejected — too long or contains disallowed patterns.
```

**Root cause:** `_run_cluster_ask` calls `_sanitize_question(question)` on line 1810, but
`question` at that point is the **assembled prompt** — it includes up to 80 lines of job
output prepended by the `ask` command handler (lines 633–644). That assembled string
routinely exceeds the 500-character limit in `_sanitize_question`.

The 500-char limit is correct for raw user input (prevents abuse), but the limit fires too
late — after the job context has already been prepended.

---

## Reproduction

1. Run any job (e.g. `up`) so that a job log exists in `JOB_DIR/<job_id>/log`.
2. In the Slack thread for that job, type: `ask gemini what happened`
3. Observe: `❌ Question rejected — too long or contains disallowed patterns.`

---

## Fix

### Change 1 — `ask` command handler: sanitize user question before context is prepended

**Exact old block (lines 626–631):**

```python
        import re as _re
        _turns_match = _re.search(r'\bturns=(\d+)\b', question)
        ask_max_turns = int(_turns_match.group(1)) if _turns_match else _ASK_MAX_TURNS_DEFAULT
        if _turns_match:
            question = _re.sub(r'\s*turns=\d+\s*', ' ', question).strip()
        _job_log = JOB_DIR / job_id / "log"
```

**Exact new block:**

```python
        import re as _re
        _turns_match = _re.search(r'\bturns=(\d+)\b', question)
        ask_max_turns = int(_turns_match.group(1)) if _turns_match else _ASK_MAX_TURNS_DEFAULT
        if _turns_match:
            question = _re.sub(r'\s*turns=\d+\s*', ' ', question).strip()
        question = _sanitize_question(question)
        if not question:
            _notify_job(job_id, "❌ Question rejected — too long or contains disallowed patterns.")
            return
        _job_log = JOB_DIR / job_id / "log"
```

---

### Change 2 — `_run_cluster_ask`: skip length check, keep injection check only

The question passed here may already include job context (up to 80 lines). The user
portion was already validated in Change 1, so only the injection check is needed.

**Exact old block (lines 1810–1813):**

```python
    question = _sanitize_question(question)
    if not question:
        _finish("❌ Question rejected — too long or contains disallowed patterns.", status="failed")
        return
```

**Exact new block:**

```python
    # Injection check only — length was validated on the raw user input before context was added
    _q_stripped = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f\x7f]', '', question.strip())
    if _INJECTION_RE.search(_q_stripped):
        _finish("❌ Question rejected — contains disallowed patterns.", status="failed")
        return
    question = _q_stripped
```
