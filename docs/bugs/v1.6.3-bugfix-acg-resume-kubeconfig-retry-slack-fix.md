# Bugfix: v1.6.3 — acg-resume kubeconfig fetch no retry + Gemini fix advice uses make command

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/acg-up`, `bin/k3dm-webhook`

---

## Problem

Two issues observed after the step-3c kubeconfig guard landed (`a4c5bc95`):

### Issue 1 — No retry in kubeconfig guard (`bin/acg-up`)

The guard calls `add_ubuntu_k3s_cluster` once with no retry. The SSH tunnel was started
in step 3 moments before the guard runs — the port may be open but the remote sshd is
not yet fully accepting connections. A single transient SSH failure causes the entire
resume to fail with exit 1.

### Issue 2 — Gemini fix advice says `make up ...` instead of `/acg-resume`

`_analyze_failure` passes Gemini the example:

```
'Run acg-up again'
```

Gemini echoes this pattern and produces fix text like:

```
Run acg-up again: make up CLUSTER_PROVIDER=k3s-aws K3DM_RESUME=1
```

Users interact with the cluster via Slack — `make up ...` is not the user-facing command.
The fix advice should say `/acg-resume`.

---

## Fix

### Change 1 — `bin/acg-up`: retry loop in step-3c guard

Replace the single `add_ubuntu_k3s_cluster` call with a 3-attempt loop (5s between
attempts). Fail with `_err` after all attempts exhausted.

**Exact old block (lines 513–517):**

```bash
# ── 3c. ubuntu-k3s kubeconfig guard ──────────────────────────────────────────
if ! kubectl config get-contexts ubuntu-k3s &>/dev/null; then
  _info "[acg-up] ubuntu-k3s context missing — re-fetching kubeconfig via SSH tunnel..."
  add_ubuntu_k3s_cluster
fi
```

**Exact new block:**

```bash
# ── 3c. ubuntu-k3s kubeconfig guard ──────────────────────────────────────────
if ! kubectl config get-contexts ubuntu-k3s &>/dev/null; then
  _info "[acg-up] ubuntu-k3s context missing — re-fetching kubeconfig via SSH tunnel..."
  _k3s_kubeconfig_fetched=0
  for _attempt in 1 2 3; do
    if add_ubuntu_k3s_cluster; then
      _k3s_kubeconfig_fetched=1
      break
    fi
    _warn "[acg-up] kubeconfig fetch attempt ${_attempt}/3 failed — waiting 5s before retry..."
    sleep 5
  done
  if [[ "${_k3s_kubeconfig_fetched}" -eq 0 ]]; then
    _err "[acg-up] Failed to fetch ubuntu-k3s kubeconfig after 3 attempts — SSH tunnel may not be fully up. Retry via /acg-resume in Slack."
  fi
fi
```

### Change 2 — `bin/k3dm-webhook`: update Gemini prompt fix example

Replace the `'Run acg-up again'` example in the Gemini prompt with the Slack command
so Gemini produces user-facing fix advice.

**Exact old string (line 794):**

```

**Exact new string:**

```
