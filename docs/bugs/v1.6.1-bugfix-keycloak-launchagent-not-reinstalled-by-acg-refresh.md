# Bugfix: v1.6.1 — Keycloak port-forward LaunchAgent not reinstalled by acg-refresh

**Branch:** `k3d-manager-v1.6.1`
**Files:** `bin/acg-refresh`

---

## Problem

After a cluster rebuild where `bin/acg-up` does not reach Step 10g.9 (or the step is
interrupted), the Keycloak port-forward plist (`~/Library/LaunchAgents/com.k3d-manager.keycloak-port-forward.plist`)
is absent. `bin/acg-refresh` calls `_launchd_ensure` for the Keycloak agent but
`_launchd_ensure` silently returns when the plist file is missing (`[[ -f "${plist}" ]] || return 0`).
Port 8880 stays dark, and `keycloak.3ai-talk.org` returns 502.

**Root cause:** `_launchd_ensure` only reloads an existing plist — it cannot regenerate
a missing one. When `acg-up` is interrupted before Step 10g.9, the wrapper script exists
at `${_ACG_STATE_DIR}/keycloak-port-forward.sh` or `${_ACG_STATE_DIR}/bin/keycloak-port-forward.sh`
but no plist is present, so `acg-refresh` silently skips the agent.

---

## Reproduction

1. Run `bin/acg-up` and interrupt it before Step 10g.9, OR run a full rebuild that
   completes `acg-down` (which `rm -f`s the plist) and does not complete `acg-up`
2. Run `make refresh`
3. Observe: no `INFO: [acg-refresh] launchd com.k3d-manager.keycloak-port-forward: ...` line
4. `nc -zv localhost 8880` → "Connection refused"
5. `keycloak.3ai-talk.org` → 502

---

## Fix

### Change 1 — `bin/acg-refresh` ~line 175: regenerate Keycloak plist when missing

**Exact old block (lines 175–178):**
```bash
  _launchd_ensure \
    "com.k3d-manager.keycloak-port-forward" \
    "${HOME}/Library/LaunchAgents/com.k3d-manager.keycloak-port-forward.plist" \
    user
```

**Exact new block:**
```bash
  _kc_pf_label="com.k3d-manager.keycloak-port-forward"
  _kc_pf_plist="${HOME}/Library/LaunchAgents/${_kc_pf_label}.plist"
  _kc_pf_log="${_ACG_STATE_DIR}/logs/keycloak-pf.log"
  _kc_pf_wrapper=""
  if [[ -f "${_ACG_STATE_DIR}/bin/keycloak-port-forward.sh" ]]; then
    _kc_pf_wrapper="${_ACG_STATE_DIR}/bin/keycloak-port-forward.sh"
  elif [[ -f "${_ACG_STATE_DIR}/keycloak-port-forward.sh" ]]; then
    _kc_pf_wrapper="${_ACG_STATE_DIR}/keycloak-port-forward.sh"
  fi
  if [[ ! -f "${_kc_pf_plist}" && -n "${_kc_pf_wrapper}" ]]; then
    _info "[acg-refresh] Keycloak port-forward plist missing — regenerating from wrapper..."
    mkdir -p "$(dirname "${_kc_pf_log}")"
    cat > "${_kc_pf_plist}" <<PLIST
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>${_kc_pf_label}</string>
  <key>ProgramArguments</key>
  <array>
    <string>/bin/bash</string>
    <string>${_kc_pf_wrapper}</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>${_kc_pf_log}</string>
  <key>StandardErrorPath</key>
  <string>${_kc_pf_log}</string>
</dict>
</plist>
PLIST
  fi
  _launchd_ensure \
    "${_kc_pf_label}" \
    "${_kc_pf_plist}" \
    user
```
