## Problem

After `make up`, the `register_app_cluster` call in step 10 applies the cluster secret
to the wrong Kubernetes cluster. `_kubectl apply -f -` inside `register_app_cluster`
uses the current kubectl context, which is `ubuntu-k3s` (the remote EC2 cluster) at the
time step 10 runs. The `cicd` namespace only exists on the local `k3d-k3d-cluster`, so
the apply either fails silently or is a no-op. The cluster secret on the local cluster
retains whatever stale IP was there from a previous run.

**Root cause:** `bin/acg-up` step 10 does not switch the kubectl current-context to
`k3d-k3d-cluster` before calling `register_app_cluster`, so `_kubectl` inside that
function targets the remote cluster instead of the local one where ArgoCD lives.

---

## Reproduction

1. Run `make up` — note the current context ends as `ubuntu-k3s` after k3sup runs
2. Run `argocd cluster list` — `ubuntu-k3s` server URL is stale (old IP from previous sandbox)
3. Run `kubectl get secret cluster-ubuntu-k3s -n cicd --context k3d-k3d-cluster -o jsonpath='{.data.server}' | base64 -d` — confirms old IP was never updated

---

## Fix

### Change 1 — `bin/acg-up` lines 261–268: switch context before `register_app_cluster`

Add `kubectl config use-context k3d-k3d-cluster` immediately before the
`register_app_cluster` call so `_kubectl apply` targets the local cluster.
Also switch to the more reliable jsonpath name filter (not `--minify`) to read
the server URL.

**Exact old block (lines 261–268):**

```bash
_cp_server=$(kubectl config view --minify --context ubuntu-k3s \
  -o jsonpath='{.clusters[0].cluster.server}' 2>/dev/null || echo "")
if [[ -z "${_cp_server}" ]]; then
  _err "[acg-up] Could not determine ubuntu-k3s API server URL — is the kubeconfig set up?"
fi
ARGOCD_APP_CLUSTER_TOKEN="${_argocd_sa_token}" \
ARGOCD_APP_CLUSTER_SERVER="${_cp_server}" \
  "${REPO_ROOT}/scripts/k3d-manager" register_app_cluster
```

**Exact new block:**

```bash
_cp_server=$(kubectl config view \
  -o jsonpath='{.clusters[?(@.name=="ubuntu-k3s")].cluster.server}' 2>/dev/null || echo "")
if [[ -z "${_cp_server}" ]]; then
  _err "[acg-up] Could not determine ubuntu-k3s API server URL — is the kubeconfig set up?"
fi
kubectl config use-context k3d-k3d-cluster >/dev/null
ARGOCD_APP_CLUSTER_TOKEN="${_argocd_sa_token}" \
ARGOCD_APP_CLUSTER_SERVER="${_cp_server}" \
  "${REPO_ROOT}/scripts/k3d-manager" register_app_cluster
```
