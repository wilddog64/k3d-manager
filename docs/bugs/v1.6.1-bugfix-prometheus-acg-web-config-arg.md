# Bugfix: v1.6.1 — Prometheus ACG StatefulSet fails to reconcile (web.config.file blocked)

**Branch:** `k3d-manager-v1.6.1`
**Files:** `scripts/etc/helm/observability/kube-prometheus-stack-acg-values.yaml`

---

## Problem

The `acg-kube-prometheus-stack` ArgoCD application is Degraded on `ubuntu-k3s`. The
`prometheus-acg-kube-prometheus-stack-prometheus` StatefulSet never gets created because
the Prometheus operator rejects the spec.

**Root cause:** `prometheus-operator v0.79.2` added `web.config.file` to its internal
managed-args denylist. Setting it via `additionalArgs` in Helm values now produces:
`can't set arguments which are already managed by the operator: web.config.file`

The operator reconciliation loop fails on every attempt, leaving Prometheus at 0 replicas.

---

## Reproduction

1. Deploy ACG observability stack on a cluster running `prometheus-operator v0.79+`.
2. Check Prometheus CR: `kubectl describe prometheus acg-kube-prometheus-stack-prometheus -n monitoring`
3. Observe `ReconciliationFailed: making statefulset failed: make StatefulSet spec: can't set arguments which are already managed by the operator: web.config.file`

---

## Fix

Remove `additionalArgs` and the `prometheus-web-config` secret mount from the ACG Helm
values. The `prometheus-web-config` secret is still created by `_prometheus_acg_web_config_secret`
but will not be injected via the operator path. Prometheus runs without basic auth on the
ACG dev cluster (acceptable for ephemeral sandbox environments).

### Change 1 — `scripts/etc/helm/observability/kube-prometheus-stack-acg-values.yaml`

**Exact old block:**
```yaml
    secrets:
      - prometheus-web-config
    additionalArgs:
      - name: web.config.file
        value: /etc/prometheus/secrets/prometheus-web-config/web.yml
    additionalScrapeConfigs:
```

**Exact new block:**
```yaml
    additionalScrapeConfigs:
```
