# Bug: azure.sh sources azure-vars.sh at plugin load time — az ad app create runs on every make up

**Date:** 2026-06-12
**Branch:** k3d-manager-v1.6.5
**File:** `scripts/plugins/azure.sh`

---

## Symptom

Every `make up` with `CLUSTER_PROVIDER=k3s-azure` silently calls `az ad app create` during
plugin load, creating a new Azure AD app in the background on every run.

---

## Root Cause

`scripts/plugins/azure.sh` sources `scripts/etc/azure/azure-vars.sh` at the top level
(lines 4–7), which runs every time the plugin is loaded:

```bash
az_eso_vars="${SCRIPT_DIR}/etc/azure/azure-vars.sh"
if [[ -r "${az_eso_vars}" ]]; then
   source "${az_eso_vars}"
fi
```

`azure-vars.sh` contains:

```bash
ESO_APP_ID="$(az ad app create --display-name azure-eso --query appId -o tsv)"
```

This line has no guard — it fires unconditionally every time the file is sourced.

`azure-vars.sh` is **already sourced inside `_create_azure_eso_store`** (line 93–98 of
`azure.sh`), which is the only function that actually needs those vars. The top-level
source is redundant and harmful.

---

## Fix

### Change 1 — `scripts/plugins/azure.sh`: remove top-level source of azure-vars.sh

**Exact old block (lines 3–7):**

```bash
# shellcheck disable=SC1090
az_eso_vars="${SCRIPT_DIR}/etc/azure/azure-vars.sh"
if [[ -r "${az_eso_vars}" ]]; then
   source "${az_eso_vars}"
fi
```

**Exact new block:**

```bash
# shellcheck disable=SC1090
```

(Keep the `shellcheck disable` comment — it covers the `source` call inside
`_create_azure_eso_store` at line 98.)
