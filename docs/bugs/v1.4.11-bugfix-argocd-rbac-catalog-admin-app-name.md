## Problem

The `catalog-admin` role in `argocd-rbac-cm.yaml` references `shopping-cart/product-catalog`,
but the canonical ArgoCD application managed by the `shopping-cart-apps` AppofApps is named
`shopping-cart-product-catalog` (with the `shopping-cart-` prefix). Users in the
`catalog-admins` LDAP group cannot sync the canonical app via the `catalog-admin` role because
the app name in the RBAC policy does not match the actual application name.

**Root cause:** The `catalog-admin` role was written with the short app name `product-catalog`,
but the shopping-cart-apps AppofApps creates apps with the `shopping-cart-` prefix. The RBAC
check `shopping-cart/product-catalog` never matches `shopping-cart/shopping-cart-product-catalog`.

---

## Fix

**File:** `argocd/config/argocd-rbac-cm.yaml`

**Exact old block:**

```yaml
    # Product Catalog Admins - Full access to product-catalog app
    p, role:catalog-admin, applications, *, shopping-cart/product-catalog, allow
    p, role:catalog-admin, logs, get, shopping-cart/product-catalog, allow
    p, role:catalog-admin, exec, create, shopping-cart/product-catalog, allow
    g, catalog-admins, role:catalog-admin
```

**Exact new block:**

```yaml
    # Product Catalog Admins - Full access to shopping-cart-product-catalog app
    p, role:catalog-admin, applications, *, shopping-cart/shopping-cart-product-catalog, allow
    p, role:catalog-admin, logs, get, shopping-cart/shopping-cart-product-catalog, allow
    p, role:catalog-admin, exec, create, shopping-cart/shopping-cart-product-catalog, allow
    g, catalog-admins, role:catalog-admin
```
