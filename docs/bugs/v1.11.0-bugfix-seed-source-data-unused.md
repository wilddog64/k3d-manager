## Problem

P3 added `_seed_source_data()` to `shopping_cart.sh` — the canonical-source reader backing
`SEED_VAULT_SOURCE_ADDR` / `SEED_VAULT_SOURCE_TOKEN` — but no call site uses it. The
generate-if-absent branches for `redis/cart`, `redis/orders-cache`, `rabbitmq/default`, and the
`postgres/*` keys always `openssl rand` a fresh password when the key is missing in the **target**
Vault, never consulting the canonical **source**. So `SEED_VAULT_SOURCE_*` is effectively dead config
in this file, and a target seeded from a configured source can drift from that source.

Separately, the four KV helpers (`_vault_kv_put`, `_vault_kv_exists`, `_vault_kv_get_field`,
`_seed_source_data`) pass the Vault token via `-H "X-Vault-Token: ${...}"`, which is visible in
`ps`/argv — the same class the PR #101 process note flagged
(`docs/issues/2026-06-28-copilot-pr101-review-findings.md`). Since these helpers are being edited
here, harden them with the proven `curl --config` (mode-600 header file) idiom already used in
`scripts/plugins/vault.sh:vault_seed_hub_into_context` (commit `b2ed5b80`, lines 1039–1055).

**Root cause:** `_seed_source_data` was added as a helper but the seed branches were never rewired to
call it before falling back to generation; and the helpers were written with `-H "X-Vault-Token: …"`
before the `curl --config` idiom was adopted in vault.sh.

**Severity:** low / latent. Not a regression — the active canonical-copy path for the in-cluster
cutover is `vault.sh:vault_seed_hub_into_context` (which already reads from the source Vault), and in
the default configuration `SEED_VAULT_SOURCE_*` resolves to the same Vault as the target, so a
fresh-target seed yields the same result. The gap only manifests when `SEED_VAULT_SOURCE_*` points at
a genuinely different source Vault.

Flagged by Copilot on PR #101 (findings 1–3); see
`docs/issues/2026-06-28-copilot-pr101-review-findings.md`.

## Reproduction

```bash
# Point the seed source at a different Vault that already holds redis/cart:
export SEED_VAULT_SOURCE_ADDR="http://source-vault:8200"
export SEED_VAULT_SOURCE_TOKEN="<source-token>"   # via env/stdin, never argv
# Seed a fresh target Vault (no redis/cart yet), then read it back:
#   actual:   target redis/cart has a NEW random password (source ignored)
#   expected: target redis/cart matches the source Vault's value
```

```bash
  local _seed_addr="${SEED_VAULT_ADDR:-http://localhost:${_vault_local_port}}"
  local _seed_token="${SEED_VAULT_TOKEN:-${_vault_root_token}}"
  local _src_addr="${SEED_VAULT_SOURCE_ADDR:-${_seed_addr}}"

      -H "X-Vault-Token: ${_seed_token}" \

  _vault_kv_get_field() {

    local field="$2"
    curl -sf \
      -H "X-Vault-Token: ${_seed_token}" \
      "${_seed_addr}/v1/secret/data/${path}" \
      | jq -r --arg field "$field" '.data.data[$field] // empty'

  # Canonical source reader (Vault = source of truth). Returns the raw KV-v2 data object as JSON,
  # empty string if the key is absent in the source Vault.

    local path="$1"
    curl -sf \
      -H "X-Vault-Token: ${_src_token}" \
      "${_src_addr}/v1/secret/data/${path}" \
      | jq -c '.data.data // empty' 2>/dev/null || true
  }
```

**Exact new block:**

```bash
  local _seed_addr="${SEED_VAULT_ADDR:-http://localhost:${_vault_local_port}}"
  local _seed_token="${SEED_VAULT_TOKEN:-${_vault_root_token}}"
  local _src_addr="${SEED_VAULT_SOURCE_ADDR:-${_seed_addr}}"
  local _src_token="${SEED_VAULT_SOURCE_TOKEN:-${_seed_token}}"
  # Token headers via curl --config files (mode 600) so the Vault tokens never appear in argv/ps.
  local _seed_hdr _src_hdr _src_json
  _seed_hdr=$(mktemp) || { _err "[acg-up] could not create temp header file"; return 1; }
  _src_hdr=$(mktemp) || { rm -f "${_seed_hdr}" 2>/dev/null || true; _err "[acg-up] could not create temp header file"; return 1; }
  chmod 600 "${_seed_hdr}" "${_src_hdr}"
  printf 'header = "X-Vault-Token: %s"\n' "${_seed_token}" > "${_seed_hdr}"
  printf 'header = "X-Vault-Token: %s"\n' "${_src_token}"  > "${_src_hdr}"
  trap 'rm -f "'"${_seed_hdr}"'" "'"${_src_hdr}"'" 2>/dev/null || true' RETURN
  _vault_kv_put() {
    curl -sf -X POST \
      --config "${_seed_hdr}" \
      -H "Content-Type: application/json" \
      -d "{\"data\":$1}" \
      "${_seed_addr}/v1/secret/data/$2" >/dev/null
  }
  _vault_kv_exists() {
    local path="$1"
    curl -sf \
      --config "${_seed_hdr}" \
      "${_seed_addr}/v1/secret/data/${path}" >/dev/null 2>&1
  }
  _vault_kv_get_field() {
    local path="$1"
    local field="$2"
    curl -sf \
      --config "${_seed_hdr}" \
      "${_seed_addr}/v1/secret/data/${path}" \
      | jq -r --arg field "$field" '.data.data[$field] // empty'
  }
  # Canonical source reader (Vault = source of truth). Returns the raw KV-v2 data object as JSON,
  # empty string if the key is absent in the source Vault.
  _seed_source_data() {
    local path="$1"
    curl -sf \
      --config "${_src_hdr}" \
      "${_src_addr}/v1/secret/data/${path}" \
      | jq -c '.data.data // empty' 2>/dev/null || true
  }
```

**Exact new block:**

```bash
  if _vault_kv_exists "redis/cart"; then
    _info "[acg-up] Reusing existing Vault secret redis/cart"
    _redis_pass_cart=$(_vault_kv_get_field "redis/cart" "password")
  else
    _src_json=$(_seed_source_data "redis/cart")
    if [[ -n "${_src_json}" ]]; then
      _info "[acg-up] Copying redis/cart from canonical source Vault"
      _vault_kv_put "${_src_json}" redis/cart
      _redis_pass_cart=$(printf '%s' "${_src_json}" | jq -r '.password // empty')
    else
      _redis_pass_cart=$(openssl rand -base64 24 | tr -d '=+/')

    fi
  fi
```
