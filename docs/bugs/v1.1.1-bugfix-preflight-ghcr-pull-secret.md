# Bugfix: v1.1.1 — preflight vcluster has no ghcr-pull-secret (shopping-cart ImagePullBackOff)

**Branch:** `k3d-manager-v1.7.1`
**Files:** `scripts/plugins/shopping_cart.sh`, `bin/cluster-preflight`

---

## Problem

In the preflight vcluster, the shopping-cart services (`basket`, `frontend`, `order`, …) stay
`ImagePullBackOff` because their private GHCR images cannot be pulled — the throwaway cluster's
`shopping-cart-apps` namespace has **no `ghcr-pull-secret`** and the `default` ServiceAccount is
not patched with `imagePullSecrets`. The real `ubuntu-hostinger` cluster gets this via
`shopping_cart_create_ghcr_pull_secret` (acg-up Step 5/12), but the preflight never runs it
against the vcluster.

**Root cause:** the preflight provisions RBAC + a token inside the vcluster but never creates the
GHCR pull secret (or patches the SA) there.

Confirmed live on `pf1` (2026-06-16):

```bash
kubectl --context ubuntu-hostinger -n vclusters get pods | grep ImagePullBackOff
#   basket-service-...-x-shopping-cart-apps-x-pf1   ImagePullBackOff
# inside the vcluster:
#   shopping-cart-apps has NO ghcr-pull-secret
```

The existing `shopping_cart_create_ghcr_pull_secret` already does exactly what is needed —
creates the `shopping-cart-{apps,payment,data}` namespaces, creates `ghcr-pull-secret` in each,
and patches the `default` SA — and it honors `APP_CONTEXT`. It just needs to run against the
vcluster, paired with `shopping_cart_resolve_ghcr_pat` (which sets the shared `_ghcr_pat` /
`_github_user` shell state) in the same process. (Bonus: this also creates the
`shopping-cart-data` namespace that was otherwise missing in the vcluster.)

---

## Fix

### Change 1 — `scripts/plugins/shopping_cart.sh`: add a resolve+create wrapper

Add a public wrapper that resolves the PAT and creates the pull secret in one process (the two
existing functions share `_ghcr_pat` / `_github_user` shell state, so they must run together).
Insert it immediately after the closing `}` of `shopping_cart_create_ghcr_pull_secret`
(currently line 359), before `function shopping_cart_create_vault_bridge()`.

**Exact old block (lines 358–361):**

```bash
  done
}

function shopping_cart_create_vault_bridge() {
```

**Exact new block:**

```bash
  done
}

function shopping_cart_provision_ghcr_pull_secret() {
  shopping_cart_resolve_ghcr_pat
  shopping_cart_create_ghcr_pull_secret
}

function shopping_cart_create_vault_bridge() {
```

### Change 2 — `bin/cluster-preflight`: provision the pull secret in the vcluster

After the argocd-manager token is minted (the vcluster kubeconfig is ready), provision the GHCR
pull secret in the vcluster's app namespaces, targeting the vcluster via its own kubeconfig +
current-context.

**Exact old block (lines 564–569):**

```bash
  _info "[preflight] Minting argocd-manager token inside ${PREFLIGHT_NAME}..."
  VC_TOKEN="$(KUBECONFIG="${VC_KCFG}" _kubectl create token argocd-manager -n kube-system --duration=8760h)"
  if [[ -z "${VC_TOKEN}" ]]; then
    _err "[preflight] Failed to mint argocd-manager token inside ${PREFLIGHT_NAME}"
  fi

```

**Exact new block:**

```bash
  _info "[preflight] Minting argocd-manager token inside ${PREFLIGHT_NAME}..."
  VC_TOKEN="$(KUBECONFIG="${VC_KCFG}" _kubectl create token argocd-manager -n kube-system --duration=8760h)"
  if [[ -z "${VC_TOKEN}" ]]; then
    _err "[preflight] Failed to mint argocd-manager token inside ${PREFLIGHT_NAME}"
  fi

  _info "[preflight] Provisioning ghcr-pull-secret in ${PREFLIGHT_NAME} app namespaces..."
  VC_CONTEXT="$(KUBECONFIG="${VC_KCFG}" _kubectl config current-context 2>/dev/null || true)"
  KUBECONFIG="${VC_KCFG}" APP_CONTEXT="${VC_CONTEXT}" \
    _run_command -- "${REPO_ROOT}/scripts/k3d-manager" shopping_cart_provision_ghcr_pull_secret

```
