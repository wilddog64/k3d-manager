# Bugfix: v1.6.3 — post-provision smoke test skipped + /acg-status inconsistency

**Branch:** `k3d-manager-v1.6.3`
**Files:** `bin/k3dm-webhook`

---

## Problem

### Part A — post-provision smoke test skipped after acg-up

`_run_post_provision_check` checks the SSH tunnel first. If `tunnel_ok` is False it
returns early (lines 1304–1307) before reaching `_smoke_test_services()`. When the tunnel
LaunchAgent hasn't reconnected yet at the 90s mark, the smoke test is silently skipped and
the user sees no post-provision service health report.

### Part B — /acg-status shows conflicting hub/tunnel ❌ and smoke test ✅

`_run_cluster_status` reports Hub cluster ❌ and SSH tunnel ❌ (kubectl/port checks), then
immediately reports Service smoke test ✅ (HTTP checks via Cloudflare tunnel /
port-forwards). Both readings are factually correct — kubectl is unreachable but services
serve traffic through independent paths — but the report gives no explanation, making it
appear broken and contradictory.

---

## Fix

### Change 1 — `_run_post_provision_check`: always run smoke test, even when tunnel is down

Remove the early `return` that skips the smoke test when tunnel is down. Instead, warn and
continue so the smoke results are always posted.

**Exact old block (lines 1304–1307):**

```python
    if not tunnel_ok:
        lines.append("⚠️ Tunnel is down — ACG cluster unreachable; skipping app health check.")
        _notify_job(job_id, f"🏥 *Post-provision check* ({provider})\n" + "\n".join(lines))
        return
```

**Exact new block:**

```python
    if not tunnel_ok:
        lines.append("⚠️ SSH tunnel down — ACG cluster API unreachable. Service endpoints may still be accessible via port-forwards.")
```

---

### Change 2 — `_run_cluster_status`: add reconciliation note when infra is down but smoke passes

After the smoke test block (after line 1600 where smoke results are logged), add a
reconciliation note when infrastructure checks failed but smoke test passed.

**Exact old block (lines 1596–1600):**

```python
        all_smoke_ok = all(ok for _, ok, _ in smoke)
        _log(f"\n*Service smoke test:*{'  all clear ✅' if all_smoke_ok else ''}")
        for svc_name, ok, detail in smoke:
            icon = "✅" if ok else "❌"
            _log(f"  {icon} {svc_name}: {detail}")
```

**Exact new block:**

```python
        all_smoke_ok = all(ok for _, ok, _ in smoke)
        _log(f"\n*Service smoke test:*{'  all clear ✅' if all_smoke_ok else ''}")
        for svc_name, ok, detail in smoke:
            icon = "✅" if ok else "❌"
            _log(f"  {icon} {svc_name}: {detail}")
        if all_smoke_ok and not tunnel_ok:
            _log("\nℹ️ _Hub/tunnel checks DOWN but service endpoints responding — port-forwards or Cloudflare tunnel still active. Run `/acg-refresh` to restore kubectl access._")
```

Note: `tunnel_ok` is already defined earlier in `_run_cluster_status` at the SSH tunnel
check block (the variable is in scope).
