## Remaining Problems

### Problem 1 — No slash command aliases in webhook

Users must type `ask gemini <q>` or `ask claude <q>` in a Slack thread.
Direct `/gemini <q>`, `/claude <q>`, `/codex <q>` commands fall through to the "Unknown
command" handler. ChatOps convention expects `/bot-name <question>`.

### Problem 2 — Claude agent refuses diagnostic tools

The Claude system prompt says "Read-only commands only" without listing what IS allowed.
Claude over-refuses `sed`, `grep`, `awk`, `curl` GET, and network tools because the
prompt wording implies they might be disallowed. Result: agents ask users to run commands
manually instead of investigating autonomously.

### Problem 3 — Gemini system prompt incorrectly says it cannot run kubectl

Gemini's `ask` system prompt says "Answer from your knowledge — you cannot run kubectl in
this context." This is misleading: Gemini doesn't run kubectl (it has no tool loop), but
it should still describe what it CAN do and frame its answer as analysis rather than
claiming a technical inability.

### Problem 4 — `k3dm-ask-bash` scope blocks diagnostic read paths

The absolute-path scope check in `k3dm-ask-bash` only allows
`${K3DM_REPO}`, `${SHOPPING_CARTS_ROOT}`, `/tmp`, `/var/tmp`, `/proc`, `/sys`.
Agents cannot read `/etc/hosts`, `~/.kube/config`, LaunchDaemon plists, or ACG state
logs — all routinely needed when diagnosing cluster and tunnel issues.

### Change 5 — `bin/k3dm-ask-bash`: expand scope allowlist to include diagnostic read paths

**Exact old block (the `for arg in "$@"` loop at the bottom of the file):**

```bash
# Block file access outside allowed repos (absolute path arguments only)
for arg in "$@"; do
  if [[ "$arg" == /* ]]; then
    real_arg="$(realpath -m "$arg" 2>/dev/null || echo "$arg")"
    if [[ "$real_arg" != "${K3DM_REPO}"* && "$real_arg" != "${SHOPPING_CARTS_ROOT}"* \
       && "$real_arg" != /tmp/* && "$real_arg" != /var/tmp/* \
       && "$real_arg" != /proc/* && "$real_arg" != /sys/* ]]; then
      _deny_scope "$arg"
    fi
  fi
done
```

**Exact new block:**

```bash
# Read-only system paths allowed for cluster diagnosis
_DIAG_PATHS=(
  /etc
  /Library/LaunchDaemons
  /Library/LaunchAgents
  "${HOME}/.kube"
  "${HOME}/.local/share/k3d-manager"
  "${HOME}/.cloudflared"
  /var/log
  /usr/local/bin
  /usr/bin
  /bin
  /opt/homebrew
)

# Block file access outside allowed repos and diagnosis paths (absolute path arguments only)
for arg in "$@"; do
  if [[ "$arg" == /* ]]; then
    real_arg="$(realpath -m "$arg" 2>/dev/null || echo "$arg")"
    _allowed=0
    for _p in "${K3DM_REPO}" "${SHOPPING_CARTS_ROOT}" /tmp /var/tmp /proc /sys "${_DIAG_PATHS[@]}"; do
      if [[ "$real_arg" == "${_p}"* ]]; then
        _allowed=1
        break
      fi
    done
    if [[ "$_allowed" -eq 0 ]]; then
      _deny_scope "$arg"
    fi
  fi
done
```
