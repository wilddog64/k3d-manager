# Bugfix: v1.1.1 — `make help` omits the `preflight` target

**Branch:** `k3d-manager-v1.7.1`
**Files:** `Makefile`

---

## Problem

`make help` does not list the `preflight` target, so there is no discoverable explanation of
`make preflight`. The target exists (`Makefile:53`) and carries a `##` doc comment
(`Makefile:52`), but the `help` target is a hand-maintained block of `@echo` lines
(`Makefile:429-470`) that does not parse `##` comments — `preflight` was simply never added
to it.

**Root cause:** the `help` recipe enumerates targets via literal `@echo` lines; the
`preflight` line was never added when the target was introduced.

---

## Reproduction

```bash
make help | grep preflight    # → no output (target is undocumented)
```

Expected: a `make preflight` line appears in the help output.

---

## Fix

### Change 1 — `Makefile`: add a `preflight` line to the `help` recipe

Insert one `@echo` line immediately after the `make test` line in the main Targets block.
Column alignment matches the existing block (description starts 19 chars after `make`).

**Exact old block (lines 437–438):**

```makefile
	@echo "    make test          Run all BATS test suites"
	@echo ""
```

**Exact new block:**

```makefile
	@echo "    make test          Run all BATS test suites"
	@echo "    make preflight     Spin up a throwaway vCluster + deploy the full stack via ArgoCD (NAME=<name> MODE=--auto|--keep, default --auto)"
	@echo ""
```

> Note: the leading whitespace on each `@echo` line is a literal TAB (Makefile recipe
> requirement), not spaces. Preserve the existing TAB indentation.
