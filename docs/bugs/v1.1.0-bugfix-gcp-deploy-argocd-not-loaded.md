## Problem

`make provision CLUSTER_PROVIDER=k3s-gcp` fails at step 6/7:

```
INFO: [gcp] Step 6/7 — Deploying ArgoCD...
/Users/.../scripts/plugins/gcp.sh: line 280: deploy_argocd: command not found
make: *** [provision] Error 1
```

Two bugs on the same line (`gcp.sh:280`):

**Bug 1 — `argocd.sh` never sourced:**
`gcp_provision_stack` is loaded by the dispatcher's `_load_plugin_function` which only
sources `gcp.sh`. Unlike `vault.sh` (which explicitly sources `eso.sh` on load),
nothing sources `argocd.sh`, so `deploy_argocd` is not defined when called.

**Bug 2 — `--skip-ldap` is not a valid flag:**
`deploy_argocd` accepts `--enable-ldap` (opt-in, default off) but has no `--skip-ldap`
flag. The unknown flag hits the `*) _err "[argocd] Unknown option: $1"` branch and
returns 1. LDAP is already disabled by default — the flag is simply wrong.

---

## Reproduction

```bash
make provision CLUSTER_PROVIDER=k3s-gcp GHCR_PAT=<pat>
# → steps 1-5 pass
# → step 6: deploy_argocd: command not found
```

---

## Fix

### Change 1 — `scripts/plugins/gcp.sh` lines 279–280: source argocd.sh and fix flags

**Exact old block:**

```bash
  _info "[gcp] Step 6/7 — Deploying ArgoCD..."
  deploy_argocd --bootstrap --skip-ldap --skip-istio || return 1
```

**Exact new block:**

```bash
  _info "[gcp] Step 6/7 — Deploying ArgoCD..."
  # shellcheck source=/dev/null
  source "${SCRIPT_DIR}/plugins/argocd.sh"
  deploy_argocd --bootstrap --skip-istio || return 1
```

**Why `source` here (not at top of file):** `argocd.sh` is only needed by
`gcp_provision_stack`. Sourcing inside the function is consistent with the lazy-load
pattern used elsewhere in the codebase.

**Why remove `--skip-ldap`:** `deploy_argocd` has no such flag. LDAP is opt-in
(`--enable-ldap`); omitting it is the correct way to skip LDAP.
