# Bugfix: v1.4.11 — order-service NPE in ExceptionTranslationFilter on actuator health probe

**Branch (spec repo):** `k3d-manager-v1.4.11`
**Branch (work repo):** `fix/order-actuator-security-npe` (create from `main` in shopping-cart-order)
**Files:** `src/main/java/com/shoppingcart/order/config/SecurityConfig.java`

---

## Problem

`order-service` crashes on every Kubernetes liveness/readiness probe. The probe hits
`/actuator/health`, which triggers a NullPointerException inside Spring Security's
`ExceptionTranslationFilter`, causing the JVM to exit and the pod to restart in a loop
(29+ restarts observed).

**Root cause:** Spring Boot 3.2.0 ships spring-security 6.2.0, which has a bug in
`HandlerMappingIntrospector` cache initialization when multiple `SecurityFilterChain`
beans exist. When an actuator endpoint is first accessed, the introspector logs a cache
miss and the exception handling path in `ExceptionTranslationFilter.doFilter` (line 126)
NPEs on a null `AtomicInteger` from `Map.get()`.

The `permitAll()` rules in both `SecurityConfig` and `OAuth2SecurityConfig` are correct
in intent but do not bypass the introspector path that triggers the bug. The fix is a
dedicated `@Order(0)` filter chain that matches `/actuator/**` and short-circuits before
the problematic chains run.

---

## Reproduction

Deploy `order-service` to Kubernetes with default liveness/readiness probes. Within the
probe interval (10s readiness, 30s liveness), the pod logs:

```
WARN HandlerMappingIntrospector : Cache miss for REQUEST dispatch to '/actuator/health' (previous null)
NullPointerException: Cannot invoke "AtomicInteger.get()" because Map.get(Object) is null
  at ExceptionTranslationFilter.doFilter(ExceptionTranslationFilter.java:126)
```

Pod enters CrashLoopBackOff.

## Fix

### Change 1 — `SecurityConfig.java`: add dedicated `@Order(0)` actuator filter chain

**Add these imports (after the existing `@Order` import if present, or add fresh):**

```java
import org.springframework.core.annotation.Order;
```

**Exact old block (lines 27–72):**

```java
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            // Disable CSRF for stateless REST API
            .csrf(AbstractHttpConfigurer::disable)

            // Stateless session management
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))

            // Security headers for XSS and other attack prevention
            .headers(headers -> headers
                // Prevent XSS attacks
                .xssProtection(xss -> xss

                // Referrer policy

                // HSTS (HTTP Strict Transport Security)

                    .includeSubDomains(true)
```

**Exact new block:**

```java
    @Bean
    @Order(0)
    public SecurityFilterChain actuatorFilterChain(HttpSecurity http) throws Exception {
        http
            .securityMatcher("/actuator/**")
            .csrf(AbstractHttpConfigurer::disable)
            .authorizeHttpRequests(auth -> auth.anyRequest().permitAll());
        return http.build();
    }

    @Bean
    @Order(1)
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            // Disable CSRF for stateless REST API
            .csrf(AbstractHttpConfigurer::disable)

            // Stateless session management
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))

            // Security headers for XSS and other attack prevention
            .headers(headers -> headers
                // Prevent XSS attacks
                .xssProtection(xss -> xss
                    .headerValue(XXssProtectionHeaderWriter.HeaderValue.ENABLED_MODE_BLOCK))

                // Prevent clickjacking
                .frameOptions(frame -> frame.deny())

                // Prevent MIME type sniffing
                .contentTypeOptions(contentType -> {})

                // Content Security Policy - prevent inline scripts
                .contentSecurityPolicy(csp -> csp
                    .policyDirectives("default-src 'self'; script-src 'self'; style-src 'self'; img-src 'self' data:; font-src 'self'; frame-ancestors 'none'; form-action 'self'"))

                // Referrer policy
                .referrerPolicy(referrer -> referrer
                    .policy(ReferrerPolicyHeaderWriter.ReferrerPolicy.STRICT_ORIGIN_WHEN_CROSS_ORIGIN))

                // HSTS (HTTP Strict Transport Security)
                .httpStrictTransportSecurity(hsts -> hsts
                    .includeSubDomains(true)
                    .maxAgeInSeconds(31536000))

                // Permissions policy (Feature policy)
                .permissionsPolicy(permissions -> permissions
                    .policy("geolocation=(), microphone=(), camera=()")))

            // Allow all requests (authentication handled at API gateway level)
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/**").permitAll()
                .anyRequest().denyAll());

        return http.build();
    }
```

Note: the `/actuator/**` matchers are removed from `securityFilterChain` because
`actuatorFilterChain` at `@Order(0)` now owns those paths.

Also update the `OAuth2SecurityConfig` to use `@Order(2)` instead of `@Order(1)` so
