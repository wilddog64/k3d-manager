# Bugfix: v1.1.1 — preflight vcluster isolated from production appsets via `environment` label

**Branch:** `k3d-manager-v1.7.1`
**Files:** `bin/cluster-preflight`

---

## Problem

A default-mode preflight run (`bin/cluster-preflight <name>`) creates + registers the
throwaway vcluster, applies the prefixed-clone ArgoCD resources, then times out the 300s
convergence wait (`rc=1`). `--auto` then keeps the cluster as a "failure" — but the flag
logic is correct; the **deploy never converges**, so delete-on-success can never be proven.

**Root cause:** the preflight registers the vcluster's ArgoCD cluster secret with
`environment: dev` (the `register_app_cluster` default). The **live production
`platform-helm` ApplicationSet** selects target clusters by `environment In [dev,infra,prod]`
**only** (no cluster-name guard), so production grabs the throwaway cluster and deploys
`vok-platform` (project `platform`, the live project) onto it. That live `platform` project
has zero destination entries for the vcluster server, so ArgoCD rejects it with
`InvalidSpecError`, the production app occupies the platform app-of-apps slot, and the
platform prerequisites (namespaces, ESO/CRDs) never install → cascade failures in the
preflight clones (`data-layer` SyncError, `shopping-cart-data` namespace never created).

Confirmed live on the `vok` repro (hub `k3d-k3d-cluster`, ns `cicd`):
- `vok-platform` `ownerReferences: [{kind: ApplicationSet, name: platform-helm}]` (the LIVE appset)
- `cluster-vok` secret labels: `{argocd.argoproj.io/secret-type: cluster, environment: dev, ...}`
- live `platform-helm` selector: `matchExpressions: [{key: environment, operator: In, values: [dev, infra, prod]}]`
- `vok-platform` condition: `... do not match any of the allowed destinations in project 'platform'`

Full diagnosis: `docs/issues/2026-06-16-vcluster-preflight-production-appset-leak.md`.

**Only `platform-helm` leaks.** Verified the other two production appsets do NOT:
`services-git` uses a git-directory generator with a hardcoded `destination.name: ubuntu-hostinger`;
`observability-acg` uses a list generator with a hardcoded `destination.name: ubuntu-hostinger`.
Neither templates `destination.server` from a cluster label, so neither selects the vcluster.
`platform-helm` is the only appset with a cluster generator templating `destination.server: {{.server}}`.

---

## Reproduction

```bash
bin/cluster-preflight vok          # default --auto, clean vclusters ns
# vcluster up, registered, clones applied, 300s wait times out → rc=1 → kept
kubectl --context k3d-k3d-cluster -n cicd get application vok-platform \
  -o jsonpath='{.metadata.ownerReferences}'   # → ApplicationSet/platform-helm (LIVE — the leak)
# Expected after fix: no vok-platform owned by the live platform-helm; the clone
# vok-preflight-platform generates instead and the stack converges.
```

---

## Fix

Register the preflight vcluster with a non-production `environment` value (`preflight`) that
no live appset selects, and make the platform-helm **clone** select that same value so the
clone still targets the throwaway cluster. `register_app_cluster` already honors
`ARGOCD_APP_CLUSTER_ENVIRONMENT` (`scripts/plugins/argocd.sh:1148`), so no subtree edit is
needed — the change is entirely in `bin/cluster-preflight`.

### Change 1 — `bin/cluster-preflight`: add the preflight environment constant

Add a new line immediately after the `APP_CLUSTER_SERVER=""` config line.

**Exact old block (lines 55–58):**

```bash
APP_CLUSTER_NAME="${PREFLIGHT_NAME}"
APP_CLUSTER_SECRET_NAME="cluster-${PREFLIGHT_NAME}"
APP_CLUSTER_SERVER=""
VC_KCFG="${VCLUSTER_KUBECONFIG_DIR}/${PREFLIGHT_NAME}.yaml"
```

**Exact new block:**

```bash
APP_CLUSTER_NAME="${PREFLIGHT_NAME}"
APP_CLUSTER_SECRET_NAME="cluster-${PREFLIGHT_NAME}"
APP_CLUSTER_SERVER=""
APP_CLUSTER_ENVIRONMENT="${APP_CLUSTER_ENVIRONMENT:-preflight}"
VC_KCFG="${VCLUSTER_KUBECONFIG_DIR}/${PREFLIGHT_NAME}.yaml"
```

### Change 2 — `bin/cluster-preflight`: pass the environment to `register_app_cluster`

Add the `ARGOCD_APP_CLUSTER_ENVIRONMENT` env var to the inline block that invokes
`register_app_cluster`.

**Exact old block (lines 578–582):**

```bash
  ARGOCD_APP_CLUSTER_NAME="${APP_CLUSTER_NAME}" \
  ARGOCD_APP_CLUSTER_SECRET_NAME="${APP_CLUSTER_SECRET_NAME}" \
  ARGOCD_APP_CLUSTER_SERVER="${APP_CLUSTER_SERVER}" \
  ARGOCD_APP_CLUSTER_TOKEN="${VC_TOKEN}" \
    _run_command -- "${REPO_ROOT}/scripts/k3d-manager" register_app_cluster
```

**Exact new block:**

```bash
  ARGOCD_APP_CLUSTER_NAME="${APP_CLUSTER_NAME}" \
  ARGOCD_APP_CLUSTER_SECRET_NAME="${APP_CLUSTER_SECRET_NAME}" \
  ARGOCD_APP_CLUSTER_SERVER="${APP_CLUSTER_SERVER}" \
  ARGOCD_APP_CLUSTER_ENVIRONMENT="${APP_CLUSTER_ENVIRONMENT}" \
  ARGOCD_APP_CLUSTER_TOKEN="${VC_TOKEN}" \
    _run_command -- "${REPO_ROOT}/scripts/k3d-manager" register_app_cluster
```

### Change 3 — `bin/cluster-preflight`: clone platform-helm selects `environment: preflight`

In `_cluster_preflight_render_platform_appset`, the rendered clone inherits the template's
`environment In [dev, infra, prod]` selector. After Change 1/2 the cluster secret carries
`environment: preflight`, so that inherited selector would no longer match and the clone
would generate nothing. Rewrite the inherited values to the preflight environment so the
clone matches the throwaway cluster (alongside the cluster-name guard the `awk` adds).

The matched line is the cluster-generator selector values from
`scripts/etc/argocd/applicationsets/platform-helm.yaml` (14 leading spaces):
`              values: [dev, infra, prod]`.

**Exact old block (lines 236–240):**

```bash
  sed \
    -e "s/^  name: platform-helm$/  name: ${cluster_name}-preflight-platform-helm/" \
    -e "s/^      name: '{{.name}}-platform'$/      name: '${cluster_name}-preflight-platform'/" \
    -e "s/^      project: platform$/      project: ${cluster_name}-preflight-platform/" \
    -e "s/^        namespace: cicd$/        namespace: cicd/" \
```

**Exact new block:**

```bash
  sed \
    -e "s/^  name: platform-helm$/  name: ${cluster_name}-preflight-platform-helm/" \
    -e "s/^      name: '{{.name}}-platform'$/      name: '${cluster_name}-preflight-platform'/" \
    -e "s/^      project: platform$/      project: ${cluster_name}-preflight-platform/" \
    -e "s/^              values: \[dev, infra, prod\]$/              values: [${APP_CLUSTER_ENVIRONMENT}]/" \
    -e "s/^        namespace: cicd$/        namespace: cicd/" \
```
