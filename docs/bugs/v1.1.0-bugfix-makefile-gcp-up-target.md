# Bugfix: v1.1.0 — Makefile `make up` inconsistency for k3s-gcp

**Branch:** `k3d-manager-v1.1.0`
**Files:** `Makefile`

---

## Problem

`make up CLUSTER_PROVIDER=k3s-gcp` only provisions the cluster (VM + k3s) but does not
deploy the plugin stack. The user must then run `make provision CLUSTER_PROVIDER=k3s-gcp`
as a second command. By contrast, `make up` for AWS runs the full stack end-to-end via
`bin/acg-up`. This forces users to remember a different workflow per provider.

Additionally, `make provision` chains commands with `;` instead of `&&`, so if `make up`
fails the plugin stack still runs against a non-existent cluster.

**Root cause:** When the k3s-gcp provider was added, `make up` was wired to `deploy_cluster`
(cluster only) and `make provision` was introduced as a separate full-stack target, instead
of folding both steps into `make up` behind `&&`.

## Reproduction

```bash
# Current broken flow — two commands required for GCP, one for AWS
make up CLUSTER_PROVIDER=k3s-gcp          # cluster only
make provision CLUSTER_PROVIDER=k3s-gcp   # plugins — runs even if make up failed
```

Expected: one command `make up CLUSTER_PROVIDER=k3s-gcp` provisions cluster + full stack,
same as `make up` (default, AWS).

## Fix

### Change 1 — `Makefile` `up` target: add explicit `k3s-gcp` case with `&&` chaining

**Exact old block (lines 13–19):**

```makefile
up:
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-aws) echo "[make] CLUSTER_PROVIDER=k3s-aws — running bin/acg-up..."; \
	           GHCR_PAT="$(GHCR_PAT)" bin/acg-up "$(URL)" ;; \
	  *)       echo "[make] CLUSTER_PROVIDER=$(CLUSTER_PROVIDER) — running deploy_cluster..."; \
	           CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" _GCP_SANDBOX_URL="$(URL)" scripts/k3d-manager deploy_cluster --confirm ;; \
	esac
```

**Exact new block:**

```makefile
up:
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-aws) echo "[make] CLUSTER_PROVIDER=k3s-aws — running bin/acg-up..."; \
	           GHCR_PAT="$(GHCR_PAT)" bin/acg-up "$(URL)" ;; \
	  k3s-gcp) echo "[make] CLUSTER_PROVIDER=k3s-gcp — provisioning cluster..."; \
	           CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" _GCP_SANDBOX_URL="$(URL)" scripts/k3d-manager deploy_cluster --confirm && \
	           echo "[make] CLUSTER_PROVIDER=k3s-gcp — deploying plugin stack..."; \
	           CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" GHCR_PAT="$(GHCR_PAT)" scripts/k3d-manager gcp_provision_stack ;; \
	  *)       echo "[make] CLUSTER_PROVIDER=$(CLUSTER_PROVIDER) — running deploy_cluster..."; \
	           CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" _GCP_SANDBOX_URL="$(URL)" scripts/k3d-manager deploy_cluster --confirm ;; \
	esac
```

### Change 2 — `Makefile` `sync-apps` target: add `k3s-gcp` no-op case

### Change 3 — `Makefile` `provision` target: remove `k3s-gcp` case (AWS-only)

**Exact old block (lines 107–118):**

```makefile
provision:
	@case "$(CLUSTER_PROVIDER)" in \
	  k3s-gcp) echo "[make] CLUSTER_PROVIDER=k3s-gcp — ensuring cluster is up..."; \
	           $(MAKE) --no-print-directory up CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" URL="$(URL)"; \
	           echo "[make] CLUSTER_PROVIDER=k3s-gcp — running gcp_provision_stack..."; \
	           CLUSTER_PROVIDER="$(CLUSTER_PROVIDER)" \
	           GHCR_PAT="$(GHCR_PAT)" \
	           scripts/k3d-manager gcp_provision_stack ;; \
	  *)       $(MAKE) --no-print-directory ssm; \
	           echo "[make] CLUSTER_PROVIDER=$(CLUSTER_PROVIDER) — running acg_provision..."; \
	           K3S_AWS_SSM_ENABLED=true scripts/k3d-manager acg_provision --confirm ;; \
	esac
```

**Exact new block:**

```makefile
provision:
	@$(MAKE) --no-print-directory ssm
	@echo "[make] CLUSTER_PROVIDER=$(CLUSTER_PROVIDER) — running acg_provision..."
	@K3S_AWS_SSM_ENABLED=true scripts/k3d-manager acg_provision --confirm
```

---

### Change 4 — `Makefile` `help` target: update GCP workflow description

Find the two GCP-specific help lines and replace with the unified workflow:

**Old lines (inside the help block):**

```makefile
	@echo "    make provision             Provision ACG stack via SSM (depends on ssm)"
	@echo "    make provision CLUSTER_PROVIDER=k3s-gcp GHCR_PAT=<pat>   Deploy full stack on GCP"
```

**New lines:**

```makefile
	@echo "    make provision             Provision ACG stack via SSM (AWS only)"
	@echo ""
	@echo "  GCP workflow (same commands as AWS):"
	@echo "    make up CLUSTER_PROVIDER=k3s-gcp GHCR_PAT=<pat>          Provision cluster + full stack"
	@echo "    make sync-apps CLUSTER_PROVIDER=k3s-gcp                   No-op (stack deployed by make up)"
```
