## Problem

`make provision CLUSTER_PROVIDER=k3s-gcp` fails at shell exit with:

```
/Users/cliang/src/gitrepo/personal/k3d-manager/scripts/plugins/argocd.sh: line 280: rendered: unbound variable
/Users/cliang/src/gitrepo/personal/k3d-manager/scripts/plugins/argocd.sh: line 1: rendered: unbound variable
make: *** [provision] Error 1
```

### Root Cause

`_argocd_deploy_appproject` (argocd.sh:811) registers an EXIT trap referencing `$rendered`
using single quotes — so `$rendered` is expanded lazily when the trap fires, not at
registration time:

```bash
trap '$(_cleanup_trap_command "$rendered")' EXIT   # line 811
```

`rendered` is a `local` variable. After the function returns normally, the EXIT trap remains
registered on the shell. When the shell later exits, bash expands `$rendered` — but the local
is out of scope → `unbound variable` under `set -u`.

The two error lines in the output are:
- `line 280` — gcp.sh's `source argocd.sh` call (bash attributes it to argocd.sh context)
- `line 1` — bash evaluating the trap string as a mini-script, starting at line 1

The RETURN trap on line 814 was intended to clean up on normal return, but it:
1. Does not clear the EXIT trap
2. Is set AFTER `_kubectl apply`, so if apply fails under `set -e`, the shell exits
   before line 814 is reached — the RETURN trap is never registered

The EXIT trap on line 811 IS correctly positioned for the error path (shell exits while
`rendered` is still in scope → cleanup runs). The bug is only the success path: the EXIT
trap is not cleared before the function returns.

## Fix

**File:** `scripts/plugins/argocd.sh`
**Function:** `_argocd_deploy_appproject` (lines 799–818)

### Exact change

**Old (lines 813–817):**
```bash
   _kubectl apply -f "$rendered" >/dev/null
   trap '$(_cleanup_trap_command "$rendered")' RETURN

   _info "[argocd] AppProject deployed: platform"
   return 0
```

**New (lines 813–817):**
```bash
   _kubectl apply -f "$rendered" >/dev/null
   rm -f "$rendered"
   trap - EXIT

   _info "[argocd] AppProject deployed: platform"
   return 0
```

### Why this is correct

- **Error path (set -e triggers during apply):** EXIT trap fires while `rendered` is still
  in scope (function hasn't returned) → cleanup runs → shell exits. No change to this path.
- **Success path:** `rm -f "$rendered"` cleans up the temp file explicitly. `trap - EXIT`
  clears the EXIT trap so it cannot fire with an unbound local after the function returns.
- The RETURN trap (line 814) is removed — it is replaced by the explicit `rm -f` and is
  no longer needed.
