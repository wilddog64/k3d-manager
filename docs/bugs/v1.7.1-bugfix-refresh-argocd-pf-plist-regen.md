# Bugfix: v1.7.1 — `make refresh` cannot regenerate a missing argocd-port-forward plist

**Branch:** `k3d-manager-v1.7.1`
**Files:** `bin/cluster-refresh`

---

## Problem

After the v1.7.1 self-heal + bootout fixes landed, `make refresh` now re-issues the
argocd-browser-https TLS material and bootstraps the socat daemon cleanly (no more error 5).
But ArgoCD is still unreachable on the proper path (`https://argocd.shopping-cart.local` → `000`)
because the socat listener crash-loops on its **upstream**:

```
socat E TCP:127.0.0.1:8080: Connection refused
```

The socat daemon TLS-terminates on `:443` and forwards plaintext to `127.0.0.1:8080`, which is the
**argocd-server port-forward** (launchd user agent `com.k3d-manager.argocd-port-forward`). If that
agent's plist is missing from `~/Library/LaunchAgents/`, nothing serves `:8080`, so every socat
connection is refused and the listener never becomes healthy.

`make refresh` cannot recover this. At `bin/cluster-refresh:331-334` it calls:

```bash
  _launchd_ensure \
    "com.k3d-manager.argocd-port-forward" \
    "${HOME}/Library/LaunchAgents/com.k3d-manager.argocd-port-forward.plist" \
    user
```

`_launchd_ensure` bootstraps the plist if `launchctl list` fails — but when the **plist file itself
is missing**, there is nothing to bootstrap and the agent never loads. Unlike the sibling
keycloak (`bin/cluster-refresh:336-375`) and grafana (`377-423`) blocks — which **regenerate the
plist from the wrapper if it is missing** before calling `_launchd_ensure` — the argocd-port-forward
block has **no regenerate-from-wrapper logic**. The wrapper itself is present
(`${_ACG_STATE_DIR}/bin/argocd-port-forward.sh`, written by `bin/cluster-up:424`); only the plist
is gone, and refresh can never rebuild it.

**Root cause:** the argocd-port-forward block in `bin/cluster-refresh` calls `_launchd_ensure` on a
plist it never regenerates, so a missing `com.k3d-manager.argocd-port-forward.plist` leaves `:8080`
dead and the browser-https listener forwards to a refused upstream — defeating the refresh
self-heal that the rest of v1.7.1 restored.

---

## Reproduction

```bash
# Remove the argocd-port-forward agent (simulates a refresh-only host that never ran the
# cluster-up step that creates the plist, or a stale-state cleanup):
launchctl bootout "gui/$(id -u)/com.k3d-manager.argocd-port-forward" 2>/dev/null || true
rm -f ~/Library/LaunchAgents/com.k3d-manager.argocd-port-forward.plist

make refresh
#   browser-https daemon bootstraps, but:
tail ~/.local/share/k3d-manager/logs/argocd-browser-https.log
#   -> socat E TCP:127.0.0.1:8080: Connection refused (crash loop)
curl -s -o /dev/null -w "%{http_code}\n" https://argocd.shopping-cart.local/healthz   # -> 000
lsof -nP -iTCP:8080 -sTCP:LISTEN   # -> (nothing)
```

Proven root cause: with the socat daemon already running, manually starting an `:8080`
port-forward (`kubectl port-forward svc/argocd-server 8080:80 -n cicd --context k3d-k3d-cluster`)
immediately makes `https://argocd.shopping-cart.local/healthz` return `200`.

Expected after fix: `make refresh` regenerates the missing argocd-port-forward plist from the
existing wrapper, the agent loads, `:8080` is served, and the browser-https listener becomes
healthy (`curl -sk https://argocd.shopping-cart.local/healthz` → `200`).

---

## Fix

Mirror the keycloak block: before `_launchd_ensure` for argocd-port-forward, regenerate the plist
from the existing wrapper if the plist is missing. The wrapper path and plist structure match
`bin/cluster-up:418-447` (`ProgramArguments` = `/bin/bash <wrapper>`, `RunAtLoad` + `KeepAlive`).

### Change 1 — `bin/cluster-refresh`: regenerate argocd-port-forward plist from wrapper if missing

**Exact old block (lines 331–334):**

```bash
  _launchd_ensure \
    "com.k3d-manager.argocd-port-forward" \
    "${HOME}/Library/LaunchAgents/com.k3d-manager.argocd-port-forward.plist" \
    user
```

**Exact new block:**

```bash
  _argocd_pf_label="com.k3d-manager.argocd-port-forward"
  _argocd_pf_plist="${HOME}/Library/LaunchAgents/${_argocd_pf_label}.plist"
  _argocd_pf_log="${_ACG_STATE_DIR}/logs/argocd-pf.log"
  _argocd_pf_wrapper="${_ACG_STATE_DIR}/bin/argocd-port-forward.sh"
  if [[ ! -f "${_argocd_pf_plist}" && -f "${_argocd_pf_wrapper}" ]]; then
    _info "[acg-refresh] ArgoCD port-forward plist missing — regenerating from wrapper..."
    mkdir -p "$(dirname "${_argocd_pf_log}")"
    cat > "${_argocd_pf_plist}" <<PLIST
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>${_argocd_pf_label}</string>
  <key>ProgramArguments</key>
  <array>
    <string>/bin/bash</string>
    <string>${_argocd_pf_wrapper}</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>${_argocd_pf_log}</string>
  <key>StandardErrorPath</key>
  <string>${_argocd_pf_log}</string>
</dict>
</plist>
PLIST
  fi
  _launchd_ensure \
    "${_argocd_pf_label}" \
    "${_argocd_pf_plist}" \
    user
```
