# Bugfix: v1.4.1 — acg-up proceeds with empty PAT after interactive read

**Branch:** `k3d-manager-v1.4.1`
**Files:** `bin/acg-up`

---

## Problem

`make up` fails mid-bootstrap with:

```
error: either --from-file or the combination of --docker-username,
       --docker-password and --docker-server is required
error: no objects passed to apply
make: *** [up] Error 1
```

This leaves `shopping-cart-payment` and `shopping-cart-data` namespaces uncreated,
causing a chicken-and-egg: subsequent `bin/rotate-ghcr-pat` runs can only reach
`shopping-cart-apps` (the one namespace that was created before step 5).

**Root cause:** when the interactive `read -r -s -p` prompt is shown but the user
presses Enter without pasting a token (e.g. paste is blocked in a subprocess terminal),
`_ghcr_pat` stays empty and the script falls through to the namespace/secret loop at
line 246 — `kubectl create secret docker-registry` then fails because
`--docker-password=""` is invalid.

---

## Reproduction

```bash
# Run make up in a terminal where clipboard paste is unavailable
make up URL=<sandbox-url>
# At the "Paste GitHub PAT" prompt, press Enter without typing
# Expected: clear error message, immediate exit
# Actual: proceeds to namespace loop, fails with kubectl error, leaves partial state
```

---

## Fix

### Change 1 — `bin/acg-up`: guard empty PAT before entering the namespace loop

Add an explicit `_err` check immediately after the PAT resolution block and before
the `for ns in ...` loop. This fires in the TTY case when `read` returns an empty
string, giving a clear message and preventing the partial-bootstrap state.

**Exact old block (lines 231–246):**

```bash
  if [[ -z "$_ghcr_pat" ]]; then
    if [[ -t 0 && -t 1 ]]; then
      read -r -s -p "[acg-up] Paste GitHub PAT (repo + read:packages) and press Enter: " _ghcr_pat
      echo ""
      if [[ -n "${_vault_root_token:-}" && -n "$_ghcr_pat" ]]; then
        curl -s -X POST -H "X-Vault-Token: ${_vault_root_token}" \
          -d "{\"data\": {\"token\": \"${_ghcr_pat}\"}}" \
          "http://localhost:${_vault_local_port}/v1/secret/data/github/pat" >/dev/null || true
        _info "[acg-up] new PAT saved to Vault"
      fi
    else
      _err "[acg-up] GHCR_PAT not set and no valid PAT in Vault — set GHCR_PAT env var or run: pbpaste | bin/rotate-ghcr-pat"
    fi
  fi
fi
for ns in shopping-cart-apps shopping-cart-payment shopping-cart-data; do
```

**Exact new block:**

```bash
  if [[ -z "$_ghcr_pat" ]]; then
    if [[ -t 0 && -t 1 ]]; then
      read -r -s -p "[acg-up] Paste GitHub PAT (repo + read:packages) and press Enter: " _ghcr_pat
      echo ""
      if [[ -n "${_vault_root_token:-}" && -n "$_ghcr_pat" ]]; then
        curl -s -X POST -H "X-Vault-Token: ${_vault_root_token}" \
          -d "{\"data\": {\"token\": \"${_ghcr_pat}\"}}" \
          "http://localhost:${_vault_local_port}/v1/secret/data/github/pat" >/dev/null || true
        _info "[acg-up] new PAT saved to Vault"
      fi
    else
      _err "[acg-up] GHCR_PAT not set and no valid PAT in Vault — set GHCR_PAT env var or run: pbpaste | bin/rotate-ghcr-pat"
    fi
  fi
fi
if [[ -z "$_ghcr_pat" ]]; then
  _err "[acg-up] GitHub PAT is empty — cannot create ghcr-pull-secret. Re-run with: GHCR_PAT=\$(pbpaste) make up"
fi
for ns in shopping-cart-apps shopping-cart-payment shopping-cart-data; do
```
