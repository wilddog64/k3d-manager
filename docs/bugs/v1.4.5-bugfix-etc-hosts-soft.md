# Bugfix: v1.4.5 ‚Äî /etc/hosts _run_command missing --soft exits script

**Branch:** `k3d-manager-v1.4.5`
**Files:** `bin/acg-up`

---

## Problem

`make up` aborts with `Error 1` immediately after `secret/argocd-secret patched`, before the `argocd-server` rollout restart. No `/etc/hosts` warning is printed, and the `else` branch is never reached.

**Root cause:** `_run_command --prefer-sudo` without `--soft` calls `_run_command_handle_failure` ‚Üí `_err` ‚Üí `exit 1` when `sh -c '... >> /etc/hosts'` fails with EPERM. An `exit 1` inside `_err` terminates the script unconditionally ‚Äî even when the call is wrapped in an `if` condition. The `2>/dev/null` redirection on the call site suppresses stderr prints but cannot intercept an `exit 1`.

---

## Reproduction

```bash
# Ensure keycloak.shopping-cart.local is NOT in /etc/hosts (so the block runs)
grep 'keycloak.shopping-cart.local' /etc/hosts   # should return nothing

# Run make up (or the Step 10e section of acg-up directly)
make up
# Expected: WARNING printed, make up continues
# Actual:   make: *** [up] Error 1 ‚Äî script dies after "secret/argocd-secret patched"
```

---

## Fix

### Change 1 ‚Äî `bin/acg-up` lines 639‚Äì640: add `--soft` to `_run_command`

With `--soft`, `_run_command_handle_failure` calls `return "$rc"` instead of `_err` when the command fails. The non-zero return propagates to the `if` condition, which then executes the `else` branch and prints the warning.

**Exact old block (lines 639‚Äì640):**

```bash
  if _run_command --prefer-sudo -- \
      sh -c 'printf "127.0.0.1 keycloak.shopping-cart.local\n" >> /etc/hosts' 2>/dev/null; then
```

**Exact new block:**

```bash
  if _run_command --prefer-sudo --soft -- \
      sh -c 'printf "127.0.0.1 keycloak.shopping-cart.local\n" >> /etc/hosts' 2>/dev/null; then
```
