# Bugfix: v1.0.4 — Replace hardcoded sandbox passwords in `acg-up`

**Branch:** `k3d-manager-v1.0.4`
**Files:** `bin/acg-up`
**Tracking:** wilddog64/shopping-cart-infra#29

---

## Problem

`bin/acg-up` seeds Vault KV with hardcoded static passwords every provision run:

```bash
_vault_kv_put '{"username":"postgres","password":"changeme123"}'   postgres/products
_vault_kv_put '{"username":"postgres","password":"changeme456"}'   postgres/orders
_vault_kv_put '{"username":"postgres","password":"CHANGE_ME"}'     postgres/payment
_vault_kv_put '{"password":"redis-cart-dev-pass"}'                 redis/cart
_vault_kv_put '{"password":"redis-orders-cache-dev-pass"}'        redis/orders-cache
_vault_kv_put '{"username":"rabbitmq","password":"rabbitmq-sandbox-pass"}' rabbitmq/default
```

**Root cause:** passwords are literals in source — identical across every sandbox run, visible in git history, never rotated.

**Exception:** `payment/encryption` key is a fixed AES-256 key (`dmF1bHQtZGV2LXNhbmRib3gtZW5jcnlwdGlvbg==`). AES keys must be stable across restarts to decrypt stored data — do NOT randomize.

---

## Fix

### Change 1 — `bin/acg-up`: generate passwords at provision time

Generate each password with `openssl rand -base64 24 | tr -d '=+/'` (URL-safe, 32 chars, no shell metacharacters).

**Exact old block (lines 261–269):**

```bash
_vault_kv_put '{"password":"redis-cart-dev-pass"}'                                              redis/cart
_vault_kv_put '{"password":"redis-orders-cache-dev-pass"}'                                      redis/orders-cache
_vault_kv_put '{"username":"postgres","password":"changeme456"}'                                 postgres/orders
_vault_kv_put '{"username":"postgres","password":"changeme123"}'                                 postgres/products
_vault_kv_put '{"username":"postgres","password":"CHANGE_ME"}'                                   postgres/payment
_vault_kv_put '{"key":"dmF1bHQtZGV2LXNhbmRib3gtZW5jcnlwdGlvbg=="}'                             payment/encryption
_vault_kv_put '{"api_key":"sk_test_placeholder","webhook_secret":"whsec_placeholder"}'           payment/stripe
_vault_kv_put '{"client_id":"paypal_sandbox_client_id","client_secret":"paypal_sandbox_client_secret"}' payment/paypal
_vault_kv_put '{"username":"rabbitmq","password":"rabbitmq-sandbox-pass"}'                         rabbitmq/default
```

**Exact new block:**

```bash
_pg_pass_orders=$(openssl rand -base64 24 | tr -d '=+/')
_pg_pass_products=$(openssl rand -base64 24 | tr -d '=+/')
_pg_pass_payment=$(openssl rand -base64 24 | tr -d '=+/')
_redis_pass_cart=$(openssl rand -base64 24 | tr -d '=+/')
_redis_pass_orders=$(openssl rand -base64 24 | tr -d '=+/')
_rabbitmq_pass=$(openssl rand -base64 24 | tr -d '=+/')

_vault_kv_put "{\"password\":\"${_redis_pass_cart}\"}"                                           redis/cart
_vault_kv_put "{\"password\":\"${_redis_pass_orders}\"}"                                         redis/orders-cache
_vault_kv_put "{\"username\":\"postgres\",\"password\":\"${_pg_pass_orders}\"}"                  postgres/orders
_vault_kv_put "{\"username\":\"postgres\",\"password\":\"${_pg_pass_products}\"}"                postgres/products
_vault_kv_put "{\"username\":\"postgres\",\"password\":\"${_pg_pass_payment}\"}"                 postgres/payment
_vault_kv_put '{"key":"dmF1bHQtZGV2LXNhbmRib3gtZW5jcnlwdGlvbg=="}'                             payment/encryption
_vault_kv_put '{"api_key":"sk_test_placeholder","webhook_secret":"whsec_placeholder"}'           payment/stripe
_vault_kv_put '{"client_id":"paypal_sandbox_client_id","client_secret":"paypal_sandbox_client_secret"}' payment/paypal
_vault_kv_put "{\"username\":\"rabbitmq\",\"password\":\"${_rabbitmq_pass}\"}"                   rabbitmq/default
```

**Note:** The `payment/encryption` key, `payment/stripe`, and `payment/paypal` blocks are unchanged — AES key must be stable; Stripe/PayPal are test placeholders that require manual configuration.
