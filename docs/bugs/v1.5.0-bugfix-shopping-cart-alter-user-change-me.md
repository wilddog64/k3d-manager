# Bug: `deploy_shopping_cart_data` corrupts PostgreSQL password with hardcoded CHANGE_ME

**Branch:** `k3d-manager-v1.5.0`
**File:** `scripts/plugins/shopping_cart.sh:109–113`
**Severity:** P1 — `order-service` and `product-catalog` enter CrashLoopBackOff on every fresh `make up`

---

## Symptom

After `make up`, `order-service` and `product-catalog` crash repeatedly:

```
order-service-86f5f685bf-czggw     0/1     CrashLoopBackOff   8 (34s ago)   18m
product-catalog-85c7744c77-ghsxf   0/1     CrashLoopBackOff   8 (79s ago)   17m
```

Logs show:

```
FATAL: password authentication failed for user "postgres"
```

---

## Root Cause

`deploy_shopping_cart_data()` runs `ALTER USER postgres WITH PASSWORD 'CHANGE_ME'` on
`postgresql-orders-0` and `postgresql-products-0` **after** PostgreSQL finishes initializing
from the correct Vault-seeded password.

PostgreSQL is initialized via the StatefulSet's `POSTGRES_PASSWORD` env var, which is
sourced from the `postgres-orders-admin` / `postgres-products-admin` k8s secrets (ESO-synced
from Vault). ESO also syncs the same Vault values into `order-service-secrets` and
`product-catalog-secrets` for use by the app pods.

The `ALTER USER ... 'CHANGE_ME'` step then **overwrites** the correct PostgreSQL password with
a hardcoded literal that no ESO-synced secret contains. Every subsequent app connection fails
authentication.

```bash
# scripts/plugins/shopping_cart.sh:109-113 — WRONG, overwrites Vault password
_info "[shopping_cart] Aligning PostgreSQL passwords to match app secrets (CHANGE_ME)..."
for pg_pod in postgresql-orders-0 postgresql-products-0; do
  kubectl exec "${pg_pod}" -n shopping-cart-data --context ubuntu-k3s -- \
    psql -U postgres -c "ALTER USER postgres WITH PASSWORD 'CHANGE_ME';" 2>/dev/null || true
done
```

---

## Fix

**`scripts/plugins/shopping_cart.sh` — delete lines 109–113 entirely:**

```bash
# DELETE these 5 lines:
  _info "[shopping_cart] Aligning PostgreSQL passwords to match app secrets (CHANGE_ME)..."
  for pg_pod in postgresql-orders-0 postgresql-products-0; do
    kubectl exec "${pg_pod}" -n shopping-cart-data --context ubuntu-k3s -- \
      psql -U postgres -c "ALTER USER postgres WITH PASSWORD 'CHANGE_ME';" 2>/dev/null || true
  done
```

After deletion, line 115 (`_info "[shopping_cart] Creating rabbitmq-credentials secret..."`)
follows directly after the `done` of the PostgreSQL rollout loop at line 107.

The resulting block (lines 103–120 after edit) should look like:

```bash
  _info "[shopping_cart] Waiting for PostgreSQL instances to be Ready..."
  for pg in postgresql-orders postgresql-payment postgresql-products; do
    kubectl rollout status statefulset/"${pg}" \
      -n shopping-cart-data --context ubuntu-k3s --timeout=120s
  done

  _info "[shopping_cart] Creating rabbitmq-credentials secret..."
  kubectl create secret generic rabbitmq-credentials \
    --context ubuntu-k3s -n shopping-cart-data \
    --from-literal=username=guest \
    --from-literal=password=CHANGE_ME \
    --dry-run=client -o yaml | kubectl apply --context ubuntu-k3s -f -
```
