## Problem

Two bugs in `acg_check_ttl` (lines 448–461):

**Bug 1 — missing `node` doesn't fail fast (line 451–452):**
`_err "[acg] node is required"` logs the error but does NOT return non-zero, so the
function continues and fails later at `node "$playwright_script" ...` with a less
informative error. Callers cannot distinguish "node missing" from "TTL parse failed".

**Bug 2 — empty string instead of -1 sentinel (line 460):**
The help text promises "-1 if unparseable", but the grep/cut pipeline returns an empty
string (exit 0) when the `REMAINING_MINS:` marker is absent. Callers doing arithmetic
on the output will get unexpected results.

---

## Fix

**Exact old block (lines 448–461):**

```bash
  local sandbox_url="${1:?usage: acg_check_ttl <sandbox_url>}"
  local playwright_script="${_LIB_ACG_ROOT}/playwright/acg_extend.js"
  if ! command -v node >/dev/null 2>&1; then
    _err "[acg] node is required — install Node.js"
  fi
  local output exit_code
  output=$(node "$playwright_script" "$sandbox_url" --check 2>/dev/null)
  exit_code=$?
  if [[ $exit_code -ne 0 ]]; then
    _warn "[acg] acg_check_ttl: node exited $exit_code"
    return 1
  fi
  printf '%s\n' "$output" | grep '^REMAINING_MINS:' | cut -d: -f2
}
```

**Exact new block:**

```bash
  local sandbox_url="${1:?usage: acg_check_ttl <sandbox_url>}"
  local playwright_script="${_LIB_ACG_ROOT}/playwright/acg_extend.js"
  if ! command -v node >/dev/null 2>&1; then
    _err "[acg] node is required — install Node.js"
    return 1
  fi
  local output exit_code
  output=$(node "$playwright_script" "$sandbox_url" --check 2>/dev/null)
  exit_code=$?
  if [[ $exit_code -ne 0 ]]; then
    _warn "[acg] acg_check_ttl: node exited $exit_code"
    return 1
  fi
  local remaining
  remaining=$(printf '%s\n' "$output" | grep '^REMAINING_MINS:' | cut -d: -f2)
  printf '%s\n' "${remaining:--1}"
}
```
