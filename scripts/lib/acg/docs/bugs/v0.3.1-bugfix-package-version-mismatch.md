## Problem

`package.json` declares `"version": "0.2.0"` and `package-lock.json` has
`"version": "0.1.0"` in both the root field and the `packages[""]` block.
The actual release is v0.3.0 (shipped, tagged). This mismatch makes it harder to
reproduce issues and verify which version of lib-acg is vendored downstream.

---

## Fix

### Change 1 — `package.json`

**Exact old line:**
```json
  "version": "0.2.0",
```

**Exact new line:**
```json
  "version": "0.3.0",
```

### Change 2 — `package-lock.json` (two occurrences)

**Exact old block (lines 2–3):**
```json
  "name": "lib-acg",
  "version": "0.1.0",
```

**Exact new block:**
```json
  "name": "lib-acg",
  "version": "0.3.0",
```

**Exact old block (lines 8–9 inside `packages[""]`):**
```json
      "name": "lib-acg",
      "version": "0.1.0",
```

**Exact new block:**
```json
      "name": "lib-acg",
      "version": "0.3.0",
```
