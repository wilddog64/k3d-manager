# Bugfix: v0.2.2 — acg-cluster.yaml removed from lib-acg but reference not updated

**Branch:** `fix/acg-cluster-yaml-path`
**Files:** `scripts/plugins/acg.sh`, `scripts/etc/acg-cluster.yaml`

---

## Problem

`scripts/etc/acg-cluster.yaml` was removed from lib-acg in the v0.2.0 fix
(`fix(acg-extend): provider-scoped button selection; remove misplaced CFn template`)
because it "belongs in k3d-manager." However, the reference in `scripts/plugins/acg.sh`
was not updated — it still looks for the file at `_LIB_ACG_ROOT/scripts/etc/acg-cluster.yaml`.

When lib-acg is embedded in k3d-manager as a git subtree, `_LIB_ACG_ROOT` resolves to
`k3d-manager/scripts/lib/acg/`, so the resolved path is
`k3d-manager/scripts/lib/acg/scripts/etc/acg-cluster.yaml` — which does not exist.
`make up` fails with `Invalid template path ...acg-cluster.yaml`.

**Root cause:** The file was deleted without updating the reference. The template must
either be restored to lib-acg at its original path, OR the code must be updated to find
it elsewhere.

**Decision:** Restore the template to lib-acg at `scripts/etc/acg-cluster.yaml`.
It was mis-categorized as "k3d-manager only" — it is part of the ACG provisioning flow
and belongs in lib-acg alongside the code that uses it.

---

## Reproduction

```bash
cd k3d-manager && make up CLUSTER_PROVIDER=k3s-aws
# → Invalid template path .../scripts/lib/acg/scripts/etc/acg-cluster.yaml
# → aws command failed (255)
```

## Fix

### Change 1 — Restore `scripts/etc/acg-cluster.yaml` in lib-acg

Create `scripts/etc/acg-cluster.yaml` with the CloudFormation template content
(copy from k3d-manager's `scripts/etc/acg-cluster.yaml` — they are identical).

The template defines a 3-node k3s cluster (server + 2 agents) on EC2.
