## Problem

`$_refs` is expanded unquoted in `printf '  %s\n' $_refs`, so filenames containing
spaces or newlines are word-split and the output becomes incorrect. This can also
cause `printf` format-string surprises if a filename begins with `%`.

**Root cause:** Line 49 — `printf '  %s\n' $_refs` — missing quotes around `$_refs`.

---

## Fix

Replace the unquoted expansion with a safe `read` loop that prints one filename per line.

**Exact old block (lines 44–51):**

```bash
while IFS= read -r -d '' _deleted; do
  _base="$(basename "$_deleted")"
  _refs="$(git grep --cached -l "$_base" -- '*.sh' '*.js' 2>/dev/null || true)"
  if [[ -n "$_refs" ]]; then
    printf 'Pre-commit: deleting %s but it is still referenced in:\n' "$_deleted" >&2
    printf '  %s\n' $_refs >&2
    exit 1
  fi
done < <(git diff --cached --name-only -z --diff-filter=D)
```

**Exact new block:**

```bash
while IFS= read -r -d '' _deleted; do
  _base="$(basename "$_deleted")"
  _refs="$(git grep --cached -l "$_base" -- '*.sh' '*.js' 2>/dev/null || true)"
  if [[ -n "$_refs" ]]; then
    printf 'Pre-commit: deleting %s but it is still referenced in:\n' "$_deleted" >&2
    while IFS= read -r _ref; do
      printf '  %s\n' "$_ref" >&2
    done <<< "$_refs"
    exit 1
  fi
done < <(git diff --cached --name-only -z --diff-filter=D)
```
