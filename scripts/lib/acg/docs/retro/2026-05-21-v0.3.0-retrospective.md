# Retrospective — v0.3.0

**Date:** 2026-05-21
**Milestone:** Make CloudFormation template path configurable
**PR:** #22 — merged to main (`3eafb3db`)
**Participants:** Claude, Copilot

## What Went Well
- One-line core fix (`ACG_CLUSTER_TEMPLATE` env var with fallback) was clean and correct on first pass
- Copilot caught three real issues: missing preflight check, wrong README link, wrong vars.sh path in spec
- All three Copilot findings were substantive — no false positives
- Phase 1/3 Haiku subagent handled PR creation, CI monitoring, and thread resolution without issues

## What Went Wrong
- Spec doc referenced `scripts/lib/vars.sh` which doesn't exist in lib-acg — spec was written from memory without verifying the path
- README Issue Logs had a wrong link (Copilot PR #18 doc instead of upstream-push incident doc) — Haiku subagent inferred filename from adjacent entry instead of looking up the actual file
- No preflight check on the new env var — a caller with a typo in `ACG_CLUSTER_TEMPLATE` would get an opaque AWS CLI error

## Process Rules Added
- Any configurable file path accepted from an env var must be validated with `-f` before use, with an error message naming the variable
- When populating README Issue Logs, always match the link target to the actual filename — do not infer from adjacent entries
- Before naming a file path in a spec doc, verify it exists with `find` or `ls`

## Decisions Made
- `acg-cluster.yaml` stays in lib-acg as the bundled default fallback; callers override via `ACG_CLUSTER_TEMPLATE`
- k3d-manager will set `ACG_CLUSTER_TEMPLATE` in its env to point at its own copy (follow-on, separate PR)
- The spec doc (`docs/bugs/`) is the right place for this kind of enhancement spec, not `docs/plans/`

## Theme
v0.3.0 was a surgical one-line fix with a good catch rate from Copilot. The core change decouples the CloudFormation template from lib-acg, letting k3d-manager own its own template while lib-acg keeps a working default. Three Copilot findings — all real — improved the robustness of the preflight check and the accuracy of the docs. The milestone confirmed the Phase 1/3 Haiku subagent pattern works well for mechanical PR work, but Haiku still makes documentation mistakes (wrong file path inference, wrong link) that require a Copilot or human catch.
